# コーディングスタイル

## TypeScript規約

- 厳格モード（`strict: true`）を使用
- `any` の使用を避け、適切な型を定義する
- インターフェースよりも型エイリアスを優先（特に理由がなければ）
- 関数の戻り値の型を明示する（公開API）

## ESLint

- ESLintでコード品質を自動チェック
- 正確なルール設定は `planner` が策定

## Prettier

- Prettierでコードフォーマットを統一
- 正確な設定は `planner` が策定
- エディタとCIの両方でフォーマットチェックを実行

## 一般原則

- 小さく焦点の合った関数/コンポーネントを書く
- マジックナンバーを避け、名前付き定数を使用する
- コメントは「なぜ」を説明する（「何を」はコードが説明する）
- 副作用を最小限に保つ
