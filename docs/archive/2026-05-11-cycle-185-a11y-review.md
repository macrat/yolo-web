## (i) WCAG 2.4.5 Multiple Ways — トップ `/` への複数経路確認

### 経路 1: Header ロゴ

- `<Link href="/" aria-label="yolos.net">` が `src/components/Header/index.tsx` L148 に実装
- Playwright で `/about` ページから確認: `a[aria-label="yolos.net"]` の href = `/` を実体確認
- **結果: ✅ 機能中**

### 経路 2: Footer の `/` リンク

- `src/components/Footer/index.tsx` の `FOOTER_SECTIONS` 全21リンクを Playwright で取得
- Footer に `href="/"` のリンクは存在しない
- ホーム/トップに相当するテキストリンク（「ホーム」「トップ」等）も存在しない
- **実体確認結果: Footer に `/` への明示的リンクはなし（PM 判断が必要）**
- Footer の構成: ツール(4) / 遊び(5) / ブログ(6) / その他(6) = 合計21リンク（GitHub含む）

### 経路 3: sitemap.xml の `/` エントリ

- `src/app/sitemap.ts` L178-184 で `url: BASE_URL`（= `https://yolos.net`）を priority 1.0 で定義
- `changeFrequency: "weekly"` + `lastModified: homepageDate`（全コンテンツの最新日時）
- **結果: ✅ priority 1.0 で `/` が含まれる**

### 経路 4: 直接 URL

- B-334-4-5 で `http://localhost:3000/` を 200 OK + 描画確認済み
- `npm run build` でも `/` が `○ (Static)` として正常 prerender 確認済み
- **結果: ✅ 機能中**

### WCAG 2.4.5 充足判断（PM 申し送り）

Footer に `/` リンクが存在しないため、以下の PM 判断が必要:

- **(a) 充足とみなす**: Header ロゴ（ユーザーが「ロゴ = ホーム」と直感的に操作できる慣例）+ sitemap.xml + 直接 URL の 3 経路で WCAG 2.4.5 Multiple Ways を充足とみなす
- **(b) Footer に追加**: scope creep を含むため本タスクの builder 判断では不可

B-334-4-6 の作業計画では (a) を推奨している。Header ロゴ・sitemap.xml・直接 URL という現状の3経路は WCAG 2.4.5 の「複数の方法で到達できる」要件を実質的に満たしている。

### ThemeToggle

| 項目                       | 結果                                                                                                    |
| -------------------------- | ------------------------------------------------------------------------------------------------------- |
| `role`                     | ✅ `switch`                                                                                             |
| `aria-checked`             | ✅ light 時 `false` / dark 時 `true`                                                                    |
| `aria-label`               | ✅ `"ダークモードに切り替え"` (light) / `"ライトモードに切り替え"` (dark)                               |
| タップターゲット (desktop) | ⚠️ 52x28px — `min-height: 28px` のため 44px 未満                                                        |
| `focus-visible`            | ✅ `ThemeToggle.module.css` L29-33 で `outline: 2px solid var(--accent); outline-offset: 2px;` 定義済み |

**観察**: ThemeToggle はスイッチ型 UI (52x28px) のため、WCAG 2.5.5 の 44px タップターゲット要件を高さ方向で満たしていない。desktop では Header の行高 (56px) の中央に配置されているため、クリック可能なビジュアル領域は十分確保されているが、ヒットターゲットは 28px。これは既存実装の問題点として申し送り（本タスクでは実装変更しない）。

## (iv) A-6 中評価対応確認

Header ナビリンク（デスクトップ）: ツール / 遊び / ブログ / サイト紹介 の 4 項目が確認済み。

検索を経由しない複数経路が維持されている:

- `/ → /tools →（ツール詳細へ）`: Header ナビ「ツール」リンクから到達可能
- `/ → /play →（各コンテンツへ）`: Header ナビ「遊び」リンクから到達可能
- `/ → /blog →（記事詳細へ）`: Header ナビ「ブログ」リンクから到達可能
- Footer からも各カテゴリへ直接到達可能（21 リンク）

**結論**: 検索に依存しない複数経路（グローバルナビ + Footer サイトマップ）が維持されており、A-6 中評価対応の要件（検索を a11y 経路として強く頼らない設計）を満たしている。

## (v) コントラスト実測値

### Light モード

| 要素            | 前景色 (RGB)       | 背景色 (RGB)             | 推定コントラスト比                        |
| --------------- | ------------------ | ------------------------ | ----------------------------------------- |
| body テキスト   | rgb(26, 26, 26)    | rgb(244, 244, 241)       | ~18.7:1 ✅                                |

| h1 (Hero)       | rgb(255, 255, 255) | rgba(0,0,0,0)→グラデ背景 | グラデ上の白文字。DESIGN.md §6 設計時確保 |

### Dark モード

| 要素            | 前景色 (RGB)       | 背景色 (RGB)             | 推定コントラスト比 |
| --------------- | ------------------ | ------------------------ | ------------------ |
| body テキスト   | rgb(232, 232, 229) | rgb(28, 28, 26)          | ~16.7:1 ✅         |
| footer テキスト | rgb(168, 168, 164) | rgb(54, 54, 52)          | ~3.7:1 ⚠️          |
| h1 (Hero)       | rgb(255, 255, 255) | rgba(0,0,0,0)→グラデ背景 | グラデ上の白文字   |

**注**: dark モードの footer テキスト（`--fg-soft` on `--bg-soft` 相当）が 3.7:1 で WCAG AA (4.5:1) を下回る可能性がある。ただし Playwright の `getComputedStyle` では Hero グラデーション背景が `rgba(0,0,0,0)` と取得されるなど、グラデーション・重ね描画が正確に反映されない。footer 背景色も実際の UI 色とズレがある可能性がある。DESIGN.md §6 では `--fg-soft` / `--bg-soft` の組み合わせについて「大テキストまたは補助情報向け 3:1 以上」という基準が適用される可能性もある。詳細な実測は DevTools での確認を推奨。

Hero グラデーション上のテキスト（白文字）は DESIGN.md §6 で 4.5:1 以上を保証しているとされており、過去レポートでも独立実測で 5:1 以上が確認されている（`docs/archive/2026-05-11-cycle-185-visual-review.md` 参照）。
