# 日本語診断・心理テスト・占い系コンテンツのSEO戦略調査

## エグゼクティブサマリー

日本語の診断・心理テスト市場では、検索上位は「診断ツール本体」だけでなく「タイプ別解説記事」が大量の検索流入を獲得している。uranaino.net のように400記事以上のMBTIタイプ解説を持つサイトが複数のキーワードで安定上位に入っている。yolos.netが月間763PVから成長するためには、診断ツールと並行して各診断の「タイプ解説ページ」を整備することが最もコストパフォーマンスの高い施策である。

### 1-1. 「性格診断 無料」「心理テスト 無料」

グローバル最大手の16Personalities（https://www.16personalities.com/ja）が圧倒的な存在感を持つ。国内サイトでは以下が競合する。

| サイト                                               | 特徴                     | コンテンツ量            |
| ---------------------------------------------------- | ------------------------ | ----------------------- |
| ホイミー (hoyme.jp)                                  | 診断＋コラム記事の複合型 | 診断20本+コラム50本以上 |
| 暇つぶし診断テスト集 (himatsubushi-shindan-test.com) | 純粋な診断アグリゲーター | 200件以上の診断         |

### 1-2. 「MBTI 診断」「INFJ とは」

- uranaino.net（「今すぐ使える心理学」）: 16タイプ×複数視点で400記事超。各タイプの解説・有名人・適職・恋愛傾向をカバー。専用の診断ツール（16TEST）も運営。
- mbti-navi.com: WordPress+CocoonテーマベースでMBTI解説記事群を展開。JSON-LDによるBlogPosting・BreadcrumbList実装。
- 16Personalitiesの各タイプ詳細ページ（/infj-personality 等）: 日本語対応で圧倒的DA（ドメインオーソリティ）。

### 1-3. 「動物診断」「面白い診断」

- unkoi.com: 動物キャラ占いで60タイプ対応
- woman.mynavi.jp: 既存メディアの信頼性を活用した診断コンテンツ
- Makko（ma-kko.com）: ユーザー投稿型診断ポータル。ハッシュタグで「#動物」「#面白い」等のカテゴリページが検索上位に出現
- 診断ドットコム（4ndan.com）: 500本以上の診断。SEO対策としてdescriptionメタタグのカスタマイズ機能を整備

## 2. 上位サイトのSEO戦略パターン

### パターンA: 診断ツール + タイプ別解説記事（最有力）

代表例: uranaino.net、mbti-navi.com

- 診断ツール本体で「MBTI 診断 無料」等の競合ワードに参入
- 各タイプの解説ページ（「INFJとは」「ENTPの特徴」等）で月間数百〜数千検索のロングテールキーワードを大量に刈り取る
- 例：「INFJ 有名人」「INTJ 恋愛」「ENFP 適職」など16タイプ×複数軸で何百もの記事が成立
- 各記事からの内部リンクが診断ツール本体に集まり、診断ページの権威性も向上する

この戦略の強み：一般的なメディアサイトと直接競合しない独自のコンテンツ資産を積み上げられる。

### パターンB: 大量の診断コンテンツ集積

代表例: 暇つぶし診断テスト集（200件以上）、診断ドットコム（500件以上）

- 幅広いジャンルの診断を大量に揃え、様々な検索クエリをカバー
- 各診断タイトルがそのままキーワードになる（「SM診断」「精神年齢診断」等）
- 小規模サイトが採用するには初期コストが高い

### パターンC: 解説コラム記事との複合運営

代表例: ホイミー（hoyme.jp）

- 「エニアグラムとは」「血液型 性格 B型」等のKnowクエリに解説記事で対応
- 診断ツール本体と解説記事を相互リンクして回遊性を高める
- 夢占い・星座・血液型などの関連テーマで記事を横展開

## 3. 技術的なSEO施策の観察

### 構造化データ

- mbti-navi.com: JSON-LDでBlogPosting・BreadcrumbList・WebPageを実装。検索結果でリッチスニペット表示を狙っている
- uranaino.net: Google Analytics（UA形式）を実装。OGP画像も整備
- 暇つぶし診断テスト集: Schema.org（WebSite・WebPage型）を実装

注意: 2023年8月のGoogleアップデート以降、FAQ構造化データはGoogle認定の権威サイト（政府・医療機関）のみ表示されるように縮小。診断サイトでのFAQリッチスニペットの恩恵は限定的になっている。

### 診断結果ページのインデックス戦略

調査した上位サイトの多くは、動的に生成される診断結果ページ（パラメータURLやセッションID付きURL）にはインデックスを与えていない。静的なタイプ解説ページ（/type/infj のような固定URL）をインデックスさせる設計が主流。

- インデックスすべきページ: タイプ別解説ページ（固定URL、独自コンテンツあり）
- noindexが適切なページ: 動的生成の診断結果ページ（URLパラメータ、重複コンテンツリスクあり）

### URL構造とサイトアーキテクチャ

- hitostat.com: `/tests/[診断名-slug]` のシンプルで意味のある構造
- uranaino.net: `/category/mbti/` でカテゴリページも整備、タイプ別は `/archives/` 以下
- カテゴリページ・タグページの整備が内部リンク強化とクローラビリティ向上に貢献

## 4. 小規模サイトが検索上位に食い込んでいる事例

### 事例1: uranaino.net（今すぐ使える心理学）

個人または小規模チームが運営と思われるサイトだが、16タイプのMBTI解説記事を体系的に積み上げた結果、「INFJ とは」「INTJ 特徴」等の中競合ワードで上位表示を実現している。診断ツール（16TEST）を別サブドメインで展開しており、記事とツールの相互送客を設計している。

### 事例2: mbti-navi.com

WordPressの無料テーマ（Cocoon）を使って構築された診断解説サイト。構造化データをJSON-LDで丁寧に実装し、タイプ別ページごとにメタキーワードを最適化することで、大手メディアと同じSERPに入り込んでいる。

### 事例3: commutest.com（ダイコミュ）

エゴグラム診断の31タイプすべてに2,000文字以上の解説を付与し、公認心理師監修を明記することでE-E-A-T（専門性・権威性・信頼性）を担保。医療・心理分野という競合の激しい領域でもニッチなキーワードで上位表示を維持している。

## 5. 「○○診断とは」「○○タイプの特徴」解説記事のSEO有効性評価

### 有効な理由

1. ロングテールキーワードの豊富さ: 16タイプ診断であれば「INFJ 恋愛」「INFJ 仕事」「INFJ 有名人」「INFJ あるある」等、1タイプあたり10〜20のキーワードが成立する。16タイプ×10記事で160本のコンテンツ機会が生まれる。

2. 検索意図との高いマッチ: 診断を受けたユーザーが「自分のタイプについてもっと知りたい」と検索する行動は自然。Knowクエリに対して深い解説を提供できれば高いユーザー満足度を得やすい。

3. 内部リンク強化: タイプ解説ページ→診断ツールの内部リンクにより、診断ページへの権威性の集約が起きる。

4. SNS拡散性: 「これが私のタイプの解説だ」という共感投稿がSNSで自然発生しやすく、外部リンク獲得につながる。

5. 競合優位性: 大手メディアはMBTI全体の概要記事を書くが、各タイプの深い解説ページは個人・専門サイトが強みを持てる領域。

### 注意点

- 解説内容の独自性が低い（他サイトと同じ情報のみ）場合は評価されにくい。自サイトの診断結果データや独自の視点を加えることが重要。
- 文字数だけ増やしても効果は低い。ユーザーの疑問に実際に答えているかが評価基準。

## 6. 診断結果ページの個別インデックスのSEO有効性評価

### 結論: 原則として非推奨（静的タイプページは別）

| ページ種別                                    | インデックス推奨 | 理由                                             |
| --------------------------------------------- | ---------------- | ------------------------------------------------ |
| 動的生成の個人結果ページ（URLパラメータ付き） | noindex推奨      | 重複コンテンツ化しやすく、クロール予算の無駄遣い |
| セッションベースの結果ページ                  | noindex推奨      | 同上                                             |
| タイプ別の固定解説ページ（/result/infj 等）   | インデックス推奨 | 独自コンテンツ、検索意図に一致                   |
| 診断トップページ                              | インデックス推奨 | 診断ツールの主要エントリーポイント               |

### 個別結果ページのインデックスが有効になるケース

- 各結果が完全に独立したユニークコンテンツ（2,000文字以上の独自解説）を持つ場合
- 固定URLで永続的にアクセス可能な設計の場合
- URLにセッションIDやランダム文字列が含まれない場合

yolos.netの現状（診断結果がJavaScriptで動的生成される設計）では、結果ページのインデックス化は難しい。代わりに静的なタイプ解説ページ（例: `/play/animal-personality/result/neko`）を静的HTMLとして作成しインデックスさせる設計変更が現実的な選択肢。

## 7. yolos.netへの示唆

### 優先度高: タイプ別解説ページの新設

yolos.netが保有する14本の診断コンテンツに対して、それぞれの診断結果タイプ別の解説ページを作成することが最も効果的な施策。

具体的なアクション例：

- 動物診断: 「ライオンタイプの特徴と性格」「うさぎタイプの特徴と性格」等
- 四字熟語診断: 各結果タイプの解説ページ
- 各MBTI系診断: 「INFJとは yolos.net版」等

各解説ページに診断ツールへの導線（CTA）を設け、診断ツール→結果→解説記事のサイクルを設計する。

### 優先度中: 解説記事からの診断誘導

「MBTI とは」「エニアグラムとは」等の情報記事を書き、記事内から自サイトの診断ツールに誘導する戦略。初期はトラフィックが少ないが、長期的な流入源になる。

### 優先度低: 大量の診断コンテンツ追加

競合サイトと同じ戦略（200〜500本の診断）をとるには規模が必要で、小規模サイトには向かない。既存14本の質を高める方が現実的。

### 技術的推奨事項

1. 各診断ページにJSON-LD形式の構造化データ（WebPage + BreadcrumbList）を実装
2. 動的生成される結果ページにnoindexを設定（または静的なタイプページに変更）
3. 診断カテゴリページの整備（/play/ 以下のカテゴリ構造の明確化）
4. 各解説ページのメタディスクリプションをキーワードを含む形で最適化
