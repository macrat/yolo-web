# 日本のカジュアル診断サイト結果ページの実態調査

## 調査の背景

yolos.netの診断コンテンツ（伝統色診断、守護キャラ診断等）の結果ページ改善にあたり、一般論やベストプラクティス記事ではなく、実際に成功している日本の診断サイトが結果ページをどのように設計しているかを調査した。特に「テキスト量」「ビジュアル有無」「シェアされやすさ」の関係を実例から明らかにすることを目的とした。

---

## 各サイトの調査結果

### 1. 診断メーカー (shindanmaker.com)

**概要**

2012年サービス開始。ユーザーが名前を入力するとランダムで結果が生成されるシンプルな仕組み。月間数千万の診断実施数を持つ日本最大のカジュアル診断プラットフォーム。UGC（ユーザー作成コンテンツ）型。

**結果ページの構成要素**

- 診断タイトル（H1）
- 名前入力フォーム
- 結果テキスト（診断後に表示）
- Xシェアボタン（1つ）
- 「みんなの診断結果」（他ユーザーのツイート一覧）

**テキスト量**

極めて少ない。代表的な形式は次のテンプレートで生成される1〜2文：
「[USER]さんは[LIST1]で[LIST2]でしょう。」

実際の結果例（推定）：

- 「太郎さんの前世はペンギン、来世はアリンコで、今の本性はニシキヘビでしょう。」（40文字前後）
- ランキング形式でも合計100文字程度
- 長くて200文字程度（テキスト形式の診断）

**ビジュアル要素**

- サイトのロゴ・ヘッダーのみ
- 診断固有のイラスト・画像は**ほぼなし**（診断作成者が設定する場合は一部あり）
- Twitter Cardとして結果ページのOGP画像が生成される
- 基本は白背景＋黒テキストのシンプル構成

**シェアボタン**

- Xシェアボタン1つ（結果テキスト直下）
- 診断名、ハッシュタグ付きでツイート生成
- Twitterの140文字制限に合わせたテキスト設計

**広告**

あり（Google AdSense）。ただし結果テキストの邪魔になる位置には配置されていない。

**ページ長**

非常に短い。1スクロール以内で収まる設計。「みんなの結果」タイムラインが続くが、これは補助コンテンツ。

**シェア実績**

一部の人気診断はTwitterトレンドになるほど拡散。結果が1〜2文でもシェアされる最大の証拠。

### 2. MIRRORZ (mirrorz.jp)

**概要**

「あなたの心を映す心理テストメディア」。4,000万人以上が診断済み。選択式質問に答えて結果が表示されるメディア型診断サイト。

**結果ページの構成要素**

- 結果タイトル（タイプ名）
- 本文説明（「あなたの特性」→「惹かれるタイプ」→「恋愛攻略法」の3段構成）
- X・LINEシェアボタン
- 関連診断リンク
- 広告（Google AdSense、アフィリエイト複数）

**テキスト量**

各結果に約200〜250文字の説明文。3段構成の場合は合計600〜750文字程度。シンプルな診断では1タイプ150〜180文字。

**ビジュアル要素**

- 診断テーマの関連画像（600×400px）を1枚掲載
- 白背景＋グレー・ブラックのテキスト基調
- イラストは最小限。テキスト中心設計
- アフィリエイトバナー複数

**シェアボタン**

「↓あなたの結果をSNSでシェアしよう（結果が自動挿入されます）」というテキスト下にXとLINEの2ボタン。結果ページごとに固定配置。

**広告**

Google AdSense + A8.net系アフィリエイト。コンテンツ内外に分散配置（やや多め）。

**ページ長**

3〜4スクロール。診断→結果→関連記事→新着情報の順に続く。

### 3. ホイミー (hoyme.jp)

**概要**

エンタメ系と本格系を混在させた人気診断サイト。8〜10問の選択式診断。ゲーム的要素（アニメ・漫画キャラ診断、ネタ系診断）が多い。

**結果ページの構成要素（実際のページより）**

- 診断結果タイプ名（例：「アクィエル」）
- キャラクターイラスト（大型・中央配置）
- キャラクター説明文
- 簡易な特徴リスト（チェックマーク形式）
- X・Facebook・LINE・URLコピーのシェアボタン4種
- 関連診断リンク（フッター：25件以上）
- 広告（Google Tag Manager経由、2箇所）

**テキスト量**

800〜900文字（診断説明本文 + 特徴箇条書き込み）。メインの説明文は約300文字、特徴リストが追加で150文字程度。

**ビジュアル要素**

- **診断固有のキャラクターイラストが必ず存在**（ホイミーの最大の特徴）
- イラストは結果ページの冒頭・中央に大きく配置
- 白背景、青/白のカラースキーム
- 各診断結果に対応した異なるイラスト・アイコン

**シェアボタン**

5種類（X・Facebook・LINE・URLコピー・その他）。コンテンツ直下にまとめて配置。

**広告**

2箇所のGoogle AdSense。コンテンツの邪魔にならない位置。

**ページ長**

2,000〜2,500px。コンテンツ部分 + 関連診断25件以上でかなり長い。

### 4. 診断ドットコム (4ndan.com)

**概要**

診断メーカーと同様のUGC型プラットフォーム。名前入力型ランダム結果診断が中心。

**結果ページの構成要素**

- 診断タイトル
- 名前入力フォーム
- 結果テキスト（UGC作成者次第）
- シェアボタン（X・Threads・Bluesky・Misskey等）

**テキスト量**

UGC作成者次第だが、診断メーカーと同様に短文（50〜200文字）が多数。一部の診断では結果ごとに画像を表示する機能を追加（新機能として案内）。

**ビジュアル要素**

基本的にイラストなし。作成者が画像を設定した場合のみ表示。

**シェアボタン**

X・Threads・Bluesky・Misskey等のマルチSNS対応。

**ページ長**

短い（1〜2スクロール）。

### 5. Kuizy (kuizy.net)

**概要**

クイズ・診断・検定を無料作成できる日本最大級のクイズメディア。正解/不正解型クイズと性格診断の両方をサポート。

**結果ページの構成要素（クイズ型）**

- スコア表示（「0/10点」「正答率86.4%・平均59.2%より上」）
- 段階別評価テキスト（5段階、各テキストは簡短）
- 「結果をツイートする！」CTAボタン（目立つ）
- X・Facebook・LINEシェアボタン
- フォローCTAセクション
- 広告（GPT広告スロット複数）

**テキスト量**

クイズ型では結果テキスト自体は非常に短い（50〜100文字程度）。段階別評価テキスト（例：「あなたは一般人です。」「貴様っ！リセットしたな！」）がシンプルで笑えるコピー。

**ビジュアル要素**

- プロフィールアバター画像
- SVGソーシャルアイコン
- シンプル。イラスト・キャラクターはほぼなし

**シェアボタン**

「結果をツイートする！」の大型CTAが最上位、X・Facebook・LINE・コピーの4種が続く。シェア導線の設計が最も積極的。

**ページ長**

中程度（2〜3スクロール）。広告スロット多め。

### 6. mgram (mgram.me)

**概要**

105問の質問に答える「超精密性格診断」。ビジネス・就活での自己分析にも使われる。無料版と有料版（深層性格レポート）あり。

**結果ページの構成要素**

- 「私を構成する8性格」（ハッシュタグ60個から上位8つを視覚的に表示）
- レーダーチャート（8次元の性格グラフ）
- 「私の4つの特異性」（日本人平均との差分を棒グラフで表示）
- 「運命の人に出逢える確率」（数値表示）
- SNS共有ボタン
- 有料版への誘導

**テキスト量**

無料版：各要素に簡潔な説明タグ程度。結果テキスト自体は300文字未満。
有料版（深層性格レポート）：約3,000文字。

**ビジュアル要素**

- **グラフが中心のビジュアル設計**（イラストではなくデータビジュアライゼーション）
- レーダーチャート・棒グラフ・ハッシュタグクラウド
- 「mgramといえばな有名な画像」とされる独自のビジュアルフォーマット
- 白背景 + データ中心のモダンデザイン

**シェアボタン**

SNSシェアボタンあり。平均20ツイート/日（2023年実績）。

**ページ長**

中〜長（3〜5スクロール）。グラフと説明が続く。

## 特に知りたかったことへの回答

### Q1: テキストが少ない（100文字以下）のにシェアされている診断サイトはあるか？

**ある。診断メーカーがその最大の証拠。**

診断メーカーの結果テキストは平均40〜100文字程度（1〜2文）だが、Twitterトレンドに入るほど拡散する人気診断が多数存在する。「あなたの構成要素は○○・○○・○○です」などの短文でも、自己表現欲求を満たすコンテンツであれば積極的にシェアされる。

Kuizyも「あなたは一般人です。」「貴様っ！リセットしたな！」等の短文評価テキストで十分にシェアされている。

**シェアされる理由**：テキスト量ではなく「自分のことを言い表されている感覚（納得感・自己開示欲求）」がシェア動機。これはPNAS 2012研究（Harvard大・Tamir & Mitchell）でも裏付けられており、自己情報の開示行為自体がドーパミン系を活性化する。

### Q2: テキストが多い（500文字以上）カジュアル診断サイトの結果ページはあるか？ユーザーに読まれているか？

**ある。ただし「読まれているか」は慎重な評価が必要。**

- MIRRORZ：1タイプ600〜750文字（3段構成）
- ホイミー：800〜900文字（説明文 + 特徴リスト）
- mgram有料版：約3,000文字

これらは確かに存在するが、「多く読まれているかどうか」を直接確認できるデータはない。業界横断的なデータ（Nielsen Norman Group）では、Webページのテキストをユーザーが実際に読む割合は最大28%とされており、カジュアル診断の文脈では「長文が必ず読まれる」とは言えない。

ホイミーで長いテキストが存在する理由の一つは、「キャラクターの世界観への没入感」であり、娯楽的価値が高ければテキスト量は関係なく読まれる可能性はある。ただし、それはテキスト量の問題ではなく「コンテンツの質と世界観の完成度」の問題と解釈すべきである。

### Q3: 結果ページにイラスト/画像がないのに人気の診断サイトはあるか？

**ある。診断メーカーと診断ドットコムがその代表例。**

診断メーカーの基本的な結果ページにはほぼ画像がない（Twitter Cardのみ）。それでも日本で最も多くシェアされている診断プラットフォームである。

Kuizyも基本的にはシンプルな数値表示のみで、キャラクターイラストはなく、シェア率は高い。

ただし、ホイミーのようにキャラクターイラストを必ず表示するアプローチも成功しており、「イラストがあることが有利かどうか」はジャンルによって異なる。

### Q4: CSSのみ（イラストなし）で視覚的に魅力的な結果ページの実例はあるか？

**mgramが最も近い事例。**

mgramはイラストではなく「データビジュアライゼーション（レーダーチャート・棒グラフ・ハッシュタグクラウド）」で視覚的魅力を実現している。これはCSSとSVGで表現可能な要素であり、外部イラスト素材に依存せずに視覚的な豊かさを作っている点が参考になる。

診断メーカーはCSSのみで非常にシンプルな設計だが、「視覚的に魅力的」ではなく「機能的にミニマル」な設計といえる。

## yolos.netの現在の結果ページとの比較

### yolos.netの現状

コードベース調査より：

- **絵文字アイコン**（result.icon）が結果タイトル上部に大型表示（font-size: 3rem）
- **タイトル**（H1）：result.title
- **クイズ名**：小さなテキスト
- **説明文**（result.description）：テキスト量は quiz データに依存。character-personalityの場合は約220〜270文字
- **detailedContent**（オプション）：「あなたの特徴」（traits 4項目）+ 「こんなところ、ありませんか?」（behaviors 4項目）+ adviceカード。この部分があると合計で1,000文字超
- **CTAボタン**（再挑戦/診断）
- **シェアセクション**（ShareButtonsコンポーネント）
- **関連クイズ**（RelatedQuizzes）
- **レコメンドコンテンツ**（RecommendedContent）

### yolos.netの強みと課題

**強み**

1. 広告なし → 読みやすく、ユーザー体験が良い
2. 絵文字アイコンによる軽量ビジュアル → 外部画像依存なしで視覚的差別化
3. detailedContentによるインタラクティブな特徴リスト → ホイミーに近い深度を実現
4. カラーアクセント（quizのaccentColor）による視覚的統一感

**課題・検討ポイント**

1. **detailedContentのテキスト量**: 合計1,000文字超は「読まれない可能性が高い」テキスト量。ユーザーが実際にスクロールするかどうかがポイント
2. **シェアテキストの工夫**: 診断メーカーのような「シェアしたくなる一言」が結果タイトルに凝縮されているか？
3. **スクロール不要で完結するコア情報**: 競合の多くはFold（最初の画面）内で「タイトル + 短い説明 + シェアボタン」が収まる設計。yolos.netでは detailedContent がある場合にCTAとシェアボタンがスクロール先に押し込まれる

## 結論と示唆

### 1. テキスト量の「正解」はない——但し短文でもシェアされる事実は重い

診断メーカー（40〜100文字）〜ホイミー（800〜900文字）まで、幅広いテキスト量で成功している。「テキストを増やせばよい」という単純な方向性は支持されない。

### 2. 「一言で刺さるタイトル」がシェアの起点

診断メーカーの結果がシェアされる理由の本質は「入力した自分の名前 + ランダムな組み合わせ = 意外性のある一言」にある。yolos.netの結果タイトル（例：「締切3分前に5手先を読む炎の策士」）はこの要件を満たす可能性があり、強みとして維持すべきである。

### 3. ビジュアルはイラストに限らない

mgramがグラフ・チャートで成功しているように、「イラスト」ではなく「ユーザー固有のビジュアル表現」であれば機能する。yolos.netの絵文字アイコンは軽量ながら固有性を持つ設計として適切。

### 4. Fold内でシェアまで完結させる設計が理想

競合の多くは、最初の1スクロールで「タイトル + 説明 + シェアボタン」が揃う。detailedContentが追加コンテンツとして存在するのは問題ないが、シェアボタンがFold内またはその直下に来る設計を維持すべき。

### 5. シェアボタンの「多様性」より「目立ちやすさ」が重要

診断メーカーはXのみで最大級の拡散を達成している。ボタン数を増やすより、「結果をシェアしたい」という動機が湧いた瞬間に迷わずタップできる配置と視認性が重要である。
