# cycle-103 タスク7修正後 再レビュー計画

## 目的

タスク7の10サブタスク（7-1〜7-10）の修正が完了した。修正前は5つの観点でレビューを実施したが、修正後の再レビューは3つに減らしてしまい網羅性が不足していた。修正によって新たな問題が生じていないか、また修正が意図通りに来訪者の体験を向上させているかを、細分化した観点ごとに1人のreviewerを割り当てて徹底的に検証する。

## 判断基準

- 「一貫性のための一貫性」ではなく「その不一致がユーザー体験を損なっているか」が基準
- ツール・辞典は現時点では有効なコンテンツ。「削除予定だから除去」ではなく「優先順位の調整」
- すべてはサイトの来訪者にとっての価値最大化のため

## レビュー分担（12人のreviewer）

各reviewerは1つの狭い観点のみを担当し、Playwrightでの視覚確認を必須とする。

### グループA: 視覚デザイン（4人）

#### R-01: ヒーローセクションのデザイン品質

- ヒーローのグラデーション、装飾絵文字、コピー、CTA、バッジの見た目
- 全バッジが静的に統一されているか（7-3の修正確認）
- モバイルでの装飾絵文字の表示
- ダークモードでのヒーロー表示
- **Playwright**: デスクトップ1280x800、モバイル375x812、ダークモード

#### R-02: カードデザインの品質と統一感

- 「まずはここから」（3列）、「もっと診断」（4列）、「デイリーパズル」（4列）のカード
- カード内要素の配置（アイコン、タイトル、説明、CTA）
- 「毎日更新」バッジがカード右上に安定配置されているか（7-4の修正確認）
- アイコンが背景円からはみ出していないか
- **Playwright**: デスクトップ、モバイル375px、320px

#### R-03: FortunePreviewセクションのデザイン品質

- FortunePreviewが「特別なコンテンツ」として意図的に差別化されているか
- 星評価のfilled（紫）/empty（グレー）の色分けが明確か（7-9の修正確認）
- padding-topが他セクションと揃っているか（7-9の修正確認）
- 「まずはここから」にdailyが含まれていないか（7-1の修正確認）
- **Playwright**: デスクトップ、モバイル

#### R-04: ブログセクションとページ末尾のデザイン

- ブログセクションに説明テキストが追加され余白が統一されているか（7-5の修正確認）
- ページ末尾（ブログ→フッター）の余白・遷移が自然か
- 「その他のコンテンツ」ナビが削除されているか
- **Playwright**: デスクトップ、モバイル

### グループB: インタラクション・挙動（3人）

#### R-05: リンク・ボタンの挙動と見た目の一致

- クリックできる要素とできない要素が視覚的に区別されているか
- ヒーローバッジが全て静的で、クリックできそうに見えないか
- カードのホバー効果（translateY、shadow）が適切か
- CTAボタンのホバー効果
- **Playwright**: デスクトップでホバー操作

#### R-06: タップターゲットとアクセシビリティ

- seeAllLink、CTA、フッターリンクのタップターゲットサイズ計測（44px以上か）
- focus-visibleスタイルの有無と一貫性
- aria-labelledbyの設定
- キーボードナビゲーションの動作
- **Playwright**: モバイル375px、320pxでサイズ計測

#### R-07: セクション末尾リンクの文言と遷移先

- 各セクションの「もっと見る」系リンクの文言は意味が明確か
- 遷移先が正しいか
- 文言が矛盾や誤解を招いていないか

### グループC: 情報設計・構造（3人）

#### R-08: ヘッダーナビゲーションの整合性

- ナビ順序が「ホーム / 遊ぶ / ツール / 辞典 / ブログ / About」になっているか（7-7の修正確認）
- トップページのコンセプトとナビの整合性
- モバイルナビ（MobileNav）にも順序変更が反映されているか
- **Playwright**: デスクトップ、モバイル

#### R-09: フッターの整合性

- フッターのセクション順序が「遊ぶ / ツール / 辞典 / その他」になっているか（7-7の修正確認）
- フッターのリンクが全て有効か
- トップページの内容とフッターの整合性
- **Playwright**: デスクトップ、モバイル

#### R-10: セクション構成と情報量のバランス

- 6セクション（ヒーロー→まずはここから→Fortune→診断→パズル→ブログ）の順序は来訪者にとって最適か
- 各セクションのコンテンツ量は適切か（多すぎ/少なすぎ）
- ページ全体のスクロール量は適切か（モバイルで計測）

### グループD: テキスト・コピー（2人）

#### R-11: ヒーローと各セクションのコピーライティング

- ヒーローのサブタイトルは感情に訴えかけているか
- AI透明性テキストがコンセプトの邪魔をしていないか
- 各セクション見出しと説明テキストは魅力的か
- ブログセクションの新しい説明テキストは適切か（7-5の修正確認）

#### R-12: shortTitleとタイトル表示の品質

- shortTitleが設定されたコンテンツで短縮タイトルが正しく表示されるか（7-10の修正確認）
- モバイルでline-clampされたタイトルが意味の通る形か
- **Playwright**: 320px幅でカードタイトルの表示確認

## 全reviewerへの共通指示

**レビューの目的は、来訪者にとって最高のユーザー体験になっているかを確認することです。修正が計画通りにできているか確認することではありません。** 計画通りに実装されていても、それがユーザーにとって最高の体験でなければ指摘してください。逆に、計画から逸脱していても、それがユーザーにとってより良い体験であれば問題ありません。

## 実行方針

- 12人全員を並行で起動する
- 各reviewerはPlaywright必須（担当観点に関連するスクリーンショットを撮影）
- 指摘はM（必須修正）/S（推奨修正）/N（備考）で分類
- 全reviewerの結果を統合し、Mがあればbuilderで修正→再レビュー
- S以下のみならPMが価値判断してクローズ
