## 0. 本ドキュメントの前提と射程

### 前提

- 新サイトコンセプト「日常の傍にある道具（と、ちょっとした息抜き）」に整合するターゲット像を策定する。
- **旧コンセプト（占い・診断）由来のユーザー像は意図的に脇に置く**。既存GA4データ（月間PV約516、デスクトップ63%）は旧コンセプト時代のもので、新ターゲット策定の根拠としては使わない。参考情報として「デスクトップ比率が高い」程度にとどめる。
- `docs/targets/README.md` の設計原則「ペルソナといえるほど具体的な人物像にはしない／年齢・性別等では限定しない」を継承する（cycle-66で確立された原則）。
- 本草案の市場規模推定はすべて公開統計からの間接推計である。一次データはないため、「仮説」として扱う部分と「事実」として扱う部分を明示する。

### 射程

- 新メインターゲットの候補像とその選定理由
- 既存サブターゲット2件（AIエンジニア／Web製作エンジニア）の扱いの選択肢

### 1-2. 新コンセプトとの適合度評価

**評価軸（新コンセプトから導出）**:

1. **道具性**: 「よく考えられた道具」を評価できる審美眼を持つか
2. **反復利用性**: 繰り返し使う行動パターンを持つか（「毎日使う」は必須ではない。繰り返し使いたい気持ちがあれば十分）
3. **多ツール利用性**: 単一ツール利用で終わらず、複数のツールに興味を持てるか（ダッシュボード構想との整合）
4. **AI運営許容度**: 「AIが運営している・間違っていることもある」というconstitution rule 3の告知を受け入れられるか
5. **日本語市場性**: 日本語を主言語として使う層か（新コンセプトは日本語特化が差別化の柱）
6. **実在規模**: マーケットとして成立する絶対数があるか

**判定基準（6軸共通）**:

- **高**: 新コンセプトと強く一致し、かつ根拠を複数明示できる
- **中**: 条件付きで一致する、または根拠が1つのみ
- **低**: コンセプトとの一致が弱い、または逆方向に働く
- **不明**: 公開データ不足で判断を保留

**総合判定ルール**:

- **総合=高**: 高が3つ以上かつ低が1つ以下
- **総合=中**: 上記未達かつ低が2つ以下
- **総合=低**: 低が3つ以上
- 規模が「不明」の場合、総合判定では「中」として扱い（注）を付ける

### 1-3. メインターゲット候補の絞り込み

上記評価から、総合=高の候補を残し、以下を**除外**する:

- **L5 学生**: 単発利用が中心で反復利用性が低い。レポート時期集中のアクセスは見込めるが、コンセプトの中核「日常の傍にある」との相性が悪い。
- **L6 一般生活者**: 絶対規模は最大だがツールの発見→定着の行動パターンが弱く、個別ツールを単発で使って離脱する典型例。ダッシュボードの価値を感じにくい層。
- **L9 ランチャー愛好家**: Mac/デスクトップアプリに比重があり、ブラウザ完結サービスとは相性が悪い。日本語市場性も低い。
- **L11 ノーコード自動化層**: 日本市場での絶対規模が小さく、かつ「バックグラウンド自動化」が主ニーズでブラウザ内ツール利用と文脈が異なる。

### 1-4. 候補層のクラスタリング（メインターゲット選定のための整理）

残った候補（総合=高のもの、および総合=中のうち戦略的に重要なもの）を、**行動動機の類似性**でクラスタ化する:

**クラスタα: 「PCでの日常作業に道具を組み込む層」**（L1 + L2 + L4）

- Webライター、事務デジタル業務、デザイナー
- 動機: 仕事・作業を効率化したい。同じ作業が週に複数回発生する。
- 道具への興味: 実用目的が主だが、「使いやすいもの」への感度は高い。
- ダッシュボード適合度: 中〜高（毎日開くブラウザに「よく使うツール」を並べたい需要）
- **クラスタ選定の根拠**: L4（デザイナー）は総合=中だが、職業的に道具を日常使いする行動パターンでL1・L2と同じ動機軸を持つため例外的にαに含める。なおL13/L14（既存サブのエンジニア2件）はダッシュボード適合度が低〜中であり「道具を日常作業に組み込む」動機よりも「AI/Web技術情報を取りに来る」動機が支配的なため、ツールユーザー系クラスタαではなくクラスタγ（実験観察者）に分類する。

**クラスタβ: 「道具そのものを愛でる層」**（L7 + L8 + L10）

- 文房具・キッチン道具好きのデジタル版、PKM愛好家、スタートページ常用者
- 動機: 自分専用の環境を作りたい。良い道具を探す行為そのものが楽しい。
- 道具への興味: 審美的・文化的関心。「Calm Technology」的価値観。
- ダッシュボード適合度: 非常に高い（Notion・Obsidian・Start.meユーザーと重なる気質）

**クラスタγ: 「AI自律運営という実験の観察者・共感者」**（L12 + L13 + L14）

- AI自律運営に興味のある読者、技術系エンジニア
- 動機: 「AIが作る道具サイト」という実験そのものに関心がある。ブログ読者層に近い。
- ダッシュボード適合度: 低〜中（ツールを使うこと自体よりメタな興味）

**クラスタδ: 「開発者的エンジニア」**（L3 単体）

- ITエンジニア・開発者
- 動機: 仕事の補助ツール。ただしCLI等の代替が強力で、Webツールへのロイヤルティは低め。
- ダッシュボード適合度: 中（it-tools.techのダイレクト54%のように刺さる人には深く刺さる）

### 1-5. 排除した層と理由のまとめ

| 層                                | 排除理由                                                                                             |
| --------------------------------- | ---------------------------------------------------------------------------------------------------- |
| L5 学生                           | 単発・時期集中型で「日常の傍にある」との乖離                                                         |
| L6 一般生活者                     | 絶対規模は大きいが発見→定着の行動パターンが弱い。ラッコツールズ等が「検索→離脱」で対応済みで差別化難 |
| L9 ランチャー愛好家               | デスクトップアプリ中心でブラウザ特化の本サイトと相性が悪い                                           |
| L11 ノーコード自動化層            | 日本市場規模が小さく、バックグラウンド自動化という文脈の違い                                         |
| L3 ITエンジニアをメインに据える案 | it-tools.tech等の既存競合が強く、CLI等の代替も強い。サブ扱いなら意味があるがメインは非推奨           |

### 1-6. L3（ITエンジニア）の扱いに関する3段階評価

L3は適合度評価で「総合=高」を得ており、メイン・サブ・対象外の3段階それぞれの妥当性を整理する。

| 扱い                           | 判定                            | 根拠                                                                                                                                                                                                                                                                                                       |
| ------------------------------ | ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| メインとして据える             | 非推奨                          | it-tools.tech・MKTools等の既存競合が強力で、L3を主軸にする差別化優位が見出しにくい。CLI/ターミナルツールという強力な代替手段があり、ブラウザWebツールへのロイヤルティが他層より低い。メインにするとコンテンツが技術特化に偏りM1の幅広さを損なう                                                            |
| サブとして追加（2Eとして議論） | 候補あり（論点2の2Eとして整理） | it-tools.techのダイレクトアクセス54%が反復利用の証拠として機能。日本語＋道具審美眼という差別化軸は既存競合（英語中心のエンジニア向けツールサイト）と重ならない可能性。既存ツールの品質基準としても機能しうる。既存サブ「AIエンジニア」との重複は属性が近いが search_intents が異なる点で一定の独立性がある |
| 完全に対象外                   | 非推奨                          | L3はM1（道具を気に入った人）の部分集合でもある。「対象外」とすると道具愛好エンジニアへのコンテンツ制作指針が失われ、機会損失になりうる                                                                                                                                                                     |

**planner結論**: L3はメインには据えないが、論点2の2Eとしてサブ追加の議論対象とする（完全対象外にはしない）。

### 2-1. メインターゲット 候補案（planner提示の複数案）

name: 毎日の作業を気に入った道具で整えたい人

  - ブラウザで何かを調べたり作業したりするのが日常であること

## 3. 市場規模の見積もり

### 3-1. 日本市場の推定（事実と推測の区別）

**事実ベースの底上げ数値**:

- 日本のインターネット利用者: 約1億704万人（総務省2024年末推計）
- PC 利用率: 46.8%（PC 利用者は約4,670万人と推計）
- Webツール月1回以上利用者: 約500〜1,500万人（下限500万=職業ベース推計〔事務職・ライター・エンジニア等の合計から反復利用層を抽出〕、上限1,500万=PV逆算推計〔ラッコツールズ等の Similarweb 推計からの逆算〕。手法が異なるため幅が広い。リサーチ1の独自推計）
- 文房具愛好家コミュニティ: 文具女子博累計70万人来場（事実）
- 「道具にこだわる」共感層: 日本の成人1億600万人のうち20〜30%（2,000〜3,000万人）との推定
- Notion 日本ユーザー: 数十万〜数百万人規模（推定。世界1億人のうち日本比率8〜12%仮定）

**推測を重ねた推計**:

「毎日の作業を気に入った道具で整えたい人」（案M1）に該当する層の規模:

| 水準   | 推定規模（日本） | 推定根拠                                                                                                                                                    |
| ------ | ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 楽観値 | 500〜800万人     | Webツール月1回以上利用者のうち、リピート利用の素養がある層。事務デジタル業務＋Webライター＋デザイナー＋道具愛好家のコア層の合計推計                         |
| 標準値 | 150〜300万人     | 「複数ツールを反復利用する」「道具の設計に美意識を持つ」行動特性を併せ持つ層。Notion＋Start.me＋Obsidian 的層の日本での推定                                 |

**現実的にリーチできる範囲**（サイト成長後の到達可能ユーザー、年単位の想定）:

- 標準値: 月間10〜50万PV（keisan.site の一部〜10015.io 相当）
- 悲観値: 月間1〜5万PV（現在の500PV から 100倍程度）

### 3-2. AI 運営ブログ読者（案M2採用時）の規模推定

- 公開統計なし。AI 関連ブログ記事のはてブ・Qiita・Zenn での反応規模から間接推測するのみ。
- 日本の ChatGPT/Claude ユーザー（数百万〜1,000万人規模と推定）のうち、「AI 自律運営の実験」に継続関心を持つのは1〜5%程度と仮定すると、数万〜数十万人規模。
- ただし「AI ブーム」の継続性に依存する層であり、規模の安定性は案M1より低い。

### 3-3. 推定の限界

- Webツールサイト利用者を直接対象にした日本の公開統計は存在しない。すべて間接推計。
- 「道具愛好」と「Webツール利用」のオーバーラップを直接示すデータは見つからなかった（調査の限界）。
- 生成AIによる代替（ChatGPT で文字数カウントや整形ができる）により、一部の単純ツール市場は縮小傾向の可能性。
- 本推計は「どの層が何万人」の精度を保証するものではなく、「メインターゲットとして成立する最小規模はあるか」の判断材料に留まる。成立する最小規模はいずれの案でも満たしていると planner は評価する。

**成立する最小規模の操作定義**: 1年後に月間10万PV規模を達成可能な読者層を国内に持つこと（現在約500PVの200倍。記事100本×月間1,000PV/記事 or ツール50個×月間2,000PV/ツール相当の現実的上限の内側）。M1（標準値150〜300万人）・M2（数万〜数十万人）とも理論上の母数では成立。M2はAIブーム継続依存性があり達成確度はM1より低い。

### M1b再設計の考え方（Owner指示「徹底的に考え抜く」に対応）

旧M1bは「道具箱（ダッシュボード）」「ツール間連携」「カスタマイズ」といった未実装の開発側機能を前提にした定義だった。これは「来訪者が何を求めているか」ではなく「我々が何を作りたいか」から出発した逆向きの記述であり、新名称「気に入った道具を繰り返し使っている人」への改名を機に全項目をゼロベースで書き直した。

「繰り返し使う」層をどう捉えたか: 彼らは必ずしも毎日来るヘビーユーザーではなく、週次・月次の頻度で「同じ作業」が発生したときに「前に使ったアレ」と思って戻ってくる人々である。ブックマーク、履歴、サイト名検索など再到達手段は多様で、ダッシュボード機能があろうとなかろうと、彼らは既に「繰り返し使っている」。彼らを引きつけているのは派手な新機能ではなく、「変わらないこと」「慣れた手順が壊されないこと」「自分が選んだものが間違っていなかったという確信の継続」という静かな価値である。したがって道具箱などの新機能は「実装後に彼らが追加で評価するかもしれないもの」であって、定義の前提ではない。彼らの本質的な不満は「久しぶりに開いたら変わっていた／壊れていた／消えていた」という、継続利用を前提としたときに初めて顔を出す種類のものである。

### Round 2 レビュー指摘への対応（Must 4件 / Should 4件 / Nit 2件）

#### Must-1: M1b の likes / interests の「旧発想の残存」解消

reviewer は以下の2点を「リピーターの実需と乖離した、審美的愛好層／開発側の語彙の残存」と指摘した:

- `interests` の「ちょっとした使い勝手の改善が、使い続けるうちに静かに積み上がること」
- `likes` の「細部まで丁寧に作られていて、使うたびに小さな気配りに気づけること」

前者は「改善を続けたい」という運営側の願望が漏れた記述、後者は「道具を愛でる人」の視点でありリピーターの本質ではない、という指摘に builder は同意した。リピーターの本質は「考えなくて済むから戻る」「前回と同じ体験が再現される」ことにある。

対応:

- `interests` の該当行を「今日もまた、昨日と同じようにその道具が動いてくれること」に置換。運営側の「積み上げる改善」ではなく、来訪者側の「変わらないことを静かに確認する」視点に反転させた。
- `likes` の該当行は単に削除するのではなく、**Must-2** で追加すべき「同じ入力に対して前回と同じ結果が返ってくること」に置換。これは「細部への気配り」の審美的視点ではなく、継続利用で最も痛点化する「再現性」の肯定形であり、M1bの軸（変わらないこと・前回と同じ）と直接整合する。
- あわせて `interests` の「考えずに使える」行の表現を「もう考えずに使えるようになった」に微修正し、「今は考えずに済んでいる」状態を描く表現に揃えた。

この置換により、M1b の likes / interests は全5〜4項目が「変わらない・前回と同じ・慣れた手順・壊されない」という単一軸に収束した。審美的愛好層（旧草案M1の道具愛好的語彙）との重なりは消えた。

#### Must-2: M1b の dislikes に再現性の痛点を追加

#### Must-3: M1b の search_intents の扱い

選択した選択肢: **(a) search_intents を空にし、コメントで主経路と理由を明示**

理由:

- リピーターの主経路は「ブックマーク・履歴・直接URL入力」であり、検索はそもそも補助的位置づけ。指名検索（「yolos」「yolos.net」等）を想像で列挙すると、AP-P01（未実測の仮定を列挙する）と同じ構造の誤りになる。
- (c) の「前回使った◯◯ もう一度」型クエリも魅力的だが、これも想像の域を出ず、Search Console で実測できていない現時点では同じ問題を抱える。
- (b) の「指名検索を残すが実測データなし注記」は、残した指名検索が参照されるリスクがある（「願望キーワード」として機能してしまう）。
- 最も正直な記述は「主経路は検索ではないこと／実測データが出るまでは埋めないこと」をコメントで明示し、`search_intents: []` とすること。これにより将来 Search Console データが出た時点で素直に埋められる。

### Step 5 申し送り追補（Should-4 対応）

M1b のターゲット定義は「変わらないこと／慣れた手順が壊されないこと／前回と同じ結果が返ってくること」を価値の中心に置いている。一方、constitution rule 5 は「Try a variety of things with creative ideas」であり、継続的な実験・変更・新機軸の導入を奨励する。この2つは原理的に緊張関係にある。

Step 5 のキャラクター設計および運用方針策定では、以下を留意点として扱うこと:

1. 「実験・創造」は主にサイト全体・ブログ・新ツールの追加といった**拡張方向**で発現させる。既存で愛用されているツールの UI や挙動の刷新は、M1b の価値を直接損なうため慎重に扱う。
2. 既存ツールに手を入れる場合は、URL・入力欄位置・挙動の互換性を優先し、「変わらないこと」を破壊しない範囲で刷新を行う。後方互換を破る変更は相応の告知・代替提供を条件とする。
3. 新しい試みは「新しい道具・新しい場所」として追加する方が、M1b と rule 5 の両立に向いている。既存道具の置き換えは両立を難しくする。
4. キャラクター設計では「変わらないものを守る静かな姿勢」と「新しいものを楽しむ好奇心」を両立させる人格を想定する。両者は矛盾しない（伝統工芸の職人・老舗の主人・長く使える道具の作り手等、現実にも両立する人格像はある）。
