```yaml
name: 日々の仕事で使える「ちょい足し」業務ツールが欲しい人
knows:
  - 自分がやりたい業務の内容
doesnt_know:
  - yolos.net が何なのか
  - yolos.net で提供しているサービス
  - yolos.net のツールの使い方や特徴
  - 高度な専門知識
interests:
  - 入力や操作が少なく、すぐ結果が出る簡単な業務支援ツール
  - 期限・日付・単位など、ミスが起きやすい領域の確認を短縮すること
  - 定型文を土台にして時短すること（メール文面など）
  - 専門知識が少なくても使えるツール
likes:
  - 迷わず使えるシンプルなUI
  - コピペで完結し、すぐ元の作業に戻れること
  - 結果が明確で、理由や注意点が簡潔に示されること
dislikes:
  - 広告・ポップアップで作業が止まること
  - 仕様や前提が曖昧で、結果を信用できないこと
  - 操作が多く、手順が複雑なツール
  - 習熟するのに時間がかかるツール
  - 専門知識が必要で、使いこなせないツール
```

## 5. 失われた知見の整理

### サイクル間で失われていた知見

**「継続利用の動機」として最も体系的に記述されていたのは2-A・2-B**  
特に「コピペで元の作業に戻れる」「仕様が曖昧で信用できないことへの忌避」は、ツール系コンテンツの継続利用を左右する核心であり、cycle-65での日本文化特化時に切り捨てられてから、cycle-167まで約2ヶ月間定義から欠落していた。

**「試行錯誤の行動パターン」**  
2-A〜Bには「専門知識なしでも使える」「すぐ結果を見て判断できる」という記述があり、これはツール利用者の試行錯誤的な利用スタイルを間接的に示している。新定義では「気軽に試せる」「何度でも使える」という継続利用動機として明示的に加えることができる。

**「信頼性への需要」**  
2-Cの「仕様や前提が曖昧で結果を信用できない」「計算結果の根拠や意味が説明されない」は、ツールに対するユーザーの信頼形成要件として重要。占い・診断コンセプト期に完全に欠落していた観点。

### 「切り捨てるべきもの」vs「普遍的な知見」

**切り捨てるべきもの（旧コンセプト固有）**

- SNSシェアを前提にした設計要件全般
- 診断結果のエンタメ性（バーナム効果・結果画像・笑え等）
- 5問・2分以内という具体数値（診断特有）
- Z世代・女性向けターゲット属性

**普遍的な知見（新コンセプトでも生きる）**

- 「登録不要」「インストール不要」はあらゆるユーザー層に共通
- 「コピペで完結」「すぐ元の作業に戻れる」は実用ツールの根幹
- 「信頼できる結果かどうか」へのこだわり
- 「広告・ポップアップで中断される」ことへの強い忌避
- 「使う前に全容が把握できること」（所要時間・入力形式等）
