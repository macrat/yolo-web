# cycle-176 レビュー観点リスト

このリストは cycle-176 の D-2「視覚評価」と reviewer 依頼テンプレートの両方で参照する SSoT。各観点に検証手段を併記。

観点リストは最低要件であり、reviewer / D-2 担当が追加観点を補足することを妨げない（cycle-176 計画書 L120 の「少なくとも以下を含める」表現と整合）。

## 検証手段の凡例

- 【視覚】= D-2（複数 viewport × ライト/ダーク のスクリーンショット画像で評価）
- 【機能】= C-3 完了条件 / Playwright シナリオ / E2E テスト で評価
- 【両方】= 視覚と機能の両面で評価（D-2 で視覚部分を、C-3 / Playwright で機能部分を評価し、両方の合格をもって観点合格とする）

## 観点リスト

1. M1a dislikes / M1b dislikes との整合（targets/\*.yaml を参照）【両方】

2. モバイル w360 でのファーストビュータイル数（前提 B-密度 の 8 タイル以上を満たすか）【視覚】

3. PC / タブレット / モバイル各 viewport の密度が前提 B-密度 と整合（PC で「12 列 12 タイル」と「4 列 8 タイル」のいずれでも通過する曖昧さがないこと）【視覚】

4. a11y 代替経路（キーボード操作 / 補助ボタン等の DnD 代替経路）が機能する（前提 B-モード分離）【機能】

5. Edit モード中はタイル本体のクリックが無効化され、使用モード中は逆に並び替え操作が効かない（操作排他）【機能】

6. Undo の確定動作が動作する：Undo 期間経過後にデータが確定し、それ以降は元に戻せない（C-3 完了条件で評価）【機能】

7. 初期デフォルトプリセットに「似たようなツールが並んで迷わせる」M1a dislike が混入していない（プリセット内のツール機能種別の被り検査）【両方】

8. 誤タップ可能性（タップターゲット 44px 規約 / 隣接タップ干渉）【視覚】

9. 編集モードの儀式性【両方】
   - 編集開始タップ数 ≦ 1（Edit ボタン 1 タップで即操作可能であること）
   - 編集中に確認モーダル等の追加ステップが介在しないこと
   - cycle-175 で批判された「編集→暗転→操作→完了」の 4 ステップ儀式が再発していないこと
   - ※視覚部分（編集モードへの遷移表現・ボタン配置）は D-2 で画像評価、機能部分（実際のタップ数・ステップ数）は Playwright シナリオ / 手動確認で評価

10. 削除誤操作の発生余地と Undo が機能要件として実際に成立しているか（C-3 完了条件で評価）【機能】

11. 「日常の傍にある道具」コンセプトとの整合【両方】

- 【視覚】以下の静的指標を D-2 の画像から評価する：
  - タイル密度（ファーストビューに役立つ道具が並んでいるか）
  - タイルサイズ（指で押しやすい大きさか）
  - タイトル文字の即可読性（フォントサイズ・コントラストが十分か）
  - アクセス障害物の不在（広告・モーダル・スプラッシュ等が反射的アクセスを妨げていないか）
- 【機能】「反射的に開ける速度感」という動的指標は静止画像では判定不能なため、E2E / 手動シナリオ（ページロード後に目的タイルへ到達するまでのステップ数）で評価する

12. DESIGN.md §4 視覚表現規約（box-shadow のみ / 半透明禁止 / 揺れアニメ不採用）の遵守【視覚】

13. 自動学習・履歴ベース並び替え等の M1b dislikes に該当する挙動が混入していないこと【機能】

14. ライト/ダークモード両方での視覚整合（背景色コントラスト・テキスト可読性・編集モード視覚表現が両モードで破綻なく成立すること）【視覚】
