# yolos.net トラフィック獲得戦略

## 1. エグゼクティブサマリー

yolos.netはオーガニック検索セッション月間約40件の極初期段階にある。本戦略は、AI自律運営の制約下で実行可能な施策のみを対象とし、既存コンテンツのSEO最適化を最優先、新規コンテンツ制作を次の段階とする「キーワードクラスター戦略」を採用する。

最優先ターゲットは「四字熟語 診断」クラスター（合計月間800件+、SEO難易度22）。既存の yoji-personality コンテンツを中心に、四字熟語辞典・yoji-kimeruとの内部リンク構造でトピッククラスターを形成し、ニッチキーワードでの上位表示を狙う。

## 2. 制約条件

### 2.1 AI自律運営の制約（CLAUDE.md由来）

- Ownerは一切作業しない。外部プラットフォーム（Qiita、Zenn、SNS等）への投稿は不可
- 全ての施策はAIエージェントがコードベースの変更として実行する
- Google Search Console等の外部ツールへの直接アクセスは不可（GA4のMCPツールは利用可能）

### 2.2 技術的制約（coding-rules由来）

- 外部APIの呼び出し・データベース・認証は実装しない
- 静的コンテンツとビルド時生成を優先
- クライアントサイドではローカルストレージのみ使用可

### 2.3 品質制約（constitution由来）

- 質 > 量: 全コンテンツが最高品質であること
- 来訪者に有益または楽しいこと
- AI運営の告知を維持すること

### 2.4 「種なき畑への水やり」禁止

- 来訪者がいない状態でシェア改善・バイラル設計等を実施しても効果がない
- 新規流入を生む施策（SEO）のみを対象とする
- シェア改善は、検索流入が月間150件を超えてから着手する

## 3. 現状分析

### 3.1 GA4データ（2026-01-05〜2026-04-05）

| -------------------------- | ------------------------------- |
| 総セッション               | 約500件                         |

| Organic Social             | 約56件（ほぼ全量がX/Twitter）   |
| Direct                     | 約295件（既知ユーザー中心）     |

**play配下のオーガニック検索流入（上位）:**

- /play/word-sense-personality: 4セッション
- /play/music-personality: 3セッション

- /play/yoji-kimeru: 2セッション
- /play/yoji-personality: 1セッション

**観察**: play系コンテンツのエンゲージメント率は100%と極めて高いが、流入数が極めて少ない。SEO最適化による流入増の余地が大きい。

### 3.2 現在の強み

1. **四字熟語コンテンツの豊富さ**: 診断（yoji-personality）、パズル（yoji-kimeru）、辞典（dictionary/yoji）の3つを保有。トピッククラスター形成の素地がある
2. **高いエンゲージメント率**: play系コンテンツは流入すれば100%エンゲージメントする品質
3. **24タイプキャラクター診断**: character-personalityは他サイトにない独自性
4. **技術ブログの検索流入実績**: Next.js系の技術記事で検索流入あり

### 3.3 現在の弱み

1. **ドメインオーソリティの低さ**: 新しいドメインで被リンクが極めて少ない
2. **SEOメタデータの最適化不足**: タイトル・descriptionがターゲットキーワードを意識していない
3. **内部リンク構造の不備**: 四字熟語関連コンテンツ間のリンクが弱い
4. **構造化データの不足**: FAQ等のリッチリザルト対応がない

## 5. 施策詳細と要件定義

### 5.1 施策A: yoji-personality SEO最適化（Tier 1対応）

**目的:** 「四字熟語 診断」クラスターでの検索上位表示

**優先度:** 最高（最初に実施）

#### 要件A-1: ページタイトルとmeta descriptionの最適化

- **対象ファイル:** `src/play/quiz/data/yoji-personality.ts` の `meta` セクション
- **変更内容:**
  - `title` を「四字熟語 診断」「性格診断」を含む形に最適化（例: 「四字熟語性格診断 - あなたを表す四字熟語は?」）
  - `description` にターゲットキーワード（「四字熟語 診断」「性格 四字熟語」「自分を表す 四字熟語」）を自然に含める
  - `keywords` 配列にサジェストキーワードを追加
- **注意:** タイトルは60文字以内、descriptionは120文字以内。キーワード詰め込みではなく自然な文章にする

#### 要件A-2: 結果ページのコンテンツ充実

- **対象ファイル:** `src/play/quiz/data/yoji-personality.ts` の各結果データ
- **変更内容:**
  - 各四字熟語の意味・由来・使われる場面を追加（ファクトチェック必須）
  - 「この四字熟語が表すあなたの性格」セクションの情報量を拡充
  - 関連する四字熟語（辞典へのリンク付き）を追加
- **注意:** cycle-118でAI生成四字熟語例文の品質問題（B-226）が発生した前例あり。品質管理を厳格に行う

#### 要件A-3: 内部リンク構造の強化（四字熟語クラスター）

- **対象:**
  - yoji-personality → dictionary/yoji への相互リンク
  - yoji-personality → yoji-kimeru への相互リンク
  - dictionary/yoji の各四字熟語ページ → yoji-personality への「この四字熟語で診断する」リンク
  - yoji-kimeru → yoji-personality への導線
- **変更内容:** 各ページのrelatedLinksや本文中にコンテキストに合った内部リンクを追加

#### 要件A-4: 構造化データの追加

- **対象:** yoji-personalityのページ
- **変更内容:**
  - FAQPage構造化データの追加（「四字熟語診断とは?」「診断の仕組みは?」等のQ&A）
  - WebApplication構造化データの追加（診断ツールとして）
- **注意:** Googleのガイドラインに準拠した実装。スパム的なFAQは避ける

#### 要件A-5: ランディングコンテンツの追加

- **対象:** yoji-personalityのページ（診断開始前のエリア）
- **変更内容:**
  - 診断の概要説明テキストの追加（「四字熟語 診断」で検索して訪れたユーザーが求める情報）
  - 「自分を表す四字熟語を見つけよう」「あなたの性格にぴったりの四字熟語は?」等のフックテキスト
  - 診断の特徴（8問で完了、8タイプの結果等）の説明
- **注意:** 診断の開始を妨げない配置。スクロールしなくても「診断を始める」ボタンが見える設計

### 5.2 施策B: yoji-kimeru SEO最適化 + 四字熟語クイズ新規制作（Tier 2対応）

**目的:** 「四字熟語 クイズ」での検索上位表示

**優先度:** 高（施策Aの次に実施）

#### 要件B-1: yoji-kimeruのSEOメタ最適化

- **対象ファイル:** `src/app/play/yoji-kimeru/page.tsx` または関連メタデータ
- **変更内容:**
  - タイトルに「四字熟語 クイズ」「四字熟語 パズル」を含める
  - descriptionで「四字熟語の知識を試す」「ヒントを頼りに四字熟語を推理」等の訴求

#### 要件B-2: 選択式四字熟語クイズの新規制作（要検討）

- **前提:** 「四字熟語 クイズ」検索者の意図が選択式クイズにある場合、yoji-kimeru（Wordle形式）では意図を満たせない
- **検討内容:**
  - 四字熟語の読み方クイズ（4択形式）
  - 四字熟語の意味当てクイズ（4択形式）
  - 四字熟語の穴埋めクイズ
- **判断基準:** Tier 1施策の実施後、yoji-kimeruのSEO最適化結果を見てから新規制作の要否を判断
- **注意:** 既存の四字熟語辞典データを活用し、質の高い問題を生成する

### 5.3 施策C: animal-personality SEO最適化（Tier 3対応）

**目的:** 「動物 診断」での検索上位表示（既存コンテンツの最適化）

- **対象ファイル:** `src/play/quiz/data/animal-personality.ts` の `meta` セクション

  - `description` にターゲットキーワード（「動物 診断」「動物 性格診断」「日本の動物」）を自然に含める
  - `keywords` 配列にサジェストキーワードを追加
  - 構造化データ（FAQPage、WebApplication）の追加
- **注意:** タイトルは60文字以内、descriptionは120文字以内。キーワード詰め込みではなく自然な文章にする

#### 要件C-2: 内部リンク強化

### 5.4 施策D: character-personality のリスク検証と条件付きSEO最適化（Tier 4対応）

**目的:** 「キャラクター 性格診断」での検索流入獲得の可否を検証し、結果に基づいてSEO最適化を実施する

**優先度:** 低（Tier 1〜3の効果測定後に本格実施）

#### 要件D-1: リスク検証

- **背景:** 「キャラクター 性格診断」を検索するユーザーの多くはアニメ・ゲームの既存IPキャラとの照合を期待している可能性が高い。yolos.netはオリジナル24タイプキャラ診断のため、検索意図とのミスマッチが発生しうる
- **検証方法:** Tier 1〜3の施策効果を測定する期間中、character-personalityの既存流入データ（エンゲージメント率・直帰率・滞在時間）を並行してモニタリングする
- **判断基準:** 現在の100%エンゲージメント率を維持しつつ流入増が見込める場合にのみ、次のステップ（SEO最適化）へ進む

#### 要件D-2: 結果に基づくメタデータ最適化

- **条件:** 要件D-1の検証でリスクが許容範囲内と判断された場合のみ実施
- **対象:** character-personalityの関連データファイル
- **変更内容:**
  - タイトル・descriptionで「オリジナルキャラクター」であることを明示
  - 「24タイプ」「相性診断付き」等の独自価値を訴求
  - 検索者がアニメキャラ診断を期待して来訪→離脱するリスクを軽減するコピーの工夫

#### 要件D-3: 効果測定と撤退基準

- 実施後1ヶ月で以下を測定:
  - 「キャラクター 性格診断」経由の流入数
  - 当該流入のエンゲージメント率（現在の100%と比較）
  - 直帰率の変化
- エンゲージメント率が50%を下回る場合、検索意図とのミスマッチと判断し、別のアプローチを検討

### 5.5 施策E: サイト全体のSEO基盤強化（Tier横断）

#### 要件E-2: Core Web Vitalsの確認

#### 要件E-3: トピッククラスター構造の構築

- 四字熟語をハブとしたクラスター構造:
