## A. cycle-210 で確立された SSoT 4 項目（複合入力型タイル固有）

### SSoT (i): AP-P21 判定基準 (i) 下限 40px の適用範囲

- **SSoT 番号**: cycle-210 (i)
- **内容**: textarea / status 領域 / エラー枠 / コピーボタン等の「visitor が直接視認・操作する要素」に対して 40px 以上の下限を適用する。複合入力型タイルでは **操作側 input（flexShrink:0 固定）にも 40px 以上を適用する**。
- **実測値**: cycle-210 T-4 で input の自然高さ 27px が基準未達 → `minHeight: 40px` を実装適用して解消。WCAG 2.5.8 下限 24px より厳しい yolos.net 独自基準（WCAG 推奨 44px に接近）。
- **text-diff への適用性**: **引用適用可**。text-diff も複合入力型タイルであり、入力 input / textarea / エラー枠すべてに 40px 下限を適用する。操作側 input（検索 / 置換に相当するフィールド）も同様。
- **出典**: cycle-210.md L841 §補足事項 1 項目目

---

### SSoT (ii): AP-P21 判定基準 (ii) 相互差 2px 以内の適用範囲

- **SSoT 番号**: cycle-210 (ii)
- **内容**: 単一 textarea タイル基準「操作側と膨張側の高さ相互差 2px 以内」は複合入力型タイルには**機械的に適用しない**。複合入力型タイルでは膨張側（本文 textarea + 結果欄）が他要素の高さに応じて再配分されるため、本文 textarea / 結果欄の相互差判定は**適用外**。**操作側 input のみ flexShrink:0 で固定要件として 2px 以内を維持**する。
- **実測値**: cycle-210 T-4 で (a)→(b) の本文 textarea 差 = 13.83px が観測されたが、これは複合入力型タイル特有の正常変化で判定 FAIL ではない（構造的に自然な再配分）。
- **text-diff への適用性**: **引用適用可**。text-diff の操作側（検索 input 等）は flexShrink:0 固定のため 2px 以内基準が適用される。膨張側の本文 textarea + 差分結果欄の相互差判定は適用外。
- **出典**: cycle-210.md L843 §補足事項 2 項目目

---

### SSoT (v): AP-P21 判定基準 (v) ±10% の適用範囲（複合入力型基準 = ±15%）

- **SSoT 番号**: cycle-210 (v)
- **内容**: 単一 textarea タイル基準 ±10%（cycle-209 で確立）は複合入力型タイルには直接適用不可。複合入力型タイルではエラー枠表示分の高さ振替が膨張側（本文 textarea + 結果欄）に分配されるため、**複合入力型タイル基準 = ±15% 以内（経験的暫定値）**を適用する。補助指標として relative 式「エラー枠高さ ÷ 本文 textarea 高さ × 100% 以内」も参考値として用いる。
- **実測値**:
  - (d)→(e) 変化率 = **11.55%**（本文 textarea 105.78px → 93.56px / エラー枠 46.09px 追加による再配分）
  - 単一 textarea 基準（±10%）では FAIL だが、複合入力型タイル基準（±15%）では PASS
  - relative 式: 46.09 ÷ 105.78 ≈ 43.6% 以内 = PASS
- **重要注記**: ±15% は **N=1 サンプルからの経験的暫定値**。B-452 で N≥3 件蓄積後に見直す予定。cycle-214（text-diff）は複合入力型 **N=2 件目**となり、B-452 の N=2 データポイントを蓄積する。
- **text-diff への適用性**: **引用適用可（ただし暫定値の再評価も兼ねる）**。text-diff での (d)→(e) 変化率を計測し、±15% 基準の妥当性を N=2 で検証する。FAIL した場合は基準値の見直し提案が必要（builder 独自解釈での PASS 判定禁止 = AP-WF03）。
- **出典**: cycle-210.md L847–852 §補足事項 4 項目目

### SSoT (vi): エラー文言枠の 1 行収納基準と実測 SSoT 値

- **SSoT 番号**: cycle-210 (vi)
- **内容**: タイル UI 内のエラー文言枠について「実測幅 ≤ 376px（= タイル幅 400px - padding 左右合計 24px）+ エラー文言が 1 行で表示されている（折り返しなし）+ 高さ実測値は下限 40px 以上で概ね 40〜60px 範囲内」を判定基準とする。計画段階では推定値のみで SSoT を確定しない。実 SSoT は T-4 実機計測値で確定する方針。
- **実測値（SSoT 確定値）**:
  - h = **46.09px**（タイル UI エラー枠 / `padding: "11px 8px"` / `fontSize: "0.8125rem"` / Playwright 実測）
  - w = **376px**
  - 1 行収納 PASS / h ≥ 40px PASS
  - エラー文言: 「入力テキストが長すぎます（最大100,000文字）」（全角 21 字 + 半角 8 字）
- **text-diff への適用性**: **再評価必要**。text-diff のエラー文言は text-replace とは異なる文言になる可能性が高い（text-diff は 2 テキスト比較のため異なる制限文言が存在しうる）。文言の字数が変わると h・w の実測値も変わるため、T-4 で独自計測して確定する必要がある。ただし「実 SSoT は T-4 実機計測で確定 / 計画段階では推定値のみ」の方針（SSoT (vi)の手順自体）は引用適用可。
- **出典**: cycle-210.md L538 / L835 / L845 §補足事項 3 項目目

## B. text-replace の AP-P21 計測手法（系統分け）

### 系統分けの構造

cycle-210 の AP-P21 計測は以下 **5 ケース（a〜e）** で実施（出典: cycle-210.md L285–298）:

| ケース | 系統の状態                                                            | 概要                 |
| ------ | --------------------------------------------------------------------- | -------------------- |
| (a)    | 空入力（膨張なし）                                                    | 初期状態             |
| (b)    | 短い入力（本文 30 字 + 検索/置換 5 字程度）                           | 低負荷               |
| (c)    | 中程度入力 + 大量置換（本文 300 字 / 「foo」20 箇所 → 「bazbazbaz」） | 結果欄膨張           |
| (d)    | 入力長制限ぎりぎり付近（9 万文字 + 検索/置換短）                      | 高負荷               |
| (e)    | 100,000 字超過時のエラー表示                                          | エラー枠条件付き表示 |

「複合入力型タイル特有の系統分け」として「**未入力時 / 入力時 / 置換実行後**」という3分類ではなく、上記 a〜e の 5 ケースで構成されており、特に (e) は「エラー枠が条件付き表示されたときの高さ変化」を CLS 視点で捕捉するために追加された。

### 系統内変化率の閾値

- **(a)(b) 相互差判定**: 操作側 input のみ ≤ 2px（膨張側 textarea / 結果欄の相互差判定は適用外 = SSoT (ii)）
- **(c)(d) 単独基準**: 本文 textarea ≥ 40px 維持 + 結果欄枠内収納（`overflowY: auto`）
- **(e) 変化率**: 複合入力型タイル基準 **±15% 以内**（経験的暫定値 = SSoT (v)）。判定は (d)→(e) の変化率を主基準とする（エラー枠表示前の最終安定状態 (d) と表示後 (e) の差が CLS 視点で意味を持つため）

### 計測した実測値

- textarea (a) 119.61px / (b) 105.78px / (c) 105.78px / (d) 105.78px / (e) 93.56px
- 検索・置換 input 各 40px（minHeight:40 適用）
- 結果 status 欄 (a) 105.61px / (b)(c)(d) 91.8px / (e) 79.56px
- (d)→(e) 変化率 = **11.55%** → ±15% 基準で PASS

### 系統数の位置づけ

5 ケース（a〜e）は「複合入力型タイル（text-replace）固有の暫定設計」として確立。出典: cycle-210.md L296 の注記「複合入力型タイル初回適用範囲を明示」。単一 textarea タイル（cycle-209）との差分として「操作側 input が複数」「エラー枠が条件付き表示」という 2 点が系統設計に影響している。

## C. 複合入力型タイルの UI 構造 SSoT

### タイルサイズ

- **recommendedSize**: cols=3 rows=3（400×400px）
- **決定根拠**（出典: cycle-210.md L367–384 §論点 1）:
  1. 「複数入力フィールド + 結果欄を視認可能に配置できる最小サイズ」= 400px 高さが必要
  2. cols=3 rows=2（264px）では本文 textarea が rows=1 相当に圧迫される = visitor 価値毀損
  3. qr-code（cycle-207）に続く 2 件目の rows=3 = Phase 8.1 内の既出先例あり
  4. target-user-mapping §推奨「複数行テキストを貼り付けて一括置換する主ユースケースに対応」と整合

### AP-P21 役割分担パターン（論点 4 案 α）

- **操作側（flexShrink:0）**: 検索 input + 置換 input（短い固定高さ / minHeight: 40px）
- **膨張側（flex:1 + overflowY:auto）**: 本文 textarea + 結果欄（残余高さを分け合う）
- **採択理由**: 本文 textarea は visitor が貼り付けた長文の確認に使う = 視認性確保が必要 = 膨張側に置く
- **出典**: cycle-210.md L492–514 §論点 4

### 入力フィールド配置（論点 3 案 X）

- **採択**: 縦並び（本文 → 検索 → 置換 → 結果）
- **根拠**: 競合多数派（9/11 サイト）= T1 期待値合致 / w375 レスポンシブが単純 / タイル 400px 幅で横並びは窮屈
- **詳細ページとの整合**: 詳細ページは検索/置換を横並び 1fr 1fr（w640 以下縦並び）だが、タイル UI は 400px 固定のため常に縦並びで差異あり。T2 likes「操作性一貫」は「縦並びの順序が同じ」レベルで担保。
- **出典**: cycle-210.md L472–490 §論点 3

### オプション省略方針（論点 2 案 A）

- **採択**: 全省略（正規表現 / 大文字小文字 / 全置換トグルをタイル UI には持たない）
- **固定値**: useRegex: false / caseSensitive: true / globalReplace: true（logic.ts デフォルト値と完全一致）
- **緩和策**: 検索フィールドの placeholder に「大文字/小文字を区別」旨を含めて visitor の誤置換リスクを緩和（案 (i) 補助ラベル採択）
- **出典**: cycle-210.md L387–424 §論点 2 / L426–470 §論点 2.1

### コピーボタン

- **存在位置**: 結果欄の下（詳細リンクの手前）/ 既存タイル 10 件と同型
- **表示条件**: 結果が空のときは非表示 or 無効化（観点 (ix) で実証 / Component.tsx L131 同型仕様 `result.output && <button>`）
- **文言変化**: 「コピー」→「コピー済み」→ 2 秒後自動復帰（= AP-P21 適用外。cycle-213 で N=3 確定 SSoT 化済み）
- **出典**: cycle-210.md L240–242 §T-3 観点 (v)(ix) / cycle-213 §補足事項 (β)(θ)

### 詳細リンク

- **位置**: タイル末尾（コピーボタンの後）
- **実装**: `<Link>` コンポーネント（既存タイル 10 件と同型）
- **URL**: `/tools/text-diff`（text-diff の場合）
- **出典**: cycle-210.md L221 §T-3 共通実装事項

### role="status" の付与

- **付与対象**: 結果欄（単一膨張側出力欄のとき count=1）
- **属性**: `role="status" aria-live="polite"`
- **出典**: cycle-210.md L216 §T-3 共通実装事項 / L312 T-4 完成条件
- **cycle-211 での更新**: 膨張側出力欄が複数の場合は **全膨張側出力欄に付与する**（cycle-211 §補足事項 (viii)）。text-diff は差分表示が 1 出力欄なら count=1 で cycle-210 と同型。

### エラー文言

- **100,000 字超過時**: 「入力テキストが長すぎます（最大100,000文字）」（logic.ts の result.error をそのまま表示 / タイル・詳細共通文言）
- **エラー枠高さ SSoT**: h=46.09px / w=376px（cycle-210 T-4 実測値 = SSoT 確定値）
- **注意**: text-diff では字数制限の文言が異なる可能性が高いため、T-4 で独自計測が必要

### タイル用テスト最低件数

- **最低 11 件**（cycle-210 §T-3 テスト観点 (i)〜(xi) / MAJOR-4 r1 対応で 8 → 11 件に拡張）
- **必須観点**:
  - (i) レンダリング（全入力フィールド + 結果欄 + 詳細リンクが DOM に存在）
  - (ii) 入力で結果更新
  - (iii) 検索空時の挙動
  - (iv) 置換空時の削除挙動
  - (v) コピーボタン押下挙動
  - (vi) ARIA / role="status"
  - (vii) 100,000 字超過エラー表示（text-diff では対応する字数制限のエラー）
  - (viii) 詳細ページリンク
  - (ix) 空入力時コピーボタン非表示
  - (x) clipboard 不在時 silent fail
  - (xi) 結果情報（置換件数 or 差分件数等）の表示
- **出典**: cycle-210.md L230–244 §T-3

## D. 論点・判断履歴（text-diff で再登場しそうな項目）

### D-1: タイルサイズの選択（複合入力型タイル固有）

cycle-210 で「3 入力 + 結果欄」を cols=3 rows=3（400×400px）に収めた。text-diff は「2 入力 textarea + 差分表示欄」という構造であり、rows=3 が適切かどうかの再検討が必要。text-replace と比べて入力欄が「3 → 2」に減るため、cols=3 rows=2（264px）でも収まる可能性はあるが、差分表示が縦長になりやすい特性を考慮すると rows=3 継続が有力と考えられる。（出典: cycle-210.md §論点 1）

### D-2: オプション省略の方針（T1 likes 最大化 vs 認知負荷低減のトレードオフ）

text-replace では「案 A 全省略」を採択し、正規表現 / 大文字小文字トグルをタイル UI に持たなかった。text-diff にも「空白無視 / 大文字小文字無視」等のオプションがある可能性があり、同様の論点が再登場する。採択方針は「T1 likes 最大化 + 詳細ページとの責務分離」。（出典: cycle-210.md §論点 2）

### D-3: AP-P21 役割分担の二分類（操作側 vs 膨張側）

text-replace では「操作側 = 検索/置換 input」「膨張側 = 本文 textarea + 結果欄」の二分類。text-diff では「操作側 = 比較モードオプション等（あれば）」「膨張側 = テキスト A textarea + テキスト B textarea + 差分結果欄」の構造となる見込み。差分結果欄が縦に長くなりやすいため、膨張側の設計が重要。（出典: cycle-210.md §論点 4）

### D-4: エラー文言の 1 行収納（計画段階では推定値のみ / T-4 実測で確定）

cycle-210 では 3 回連続事実誤認（R1-CRIT-1 / R2-CRIT-5 / R4-CRIT-6）を踏まえ、「数値 literal は計画段階で推定値のみ記載 / 実 SSoT は T-4 実機計測で確定」方針を確立した。text-diff のエラー文言は text-replace と異なるため、同じく「計画段階では推定値 + T-4 実機計測確定」の方針で進める。（出典: cycle-210.md §論点 5 / §R5 同型事故防止メモ）

### D-5: 大文字小文字区別等の仕様を visitor に伝える補助ラベル（論点 2.1 同型）

cycle-210 論点 2.1 で「オプション全省略採択に伴い、固定仕様（caseSensitive: true 等）を placeholder 等で visitor に伝える」補助ラベル案 (i) を採択した。text-diff でもオプション固定値を採択した場合、同様の補助ラベル設計論点が発生する可能性がある。（出典: cycle-210.md §論点 2.1）
