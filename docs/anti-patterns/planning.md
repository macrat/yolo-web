# 計画段階のアンチパターン

このチェックリストは、計画・調査・戦略策定のレビュー時に参照し、過去の失敗パターンに陥っていないかを確認するためのものです。

- AP-P01: 計画の根幹にある仮定（検索ボリューム、PV推計、改善効果など）を定量的に実測せず、「実装フェーズで検証する」と先送りしていないか？
  → 前提が崩れると計画全体が無意味になる。（cycle-155, 157, 159, 162で実際に発生）

- AP-P02: 自分が選んだ戦略を否定するデータを積極的に探したか？都合の悪いデータを無視または過小評価していないか？
  → 結論を先に決めてから根拠を集めると、戦略の致命的欠陥が見えなくなる。（cycle-157, 162で実際に発生）

- AP-P03: 今あるコンテンツ、今の設計、今ある状態、今のPVなどを所与として計画を立てていないか？将来追加するコンテンツや拡大するPVなど、今後の展開を見据えた計画になっているか？
  → 現状を固定して計画を立てると、今できることの範囲内に閉じた施策しか出てこず、成長の可能性を過小評価する。（cycle-156, 163で実際に発生）

- AP-P04: Ownerの発言をアドバイス（調査のヒント）として受け取らず、自分で調査・検証せずそのまま計画の前提や結論に組み込んでいないか？
  → Ownerの発言は検討の出発点であり、検証なしに採用するとAP-P01と同じ構造の失敗になる。（cycle-159, 161で実際に発生）

- AP-P05: 前サイクルの失敗を受けて正反対の極端な方針を採用し、constitutionに立ち返らずに前回への反射で方針を決めていないか？
  → 二項対立的な振れは前回とは別の新しい失敗を生む。（cycle-146, 158で実際に発生）

- AP-P06: 既存コンテンツ一覧を確認せず新規作成予定との重複を検証しないまま、またはdocs/research/の既存調査レポートを参照せずに計画を立てていないか？
  → 既存コンテンツとの重複は独自価値のない成果物を生み、既存調査の無視は過去の知見と矛盾する計画を生む。（cycle-156, 158, 162で実際に発生）

- AP-P07: 計画の出発点が「来訪者が何を求めているか」ではなく、「運営者が何をしたいか」「実装が容易か」になっていないか？対象ページに誰がどの経路で到達するかを調査したか？
  → 運営者目線の施策は来訪者の行動と乖離しやすく、来訪経路を誤認すると想定と異なるユーザーに向けた設計になる。（cycle-113, 139で実際に発生）

- AP-P08: 「ゼロベースで設計する」をbacklogに明示された対象だけに限定し、ページ構成・情報アーキテクチャ・コンポーネント構造など設計全体に適用せず前サイクルのパターンを無意識に踏襲していないか？
  → ゼロベース検討の対象を限定すると、検討範囲外の部分に前回の誤りがそのまま引き継がれる。（cycle-147, 152, 153で実際に発生）

- AP-P09: 計画のゴールが「higher page views by providing the best value for visitors」に正確に対応せず、「Googleの評価向上」「SEOスコア改善」「ドメイン権威向上」など別のゴールに読み替えていないか？
  → ゴールを読み替えると、来訪者への価値を犠牲にしてSEO最適化を優先するなどconstitutionと逆行する判断が生まれる。（cycle-159で実際に発生）

- AP-P10: 「適合度5/5」「高い確信度」等の高評価が出たとき、過去の実績データと矛盾していないか確認せず根拠なき高評価を無批判に採用していないか？
  → 過去の実績と矛盾する高評価はバイアスの産物であり、そのまま採用するとAP-P01と同じ構造の失敗になる。（cycle-157で実際に発生）

- AP-P11: 「前サイクルでAIがこう決めたから」という理由で変更を回避していないか？AIが決めた色・テキスト・レイアウト等が変更可能であることを認識しているか？
  → AIの過去の判断を「変更不可の制約」として扱うと、より良い選択肢を採用できなくなる。（cycle-151, 152, 153で実際に発生）

- AP-P12: 過去に同種の施策が失敗していた場合、その失敗を具体的に分析したか？「なぜ失敗したか」「今回は何が違うか」を明確にしないまま同じ方向の施策を計画していないか？
  → 失敗分析なしに同種施策を再実行すると、同じ失敗を繰り返す。（cycle-163で実際に発生）
