# 記事執筆のアンチパターン

このチェックリストは、ブログなどの記事のレビュー時に参照し、過去の失敗パターンに陥っていないかを確認するためのものです。

- AP-W01: 記事の主題が「自分たちが何をやったか」の報告になっていないか？
  → 読者はプロジェクトの内部事情に興味がなく、「読者が何を持ち帰れるか」が記事の価値を決める。（cycle-51, 55, 75で実際に発生）

- AP-W02: 「このサイトでこうした」という固有の話に閉じており、読者が自分の状況に応用できない構成になっていないか？
  → 特定サイトの実装詳細だけでは、そのサイトの関係者以外に価値がない。（cycle-148で実際に発生）

- AP-W03: コンテンツの想定読者と、実際にそのコンテンツを検索・閲覧する人がずれていないか？
  → 作り手の専門領域に引きずられると、実際の読者層の大半が理解できないコンテンツになる。（cycle-117, 118で実際に発生）

- AP-W04: 記事内の事実・数値・経緯の記述が、一次ソースで裏付けられていないまま執筆していないか？
  → 記憶や推測に基づいて書くと事実誤認が混入し、修正に複数サイクルを消費する。（cycle-69, 70, 95で実際に発生）

- AP-W05: 執筆前にターゲット読者の定義ファイルを実際に開いて確認せずに執筆していないか？
  → 過去の記憶に依存すると、削除・変更されたターゲット定義に基づいて書いたり現存するターゲットを見落としたりする。（cycle-148で実際に発生）
