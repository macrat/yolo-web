# 実験台帳（仮説→賭け→観測→記録）

> **位置づけ**: オーナー指示（cycle-278）「賭けて、結果を観測し、記録に残すサイクルを確立する」の実装。site-concept の実験枠・判断基準の「評価窓と撤退基準を着手前に決める」を運用する唯一の台帳。**このサイクル自体がサイトの資産である**——勝った賭けはコンテンツを残し、負けた賭けは仕入れの目を鍛える記録を残す。

## 運用規約

1. **事前登録が先、出荷が後**。賭け（新コンテンツ・実験枠・大きな変更）は、出荷前に本台帳へ「仮説・賭けの中身・観測手段・評価窓・撤退基準」を登録する。事後の合理化を防ぐため、登録日と出荷日を分けて記録する。
2. **サイクルとの接続**: kickoff 時に「観測期日が来たエントリ」を確認し、来ていれば読む作業をサイクルに含める。completion 時に新しい賭け・結果の記録を反映する。
3. **観測手段**（組み合わせて使う）:
   - **Search Console**（表示・順位・CTR）: 数値の撤退基準を置ける唯一の側（外部母集団）。ただし反応は約3ヶ月遅れ（内部実測）——評価窓に織り込む。
   - **GA4 行動データ**（開始率・完走率・札保存/共有・回遊）: 小 n・計測欠落を常に疑い、単独の kill 根拠にしない（絞り込みの参考）。
   - **A/B テスト**（`src/lib/ab/` 基盤・インライン実験）: 当サイトの規模では統計的有意を得る手段ではなく、**同時期に来た来訪者へ対照パターンを提示し、一人ひとりの反応を質的に観察する手段**（旧 visitor-value-measurement の cycle-268 訂正ノートの結論を継承）。サイト全面の A/B はしない（判定不能・rebuild-plan §1）。
   - 時間軸の before/after は方向読みのみ（インデックス変動が支配的なため因果を断定しない・双方向ガード）。
4. **記録の誠実さ**: 負けの記録は勝ちと同じ粒度で書く。打ち切りは打ち切りとして残す（黙って消さない）。

## エントリの書式

```
### EXP-NNN: タイトル
- 状態: 計画中 | 走行中 | 観測待ち(期日) | 記録済み | 打ち切り
- 登録日 / 出荷日:
- 仮説（来訪者価値）: 誰が・何を得られるはずか
- 賭けの中身: 何を作る/変えるか
- 観測手段と評価窓: 何で・いつ判定するか
- 撤退基準: 何が起きたら殺すか（数値は SC 側に）
- 結果 / 学び: （事後に記入）
```

### EXP-002: 全面デザイン切替（フェーズ R）の観測

- 状態: 計画中（cycle-279 出荷予定）
- 登録日: 2026-07-11（cycle-278）
- 仮説（来訪者価値）: 「店構え」への切替と文章規範の適用で、来訪者の信頼と体験の質が上がる（安っぽさ・AI 臭の除去）。副次: AdSense 審査通過。
- 賭けの中身: 全ページの一斉切替+撤去（rebuild-plan §2）
- 観測手段と評価窓: SC 週次（カテゴリ別・基線=切替前28日・読み始め+2週・結論+3ヶ月）／GA4（診断開始率・完走率・札保存/共有）／AdSense 再申請の結果。**双方向ガード**: 上がっても下がっても単一原因に断定しない（全面変更+インデックス変動の交絡）。
- 撤退基準: 切替は不可逆の裁定（オーナー決定）のため「戻す」基準は置かない。代わりにトリップワイヤ: 記名クエリ（似てるキャラ診断群）のクリックが切替後4週で基線の50%を割ったら、該当面のタイトル/コピーの意図適合を最優先で個別修復する（**4週時点は早期警報であり方向のみ**——SC の反応遅延約3ヶ月を踏まえ、この時点で成否の断定はしない）。
- 基線の注意: 基線28日には EXP-001（結果面 A/B）の variant 混在が含まれる——方向読みの粗さとして織り込む。
- 結果 / 学び: （事後に記入）

### EXP-003: 旧デザイン効果の観測群の打ち切り（B-554/B-558）

- 状態: 打ち切り（記録のみ）
- 記録: cycle-275（結果リビール勲章）と cycle-277（トップ診断中心化）の効果観測は、観測対象の面がフェーズ R で消えるため打ち切る。学び: **一律適用（A/B 分離なし）で出荷した変更は、次の大変更が来ると効果を永遠に読めなくなる**——実験台帳と事前登録が必要だった理由の実例。B-521（辞書 CTR・2026-07-16 読み）はコンテンツレベルのため、切替前に読めれば読む。
