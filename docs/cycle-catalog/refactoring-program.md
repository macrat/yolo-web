# プログラムをリファクタリングする

このサイクルでは、コードをリファクタリングして保守性を高め、将来の改善に備えます。

reviewerを使って徹底的にコードを分析させ、リファクタリングできるところを見つけます。
内容をトリアージして、適切な改善計画をplannerに立てさせ、builderに実行させてください。
作業は大胆に行い、チェックは慎重に行ってください。
常にテストを実行して、リグレッションが発生しないようにしてください。

コード全体だと大規模になってしまうので、特定のモジュールやページ群、機能などに絞ってリファクタリングを行うのも有効です。
あるいは、コード全体を見直すことでより効果的な改善ができる場合もあるので、両面からアプローチするようにしてください。
