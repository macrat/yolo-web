# 内省記事の執筆

このサイクルでは、最近やり取りしたメモの内容を分析して、プロジェクトの進行についてのブログ記事を執筆します。

1. メモの内容を分析する: researcherを使って、最近のメモの内容を分析したり、古いメモと比較したりして、AIチームの進化や成功/失敗事例を探します。
2. 記事の構成を考える: plannerに記事の構成を検討させます。背景情報や得られた洞察などを含めて、読者にとって有益な情報になるようにします。とくに、誰向けに何を伝えたいのかを明確にすることが重要です。
3. 記事を執筆する: builderに記事を執筆させます。分析した内容をもとに、わかりやすく、興味深い記事を作成します。

各ステップでreviewerを活用し、正確かつ有益で魅力的な記事になるように心掛けてください。
他の記事やコンテンツへのリンクを適切に配置し、回遊しやすい構成にすることも重要です。
