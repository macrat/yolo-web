# yolos.net Design System

> **【コンセプトと再適合（cycle-261 / B-539）】** 本デザインシステムは元々「道具箱-as-core」前提で書かれていたが、cycle-257（B-535）の実測でサイトコンセプトは **「自分を知り、楽しむ。そのための場所」（診断中心）** に再センタリングされ（`docs/site-concept.md`）、cycle-261 で本ドキュメントと `/DESIGN.md`・`.claude/skills/frontend-design/SKILL.md` を一体で診断中心へ再適合した。
>
> - **デザイントークン（色・タイポ・余白・モーション）と UI コンポーネント（パネル・ボタン・タグ等）の規約は引き続き有効**——これらは概念中立な視覚言語の基盤で、診断・文化・道具のいずれにも使える。抑制された上質さは品質の基盤として維持する。
> - そのうえで、**診断のタッチポイント（診断入口・結果面）に限り**、結果固有色・象徴アイコンを主役にした表現を許す（`/DESIGN.md` §7「診断の視覚言語」が単一情報源。本ドキュメントはそれを参照する）。この拡張は診断面に限定し、道具・辞典本文・道具箱の基調には広げない。
> - 確定した視覚言語の方向と実デザイン探索の根拠は `docs/cycles/cycle-261.md`。診断結果面への本番出荷は走行中 A/B `quiz_result_visual_v1`（B-526・2026-07-21目安に結論）と協調する下流タスク（結果面=B-523・トップ=B-542）で行う。

診断（自分を知り楽しむ体験）を主軸に、文化コンテンツ（辞典）と実用的なオンラインツール群を支えるサイトのためのデザインシステム。

## 1. プロダクト概要

**yolos.net** は「自分を知り、楽しむ。そのための場所」。3つの層で構成される（`docs/site-concept.md`）：

- **主軸：自分を知り、楽しむ体験（診断・占い・ゲーム）** — `/play`。性格診断・キャラ診断のクラスタが集客・滞在の両軸で頑健に1位で、新規来訪の主な着地点。サイトの顔。診断の入口・結果面は専用の視覚言語（`/DESIGN.md` §7）で「発見を楽しむ・人に見せたくなる」体験を伝える。
- **支え：文化コンテンツ（辞典・伝統色）** — `/dictionary`。地味だが精読される良コンテンツ。主軸を文化的な厚みで支える。診断結果の伝統色などで主軸と橋渡しされる。
- **道具（オンラインツール・道具箱）** — `/tools`・道具箱。直接来た人・回遊する人に役立つ実用層として残す。サイト全体を道具箱に合わせて曲げることはしない。

すべてのコンテンツは「**パネル**」という統一フォーマットを基本に表現される。シンプルな矩形の中にコンテンツが収まる。

### 個別ツールページの構造（道具層）

1. トップに **名称**
2. その下に大きく表示された **パネル**（道具そのもの）
3. その下に **使い方の説明**

### 道具箱（ダッシュボード・道具層）

- 複数のパネルを好きな順番・サイズで並べる
- 道具箱は複数作成できる（仕事用、勉強用、など）

※ 診断面（`/play`）は道具と同じパネル基盤の上に、§7 の診断専用の視覚言語を重ねる。道具・辞典本文・道具箱の基調は抑制を保つ。

## 2. CONTENT FUNDAMENTALS — 言葉の指針

### トーン

実用的な道具らしく、淡々として丁寧。説明過多にせず、機能と用途だけを置く。装飾的・情緒的な言い回しは避ける。

### 文体

- **常体／です・ます混在** — 見出しや短文は体言止め。説明文は「です・ます」。命令形は使わない。
- **「あなた」「私たち」は使わない** — 主語を消す。
- **絵文字・顔文字は使わない**
- **句読点** — 「、」「。」を使う。半角カンマ・ピリオドは英数字内のみ。
- **数字** — 半角。単位の前にスペースは入れない（例: 5分、3個）。

### よく使うラベル

| 場面 | 言葉           |
| ---- | -------------- |
| 起動 | はじめる       |
| 追加 | 道具箱に入れる |
| 戻る | もどる         |
| 保存 | 保存           |
| 削除 | 削除           |
| 設定 | 設定           |
| 検索 | さがす         |

## 3. VISUAL FOUNDATIONS — 視覚の指針

### 全体の方向性

ニュートラルなオフホワイトを基調に、機能的な配色とミニマルな線画で構成する。基調は装飾を排し情報のヒエラルキーだけで成立させる。診断のタッチポイント（入口・結果面）に限り、この抑制を土台として保ったまま結果固有色・象徴を主役にした表現を重ねる（`/DESIGN.md` §7）。

### 色

基調（道具・辞典本文・道具箱・共通UI）では、色は機能を伝えるためだけに使う。診断のタッチポイント（入口・結果面）に限り、結果を象徴する固有色（可能なら最近傍の伝統色）を主役の色として使ってよい——単一情報源は `/DESIGN.md` §2「色の使用ルール」＋§7「診断の視覚言語」。結果固有色・伝統色は面/象徴のアクセントに使い、テキストには WCAG AA を満たす濃色版を使う（多くの伝統色はそのままでは白文字・テキストでコントラスト不足）。

#### Surface（5階調）

- `--bg` `#FFFFFF` — デフォルトのパネル面
- `--bg-soft` `#F4F4F1` — 沈んだ背景／hover on bg
- `--bg-softer` `#E6E6E2` — ghost ボタンの hover、ヘアライン罫
- `--bg-invert` `#1A1A1A` — primary ボタン、強い罫線
- `--bg-invert-soft` `#555555` — primary の hover

#### Text（5階調）

- `--fg` `#1A1A1A` — 本文
- `--fg-soft` `#555555` — 補助
- `--fg-softer` `#888888` — caption / disabled
- `--fg-invert` `#FFFFFF` — 暗い面の上のテキスト
- `--fg-invert-soft` `#F4F4F1` — 暗い面の上の補助テキスト

#### Border

- `--border` = `var(--bg-softer)` — デフォルトのヘアライン
- `--border-strong` = `var(--bg-invert)` — 強い罫線

罫線色は独立した値を持たず、surface パレットのエイリアスとして定義される。

#### Accent & Status

4色とも OKLCH 明度 L=0.62 / 彩度フルで定義。色相は **カラーユニバーサルデザイン推奨配色セット ver.4.2 (CUD)** に準拠。

| トークン    | 色相       | 用途                             |
| ----------- | ---------- | -------------------------------- |
| `--accent`  | 264° (青)  | リンク、フォーカス、操作可能要素 |
| `--success` | 162° (緑)  | 完了                             |
| `--warning` | 80° (琥珀) | 準備中・注意                     |
| `--danger`  | 35° (朱)   | エラー・停止                     |

各色には `-strong` (L 0.50)・base (L 0.62)・`-soft` (L 0.93, chroma 0.04) の3段が定義されている。共通の操作で生成しているため階調が揃う。

> 1画面内に同時に出すアクセント色は1つを基本とする。

詳細は `colors_and_type.css` を参照。

### タイポグラフィ

- **メイン**：Noto Sans JP — 本文・UI・見出しすべて
- **モノ**：Noto Sans Mono — 数値、コード的表示、トークン名
- **サイズ**：12 / 13 / 14 / 15 / 16 / 18 / 20 / 24 / 32 / 44 / 60 px
- **行送り**：日本語向けに広め（1.7 がデフォルト）
- **大文字化しない** — 日本語の文脈で `text-transform: uppercase` は使わない

### コーナー半径（角丸）

**2 値のみ**：

- `--r-normal` `2px` — **原則すべて**（パネル、カード、タグ、チップ、モーダル等）
- `--r-interactive` `8px` — **操作可能要素のみ**（ボタン、入力欄、セレクト、テキストエリア）

それ以外の値は使わない。

### スペーシング

8px グリッド：4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96。余白は惜しまず取る。

- 本文の最大幅：720px
- 道具箱の最大幅：1200–1440px

### Elevation（影）

**3レベルのみ**：

- `--elev-0` — デフォルト。すべての要素はここから始まる（影なし）
- `--elev-button` (= `--elev-1`) — ボタンの通常時のみ
- `--elev-dragging` (= `--elev-2`) — ドラッグ中の要素のみ

ダイアログ・ポップオーバー・カード等にも影は使わない。階層は枠線とスペースで表現する。

### ホバー / プレス状態

| 種類    | 通常               | hover              |
| ------- | ------------------ | ------------------ |
| default | `--bg` (白)        | `--bg-soft`        |
| primary | `--bg-invert` (墨) | `--bg-invert-soft` |
| ghost   | `--bg-soft`        | `--bg-softer`      |

すべて hover 前後の **明度差は ΔL ≈ 0.035 で揃える**（surface の3階調が同じ間隔で並んでいるため）。border 色は hover で変えない。透明度や scale もいじらない。

### フォーカス

`outline: 2px solid var(--accent); outline-offset: 2px;` — 藍色の見えるアウトライン。

### モーション

- **基本ease**：`cubic-bezier(0.2, 0.6, 0.2, 1)`
- **時間**：120ms (fast) / 200ms (base) / 320ms (slow)
- **fade と short slide のみ**。bounce / spring / 大きな移動は使わない

### レイアウトルール

- **左揃え** が基本。中央揃えはタイトルや空状態など限定的に
- ヘッダーは固定しない（ページに溶け込ませる）
- フッターは細い罫線一本＋小さなテキストのみ

## 4. ICONOGRAPHY

### 方針

**アイコンは最小限。** 文字（ラベル）が第一言語。アイコンを並べてビジュアルを作らない。

### 採用するアイコンセット

**[Lucide](https://lucide.dev/)**。

- ストロークウェイト 1.5px
- 線画ベース、square caps
- サイズは **16px / 20px / 24px** の3サイズに制限
- 色は `currentColor` で文字色に追従
- アイコン単独のボタンには**必ず aria-label** を付ける

### 絵文字

UIの装飾・機能ラベル（ボタン・メタ情報・ナビ等）には**使わない**（Lucide 線画に統一）。**例外は診断結果の象徴アイコンのみ**——診断結果データが持つ象徴アイコン（`icon`）を結果の「勲章」として主役に使ってよい（`/DESIGN.md` §3・§7）。メタ情報や一般UIに絵文字を散らさない。

### ロゴ

`assets/yolos-logo.svg` — `yolos.net` のサンセリフ表記、`.` を `--accent`（青）にハイライト。mark 版は `y.` で同様に `.` をアクセント化。
