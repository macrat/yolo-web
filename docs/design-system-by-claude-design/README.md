# yolos.net Design System

実用的なオンラインツール群と、それを並べる道具箱（ダッシュボード）のためのデザインシステム。

## 1. プロダクト概要

**yolos.net** は、日々使えるオンラインツールを集めたサイト。

中心となる体験は3つ：

- **ツール（道具）** — サイトのコア。よく考えられた単機能ツール群。
- **道具箱（ダッシュボード）** — 気に入ったツールを一画面に並べておける場所。複数作成可能。
- **息抜きコンテンツ** — 占い、パズル、クイズなど。全体の1割以下。**ツールと同じ見た目で並ぶ**（明示的な区別はしない）。

すべてのコンテンツは「**パネル**」という統一フォーマットで表現される。シンプルな矩形の中にあらゆる道具が収まる。

### 個別ツールページの構造

1. トップに **名称**
2. その下に大きく表示された **パネル**（道具そのもの）
3. その下に **使い方の説明**

### 道具箱（ダッシュボード）

- 複数のパネルを好きな順番・サイズで並べる
- 道具箱は複数作成できる（仕事用、勉強用、など）
- ツールと息抜きは見た目で区別しない

## 2. CONTENT FUNDAMENTALS — 言葉の指針

### トーン

実用的な道具らしく、淡々として丁寧。説明過多にせず、機能と用途だけを置く。装飾的・情緒的な言い回しは避ける。

### 文体

- **常体／です・ます混在** — 見出しや短文は体言止め。説明文は「です・ます」。命令形は使わない。
- **「あなた」「私たち」は使わない** — 主語を消す。
- **絵文字・顔文字は使わない**
- **句読点** — 「、」「。」を使う。半角カンマ・ピリオドは英数字内のみ。
- **数字** — 半角。単位の前にスペースは入れない（例: 5分、3個）。

### よく使うラベル

| 場面 | 言葉           |
| ---- | -------------- |
| 起動 | はじめる       |
| 追加 | 道具箱に入れる |
| 戻る | もどる         |
| 保存 | 保存           |
| 削除 | 削除           |
| 設定 | 設定           |
| 検索 | さがす         |

## 3. VISUAL FOUNDATIONS — 視覚の指針

### 全体の方向性

ニュートラルなオフホワイトを基調に、機能的な配色とミニマルな線画で構成する。装飾はなく、情報のヒエラルキーだけで成立させる。

### 色

実務的なツールであることを最優先。色は機能を伝えるためだけに使う。

#### Surface（5階調）

- `--bg` `#FFFFFF` — デフォルトのパネル面
- `--bg-soft` `#F4F4F1` — 沈んだ背景／hover on bg
- `--bg-softer` `#E6E6E2` — ghost ボタンの hover、ヘアライン罫
- `--bg-invert` `#1A1A1A` — primary ボタン、強い罫線
- `--bg-invert-soft` `#555555` — primary の hover

#### Text（5階調）

- `--fg` `#1A1A1A` — 本文
- `--fg-soft` `#555555` — 補助
- `--fg-softer` `#888888` — caption / disabled
- `--fg-invert` `#FFFFFF` — 暗い面の上のテキスト
- `--fg-invert-soft` `#F4F4F1` — 暗い面の上の補助テキスト

#### Border

- `--border` = `var(--bg-softer)` — デフォルトのヘアライン
- `--border-strong` = `var(--bg-invert)` — 強い罫線

罫線色は独立した値を持たず、surface パレットのエイリアスとして定義される。

#### Accent & Status

4色とも OKLCH 明度 L=0.62 / 彩度フルで定義。色相は **カラーユニバーサルデザイン推奨配色セット ver.4.2 (CUD)** に準拠。

| トークン    | 色相       | 用途                             |
| ----------- | ---------- | -------------------------------- |
| `--accent`  | 264° (青)  | リンク、フォーカス、操作可能要素 |
| `--success` | 162° (緑)  | 完了                             |
| `--warning` | 80° (琥珀) | 準備中・注意                     |
| `--danger`  | 35° (朱)   | エラー・停止                     |

各色には `-strong` (L 0.50)・base (L 0.62)・`-soft` (L 0.93, chroma 0.04) の3段が定義されている。共通の操作で生成しているため階調が揃う。

> 1画面内に同時に出すアクセント色は1つを基本とする。

詳細は `colors_and_type.css` を参照。

### タイポグラフィ

- **メイン**：Noto Sans JP — 本文・UI・見出しすべて
- **モノ**：Noto Sans Mono — 数値、コード的表示、トークン名
- **サイズ**：12 / 13 / 14 / 15 / 16 / 18 / 20 / 24 / 32 / 44 / 60 px
- **行送り**：日本語向けに広め（1.7 がデフォルト）
- **大文字化しない** — 日本語の文脈で `text-transform: uppercase` は使わない

### コーナー半径（角丸）

**2 値のみ**：

- `--r-normal` `2px` — **原則すべて**（パネル、カード、タグ、チップ、モーダル等）
- `--r-interactive` `8px` — **操作可能要素のみ**（ボタン、入力欄、セレクト、テキストエリア）

それ以外の値は使わない。

### スペーシング

8px グリッド：4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96。余白は惜しまず取る。

- 本文の最大幅：720px
- 道具箱の最大幅：1200–1440px

### Elevation（影）

**3レベルのみ**：

- `--elev-0` — デフォルト。すべての要素はここから始まる（影なし）
- `--elev-button` (= `--elev-1`) — ボタンの通常時のみ
- `--elev-dragging` (= `--elev-2`) — ドラッグ中の要素のみ

ダイアログ・ポップオーバー・カード等にも影は使わない。階層は枠線とスペースで表現する。

### ホバー / プレス状態

| 種類    | 通常               | hover              |
| ------- | ------------------ | ------------------ |
| default | `--bg` (白)        | `--bg-soft`        |
| primary | `--bg-invert` (墨) | `--bg-invert-soft` |
| ghost   | `--bg-soft`        | `--bg-softer`      |

すべて hover 前後の **明度差は ΔL ≈ 0.035 で揃える**（surface の3階調が同じ間隔で並んでいるため）。border 色は hover で変えない。透明度や scale もいじらない。

### フォーカス

`outline: 2px solid var(--accent); outline-offset: 2px;` — 藍色の見えるアウトライン。

### モーション

- **基本ease**：`cubic-bezier(0.2, 0.6, 0.2, 1)`
- **時間**：120ms (fast) / 200ms (base) / 320ms (slow)
- **fade と short slide のみ**。bounce / spring / 大きな移動は使わない

### レイアウトルール

- **左揃え** が基本。中央揃えはタイトルや空状態など限定的に
- ヘッダーは固定しない（ページに溶け込ませる）
- フッターは細い罫線一本＋小さなテキストのみ

## 4. ICONOGRAPHY

### 方針

**アイコンは最小限。** 文字（ラベル）が第一言語。アイコンを並べてビジュアルを作らない。

### 採用するアイコンセット

**[Lucide](https://lucide.dev/)**。

- ストロークウェイト 1.5px
- 線画ベース、square caps
- サイズは **16px / 20px / 24px** の3サイズに制限
- 色は `currentColor` で文字色に追従
- アイコン単独のボタンには**必ず aria-label** を付ける

### 絵文字

**使わない。** 例外なし。

### ロゴ

`assets/yolos-logo.svg` — `yolos.net` のサンセリフ表記、`.` を `--accent`（青）にハイライト。mark 版は `y.` で同様に `.` をアクセント化。
