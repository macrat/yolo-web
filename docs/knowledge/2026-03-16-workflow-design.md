# ワークフロー設計の教訓

cycle-95（2026-03-16）で得られた知見。

## ワークフロー設計の教訓

- **詳細定義+臨機応変 vs シンプル定義+厳格ワークフローの対比**: 詳細なドキュメントとエージェント定義で複雑な仕組みを説明し、ワークフローを臨機応変に運用する方式は生産性は高いがルール逸脱が多く成果物の品質が安定しにくい。一方、ドキュメントを削ぎ落としてシンプルにし、定義されたワークフロー（スキル）を厳格に踏ませる方式はルール遵守率が上がり品質を担保しやすい一方で、1タスクあたりの手順が増えてコンテキスト消費が増大する。どちらが優れているかではなく、プロジェクトの目標（品質重視か速度重視か）に応じて使い分けることが重要。

- **各フェーズのメリット・デメリットを客観的に比較することの重要性**: ワークフローの変遷をレビューや記録として残す際、ある時代や方式をネガティブまたはポジティブな一面だけで評価しない。各方式には必ずトレードオフがある。たとえば第1期は「生産性が高いが品質が低い」、第2期は「品質が高いがコンテキスト消費が大きい」のように、それぞれの特徴をフラットに記述することで、なぜ次の方式に移行したかの理由が自然に導かれる。ネガティブな見出しや一方的な評価はこのトレードオフを隠してしまう。

- **段階的移行 vs 一括移行の判断**: AIを使ったワークフローの変更において、2種類の方式を共存させる中間状態は機能しにくい。AIは複数の選択肢を提示されると意図通りに使い分けられず、廃止すべき古い方式を使い続けてしまうことがある。後方互換性を捨てて一気にドラスティックな変更をする方が、結果として少ない工数で完結することが多い。段階的移行を選ぶ場合は、中間状態を厳密に設計する必要があり、かえって手間がかかる。

- **Ownerが手動で作業する判断基準**: ドキュメント更新のような作業を AI に任せると、特定のケースにしか適用できない記述や冗長な説明が入り込むことがある。「意図通りに確実な更新をしたい」「変更内容が軽微で手作業の方が速い」という条件が重なる場合は、Ownerが手動で実施する方が品質と意図の正確性を保ちやすい。

- **チェックリスト方式 vs 指示文方式の比較**: ワークフローやルールを指示文・ドキュメントの構造として定義するよりも、チェックリストを渡して1つ1つチェックを入れさせる方式の方が、AIは確実に動作する。同様に、過去の履歴を振り返らせて指示への遵守を自己点検させるよりも、チェックリストのすべての項目にチェックが入っているかを確認させる方が安定的に機能する。複雑な手順の遵守を期待するよりも、シンプルで汎用性の高い1つの仕組みにした方がよい。

- **第2期でのワークフロー遵守率の変化に関する注意点**: 厳格なワークフローに移行することで、ワークフロー全体に対するルール逸脱は顕著に減少する。しかし、長期間同じワークフローを動作させるようになった分だけ、別のルール（例: メモ管理に関するルール）から逸脱するという問題が顕在化しやすくなる。ルールを改善するアプローチはあまり効果的ではなく、根本的な仕組みを変えることが必要になることがある。
