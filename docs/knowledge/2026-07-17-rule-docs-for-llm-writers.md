# LLMが適用するルール文書の書き方に関する知見

2026-07-17、認知リズム規範（`docs/cognitive-rhythm-writing.md`）を執筆ルールへ取り込む作業で得られた知見。

- **ルール文書の例外を「ただし、〜は別物」という抽象条件だけで書くと、LLMは適用時に判定できず混乱する。** 禁止ルールに例外を設けるとき、「回収されるなら可」のような抽象的な条件文だけを添えても、生成中のLLMは自分が書いた文がどちら側かを判定できない。例外の線引きには、可の例文と不可の例文を必ず対で併記し、両者を分ける特徴が例文の中で目に見える状態にする。対は同じ表面形を共有する最小対にする。表面形が違う対（可が「〜に違いない」、不可が「〜ではないでしょうか」など）は、判定軸ではなく語尾の違いを教えてしまう。同じ線引きを複数のドキュメントが共有する場合は、同じ例文を使うと判定基準のブレを防げる（適用例: `docs/japanese-ai-slop.md` と `docs/cognitive-rhythm-writing.md` の縫合箇所）。

- **例外条項は、原則と同じ場面・同じ表面形に実際に重なるときにだけ書く。** 重ならない話を例外として持ち込むと、原則と例外が別のシチュエーションを語る条項になり、かえって読み手を混乱させる。実例: 曖昧語尾（「〜ではないでしょうか」）の禁止に対する例外として、禁止形にそもそも該当しない断定口調の例文（「〜に違いない」）を挙げていた。禁止形に当たらないものは例外にする必要がない。例外を書く前に「その例は原則の禁止形に本当に該当するか」を確認する。
