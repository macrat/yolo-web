# インタラクティブコンテンツ一覧ページ UI/UXベストプラクティス調査レポート

- 調査日時: 2026-03-18
- 調査目的: /play一覧ページのUI/UX改善のためのベストプラクティス調査（カテゴリ分け、カードUI、モバイル表示、回遊性向上の観点）
- 情報源: Nielsen Norman Group、eleken.co、UX Collective、U-Site (NN/G日本版)、Baymard Institute、aufaitux、Design4Users

---

## 1. カテゴリ分けのUXベストプラクティス

### 現在の分類名の評価

「占い」「性格診断」「知識テスト」「ゲーム」という名称は、コンテンツの種類を正確に伝える機能的なラベルとして妥当。ただし、来訪者の行動動機（好奇心・楽しさ・自己理解）と直接結びつく言葉ではないため、「正確だが感情を喚起しない」という特性がある。

### カテゴリ名の改善方向性

NN/Groupの推奨によれば、ラベルはユーザーの語彙・動機に合致する言葉を使うべきとされている。

| 現在の名称 | 機能的だが冷たい             | 感情・動機に寄り添う代替案             |
| ---------- | ---------------------------- | -------------------------------------- |
| 占い       | 情報として正確               | 「今日の運勢」「今日を占う」           |
| 性格診断   | やや教科書的                 | 「あなたはどのタイプ？」「自分を知る」 |
| 知識テスト | テスト感が強く敬遠されやすい | 「知識を試す」「どこまで知ってる？」   |
| ゲーム     | 明確だが広すぎる             | 「毎日のパズル」「今日のゲーム」       |

日替わりコンテンツという独自性があるため、「今日の〇〇」系のフレームが有効。ただし全カテゴリをそう呼ぶと単調になるため、2カテゴリ程度に限定して差別化するのが現実的。

### カテゴリの表示順序

BaymardおよびKuizyの調査結果から、以下の基準が有効。

- **訪問頻度が高いもの（デイリー系コンテンツ）を先頭に置く**: 占い・パズルは毎日来訪動機になるため最上位が適切
- **新規訪問者が最初に試しやすいものを前半に置く**: 軽く試せる診断系は初回体験として向いている
- **最も重い体験（知識テスト・長いゲーム）は後半**: 最初のハードルを下げる設計

推奨順序: デイリー系（占い）→ 即試しやすい（性格診断）→ 頭を使う（知識テスト）→ 繰り返し楽しめる（ゲーム）

## 2. カードUIのベストプラクティス

### サムネイル画像がない場合の工夫

UX CollectiveおよびEleken.coの調査によれば、画像がない場合に有効なアプローチは以下の通り。

**タイポグラフィによる補完**

- タイトルを大きく・太く設定してビジュアルの役割を持たせる
- カード上部にアクセントカラーの帯（color block）を設け、視覚的な差別化を図る

**カラーブロックによる識別**

- カテゴリごとに異なる背景色をカード上部に適用する（Googleカレンダーや Notion のカバー機能に近い発想）
- これにより画像がなくても「どのカテゴリに属するか」が即座に視覚識別できる

**情報の密度を抑える**

- 「Less is more」原則: カードには最大で「アイコン・タイトル・1行説明・CTA」の4要素に限定
- 100文字以内のテキストが「読む気にさせる」上限とされている（eleken.co）

### 絵文字アイコンを効果的に使うパターン

- **サイズ**: 2remは小さい。3〜4rem（48〜64px相当）に拡大することで、アイコンがカードのビジュアル的な主役になる
- **配置**: カード上部の色帯の中央に白地で絵文字を大きく表示する構成が視覚的インパクトを持つ
- **一貫性**: 1カードにつき絵文字は1つに限定。複数使うとノイズになる
- **文字との関係**: 絵文字の直下にタイトル、さらに下に説明文という縦の流れが視線の動きに合致する

### カードに表示すべき情報の優先順位

NN/GroupおよびBaymardのカードUX研究から、情報の優先順位は以下の通り。

1. ビジュアル要素（絵文字 or カラーヘッダー）— スキャン時の識別に使われる
2. タイトル（太字・大きめ）— コンテンツを理解する最重要テキスト
3. 所要時間または問数（グレーの小テキスト）— クリック前の意思決定を支援
4. コンテンツタイプバッジ（「診断」「クイズ」「ゲーム」）— 種類の即時識別
5. 1〜2行の説明文 — ある方が良いが省略可能
6. CTAボタン — カード自体がクリッカブルなら省略も検討可

「毎日更新」「デイリー」バッジは、リピーター獲得のため運勢・パズル系カードには必須で表示することが推奨される。

## 3. モバイルでのコンテンツ一覧表示

### 19種のアイテムをモバイルで効率的に表示する方法

u-site.jpの研究によると、サブカテゴリ数が6〜15個の場合は「セクションメニュー」が適切とされている。19種という数は上限に近く、以下のアプローチが有効。

**2カラムグリッドの維持**

- 375px幅で2カラムを維持するには、カード幅を約165〜170pxに設計する
- カードコンパクト化の条件: 説明文を非表示 or 1行に、アイコンを上部に固定
- Yahoo!ゲームはこのパターンをモバイルでも採用しており実績がある

**「上からすべて並べる」の回避**

- 全19種を1カラムで縦並びにすると、スクロール量がページの3倍以上になる
- 2カラム化すれば同じコンテンツ数でスクロール量が約50%削減できる

### カテゴリナビゲーションのパターン

| パターン                      | 向いているケース                 | 注意点                                       |
| ----------------------------- | -------------------------------- | -------------------------------------------- |
| 横スクロールタブ（pill tabs） | カテゴリ数が4〜7個               | 全タブが見えないと存在に気づかれにくい       |
| スティッキーカテゴリバー      | スクロール量が多いページ         | 実装コストが高く、モバイルの表示領域を狭める |
| ページ内アンカーリンク        | カテゴリが3〜4個かつページが縦長 | スクロール先まで遠いと迷子になりやすい       |
| 折りたたみアコーディオン      | カテゴリ数が多い場合             | 初回訪問者はコンテンツの存在に気づきにくい   |

4カテゴリという構成では、「横スクロールタブ（全タブが画面内に収まるサイズ）」がもっともシンプルで効果的。

## 4. 回遊性を高めるUIパターン

### 「おすすめ」「人気」セクションの効果

NN/Groupの推奨コンテンツガイドライン調査によれば、おすすめセクションは以下の設計が有効。

**配置**: ページ上部（カテゴリセクションより前）に置くことで最も目に入る。後半に埋めると無視される。

**分類の分離**: 「みんながやっている（人気）」「毎日更新（デイリー）」「初めての人へ（入門）」などを明確に分けると、それぞれに異なる動機を持つユーザーに刺さる。

**ラベルの明確化**: 「おすすめ」という言葉だけでは信頼されにくい。「今日更新」「今月最も遊ばれた」のように根拠を示す言葉の方がクリック率が高い。

**カード数**: おすすめセクションは3〜6枚程度に絞る。多すぎると「おすすめ」の意味が薄れる。

### デイリー更新コンテンツの目立たせ方

**バッジ方式（軽量）**

- カードに「今日の」「毎日更新」「本日限り」などのバッジを貼る
- 色は赤・オレンジ等の高彩度色が注目を集める（ただし多用すると麻痺する）

**ヒーロースポット（高インパクト）**

- ページ最上部に「今日のコンテンツ」として1〜2枚を大きく表示する（Vonvonの手法）
- 日次更新であることをタイトルや日付で明示することで「今日も来る理由」を強調できる

**日付/曜日の表示**

- 「本日（3/18月）の運勢」のように具体的な日付を入れることで、コンテンツが実際に更新されていることをユーザーが信頼できる

## 5. 統合推奨事項

### 優先度A（UX改善に直結）

1. **モバイル2カラム化**: 640px以下を1カラムから2カラムに変更。カードをコンパクト化（説明文はモバイルでは非表示）
2. **「今日のおすすめ」セクションをカテゴリ前に追加**: 3枚程度を選んでページ上部に表示
3. **絵文字の大型化とカードビジュアルの強化**: 絵文字を3〜4rem程度に拡大し、カード上部にカテゴリ別カラーブロックを設ける

### 優先度B（回遊性の向上）

4. **デイリーバッジの導入**: 運勢・パズル系カードに「毎日更新」バッジを貼る
5. **カテゴリタブをモバイルで追加**: 4カテゴリをpill tab形式でページ上部に表示

### 優先度C（長期的改善）

6. **カテゴリ名の感情的表現への変更**: 占い→「今日の運勢」、ゲーム→「毎日のパズル」
7. **所要時間表示の全コンテンツ展開**: 問数のみの表示を「約3分」「毎日更新」等の時間感覚に翻訳
