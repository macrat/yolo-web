# character-personality 判定の検証スクリプト（cycle-284）

`../2026-07-16-character-personality-scoring-analysis.md` の解析を再現するための計器一式。
**`tmp/` は保証の無い場所なので、ここへ恒久化した**（cycle-284 の唯一の成果物であり、
消えると cycle-285 が 4^12 検証と 48 重み設計をゼロからやり直すことになるため）。

| ファイル                 | 役割                                                                        |
| ------------------------ | --------------------------------------------------------------------------- |
| `archetype-weights.json` | **設計した 48 選択肢のアーキタイプ重み**（受け入れ基準を通過・未出荷）      |
| `aw_common.py`           | 判定規則（アーキタイプ得点 → 主軸/副軸 → タイプ・閾値と逆順フォールバック） |
| `aw_ideal.py`            | **理想回答者テスト（∀ 版・これが正典）**。分割モデルの実装                  |
| `aw_exhaustive.py`       | 新設計の全数検査（4^12・meet-in-the-middle）                                |
| `exhaustive.py`          | **現行**判定の全数検査（同点 36.20%・偏り 20倍 を出したもの）               |
| `ideal.py`               | **現行**判定の理想回答者テスト（2/24 失敗を出したもの）                     |

## 実行方法

```bash
cd docs/research/character-personality-scoring-cycle284
python3 aw_ideal.py        # 新設計の理想回答者テスト(∀版・数秒)
python3 aw_exhaustive.py   # 新設計の全数検査(4^12・数十秒)
python3 ideal.py           # 現行判定の理想回答者テスト -> 2/24 失敗を再現
python3 exhaustive.py      # 現行判定の全数検査 -> 同点36.20%・20倍を再現
```

- 依存: `numpy` のみ。
- 設問・タイプの定義は `src/play/quiz/data/` から**リポジトリルート相対で解決**する（どのディレクトリから実行してもよい）。
- **`tmp/` に依存しない**ことを、`tmp/` が存在しない環境へコピーして実行し確認済み（cycle-284）。

**未同梱**: 29.63%／68倍（＝未決着を一律「未決着」と数える保守的定義）と、∃版の罠の実測（ゴミ設計が通ること）を出すコードは同梱していない。これらは解析文書 §3・§4.1 に**数値と定義だけを記載**しており、コードからは再現できない。再現が要るなら書き起こすこと。

## 使う前に必ず読むこと

**計器そのものを疑うこと。** cycle-284 の PM は自作の検証スクリプトに
「同点を配列の若い順で選ぶ」バグ——**そのサイクルの主題そのもの**——を仕込んだまま、
合格している設計を棄却した。

- **理想回答者テストは ∀ 版でなければ無効。** ∃ 版（「そういう分割が一つでも存在すれば合格」）では
  TH=3〜12 が全部 24/24 を通り、**重みをランダム化したゴミ設計すら通る**。
- **新しい計器を書いたら、既知のゴミ設計に対して検定すること。** 通ってしまうならその計器は無効。
- 詳細は解析文書の §4.1・§4.4。
