方法: 実サイト yolos.net を来訪者として接地。(1) 単独結果ページ（第三者向けシェアランディング）を take-screenshot で撮影、(2) Playwright で実際に12問を遊び、本人向けインライン結果（ResultCard）に到達して全ページ撮影。スクリーンショットは `tmp/screenshots/`（非コミット）。

## 接地のレンズ（確立知見・実体確認済み）

- cycle-137 line 33: シェアの動機は「自己開示の報酬性」。「自分のことを言い当てられた感覚」＝「これ自分だ!」体験がシェアを促す（Harvard PNAS 2012）。
- cycle-136 line 81 / cycle-142 line 58,92: キャラ口調の「あるある（behaviors）」は共感とシェア動機を生む最重要コンテンツ。
- cycle-142 line 36: 来訪者の多数派は友人のシェアリンクから来る。
- ターゲット `自分に似たキャラや自分の性格を知りたい人.yaml`: likes=「当たってる」「新しい一面」の手応え・読み物として面白い独自性・自己紹介ラベルになり SNS に貼れる。

要するに、結果体験が「これ自分だ!」を強く生むほど、満足・シェア・再訪につながる。本接地は、その自己認識の山場がどこで生まれ/冷めているかを指差すために行った。

## 2つの面（コード上の事実）

`src/app/(new)/play/[slug]/result/[resultId]/` 冒頭コメントが明示する通り、結果体験には2面ある:

- **本人向けインライン結果**（`src/play/quiz/_components/ResultCard.tsx`）: 診断を遊んだ本人が `/play/<slug>` 上で見る。description は**全文表示**（切り詰めなし）。「これ自分だ!→見せたい」が最も効く面。
- **第三者向け単独結果ページ**（`/play/<slug>/result/<id>`）: 友人のシェアリンク等から来た、まだ診断していない第三者が見る。description は `DescriptionExpander` で4行クランプ＋「続きを読む」。

## 指差せる発見

### Finding B（本サイクルで是正）: 「このキャラの成り立ち」に内部の英語元型名が露出している

本人向けインライン結果で、description（あなたへの一人称メッセージ）の直後に出る「このキャラの成り立ち」カード（`CharacterPersonalityContent.tsx:218` が `archetypeBreakdown` を直接描画）の本文が、来訪者に見せる日本語コピーの中で内部分類の英語名をそのまま使っている。

実例（炎の詩人 blazing-poet の成り立ち本文・実画面で確認）:

> 「**commander**の爆発的推進力に**dreamer**の柔らかい感性が融合した形。…」

- 英語元型名（commander / professor / dreamer / trickster / guardian / artist）は `character-personality.ts:20` のコードコメントにのみ定義される**内部分類**で、FAQ 等を含めユーザーには一切説明されない。
- 全24結果の archetypeBreakdown に体系的に及ぶ。是正前の値を git diff で全件確認した結果、**24/24 すべての archetypeBreakdown 本文が英語元型名を含んでいた**（易しい一例の一般化ではなく全件の構造的欠陥）。なお grep の素の行ヒット数は batch1/2/3 で 46/43/33 件あるが、これはコメント・id スラッグ・color/scoring キー等を含む全行数で、archetypeBreakdown プロースの件数（24件）とは別である。
- 来訪者は自分について読んでいる流れの真ん中で、意味の分からない英語ジャーゴンに当たる。「これ自分だ!」の没入が冷める。constitution Rule 4（あらゆる面で最高の品質）に対する明確な品質欠陥であり、読者の誰が見ても「平易な自己認識の日本語の方が良い」と同意できる自明な改善対象。
- 制約: batch テストが archetypeBreakdown を 80〜150 字に制約（`character-personality-results-batch1.test.ts:71-76`）。書き直しもこの範囲を守る。

### Finding A（本サイクル非対象・既存 B-544 / B-526 へ）: 結果到達のリビール（山場）が無い

12問を答え終えても、結果は診断トップの静的な説明文の下に静かにインライン描画されるだけで、到達感・お祝い・視覚的な山場が無い。2分投資した来訪者の感情のピーク（リビールの瞬間）が平板。

これは emotional value としては Finding B より大きいが、(1) 視覚的クライマックス化は設計方向への賭けで cycle-268〜271 が4度失敗した領域、(2) 既に B-544（§7 フルクライマックス化）と B-526（インライン A/B `quiz_result_visual_v1` の月次読み）で、A/B の枠組みを経て進める前提で管理されている。本サイクルの「読者として自明な改善のみ出荷」方針の外にあるため本サイクルでは触れず、その前提を再確認した事実として記録する。

### Finding C（本サイクル非対象・carryover）: 第三者向け単独ページが description を切り詰める

友人のシェアリンクから来た第三者（来訪者の多数派ルート）が着地する単独結果ページで、description が4行で切り詰められ「続きを読む」になる。第三者は自分の話を読んでいるわけではないため「これ自分だ!」への直接の影響は本人向けより小さく、また「ティーザー＋診断 CTA」という変換設計としての一定の妥当性もある。価値優先（来訪者に結果を最後まで楽しんでもらう）の観点では再考余地があるが、本サイクルの自明改善の枠では Finding B より優先度が低い。次サイクル以降で判断する。

## 本サイクルの結論

Finding B を是正する。全24結果の archetypeBreakdown を、英語元型名を除き、来訪者自身について平易な日本語で語る自己認識の文に書き直す（「二つの面が同時に働く」という洞察は保持し、各結果のトーンに合わせ、80〜150字を守る）。Finding A・C は本ドキュメントに記録し、A は既存 B-544/B-526 の前提として、C は次サイクル判断として申し送る。
