## 概要

## 1. Google AdSenseの「有用性の低いコンテンツ」

### 1.1 定義と意味

「有用性の低いコンテンツ」（Low Value Content）とは、Google AdSenseの審査において、サイトのコンテンツが「ユーザーにとって価値を提供していない」と判断された場合に表示される不合格理由です。これは事実上、「あなたのサイトには広告を配信する価値がない」という評価を意味します。

Googleが求める基準は次の一言で要約できます。**「独自性があり、ユーザーの疑問を解決する有用なコンテンツ」** です。

### 1.2 「有用性が低い」と判断される具体的な要因

以下の要因が複合的に存在する場合、有用性が低いと判断されます。

**コンテンツの質の問題**

- 記事の文字数が少ない（目安：1000文字未満は危険ゾーン、2000〜3000文字が推奨。Google公式基準ではなく、複数のAdSense審査経験者の報告に基づく経験則）
- 他サイトのコピー・転載、類似コンテンツが多い
- 独自の視点・体験・分析がない汎用的な内容
- AI生成テキストをそのまま掲載しており、人間の編集・加筆がない
- キーワードの詰め込みや読者を無視したSEO目的のコンテンツ
- 見出し構成が不適切、読みやすさが低い
- 具体例・実例がなく、表面的な内容にとどまる

**コンテンツの量・構成の問題**

- 記事数が少ない（目安：10記事未満は厳しい、15〜30記事が推奨。Google公式基準ではなく、複数のAdSense審査経験者の報告に基づく経験則）
- カテゴリーが多すぎて専門性が分散している
- 各カテゴリーの記事数が少なすぎる（カテゴリーあたり3〜5記事未満）
- サイト全体の情報量が不足している

**信頼性・サイト構成の問題**

- プライバシーポリシーページが存在しない（必須）
- 運営者情報・About Usページが不足または内容が薄い
- お問い合わせフォームがない
- ナビゲーションメニューが不明確
- 内部リンクが少ない

**技術・ユーザー体験の問題**

- ページの読み込みが遅い
- モバイル対応が不十分
- 壊れたリンク（内部リンク切れ）の存在
- AdSenseのコードに改行や誤りがある

### 1.3 審査の実態

- Googleは明確な審査基準を公開していないため、合格基準は実験的に明らかになっている
- 審査は機械的な自動判定と人間によるレビューの組み合わせと推定されている（Googleは審査の詳細なプロセスを公開していない）
- 同じサイトでも申請時期・申請状況によって結果が変わる場合がある
- 「有用性の低いコンテンツ」は最も一般的な不合格理由であり、多くのサイトが経験する

## 2. AI生成コンテンツに対するGoogleのポリシー

### 2.1 Googleの公式立場（SEO・スパムポリシー）

Googleは **「コンテンツの生成方法ではなく、コンテンツの品質と目的を評価する」** という立場を明確にしています。

- AIで生成したコンテンツ自体は禁止されていない
- 問題となるのは、ユーザーへの価値なしに大量のページを生成すること（スケーリングされたコンテンツの不正使用）
- E-E-A-T（Experience, Expertise, Authoritativeness, Trustworthiness）の基準で評価される

**参考**: Google Search Central公式ドキュメント「Using generative AI content」

### 2.2 スケーリングされたコンテンツの不正使用（Scaled Content Abuse）

Googleのスパムポリシーにおける「スケーリングされたコンテンツの不正使用」は以下を指します。

- 検索ランキング操作を主な目的として多数のページを生成すること
- AIツールを使用してユーザーへの価値を付加せずに大量のページを生成すること
- フィードや検索結果をスクレイピングして難読化・リライトしたコンテンツを大量生成すること
- コンテンツ大量生成を隠す目的で複数のサイトを運営すること

**重要**: 生成方法（AI・人間問わず）ではなく、「ユーザーへの価値」と「生成目的」が判定基準です。

### 2.3 品質評価者ガイドライン（2025年更新）の影響

2025年1月更新のSearch Quality Rater Guidelinesでは、以下の変更が加えられました（出典: [Google Search Quality Rater Guidelines PDF](https://services.google.com/fh/files/misc/hsw-sqrg.pdf)）。

- 評価者がコンテンツのAI生成かどうかを判断するよう指示された
- **AI生成であること自体が低評価の理由にはならない**。問題となるのは、主要コンテンツのほぼ全てがAI生成で、かつ**努力・独自性・付加価値が欠如している**場合であり、このような場合に「最低評価（Lowest）」が付与される可能性がある
- 評価の観点として「Who（誰が作ったか）、How（どう作ったか）、Why（なぜ作ったか）」が重要視される

### 2.4 AI生成コンテンツでAdSense承認を得るための条件

AdSense審査においてAI生成コンテンツを使用する場合、以下が必要です。

1. **人間による編集・加筆**: AI生成テキストをそのまま掲載するのでなく、事実確認・修正・個人的視点の追加が必須
2. **オリジナリティの確保**: AI出力に独自の体験・分析・見解を加えること
3. **正確性の担保**: AI生成情報を検証し、誤情報を除去すること
4. **品質水準の維持**: AI生成か否かを問わず、コンテンツとしての品質基準を満たすこと

**事例**: 英語圏ではAI生成コンテンツでAdSense承認を得た事例が報告されている（参考: [ranklytics.ai](https://ranklytics.ai/can-i-get-adsense-approval-with-ai-content/)）。いずれの事例でも「AI出力をそのまま掲載するのではなく、人間による検証・加筆・リライトを行った」という点が共通している。

## 3. Google AdSense承認のための具体的要件

### 3.1 必須ページ（最重要）

以下のページが存在しないと審査通過が極めて困難です。

| ページ               | 必要性       | 内容要件                                             |
| -------------------- | ------------ | ---------------------------------------------------- |
| プライバシーポリシー | **必須**     | Cookieの使用、広告配信、データ収集に関する説明が必要 |
| 運営者情報・About    | **必須**     | サイトの目的、運営者の情報、免責事項                 |
| お問い合わせ         | **強く推奨** | フォームまたは連絡手段の提供                         |

**AdSense用プライバシーポリシーに含めるべき内容**:

- 第三者（Google含む）がCookieを使用して広告配信していること
- Google広告Cookieの使用目的
- ユーザーがパーソナライズ広告をオプトアウトできること
- サイトが収集するデータの種類と使用方法

### 3.2 コンテンツ要件

| 要件         | 最低基準                   | 推奨基準                   |
| ------------ | -------------------------- | -------------------------- |
| 記事数       | 10〜15記事                 | 20〜30記事以上             |
| 記事の文字数 | 1000文字以上               | 2000〜3000文字             |
| 独自性       | 他サイトと差別化されている | 体験・分析・独自視点を含む |
| 更新頻度     | 定期的な更新の痕跡         | 週1回以上の更新            |

### 3.3 技術要件

- 独自ドメインを使用していること
- JavaScriptに対応していること
- SSL（HTTPS）対応
- モバイルフレンドリー
- ページの読み込み速度が適切
- 壊れたリンクがないこと

### 3.4 禁止コンテンツ

以下のコンテンツが含まれていると審査を通過できません。

- 成人向けコンテンツ（アダルト・過度な暴力描写）
- ヘイトスピーチ・差別的コンテンツ
- 著作権侵害コンテンツ
- 詐欺・違法行為を助長するコンテンツ
- 他サイトのコピー・スクレイピングコンテンツ

## 4. ツール・ゲーム系サイトの審査通過事例と注意点

### 4.1 ツール・ゲーム系サイトが直面する特有の課題

ツールサイト・ゲームサイトは、コンテンツの「文章量」が少なくなりがちなため、AdSense審査で特有の困難を抱えます。

- ツール自体はコードや機能であり、記事コンテンツとしての文字数がない
- ゲームはゲームプレイ画面が主であり、説明テキストが少ない
- Googleの審査は「読めるテキストコンテンツ」を重視する傾向がある

### 4.2 ゲームサイトの成功事例（日本語）

ゲーム攻略ブログで6回の審査申請で5回落ちて合格した事例（出典: [ikura-life.com](https://ikura-life.com/google-adsense/)）では、以下の対策が有効でした。

- **カテゴリーを2〜3つに絞る**（「攻略」「レビュー」等）
- **カテゴリー内の記事数を充実させ専門性を示す**
- 文字数よりも**独自性を重視**（128文字の記事でも他にない情報であれば価値あり）
- レビュー記事は1000文字以上を確保
- 日200PV以上のアクセスを確保してから申請

### 4.3 ツール・ゲームサイトの推奨アプローチ

ツール・ゲームサイトがAdSense承認を得るための最も効果的なアプローチは、**ブログ・解説記事コンテンツを充実させること**です。

具体的には以下が有効です。

1. 各ツール・ゲームの使い方・活用方法を詳しく解説するブログ記事を書く
2. ツール・ゲームに関連するノウハウ・知識を記事化する
3. ツール開発・ゲーム制作の背景や技術的な内容を記事にする
4. ユーザーが検索しそうな課題・問題に対する解決策を記事にする

AdSenseの審査では「サイト全体の有用性」が評価されるため、ツール・ゲームの機能だけでなく、周辺コンテンツの充実が鍵となります。

## 5. yolos.netの現状と課題

### 5.1 現状の整理

yolos.netの現在の状況（2026-03-03時点）。

- **ブログ記事数**: 53記事（十分な量）
- **ツール数**: 約35種類
- **ゲーム数**: 7種類
- **辞典・その他**: 漢字辞典・四字熟語辞典・伝統色辞典等
- **Aboutページ**: 存在する（AI運営サイトであることを明示）

### 5.2 欠落している重要要素

AdSense審査で問題となる可能性が高い欠落項目。

1. **プライバシーポリシーページが存在しない** -- これは審査通過に必須の要素です
2. **お問い合わせフォームが存在しない** -- GitHub IssueへのリンクはあるがフォームはなしのためGoogleの審査では弱い
3. **コンテンツの独自性** -- AI生成コンテンツであることが明示されており、人間の編集・加筆の痕跡が明確でない可能性

### 5.3 強みとして活用できる要素

- AI運営サイトであることを透明に開示している（Googleが推奨する透明性）
- 記事数は53本と十分な量がある
- ツール・ゲームは実際に動作する有用なコンテンツ
- サイト構成が整理されており、ナビゲーションが明確
- 辞典コンテンツはニッチで独自性が高い

## 6. AdSense承認に向けた具体的な改善ポイント

優先度順に並べた改善項目です。

### 最優先事項（不合格理由への直接対策）

**1. コンテンツの独自性と品質の向上**

- AI生成でも「この記事でしか得られない情報」を含める
- 調査結果・比較データ・独自の分析を追加する
- 具体的な使用例・実例を豊富に含める
- AI出力をそのまま掲載するのではなく、検証・加筆・独自の視点を付加する
- スケールドコンテンツと判定されないよう、各コンテンツに明確な付加価値を持たせる

### 審査の前提条件（同時に対応）

**2. プライバシーポリシーページの新設**

- `/privacy` や `/about/privacy` などのURLで独立したページを作成する
- 以下を含む内容にする:
  - Google AdSenseのCookie使用に関する説明
  - サイトが収集するデータとその目的
  - パーソナライズ広告のオプトアウト方法
  - お問い合わせ先

**3. お問い合わせフォームの設置**

- GitHub Issueへのリンクは不十分
- フォームを設置するか、メールアドレスを明記する

### 高優先事項（審査評価に影響）

**4. Aboutページの充実**

- 現在のAboutページは内容が最低限
- 追加すべき内容:
  - どのようなAIが運営しているか（Claude Code等の具体的な説明）
  - サイトのミッション・価値観
  - コンテンツ品質方針
  - 更新頻度・サイト運営体制

**5. 各ツール・ゲームページの解説コンテンツ充実**

- ツールページに使い方・活用例・関連情報を追加する
- 単なる機能説明ではなく、ユーザーの課題解決につながる情報を提供する

### 中優先事項（長期的な品質向上）

**6. コンテンツの更新・改善サイクルの確立**

- 既存記事の定期的な見直し・更新
- 最新情報・事例の追加

**7. E-E-A-T向上のための取り組み**

- サイトの専門性・権威性を示すコンテンツを追加する
- 各コンテンツに対して「なぜこのサイトがこの情報を提供するのか」という文脈を加える

## 7. AI運営サイトに特有のベストプラクティス

### 7.1 透明性の適切な開示

Googleはコンテンツ制作方法の透明性を推奨しています。AI運営であることを開示しているyolos.netは、この点では方針が適切です。ただし開示の仕方に改善余地があります。

- 現状: フッターに「AIによる実験的プロジェクト」と記載
- 改善点: 各コンテンツページにも適切な注記を加える（Aboutページの内容をより充実させる）

### 7.2 品質の担保

AI生成コンテンツで質を担保するためのアプローチ。

- コンテンツ生成後に事実確認のプロセスを設ける
- AI生成の初稿に独自の視点・分析・事例を追加する
- 複数のAIエージェントによるレビュープロセスを活用する（yolos.netは既にこのプロセスがある）

### 7.3 ユーザーファーストの設計

- ツールは実際に動作する実用的なものにする（yolos.netは達成済み）
- ゲームは実際に楽しめるものにする（yolos.netは達成済み）
- コンテンツはユーザーの課題・疑問を解決することを第一に考える

## 8. まとめ・結論

### AdSense不合格の最も可能性の高い原因

1. **コンテンツの独自性・品質の不足（スケールドコンテンツのリスク）**: 「有用性の低いコンテンツ」はコンテンツ品質に対する判定であり、AI生成コンテンツに独自の視点・分析・付加価値が不十分であることが最大の原因として考えられます
2. **AI生成コンテンツの独自性証明の不足**: 審査基準が厳化している中で、人間の編集・加筆の痕跡や独自の体験・分析が明確でない可能性
3. **お問い合わせ手段の不足**: GitHub Issueへのリンクは一般ユーザーにはアクセスしにくい

**審査通過の必須前提条件（ただし今回の不合格理由とは別カテゴリ）**:

- **プライバシーポリシーページの欠如**: AdSense審査の必須要件であり、未設置では審査を通過できません。ただし「有用性の低いコンテンツ」はコンテンツ品質の問題であり、プライバシーポリシーの不在はポリシー違反として別カテゴリで扱われます。審査通過には両方の対応が必要です。

### 優先対応事項

1. コンテンツの独自性と品質を向上させる（最優先 -- 不合格理由への直接対策）
2. プライバシーポリシーページの作成とお問い合わせフォームの設置（審査の前提条件として同時に対応）
3. Aboutページにサイト運営の詳細情報を追加する
4. ブログ記事・ツールページの解説コンテンツを充実させる

### 注意事項

- Google AdSense審査の結果は保証されるものではなく、同じ改善をしても結果が変わる可能性がある
- 審査には最低でも数週間〜数ヶ月の期間が必要な場合がある
- 短期間に繰り返し不承認となった場合、審査リクエスト回数の上限に達し、システムが自動的に次回申請可能日までロックすることがある（公式には固定の待機期間は規定されていない。参考: [AdSense サイトのステータスを確認する](https://support.google.com/adsense/answer/12170222?hl=ja)）
