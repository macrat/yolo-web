# 文字列置換ツール 競合 UI 調査レポート

## 調査概要

cycle-210 において yolos.net は `text-replace` ツールをダッシュボードのタイル UI（400×264px）と詳細ページ（フル機能）の二段構えで提供する。本レポートは、タイル UI に載せる「最小機能セット」の判断材料として、国内外 11 サイトの実物 UI を Playwright で確認・分析した結果をまとめる。

スクリーンショット保存先: `docs/research/cycle-210-competitor-text-replace-screenshots/`

## 1. 入力フィールドの配置

### フィールド順序パターン（全 11 サイト）

| パターン                             | サイト数 | 代表サイト                                                        |
| ------------------------------------ | -------- | ----------------------------------------------------------------- |
| 検索 → 置換 → 本文 → 結果（縦順）    | 5        | ラッコ手帳、LMN、UpdateYouSet                                     |
| 本文 → 検索 → 置換 → 結果（縦順）    | 3        | TAG index（入出力兼用）、MoreOnlineTools、TextMechanic            |
| 本文 + 検索群（上下二分割）          | 2        | webw0rk（本文左/結果右 + 検索ペア群下段）、俺のテキスト置換ツール |
| 複数検索ペア群 → 本文 → 結果（縦順） | 1        | ベンリスト、Toolpods                                              |

### フィールドサイズ感

- **検索・置換フィールド**: 全サイト共通で 1行 input（`<input type="text">`）。textarea は 1 サイトも使用していない。
- **本文フィールド**: textarea が標準。行数は 5〜12 行が多く、ラッコ手帳は約 5 行、LMN は可変高、MoreOnlineTools は 8 行程度。
- **結果フィールド**: readOnly textarea が多数。TAG index のみ入出力を同一 textarea にしていてユニーク。

### 横並び vs 縦並び

- **検索・置換の横並び配置**: MoreOnlineTools のみ「Find / Replace With」を左右横並びに配置（横 2 列 input）。他の全サイトは縦並び。
- **本文・結果の左右分割**: webw0rk のみ本文（左）と結果（右）を左右 50% に分割。他は縦並び一本道。

## 2. オプション提供状況

### 機能提供率（11 サイト中）

| オプション                 | 提供サイト数 | 提供率 | 備考                                                                                                                                    |
| -------------------------- | ------------ | ------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| 全置換（デフォルト ON）    | 11           | 100%   | 全サイトが全置換をデフォルト動作として提供                                                                                              |
| 大文字小文字を区別         | 7            | 64%    | ラッコ手帳は仕様として「区別する」固定、TAG index は任意置換モードでのみ提供                                                            |
| 正規表現モード             | 5            | 45%    | TextMechanic・MoreOnlineTools・UpdateYouSet・LMN（暗黙的 regex）・webw0rk なし                                                          |
| 最初の 1 件のみ            | 3            | 27%    | TextMechanic・MoreOnlineTools（Replace / Replace All の 2 ボタン）・LMN なし                                                            |
| 全角・半角変換プリセット   | 2            | 18%    | TAG index のみ充実、Toolpods で一部                                                                                                     |
| 複数ペア（バッチ置換）     | 4            | 36%    | ちょっと便利なツール（4 ペア）・webw0rk（4 ペア＋追加可）・ベンリスト（6 ペア）・Toolpods（複数）・俺のテキスト置換ツール（改行区切り） |
| 単語境界一致（Whole Word） | 1            | 9%     | MoreOnlineTools のみ                                                                                                                    |

### オプションの UI 種別

- **チェックボックス**: ラッコ手帳（大文字小文字注記のみ）、LMN（なし）、TextMechanic（Global matching / Case sensitive / Enable regex）、MoreOnlineTools（Case Sensitive / Whole Word / Regex Mode）、UpdateYouSet（正規表現を使用する / 大文字小文字を区別する）
- **ラジオボタン**: TAG index（プリセット変換 vs 任意文字置換）、TextMechanic（Unix/Dos 改行）
- **セグメントコントロール / トグルボタン**: MoreOnlineTools が最も洗練（チェックボックスだが視覚的にトグル的）
- **ボタン pair（Replace / Replace All）**: MoreOnlineTools、TextMechanic（このパターンのみで「最初の 1 件」オプションを実現）

### デフォルト動作

- **全置換 ON がデフォルト**: 11 サイト全て。「最初の 1 件のみ」をデフォルトとするサイトは 0。
- **大文字小文字区別デフォルト**: ON（区別する）が多数。ラッコ手帳は区別する固定。TextMechanic はデフォルト区別なし。
- **正規表現デフォルト**: OFF がデフォルト（全サイト共通）。LMN のみ「正規表現を使用しているため（）.-?$の記号は前に \ をつける必要があります」という注記があり、内部的に常に regex を使用している。

## 3. 結果表示

### 結果欄の配置

| 配置パターン                    | サイト数 | 代表サイト                                               |
| ------------------------------- | -------- | -------------------------------------------------------- |
| 別欄 readOnly textarea（下）    | 8        | ラッコ手帳、LMN、MoreOnlineTools、UpdateYouSet、Toolpods |
| 入力と同じ textarea に差替え    | 1        | TAG index（入出力兼用、「更新する」ボタン押下で差替え）  |
| 結果を div/p に表示（テキスト） | 1        | 俺のテキスト置換ツール（ハイライト機能あり）             |
| 左右分割（左:入力, 右:結果）    | 1        | webw0rk                                                  |

### 置換件数の表示

- **表示あり**: MoreOnlineTools（Matches Found / Replaced をダッシュボード形式で 4 指標表示）
- **暗黙的**: LMN、UpdateYouSet（ハイライト表示）
- **表示なし**: ラッコ手帳、TAG index、ベンリスト、webw0rk、俺のテキスト置換ツール、Toolpods

yolos.net の既存 Component.tsx はすでに「N件置換しました」というステータス表示を持っており、競合よりも先進的。

### リアルタイム反映 vs ボタン押下式

| 方式                           | サイト数 | 代表サイト                                                                                                    |
| ------------------------------ | -------- | ------------------------------------------------------------------------------------------------------------- |
| リアルタイム反映（ボタンなし） | 5        | MoreOnlineTools（マッチ数のみリアルタイム）、UpdateYouSet（ハイライトリアルタイム）、LMN は「置換」ボタンあり |
| ボタン押下式                   | 7        | ラッコ手帳、LMN、TAG index、ベンリスト、webw0rk、Toolpods、TextMechanic                                       |

yolos.net の既存 Component.tsx は useMemo でリアルタイム反映済み（競合より優れた設計）。

## 4. 正規表現エラー時の UX

### エラー表示のある/なし

- **エラー表示あり**: TextMechanic（regex エラー時に結果欄が空になる）、MoreOnlineTools（「Invalid regex pattern」をリアルタイムでフォーム下に表示）
- **注記のみ**: LMN（「（）.-?$の記号は前に \ が必要」という静的注記）、UpdateYouSet（同様の注記）
- **エラー対応なし（regex 非対応）**: ラッコ手帳、TAG index、ベンリスト、webw0rk、俺のテキスト置換ツール、Toolpods

### エラー表示位置・タイミング

- **MoreOnlineTools**: リアルタイムでフォーム直下に赤いアラートボックス。結果欄には「Invalid pattern」などのメッセージを表示（元テキストは保持）。最も洗練されたエラー UX。
- **TextMechanic**: ボタン押下時のみ。エラー表示は alert() かブラウザのデフォルト（確認できず）。

yolos.net の既存 Component.tsx は result.error を role="alert" で表示するパターンが実装済み。

## 7. プリセット機能

### 概要

プリセット（典型ユースケースのクイック入力）を提供しているサイト:

1. **TAG index**: プリセット変換ドロップダウンが最も充実。25 種類以上の変換パターンを提供。
   - 全角英数字 → 半角英数字、全角記号 → 半角記号、半角カナ → 全角カナ
   - 全角スペース → 削除、タブ → 半角スペース × 2、タブ → 削除 など

2. **Toolpods**: 追加オプションとして以下を提供。
   - 「アルファベットをカタカナに変換」（チェックボックス）
   - 「記号を削除」（チェックボックス）

3. **LMN Apps**: プリセットボタンはないが、使い方解説の表形式で典型ユースケースを提示。
   - 「改行を削除する（置換前: \n / 置換後: 空欄）」
   - 「特定の文字列の位置で改行する（例: 。→ 。\n）」

4. **俺のテキスト置換ツール**: アルファベット大文字小文字変換、括弧矯正、ハイフン統一等の専用タブを複数持つ。

5. **MoreOnlineTools**: Usage Examples として 3 種のプリセット例を提供（ボタンクリックでフォームに自動入力）。
   - Simple Text Replace
   - Regex Pattern Matching
   - Whole Word Replace

### 改行関連プリセット

改行操作（\n の置換）は全サイトで「使い方」「応用例」として解説されており、visitor の主要ニーズであることが確認される。PDF コピペ後の余分な改行除去は最も頻繁に挙げられるユースケース。

## 8. MoreOnlineTools 詳細分析（最も洗練された競合）

MoreOnlineTools の Find and Replace が全競合中で最も機能的・UX 的に優れていた。特徴を詳細にまとめる。

### レイアウト（デスクトップ）

- **Source Text**: 大きな textarea（約 8 行）がトップ
- **Find / Replace With**: 横 2 列の input（唯一の横並び採用サイト）
- **オプション 3 つ**: Case Sensitive / Whole Word / Regex Mode をチェックボックス（横並び）
- **ボタン 4 つ**: Replace / Replace All / Clear / Copy Result（横並び）
- **プレビュー**: 「Preview (matches highlighted)」エリア（リアルタイムでマッチ箇所をハイライト）
- **統計パネル**: Matches Found / Replaced / Original Length / Result Length を 4 グリッドで表示

### UX 上の強み

- リアルタイムマッチハイライト（キー入力と同期）
- Replace（最初の 1 件）と Replace All の 2 段階
- 統計情報の即時表示
- Whole Word オプション（他サイトにない）
- 「All processing happens in your browser」のプライバシー表示

### UX 上の弱点

- 情報密度が高く、初見 visitor には情報過多
- Source Text が最上部で「まず何を入れるか」が分かりにくい

## 10. 最小機能セット候補の提案

タイル UI の制約: **400×264px の固定枠**、yolos.net のデザイントークン使用、inline style で実装（CSS Module 不使用）、LineBreakRemoverTile.tsx のパターンを踏襲。

### 案 A: 超ミニマム（検索 + 置換 + 本文）

**構成**:

- 本文 textarea（rows=2）
- 検索文字列 input（1行）
- 置換文字列 input（1行）
- 結果表示 div（flex: 1）
- コピーボタン + 詳細ページリンク
- オプション: なし（全置換 ON 固定、大文字小文字区別 ON 固定）

**想定動作**: 入力と同時にリアルタイム置換（debounce なし）。

**メリット**:

- 最もシンプルで 400×264px に余裕を持って収まる
- 学習コストゼロ
- 日本語テキストの単純な単語置換ユースケースに完全対応
- 全競合サイトの「最低共通」機能セットをカバー

**デメリット**:

- 大文字小文字区別ができない（英語文書での誤置換リスク）
- 正規表現が使えないためパワーユーザーに詳細ページ誘導が必要
- 全置換固定のため「最初の 1 件のみ」が使えない

**想定 visitor 体験**: PDF コピペ後の不要な文字列除去など単純作業なら 1 タイルで完結。満足度高い。英語混じりのテキストで区別が必要な場合は「あれ、全部変わってしまった」が起きる可能性あり。

---

### 案 B: バランス型（案 A + 大文字小文字トグル）

**構成**:

- 本文 textarea（rows=2）
- 検索文字列 input（1行）
- 置換文字列 input（1行）
- オプション: 「大文字小文字を区別」トグルボタン（デフォルト ON）
- 結果表示 div（flex: 1）
- コピーボタン + 詳細ページリンク

**メリット**:

- 最も誤操作しやすい「大文字小文字の誤置換」を防げる
- トグル 1 つなので UI 複雑度の上昇は最小限
- 日本語・英語混在テキストの両方に対応

**デメリット**:

- 正規表現なし・全置換固定は案 A と同じ
- 264px の高さ内に 5 要素が入るが、本文 textarea の行数が 2 行に制限されやや窮屈

**想定 visitor 体験**: 日常的な置換タスクの 90% をカバー。「大文字小文字」オプションの存在で信頼感が上がる。

---

### 案 C: 充実型（案 B + 正規表現トグル + 全置換トグル）

**構成**:

- 本文 textarea（rows=2）
- 検索文字列 input（1行）
- 置換文字列 input（1行）
- オプション 3 つ（横並び）: 正規表現 / 大文字小文字を区別 / すべて置換
- 置換件数表示（「N件置換」ラベル）
- 結果表示 div（flex: 1、縮小）
- コピーボタン + 詳細ページリンク

**メリット**:

- 既存 Component.tsx の機能セットをタイルに凝縮
- 開発者・ライター・パワーユーザー全層をカバー
- MoreOnlineTools と同等の機能をコンパクトに提供

**デメリット**:

- 264px の高さに 7 要素（textarea 2 + input 2 + オプション行 + カウント + コピー行）が入るため、各 textarea が非常に小さくなる（rows=1 相当）
- 正規表現エラー時のエラー表示領域をどこに置くか設計が難しい
- 初見 visitor には情報過多で「どこから始めればいいか」の迷いが生じる可能性

**想定 visitor 体験**: 機能的な満足度は高いが、タイル内が密集して視覚的に重い。タイルとしての「瞬時に使える」感が弱くなるリスクあり。正規表現エラー時の表示領域確保も課題。

---

### 案 D: 異質型（複数ペア特化 / 最小 2 ペア）

**構成**:

- 検索 1 → 置換 1（input ペア）
- 検索 2 → 置換 2（input ペア）
- 本文 textarea（rows=2）
- 実行ボタン（リアルタイムではなくボタン押下式）
- 結果 div（flex: 1）
- コピーボタン

**メリット**:

- ベンリスト・webw0rk のように「複数ペア同時置換」という差別化ができる
- PDF コピペ後の「複数の不要文字列を一括削除」というユースケースに対応

**デメリット**:

- ボタン押下式が必要でリアルタイムの快適さが失われる
- 競合（ベンリスト等）とのバッティングが強くなる
- 1 ペアの使い方として自然ではなく、初見でペア入力の意図が伝わりにくい

**想定 visitor 体験**: 複数パターンを一括処理したいヘビーユーザーには刺さるが、シンプルな単語置換だけしたい visitor には複雑に感じる。

---

## 11. 推奨案と根拠

### 推奨: 案 B（バランス型）

**根拠**:

1. **400×264px の物理制約**: 案 C は情報密度が高すぎてタイルとしての快適さが失われる。案 B は本文 textarea rows=2 + 検索/置換 input × 2 + オプション 1 行 + 結果 div + コピー行の構成で、LineBreakRemoverTile.tsx（264px 確認済み）と同等の余白感を実現できる。

2. **visitor の主目的**: 競合 11 サイトの分析から、最も多いユースケースは「テキスト中の特定の文字列を別の文字列に全置換する」という単純作業。正規表現は詳細ページで十分に対応できる。

3. **差別化: リアルタイム反映**: 競合の 6 サイトがボタン押下式のなか、yolos.net はリアルタイム反映という優位性がある。案 B ではこの優位性を最大活用できる。

4. **大文字小文字オプションの必要性**: 64% の競合が大文字小文字区別オプションを提供しており、visitor の期待値として「区別できること」を前提とするサイトが多い。案 A では全置換時に英文字の誤置換が起きるリスクがある。

5. **ラッコ手帳との比較**: 競合 1 位と思われるラッコ手帳は「置換する」ボタン + 大文字小文字固定（区別あり固定）のシンプル構成。yolos.net 案 B は「リアルタイム + 大文字小文字トグル」でラッコ手帳より一歩優れた UX を提供できる。

### 正規表現の扱い方針

正規表現はタイル UI に含めず、詳細ページに任せる。理由:

- 正規表現の提供率は 45% で、使う visitor は限定的
- 正規表現エラー時のエラー表示領域が 264px 内に収まらない
- 詳細ページへの誘導ポイント（「正規表現を使いたい場合は詳細ページへ」）として機能させる

## 付記: スクリーンショット撮影時に気づいた細部

1. **ラッコ手帳（01）**: サイドバーなしの 1 カラム構成。置換前/置換後 input → 本文 textarea の順で、先に「何を置換するか」を設定してから本文を貼り付ける設計。初見 visitor には「まず検索文字列を入れる」という設計意図が伝わりやすい可能性。

2. **TAG index（04）**: プリセット変換ドロップダウンが「全角英数字 → 半角英数字」等 25 種類あり、全角・半角変換ニーズへの対応が充実。ただし、任意置換モードに切り替えると radio button で UI が切り替わる 2 段階設計で初見ではわかりにくい。

3. **MoreOnlineTools（07）**: Replace / Replace All の 2 ボタン + リアルタイムハイライト + 統計パネルの組み合わせが最も完成度が高い。ただし、スクリーンショット確認時点でページ内に 5 件のコンソールエラーが発生していた（外部 iframe の広告関連と推測）。

4. **LMN Apps（03）**: 「正規表現を使用しているため（）.-?$の記号は前に \ をつける必要があります」という警告が常時表示される設計。実装コストを下げるために内部で常に regex を使用している。visitor には混乱を招くことがある。

5. **俺のテキスト置換ツール（12）**: 「置換前文字列,置換後文字列」をカンマ区切りで改行入力するという独自フォーマットを採用。API 的な入力方式で慣れが必要だが、大量のペアを高速入力できるパワーユーザー向け設計。

6. **webw0rk（11）**: 本文（左）/ 結果（右）の左右分割 + 複数ペア（下段）という独特のレイアウト。シンプルかつ効率的だが、スマホでは崩れる可能性が高い。

7. **TextMechanic（10）**: ファイルアップロード・ダウンロード機能は競合中で唯一の機能。大量テキストの処理ニーズに応える設計。Unix/Dos 改行コードの切り替えも開発者向け機能として提供。
