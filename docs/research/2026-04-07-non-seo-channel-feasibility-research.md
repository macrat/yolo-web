# yolos.net向け非SEOチャネル実現可能性調査

## 調査前提・制約条件の確認

本調査の対象サイトyolos.netの現状：

- 月間約224セッション / 601PV（調査時点）
- トラフィック構成：Direct 113セッション、Organic Search 93セッション、Organic Social 2セッション
- ドメイン年齢：約2ヶ月（初回コミット 2026-02-13）
- 運営形態：Claude Codeによる自律AI運営（人間によるSNS操作なし）
- フォロワー数：X（Twitter）フォロワーゼロ
- コンテンツ種別：診断、辞典、ツール、ブログ

## 1. Google Discover掲載の実現可能性

### 1-1. 掲載要件と技術条件

Google公式ドキュメント（developers.google.com/search/docs/appearance/google-discover）によれば、Discoverへの掲載に必要な条件は以下のとおりである。

**必須条件（技術）：**

- コンテンツがGoogleにインデックス登録されていること
- Discoverのコンテンツポリシーを満たすこと
- アイキャッチ画像の幅が1200px以上であること（`max-image-preview:large` metaタグの設定も必要）
- モバイルフレンドリーであること

**コンテンツ品質条件：**

- E-E-A-T（経験・専門性・権威性・信頼性）を満たす独自コンテンツ
- クリックベイトではないタイトル
- 一次情報または独自の分析・視点を含む記事
- 公開から24時間以内の記事が優先されやすい傾向

**2026年2月のDiscoverコアアップデートの影響：**
2026年2月5日、Googleは初のDiscover専用コアアップデートをリリースした。主な変更点は3点である。

1. **地域関連性の強化**：ユーザーの居住国・言語圏のサイトを優先。日本語サイトにとっては追い風。
2. **クリックベイト削減**：誇張表現や煽りタイトルの掲載頻度が低下。
3. **専門性と一次情報の評価強化**：他サイトのまとめではなく、独自調査・独自視点を持つコンテンツが優遇。

### 1-2. AI生成コンテンツのDiscover掲載可否

Googleの公式見解（developers.google.com/search/docs/fundamentals/using-gen-ai-content）では、「AIによって生成されたかどうかにかかわらず、コンテンツはユーザーのために作成され、価値を提供することが必要」とされており、AI生成コンテンツを明示的に禁止するポリシーは存在しない。ただし以下が条件となる。

- 大量生成で独自価値のないページは「スパムポリシー違反」のリスクがある
- AI生成であることをユーザーに通知することが推奨される（yolos.netはConstitutionで既に対応済み）

### 1-3. 小規模サイト・新規ドメインの掲載実態

2024年8月のGoogleコアアップデート以降、「小規模サイトや個人運営サイトを含む高品質なサイトをユーザーと結びつけることを目指す」との方針が示された。ただし以下の障壁がある。

- **ドメイン年齢の問題**：新規ドメインがSearch結果に安定表示されるまで最低3〜6ヶ月かかるとされており、Discoverも同様の傾向がある
- **2024年コアアップデートの打撃**：2024年3月のコアアップデートで個人ブログのDiscover流入が激減し、その後も回復途上のサイトが多い
- **2023〜2024年の傾向**：個人ブログがDiscoverから除外されるケースが増加したが、2024年末〜2025年にかけて回復傾向が報告されている

### 1-4. yolos.netコンテンツタイプ別のDiscover適性

| コンテンツタイプ | Discover適性 | 理由                                                               |
| ---------------- | ------------ | ------------------------------------------------------------------ |
| ブログ記事       | 中           | 鮮度・独自性があれば掲載対象。ただし継続的な新記事投稿が必要       |
| 診断コンテンツ   | 低           | 静的ページとして更新されにくく、Discoverの「鮮度優先」に不利       |
| 辞典・用語集     | 低           | 常設ページはDiscoverに不向き                                       |
| デイリーゲーム   | 低〜中       | 毎日更新される「今日の問題」ページをインデックスできれば可能性あり |

### 1-5. 実現可能性評価

**実現可能性：低（現時点）→ 中（6ヶ月後）**

**yolos.netでの実現可否：** 現時点では困難。理由は以下のとおり。

- ドメイン年齢が2ヶ月と短く、Googleからの信頼度が低い
- 月間600PV規模のサイトは被リンクも少なく、トラスト指標が弱い
- AI生成コンテンツへの2026年2月アップデートの影響が不明確

**期待できるPVインパクト：** Discoverに掲載されると1記事あたり数千〜数万PVの瞬間流入が可能（ただし継続性はなく一時的）。月次影響は予測困難。

**実現までの時間軸：** 最短6ヶ月以上（ドメイン成熟後）。技術要件（1200px画像）を今から整備しておくことは有効。

**このチャネルが機能しない場合：** ドメイン年齢が成熟してもDiscoverに掲載されないサイトは多数存在する。E-E-A-Tが低く評価されたAI生成コンテンツは引き続き除外される可能性が高く、このチャネルへの過度な依存は危険。SEOによるOrganic Searchを主軸に据えつつ、Discoverは「ボーナス」として位置づける設計が適切。

## 2. Yahoo!ニュース・ポータルサイト掲載

### 2-1. Yahoo!ニュースへの掲載方法

Yahoo!ニュースへ個人サイトのコンテンツを掲載するルートは以下の2系統がある。

**Yahoo!ニュース エキスパート（旧：個人オーサー）：**

- 約2,800名のエキスパート（うちオーサー約700名）が参加する専門家登録制度
- 「専門性」「実績」「発信力」「時勢・ニーズ」を踏まえた厳正な審査
- Yahoo!側からの審査・招待制が基本であり、個人サイトが自発的に応募して通過するハードルは高い
- AI運営サイトとしての「実績・発信力」の証明が困難

**媒体提携（企業向け）：**

- 一般的に一定規模の法人・メディアが対象
- 個人サイトの掲載は事実上困難

### 2-2. SmartNews

SmartNewsは新規媒体からのSmartFormat申請受付を全面停止している（2024年以降）。掲載される唯一の方法は既存の提携媒体に転載されることであり、新規参入は現実的でない。

### 2-3. PR TIMESによるプレスリリース配信

PR TIMESは月間1億PV超のプレスリリース専門サイトで、配信されると多数のメディアの目に触れる。ただし以下の限界がある。

- 企業または法人格が前提であり、個人や無法人は原則対象外
- AI運営サイトとして注目性は高い可能性があるが、「AI実験サイト」はニュース性として弱い
- 配信費用（スタートアップ向けで月5,000〜15,000円程度）に対してROIが不明確

**注目すべき事例：** Makko（診断ポータルサイト）はPR TIMESでプレスリリース配信を行い「リリースから8ヶ月で月間3,000万PV・参加総数1,500万回突破」を発表している。ただしMakkoは法人（株式会社ギミックプラス）が運営しており、PRTIMESを活用できる条件が異なる。

### 2-4. 実現可能性評価

**実現可能性：低（Yahoo!ニュース・SmartNews）/ 低〜中（PR TIMES）**

**yolos.netでの実現可否：** Yahoo!ニュースとSmartNewsはほぼ不可。PR TIMESは法人格取得を前提とすれば可能性はあるが、現状では対象外。

**期待できるPVインパクト：** PR TIMESで話題性のあるリリースが出た場合、数日間で数千〜1万PV程度の可能性があるが保証なし。

**実現までの時間軸：** PR TIMESは法人化対応の問題もあり、短期では実現困難。

**このチャネルが機能しない場合：** ポータルサイト・ニュースアグリゲーター経由のトラフィックは現時点でyolos.netの有効な選択肢ではない。このチャネルに時間・コストを投じる優先度は低い。

## 3. はてなブックマーク戦略

### 3-1. ホットエントリー掲載の仕組みと最低ブックマーク数

はてなブックマーク（以下はてブ）のエントリー掲載の仕組みは以下のとおりである。

| 段階                                   | ブクマ数の目安    | 掲載場所                         |
| -------------------------------------- | ----------------- | -------------------------------- |
| 新着エントリー                         | 3ブクマ（旧仕様） | 現在は数のみでは不掲載の場合あり |
| カテゴリ人気エントリー                 | 10〜15ブクマ程度  | カテゴリ別人気に出現             |
| 総合人気エントリー（ホットエントリー） | 50〜100ブクマ以上 | TOPページに掲載                  |

**重要な注意点：** 2024年以降、「3ブクマで新着」という仕様は廃止されている。はてな公式によれば「ブックマーク数のみを指標とせず、独自のアルゴリズムで掲載基準を調整している」。スパムフィルターが強化されており、新規・低権威サイトはブクマ数があっても掲載されないケースが報告されている。

### 3-2. セルフブックマークと互助会の規約状況

**セルフブックマーク（自分のサイトをブクマ）：**

- はてな公式は「自身のサイトを自身のアカウントでブクマすること自体は表示制限・利用停止の対象ではない」としている
- ただし「ブクマ数増加のためだけ」の利用は推奨されない
- メインアカウントとサブアカウントでの複数ブクマはスパム扱い

**互助会（相互ブクマ）：**

- はてな公式は「明確な規約違反」として対応を宣言している
- 互助会が発覚したサイトはエントリーから除外、ユーザーは利用制限の対象
- クラウドソーシングやSNSでの会員募集は「発見次第対応」としている

**結論：** セルフブクマ1つは問題ないが、組織的・意図的なブクマ数増加はリスクが高い。自然なブクマ獲得以外の方法は規約違反または事実上の不可。

### 3-3. 自然なブクマ獲得パターンと事例

**はてブでバズりやすいコンテンツの傾向（実例含む）：**

技術系ブログの実例として、「はてブでバズる前の10倍以上のPV（722PV）を獲得した」事例では35ブクマで学びカテゴリと総合カテゴリの人気上位に達したと報告されている。また1100ブクマ+500ブクマが付いた事例ではPVと収益が大幅増となっている（note.comの公開データより）。

ただし2020年代以降、はてブ流入の「単価」は低下している。2009年頃は150ブクマで5,000〜10,000PVのはてブ流入があったが、最近は150ブクマでも2,000PV程度に留まるとの分析がある。

**バズりやすいコンテンツタイプ：**

- 技術解説・ハウツー記事（エンジニア・はてなユーザー層が多い）
- 論考・考察記事（長文・独自視点）
- 「まとめ・一覧」型コンテンツ（保存して後で読む用途）
- 炎上・議論を呼ぶテーマ（ただしネガティブリスクあり）

**診断系コンテンツのはてブ適性：** 診断コンテンツはTwitter/XとのSNS連携でバズる設計であり、はてブユーザー層（テキストコンテンツの保存・共有）との相性は弱い。診断ページ単体でホットエントリーを狙う事例は見当たらず、診断を「ツールとして紹介するブログ記事」のほうがはてブ向きと言える。

### 3-4. フォロワーゼロでの最初のブクマ獲得方法

最初の3ブクマ（旧仕様での新着掲載の閾値）を自然に獲得する現実的な方法は以下のとおりであるが、いずれも保証はない。

1. **はてなブログとの連携**：はてなブログで記事を書くと、はてなユーザーのフィードに露出する確率がある
2. **Xでのシェア**：記事URLをXに投稿してブクマを促す（フォロワーがいない場合は効果限定的）
3. **コンテンツの質のみ**：技術的・教育的価値の高いコンテンツがGoogleやXで発見されてブクマされる（最も時間がかかるが最も持続可能）
4. **セルフブクマ1つ**：規約上許容範囲だが、スパムフィルターの影響で新着掲載される保証なし

### 3-5. 実現可能性評価

**実現可能性：低〜中（自然獲得に限定した場合）**

**yolos.netでの実現可否：**

- 現時点でスパムフィルターの影響を受けやすい新規ドメインであり、数ブクマでは掲載されない可能性が高い
- フォロワーゼロの状態でブクマの連鎖を起こすことは極めて困難
- 技術ブログ・考察記事など「はてなユーザー向けコンテンツ」を意図的に作成するアプローチが現実的

**期待できるPVインパクト：**

- ホットエントリー（50〜100ブクマ）達成時：数千〜1万PV程度の一時的流入
- カテゴリ人気エントリー（10〜15ブクマ）達成時：500〜2,000PV程度

**実現までの時間軸：** 最短3〜6ヶ月（高品質コンテンツを継続投稿した場合）。ホットエントリー到達は1年以上かかる可能性が高い。

**このチャネルが機能しない場合：** yolos.netのコンテンツがはてなユーザー層（IT系・テキスト指向）とミスマッチの場合、ブクマは自然に集まらない。診断・辞典コンテンツはXのほうが適合性が高く、はてブに過度な期待をかけることはリスクがある。

## 4. バイラル/SNS拡散の起点獲得

### 4-1. 診断コンテンツのSNS拡散モデル

診断コンテンツがXでバイラルする仕組みは「シェアによる自己表現」の心理にある。「自分の意外な一面を知りたい」「結果を誰かに言いたい」という欲求が、シェアを自然に促す。

ShindanMaker（診断メーカー）のモデルはこの設計の典型例で、以下の構造を持つ。

- 誰でも診断を作成・投稿できるプラットフォーム設計
- 診断結果に関連するハッシュタグがXトレンドに出現するケース
- 一度バズると「診断してみた」という連鎖投稿が発生

**yolos.netとの差異：** ShindanMakerはプラットフォームとしての規模と信頼がある。yolos.netは独立サイトであり、「発見してもらう」ための入口が存在しない。

### 4-2. フォロワーゼロでのバイラル可能性

フォロワーゼロの個人サイトが診断コンテンツで自然拡散した事例は公開情報から特定できなかった。Makkoの事例では：

- 法人（株式会社ギミックプラス）が運営
- PR TIMESでのプレスリリース配信
- 「プラットフォーム型」設計（ユーザーが診断を作成・投稿できる仕組み）
- SNSシェア機能の組み込み

これらの要素を組み合わせており、「フォロワーゼロの個人サイトが自然拡散」というモデルとは異なる。

**Xアカウント運用なしでの自然拡散条件：**
診断コンテンツが運営者のXアカウントなしで拡散された実例として確認できる条件は以下のとおり。

1. シェアボタンに結果のサマリーテキスト（絵文字・フォーマット含む）が自動的に入る設計
2. 結果の意外性・自己肯定感の高さ・笑える要素
3. 「○○タイプ」「あなたの前世」等、アイデンティティに関わるテーマ
4. Xトレンドや他媒体で最初の発見者が現れること（運次第）

### 4-3. しいたけ占いのリテンションモデルの分析

しいたけ占いは毎週月曜更新の12星座占いであり、「月曜の朝のルーティン」として定着することでリピーターを形成している。TikTokでの「#しいたけ占い」関連動画の総再生回数は約1億回に達しており、これは読者が二次創作（週間vllog投稿等）を自発的に行うことで生まれた。

**yolos.netへの示唆：** 「毎週更新・毎日更新」のコンテンツがルーティン化されれば、リピーターからのシェアが新規流入を生む経路が形成される。ただしこの経路が機能するまでには、初期リピーター層の形成（最低数十〜数百人規模）が前提となる。

### 4-4. 実現可能性評価

**実現可能性：低（短期）/ 中（長期）**

**yolos.netでの実現可否：**

- Xアカウントを運営者（AI）が運用できない制約のもとでは、「発見の起点」を作ることが困難
- 診断コンテンツの設計次第では、1人のユーザーがXでシェアすれば連鎖が起こる可能性はある
- ただし「最初の1人に発見されること」自体がOrganic Search等の他チャネルに依存する

**期待できるPVインパクト：**

- バイラル成功時：数万〜数十万PV（ただし確率は極めて低い）
- 継続的なシェアによる積み上げ：月数百PV程度が現実的な上限

**実現までの時間軸：** 不定（バイラルは予測不能）。計画的に達成できるものではない。

**このチャネルが機能しない場合：** X経由のバイラルは「運とコンテンツ品質」の掛け合わせであり、計画的に達成する手段が現状では存在しない。フォロワーゼロの状態が続く限り、SNS経由の安定流入は期待できない。SEOで流入を積み上げ、そこから派生するシェアを待つアプローチが最も現実的。

## 5. デイリーコンテンツ（Wordle型）のリテンション

### 5-1. Wordleの拡散モデルと日本での普及

Wordleは2021年10月に一般公開され、2022年1月時点でMAU30万人を達成した。日本でも2022年1月に急速に普及した。

**バイラル構造：**

- 全員が同じ問題を解く「共体験」設計
- 結果を絵文字グリッド（格子模様）として1クリックでXにコピーできる機能
- 「結果のグリッド」を見た人が「何これ？」と調べてアクセスする二次的拡散
- 正解/不正解に関わらず「今日のは難しかった」という会話が生まれる社会的接着剤

日本語版（「言葉で遊ぼう」/ kotobade-asobou など）はGitHub上で公開されているOSSプロジェクトが多く、専用の公式日本語版は存在しない状態。

### 5-2. デイリーコンテンツでのリピーター形成事例

**しいたけ占い（週次更新）：**

- 毎週月曜のルーティンとして定着 → 「ないと週が始まらない」というユーザー心理
- TikTok関連動画の総再生1億回はリピーターの二次創作から生まれた

**毎日脳トレ・パズル系：**

- 「今日だけプレイできる」という希少性が習慣形成を促す
- 「友達と比べたい」というソーシャル動機でシェアが起きる

**デイリーコンテンツがリテンションを生む条件：**

1. 毎日同じ時間に新しいコンテンツが更新される信頼性
2. プレイ結果の「シェア可能なフォーマット」の存在
3. 「今日はクリアできたか」という達成感の設計
4. 難易度の適切さ（簡単すぎず難しすぎない）

### 5-3. yolos.netのデイリーコンテンツ設計への示唆

yolos.netに類似のデイリーゲームが存在する場合、以下が必要になる。

- 結果をXに投稿できるワンクリックシェアボタン（テキストフォーマット付き）
- 「今日の結果：○○（絵文字グリッドまたはスコア）」の自動生成
- 毎日確実に新コンテンツが生成される安定性

**リピーターがシェアして新規ユーザーを獲得する経路：**

- リピーターがXに結果を投稿 → フォロワーが見る → フォロワーがプレイ → リピーターになる
- この経路が機能するには最初のリピーター層（数十〜数百人）が必要

### 5-4. 実現可能性評価

**実現可能性：中（コンテンツ設計次第）**

**yolos.netでの実現可否：**

- デイリーゲームそのもの（技術的実装）は可能
- 「最初のリピーターを形成すること」がボトルネック
- SEOで初回訪問者を獲得し、そこからリピーターに変換する設計が必要
- Wordleのバイラルグリッド型シェア機能は、日本語のひらがな/漢字パズルにも応用可能

**期待できるPVインパクト：**

- リピーター100人 × 毎日アクセスの場合：月3,000PV増
- リピーター1,000人達成時：月30,000PV増
- ただしこのレベルのリピーター形成には6ヶ月〜1年以上かかる見込み

**実現までの時間軸：** 設計・実装は1〜2ヶ月。リピーター形成は6ヶ月〜1年。

**このチャネルが機能しない場合：** デイリーコンテンツへの訪問者が定着しない場合（初回訪問→離脱）は、コンテンツの難易度・楽しさ・UIに問題がある可能性が高い。A/Bテストで継続率を測定し、リピーター率が低い場合はコンテンツ改善またはジャンル変更を検討する必要がある。

## 6. 総合評価と優先順位

### チャネル別サマリー

| チャネル                        | 実現可能性 | PVインパクト | 時間軸           | yolos.net適合性      |
| ------------------------------- | ---------- | ------------ | ---------------- | -------------------- |
| Google Discover                 | 低→中      | 大（一時的） | 6ヶ月以上        | 中（ブログ記事のみ） |
| Yahoo!ニュース                  | 低         | 大           | 不可（現状）     | 低                   |
| SmartNews                       | 低         | 大           | 不可（新規停止） | 低                   |
| PR TIMES                        | 低         | 中           | 要法人化         | 低                   |
| はてなブックマーク              | 低〜中     | 中           | 3〜12ヶ月        | 中（技術ブログ）     |
| SNSバイラル（X）                | 低         | 大（不確実） | 不定             | 中（診断コンテンツ） |
| デイリーコンテンツ+リテンション | 中         | 中（累積）   | 6〜12ヶ月        | 高                   |

### 推奨アプローチ（優先順位順）

1. **最優先：SEO（Organic Search）を主軸に継続**  
   現在のトラフィックの41%を占め、かつ唯一安定して計画的に成長できるチャネル。

2. **第2優先：デイリーコンテンツのリテンション設計**  
   Wordleモデルのシェア機能実装とリピーター形成は、中長期での累積PVに最も安定した貢献をもたらす。

3. **第3優先：はてなブックマーク対応コンテンツの作成**  
   技術的・論考的なブログ記事をはてなユーザー向けに設計することで、自然なブクマ獲得を狙う。期待値は低いが実施コストも低い。

4. **第4優先：Google Discoverのための技術準備**  
   今から1200px画像・max-image-preview:large 設定を整備しておくことで、ドメイン成熟後の掲載機会に備える。

5. **見送り：Yahoo!ニュース・SmartNews・PR TIMES**  
   現状では制度的・法人的制約により参入不可または意味がない。

### 否定シナリオの整理

- Google Discoverに1年以上掲載されない場合：E-E-A-TとAI生成コンテンツ評価の問題である可能性が高く、コンテンツの独自性強化が必要。
- はてブでブクマが自然につかない場合：yolos.netのコンテンツがはてなユーザー層（IT系男性・テキスト指向）と合致していない可能性。ターゲット層の再定義が必要。
- デイリーコンテンツのリピーター率が低い場合：コンテンツの楽しさ・難易度・UI改善を実施。改善後も定着しない場合はジャンル変更。
- SNSバイラルが6ヶ月起きない場合：コンテンツのシェア設計（ワンクリックXポスト機能等）の実装状況を確認し、シェア動線の改善を行う。それでも拡散しない場合は、バイラルチャネルへの期待を下げSEO・リテンションに集中する。
