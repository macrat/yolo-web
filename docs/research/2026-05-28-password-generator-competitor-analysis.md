## 1. 各競合の詳細分析

### 1-1. 1Password Password Generator (英語圏)

**URL:** https://1password.com/password-generator

**生成方式:** クライアント完結（ページ内 JavaScript。パスワード文字列がサーバーに送信される証拠なし）

**UI 動線:** 最短 1 クリック（ページロード時点でパスワード生成済み → "Copy password" 1 クリックでコピー完了）

**オプション:**

- パスワードタイプ: Random / Memorable / PIN の 3 種類をタブで切り替え
- 文字数: スライダー + スピンボタン（デフォルト 20 文字）
- Numbers オン/オフ（デフォルト: オン）
- Symbols オン/オフ（デフォルト: オフ）

**デフォルト値:** 20 文字、Numbers オン、Symbols オフ（大文字小文字はオン固定で変更不可）

**差別化軸:**

- "Memorable" モード（複数単語を組み合わせたパスフレーズ型）が目立つ
- "PIN" モード（数字のみ）もワンタッチ切り替え
- パスワード文字が文字ごとにスペース区切りで表示されており、視認性が高い（「i d J a j v i y K 8...」）
- サービス訴求（パスワードマネージャーへの誘導）がメイン目的

**タイル動線可否:** ページロード直後に生成済みのため、タイル上に表示して "コピー" ボタン 1 つで完結可能。「生成」ボタン不要とも言えるが、再生成ボタンは必要。

**UI の見た目:** 白背景でクリーン。ヒーローセクションにパスワード表示エリアとオプション、その下は製品説明・CTA が縦に並ぶ構造。オプション部分は区切られたカード内。

### 1-2. Bitwarden Password Generator (英語圏)

**URL:** https://bitwarden.com/password-generator/

**生成方式:** クライアント完結（「passwords are generated locally on your device using cryptographic randomness; they never travel across the internet」と明記）

**UI 動線:** ページロード時点で生成済み → "Copy password" 1 クリックでコピー完了

**オプション:**

- Password / Passphrase の 2 モード（ラジオボタン切り替え）
- 文字数: スライダー（デフォルト 14 文字）
- A-Z（大文字）オン/オフ（デフォルト: オン）
- a-z（小文字）オン/オフ（デフォルト: オン）
- 0-9（数字）オン/オフ（デフォルト: オン）
- !@#$%^&\*（記号）オン/オフ（デフォルト: **オフ**）

**デフォルト値:** 14 文字、大文字・小文字・数字オン、記号オフ

**差別化軸:**

- Passphrase モード（単語列の覚えやすいパスワード）を同等に並列提示
- 「Password strength: strong」「Estimated crack time: centuries」の強度＋クラック推定時間表示が強力な安心感を提供
- クリップボードコピー後に「Now save it securely in your Bitwarden vault」モーダルが出て自然なアップセル
- オープンソース訴求が信頼感を補強
- フリーツールとして Password Generator / Passphrase Generator / Password Strength Tester / Username Generator の 4 つを無料提供

**タイル動線可否:** ページロード即生成 + 1 クリックコピーが確立。タイル化すれば「ボタン→コピー」の 1 ステップに圧縮できる。

**UI の見た目:** ダークヘッダー＋白メインのコントラスト構成。生成ボックスは中央配置の半透明グレー背景カード。パスワード文字は大きく表示。強度バーと "centuries" テキストが視覚的に分かりやすい。

- Uppercase オン/オフ（デフォルト: オン）
- Lowercase オン/オフ（デフォルト: オン）
- Numbers オン/オフ（デフォルト: オン）
- Symbols オン/オフ（デフォルト: オン）

**デフォルト値:** 12 文字、全文字種オン（記号含む）

**差別化軸:**

**タイル動線可否:** スクロール不要エリアにジェネレーターがあるが、ヒーロービジュアルが邪魔で初見は迷う。タイル化においてはヒーロー不要。

**UI の見た目:** ブラック＋ホワイト系のモダンデザイン。パスワードテキストボックスが入力フォームと同じ白背景で見やすい。コピーボタンとリフレッシュボタンが横並び。"Strong" インジケータがパスワードフィールドの下に緑テキストで表示。

**UI の見た目:** 極めてシンプルなテキスト主体のレイアウト。1990 年代風のデザインで装飾なし。フォームエリアは白背景の小さい 2 フィールド + ボタン。

## 3. ブラウザ組み込み機能との比較

### Safari（iOS / macOS）の自動パスワード生成

- ログインフォームにフォーカスした際に自動提案
- 20 文字前後のランダム文字列（記号含む）を 1 タップで入力・キーチェーンに保存
- **ユーザーが意図的にカスタマイズする手段がない**（文字数・文字種の変更不可）
- iCloud キーチェーンとの自動連携がシームレス

### Chrome（Google パスワードマネージャー）の自動パスワード生成

- Safari と同様、ログインフォーム時に「パスワードを提案」
- 生成されたパスワードは Google アカウントに同期
- カスタマイズは不可

### ブラウザ組み込み機能 vs. 専用ツールの差異

| 比較軸                     | ブラウザ組み込み                    | 専用ツール（yolos.net など）                   |
| -------------------------- | ----------------------------------- | ---------------------------------------------- |
| カスタマイズ性             | なし（固定パターン）                | あり（文字数・文字種・記号等）                 |
| 複数個生成                 | なし（1 個のみ）                    | あり                                           |
| 特定サービスへの要件対応   | 不可（記号禁止サービス等）          | 可（記号除外チェックボックス）                 |
| 利用場面                   | ブラウザ内ログインフォーム限定      | API キー、Wi-Fi パスワード、任意のテキスト入力 |
| パスワードマネージャー依存 | あり（Apple/Google アカウント必須） | なし（ブラウザ完結・登録不要）                 |

**結論:** ブラウザ組み込み機能が普及している現在、「ブラウザフォームに入力しないもの」（Wi-Fi パスワード、社内システム、API キー等）や「記号禁止サービスへの対応」「複数個一括生成」においては専用ツールへの需要が維持される。

## 4. yolos.net の現状評価

### 強み

1. **デフォルト 16 文字**: 英語圏大手（1Password: 20 文字、Bitwarden: 14 文字、LastPass: 12 文字）と比べて現代的推奨値をすでに設定済み。日本語圏競合（LUFT: 8 文字、cman: 8 文字）より高い。

2. **記号がデフォルトオン**: Bitwarden・1Password・日本語圏すべてが記号デフォルトオフ。yolos.net はデフォルトオン（LastPass と同等）。これはセキュリティ観点で高評価。

3. **エントロピー計算による強度表示**: 数値根拠のある強度評価（entropy < 40/60/80/∞ で 4 段階）。競合の多くは経験則ラベルか未搭載。

4. **広告なし**: 日本語圏競合（LUFT・cman）は広告が非常に多い。yolos.net は完全広告フリー（Google AdSense は申請中だが現状なし）。

5. **クライアント完結・登録不要**: 英語圏大手（1Password・LastPass）はサービス誘導が主目的。yolos.net は pure tool として機能。

### 弱み

1. **ページロード後に手動生成が必要**: 英語圏大手 3 社と Graviness はロード直後に生成済み。yolos.net は「パスワード生成」ボタンを 1 回押す必要があり、最短動線が 2 クリック（生成 → コピー）。

2. **パスフレーズ（Memorable）モードなし**: 1Password と Bitwarden が差別化として強調するパスフレーズ型生成がない。

3. **複数個生成なし**: 一括生成（最大 1,000 個）+ CSV ダウンロードは日本語圏競合（LUFT・cman・Graviness）の強みだが、yolos.net は 1 個のみ。

4. **タイル化されていない**: トップページへの露出がなく、ツール直 URL への流入に依存。

## 5. タイル化における最小機能セット（PM 判断材料）

### トップページタイル化の要件整理

タイル上に「ボタン → 生成 → コピー」を 1 タイルで実現するとした場合の機能整理:

#### 必須機能（タイルに含める）

| 機能                       | 理由                                                    |
| -------------------------- | ------------------------------------------------------- |
| タイルロード時に自動生成   | 英語圏大手 3 社の標準。「すぐ使える」体験が差別化の前提 |
| パスワード表示エリア       | 生成結果の視認性（モノスペースフォント推奨）            |
| コピーボタン               | 1 クリックコピー（コピー後フィードバック付き）          |
| 再生成ボタン               | 気に入らなければ再生成できる安心感                      |
| ツール詳細ページへのリンク | カスタマイズしたいユーザーを誘導                        |

#### 割愛可能（詳細ページに委譲）

| 機能                         | 理由                                                                     |
| ---------------------------- | ------------------------------------------------------------------------ |
| 文字数スライダー             | タイルの面積では操作が困難。デフォルト 16 文字で十分                     |
| 文字種チェックボックス       | タイルでは操作ノイズになる。デフォルト（大小英字+数字+記号）で十分       |
| 紛らわしい文字除外オプション | ニッチな需要。詳細ページで提供                                           |
| 強度メーター                 | タイルでは補足情報として視覚ノイズになる可能性。含めるなら小さいバー表示 |

#### タイルで追加すべき差別化要素（検討）

| 機能                       | 競合比較                                                                                             | 判断     |
| -------------------------- | ---------------------------------------------------------------------------------------------------- | -------- |
| 「サーバー送信なし」バッジ | 英語圏大手は詳細ページで説明、日本語圏は小さく注記。タイルで強調すれば希少な差別化                   | **推奨** |
| 強度バー（小サイズ）       | Bitwarden は crack time 付き。yolos.net のエントロピーベースは競合優位。小さく表示するだけで価値ある | **推奨** |
| パスフレーズモード切り替え | 1Password・Bitwarden の強みだが実装コスト高。タイルには不要                                          | 見送り   |

### 推奨タイルデザイン仕様（概要）

```
[パスワードジェネレーター]
[サーバー送信なし 🔒]

╔═══════════════════════════════╗
║  G#mP9aKs!2wQvTn             ║
╚═══════════════════════════════╝
[強度: ████████ 強い]

[再生成] [コピー]

→ オプションを設定して生成
```

- タイルカテゴリ: セキュリティ
- 初期状態: ページロード時に自動生成（デフォルト 16 文字・全文字種オン）
- コピー後: "コピー済み ✓" に 2 秒表示変更
- 「オプションを設定して生成」リンク → 詳細ページ（既存ページ）

## 6. yolos.net の差別化軸評価

調査目的だった 3 つの差別化軸候補の競争力を評価する:

### (a) ブラウザ完結 + サーバー送信なし（プライバシー）

**競争力: 中**

英語圏大手 3 社はいずれもクライアント完結。日本語圏競合（LUFT・cman）もクライアント完結。RANDOM.ORG のみサーバー処理（ただし「大気ノイズ真乱数」で独自路線）。

唯一の差別化点は「信頼できるプライバシー説明の可視性」。英語圏大手は詳細ページで説明、LUFT・cman はページ下部の小テキスト。yolos.net がタイル上に「サーバー送信なし」バッジとして視覚化すれば、プライバシー意識の高いユーザーへの訴求力が向上する。

### (b) シンプル / 登録不要 / 広告少（UX）

**競争力: 高**

英語圏大手は全てパスワードマネージャーへの誘導が目的で、ランディングページは製品訴求コンテンツが多い。純粋なツールとして使うには離脱 CTA が邪魔。日本語圏競合は広告過多でページが重い。Tech-Unlimited は広告少・シンプルだが機能が薄い。

yolos.net は「登録不要・広告なし・シンプルな 1 機能ページ」の組み合わせが現状でも差別化できている。これをトップページタイルでさらに短縮すれば競争力は高い。

### (c) タイル UI による最短動線（トップページから 1 ボタンで生成）

**競争力: 高（実装後）**

競合でトップページからワンタッチでパスワード生成できる構造はない。1Password・Bitwarden・LastPass はパスワードジェネレーターが独立 URL。日本語圏競合も専用ページのみ。

**タイル化によりパスワード生成の最短動線を競合他社よりも短くできる**。「サイトトップ → タイルでコピー」という 0 ナビゲーションの体験は現在の競合にはない。ただし実装後の価値は実際の訪問者の行動データで確認が必要。

## 7. 結論と推奨

### 実装推奨優先度

1. **タイルロード時の自動生成**: 英語圏大手の標準。現在の yolos.net は生成ボタン必要で 1 クリック分の摩擦がある。これをなくすだけで体験が大幅改善。

2. **「サーバー送信なし」の可視化**: タイルまたは詳細ページの目立つ位置に表示。現在は実装されているが訴求できていない。

3. **強度バーの維持**: エントロピーベースの強度表示は競合の多くにない。タイルでも小さく表示する価値あり。

4. **16 文字デフォルトの維持**: すでに競合より長い。変更不要。

5. **記号オンデフォルトの維持**: Bitwarden・1Password はオフ。yolos.net がオンを維持することは差別化。

### 不要な機能追加

- パスフレーズモード（実装コスト高、タイルに向かない）
- 1,000 個一括生成（yolos.net のコンセプト「シンプル」に反する）
- CSV ダウンロード（同上）
- zxcvbn ライブラリ導入（エントロピー計算で代替できている）
