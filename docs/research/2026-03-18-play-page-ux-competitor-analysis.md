# /play ページ UI/UX 競合調査レポート

- 調査日時: 2026-03-18
- 調査目的: /play一覧ページのUI/UX改善のための競合サイト調査（19種のインタラクティブコンテンツを統合表示する際のベストプラクティスを把握する）
- 調査手法: Playwright (headless Chromium, locale: ja-JP) によるスクリーンショット取得 + 視覚分析
- 情報源: 診断メーカー、Kuizy、Vonvon、Yahoo!ゲーム、MiniClip（計5サイト）

## 2. 各サイト詳細分析

### 2-1. 診断Bot (shindan-maker.com)

**定性的観察:**

- トップページはBtoB向けSaaSのランディングページ構成。一般ユーザー向けの診断一覧ページではなく、「診断ツールを作りたい企業」向けの営業ページ
- ヒーローセクション: 左テキスト/右イラストの2カラムレイアウト。キャッチコピーは「オリジナルの診断で友だちに楽しく商品紹介」
- モバイルでは「スマートフォンでのアクセスはできません」と表示 — モバイル対応なし
- カラーパレット: 白背景、ネイビー系テキスト、アクセントカラーはシアンブルー (#00a8e0 系)
- フォント: ゴシック体、読みやすいサイズ

**コンテンツ分類:** 機能別（診断エンジン、LINE連携、テンプレート）

**カードデザイン:** なし（一覧型ではなく縦スクロールLP）

**初回訪問者への導線:** 「利用プランを見る」CTAボタン1つに集約

**回遊促進:** なし（直帰率高そうな設計）

**学習ポイント:**

- ユーザーが求めるのは「診断を作る」ではなく「診断を体験する」。yolos.netの/playページが目指すべき方向とは異なるモデル

---

### 2-2. Kuizy (kuizy.net)

**定性的観察:**

- トップナビゲーション: 「クイズ」「診断」「お絵描き診断」「大喜利」「ログイン」とコンテンツタイプ別にタブ分類
- ヘッダー下にサービスコピー: 「クイズや診断がかんたんに作れる 日本最大級のクイズ・診断メディア」
- サービス説明直下に「クイズ・診断を作る」CTAボタン（オレンジ色）— UGC型であることを強調
- コンテンツ一覧は縦型リスト: サムネイル画像 (左) + タイトル・カテゴリラベル (右) の横並びカード
- カテゴリラベルは色付きバッジ（「診断」「お絵描き診断」など）でコンテンツタイプを識別
- モバイルでは初回に広告モーダルが表示される（「続けてkuizyを読む」— 広告視聴ゲート）
- モバイルUIはデスクトップとほぼ同一構造でレスポンシブ対応

**コンテンツ分類:** タイプ別タブ（クイズ/診断/お絵描き診断/大喜利）

**カードデザイン:**

- サムネイル画像必須（ユーザー投稿画像）
- タイトル + カテゴリバッジ
- 問数・プレイ回数等の数値は表示されていない

**色使い:** 白背景、青系アクセント（ブランドカラー）、カテゴリバッジは複数色

**初回訪問者への導線:** 上部に「現在人気の診断」的なコンテンツが並ぶ。ランキング等の明示はなし

**回遊促進:**

- 「詳細を見る」で関連コンテンツのグループページへ誘導
- コンテンツが縦に大量並列されており、スクロールで次々発見できる構造

**学習ポイント:**

- コンテンツタイプをタブで分類するナビゲーションパターンは有効
- UGCサイトのためサムネイル画像が豊富。固定コンテンツサイトでは画像のない状態が課題になりうる
- モバイルでの広告モーダルはUXを著しく損なっている（反面教師）

### 2-3. Vonvon (vonvon.me/ja)

**定性的観察:**

- ヒーローエリア: フルワイドのグラデーション背景（紫〜ピンク〜オレンジ）に診断タイトルを大きく表示。「私のカラー感度は何レベル？」のような感情訴求型コピー
- CTA: 「スタート！」の赤いボタン — コンテンツ一覧より先に「今すぐ体験」を促す
- その下に「Vonvonの人気診断」として 2カラムのカードグリッドを配置
- カードはサムネイル画像主体。画像クオリティは高く、感情・テーマに合ったビジュアル
- 診断グループ（「友情診断」「Vonvonミニ」）をカード内で示している
- カラーパレット: 鮮やかなグラデーション多用、賑やかで視覚的に引き付ける
- モバイルUIはデスクトップとほぼ同一。スクロール量は少なく、コンパクト

**コンテンツ分類:** 特定カテゴリ分類なし。「人気診断」として並列表示

**カードデザイン:**

- サムネイル画像 (大きく表示) が主役
- カード下部にタイトルと簡単な説明文
- バッジ・問数等なし

**色使い:** 鮮やかなグラデーション（パステルからビビッドまで）。SNS映えを意識したビジュアル

**初回訪問者への導線:**

- ヒーローエリアで「今すぐスタート」を強調
- 最初に1つの診断に没入させる戦略（一覧より体験優先）

**回遊促進:**

- 各診断完了後のシェア誘導が強い（SNSシェアでバイラル想定）
- トップページは「入口」として機能し、完了後に次の診断へ誘導する設計と思われる

**学習ポイント:**

- サムネイル画像のビジュアルクオリティが訴求力に直結している
- ヒーローで1コンテンツを強くプッシュして没入させる戦略は、SEO流入後の直帰対策にも有効
- 感情・好奇心をくすぐるコピーライティング（「○○は何レベル？」形式）

### 2-4. Yahoo!ゲーム (games.yahoo.co.jp)

**定性的観察:**

- ヘッダー: Yahoo!ロゴ + ゲームカテゴリナビ（ゲーム一覧、かんたんゲーム等）
- ヒーローエリア: 大型バナー（ローテーション）で注目タイトルを前面プッシュ。右側に小さなバナーを複数配置（雑誌の表紙構成）
- 「おすすめゲーム」「ピックアップ」「トップゲーム」「人気ゲーム」「LINE GAME」等のセクション分類
- ゲームカードは正方形サムネイル + タイトルの最小構成。画像が情報の大半
- カラーパレット: Yahoo!ブランドカラー（赤）＋白背景。情報密度が高く、賑やかな印象
- モバイル: タブ形式（トップ/ゲーム一覧/かんたんゲーム）。「おすすめゲーム」は横スクロールカルーセル形式
- モバイルでのカルーセルは2枚横並び + 次のカードが見切れる設計（スワイプ誘導）

**コンテンツ分類:**

- 「おすすめ」「ピックアップ」「トップ（ランキング）」「人気」「ジャンル別（LINE GAME等）」
- 人気・新着・ジャンルの3軸でコンテンツを整理

**カードデザイン:**

- サムネイル画像 (正方形、大) が主役
- タイトルのみ（説明文なし）
- 評価スター等なし — シンプルに画像とタイトルのみ

**色使い:** Yahoo!ブランドで統一。ゲームサムネイルが色のほぼすべてを担っている

**初回訪問者への導線:**

- 大型ヒーローバナーで即座に注目タイトルへ誘導
- 「おすすめ」セクションが最初のゲーム一覧（キュレーション）

**回遊促進:**

- 大量のゲームが並ぶため、発見のしやすさが強み
- カテゴリ・ジャンル横断ブラウジングが可能

**学習ポイント:**

- セクション分けによる「おすすめ→人気→ジャンル別」の流れは、初回訪問者から常連まで対応
- モバイルでのカルーセル型横スクロールはスペース効率が高い
- サムネイルのビジュアルクオリティが訴求力のほぼすべて — テキストは二次的

### 2-5. MiniClip (miniclip.com)

**定性的観察:**

- トップページはゲームポータルではなく企業コーポレートサイト構成
- ヒーロー: 自社ゲーム「8 Ball Pool」「Agar.io」を大きく表示。「Play our most popular games now!」+ 「PLAY NOW」CTA
- その下はニュース・プレスリリース・採用情報等のコーポレートコンテンツ
- ゲーム一覧ページは別URL（/web-games 等）の模様
- カラーパレット: 黒背景 + 赤・白のブランドカラー。ゲーミング感を演出
- モバイル: 大型ヒーローがほぼ全画面を占有。コンテンツは縦スクロール
- Cookieコンセントが画面下部に表示される（GDPR対応）

**コンテンツ分類:** コーポレートサイトのため参考にしにくい

**カードデザイン:** ニュースカード形式（テキスト + サムネイル）。ゲームカードではない

**学習ポイント:**

- 黒背景のゲーミングデザインは特定ターゲット層（ゲーマー）への強いシグナル
- トップページで最も人気の自社ゲームを1〜2タイトルに絞って大きく訴求するアプローチは、エントリーポイントとして有効
- ゲームポータルとしての参考事例としては限定的

## 4. 現在の /play ページとの比較

### 現在の /play ページの構成

- ヒーローバナー（テキストのみ、「遊ぶ」という見出し）
- カテゴリ別セクション（占い/性格診断/知識テスト/ゲーム）
- 各カテゴリ内でグリッドカードを表示
- カードは絵文字アイコン + タイトル + 短い説明文 + CTAバッジ
- ホバーエフェクトあり（アクセントカラーボーダー + 浮き上がり）
- モバイル: 640px以下で1カラム

### 現在の設計の強み

- カテゴリ分類が明確でコンテンツの種類を即座に把握できる
- カードに説明文があり、クリック前に内容を理解できる
- 問数バッジで所要時間の目安を提供（知識テスト系）
- シンプルで読み込みが速い（サムネイル画像なし）

### 現在の設計の課題

1. **視覚的な引き付けが弱い**: 絵文字のみでサムネイル画像がない。競合は全サイトで画像によるビジュアル訴求が中心
2. **ヒーローが弱い**: テキストのみのヒーローは「今すぐ体験したい」という感情を喚起しにくい
3. **「おすすめ」「人気」「新着」の概念がない**: すべて同等に並んでいるため、初回訪問者が何から始めればいいか迷いやすい
4. **モバイルでの情報密度が低い**: 640px以下で1カラムになるため、スクロール量が大幅に増える
5. **回遊促進の仕組みがない**: 各コンテンツへのリンクのみで、関連コンテンツや「次はこれを試して」的な誘導がない

## 5. yolos.net /play ページへの改善提案

### 5-1. 優先度高: ヒーローセクションの強化

**現状:** テキスト「遊ぶ」+ サブテキストのみ

**提案:**

- ヒーローで1コンテンツを「今日のピック」として強くフィーチャー
- 例: 「今日のユーモア運勢」や話題の診断を大きなカード形式で表示
- 直接体験を促す「今すぐ始める」ボタン
- Vonvonの手法を参考に、感情・好奇心を刺激するコピーを添える

### 5-2. 優先度高: 「おすすめ」セクションの追加

**現状:** カテゴリ別のみ（占い/性格診断/知識テスト/ゲーム）

**提案:**

- カテゴリセクションの前に「まずはここから」「編集部おすすめ」等のセクションを追加
- 初回訪問者が迷わず体験できる導線を提供
- ロジック: 代表的なコンテンツを2〜3点選んで固定表示

### 5-3. 優先度中: カードのビジュアル強化

**現状:** 絵文字アイコン (2rem) + テキスト構成

**提案オプションA (低コスト):** 絵文字を大きくする + カードにアクセントカラー背景を適用

- カード上部にアクセントカラーの帯/背景を設けてビジュアル的な差別化
- 絵文字を3〜4rem程度に大きくして視認性向上

**提案オプションB (高コスト):** OGP画像をカードサムネイルとして活用

- 各コンテンツが持つOGP画像をカードのサムネイルとして再利用
- 画像があることで競合と同等のビジュアル訴求力を確保

### 5-4. 優先度中: モバイルでの2カラムグリッド

**現状:** 640px以下で1カラム（縦に長い）

**提案:**

- 375px幅でも2カラムグリッド (各カード約170px幅) を維持
- カードをよりコンパクトなデザインに変更（説明文を省略または折り畳み）
- Yahoo!ゲームの横スクロールカルーセル型も選択肢（ゲームセクションのみ適用等）

### 5-5. 優先度低: コンテンツバッジの充実

**現状:** 問数バッジ（クイズのみ）+ CTAバッジ「遊ぶ」

**提案:**

- 「毎日更新」バッジ（運勢・パズル系）
- 「5分以内」等の所要時間目安
- 「人気」「NEW」等のラベル
- カテゴリタイプを示すバッジ（診断/クイズ/ゲーム）

### 5-6. 優先度低: カテゴリ名の見直し

**現状:** 占い / 性格診断 / 知識テスト / ゲーム

**提案:**
競合サイトの分類を参考にした代替案:

| 現在のカテゴリ | 提案名            | 理由                                                    |
| -------------- | ----------------- | ------------------------------------------------------- |
| 占い           | 今日の運勢        | 具体性があり「今日も来たくなる」感を演出                |
| 性格診断       | あなたは何タイプ? | 疑問形で好奇心を刺激                                    |
| 知識テスト     | 知識を試す        | そのままでも可                                          |
| ゲーム         | 毎日パズル        | yolos.netのゲームが全て日替わりパズルであることを活かす |

## 6. カード情報とレイアウトの推奨事項

### カード表示情報（推奨優先順）

1. ビジュアル要素（絵文字大きめ or サムネイル画像）
2. タイトル（太字、1〜2行）
3. コンテンツタイプバッジ（診断/クイズ/ゲーム/占い）
4. 短い説明文（1〜2行）
5. 所要時間または問数（グレーテキスト）
6. CTAボタン「遊ぶ」「試す」

### レイアウト推奨

- デスクトップ (1280px): 3カラム（現状の `minmax(280px, 1fr)` はほぼ3カラムで適切）
- タブレット (768px): 2カラム
- モバイル (375px): 2カラム（現状の1カラムから変更。カードをコンパクト化）

### カードサイズ

- デスクトップ: 現状維持（280px以上）
- モバイル2カラム: カード幅 ~160px。絵文字+タイトルのみのコンパクト版も検討

## 7. まとめ

競合調査から得られた最も重要な示唆は以下の3点:

1. **ビジュアルが最優先**: すべての競合がサムネイル画像または大きなビジュアル要素でコンテンツを訴求している。yolos.netの絵文字アイコンは軽量だが視覚的インパクトが弱い

2. **入口での没入体験**: Vonvon, Yahoo!ゲーム, MiniClipはヒーローで1〜2のコンテンツを強力にプッシュし、すぐ体験させる設計。一覧ページでの迷いを減らす

3. **モバイルファースト**: Yahoo!ゲームのカルーセル等、モバイルに最適化されたUXパターンが有効。現状の640px以下1カラムは改善余地がある

yolos.netが差別化できる点は「AIによる日替わりコンテンツ」「毎日来る理由がある」という部分であり、これをページの冒頭で伝える設計が競合との差別化につながる。
