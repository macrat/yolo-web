# yolos.net 競合調査レポート

## 調査概要

本レポートは、yolos.netが提供するカテゴリ（Webツール・チートシート・単語ゲーム・クイズ・辞典・技術ブログ）の競合環境を調査し、競合との差別化ポイントと改善施策を整理したものです。

---

## カテゴリ1: Webツール（文字数カウント・変換・フォーマッタなど）

### 主要競合サイト

| サイト名                    | URL                                        | 特徴                                                                                              |
| --------------------------- | ------------------------------------------ | ------------------------------------------------------------------------------------------------- |
| ラッコツールズ              | https://rakko.tools/                       | 100種類以上のツール、20以上のカテゴリ、16言語対応、SEO・SSL・テキスト・ネットワーク等の幅広い領域 |
| Web便利ツール@ツールタロウ  | https://tool-taro.com/                     | 約35種のWeb開発者向けツール。Whois、パスワード生成、QRコード、HTMLバリデーション等                |
| 文字数カウント (luft.co.jp) | https://www.luft.co.jp/cgi/str_counter.php | 文字種別分析・読了時間推定・原稿用紙換算・正規表現除外設定など高機能な単一ツール                  |
| クロクロ・ツールズ          | https://crocro.com/tools/                  | 2015年開設、15種類の開発者向けWebアプリ群                                                         |

### 競合の成功要因

- **ラッコツールズ**: 関連サービス（RakkoKeyword, RakkoServer等）とのエコシステム連携。無料・ブラウザ完結・16言語対応の圧倒的アクセシビリティ。「くじ引き」のような非技術者向けツールも含む幅広さ
- **ツールタロウ**: プライバシー配慮（サーバーで即座に破棄）を明示しているため開発者の信頼を獲得
- **luft文字数カウント**: 単一ツールに深く特化し、文字種別解析・原稿用紙換算・特殊文字検出まで徹底して深化させた圧倒的な機能充実度

### 競合が提供していてyolos.netが持っていないもの

1. **解説・使い方ガイド**: 競合の多くは各ツールに詳細な使い方説明、具体的な使用場面の例示を設けている
2. **ツール数の規模感**: ラッコツールズは100種以上に対しyolos.netは約35種（量より質が重要ではあるが、認知されるための品揃えが不足）
3. **ユースケースの説明**: "なぜこのツールが必要か"を具体的に説明するコンテキストが薄い

### yolos.netが差別化できる方向性

- **深度戦略**: ラッコツールズが100種を浅く提供するのに対し、各ツールページを「このツールの使い方・活用例・関連知識」まで深化させる
- **AI由来の透明性**: AI運営であることを活かし、「このツールのアルゴリズムと仕様」を技術的に丁寧に説明する（E-E-A-T強化）
- **学習コンテキスト**: ツール使用時に関連するチートシート・解説記事へのリンクを設け、サイト全体の内部リンク強化

- **実例・ユースケース付き**: 「このコマンドはどんな時に使うか」の具体例をセットで提供（Qiitaの記事形式との差別化）

4. **収録数の規模**: 競合が数千〜9000語以上なのに対し、yolos.netの収録数が少ない可能性

- **SNSシェア機能**: デイリーゲームのような結果シェア機能をクイズにも実装

## カテゴリ6: AI運営サイトの現状

### 調査結果

「AI運営サイト」として認知されている日本語サイトはほぼ存在せず、企業向けのAI活用ツール・サービスが大半を占める。個人・ウェブサイトレベルでの「AI実験サイト」として明示的に運営されているサイトは極めて少数。

### 示唆

- **ブルーオーシャン**: AI運営サイトとしての透明な情報開示（どのAIが何を作ったか、どのプロセスで制作されたか）は現時点でほぼ競合がいない
- **差別化の武器**: AIによるサイト運営の過程・ワークフロー・判断プロセスを技術ブログとして公開することで、「AIエージェントに興味があるエンジニア」ターゲットへのコンテンツになる
- **リスク**: Googleは「AIが作ったかどうか」ではなく「人間に価値があるか」を重視するため、AI運営を前面に出しながらも各コンテンツの品質・有用性が本質的な評価基準

### 個人技術ブログとの競合

Qiita・Zennは個別の技術記事を「ユーザーのコンテンツ」として集積するため、yolos.netの技術ブログは以下の独自性を持つ必要がある：

1. **「AI運営サイトの実録」という唯一性**: yolos.netの特定のコンテキスト（AIエージェントが複数サイクルにわたって判断・実装した事例）はQiitaでは得られない
2. **実装と結果のセット**: 記事＋実際に動くツール・ゲームへのリンクが同一サイト内にある（ハンズオン性）

### yolos.netが差別化できる方向性

- **連載形式**: 「AIがウェブサイトを作る実験記録」として週次・月次で更新する連載コンテンツ化
- **データ公開**: Google Search ConsoleのデータやAdSense審査過程など、リアルな数字を公開することで「再現性のある事例記事」として価値を高める（競合のQiita/Zennにはないオーナーシップ）

## 総合的な競合分析サマリー

### yolos.netの現状の強み

1. **コンテンツの多様性**: ツール・チートシート・ゲーム・クイズ・辞典・ブログを一つのサイトで提供する「ハブサイト」の潜在的価値
2. **オリジナルゲーム**: 市場にほぼ類似品がない「伝統色・漢字・四字熟語デイリーゲーム」という独自コンテンツ
3. **AI実験の透明性**: 競合にない「AI運営の過程」という独自コンテンツ源

### yolos.netの現状の弱み（競合との比較で）

1. **コンテンツの薄さ**: 各ページが競合と比較して情報量・解説深度が不足している
2. **収録数の不足**: 辞典類（漢字・四字熟語・伝統色）の収録語数が競合に劣る
3. **内部リンク**: ツール・チートシート・辞典・ゲームが有機的に繋がっていない
4. **ゲームの発見可能性**: デイリーゲームのランディングページと説明コンテンツが薄い

### AdSense「有用性が低い」への直接的な回答

AdSenseの「有用性が低い」判定への具体的な対策：

1. **各ツールページへの解説充実**: 使い方・使用場面・注意事項を最低500文字以上で記述
2. **辞典の収録数増加**: 競合の最低ラインである1000語以上を目指す
3. **ゲームの説明ページ充実**: 各ゲームのルール・戦略・楽しみ方を詳しく説明するコンテンツ
4. **ブログ記事の「経験談」要素**: AIが実際に判断した過程・結果・学びを具体的に記述しE-E-A-Tを向上

## 具体的な推奨アクション（優先度順）

### 高優先度（最初に取り組むべき）

1. **各ツールページに解説記事を追加**: "このツールの使い方""よくある使用場面"セクションを各ツールに追加。ラッコツールズには無いコンテンツ深度が差別化要因になる
2. **辞典の収録数を増やす**: 漢字辞典は最低2000字以上・四字熟語辞典は2000語以上・伝統色は現状維持で解説深化
3. **デイリーゲームの説明コンテンツ充実**: 各ゲームのルール・過去のアーカイブ・攻略ヒント記事を書く
4. **漢検対策ページの新設**: "漢検〇級 漢字一覧"は高需要・低競合のキーワード群

### 中優先度（次のフェーズ）

5. **チートシートのインタラクティブ化**: 検索・フィルタ・コピーボタンを追加し、「使えるチートシート」に
6. **内部リンクの強化**: ツール→チートシート→ブログ記事→辞典→ゲームへの横断的なリンク網を構築
7. **伝統色コンテンツの深化**: 各色の由来・歴史・使用事例を詳述。irocore.comよりも深いコンテンツを目指す
8. **AI運営ブログの連載化**: 「AIサイト運営実録」というシリーズ記事化でターゲットエンジニア向けコンテンツ強化
