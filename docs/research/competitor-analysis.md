# yolos.net 競合調査レポート

## 調査概要

本レポートは、yolos.netが提供するカテゴリ（Webツール・チートシート・クイズ・辞典・デイリーブラウザゲーム・ことわざ・慣用句・技術ブログ）および関連領域（診断コンテンツ）の競合環境を調査し、競合との差別化ポイントと改善施策を整理したものです。

---

## カテゴリ1: Webツール（文字数カウント・変換・フォーマッタなど）

### 主要競合サイト

| サイト名                    | URL                                        | 特徴                                                                                                                                                                                                                     |
| --------------------------- | ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ラッコツールズ              | https://rakko.tools/                       | 100種類以上のツール（2021年時点、現在はさらに増加）、20以上のカテゴリ、16言語対応、SEO・SSL・テキスト・ネットワーク等の幅広い領域（出典: [value-press プレスリリース](https://www.value-press.com/pressrelease/238840)） |
| Web便利ツール@ツールタロウ  | https://tool-taro.com/                     | 約35種のWeb開発者向けツール。Whois、パスワード生成、QRコード、HTMLバリデーション等                                                                                                                                       |
| 文字数カウント (luft.co.jp) | https://www.luft.co.jp/cgi/str_counter.php | 文字種別分析・読了時間推定・原稿用紙換算・正規表現除外設定など高機能な単一ツール                                                                                                                                         |
| クロクロ・ツールズ          | https://crocro.com/tools/                  | 2015年開設、15種類の開発者向けWebアプリ群                                                                                                                                                                                |

### 競合の成功要因

- **ラッコツールズ**: 関連サービス（RakkoKeyword, RakkoServer等）とのエコシステム連携。無料・ブラウザ完結・16言語対応の圧倒的アクセシビリティ。100種類以上（2021年時点、現在はさらに増加）のツール群に「くじ引き」のような非技術者向けツールも含む幅広さ
- **ツールタロウ**: プライバシー配慮（サーバーで即座に破棄）を明示しているため開発者の信頼を獲得
- **luft文字数カウント**: 単一ツールに深く特化し、文字種別解析・原稿用紙換算・特殊文字検出まで徹底して深化させた圧倒的な機能充実度

### 競合が提供していてyolos.netが持っていないもの

1. **解説・使い方ガイド**: 競合の多くは各ツールに詳細な使い方説明、具体的な使用場面の例示を設けている
2. **ツール数の規模感**: ラッコツールズは100種以上（2021年時点、現在はさらに増加）に対しyolos.netは約35種（量より質が重要ではあるが、認知されるための品揃えが不足）
3. **ユースケースの説明**: "なぜこのツールが必要か"を具体的に説明するコンテキストが薄い

### yolos.netが差別化できる方向性

- **深度戦略**: ラッコツールズが100種以上を浅く広く提供するのに対し、各ツールページを「このツールの使い方・活用例・関連知識」まで深化させる
- **AI由来の透明性**: AI運営であることを活かし、「このツールのアルゴリズムと仕様」を技術的に丁寧に説明する（E-E-A-T強化）
- **学習コンテキスト**: ツール使用時に関連するチートシート・解説記事へのリンクを設け、サイト全体の内部リンク強化

## カテゴリ2: チートシート（Git・正規表現・Markdown・Cronなど）

### 主要競合サイト

| サイト名               | URL                                 | 特徴                                                                               |
| ---------------------- | ----------------------------------- | ---------------------------------------------------------------------------------- |
| Qiita（記事群）        | https://qiita.com/                  | Markdownチートシート記事が数万いいね。コミュニティ更新型で常に最新情報が更新される |
| GitHub Gist            | https://gist.github.com/            | GitHubユーザーによる非公式チートシートが多数。バージョン管理可                     |
| GitHub公式ドキュメント | https://docs.github.com/            | Gitチートシート公式PDF（日本語版あり）                                             |
| meganelog.site         | https://meganelog.site/cheat-sheet/ | 2025年版Web制作者向けHTMLCSS/VS Code/Cursor日本語チートシートまとめ                |
| coliss.com             | https://coliss.com/                 | フロントエンド開発者向けチートシートキュレーションサイト                           |

### 競合の成功要因

- **Qiita/Zenn**: コミュニティ由来の「実践から生まれた具体的なユースケース」が豊富。「自分が困ったこと＋解決策」の構造が検索意図に一致しやすい
- **GitHub公式**: 権威性（E-A-T）が高く「git チートシート」で上位表示されやすい
- **キュレーション系サイト**: 最新ツール（CursorなどAIエディタ）への対応が速い

### 競合が提供していてyolos.netが持っていないもの

1. **インタラクティブ要素**: 多くの競合はPDF/静的ページだが、インタラクティブな検索・フィルタ機能を持つものは少ない
2. **コピーボタン付きコードスニペット**: すぐコピペできるUI
3. **バージョン対応情報**: "Git 2.x系での変更点"など時系列情報

### yolos.netが差別化できる方向性

- **インタラクティブチートシート**: 単なる一覧ではなく、コマンド検索・フィルタ・コピーボタン付きの「使えるチートシート」に
- **実例・ユースケース付き**: 「このコマンドはどんな時に使うか」の具体例をセットで提供（Qiitaの記事形式との差別化）
- **バージョン履歴対応**: ツールのバージョンごとの差異を明示し、技術的信頼性を高める

### 競合の成功要因

- **四字熟語データバンク (sanabo.com)**: 圧倒的な収録数（9241語、サイト自己申告値）と「[世界最大の四字熟語情報サイト](https://sanabo.com/words/)」の権威性。テーマ別分類・人気ランキング・座右の銘カテゴリなど独自機能が豊富
- **四字熟語の百科事典**: 「楽しく学ぶ」というコンセプトを徹底し、学年別・場面別・テーマ別の多軸分類で幅広いユーザーをカバー
- **ベネッセ**: 教育ブランドの信頼性＋丁寧な由来・解説で「YMYL」的なクオリティ

### 競合が提供していてyolos.netが持っていないもの

1. **デイリー更新クイズ**: "毎日1問"形式が習慣化させる動線として機能している
2. **レベル分け**: 初級〜上級の難易度分類
3. **由来・背景の詳細解説**: クイズの正解後に「由来や歴史的背景」を詳しく説明するコンテンツ深度
4. **収録数の規模**: 競合が数千〜9000語以上なのに対し、yolos.netの収録数が少ない可能性

### yolos.netが差別化できる方向性

- **デイリー形式との統合**: 既存のデイリーゲームとクイズを統合し、「今日の四字熟語クイズ」をデイリーゲームとして提供（習慣的な再訪促進）
- **AI解説の深化**: AIを活用した四字熟語の「現代での使用例」「SNSでの用法」「誤用例」など現代的コンテキストの充実
- **SNSシェア機能**: デイリーゲームのような結果シェア機能をクイズにも実装

#### 診断コンテンツがSNSで拡散する理由

- **自己表現欲求**: 「こんな自分がいた」を他者に伝えたい心理
- **シェアインセンティブ**: 結果共有でキャンペーン参加、抽選応募などの仕組み
- **娯楽性**: 気軽に短時間で楽しめるため参加ハードルが低い
- 人気診断は数百万〜1,000万回近い利用回数に達することもある（サイト公開値。出典: [診断メーカー 総合ランキング](https://shindanmaker.com/list/overall)）

#### 2025年以降のトレンド

- Xのアルゴリズム変化（機械学習による行動予測モデル）により、単純な「拡散狙い」は効果が薄れつつある
- 「素材配布 × 大喜利」のような参加型コンテンツが新たなバズパターンとして台頭

#### yolos.netとの相性と差別化ポイント

- **高い相性**: yolos.netはインタラクティブコンテンツが得意であり、診断系はその延長線上にある
- **差別化ポイント**: 漢字・四字熟語・ことわざなど日本語知識系の診断（「あなたの語彙力は?」「あなたに合う四字熟語は?」など）はほぼ空白地帯
- **注意点**: 診断メーカーという強力なUGCプラットフォームが存在するため、「場所」より「コンテンツの質とオリジナリティ」で差別化する必要がある

- **伝統色の独自価値**: カラーコード＋歴史解説だけでなく、「その色が使われた歴史的作品・衣装・建築物の画像」など視覚的コンテンツを充実させ、デザイナーだけでなく日本文化ファンも取り込む
- **漢検対策ページ**: 「漢検N級 漢字一覧」ページは高需要キーワードを持つ。既存辞典コンテンツを漢検級別に再整理するだけでも高い検索流入が見込める
- **辞典×クイズの連携**: 辞典エントリから直接「この漢字のクイズに挑戦」ボタンを設ける内部リンク強化

1. **辞書型コンテンツの網羅性**: 既存競合は数千語規模の収録数を持つ
2. **逆引き検索**: 意味や場面から逆引きできる機能は差別化要因として強い
3. **学習段階別の構成**: 小学生・中学受験・高校入試・大学受験といった段階別アプローチ
4. **専門家監修による権威性**: E-E-A-Tの観点で重要

### 市場の規模感

- 「ことわざ」は日本語学習・教育の定番カテゴリであり、受験シーズン（秋〜冬）に検索数が増加するシーズナルな傾向がある
- kotowaza.jitenon.jp がSimilarWebに掲載されていることから、相当のアクセス規模があると推定される
- モバイルアプリ分野でも「ことわざクイズ - はんぷく一般常識」（App Store）などが人気を集めている

### 参入余地の分析

- **辞書系は参入余地が限定的**: 既存サイトがSEOを抑えており、同様の辞書コンテンツでは差別化が難しい。なお、大量のことわざエントリを機械的に生成する手法はGoogleの[スケールドコンテンツに関するスパムポリシー](https://developers.google.com/search/docs/essentials/spam-policies?hl=ja#scaled-content)に抵触するリスクがあるため、品質を伴わない大量生成は避けること
- **インタラクティブ・ゲーム要素での差別化は有望**: 既存競合はほぼ静的な辞書形式であり、ことわざを題材にしたゲーム（Wordleのことわざ版、穴埋めパズルなど）は競合が薄い
- **ことわざ × SNS拡散（診断形式）の組み合わせはほぼ空白地帯**: yolos.netのコンセプト（インタラクティブ・エンタメ性）との相性が非常に良い

### yolos.netが差別化できる方向性

- **ゲーム型アプローチ**: Wordleのことわざ版、穴埋めパズル、意味からことわざを当てるクイズなど、既存競合にないインタラクティブ体験
- **診断型コンテンツ**: 「あなたに合うことわざは?」のようなSNS拡散を狙った診断コンテンツ
- **既存コンテンツとの連携**: 四字熟語クイズ・漢字辞典との横断的リンクで日本語学習ハブとしての価値を高める

## カテゴリ7: AI運営サイトの現状

### 調査結果

「AI運営サイト」として認知されている日本語サイトはほぼ存在せず、企業向けのAI活用ツール・サービスが大半を占める。個人・ウェブサイトレベルでの「AI実験サイト」として明示的に運営されているサイトは極めて少数。

### 示唆

- **ブルーオーシャン**: AI運営サイトとしての透明な情報開示（どのAIが何を作ったか、どのプロセスで制作されたか）は現時点でほぼ競合がいない
- **差別化の武器**: AIによるサイト運営の過程・ワークフロー・判断プロセスを技術ブログとして公開することで、「AIエージェントに興味があるエンジニア」ターゲットへのコンテンツになる
- **リスク**: Googleは「AIが作ったかどうか」ではなく「人間に価値があるか」を重視するため（出典: [Google検索セントラル AI生成コンテンツに関するガイダンス](https://developers.google.com/search/docs/fundamentals/using-gen-ai-content)）、AI運営を前面に出しながらも各コンテンツの品質・有用性が本質的な評価基準

## カテゴリ8: 技術ブログ（Zenn/Qiita競合）

### 主要競合プラットフォーム

| プラットフォーム | 特徴                                                                           |
| ---------------- | ------------------------------------------------------------------------------ |
| Qiita            | 2011年開設、最大規模の日本語エンジニアコミュニティ、記事の検索流入が非常に強い |
| Zenn             | 2020年開設、GitHub連携、電子書籍販売、TypeScript/フロントエンド系に強い        |
| Note             | 技術以外も含む幅広いクリエイタープラットフォーム                               |

### 個人技術ブログとの競合

Qiita・Zennは個別の技術記事を「ユーザーのコンテンツ」として集積するため、yolos.netの技術ブログは以下の独自性を持つ必要がある：

1. **「AI運営サイトの実録」という唯一性**: yolos.netの特定のコンテキスト（AIエージェントが複数サイクルにわたって判断・実装した事例）はQiitaでは得られない
2. **実装と結果のセット**: 記事＋実際に動くツール・ゲームへのリンクが同一サイト内にある（ハンズオン性）

### yolos.netが差別化できる方向性

- **連載形式**: 「AIがウェブサイトを作る実験記録」として週次・月次で更新する連載コンテンツ化
- **データ公開**: Google Search ConsoleのデータやAdSense審査過程など、リアルな数字を公開することで「再現性のある事例記事」として価値を高める（競合のQiita/Zennにはないオーナーシップ）

## 総合的な競合分析サマリー

### yolos.netの現状の強み

1. **コンテンツの多様性**: ツール・チートシート・ゲーム・クイズ・辞典・ブログを一つのサイトで提供する「ハブサイト」の潜在的価値
2. **オリジナルゲーム**: 市場にほぼ類似品がない「伝統色・漢字・四字熟語デイリーゲーム」という独自コンテンツ
3. **AI実験の透明性**: 競合にない「AI運営の過程」という独自コンテンツ源

### yolos.netの現状の弱み（競合との比較で）

1. **コンテンツの薄さ**: 各ページが競合と比較して情報量・解説深度が不足している。なお、品質の低いコンテンツの削除・統合はconstitution.mdのRule 5（try a variety of things）とは矛盾しない。Rule 4（質の優先）に基づき、低品質コンテンツの整理は品質向上の一環として正当化される
2. **収録数の不足**: 辞典類（漢字・四字熟語・伝統色）の収録語数が競合に劣る
3. **内部リンク**: ツール・チートシート・辞典・ゲームが有機的に繋がっていない
4. **ゲームの発見可能性**: デイリーゲームのランディングページと説明コンテンツが薄い

### AdSense「有用性が低い」への直接的な回答

AdSenseの「有用性が低い」判定への具体的な対策：

1. **各ツールページへの解説充実**: 使い方・使用場面・注意事項を最低500文字以上で記述
2. **辞典の収録数増加**: 競合の最低ラインである1000語以上を目指す（ただし各エントリに独自の深い解説を伴うことが条件。量のみの拡大はGoogleの[スケールドコンテンツに関するスパムポリシー](https://developers.google.com/search/docs/essentials/spam-policies?hl=ja#scaled-content)に抵触するリスクがあるため、品質を伴わない大量生成は避けること）
3. **ゲームの説明ページ充実**: 各ゲームのルール・戦略・楽しみ方を詳しく説明するコンテンツ
4. **ブログ記事の「経験談」要素**: AIが実際に判断した過程・結果・学びを具体的に記述しE-E-A-Tを向上

## 具体的な推奨アクション（優先度順）

### 高優先度（最初に取り組むべき）

1. **各ツールページに解説記事を追加**: "このツールの使い方""よくある使用場面"セクションを各ツールに追加。ラッコツールズには無いコンテンツ深度が差別化要因になる
2. **辞典の収録数を増やす**: 漢字辞典は最低2000字以上・四字熟語辞典は2000語以上・伝統色は現状維持で解説深化。ただし各エントリに独自の深い解説を伴うことが条件であり、品質を伴わない大量生成はGoogleの[スケールドコンテンツに関するスパムポリシー](https://developers.google.com/search/docs/essentials/spam-policies?hl=ja#scaled-content)に抵触するため避けること
3. **デイリーゲームの説明コンテンツ充実**: 各ゲームのルール・過去のアーカイブ・攻略ヒント記事を書く
4. **漢検対策ページの新設**: "漢検〇級 漢字一覧"は高需要・低競合のキーワード群

### 中優先度（次のフェーズ）

5. **チートシートのインタラクティブ化**: 検索・フィルタ・コピーボタンを追加し、「使えるチートシート」に
6. **内部リンクの強化**: ツール→チートシート→ブログ記事→辞典→ゲームへの横断的なリンク網を構築
7. **伝統色コンテンツの深化**: 各色の由来・歴史・使用事例を詳述。irocore.comよりも深いコンテンツを目指す
8. **AI運営ブログの連載化**: 「AIサイト運営実録」というシリーズ記事化でターゲットエンジニア向けコンテンツ強化

### 低優先度（余裕ができたら）

9. **SNSシェア機能の強化**: ゲーム・クイズの結果シェアUI改善
10. **多軸検索の実装**: 辞典の検索機能強化（部首・漢検級・テーマ別）
11. **診断型コンテンツ**: 「今日の伝統色」「あなたの漢字」のような診断要素でエンゲージメント向上
12. **ことわざ・慣用句ゲームの新設**: ことわざWordle・穴埋めパズルなどインタラクティブ型コンテンツの開発

### 検索ボリュームから見た戦略的示唆

- **即効性が高いカテゴリ**: Webツール（QRコード・JSON整形など）は実用的で反復利用率が高く、安定したアクセスを見込める
