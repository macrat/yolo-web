# 人気診断・クイズサイト結果ページのUI構成調査（競合比較）

## 調査の制約

16Personalities（Cloudflare保護）、診断メーカー（Cloudflare保護）はPlaywrightのボット検出によりアクセス不能だった。代替として、既存リサーチ（2026-03-31-japanese-casual-diagnosis-result-page-analysis.md）の詳細データ、BuzzFeed（直接アクセス可能）、mgramのトップページ（直接確認）、WebFetchによるBuzzFeedの内部構造解析を活用した。

---

## 各サイトの結果ページ構成

### 1. 診断メーカー (shindanmaker.com)

既存調査より確認。

**要素の配置順序（上から下）:**

1. 診断タイトル（H1）
2. 名前入力フォーム（再診断用）
3. 結果テキスト（1〜2文、40〜200文字）
4. **Xシェアボタン（1種のみ）**← 結果テキストの直下
5. 「みんなの診断結果」（他ユーザーのツイート一覧）

**シェアボタンの特徴:**

- 結果テキストの直下（テキストとボタンの間に別コンテンツが入らない）
- X（旧Twitter）への単一ボタン
- ボタンデザインはシンプルな青のテキストボタン
- モバイル版ではWeb Share APIボタンも追加（OSのシェアシートを呼び出す）

**来訪者の体験フロー:**
名前を入れて診断 → 結果テキストを読む → 直下のシェアボタンをタップ → Xに投稿、という最短距離の設計。結果を読んで「笑える」「友達に見せたい」と思った瞬間にボタンが視界に入る。ページをスクロールする必要がない（1スクロール以内）。

---

### 2. mgram (mgram.me)

既存調査・Playwrightによるトップページ確認より。

**要素の配置順序（上から下）:**

1. 「私を構成する8性格」ハッシュタグクラウド（カラフルなタイル表示 ← シェア用のビジュアルになっている）
2. レーダーチャート（8次元の性格グラフ）
3. 「私の4つの特異性」棒グラフ（日本人平均との比較）
4. 「運命の人に出逢える確率」数値表示
5. **SNSシェアボタン**
6. 有料版への誘導CTA

**シェアボタンの特徴:**

- データビジュアライゼーション（グラフ群）の後に配置
- SNSシェアボタン（複数、X・Twitter等）
- シェア用のビジュアルフォーマット（ハッシュタグクラウド）が自動生成される仕組み

**来訪者の体験フロー:**
105問という長い診断を経て到着する結果ページ。まず「私を構成する8性格」ハッシュタグが視覚的に飛び込んでくる（これがmgramの代名詞的なビジュアル）。グラフを眺めてデータを吸収した後、シェアボタンへ到達する。長い質問への投資感があるため、シェアモチベーションは高い。ハッシュタグクラウドは「そのままスクショしてシェアしたくなる」ビジュアルになっている。

---

### 3. BuzzFeed (buzzfeed.com/jp)

Playwrightによる直接アクセス・WebFetchによる構造解析より。

**要素の配置順序（上から下、クイズ形式）:**

1. 記事タイトル（H1）
2. 著者・公開日
3. 質問タイル（選択肢を画像付きタイルで表示）
4. ← 選択肢を全て答えると自動で結果表示 →
5. 結果カード（result-card: グラデーション背景 + 結果画像 + 結果テキスト）
6. **シェアボタン群（Facebook・Twitter・WhatsApp・LINE・コピーの5〜6種）**← 結果カード直後
7. 「もう一度やってみる」ボタン（retake）
8. 関連記事・他のクイズへのリンク

**シェアボタンの特徴:**

- 結果カードの直後（ページ下部ではない）
- 複数SNSに対応（Facebook・Twitter・WhatsApp・LINE・Pinterest・コピー）
- シェアボタン群が「share-container」として結果カードとセットになっている
- 「retake」（再挑戦）ボタンはシェアボタンの後ろに配置

**来訪者の体験フロー:**
最後の質問に答えると画面が自動でスクロールして結果カードが表示される（タイルを選ぶ瞬間が一種のゲーム感）。結果カードが現れた直後にシェアボタンが続く。「自分はどのキャラだろう？」という好奇心の答えが出た興奮のまま、すぐ下のシェアボタンを押せる設計。

---

### 4. 16Personalities (16personalities.com)

Cloudflareブロックにより直接確認不能。既存調査・公開情報より。

**結果ページの推定構成（公開情報・サードパーティ解説より）:**

1. MBTIタイプのビジュアルバナー（カラーグラデーション + タイプコード）
2. タイプ名（例：「提唱者型 - INFJ」）
3. 5つの軸スコアバー（E/I・N/S・T/F・J/P・A/T）
4. タイプの概要説明文（数百文字）
5. 詳細説明への誘導（内部リンク、ログイン促進）
6. シェアボタン（比較的控えめな配置）
7. 詳細なタイプ解説セクション（長文）

**シェアボタンの特徴:**

- 5軸スコアの表示後、詳細説明前後に配置（目立つ位置ではない）
- ログイン/アカウント作成へのCTAが優先されている
- 「メール送信」や「印刷」などを含むシェア形式

**来訪者の体験フロー:**
世界的に人気があり、「SNSでMBTIを語る文化」が既に形成されているため、シェアボタンがなくてもユーザーが自発的にスクショをシェアする設計。シェアボタンより「アカウント登録してもっと詳しく見る」CTAを優先している。

## 横断比較：シェアボタンの位置に関する共通パターン

| サイト                            | シェアボタン位置               | 結果コンテンツとの前後関係         | ボタンの目立ち具合             |
| --------------------------------- | ------------------------------ | ---------------------------------- | ------------------------------ |
| 診断メーカー                      | 結果テキスト直下               | 詳細説明なし・その他コンテンツなし | 最も目立つ（唯一のアクション） |
| BuzzFeed                          | 結果カード直後                 | retake(再挑戦)より前               | 目立つ（6種類のSNSボタン群）   |

## 分析：なぜこの配置なのか（体験フローの観点）

### 「シェアしたい衝動のピーク」に合わせる設計原則

来訪者がシェアしたいと思う瞬間は、診断の種類によって異なる。

**短文診断型（診断メーカー）:** 結果テキスト（1〜2文）を読んだ瞬間がピーク。だから結果の直下にボタンを置く。このタイプはシェアが主目的のため、シェアボタンを「唯一の行動選択肢」として際立たせている。

**タイル選択型（BuzzFeed）:** 最後の選択肢を選ぶ「ゲームの終わり」がピーク。結果カードが現れた直後にシェアボタンを置くことで、興奮冷めやらぬうちにシェアを促す。retake（再挑戦）はシェアの後に置かれ、シェアの機会を先に与えている。

**データ蓄積型（mgram）:** 105問という投資を経てデータが可視化された瞬間がピーク。ハッシュタグクラウドというSNS映えするビジュアルが最初に現れるため、「これをシェアしたい」という衝動がビジュアルと共に生まれ、その後のシェアボタンに繋がる。

### retake（再診断）ボタンとシェアボタンの順序

BuzzFeedは「シェアボタン → retake」の順。診断メーカーは「シェアボタン（のみ）」でretakeは名前フォームとして常時表示。

どちらも「シェアを先にする」設計であり、「もう一度やる」を先に提示するとシェアへの動線が切れる可能性を避けている。

## yolos.netへの示唆

1. **シェアボタンはdescription直後が正解。** 競合（診断メーカー・BuzzFeed）はいずれも「結果コンテンツの直後」にシェアボタンを配置している。detailedContentの後ではなく前が正しい位置。

2. **detailedContent後にもシェアボタンを設ける。** detailedContentを全部読んだ後に「やっぱりシェアしよう」と思った来訪者がシェアボタンを探すことになる。ダブル配置が最も来訪者フレンドリーである（競合でこれを明示的にやっているサイトは少ないが、来訪者心理からは合理的）。

3. **シェアボタンはretake（再診断）ボタンより前。** BuzzFeedと診断メーカーがそれを示している。trySection（診断してみようボタン）の後ろにシェアボタンを置く現在のデプロイ済み配置は逆転している。

4. **シェアボタンのビジュアルは「目立つ」方が良い。** 診断メーカーは1種類だけで最大限目立たせている。BuzzFeedは複数種をカラフルに並べている。yolos.netのX/LINE/コピーの3種は適切なサイズで目立つ配置が必要。
