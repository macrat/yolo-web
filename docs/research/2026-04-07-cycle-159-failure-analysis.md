# cycle-159 失敗分析: fullwidth-converterへの解説コンテンツ追加とUX悪化

関連サイクル: cycle-160（2026-04-06T18:27:46+0900 〜 20:04:56+0900）

## 1. 何が追加され、なぜ削除されたのか

### 追加されたもの

cycle-159のStep 5において、fullwidth-converter（全角半角変換ツール）に以下が追加された:

- **解説コンテンツ3セクション**（ツールUI下部に追加）
  - 「全角と半角の違いとは」セクション: 全角・半角の定義、文字コードの背景（Unicode）
  - 「全角半角変換が必要になる場面」セクション: データ入力の統一、CSVの処理、フォーム入力の正規化等の具体的ユースケース
  - 「変換対象の文字一覧」セクション: 数字/英字/カタカナ/記号/スペースの全角半角対応を表形式で表示
- **FAQを3問から5問に拡充**
- **descriptionを65文字から124文字に拡充**
- **全ツールのtitle生成ロジック修正**（「全角半角変換 - tools | yolos.net」→「全角半角変換 | yolos.net」）

### 削除された理由

Ownerが「来訪者のUXを悪化させている」と指摘した。具体的には2つの問題が生じていた:

1. **モバイルでツール本体がファーストビューから押し出された**: description拡充（65→124文字）により、ツール本体の前に表示されるテキスト量が増加し、モバイル（375x667px）でツール本体がファーストビューの外に押し出された。「全角半角変換」で検索してページに来た来訪者が、スクロールしないとツールを使えない状態になった。

2. **解説コンテンツが来訪者の目的と一致していない**: 「全角半角変換」で検索してページに来る来訪者の目的は「テキストを変換すること」であり、「全角と半角の違いとは何か」「変換が必要な場面はどこか」を学ぶことではない。解説コンテンツはSEO的なテキストコンテンツの増量が目的であり、来訪者の即時ニーズを充足するものではなかった。

ビジュアルテストと実装レビューでは「計画通りに表示されているか」の観点のみで確認し、「来訪者として最初にページを開いたとき、最高の体験か」の観点での確認を行わなかった。そのため問題が検出されなかった。

## 2. 「来訪者価値」と「SEO施策」の混同とは何か

### 根本にある認識の誤り

cycle-159の計画では「競合（so-zou.jp）は解説コンテンツがない。解説コンテンツを追加すれば差別化できる」と分析していた。この分析自体は間違いではない。しかし、問題は **解説コンテンツが「誰のためのものか」を正確に問わなかった** ことにある。

解説コンテンツが追加された理由は（cycle-159の計画を読む限り）:

- 「薄いコンテンツと判定されている可能性を解消する」（Googleへの対応）
- 「検索エンジンに対してページの専門性を示す」（SEO的なシグナル）
- 「so-zou.jpに対する差別化ポイント」（競合分析の結論）

来訪者への価値として計画書に書かれていたのは「来訪者が実際に読んで役に立つ内容にすること」という注意書きのみで、これは「来訪者の何の問題を解決するのか」という問いに答えていない。

### 混同のメカニズム

「SEO施策」は「Googleに高く評価されるためにやること」であり、「来訪者価値」は「来訪者の問題を解決したり来訪者を喜ばせたりすること」である。この2つは重なる部分があるが、同一ではない。

混同が生じるのは次のようなロジックで動いたときである:

- 「解説コンテンツを追加すれば、Googleはページを評価しやすくなる」（SEO仮説）
- 「Googleが評価するということは来訪者に役立つということだ」（誤った等式）
- よって「解説コンテンツ追加 = 来訪者価値向上」（誤った結論）

実際には、Googleがページのテキスト量を評価することと、そのテキストがその瞬間にそのページを訪れた来訪者の役に立つことは別の話である。

cycle-159の計画では「来訪者にとって実際に役立つ内容にすること（constitution Rule 4: 品質優先）」という注記があったが、「役立つ内容かどうか」の検証を来訪者の目的から逆算して行わなかった。代わりに「競合が持っていないから差別化になる」「Googleが高く評価するから良い」という別の基準で判断した。これが混同の実体である。

### AP-P09として記録されているアンチパターン

この失敗はdocs/anti-patterns/planning.md のAP-P09として記録されている:

> AP-P09: 計画のゴールが「higher page views by providing the best value for visitors」に正確に対応せず、「Googleの評価向上」「SEOスコア改善」「ドメイン権威向上」など別のゴールに読み替えていないか?

cycle-159では、constitutionのゴール（来訪者への最高の価値提供）が「Googleの評価向上」というゴールに読み替えられた。

### 対比の核心

cycle-159の失敗とcycle-160の成功の違いは「設計の出発点」にある。

cycle-159は「現状のツールに何を追加するか」を考えた。追加するものを「SEOに効くもの」から選んだ。来訪者は後から「役に立つ内容にする」という修飾語として登場した。

cycle-160は「来訪者が何を求めているか」から考えた。求めていることを実現するためにページ構成を設計した。SEOは「来訪者体験を妨げない範囲で考慮する」という制約条件として扱われた。

cycle-160の計画書には明示的に書かれている:

> いずれのパターンでも、来訪者が最も早くツール本体にたどり着けることが最重要。

howItWorksをツール本体の「下」に配置した理由もここにある:

> 「ツールを使ってみてもう少し詳しく知りたい」と思った来訪者、または使う前に仕組みを確認したい来訪者のための補助情報。ゾーン1ではなくゾーン3に置くことで、ファーストビューを圧迫しない。

cycle-159が追加しようとした解説コンテンツと、cycle-160が追加したhowItWorksは表面的には同種のものに見えるが、設計の論理が根本的に異なる。

## 4. 教訓: cycle-159と同じ失敗を避けるために

### cycle-163のツール系ページSEO強化を行う際のチェックポイント

#### チェック1: 施策の出発点を確認する

施策が「来訪者の何の問題を解決するのか」から出発しているか、「SEOスコアを上げるために何を追加するか」から出発していないか。

- 誤った出発点の例: 「テキストコンテンツが少ない → 解説コンテンツを追加してGoogleの評価を上げよう」
- 正しい出発点の例: 「このツールを使いに来た来訪者が求めているものは何か → ツールにすぐアクセスできること → ファーストビューにツールを配置しよう」

#### チェック2: 来訪者の目的を明示する

「このページに来る来訪者の目的は何か」を具体的に書き出す。「全角半角変換で検索してきた人の目的はテキストを変換することであり、全角半角の歴史を学ぶことではない」のように。その目的に対して施策がどのように貢献するかを説明できなければ、その施策は来訪者価値を生まない可能性が高い。

#### チェック3: 追加要素がファーストビューに与える影響を事前に評価する

コンテンツを追加する際は、モバイル（375x667px）でツール本体がファーストビューに表示されるかを事前に確認する。ツール本体がファーストビューから押し出される変更は、来訪者価値を損なう。

#### チェック4: ビジュアルテストを「来訪者目線」で実施する

「計画通りに表示されているか」だけでなく「来訪者として最初にページを開いたとき、目的を達成できる最高の体験か」を評価する。cycle-159ではビジュアルテストで「PC/モバイル/ダークモード全項目問題なし」と判定されたが、「来訪者がツールにたどり着けるか」という観点での確認を行わなかった。

#### チェック5: SEO効果と来訪者価値が矛盾していないかを確認する

SEO施策を計画する際は「この施策は来訪者の体験を妨げないか」を常に問う。妨げる場合は、妨げない形に設計し直す（「妨げるが効果が大きいから許容」は認めない）。

cycle-160が示した解は「SEOテキストコンテンツをツール本体の下に配置し、来訪者が使った後に自然に到達する位置に置く」ことで両立させることだった。これは「追加するかしないか」ではなく「どこに配置するか」という設計の問題だった。

### 最も重要な原則

constitutionのゴールは「achieve higher page views by providing the best value for visitors」である。SEO施策は「page views を増やす手段の一つ」であり、「best value for visitors」と矛盾してはならない。

「Googleの評価を上げれば来訪者が増える」は一つの仮説であり、手段と目的の混同ではない。しかし「Googleの評価を上げるために来訪者体験を犠牲にする」は手段と目的の逆転であり、constitutionに違反する。
