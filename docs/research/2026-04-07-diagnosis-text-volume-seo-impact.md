# 診断サイトのタイプ別結果ページにおけるテキスト量とSEO影響の調査

## エグゼクティブサマリー

yolos.netの診断結果ページへの自然検索流入は、2026年3月〜4月にかけてゼロであることをGoogle Analyticsで確認した。「初志貫徹 性格」「ニホンザル 性格 診断」等の関連キーワードの実際のSERP（検索結果）でもyolos.netは一切表示されない。

主要因は**テキスト量の絶対的不足**である。競合が1,200〜5,500文字のコンテンツを提供する中、yolos.netの結果ページ本文は約300〜400文字しかない（ページ全体でも約1,200〜1,750文字）。このテキスト量では、Googleがページを「診断タイプについての解説コンテンツ」として認識できず、関連キーワードでの掲載に至らない。

## 4. 「指名検索しか機能しない」問題の構造的原因

### 原因1：本文テキスト量が競合比で1/5〜1/10

「ニホンザル 性格 診断」でGoogle検索した際、yolos.netは表示されない（実際にWebSearchで確認）。代わりに「おさるランド（osaruland.jp）」「ホンシェルジュ（honcierge.jp）」等の動物生態解説サイトが上位。yolos.netのページに存在するニホンザル関連テキストは動物生態の説明数十文字と性格特徴3項目程度（総計約150〜200文字）で、Googleが「ニホンザルについてのコンテンツ」と認識するには不十分。

「初志貫徹 性格 特徴 診断」でGoogle検索した際も、yolos.netは表示されない。上位は「quizcastle.com」「motivation-up.com」「weblio.jp」など、「初志貫徹」という言葉の意味・特徴を2,000文字以上で解説するページ。

### 原因2：コンテンツが「診断インターフェース」として設計されており「解説記事」ではない

yolos.netの結果ページは、診断完了後のユーザーが自分の結果を確認するUIとして設計されている。descriptionフィールドはシェア用の短文として機能するテキストであり、SEOの観点では「検索クエリに対する答え」として機能するテキスト量を持っていない。

検索エンジンが評価するのは「このキーワードで検索したユーザーにとって有用な情報があるか」であり、現状の構成はその判断ができないレベルのテキスト量。

### 原因3：検索意図のミスマッチ

「ニホンザル 性格」「初志貫徹 特徴」で検索するユーザーは、そのキーワードについて詳しく知りたい人である（Knowクエリ）。yolos.netの結果ページは「診断結果の確認」というDoクエリのユーザーを想定した構成になっており、Knowクエリへの対応テキストが存在しない。

### 原因4：内部リンク構造が外部クローラーに対して薄い

各結果ページは「他の動物も見てみよう」のリンク集はあるが、それぞれのページへの深い誘導はない。uranaino.netのように「このタイプと相性の良いタイプ → 対応ページへのリンク」という構造がなく、クローラーがサイトの深い構造を把握しにくい。

## 5. ロングテール検索流入を得るために必要なテキスト要素

以下は競合サイトの実態調査と一般的なSEO知見を組み合わせた分析。

### 必須要素（テキスト量の観点）

**本文最低2,000文字**が実際の競合の実態から見た目安（uranaino.net記事参照）。ただし2,000文字を「詰め込む」のではなく、以下の各テーマを適切に展開した結果として2,000文字になることが重要。

### 流入を生むコンテンツ軸（検証済みパターン）

1. **タイプの性格説明**（現状あり、ただし量が不足）
   - 現状：150〜200文字
   - 競合基準：400〜600文字
   - 対象キーワード：「〇〇タイプ 性格」「〇〇 診断 特徴」

2. **強み・弱みの詳細説明**（現状あり、ただし箇条書きのみ）
   - 現状：各1〜3行の箇条書き
   - 競合基準：各項目を段落で展開（100〜200文字/項目）
   - 対象キーワード：「〇〇タイプ 強み」「〇〇 短所 仕事」

3. **恋愛・人間関係傾向**（現状なし）
   - 現状：なし
   - 競合基準：150〜300文字
   - 対象キーワード：「〇〇タイプ 恋愛」「〇〇 相性」「〇〇 付き合い方」
   - 注：このキーワードは診断系サイトで最も検索ボリュームが高い部類

4. **仕事・適職傾向**（現状なし）
   - 現状：なし
   - 競合基準：150〜300文字
   - 対象キーワード：「〇〇タイプ 仕事」「〇〇 向いている 職業」

5. **有名人・キャラクター情報**（現状なし）
   - 現状：なし
   - 競合基準：5〜15件のリスト
   - 対象キーワード：「〇〇タイプ 有名人」「〇〇に似たキャラ」
   - 注：SNS拡散性が高く、被リンク獲得にもつながりやすい

6. **あるある・行動パターン**（現状あり）
   - 現状：4項目の箇条書き（良い状態）
   - 対象キーワード：「〇〇タイプ あるある」「〇〇 行動」

7. **タイプ間相性情報**（現状あり・動物診断のみ）
   - 現状：動物診断にはcompatibilityが存在する
   - 対象キーワード：「〇〇と〇〇 相性」「〇〇タイプ 合わない」

### yolos.netの診断タイプ別・有効なテキスト軸

**動物性格診断の場合**:

- 「ニホンザル タイプ 性格」→ タイプ説明の充実（現在50文字程度のキャッチフレーズのみ）
- 「ニホンザル 診断 仕事」→ 仕事傾向の追加（現状なし）
- 「ニホンザル 恋愛 相性」→ 恋愛傾向の追加（現状なし）
- 「ニホンザル 有名人」→ 有名人リストの追加（現状なし）
- 「動物性格診断 ニホンザル」→ 診断タイトルとの組み合わせ（現状対応）

**四字熟語性格診断の場合**:

- 「初志貫徹 タイプ 診断」→ 診断とのひもづけを強化

## 6. 「検証済みの事実」と「推測」の峻別

### 検証済みの事実（本調査で直接確認）

- yolos.netの動物性格診断結果ページの本文テキスト量は約350文字（Playwright実測）
- yolos.netの四字熟語性格診断結果ページの本文テキスト量は約300文字（Playwright実測）
- 2026年3月〜4月のyolos.netへの自然検索流入において、/play/_/result/_ ページへの流入はゼロ件（Google Analytics実測）
- 「ニホンザル 性格 診断」「初志貫徹 性格 特徴 診断」のWebSearchでyolos.netは表示されない（WebSearch実測）
- hoyme.jpの猫診断結果ページのページ全体テキスト量は約2,188文字（Playwright実測）
- uranaino.netのINFJ記事のページ全体テキスト量は5,541文字（Playwright実測）
- commutest.comのエゴグラム結果ページのページ全体テキスト量は2,788文字（Playwright実測）
- mbti-navi.comのINFJ解説ページのページ全体テキスト量は4,109文字（Playwright実測）
- yolos.netの結果ページのrobotsメタタグは `index, follow`（インデックスは正しく許可）

### 推測（根拠はあるが直接確認していない）

- 「テキスト量が少ないからGoogleが評価していない」という因果関係：技術的にrobots/canonical設定が正常であることと、実際にキーワードでSERPに表示されないことから強く推測できるが、他の要因（ドメインオーソリティ、リンク数など）の影響も否定できない。
- 「2,000文字を超えると自然検索流入が発生する」という具体的な閾値：SEO専門メディア（ds-b.jp等）で言及されており、競合の実態とも整合するが、キーワードによって異なるため絶対値ではない。
- 「恋愛・仕事傾向のコンテンツ追加がロングテール流入に最も効果的」：uranaino.netの記事構成からの類推であり、yolos.netの特定の診断コンテンツで同じ効果が得られるかは未検証。

## 7. 具体的な対策の方向性（本調査から導出）

### 優先度：高

**各結果ページの本文を現状の10倍規模（2,000〜3,000文字）に拡充する**

現状の結果ページ構成に以下のセクションを追加することで達成可能：

| 追加セクション               | 推定文字数   | 対応キーワード                       |
| ---------------------------- | ------------ | ------------------------------------ |
| タイプ性格の詳細説明（散文） | 300〜500文字 | 「〇〇タイプ 性格」                  |
| 恋愛・人間関係の傾向         | 200〜300文字 | 「〇〇タイプ 恋愛」「〇〇 相性」     |
| 仕事・得意分野の傾向         | 200〜300文字 | 「〇〇タイプ 仕事」「〇〇 向いてる」 |
| このタイプの有名人・キャラ   | 100〜200文字 | 「〇〇タイプ 有名人」                |
| ストレス・苦手なこと         | 150〜200文字 | 「〇〇タイプ 苦手」「〇〇 疲れる」   |

これにより、1タイプあたり約2,000〜3,000文字の独立したコンテンツページとなり、複数のロングテールキーワードに対応できる。

### 優先度：中

- 各タイプの metadescription を診断結果の短文から、キーワードを含む180文字程度のものに変更
- 構造化データ（JSON-LD）の実装（現状の確認が必要）
- 相性情報のあるページ（動物診断）での相性キーワード対応の強化

### 注意点

- 内容の独自性：競合と同じ情報を並べるだけでは評価されにくい。yolos.netの診断の特徴（日本固有種をベースとした動物診断など）を活かした独自の切り口が必要。
- 信頼性の表記：commutest.comのような専門家監修の明記はyolos.netの方針（AIによる実験的コンテンツ）と相容れないが、エンターテインメントであることを明示しつつ独自性を高める方向が現実的。
