# 診断・クイズ結果ページのモバイルUX実地体験レポート

## 重要な前置き：ソースコードとデプロイ済みサイトの乖離

調査時点（2026-03-31）において、**ローカルのソースコード（page.tsx）と実際にデプロイされているサイトの構造が異なっている**ことが判明した。

### ソースコード（page.tsx）の構造

```
icon → title → quizName → description → [shareSection] → [detailedContent] → trySection
```

shareSectionはdescription直後（detailedContentより前）に配置されている。

### 実際のデプロイ済みサイトの構造

DOM要素の垂直位置（JavaScript計測）では：

```
icon(147px) → title(232px) → quizName(333px) → description(372px) → detailedSection(730px) → trySection(1690px) → shareSection(1822px)
```

shareSectionはtrySection（診断してみようボタン）の後ろに配置されている。

**これはソースコードが変更されたが、まだデプロイされていないことを示している。**

以下のUX体験レポートは**現在のデプロイ済みサイト（旧配置）**に基づいている。

## ページ1: あなたに似たキャラ診断（detailedContentあり）

URL: https://yolos.net/play/character-personality/result/blazing-strategist

### ファーストビュー（スクロールなし、0px〜812px）

画面上部にナビゲーションバー（yolos.netロゴ、検索・テーマ・ハンバーガーメニュー）、パンくずリスト（ホーム / 遊ぶ / あなたに似たキャラ診断 / 結果）。

その下にアイコン絵文字（🌩️ 雷雲マーク）が大きく表示され、続いて診断タイトル「締切3分前に5手先を読む炎の策士」がh1で太字大文字表示される。「あなたに似たキャラ診断の結果」という小さなサブテキスト。

descriptionテキスト（「おっ、来たな。お前、追い込まれてから...」）が長文で続き、画面の下部では「あなたの特徴」という見出しと最初の特徴テキストが切れた状態で見えている。

**ファーストビューで見えるもの**: アイコン、タイトル、クイズ名、長いdescriptionテキスト、「あなたの特徴」セクションの先頭部分

**シェアボタン**: ファーストビューには一切見えない。

### スクロール体験（測定値）

各要素の位置（ページトップからのpx）：

| 要素             | 開始位置   | 内容                                                                                                 |
| ---------------- | ---------- | ---------------------------------------------------------------------------------------------------- |
| icon             | 147px      | 🌩️絵文字                                                                                             |
| title            | 232px      | 締切3分前に5手先を読む炎の策士                                                                       |
| quizName         | 333px      | あなたに似たキャラ診断の結果                                                                         |
| description      | 372px      | 長文診断テキスト（358px分）                                                                          |
| detailedSection  | 730px      | あなたの特徴 + 特徴リスト + こんなところ、ありませんか？ + behaviorsリスト + adviceカード（計960px） |
| trySection       | 1690px     | あなたはどのタイプ？診断してみようボタン（108px）                                                    |
| **shareSection** | **1822px** | **Xでシェア / LINEでシェア / コピーボタン（153px）**                                                 |

**総ページ高さ（メインカード部）**: 1975px
**シェアボタンに到達するまでのスクロール量**: 約1010px（ビューポート高さの1.25画面分）

### 来訪者がシェアしたいと思うタイミングとシェアボタンの位置

来訪者が「面白い、友達に見せたい」と思う瞬間は2段階ある。

**第1段階**: descriptionを読んだとき（372〜730px付近）。「これ自分のことだ！」という共感・驚きが生まれる。この時点ではシェアボタンは画面外の1822px地点にある。来訪者が「シェアしたい」と衝動的に思った瞬間にシェアボタンが見えない。

**第2段階**: detailedContent（traits、behaviors）を読み進めたとき。「あるある！」という追加の共感が積み重なる。しかし、このタイミングでもシェアボタンはまだ1090px以上下にある（730px + 960px = 1690px のtrySectionより下）。

診断ボタン（trySection: 1690px）を過ぎてさらにスクロールした1822px地点でようやくシェアボタンが現れる。

**問題**: 来訪者がシェアしたいと思った瞬間（descriptionを読み終えた頃）から、シェアボタンにたどり着くまでに約1450px（ほぼ2画面分）のスクロールが必要。

### ページの流れの自然さ

descriptionを読んで「面白い」と思った直後に、「あなたの特徴」という追加コンテンツが始まる。これはengagingだが、シェアへの動線が絶たれる。

detailedContent → trySection（診断ボタン）→ shareSection という順序は、来訪者の気持ちの流れとずれている。診断ボタンを押したい気持ちとシェアしたい気持ちが混在する中で、診断ボタンを先に見てしまうと「シェアする前に診断してみよう」と思う来訪者が多い可能性がある。

## 全体所見：来訪者目線での率直な感想

### character-personality（detailedContentあり）

**良い点**:

- アイコン+タイトルの組み合わせが視覚的にインパクトがある
- descriptionの文体（キャラクターが語りかけてくる口語体）が個性的で印象に残る
- detailedContentの「あなたの特徴」「こんなところ、ありませんか？」が非常に共感を呼ぶ内容で、読み進めるほど「これ私のことだ！」という感情が高まる
- adviceカードが来訪者への直接メッセージとして機能している

**問題点**:

- シェアボタンが最後に追いやられている（デプロイ済み版の問題）
- descriptionを読んで「シェアしたい！」と思った瞬間にシェアボタンが見えない
- detailedContentを全部読み終えた後、診断ボタン（trySection）が現れるため、「もう一度診断しようかな」という思考に転換してしまい、シェアへの動線がさらに薄れる

### kanji-level（detailedContentなし）

**良い点**:

- シェアボタンがファーストビュー内に収まっており、シェアしたい気持ちがあれば即座にできる
- ページ構造がシンプルで迷いがない

**問題点**:

- コンテンツが薄く、「友達に見せたい！」という衝動を生む感情的なフックが弱い
- 「漢字中級者」という結果タイトルと短いdescriptionだけでは、シェアする動機が弱い

## ソースコード変更後（新配置）の予測

現在のソースコード（page.tsx）では、shareSectionがdescriptionの直後に移動している。デプロイされれば：

character-personalityの新配置:

```
icon → title → quizName → description → [shareSection] → detailedContent → trySection
```

shareSection位置は約730px付近（descriptionの直後）になる見込み。ビューポート（812px）内にほぼ収まり、descriptionを読み終えたタイミングでシェアボタンが目に入るようになる。

これは**来訪者がシェアしたいと思う瞬間（description読了直後）にシェアボタンが見える**という観点では大幅に改善される。

ただし懸念点もある：descriptionを読み終えた直後にシェアボタンが表示されるが、その後detailedContentが続く。来訪者がシェアボタンを素通りしてdetailedContentを読んでしまうと、「やっぱりシェアしよう」と思ったときに再度シェアボタンを探す必要が生じる。detailedContent後のshareSectionが削除されているため、trySection以降にシェアボタンがなくなる可能性がある。

推奨：shareSectionをdescription直後に置きつつ、detailedContent後（trySection前）にも2つ目のshareSection（または同一コンポーネント）を配置するダブル配置が最も来訪者フレンドリーになる可能性がある。

## 結論

| 観点                                 | character-personality（旧配置）    | kanji-level                       |
| ------------------------------------ | ---------------------------------- | --------------------------------- |
| シェアボタンの視認性                 | 悪い（1822px地点）                 | 良い（538px＝ファーストビュー内） |
| シェアしたいと思うコンテンツ力       | 高い                               | 低い                              |
| 全体的なページ流れの自然さ           | やや不自然（シェアが最後）         | 自然                              |
| 来訪者が「友達に見せたい」と思う瞬間 | descriptionを読んだ後（730px付近） | ほぼない（弱い）                  |
| シェアボタンへの到達まで             | 約1.25画面分スクロール必要         | 不要（ファーストビュー内）        |

character-personalityのデプロイ済みサイトでは、**最も感情的に高揚するdescription読了タイミングと、シェアボタンの位置に大きなギャップがある**のが最大の問題。ソースコードの変更（shareSection移動）はこの問題を解決する方向に進んでいる。
