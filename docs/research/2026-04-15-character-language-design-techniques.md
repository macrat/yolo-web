# キャラクター言語設計技法

本レポートは、yolos.net PMエージェントのキャラクター設計における言語的要素（口癖・思考パターン・文体）の設計技法を体系化することを目的とする。小説執筆論・言語学・VTuber事例・AIプロンプト工学・ブランドボイス設計論を横断的に調査した結果をまとめる。

---

## 1. 役割語と個人言語：理論的基盤

### 1-1. 役割語とは何か

金水敏（大阪大学）は著書『ヴァーチャル日本語 役割語の謎』（岩波書店、初版2003年）において、「役割語（yakuwarigo）」という概念を提唱した。役割語とは、特定の人物像（年齢・性別・職業・階層・時代・容姿・性格など）を即座に想起させる、フィクションの中で慣用的に用いられる言葉遣いのことである。

代表例：

- 「〜じゃ、〜のう」→ 老賢者・博士
- 「〜わよ、〜かしら」→ 上流階級の女性
- 「〜ぜ、〜だぜ」→ 粗野な男性
- 「〜ますわ、〜ですわ」→ お嬢様

これらはメディアを通じて世代間で伝達・強化された「仮想的な日本語」であり、実際の話者が使うとは限らないが、キャラクター理解の即効性を高める機能を持つ。

### 1-2. 役割語を超えた「個人言語」

近年の研究（『役割語とキャラクター』研究社）では、役割語というカテゴリに収まらない「個人言語」が注目されている。マンガ・アニメ・ゲームにおいて、特定のキャラクターだけが使う固有の言葉遣い（「ぺこ」「にぇ」等）は、典型的なステレオタイプではなくそのキャラクター固有の印章として機能する。

**設計上の示唆**：ステレオタイプな役割語（老賢者、お嬢様など）はあくまで「わかりやすさのショートカット」。新キャラクターを設計する際は、役割語を土台にしつつも、そのキャラクター固有の個人言語要素を重ねることで「どこかで見たが、この人だ」という認識が生まれる。

## 2. 口癖・語尾の設計技法

### 2-1. 口癖の機能と種類

口癖（よく使う言い回し・語尾）はキャラクターの認識標識として機能する。その形態は以下に分類できる：

| 種類             | 例                                         | 効果                           |
| ---------------- | ------------------------------------------ | ------------------------------ |
| 語尾付加         | 「〜ですよ」「〜ですね」「〜かな」         | 温度感・距離感の調整           |
| 終助詞の癖       | 「〜よね」「〜でしょう」「〜じゃない?」    | 性格の内省的/外向的印象        |
| 枕詞・接続詞の癖 | 「まあ」「そういえば」「ふと思うんだけど」 | 思考の流れ方・ペースの可視化   |
| 専門語・特定語彙 | 職業由来の言葉を平常会話に持ち込む         | 価値観・バックグラウンドの滲み |
| 擬音・造語       | 独自の感嘆符や言い回し                     | 即認識性・SNS親和性            |

### 2-2. 頻度コントロール：自然さの原則

口癖が「うるさい」と感じられる原因は頻度過多である。創作コミュニティからの知見によれば：

**過剰になるサイン**：

- 同じ語尾が連続する段落に3回以上現れる
- 意味を持たない場所にまで口癖が差し込まれる
- 口癖が感情の起伏に関係なく機械的に出現する

**自然に保つ技法**：

- 「感情が動いた瞬間」に口癖を配置する（クセが感情表現の一部になる）
- 口癖を使わない文を意図的に多く書き、口癖が出る場面を際立たせる
- 対話中の相手（読者）の状態を想像して口癖の出現タイミングを選ぶ

**ガウル・グラ（Gawr Gura / ホロライブEN）の事例**：口癖「a（あ）」と「oh nyo」は配信中に自然発生したものが定着した。人工的に埋め込んだのではなく、キャラクターの「すっとぼけた天然さ」という性格から有機的に生まれた言葉が反復されることで印象に刻まれた。**学び：口癖は性格の自然な帰結として設計せよ。性格と切り離された口癖は記号に過ぎない。**

### 2-3. 話し言葉として自然・書き言葉では冗長な表現の扱い方

語尾・枕詞の多くは、対話における「間（ま）」や「温度感」を担う。書き言葉ではこの「間」を別の手段で表現できる。

| 話し言葉の口癖                           | 書き言葉での代替                                       |
| ---------------------------------------- | ------------------------------------------------------ |
| 「まあ、そうですね」（ペース調整）       | 段落の区切り、一行空き                                 |
| 「〜なんですけど」（柔らかさ）           | 文末の断定を避ける表現「〜のかもしれない」             |
| 「ちょっと聞いてほしいんですが」（導入） | 書き出しの問いかけ形式「〜と思ったことはないだろうか」 |
| 「そういえば」（話題転換）               | 小見出し・行アキ・接続詞「ところで」                   |

**設計方針**：話し言葉の口癖を書き言葉に「翻訳」するのではなく、その口癖が担っていた「機能」（間・温度・転換）を書き言葉の構造的要素で果たすと考える。

## 3. 思考パターンの設計：価値観と世界観の滲ませ方

### 3-1. 比喩の源泉をキャラクターの背景に置く

比喩の選択は、話者の認知構造を反映する。キャラクターの職業・経験・美学によって、同じ対象を描写する言葉が根本的に異なる。

playmaker.jpが示した具体例：

- **少年**：月を「半分かじったクッキー」（無邪気で食欲的）
- **詩人**：月を「破れた銀紙」（傷つきやすく美的）
- **科学者**：月を「重力に引かれながら静止した古い衛星」（論理的・客観的）
- **ロマンチスト**：雨を「閉じた傘の中で眠る小鳥の羽音」（内向きの感覚美）
- **冷めた老人**：雨を「錆びた蛇口の滴り」（疲弊・倦怠感）

**設計の原則**：キャラクターが物事を「どの語彙系から引っ張ってくるか」を決める。道具好きなら道具の比喩、料理好きなら食の比喩、工程を重視するなら工程の比喩。比喩の引き出しがキャラクターの「何を見ているか」「何に価値を置くか」を読者に伝える。

### 3-2. 「引っかかり」の設計：何を良しとし何に違和感を覚えるか

電通クリエイターの知見（ウェブ電通報）によれば、記憶に残るキャラクターは「思わずツッコミを入れたくなるポイント」を意図的に持っている。これは見た目の奇抜さではなく、**価値観のクセ（こだわり）から生まれる行動・言動上の違和感**である。

キャラクターの価値観設計は以下を明確にすることで機能する：

- **何を「良いもの」と思うか**：仕上がりの美しさ、プロセスの誠実さ、意外性、機能美など
- **何が「引っかかる」か**：曖昧さ、雑さ、見た目だけの洗練、等
- **どこに「発見の喜び」を感じるか**：細部、意外な共通点、歴史の層、など

これらの価値観が言動の随所に滲むとき、キャラクターは「こういう人だ」という輪郭を持つ。

**実用技法**：キャラクターが意見を言う場面で、単に評価するだけでなく「なぜそれが気になるのか」の理由を語る口調を与える。「それ、面白いですね」ではなく「それ、ちょっと待ってください——なぜこの形なんでしょう。機能的な理由があるのか、それとも昔からの習慣が残っているのか」という反応を設計する。

### 3-3. 観察の視点：何に気づき、何を見落とすか

キャラクターの「観察眼の癖」は個性を強力に伝える。小説技法において、地の文で語り手が何に言及し、何を無視するかは、キャラクターの価値観を直接反映する。

設計時に問うべき問い：

- このキャラクターは部屋に入ったとき、最初に何を見るか（光？匂い？道具の配置？人の顔？）
- 失敗に直面したとき、どこに原因を求めるか（自分？素材？手順？道具？）
- 他者を評価するとき、どの尺度を使うか（誠実さ？技術？センス？継続性？）

## 4. 話し言葉と書き言葉の同一キャラクター運用

### 4-1. ボイスとトーンの分離

Mailchimpのコンテンツスタイルガイドが提示する「ボイス（voice）とトーン（tone）の分離」は、キャラクター設計に直接応用できる最も実践的なフレームワークである。

- **ボイス（Voice）**：常に一定。キャラクターの人格、価値観、語彙の傾向、何を面白いと思うか。これは文脈に関わらず変わらない。
- **トーン（Tone）**：文脈依存。対話なら軽快、解説なら穏やか、重い話題なら抑制的。同じ人格が異なる感情レジスターで表現される。

**実例（Mailchimp）**：「ユーモアがある、平易、経験豊富な友人のような」ボイスは一定。購入完了メールでは「軽快・楽しい」トーン、クレーム対応では「落ち着いた・共感的」トーン。いずれも同一人格から出た言葉に聞こえる。

### 4-2. 話し言葉のクセを書き言葉に翻訳する技法

同一キャラクターが「対話モード」と「執筆モード」を切り替える際、以下の要素を「共通化するもの」と「モードで分岐させるもの」に整理すると設計しやすい。

**共通化する要素**（ボイス＝人格の本質）：

- 語彙の選好（専門用語の使い方、比喩の引き出し）
- 何に言及し何に言及しないか（観察の癖）
- 結論の出し方（断定的か、留保を置くか）
- 感情表現の仕方（直接的か、比喩に包むか）
- ユーモアの種類（観察系か、自虐系か、言葉遊びか）

**モードで分岐させる要素**（トーン＝文脈への適応）：

| 要素       | 話し言葉（対話）                     | 書き言葉（ブログ）                         |
| ---------- | ------------------------------------ | ------------------------------------------ |
| 語尾       | 「〜ですよね」「〜かな」「〜です！」 | 「〜である」「〜かもしれない」「〜と思う」 |
| 文の長さ   | 短文・口語的切れ                     | 中〜長文・論理的な流れ                     |
| 枕詞・接続 | 「そういえば」「まあ」「ちょっと」   | 「〜という点に着目すると」「ところで」     |
| 感嘆・共感 | 「あ、それ面白い！」                 | 「これは興味深い」「注目したいのは」       |
| 問いかけ   | 直接的な質問「どう思います?」        | 修辞疑問「〜なのではないだろうか」         |

**一人称の扱い**：話し言葉・書き言葉で一人称を変える手法は有効。例として「私」を対話でも文章でも使いつつ、書き言葉では「筆者」を使う作家がいるように、一人称の使い分けがトーンのシフトを明示する。

### 4-3. 地の文口調化という技法

一部の小説家・エッセイストは、地の文（説明・描写部分）も会話口調に近づける「地の文口調化」を採用する。これは書き言葉と話し言葉の境界をわざとぼかす手法であり、キャラクターの親密さや独特の語り口を強調する効果がある。

ブログ記事においてこの技法を使うと、文章が「語りかける」感触を持つ。ただし過剰に口語化すると読みにくくなるため、「話し言葉の流れ感」と「書き言葉の情報密度」のバランスが課題となる。

## 5. 既存キャラクターからの学び

### 5-1. VTuberの口癖設計：有機的発生と意図的定着

VTuberのキャラクター言語設計から得られる主な教訓：

**兎田ぺこら（ホロライブ）の「ぺこ」**：語尾に固有名詞的要素を付加することで、セリフを引用するだけでキャラクターが想起される即認識性を達成。これは個人言語の最も単純かつ強力な形式。

**がうる・ぐら（ホロライブEN）の「a」**：単音の感嘆詞が反復されることで、「予期せず言葉に詰まる」「すっとぼける」という性格描写の機能を果たす。複雑な語彙設計より、シンプルな反応パターンの方が印象に残りやすい。

**にじさんじの多様性**：にじさんじのライバーは各自が強烈な個性を持ち、口調の多様性が共存する。一方ホロライブはアイドル性という共通ボイスの中で各自のトーンが変化する構造。

**VTuber設計の核心原則**（Vtuber Senseより）：

1. 「選んだ性格と発言スタイルに一貫して従うこと」が信頼を生む
2. キャッチフレーズは「強制的に埋め込む」のではなく、「ペルソナから有機的に発生する」ものにする
3. 感情表現（喜怒哀楽の表し方）がスタイルを維持しながら自然な反応を可能にする

### 5-2. AIキャラクター設計のプロンプト工学知見

LLMのキャラクター設計から得られる知見は「キャラクターの文書化」の方法として応用できる。

**効果的なキャラクター定義の要素**（AXメディアの調査より）：

- 役割・職業（「あなたは〜です」という存在の定義）
- 性格・価値観（何を大切にするか）
- 一人称・口調・口癖（具体的な言語的特徴）
- 背景・経歴（比喩や観察眼の源泉）
- 知識レベル・専門分野
- ユーザーとの関係性（どんな距離感で話すか）

**注意点**（note「AIに口調設定する限界」より）：口癖の例文を大量に列挙すると逆効果になる。AIは大量のセリフを「パターンの抽象化」として処理するため、核となる要素がぼやける。「簡明インスト」＝総体的記述と具体例の適切な組み合わせが有効。

**生成AI文章の「違和感」から学ぶ**（malna株式会社の調査より）：AIが書く文章が違和感を持つ原因の一つは「最も統計的に安全な言葉」を選ぶことによる画一性。これはキャラクター設計でも同様で、「よく見る言い回し」「汎用的な感嘆詞」を避け、そのキャラクターが使いそうな固有の言葉を意図的に選ぶことで個性が生まれる。

### 5-3. 文体と人格の一致：作家の事例

村上春樹の文体分析（プリンストン大学 牧野成一による研究）から：

- 村上は「リズム」を文章の必須要素と述べており、文体はアイデンティティそのものである
- 主人公「僕」があまり断言せず「〜だ。あるいは〜だったかもしれない」と必ず別の可能性を示すスタイルは、語り口という次元を超えてキャラクターの認識論（世界の不確かさへの態度）を表現している

**学び**：優れた文体は「誰が書いたかわかる」固有性を持つ。その固有性は単語選択の癖ではなく、「世界をどう見るか」という認識の構造から生まれる。

## 6. 設計の実践的まとめ：yolos.net PMキャラクター設計への応用指針

以下は、本調査を踏まえてキャラクター候補を列挙するStep 3 plannerが参照すべき技法の要点である。

### 口癖・語尾の設計指針

1. 語尾は「感情が動いた瞬間に出る」ように設計する。常に出続けるものにしない。
2. 枕詞（「まあ」「そういえば」「ふと）は思考の流れを見せる道具として使う。
3. 専門語・生活語の混用：「日常の道具」というコンセプトに沿った語彙系（職人、手仕事、使い込み、馴染む等）を口癖の基盤に置く。
4. 書き言葉ではトーンのシフト（文末の柔化、段落の間の活用）で口癖の機能を代替する。

### 思考パターンの設計指針

1. 「何を最初に見るか」を決める（道具の細部？ 素材の手触り？ 用途の合理性？）。
2. 「何が引っかかるか」の価値観を明確にする（雑さ？ 過剰な装飾？ 本来の使い方からの逸脱？）。
3. 比喩の引き出しを「道具・工程・材料・時間の積み重ね」系に統一する。

### 話し言葉と書き言葉の分岐指針

- **共通（ボイス）**：道具や日常への目線の近さ、静かな熱量、留保を置く語り口、「使い手の視点」への共感
- **話し言葉（トーン：軽快）**：短文、問いかけ、感嘆の直接表現、枕詞
- **書き言葉（トーン：落ち着き）**：中〜長文、修辞疑問、比喩での感情包含、論理的なつながり
