# play系コンテンツ カスタマージャーニーマップ

## 概要

このドキュメントはplay系コンテンツへの全到達経路を5シナリオに整理し、各ステップのユーザー心理・アクション優先順位・離脱リスクと対策を明記したものである。来訪者への価値提供を最大化し、ページビュー増加につなげるための改善方向の基礎資料として使用する。

### ターゲットユーザーの特徴

- 手軽で面白い占い・診断を楽しみたい
- 特別な前提知識は不要、SNSの基本的な使い方を知っている
- yolos.netが何なのか知らない状態で来訪する
- **求めるもの**: 登録不要ですぐ遊べること、ユーモアと意外性、シェアしやすさ
- **嫌うもの**: ガチスピリチュアル、こじつけ、長すぎるコンテンツ
- **特性**: 「友達に見せたくなる」「SNSでシェアしやすい」コンテンツを好む — SNSバイラルが最も自然な成長ドライバーである

### GAデータが示す現状と最重要課題

- サイト全体: 直近7日間 PV 96（セッション52）。そのうちplay系コンテンツへの流入はmusic-personalityの3PVのみ（全体の約3%）
- **93PVが非play系（ブログ・辞典等）に流入している** — これらのユーザーからplay系への導線強化が即効性の最も高い施策である
- 流入の大半はOrganic Search（60%）とDirect（35%）だが、play系には到達していない
- **最重要課題は「play系への流入自体がない」こと**。回遊導線をいくら改善しても、play系来訪者が週3PV程度の現状では効果は極めて限定的である（回遊率2倍でも+3PV程度）
- **最優先施策は「流入増加」**: (1) 既存93PVからの導線強化、(2) SNSバイラル体験の改善（シェア体験の質向上）。回遊導線改善はplay系への流入が一定規模に達した後に効果が顕在化する施策として位置づける

## シナリオ1: Organic Search経由の新規来訪者

**現時点での発生頻度: 月0〜数件程度（GAデータ上、play系へのOrganic Search流入はほぼ計測されていない）**

### 到達経路

```
Google検索「性格診断 面白い」「占い 無料」「〇〇診断」
  ↓
検索結果クリック
  ↓
/play/[slug]（個別クイズページ） ← 最多ルート
または /play（一覧ページ）
または /（トップページ）→ /play
```

### ユーザーの心理状態

来訪者は検索意図として「面白い診断があれば試してみたい」程度の軽い気持ちを持っている。しかし**積極的にシェアしたいわけではなく**、暇つぶしの選択肢の一つを探しているに過ぎない。

ページを開いた瞬間から「これは自分が期待したものか?」の無意識的な評価が始まる。以下の懸念が生じるとすぐに離脱する:

- 「長そう」「面倒くさそう」
- 「なんか怪しいサイト」「スパムっぽい」
- 「知らないサイトで登録が必要かも」
- 「期待した診断と違う内容」

### ユーザーが実際に見る画面要素

**ファーストビュー（/play/[slug]、introフェーズ）**

`QuizContainer.tsx`のintroフェーズ（phase === "intro"）で以下が表示される:

- アイコン（`quiz.meta.icon`）
- タイトル（`quiz.meta.title`、h1）
- 説明文（`quiz.meta.description`）
- **問数とタイプ**（`全{quiz.meta.questionCount}問 / 診断 or 知識クイズ`）← 既実装
- スタートボタン（`accentColor`でスタイリング）
- 関連リンク（`quiz.meta.relatedLinks`が存在する場合）

**ファーストビュー（/playの場合）**

- ヒーローバナー（コンテンツ総数、今日のピックアップ）
- カテゴリナビゲーション（sticky、常時表示）
- 「イチオシ」セクション（おすすめ理由バッジ付きカード3件）
- 各カードに**問数（`{questionCount}問`）**が表示される← 既実装（quizQuestionCountBySlugが定義されているコンテンツのみ）

### アクション優先順位

1. **クイズ・診断をプレイ開始する**（最重要。ここで離脱すると他は何も達成されない）
2. **他のコンテンツも試す（回遊）**
3. シェアする

### 離脱リスクと対策

| ステップ             | 離脱要因                         | 対策の方向性                                                |
| -------------------- | -------------------------------- | ----------------------------------------------------------- |
| 検索結果から来訪直後 | タイトル・説明文が期待と合わない | メタタイトル・ディスクリプションを検索意図に合わせる        |
| ファーストビュー     | 「怪しい」と感じる               | 「登録不要」「無料」「AIが作った診断サイト」の明示          |
| 設問途中             | 設問が多すぎる・退屈             | 進捗バー表示（既実装）、introフェーズでの問数表示（既実装） |
| /play一覧ページ      | 何を試せばいいかわからない       | 「イチオシ」セクション（既実装）、問数表示（既実装）        |

### 現状の導線の問題点

- **introフェーズに問数は既に表示されている**（`全{questionCount}問 / 診断 or 知識クイズ`）
- **/play一覧カードにも問数は既に表示されている**（quizQuestionCountBySlugで定義されているコンテンツのみ。ゲーム等、問数が定義されていないコンテンツは非表示）
- トップページから/playへの導線が弱く、サイト名検索で来訪したユーザーがplay系コンテンツに到達しにくい
- 検索からの流入は全体の60%を占めるが、play系コンテンツへの流入は極めて少ない。ただし`src/play/seo.ts`の確認により、SEOメタデータ（OGP・JSON-LD・canonical等）は既に実装済みである。流入不足の主因は「コンテンツの発見可能性」または「検索インデックス未成熟」の可能性が高く、メタデータ最適化が即効性ある解決策になるとは限らない

## シナリオ2: サイト内回遊でplay系に到達する来訪者

**現時点での発生頻度: 月数件〜十数件程度（推定）。93PVが非play系に流入しており、現状play系への内部導線がほぼないため実際の到達数は極めて少ないと推定される**

**このシナリオは現時点で最も投資対効果が高い改善領域である。**

### 到達経路

```
ブログ記事を読む
または 辞典ページを見る
または トップページを訪れる
  ↓
記事内のリンク・バナー・サイドバー等からplay系へ誘導される
  ↓
/play/[slug]（個別コンテンツページ）
または /play（一覧ページ）
```

### ユーザーの心理状態

すでにyolos.netにいる（つまり離脱リスクが低い）ユーザーが対象である。非play系コンテンツに来訪したということは、すでにサイトに対して一定の関心・信頼がある状態であり、play系への誘導に対する心理的抵抗は低い。

「暇つぶしに面白いことをしたい」というニーズは常に潜在しており、記事を読みながらでも「ちょっと診断してみようかな」という気持ちに切り替わりやすい。

- 既にサイトの雰囲気を把握しているため「怪しい」という懸念が小さい
- 記事のテーマと関連した診断であれば関心度が高い（文脈的誘導）
- ただし「今は記事を読みに来た」というモードのため、押しつけがましい誘導は逆効果

### ユーザーが実際に見る画面要素

現状の問題: **ブログ記事・辞典ページからplay系への内部リンクがほぼ存在しない**。トップページの「遊ぶ」ナビゲーションは存在するが、コンテンツページ内でplay系を積極的に紹介する導線がない。

改善後に期待できる接点:

- ブログ記事末尾の「関連する診断を試してみる」ブロック
- 辞典ページのサイドバーやインライン推薦
- トップページにおける「今日のおすすめ診断」の目立つ配置
- ヘッダーナビゲーションの「遊ぶ」リンクの視認性向上

### アクション優先順位

1. **play系コンテンツに誘導されてプレイ開始する**（このシナリオの唯一の目的）
2. **プレイ後にシェアする**（既存ユーザーは信頼感があるためシェア率が高い可能性）
3. **さらに別のplay系コンテンツに回遊する**

### 離脱リスクと対策

| ステップ                     | 離脱要因                           | 対策の方向性                                                       |
| ---------------------------- | ---------------------------------- | ------------------------------------------------------------------ |
| コンテンツページ閲覧中       | play系への誘導リンクが存在しない   | ブログ記事・辞典ページにplay系推薦ブロックを追加                   |
| play系への誘導を発見した瞬間 | 「記事と関係ない」と感じて無視する | 記事テーマと関連した文脈的推薦（例: 音楽記事なら音楽系診断を推薦） |
| /play/[slug]到達後           | 期待と違う内容                     | introフェーズでコンセプトを明確に伝える（既実装要素の活用）        |

### 現状の導線の問題点

- **最大の問題**: 93PVが非play系に流入しているにもかかわらず、それらのページからplay系への内部導線がほぼない。これは「最もコスト低く流入を増やせる施策が未着手」であることを意味する
- トップページのナビゲーションに「遊ぶ」リンクはあるが、ブログ・辞典の個別コンテンツページ内での推薦が存在しない
- 文脈的推薦（記事テーマと関連するplay系コンテンツの紹介）がないため、サイト内回遊のクロスセル機会を逃している

## シナリオ3: プレイ完了者（クイズ・診断・ゲーム）

**現時点での発生頻度: 月数件程度（play系3PVの一部がプレイ完了に至ると推定。現状は実装品質よりも流入増加が先決）**

このシナリオはかつて「シナリオ3（クイズ・診断）」と「シナリオ4（ゲーム）」に分割されていたが、完了後の基本的な行動パターン（シェア・回遊）は共通しており、UI上の差異（ResultCard vs. ResultModal）は実装の詳細に過ぎないため統合した。

### 到達経路

**クイズ・診断の場合**

```
/play/[slug]（クイズ・診断ページ）
  ↓
全設問に回答
  ↓
インライン結果表示（ResultCard）← 現在地
```

**ゲームの場合**

```
/play/[slug]（ゲームページ）
  ↓
ゲームをプレイ
  ↓
ResultModal（モーダル）が表示 ← 現在地
```

### ユーザーの心理状態

**クイズ・診断プレイ完了後**

| タイプ   | 心理状態                                           | 次アクションへの動機                                               |
| -------- | -------------------------------------------------- | ------------------------------------------------------------------ |
| 満足型   | 「なるほど、わかる気がする」「まあそうかも」達成感 | 弱い。「次もあれば試すかも」程度                                   |
| 興奮型   | 「当たりすぎ！」「ウケる」共感・驚き               | シェアしたい衝動。ただし次のアクションへの導線がないと衝動が冷える |
| 不満型   | 「これは違う」「的外れ」                           | もう一度やるか、即離脱の二択                                       |
| 無関心型 | 「まあそうか」興味を持てなかった                   | 離脱が最も速い                                                     |

**ゲームプレイ完了後**

| タイプ       | 心理状態                                     | 次アクションへの動機                                           |
| ------------ | -------------------------------------------- | -------------------------------------------------------------- |
| 達成感型     | 「クリアできた」「スコアが良かった」満足感   | 達成を誰かに伝えたい（シェア動機）、別のゲームを試す余裕がある |
| 悔しさ型     | 「もう少しだった」「もう一回やりたい」       | リトライが最優先。他コンテンツへの回遊動機は低い               |
| ルーティン型 | 「今日も終わった」（毎日の習慣としてプレイ） | 次のゲームやカウントダウン確認が自然な動作。長期的な継続が前提 |

**重要（共通）**: 興奮型・達成感型でさえ「次の行動先」が明示されていなければ離脱する。次のアクションへの動線は結果表示の直後に提示されるべきである。また、ターゲットユーザーの特性（「友達に見せたくなる」「SNSでシェアしやすい」）から、**シェア体験の質がこのシナリオのKPIを最も大きく左右する**。

### ユーザーが実際に見る画面要素

**クイズ・診断: ResultCard内（`ResultCard.tsx`が描画する要素）**

- アイコン（`result.icon`が存在する場合のみ）
- 結果タイトル（`result.title`、h2）
- **スコア表示**（`quizType === "knowledge"` かつ `score`と`totalQuestions`が両方存在する場合のみ: `{totalQuestions}問中{score}問正解`）
- 結果の説明文（`result.description`）
- **推薦リンク**（`result.recommendation`と`result.recommendationLink`が**両方**存在する場合のみ: リンクとして表示）
- ShareButtons
- 「もう一度挑戦する」ボタン

**クイズ・診断: ResultCardの直後（ResultExtraLoaderが表示する要素）**

- InviteFriendButton（友達に診断を送る）※quiz種別に応じて表示
- CompatibilitySection（相性診断）※quiz種別に応じて表示
- その他クイズ固有のコンテンツ

**クイズ・診断: さらに下（スクロール必要）**

- FAQ（5問程度）
- さらに下にRelatedQuizzes/RecommendedContent

**ゲーム: ResultModal内（モーダルが開いた状態）**

- ゲーム結果（スコア・クリア状況等）
- GameShareButtons（シェアボタン・画像保存ボタン）
- CountdownTimer（次のパズルまでのカウントダウン）
- NextGameBanner（同カテゴリのゲーム導線）
- 「統計を見る」フッターボタン（モーダルを閉じて統計画面を表示）

**ゲーム: モーダルを閉じた後**

- ゲーム画面（リトライ可能な状態）
- 下スクロールでRelatedGames/RecommendedContent

### アクション優先順位

1. **結果を楽しむ（読み込む）**（感情的報酬の確保。これがなければ次のアクションは生まれない）
2. **シェアする / 友達に診断を送る**（興奮型・達成感型の衝動を逃さないための即時CTAが最重要 — ターゲットユーザーの「シェアしやすさを求める」特性から最も優先度が高い）
3. **別のクイズ・診断・ゲームを試す（回遊）**（好奇心の連鎖を設計）
4. もう一度挑戦する（不満型・悔しさ型への対応）

### 離脱リスクと対策

**クイズ・診断**

| ステップ                         | 離脱要因                                     | 対策の方向性                                                                   |
| -------------------------------- | -------------------------------------------- | ------------------------------------------------------------------------------ |
| 結果表示直後                     | 結果が面白くない・当たっていない感           | コンテンツ品質の向上（根本対策）、「もう一度」CTAを目立たせる                  |
| シェアしたい衝動が生じた瞬間     | ShareButtonsが見えない・操作しにくい         | ShareButtonsをファーストビュー内に確実に配置                                   |
| 次コンテンツへの関心が生じた瞬間 | 関連コンテンツ導線がFAQ5問の下に埋もれている | **RelatedQuizzesをResultCardの直下（FAQより前）に移動**                        |
| 回遊ポイント                     | 「どれが面白そうか」判断できない             | **文脈付き推薦**: 「あなたのタイプと相性がいい診断」「同じくXXな人はこちらも」 |

**ゲーム**

| ステップ             | 離脱要因                                                       | 対策の方向性                                                   |
| -------------------- | -------------------------------------------------------------- | -------------------------------------------------------------- |
| ResultModal表示直後  | GameShareButtonsがモーダル内の下部にあり気づきにくい場合がある | GameShareButtonsの配置・視認性を確認し、結果の直下に確実に表示 |
| NextGameBanner確認後 | 同カテゴリのゲームに興味がない場合、他への導線がない           | モーダル内に診断・占いカテゴリへの簡易導線を追加               |
| モーダルを閉じた後   | RelatedGames/RecommendedContentが遠い                          | モーダルを閉じると関連コンテンツが目に入る位置への配置         |
| リトライ後           | 同じ離脱ループが繰り返される                                   | リトライ回数に応じて「他のゲームも試す?」の訴求を強化          |

### 現状の導線の問題点

**クイズ・診断**

- **最大の問題**: RelatedQuizzes/RecommendedContentへ到達するには、FAQ5問 + ResultExtraLoaderのコンテンツを超えるスクロールが必要。プレイ完了直後の「次も試したい」という気持ちが生じても、導線が遠すぎて到達前に離脱する
- InviteFriendButtonはResultExtraLoaderが対応するquiz種別（music-personality等）のみに表示されており、全コンテンツには存在しない。シェア動線の非一貫性
- 「もう一度挑戦する」ボタンが結果の下に配置されているが、不満型ユーザーが気づく前に離脱している可能性がある

**ゲーム**

- GameShareButtonsはResultModal内に存在する。ただしモーダル内のレイアウト（結果 → シェアボタン → タイマー → NextGameBanner の順序）によっては、シェアボタンが結果に比べて目立たない配置になっている可能性がある
- **ResultModal内に他カテゴリ（診断・占い）への導線が一切ない**。NextGameBannerは同カテゴリ限定であり、ゲームに飽きたユーザーが診断に流れる経路が存在しない
- モーダルを閉じた後のRelatedGames/RecommendedContentへのスクロール距離が膨大

### ベストプラクティスの反映

- **シェア体験の最優先**: ShareButtonsをファーストビュー内に確実に配置し、シェアしやすいUI（テキストのコピーしやすさ・OGP画像の品質）を重視する
- **摩擦の最小化**: 結果確認後、スクロールなしで次のアクション（シェア or 次コンテンツ）に到達できる設計
- **好奇心の刺激**: 結果確認後すぐに関連コンテンツが目に入ることで、「他も気になる」という探索意欲を持続させる
- **文脈付き推薦**: 「あなたの結果と相性がいい診断」というパーソナライズされた文言で回遊率を高める
- **文脈付きクロスセル（ゲーム）**: 「ゲームが好きなあなたは、このタイプ診断も面白いかも」という文脈でクロスセル

## シナリオ4: /play一覧ページからの探索者

**現時点での発生頻度: 月数件程度（Direct経由の一部がplay一覧を訪問していると推定されるが、実数は不明）**

### 到達経路

```
ヘッダーの「遊ぶ」クリック
または トップページのCTA「遊ぶ」クリック
または ブックマーク・Directからの直接来訪
  ↓
/play（一覧ページ） ← 現在地
```

### ユーザーの心理状態

「何があるか見てみよう」「今日は何を遊ぼうか」という探索モード。リピーターの場合は「新しいコンテンツはあるか」という確認行動。

ただし「探索」は積極的なものではなく、以下の条件が揃わないと「何もせず離脱」になる:

- パッと見て気になるコンテンツが1秒以内に目に入ること
- 「面白そう」と「どれくらいかかるか」が瞬時に判断できること
- 消極的ユーザーは「長そう」「難しそう」と感じると試さない

### ユーザーが実際に見る画面要素

**ファーストビュー（スクロールなし）**

- ヒーローバナー（コンテンツ総数、今日のピックアップ）
- **カテゴリナビゲーション（sticky化済み: `position: sticky; top: 0; z-index: 10`）** ← 既実装（`page.module.css`の`.categoryNav`クラス）
- 「イチオシ」セクションの上部

**「イチオシ」セクション（既実装）**

- おすすめ理由バッジ（`recommendReason`）付きカード3件
- 各カードに**問数（`{questionCount}問`）**が表示される（quizQuestionCountBySlugで定義されているコンテンツのみ）

**カテゴリ別セクション（スクロール後）**

- 各カードに**問数（`{questionCount}問`）**が表示される（quizQuestionCountBySlugで定義されているコンテンツのみ）
- 毎日更新コンテンツに「毎日更新」バッジ（既実装）

### アクション優先順位

1. **気になるコンテンツを1つ選んでプレイ開始**（一覧ページの唯一の目的）
2. **カテゴリを切り替えて他のコンテンツも確認**（探索を促進）
3. **（リピーターの場合）新着・未プレイのコンテンツを発見**（継続来訪の動機づけ）

### 離脱リスクと対策

| ステップ           | 離脱要因                           | 対策の方向性                                                   |
| ------------------ | ---------------------------------- | -------------------------------------------------------------- |
| 一覧ページ到達直後 | 何を試せばいいかわからない         | 「イチオシ」セクション（既実装）、今日のピックアップ（既実装） |
| カード閲覧時       | カード同士の差別化が見えない       | 各カードにユニークな訴求文を追加                               |
| カテゴリ切り替え時 | 見たいカテゴリへのスクロールが面倒 | カテゴリナビゲーションのsticky化（**既実装**）                 |
| リピーター来訪時   | 新しいコンテンツがどれかわからない | 「新着」バッジや「未プレイ」フラグの表示                       |

### 現状の導線の問題点

- カードの説明文が内容を説明するに留まり、「なぜ今これを試すべきか」という動機づけが弱い（「イチオシ」セクションの`recommendReason`バッジはあるが、通常カードには未適用）
- リピーターへの「新着・未プレイ発見」機能が存在しない。継続来訪の動機が薄れる
- **CategoryNavのsticky化は既実装。問数表示も既実装**。これらは改善済みである

### ベストプラクティスの反映

- **コスト感の可視化**: 問数表示（既実装）で消極的ユーザーの参入障壁を下げる
- **好奇心ギャップ**: 答えを示さない文言（「あなたのタイプは意外な結果かも」）でクリック動機を作る

## シナリオ5: SNSシェアリンク経由の来訪者【将来シナリオ】

**現時点での発生頻度: 月0件程度（シェア実績がほぼないため、このシナリオはほぼ発生していない）**

**位置づけ**: このシナリオは現時点での投資対効果が低い。シェア実績が積み上がった後（シナリオ3のシェア体験改善が先行する必要がある）に効果が出る施策である。設計の参照資料として保持するが、現時点での改善優先度は低い。

### 到達経路

```
友人・知人のSNS投稿（X/Twitter、LINE、Instagram等）
  ↓
シェアリンクをクリック
  ↓
/play/[slug]/result/[resultId]（結果ページ）← 直接到達
```

### ユーザーの心理状態

「ちょっと気になったから開いてみた」「うっかりタップしてしまった」という程度の関与度。「自分もやってみたい！」という積極性はまだない。シェアした友人への関心（「友達がどんな結果だったか見たい」）の方が、コンテンツへの関心より強い場合が多い。

### 判断フロー

```
Step 1: 「これは何?」を数秒で判断
  ↓ わからない・理解に時間がかかる → 即離脱

Step 2: 「友達の結果はどれ?」を確認
  ↓ 結果が面白くない・意味不明 → 離脱

Step 3: 「自分もやったら面白いかな?」を検討
  ↓ コストが高そう（長い・難しい・登録必要） → 離脱

Step 4: プレイ開始
```

### ユーザーが実際に見る画面要素

**ファーストビュー（スクロールなし）**

- 友人の診断結果のタイトル・タイプ名（大きく表示されているか?）
- 結果の簡潔な説明文
- 「あなたも挑戦してみる?」CTAボタン（診断名のアクセントカラーで表示）

**スクロール後に見える要素**

- シェアボタン
- CompatibilityDisplay（相性結果。`with`パラメーター付きURLの場合のみ表示）
- RelatedQuizzes
- RecommendedContent

### アクション優先順位

1. **「あなたも挑戦してみる?」CTAをクリック**（シェアリンク経由のユーザーを新規プレイヤーに変換する最重要アクション）
2. クイズをプレイして結果を見る
3. 自分の結果をシェアする

### 離脱リスクと対策

| ステップ               | 離脱要因                                                     | 対策の方向性                                                       |
| ---------------------- | ------------------------------------------------------------ | ------------------------------------------------------------------ |
| 結果ページ到達直後     | 「これは何?」が数秒でわからない                              | 診断名・コンセプトをファーストビューに大きく表示                   |
| 結果確認後             | 「自分もやるのは面倒」                                       | CTAに問数・登録不要等のコスト低減訴求を含める                      |
| プレイ開始への誘導     | CTAが結果の下に埋もれている                                  | CTAをファーストビュー内（スクロール不要の位置）に配置              |
| シェアリンク閲覧者特有 | 「これは友人個人のリンク」と誤解して自分向けではないと感じる | 「この診断、あなたはどのタイプ?」という2人称の問いかけで自分ごと化 |

### 現状の導線の問題点

- 静的結果ページ（`/play/[slug]/result/[resultId]`）に「あなたも挑戦してみる?」CTAボタンは既に存在する。しかしCTAの**位置・訴求の質**に改善余地がある:
  - CTAがタイトル・説明文の直後に配置されているが、コスト感の訴求（問数・登録不要等）が含まれていない
  - CTAテキスト「あなたも挑戦してみる?」は訴求としてニュートラルであり、「シェアリンクを見た人」向けに未完了感を演出する文言ではない
- シェアボタンがCTAより下に配置されており、ファーストビューには見えない場合がある

## 横断的な問題点の整理

### 0. 最優先課題: play系への流入自体がない

GAデータによれば、play系コンテンツへの流入はmusic-personalityの3PVのみ（全体96PVの約3%）。回遊導線をいくら改善しても、来訪者がいなければ効果は出ない。**回遊導線改善は流入増加施策と並行して取り組む必要があるが、流入が一定規模に達するまでは回遊導線改善の優先度を下げるべきである**。

### 0-A. 既存93PVからの内部導線が未整備（即効性最高）

93PVが非play系（ブログ・辞典等）に流入している。これらのページからplay系への内部導線を強化することが、最も低コストで流入増加効果を得られる施策である。

**改善の方向性**: ブログ記事・辞典ページへのplay系推薦ブロック追加。記事テーマと関連したコンテンツの文脈的推薦を優先する。

### 0-B. SNSバイラルがplay系の本命成長ドライバー（ただし現状では根拠の検証が困難）

ターゲットユーザーの特性（「友達に見せたくなる」「SNSでシェアしやすい」）から、SNSバイラルがplay系コンテンツの最も自然な成長経路である。シェア体験の改善（ShareButtonsのUI、シェアテキストの魅力）は戦略的に重要な施策である。

**重要な留意点**: 現在のplay系への流入はweekly 3PV程度であり、シェアが起きていない原因が「シェア体験の質が低いから」なのか「そもそもプレイ完了者の母数が存在しないから」なのかを現状のデータから区別することはできない。「シェア体験の質が低くSNSバイラルが起きていない」という主張は現時点では根拠不十分である。シェア体験の改善は将来的に流入が増えた際に効果を発揮する施策として位置づけるべきであり、現時点での最優先事項は0-Aの流入増加施策である。

**改善の方向性**: ShareButtonsを結果のファーストビューに確実に配置し、シェアしやすいUI・シェアテキストを整備しておく。OGPメタデータについては`src/play/seo.ts`により既に実装済みであるため、新規実装は不要。

### 1. 回遊導線の断絶（流入増加後に効果が顕在化する施策）

現状のユーザーフローは「プレイ → 結果 → 離脱」となっており、「プレイ → 結果 → 次のコンテンツ → プレイ」という連鎖が設計されていない。

**根本原因**: RelatedQuizzes/RecommendedContentがFAQや長いコンテンツの下に配置されており、スクロールしなければ到達できない。

**改善の方向性**: ResultCard直下（FAQより前）への関連コンテンツ配置。**この施策はplay系への流入が週3PV程度の現状では回遊率2倍でも+3PV程度の効果しかなく、流入増加施策（0-A）が先行すべきである。**

### 2. シェア動線の課題

- シェアボタン（ShareButtons）はプレイ完了者向け（本人がシェアする用）のみ設計されており、シェアリンク経由の新規来訪者向けのCTA訴求が不十分
- 既存の「あなたも挑戦してみる?」CTAはコスト感の訴求（問数・登録不要等）を含んでおらず、訴求の質に改善余地がある
- InviteFriendButtonがResultExtraLoader対応のquiz種別限定で非一貫的
- ゲームのResultModal内には他カテゴリへの導線がなく、ゲームユーザーが診断・占いに流れる経路が存在しない

**改善の方向性**: ShareButtonsのファーストビュー配置確認・改善。「あなたも挑戦してみる?」CTAにコスト感訴求を追加し、訴求テキストをシェアリンク経由ユーザーに響く文言に最適化。

### 3. SEO起点の流入最適化

GAデータによれば、Organic Searchが流入の60%を占めるが、play系コンテンツへの流入はほぼゼロ（music-personalityの3PVのみ）。ただし**`src/play/seo.ts`の確認により、OGP・JSON-LD・canonical・Twitter Cardを含むSEOメタデータは既に実装済み**である。流入不足の主因はメタデータの不在ではなく、コンテンツの検索インデックス未成熟や競合環境の可能性が高い。

**改善の方向性**: メタデータの大幅な追加実装は優先度低。既存実装の品質確認（各コンテンツのdescription・keywordsが検索意図に合っているか）を行い、コンテンツ品質の向上に注力する。
