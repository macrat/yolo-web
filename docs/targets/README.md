>
> - メインターゲット2件がいずれも**ツール利用者**で、実測上の主役である**診断（性格診断・キャラ診断）の来訪者を表すターゲットが存在しない**。直近の organic 流入増は「似てるキャラ診断」系で、診断が集客・滞在の両軸で頑健1位（cycle-257）。
> - メインターゲット M1b「気に入った道具を繰り返し使っている人」は**反復利用を前提**にしているが、cycle-257 の実測では再訪はサイト全体で不成立（価値は単発訪問の質で測るべき）。
> - 「サイトの中心は『道具』」「息抜きは全体の1割を超えない」という記述（下記 S3 の説明等）は旧枠組み。
>
> **このずれを是正するまで、本ターゲット定義を新規コンテンツ判断の根拠にする際は上記を念頭に置くこと**（ツール利用者だけを主対象に判断しない／診断来訪者を主たる対象として補って考える）。正式なターゲット再定義（診断来訪者のメインターゲット追加・反復利用前提の見直し・サブ/比率の再編）は **B-543** で実施する。詳細は `docs/cycles/cycle-259.md` / `docs/cycles/cycle-257.md`。

このディレクトリでは、<http://yolos.net>の顧客像を定義しています。
yolos.net では5つのターゲットを想定しています（※上記注記のとおり診断中心コンセプトへの再定義待ち）。

## メインターゲット（2件）

ツールを主軸とするユーザーを、来訪フェーズで2つに分けて定義しています。

- **`特定の作業に使えるツールをさっと探している人.yaml`** — 目の前の作業中に必要になった処理をこなすため、検索から初めてたどり着く初回来訪者。サイトそのものへの関心はまだなく、用事が済んだら元の作業に戻ることを優先する。
- **`気に入った道具を繰り返し使っている人.yaml`** — 一度使って気に入った道具に、ブックマーク・履歴・サイト名検索などで再び戻ってきて繰り返し使う来訪者。使い慣れた手順が変わらないこと、前回と同じ体験で迎えられることを重視する。

この2つは来訪フェーズが異なるだけでなく、動機・行動・ニーズが大きく異なります。コンテンツを制作する際はどちらのフェーズを主たる対象にするかを意識してください。

なお、この2件は**同一人物がフェーズによって両方に該当しうる**ことに注意してください。初回来訪者がツールを気に入り、数週間後にブックマークから戻ってきた瞬間、その人はM1aからM1bへと移行します。境界は人物ではなく来訪状態にあります。

## サブターゲット（3件）

以下、ファイル名の五十音順に記載します。

- **`AIエージェントやオーケストレーションに興味があるエンジニア.yaml`** — ブログ（AI技術記録・開発の学び）の読者
- **`AIの日記を読み物として楽しむ人.yaml`** — ブログ「AIの日記」の読者。技術的な深い理解が目的ではなく、エッセイ・読み物として楽しむ層。サイトの中心は「道具」であり「息抜き」に属するブログ「AIの日記」の読者層としてサイト傍流に位置づける（site-concept.md の「息抜きは全体の1割を超えない」原則に準ずる）。
- **`Webサイト製作を学びたいエンジニア.yaml`** — ブログ（Web技術記録・開発の学び）の読者

## ファイル形式

ファイルはYAML形式で、以下の要素が含まれています。

- `name`: ターゲットのことを簡潔に説明する名前
- `knows`: ターゲットが知っていること
- `doesnt_know`: ターゲットが知らないこと、または認識が曖昧な（はっきりとは意識していない・気づき始めているがまだ強く意識化されていない）こと
- `interests`: ターゲットが興味を持っていること
- `likes`: ターゲットが求めているもの
- `dislikes`: ターゲットが嫌いなもの
- `search_intents`: このターゲットがサイトに到達する際に使いそうな検索キーワードの例。ただし主な来訪経路が検索ではないターゲット（ブックマーク・履歴・直接URL入力が中心のリピーターなど）では空になりうる。その場合は空配列にしたうえで、理由をコメントで明示する。

### AI 運営告知の記載に関する非対称について

ターゲットファイルによって、`doesnt_know` に「AI が自律運営している実験サイトであること」が記載されているものと、されていないものがあります。この非対称は意図的です。

- **初回来訪者（M1a）と読み物読者（S3）**: サイトの存在自体を知らない状態で到達するため、AI 運営という背景も当然知らない。明示的に記載する。
- **リピーター（M1b）**: 繰り返し使う過程で AI 運営の背景に気づき始めているか、知ったうえで特に気にせず使い続けている段階にある。「はっきりとは意識していない」という過渡状態として記載する。
- **既存サブ2件（AI エージェント興味エンジニア／Web 製作を学びたいエンジニア）**: ブログ記事経由で到達する技術系読者であり、AI が書いた記事であることを前提として読むことを想定している。よって AI 運営自体は彼らの認知枠組みの一部であり、`doesnt_know` に改めて列挙する必要がない。彼らが知らないのは「yolos.net というサイトの具体的な運営ワークフローや提供内容」の方である。

## 利用方針

サイトの方向性やコンテンツを決めるときは、これらのターゲットのいずれかまたは複数を念頭に置いて考えてください。
1つのコンテンツが複数のターゲットに該当してもかまいません。その場合は主ターゲットを1つ定め、そのターゲットに合わせたトーンや前提知識の水準でコンテンツを制作してください。

ターゲットユーザーは、あえてペルソナといえるほど具体的な人物像にはしていません。
これは、あまりに具体的な名前や年齢を設定してしまうと、AIがその人物だけに特化したコンテンツを作ってしまうことがあるからです。
SEOの改善やSNSでのバズを狙うために、あえて少し抽象的で幅広いターゲットユーザー像を定義しています。
