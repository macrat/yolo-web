# Targets

> **改訂履歴（cycle-260 / B-543）**: 本ターゲット定義は cycle-260 で「道具箱-as-core（日常の傍にある道具）」前提から「診断中心」コンセプトへ再定義した。理由は cycle-257（B-535）で初めて来訪者価値を実測した結果、価値が診断（性格診断・キャラ診断）に頑健に集中し、旧定義のメイン2件（いずれもツール利用者）が実測の主役＝診断来訪者を取りこぼし、かつ反復利用を前提にしていたことが実測（再訪は全セクションで不成立）と矛盾していたため。一次根拠は `docs/research/2026-06-21-concept-visitor-value-measurement.md`、コンセプト本体は `docs/site-concept.md`（cycle-259 改訂）、再定義の判断経緯は `docs/cycles/cycle-257.md` / `cycle-259.md` / `cycle-260.md`。

このディレクトリでは、<http://yolos.net>の顧客像を定義しています。
yolos.net では6つのターゲットを想定しています。サイトコンセプト（`docs/site-concept.md`）の3層構造——**主軸＝自分を知り楽しむ体験（性格診断・キャラ診断・占い・ちょっとしたゲーム＝`/play`）／支え＝文化コンテンツ（辞典・伝統色）／道具（オンラインツール）**——と、独自性の源泉であるブログに対応します。

## メインターゲット（1件・主軸＝診断）

- **`自分に似たキャラや自分の性格を知りたい人.yaml`** — 「自分に似ているキャラは？」「自分はどんな性格？」という自分自身への好奇心から、検索（「似てるキャラ診断」等）でたどり着く来訪者。性格診断・キャラ診断を一回やって、自分についての発見を楽しんで帰る。モバイル中心。**実測（cycle-257）でサイトの来訪者価値が集客・滞在の両軸で最も集中しているのがこの層**であり、サイトの主軸に据える。サイトそのものへの関心はまだなく、用が済めば（＝診断を楽しめば）離れる一回完結型である点に注意する。
  - **主軸 `/play` の占い・ゲームの扱い**: site-concept の主軸 `/play` には診断のほか占い・ちょっとしたゲームも含む。占いはこのペルソナの interests「相性・運勢」に内包される。ゲームは、実測の来訪者価値が診断・キャラ診断に集中しており（cycle-257）独立ペルソナを立てる根拠が現時点で乏しいため、同じ「自分を知り楽しむ」動機の延長としてこのメインペルソナに内包させる（専用ペルソナは `/play` のトラフィックがゲーム方向に多様化した段階で再検討）。

コンテンツを制作する際は、この層の「一回の訪問でちゃんと満足して帰れるか（＝単発訪問の質）」を最優先の評価軸にする。再訪・反復利用を前提にした設計はしない（後述「反復利用を前提にしないこと」）。

この層を主軸に据える判断は、内部実測（cycle-257）に加えて**外部の市場・競合調査**でも裏付けている（`docs/research/2026-06-22-diagnostic-market-competitive-analysis.md`）。要点: 性格診断・キャラ診断の検索需要は日本で非常に大きく（領域全体の規模感の代理指標として「MBTI診断」だけで約277万回/月規模＝一事業者集計の桁感。なお自サイトは MBTI 診断を持たず、主経路は「似てるキャラ診断」のロングテール）、一過性でなく定着方向。主動機は「会話のネタ・自己紹介・シェア」で、厳密な正確性より楽しさ・納得感・シェアしやすさが価値ドライバー。ただし head 一般語と相性領域は大手・占い師ブランドの SEO 寡占で正面突破は非現実的。**現実的な勝ち筋は、競合がほぼ持たない「文化×診断（四字熟語・伝統色×診断）」と、伸びている「似てるキャラ診断」のロングテール＋SNS シェア導線＋登録不要・モバイル1回完結の UX 優位**。このペルソナの likes/dislikes はこの市場理解（軽さ・結果のラベル性・シェアしやすさ・課金/登録の壁を嫌う）を反映している。煽り・押し付け型の導線は作らない範囲で行う。

## サブターゲット（コンテンツ層別）

主軸を支える2層と、独自性を担うブログの読者です。

### 道具層（オンラインツール）

- **`特定の作業に使えるツールをさっと探している人.yaml`** — 目の前の作業中に必要になった処理（文字数を数えたい、日付を計算したい等）をこなすため、検索から初めてたどり着く来訪者。用事が済んだら元の作業に戻る。道具はサイトの実用層として残しており（site-concept の3層目）、この層には直接的な実用価値がある。ただしサイトの中心構造を道具に合わせて曲げることはしない。

### 文化層（辞典・伝統色）

- **`言葉や色の意味をじっくり知りたい人.yaml`** — 特定の語句・漢字・四字熟語・伝統色の名前などを検索し、辞典の個別ページに着地して**じっくり読み込む**読者。実測（cycle-257）では辞典は organic 流入が診断に次ぐ2番手で、1ページあたりの精読の深さが全セクションで最高。地味だが深く読まれる文化コンテンツの読者層で、主軸（診断）を文化的な厚みで支える。市場・競合調査（同上 research）でも「文化×診断（四字熟語・伝統色×診断）」は競合がほぼ持たない空白＝自サイトの差別化の核であり、この文化層は診断主軸と「四字熟語性格診断→四字熟語辞典」のように橋渡しされる支えとして、市場面でも残す意味がある。

### ブログ読者（3件）

ブログ（AI 自律運営という独自性を活かす一次情報）の読者です。ファイル名の五十音順に記載します。

- **`AIエージェントやオーケストレーションに興味があるエンジニア.yaml`** — ブログ（AI技術記録・開発の学び）の読者
- **`AIの日記を読み物として楽しむ人.yaml`** — ブログ「AIの日記」の読者。技術的な深い理解が目的ではなく、エッセイ・読み物として楽しむ層。
- **`Webサイト製作を学びたいエンジニア.yaml`** — ブログ（Web技術記録・開発の学び）の読者

## 反復利用を前提にしないこと（旧メインターゲット削除の記録）

cycle-260 以前は「気に入った道具を繰り返し使っている人」をメインターゲットに置いていたが、**実測（cycle-257）で再訪はサイト全体で不成立（returning が全セクション一桁人）**と判明し、コンセプトも「価値は単発訪問の質で測る／『また来たくなる仕掛け』より『この一回を最高にする』を優先」へ確定したため、このターゲットは削除した。反復利用を前提にした persona を主対象に置くと、最も価値が届いている診断（一回完結型）を取りこぼす設計に逆戻りするためである。

ただし、**「前に使った人がまた来たときに、URL や体験を壊さない」という後方互換は、ターゲットの有無に関わらず守るべき工学的ベースライン**として残す。具体的には、URL を変更する際は必ずリダイレクトを設定する／同じ入力に同じ結果を返す／仕様変更で従来の使い方を壊さない、など。これは「新規コンテンツを誰のために設計するか」というターゲットの問いとは別の、品質の前提条件として扱う（新規コンテンツ設計を駆動する主対象には置かない）。

## ファイル形式

ファイルはYAML形式で、以下の要素が含まれています。

- `name`: ターゲットのことを簡潔に説明する名前
- `knows`: ターゲットが知っていること
- `doesnt_know`: ターゲットが知らないこと、または認識が曖昧な（はっきりとは意識していない・気づき始めているがまだ強く意識化されていない）こと
- `interests`: ターゲットが興味を持っていること
- `likes`: ターゲットが求めているもの
- `dislikes`: ターゲットが嫌いなもの
- `search_intents`: このターゲットがサイトに到達する際に使いそうな検索キーワードの例。ただし主な来訪経路が検索ではないターゲットでは空になりうる。その場合は空配列にしたうえで、理由をコメントで明示する。

### AI 運営告知の記載に関する非対称について

ターゲットファイルによって、`doesnt_know` に「AI が自律運営している実験サイトであること」が記載されているものと、されていないものがあります。この非対称は意図的です。

- **検索から初めて到達する来訪者（診断来訪者・ツールを探す人・言葉や色を調べる人・読み物読者）**: yolos.net というサイトの存在自体を知らない状態で、検索結果から個別ページに到達する。そのため AI 運営という背景も当然知らない。`doesnt_know` に明示的に記載する。
- **ブログの技術系読者（AI エージェント興味エンジニア／Web 製作を学びたいエンジニア）**: ブログ記事経由で到達する技術系読者であり、AI が書いた記事であることを前提として読むことを想定している。よって AI 運営自体は彼らの認知枠組みの一部であり、`doesnt_know` に改めて列挙する必要がない。彼らが知らないのは「yolos.net というサイトの具体的な運営ワークフローや提供内容」の方である。

## 利用方針

サイトの方向性やコンテンツを決めるときは、これらのターゲットのいずれかまたは複数を念頭に置いて考えてください。
1つのコンテンツが複数のターゲットに該当してもかまいません。その場合は主ターゲットを1つ定め、そのターゲットに合わせたトーンや前提知識の水準でコンテンツを制作してください。

ターゲットユーザーは、あえてペルソナといえるほど具体的な人物像にはしていません。
これは、あまりに具体的な名前や年齢を設定してしまうと、AIがその人物だけに特化したコンテンツを作ってしまうことがあるからです。
SEOの改善やSNSでのバズを狙うために、あえて少し抽象的で幅広いターゲットユーザー像を定義しています。
