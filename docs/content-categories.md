# コンテンツカテゴリ定義

本ドキュメントは、サイト内のインタラクティブコンテンツを4つのカテゴリに分類し、それぞれの定義と品質基準を定める。

---

## 1. 診断 (Personality Quiz)

**定義**: ユーザーの回答に基づいて決定論的に結果が決まるコンテンツ。同じ回答をすれば常に同じ結果になる。

**主な特性**:

- 結果の決定方法: 質問への回答をスコアリングし、得点分布でタイプを決定
- 再現性: 完全に再現可能（同じ回答 → 同じ結果）
- ランダム要素: なし
- 来訪者の期待: 「自分の性格・タイプを知りたい」「答えが当たっているか確かめたい」

**判別基準**: 「同じ回答を2回しても同じ結果が出るか?」 → YES なら診断

### 品質基準

| 項目         | 最低基準 | 推奨基準            | 理由                                   |
| ------------ | -------- | ------------------- | -------------------------------------- |
| 結果タイプ数 | 4タイプ  | 6〜8タイプ          | 少ないと「外れ感」が強くなる           |
| 質問数       | 6問      | 8〜12問             | 少ないと精度が出ない。多すぎると離脱増 |
| 結果文字数   | 200文字  | 300〜500文字        | 読み応えと納得感に必要                 |
| 相性診断     | 任意     | n\*(n+1)/2 組を推奨 | リピート・シェアのきっかけになる       |

---

## 2. 占い (Fortune-telling)

**定義**: ユーザーの固有属性（生年月日・名前・血液型等）または日時情報を使って結果を出すコンテンツ。質問への回答ではなく「固有属性」が入力値。

**主な特性**:

- 結果の決定方法: 生年月日・星座・時間帯等の外部属性から計算
- 再現性: 属性が変わらない限り変わらない。時間依存型（今日の運勢）は日付で変わる
- ランダム要素: なし（決定論的）だが、属性への依存で「変えられない感」が占い的な体験を生む
- 来訪者の期待: 「自分の運命・未来を知りたい」「今日の運気を確認したい」

**判別基準**: 「質問に答えず、属性（生年月日等）だけ入力して結果が出るか?」 → YES なら占い

### 品質基準

現時点でサイト内に「占い」カテゴリのコンテンツは存在しない。コンテンツ追加時に品質基準を策定する。

---

## 3. おみくじ (Omikuji)

**定義**: ランダム性が主体で、毎回引くたびに（または毎日）異なる結果が出るコンテンツ。「運」を試す体験が目的。

**主な特性**:

- 結果の決定方法: 乱数（または日付 x ユーザーseedの擬似乱数）で選択
- 再現性: 意図的に低い（同じ日なら同じ結果、翌日は変わる等の設計もある）
- ランダム要素: 中核的要素
- 来訪者の期待: 「今日の運を試したい」「面白い結果が出るかな」。回答ベースではなく「引く」体験

**判別基準**: 「何も答えず、ただ引くだけで結果が出るか?」 → YES ならおみくじ

### 品質基準

| 項目               | 最低基準 | 推奨基準                   | 理由                             |
| ------------------ | -------- | -------------------------- | -------------------------------- |
| パターン数         | 10       | 30〜60                     | 少ないと「また同じ」感が出やすい |
| 毎日変化           | 必須     | 必須                       | ユーザーの期待値に直結           |
| ラッキー要素       | 任意     | アイテム+アクションの2要素 | コンテンツの面白さを補完         |
| 各パターンの文字数 | 100文字  | 150〜250文字               | 短すぎると価値を感じない         |

---

## 4. 知識テスト (Knowledge Quiz)

**定義**: 正誤がある問題に回答し、正答率でランク判定するコンテンツ。学習価値がある。

**主な特性**:

- 結果の決定方法: 正答数・正答率によるランク判定
- 再現性: 同じ回答なら同じ結果（決定論的）
- ランダム要素: なし
- 来訪者の期待: 「自分の知識レベルを確かめたい」「新しいことを学びたい」

**判別基準**: 「各質問に正解・不正解があるか?」 → YES なら知識テスト

### 品質基準

| 項目       | 最低基準             | 推奨基準        | 理由                     |
| ---------- | -------------------- | --------------- | ------------------------ |
| 問題数     | 10問                 | 10〜15問        | 適切な達成感             |
| 難易度分布 | 易〜難をバランスよく | 易3:中5:難2程度 | 幅広いユーザーが楽しめる |
| 解説文     | 全問に説明           | 全問+参考情報   | 学習価値が高まる         |
| スコア段階 | 5段階                | 5段階           | 現状の実装は適切         |

## カテゴリ判別フローチャート

```
結果はランダムまたは日付/時刻で変化するか?
  |
  +-- YES --> ユーザー属性（生年月日等）も使うか?
  |             |
  |             +-- YES --> 「占い（時間依存型）」
  |             +-- NO  --> 「おみくじ」
  |
  +-- NO  --> ユーザーが質問に答えるか?
                |
                +-- YES --> 各質問に正解・不正解があるか?
                |             |
                |             +-- YES --> 「知識テスト」
                |             +-- NO  --> 「診断」
                |
                +-- NO（属性のみ入力）--> 「占い（属性型）」
```
