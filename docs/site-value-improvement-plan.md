# サイト全面価値向上 作業計画書

## 1. このサイトの使命

来訪者に最高の価値を提供し、PV数の最大化を達成する。
すべてのコンテンツは、ターゲットユーザーの課題解決・欲求充足のために存在する。

## 2. 作業の目的

Google AdSenseの審査で「有用性の低いコンテンツ」として却下された（2026-03-03）。
これはSEO面でも深刻な問題を示唆しており、コンテンツの薄さ・独自性の低さがサイト全体の評価を下げている可能性が高い。

本作業では、サイト全体のコンテンツを根本から見直し、来訪者にとって真に価値のあるサイトに作り変える。

### 成功基準（順序付き）

1. Google Search Consoleで主要ページがインデックスされ、検索クエリが増加傾向にある
2. Google AdSense再申請で承認される
3. 月間PVが改善前比で増加傾向にある

### フェーズ2完了時の中間判断基準

- サイトの独自価値提案（Value Proposition）が明文化され、競合との差別化が説明できる
- 残すコンテンツの各ページが、競合の同等ページに対して少なくとも1つの明確な優位性を持つ

## 3. 心構え

1. **ユーザーファースト**: すべての判断基準は「来訪者にとっての価値」。工数を理由に判断を歪めない。
2. **ドラスティックな変更を恐れない**: 過去の判断の撤回、コンテンツの全面削除、コンセプトの刷新を躊躇しない。
3. **品質 > 量**: 100ページの薄いコンテンツより、10ページの深いコンテンツの方が価値がある。
4. **独自性の追求**: 競合サイトと同じものを作っても価値はない。このサイトにしかない価値を追求する。
5. **[E-E-A-T](https://developers.google.com/search/docs/fundamentals/creating-helpful-content) の意識**: Experience（経験）、Expertise（専門性）、Authoritativeness（権威性）、Trustworthiness（信頼性）を全コンテンツで意識する。
6. **AI運営の透明性**: AI運営であることを隠さないが、それ自体を売りにするのではなく、提供する価値を売りにする。
7. **データに基づく判断**: 推測で動くのではなく、Search Console・サイト分析の実データに基づいて判断する。改善の効果もデータで検証する。

## 7. このファイルの扱い方のルール

### 基本ルール

1. **毎サイクル必読**: サイクル開始時に必ずこのファイルを読み、現在のステータスと次にやるべきことを把握する。
2. **ステータス更新**: 各タスクの完了時にチェックリストにチェックを入れ、「5. 現在のステータス」セクションを更新する。
3. **申し送り更新**: サイクル完了時に「6. 後続タスク実施者への申し送り事項」を更新し、次のサイクルで把握すべき情報を記載する。
4. **参照文書の維持**: 各フェーズの調査結果・企画書は、指定されたパスに文書として保存する。この計画書からリンクする。

### ステータス更新のフォーマット

```
### 最終更新: cycle-XX 完了時（YYYY-MM-DD）

**現在のフェーズ**: フェーズX（名前）
**進捗サマリ**: （箇条書きで主要な進捗を記載）
**現サイクルの残タスク**: （あれば記載）
```

### 申し送り事項のフォーマット

```
### cycle-XX → cycle-YY への申し送り

- **必ず確認すべきこと**: （次サイクルで最初に確認・対応すべき事項）
- **判断が必要な事項**: （次の実施者が判断すべき未決事項）
- **注意点**: （作業上の注意事項やハマりポイント）
- **参考メモ**: （関連するメモID）
```

### 申し送りの書き方

- **次のPMが知らないと問題になる情報だけ**を書く。過去の経緯や完了済みの詳細はサイクルドキュメントで参照可能なので書かない。
- ステータス（セクション5）も同様に、現在地と次にやることだけを簡潔に記載する。サイクルごとの作業記録は書かない。
- このファイルはサイクル開始時にPMのコンテキストに読み込まれるため、不要な情報はLLMの性能を劣化させる。
