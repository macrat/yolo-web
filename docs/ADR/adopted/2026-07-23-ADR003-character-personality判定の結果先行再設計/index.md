# ADR003: character-personality 判定の結果先行再設計と、受け入れ基準の撤回方法論

- 起票日 / 出荷日: 2026-07-23（cycle-295・出荷と同日=正確性は測度非依存ゲートで確定済のため観測窓を要さない純粋な決定）
- 決定・仮説:
  1. **玄関 character-personality（サイト最高PVページ）の判定を結果先行で作り直す**。旧=24タイプへ直接配点し同点を `quiz.results` の配列順で決着（回答でなくファイル並び順が36〜46%の結果を決め、2タイプ academic-artist/clever-guardian は本人の最大化回答でも永久に到達不能）=constitution Rule4 違反。新=各選択肢の主signalアーキタイプの**被選択回数（count軸）**で主軸/副軸を決め、6アーキタイプの主×副で24タイプへ写像する専用判定 `determineCharacterPersonalityResult`（science-thinking 前例に倣う）+反同点配点。設問文も結果先行で再設計。得点軸でなく count 軸にしたのは副signalの cross-talk を軸決定から排除し理想回答者テストを 24/24 にするため。
  2. **受け入れ基準の撤回方法論（precedent）**: 設計着手前に凍結した合否ゲートが原理的に不能と判明した場合の是正手順。本サイクルでは feasibility spike で「同型を gap≥TH で判定＋一様測度に同型フロア G4c を課す」構造が G4a と機構非依存で数学的に相反すると確定（独立再実装で検証）。PM が constitution Goal（来訪者価値）に接地して **G4c を撤回**し、**独立 reviewer の白紙審査**で「原則上の是正か AP-P02 のゴールポスト移動か」を判定させ承認（AP-P02 の4テストすべて不該当）。

- 確定の根拠（観測窓を要さない理由）: 決定の正当性は**測度非依存の受け入れ基準**で確定済み——G1（∀理想回答者 24/24・本番判定）・G2（悉皆 4^12 到達不能0）・G4a（現実的測度 c=1.0/0.85 で全24≥1%）・G4b（比 2.54×）・G5（最悪c 恣意決着率 1.576% ≤5%）。本番 vs 高速経路 70,000/70,000 一致・計器のゴミ検定で検出力確認。旧・本人到達不能の2タイプが到達回復。行動データ（share・滞在）を成功基準に置かない（cycle-284 の確定=当ページの行動データは現状読めずトートロジーになる）。
- 従うべき先例（このADRが precedent として残す判断）:
  - **性格診断は「タイプが先・設問が判別する」設計**（cycle-294/incident-2 line 100）。「タイブレーク」が判定の中心語になったら後付け割り振りの壊れた枠。既存設問を固定した配点調整（cycle-284 §5）は同じ枠なので採らない。
  - **受け入れ基準の撤回は「価値性質ゲート」と「機構固有分布ゲート」を区別して行う**（撤回論理の封じ込めガードレール）。G5/G1/G2 は「回答が結果を一意に決める・本人に届く」という**全回答者共通の価値性質**を測る=緩和不可。G4c（撤回）は**機構固有の型分布**（一様ランダム回答での同型頻度）を要求していた=来訪者価値に無関係。**「一様は実来訪者でない」という撤回論理は機構固有分布ゲートにのみ適用し、価値性質ゲート（特に G5-at-uniform）の緩和に転用してはならない**——転用は芯の実害の測度を落とす真正の AP-P02。
- 観測: 独立の観測窓は設けない（正確性は確定済）。ただし**来訪者エンゲージメントへの効果は ADR001（サイト刷新）の観測窓の交絡要因**として記録済み——2026-07-27 に GA4 の診断開始率/完走率/share を読む際、本変更（07-23）を切替（07-13）と別要因として分けて読む（ADR001 経過記録・次回確認 to-do に記載）。
