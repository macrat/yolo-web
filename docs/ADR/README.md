# ADR（意思決定と実験の記録）

重大な意思決定と実験（賭け）を、**1件=1ディレクトリ**で記録する。「決定を記録するもの」と「賭けて観測するもの」を同じ書式で扱う（ADR 兼実験記録）。

## 構造（ステータス=置き場所・三状態）

- `docs/ADR/open/YYYY-MM-DD-ADR###-タイトル/` — **実験中**（観測・フォローアップが残っている）。**先頭の日付は「次回確認日」**（起票日ではない）。
- `docs/ADR/adopted/YYYY-MM-DD-ADR###-タイトル/` — **確定**（採用され、現在も効力を持つ判断=従うべき先例）。先頭の日付は確定日。観測不要の純粋な決定は起票時からここに置いてよい。
- `docs/ADR/expired/YYYY-MM-DD-ADR###-タイトル/` — **失効**（後の決定に覆された・賭けに負けて撤退した・環境変化で無効になった）。先頭の日付は失効日。**index.md に「何によって失効したか」（覆した ADR 番号・観測結果等）を必ず書く**。
- adopted と expired は参照意味が正反対（従うべき先例／もう従ってはいけない記録）なので混ぜない。
- index.md が正典。同一ディレクトリに任意の参考資料（調査メモ・データ・試作コード等）を置いてよい。
- **参照は必ず「ADR###」の番号で行う**（ディレクトリ名はリネームされるため、パス参照は壊れる）。番号は全状態通しの連番（最大+1）。

## index.md の書式

```
# ADR###: タイトル
- 状態: 実験中（open）| 確定（adopted）| 失効（expired）
- 起票日 / 出荷日:
- 決定・仮説: 何を決めた/何に賭けたか（実験は来訪者価値の仮説を明記）
- 根拠: 一次資料・調査・実測へのポインタ
- 観測手段と評価窓 / 撤退基準:
- 次回確認時にやること: ←【必須】期限で開いた PM が、これだけ読めば動けるように書く
- 経過記録: （確認のたびに日付付きで追記）
- 結果・学び: （クローズ時に記入。負けの記録は勝ちと同粒度で）
```

## ライフサイクル（サイクル工程への組み込み済み・スキルが正）

1. **kickoff**（cycle-kickoff スキルに手順あり）: `ls docs/ADR/open/` を実行し、**先頭日付が今日以前のものだけ** index.md を読んで「次回確認時にやること」をサイクル作業に含める。未来日のものは開かない（コンテキスト節約）。
2. **サイクル中**: 重大な決定・賭けをしたら**出荷前に起票**する（事前登録が先・出荷が後。事後の合理化を防ぐため起票日と出荷日を分けて記録）。
3. **completion**（cycle-completion スキルに手順あり）: このサイクルに記録すべき重大決定・出荷した賭けがないか確認し、あれば起票/更新。観測したら経過を追記し、**ディレクトリを次回確認日へリネーム**。決着したら結果・学びを書いて **adopted/**（賭けに勝った・決定が確認された）または **expired/**（負けて撤退・無効化）へ移動。
4. **失効は覆す側の操作**: 新しい決定が過去の ADR を覆すときは、新 ADR に旧番号を明記し、**旧 ADR を expired/ へ移して「ADR◯◯◯により失効」と記入**する（相互参照）。adopted の判断にあとから矛盾する変更を入れる前に、その領域の adopted を確認する。

## 観測の規律

- **Search Console**: 数値の撤退基準を置ける唯一の側（外部母集団）。反応は約3ヶ月遅れ——評価窓に織り込む。
- **GA4**: 小 n・計測欠落を常に疑い、単独の kill 根拠にしない。
- **A/B**（`src/lib/ab/`・面単位のインライン実験）: 統計的有意の道具ではなく、同時期対照の質的観察の手段。サイト全面の A/B はしない（`docs/rebuild-plan.md` §1）。
- 時間軸の before/after は方向読みのみ（因果を断定しない・双方向ガード）。
- 打ち切りは打ち切りとして残す。ただし結果がどう出ても使い道のない記録は残さない（記録は目的でなく手段）。
