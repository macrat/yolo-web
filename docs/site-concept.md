# yolos.net サイトコンセプト

> **この文書の使い方**: コンテンツについて「追加すべきか？（削除すべきか？）」「どう作るべきか？」に迷ったとき、判断基準として参照する。ここには具体的なコンテンツや計画は書かない——それらは各サイクルの計画文書・調査文書で扱う。
>
> **改訂履歴**: cycle-278（オーナー主導）で旧版（「診断中心」コンセプト）を削除し、ゼロから書き直した。導出の全根拠（内部実測・外部調査・判定とレビューの経緯）は `docs/research/2026-07-11-market-research-cycle278.md`。

## このサイトは何か

**yolos.net は、検索してたどり着いた人が「読むだけでは手に入らないもの」をその場で受け取れるサイト。** 自分の入力から生まれる結果、持ち帰れる成果物、遊べる体験を提供する。AI が運営する実験サイトであることを明示する（constitution rule 3）。

サイト上の自己表示（トップ・about・メタデータ）の書き換えは、実体が伴った時点で行う。この文書の自己定義は「作る際の基準」であり、宣言を実装に先行させない。

## 唯一のゴール

constitution の Goal——**来訪者に最良の価値を提供することによる PV の最大化**。この文書のすべての規則はこのゴールの手段であり、規則がゴールと衝突したら規則の方を改訂する。

## 中心原則と弁別子

**「検索者の『したい』に、関与して得る体験・成果物で応える」**

- 『したい』は広義に取る——試したい・作りたい・計算したい・変換したい・印刷したい・遊びたい・自分を知りたい等。SEO 用語の取引/タスク意図に狭めない（狭めると娯楽が落ちる）。
- **弁別子**: 「その価値は、検索結果の画面上でテキストとして配達しきれるか（＝来訪せずに得られるか）」。
  - **配達しきれるなら外（参照型）**: 辞典の語義・全員共通の静的テキスト・解説記事・検索エンジンが SERP 内ウィジェットで即答する単純な計算/換算。これらはゼロクリック環境で構造的に訪問を失う。
  - **来訪して関与しないと得られないなら内（体験型）**: (a) 入力から生まれる結果、(b) 持ち帰る成果物（印刷物・ファイル・スコア）、(c) 遊ぶ体験そのもの（全員共通の出題でも、価値が問題文でなくプレイにあるなら内）。
  - 注意: **操作・入力・単一の lookup を挟むこと自体は体験型の証明にならない**——中身が静的に配達可能なら参照型である。
- この弁別子のうち「参照型が訪問を失う」ことは強い定量で確認済み、「体験型が耐性を持つ」ことは中強度仮説である（後述のトリップワイヤで検証を続ける）。

## 構造の原則: エンジン × コンテンツ

サイトは「**磨き込んだ少数のエンジン**（体験の器）× エンジンごとの**多数のコンテンツ**（1ページ=1実在需要）」の形を取る。集中（器の品質に複利が効く）と探索（当たりを引く試行回数）を両立し、個々のページの反応を自サイトの少ないトラフィックではなく検索という外部の巨大な母集団で測れる、当サイトの規模で成立する最良の形だからである。

- 傘（サイト全体のテーマ）はピンポイントである必要はない。ただし集中の複利は**エンジン単位**で効き、エンジンを開き過ぎればその分だけ薄まる——新しいエンジンを開くことは、既存エンジンに中身を足すことより常に重い判断である。
- **AI の無限供給・量産力は差別化の第一の柱にしない**。効くのは、器の品質が確立し人気が出た後の維持コスト優位としてのみ。

## 判断基準 1: このコンテンツを追加すべきか

以下を**すべて**満たすときだけ追加する。

1. **中心原則の内側にある**（弁別子テストに合格する）。
2. **実在する検索需要に応える**——需要（クエリ）から出発し、需要の実在を証拠（サジェスト・供給の横並び・実測）で確認してから作る。「作れるから作る」は禁止。現状の流入クエリを需要の全体像と混同しない（今当たっているものは早咲きにすぎない可能性があり、当たっていないものは遅咲きかもしれない）。
3. **それ単体で独立したユーザー価値を持つ**——薄いページの量産は、品質の憲法違反（rule 4）であると同時に検索エンジンのスパム認定（scaled content abuse）の直撃圏である。面の量産で露出を稼ぐことはどのエンジンでも禁じ手。
4. **既存エンジンの中身として作れる**か、新エンジンを開くなら集中の複利の希薄化と品質保守力を考量した明示的な判断を経ている。
5. **ガード（後述）に抵触しない**。
6. **着手前に評価窓と撤退基準を決めてある**（決めずに作り始めない）。

## 判断基準 2: このコンテンツを削除（撤退）すべきか

- **評価窓を経てから判断する**。検索の反応は遅い（当サイトの実測で、当たったページも順位獲得は作成の約3ヶ月後）。早く切りすぎる失敗（遅咲きの誤殺）と、サンクコストによる塩漬けの両方を、事前に決めた評価窓と撤退基準だけが防ぐ。
- **数値の撤退基準は Search Console（表示・順位・CTR）側に置く**。オンサイトの即時指標（完了率・エンゲージ）は母数が小さく統計的判定に耐えないため、絞り込みの参考に留め、単独の削除根拠にしない。
- **外れは放置しない**——低品質ページの放置はサイト全体の評価を引き下げる。評価窓を経て外れと判断したものは改善するか消す。
- **勝者が出たら探索を続けず、勝者への集中に切り替える**。
- 継続の理由が「もう投資したから」「愛着があるから」になっていたら、それは撤退のサインである。
- **開く判断（基準1）と閉じる判断（基準2）は、別々に運用せず同じ評価設計の中で行う**——何かを追加するとき、その撤退条件も同時に決まっていなければならない。

## 判断基準 3: どう作るべきか

- **品質と面白さの審判は、来訪者の行動データのみ**。自分やレビュアーの「面白い」を判定に使わない。ただし行動データを読むときは母数の小ささ・計測の欠落を常に疑う（集計値の中身を確かめずに満足と読まない）。
- **AdSense 等の外部審査は「存続の必須条件（ゲート）」であって品質の定義ではない**。審査を通すためのコンテンツ作りに滑らない。
- **「LLM のウィット」を品質の前提にしない**。LLM が狙って書く面白さは大抵人間に伝わらない。面白さは機構（偶発的なズレ・意外な組み合わせ・自分の入力との衝突）から出す。LLM の役割は部品の大量供給と検品である。
- **来訪者の目に触れる生成物はすべて出荷前に検証する**。機械検証できる題材（決定的計算・辞書照合・解の一意性）を優先する。検証できない自由生成を大量に出荷しない。
- **シェア可能な成果物（結果カード・スコア・生成物）を最初から設計に内蔵する**——SNS がリンクを抑制する環境で機能する唯一の拡散型。ただし独自の記号が「共有できる記号」に育つかは未検証の賭けであり、行動データで検証する。
- **デイリー/ストリーク等の再訪装置は、検索で人が付いた後の後段装備**。先行させない（失敗済みの方向）。後段で扱う場合も固有リスク（目新しさの減衰・着火の初速がゼロであること）を前提に置く。
- **探索しやすく整理する**（rule 4 後半）——コンテンツを増やすことは、来訪者にとっての分類・回遊・見つけやすさの設計義務を伴う。

## ガード（constitution 接地）

- **rule 1（法令）**: 実在キャラクター名・実在作品名を返すコンテンツで検索意図を満たさない（著作権）。商標を回避する。意図の充足は、独自の記号の魅力と正直な約束（タイトル・説明が中身と一致すること）の側で追求する。
- **rule 2（害の禁止）**: 占い・鑑定系は、断定・不安煽り・依存助長の禁止、重大な人生の決断（結婚・離別・名付けの吉凶等）への非関与、不安を残さない締めを必須設計とする。医療・金融等の重大判断（YMYL）に踏み込まない。来訪者のデータを預かる体験は喪失から守る。
- **rule 3（AI 明示）**: AI 運営の明示を維持する。人間の著者を装う偽装は検索品質評価上も最低評価に直行し、明示はむしろ保護的に働く（一次資料確認済み）。
- **rule 4（量より質・易探索）**: 判断基準 1-3・基準 3 末項に接地。
- **rule 5（創造的多様性）**: 新しい試みは歓迎する。ただし評価窓と撤退基準を伴わせ、賭けであることを記録してから行う。

## この原則が禁じるもの

参照型（読んで終わる情報）の量産／薄いページの量産／飽和した定番ヘッドワードへの汎用再実装／ストリーク・デイリー先行の設計／LLM ウィット依存の品質設計／実在キャラ名による検索意図の充足／「作れるから作る」供給起点の追加。

## この文書の地位と検証

- 本書は検証済み結論と仮説の混合である。**参照型の敗北**は強い定量と内部実測で確認済み。**体験型の耐性**と**独自記号のシェア可能性**は仮説であり、実測で検証を続ける。岩盤は constitution と生の観測データのみで、本書自体もその上の改訂可能な層である。
- **トリップワイヤ**（該当したら本書の該当部を再検証する）:
  - TW-A: 新設した体験型ページ群が、評価窓を経ても参照型と同型の敗北（表示のみ・クリックなし）を示す → 中心原則の体験型耐性仮説を再検証。
  - TW-B: 検索エンジンの SERP 挙動の変化（ウィジェット/AI 回答の適用拡大）が弁別子の線引きを侵食 → 弁別子を再検証。
  - TW-C: 検索流入全体の構造的下落 → 「検索を主戦場とする」前提自体を再検討（この前提は、AI 運営の過程それ自体を見せる方向などの固有資産を後段に回すという代償を伴って選ばれている）。
- 本書の改訂の正当な理由は、観測（生データ）または constitution の変更のみ。慣性・愛着・サンクコストは理由にならない。
