# yolos.net サイトコンセプト

> **改訂履歴（cycle-259 / B-538）**: 本コンセプトは cycle-259 で「日常の傍にある道具（道具箱-as-core）」から「自分を知り、楽しむ体験を主軸に（診断中心）」へ再センタリングした。理由は cycle-257（B-535）で初めて来訪者価値を実測した結果、価値が診断（`/play`）に頑健に集中し、道具箱トップ `/` は organic 集客ほぼ0・中核機能の利用ゼロと判明したため。約50サイクルにわたり道具箱-core を「不変項」に固定し、最も価値の届く診断を「息抜き1割以下・整理対象」と格下げしていた未計測の前提を是正する。判定の一次根拠は `docs/research/2026-06-21-concept-visitor-value-measurement.md`、判定本体と承認は `docs/cycles/cycle-257.md`。

## コアコンセプト

**「自分を知り、楽しむ。そのための場所」**

来た人が「自分のこと」を発見して楽しめる体験——性格診断・キャラ診断・占い・ちょっとしたゲーム——を主軸に置く。それを支えるのが、日本語や伝統色といった文化コンテンツ（読みごたえのある辞典）と、必要なときにすぐ使えるオンライン道具。

このサイトは「いろいろ作って、伸びたものを主軸にさらに展開する」という仮説検証で運営している（後述「戦略」）。**現時点でデータが示す主軸は診断**であり、これは固定された教義ではなく、計測で見直し続ける現在の重心である。最も価値が届いているものを中心に据え、そこへ来た人がもっと多くの価値に出会えるようにする——それがこのコンセプトの芯。

### 3つの層

1. **主軸：自分を知り、楽しむ体験（診断・占い・ゲーム）** — `/play`。性格診断・キャラ診断のクラスタが集客・滞在の両軸で頑健に1位（全期間84日）。新規来訪の主な着地点。サイトの顔はここ。
2. **支え：文化コンテンツ（辞典・伝統色）** — `/dictionary`。地味だが精読される良コンテンツ（イベント単位の読み込み深さが全セクション最高）。主軸を文化的な厚みで支える。
3. **道具（オンラインツール・道具箱）** — `/tools`・道具箱。削除はしない。直接来た人・回遊する人に役立つ実用層として残す。ただし**サイト全体を道具箱に合わせて曲げることはしない**。将来の道具箱への投資は、来訪者トラクションがデータで示されたときに行う（未計測のまま「核」に固定しない）。

## このコンセプトに至った背景

### なぜ「診断」が主軸なのか（データで決めた）

- cycle-257 で初めて来訪者価値を実測した結果、`/play`（診断・ゲーム）が **集客（organic 着地の大半）と滞在の深さ（中央エンゲージが他セクションの15〜20倍）の両軸で頑健に1位**だった。これは小母数でも揺るがず、複数サイクルの観測と一致する。
- 一方、道具箱トップ `/` は全期間を通じて **organic 集客がほぼ0**、道具箱の中核機能（並べ替え・プリセット・カスタマイズ）の利用は観測ゼロだった。「道具を核・診断を息抜きに格下げ」する設計は、データと逆を向いていた。
- 直近では「似てるキャラ診断」系のクエリで organic 流入が増加し、診断へ市場が引いている（着地後のエンゲージは良好だが回遊・再訪はまだゼロ＝伸びしろ）。なおこの直近増はトレンドか減衰か未確定の補強材料であり、主軸判断の確定根拠は上記の全期間84日・両軸1位の方（直近増がなくても成立する）。

主軸を診断に置くのは、好みや思いつきではなく、この実測に基づく。これは下記「戦略：仮説検証アプローチ」——「数ヶ月改良を続け、結果的に伸びたものを主軸にさらに展開する」——の素直な適用である。道具箱を未計測のまま不変項に固定したことが、むしろこの戦略への先走った背反だった。

### なぜ「自分を知り、楽しむ」なのか

- 検索から来る人の多くは「自分に似ているキャラは？」「自分の性格は？」という、自分自身への好奇心で訪れる。診断・占いは、その問いに一回の訪問で気持ちよく答えを返せる。
- 価値は「毎日使う・反復利用」では測れない（現データでは再訪は全セクションで不成立）。**価値は単発訪問の質**——来た人がその一回でちゃんと満足して帰れるか——で測る。だから設計は「また来たくなる仕掛け」より「この一回を最高にする」を優先する。

### なぜ「文化コンテンツ」で支えるのか

- 辞典（漢字・四字熟語・伝統色）は派手ではないが、来た人がじっくり読み込む（精読の深さが最高）。診断で楽しんだ人が、関連する文化コンテンツへ自然に出会えれば、訪問の価値が一段深まる。
- 文化コンテンツは「短い文章を一定品質で大量に」作る難しさがあるが、データ資産（常用漢字2,136字・四字熟語400語・伝統色250色）に裏打ちされた事実中心のコンテンツなので、品質を保ちやすい。

### なぜ「道具」を残すのか

- オンライン道具は AI による自律開発で比較的安定して作りやすく、constitution rule 5「いろいろ試す」の実験の幅を保てる。検索から直接道具に来て使って帰る人には実用的な価値がある。
- ただし「道具箱（タイルを並べる個人ダッシュボード）」という形態自体には、現時点で来訪者トラクションの兆候がない。だから道具は**実用層として残しつつ、サイトの中心構造はそこに合わせない**。道具箱形態への追加投資（タイル化の全面展開・連携・シェア等）は、データで地位を獲得してから行う。

## ブログの位置づけ

ブログは、AI が自律運営しているというこのサイト固有の特殊性を活かす手段である。AI 自律運営の過程で生まれる一次情報（開発の学び、AI の視点や感想）は他では得られない独自コンテンツで、サイトの独自性の源泉になりうる。2種類の記事を書く:

1. **開発の学び**: 開発やワークフローで得られた学び・失敗談。サイクルの中で記録に値する知見が得られたときに書く。読者にも学びを提供することを目指す。
2. **AI の日記**: AI の主観で書いた作業日記。読み物として昇華することを目指す。

ただしブログは現データでは読み捨て傾向（organic は3番手だがエンゲージ・回遊は最下位クラス）であり、**テコ入れか縮小かの判断対象**である。判断の際は metric により評価が逆転する点に注意する（イベント単位の精読では辞典が上だが、セッション滞在ではブログが辞典より長い）。片方の metric だけで縮小を決めない。

## 戦略

### 仮説検証アプローチ

事前に完璧な計画を立てるのではなく、「来訪者への価値」を主目的・「SEO 対策」を副目的としながら、いろいろな方向性のコンテンツを作る。数ヶ月改良を続け、**結果的に伸びたものを主軸として**さらにいろいろな方向性のコンテンツを作る。このサイクルを回すことで仮説検証を行う。

重要なのは、**「主軸」は仮説検証の結論であって前提ではない**こと。主軸（現在は診断）はデータで決め、データで見直し続ける。コンセプトを「不変項」に固定して計測をやめると、cycle-175〜257 で起きたように、最も価値の届くものを取りこぼし続ける。コンセプトの監視と立て直しは PM の継続責務である（憲法ルール5「いろいろ試す」とも整合）。

### 新規開発と改良の比率

新コンテンツの開発1サイクルに対して、既存コンテンツの改良1〜2サイクルの割合で進める。改良サイクルでは以下を基準に作業対象を決める:

- GA4 や Search Console の状況
- そのコンテンツが最後に改良された日からの経過日数

しばらく改良されていないコンテンツを中心に、一番効果が出そうな作業をする。この比率により、新コンテンツの量産に偏ったり、小さな改良に集中して新コンテンツが生まれないといった問題を防ぐ。

### 計測駆動で中心を見直す

主軸・支え・道具の位置づけは、定期的に来訪者価値（集客・滞在の質・回遊）で計測し、ズレていれば立て直す。投資の前に計測する（未計測の前提の上に作業を積まない）。回遊や再訪を増やす施策は「単発訪問の質を損なわず付加価値を足すもの」に限り、数値を上げるためだけの煽り・押し付け型の導線は作らない（ダークパターン禁止）。

### PM のキャラクター性

過去の経験から、理念的な判断基準（「来訪者への価値を最優先とする」等）は明文化しても一貫して適用されにくいことが分かっている。そこで PM にキャラクター性を持たせてロールプレイさせる。「『使う人は喜ぶかな？』が口癖」「使い勝手をいつも気にしている」といったパーソナリティを設定すれば、抽象的な理念よりも日々の判断に自然に反映されやすくなる。

キャラクター性の導入で期待する効果:

- 判断基準の一貫性: 理念の解釈がブレても、キャラクターの行動原理はブレにくい
- ブログ記事の個性: 「AI の主観」が感じられる文章の土台になる
- サイトの世界観の統一: コンテンツの選定・設計・文体にキャラクターが一貫性を与える

具体的なキャラクター像は別途定義する。
