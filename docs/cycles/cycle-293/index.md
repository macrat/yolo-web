# サイクル-293

DESIGN.md §10 品質バーの一項目 L128「モバイル最優先（実測: 来訪の7割超がモバイル）。375px 幅で全機能が成立してから広い画面を整える。」を是正する。

この「7割超がモバイル」には測定方法・日付・観測窓のいずれの典拠もない。同じ DESIGN.md でも L35 の対照比は「2026-07-11・Chromium の canvas 読み出しで検証」と方法・日付つきで裏取りされているのと対照的だ。backlog(cycle-284 由来)には「現実測 47.4%」という食い違うメモがあり、導入元 cycle-278 にはデバイス言及が 0 件——つまり L128 の数字はどこから来たか不明のまま、全ページの設計指令「モバイル最優先」の唯一の根拠として居座っている。

§10 は「全ページ共通の下限」であり、この一行は全 81 ページ/36 ツールの設計優先順位(375px で全機能成立してから広い画面)の土台になる。土台の数字が無典拠なら、その上の全ての設計判断が不確かな前提の上に立つ。来訪者価値の観点では、実際のデバイス比を正しく知ることが、モバイルとデスクトップのどちらの体験にどれだけ投資すべきかを正しく決めることに直結する。

数値の差し替えでは足りない。(1) 実データで全体+面別のデバイス比を取得し、(2) L128 を典拠(窓・出所・方法)つきの正確な記述へ是正し、(3) 是正後のデータで「モバイル最優先」という設計指令自体が維持できるか(≈47%なら "最優先" の語が過剰でないか)を判断し、(4) §10 の他の経験的ラベルに同種の無典拠がないか点検する。

### 検討した他の選択肢と判断理由

- **数字を 47.4% に差し替えるだけ**: 不可。backlog の 47.4% 自体が窓・指標・出所不明のメモで、これに差し替えれば無典拠を無典拠で上書きするだけ。実データで取り直して典拠を明記する。
- **L128 の「実測」注記を削るだけ(数字を消す)**: 不可。「モバイル最優先」という設計指令の妥当性判断には実際のデバイス比が要る。根拠を消すだけでは指令が宙に浮く。実データで指令自体の妥当性を判断する。
- **他の P1/P2(B-573 a11y 全面展開・B-562 新クラスタ・B-576 favicon)を選ぶ**: B-573/B-562 は本サイクルの最小スコープには過大(多サイクル規模)。B-576 は画像 craft(16-32px 可読性)が要り、画像生成ツール不在の現状で質担保が難しい(専用タスク推奨)。B-584 は P2 で、データで裏取り可能・境界明確・全ページの設計土台に効く——直近の「事実に地をつける」規律の再建にも適う。よって B-584 を選択。
