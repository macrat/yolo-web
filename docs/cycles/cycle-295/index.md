# サイクル-295

サイトの玄関=最高PVページ `/play/character-personality`(全PVの31%)は「あなたの回答があなたのキャラを決めた」と手続きを名乗るのに、実際には **36〜46% の回答でトップが同点になり、`quiz.results` 配列の並び順で結果が決まっている**(`scoring.ts` の strict `>` タイブレーク)。しかも **2タイプ(academic-artist / clever-guardian)は、そのタイプを最大化する回答=「本人」が来ても永久に届かない**(4^12=16,777,216 通りの全数検査で確定・cycle-284)。診断が、自分が名乗っている操作を実行していない=constitution **Rule 4 違反**。

**是正の順序を、cycle-294 の教訓に従って正す。** 性格診断は「先に設問があり、点数最大のタイプに当て、ぶつかったら適当に決める」ものではない。**先に診断結果(24タイプ=6アーキタイプの主軸×副軸)が在り、そこへ正しく判別するために設問を設計する**(cycle-294/incident-2.md line 100・オーナー指摘)。「タイブレーク」という語が判定の中心に来た時点で、後付け割り振りの壊れた枠に入っている——36% も同点が出ること自体が設問設計の不全の証拠。したがって cycle-284 §5 の「**既存12問を固定して選択肢の配点だけ振り直す**」案は、同じ壊れた枠(設問が先・割り振りは後)なので**本サイクルの解として採らない**(同型6タイプの潰れ・残る同点6.64% が配点調整で消えなかったのは、設問が結果から設計されていないため)。

本サイクルは **6アーキタイプで採点し主軸×副軸でタイプを決める判定**を実装し、**各アーキタイプが均等に主軸/副軸になれるよう設問を結果先行で再設計**して、来訪者に「自分の回答が決めた結果」を返す。結果テキスト品質(水増し・テンプレート・24タイプ一覧=T2/T3)は **B-603** の仕事で本サイクル対象外(判定が正しくなる前に本文だけ鋭くすると、他人の型を受け取った人に鮮明に外れた肖像を読ませ悪化する=cycle-284 の T1→T2 順序)。

## 作業計画

### 目的

12問に答えてくれた人に、**その人の回答が決めた、その人のための結果**を返す。現状は3人に1人以上がファイルの並び順で決められた結果を受け取り、2タイプは本人にすら届かない。判定は性格診断の「中身」そのものであり、Rule 4(every aspect で最高品質)の最も荷重のかかる aspect。

### 作業内容

**結果先行で診断を組み直す。** 24タイプ(6アーキタイプの主軸×副軸)を所与とし、そこへ正しく判別できるよう設問を設計する。cycle-284 は全数検査で欠陥(同点36.20%・配列順決着・2タイプ到達不能)を確定し、6アーキタイプ方式(§3)が正しい判定モデルだと示した。だが §5 の重み表は既存設問を固定した配点調整に留まり、同型6タイプの潰れ・残る同点6.64% を消せなかった——**設問が結果から設計されていないから**。本サイクルは設問そのものを結果先行で再設計してこの根を断つ。判定は機械であり性質は全数検査で確定できる——**ただし cycle-284 自身が「自作ハーネスに主題そのもののバグ(配列順タイブレーク)を仕込んだまま合格設計を棄却した」ため、過去の数値は鵜呑みにせず、本番経路を呼ぶハーネスで再検証し、ハーネス自体を既知のゴミ設計で検定する**(D1)。

### 検討した他の選択肢と判断理由

- **cycle-284 §5(既存設問を固定して配点だけ振り直す)をそのまま出荷**: **不採用**。これは「設問が先・割り振りは後付け」の壊れた枠で、cycle-294 の教訓(タイプが先・設問が判別する)に反する。実際 §5 は同型6タイプの潰れ・残る同点を消せなかった。ただし cycle-284 の分析(6アーキタイプが正しいモデル・受け入れ基準・ハーネスの罠)は土台として活かす。
- **判定はそのままで結果テキストだけ改善(B-603 相当)**: 不採用。判定が壊れたまま本文を鋭くすると悪化する(cycle-284 の T1→T2 順序)。判定が先。
- **同点を「混合型」として提示(T1-b)**: 不採用(本サイクル)。24タイプ本文は単一タイプ前提で書かれ、混合提示は本文全再設計(T2/T3)を前提にする。設問を結果先行で設計すれば同点は構造的に稀になるため、混合提示に頼る必要がない。
- **タイプ名の書き換え(D 案)**: 不採用。タイプ名は資産(一覧・相性が title のみ描画=全面共通資産)。壊すリスクが利得を上回る。
- **他の backlog 項目(B-573 a11y全面展開・B-594 daily h1 等)**: 見送り。玄関の Rule 4 違反(全PVの31%・診断が診断していない)より来訪者価値の大きい着手対象は現状ない。

## 補足事項

- **計画時に壊れた枠を一度踏みかけ、オーナー指摘で是正した(near-miss の開示)**: 初版計画は cycle-284 §5(既存設問を固定した配点調整)を土台にし、タイブレーク規則を中心タスクに据えていた——これは cycle-294/incident-2.md line 100 が名指しした「後付け割り振りの壊れた枠」の再演だった。オーナーの指摘を受け、記録(recall・incident-2.md)で教訓を確認し、結果先行の設問再設計へ計画を全面改訂した。新 AP は追加しない(既存の cycle-284 規律#1・AP-P09/P12・incident-2 line 100 で覆われている)。守れなかったのは枠であってルールではない。
