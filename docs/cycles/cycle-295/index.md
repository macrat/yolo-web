# サイクル-295

サイトの玄関=最高PVページ `/play/character-personality`(全PVの31%)は「あなたの回答があなたのキャラを決めた」と手続きを名乗るのに、実際には **36〜46% の回答でトップが同点になり、`quiz.results` 配列の並び順で結果が決まっている**(`scoring.ts` の strict `>` タイブレーク)。しかも **2タイプ(academic-artist / clever-guardian)は、そのタイプを最大化する回答=「本人」が来ても永久に届かない**(4^12=16,777,216 通りの全数検査で確定・cycle-284)。診断が、自分が名乗っている操作を実行していない=constitution **Rule 4 違反**。

**是正の順序を、cycle-294 の教訓に従って正す。** 性格診断は「先に設問があり、点数最大のタイプに当て、ぶつかったら適当に決める」ものではない。**先に診断結果(24タイプ=6アーキタイプの主軸×副軸)が在り、そこへ正しく判別するために設問を設計する**(cycle-294/incident-2.md line 100・オーナー指摘)。「タイブレーク」という語が判定の中心に来た時点で、後付け割り振りの壊れた枠に入っている——36% も同点が出ること自体が設問設計の不全の証拠。したがって cycle-284 §5 の「**既存12問を固定して選択肢の配点だけ振り直す**」案は、同じ壊れた枠なので**本サイクルの解として採らない**(同型6タイプの潰れ・残る同点6.64% が配点調整で消えなかったのは、設問が結果から設計されていないため)。

本サイクルは **6アーキタイプで採点し主軸×副軸でタイプを決める判定**を(science-thinking と同様の専用判定経路として)実装し、**各タイプが現実的に到達可能になるよう設問を結果先行で再設計**して、来訪者に「自分の回答が決めた結果」を返す。結果テキスト品質(水増し・テンプレート・24タイプ一覧=T2/T3)は **B-603** の仕事で本サイクル対象外(判定が正しくなる前に本文だけ鋭くすると悪化する=cycle-284 の T1→T2 順序)。

## 受け入れ基準(到達性 G1-G4・恣意決着率 G5)(**設計に着手する前に確定・自分の数値を見て動かさない**)

cycle-294 が「作業を終わらせられる方向へ基準を動かす(AP-P02)」で事故を起こした領域なので、合否の物差しを設計前に固定する。以下は verification.md でも正典とする。

- **G1(ハード・測度非依存)**: ∀理想回答者テスト(research §4.1 の∀分割モデル)が **24/24**。現行は 2/24 で失敗=自明でないゲート。
- **G2(ハード・測度非依存)**: 全数検査(4^12 悉皆)で**到達不能タイプ 0**。
- **G3(ハード・決着機構)**: 残る同点の決着が**配列順に依存しない**(受け入れ基準 §4.2.3)。**残余の決着は意味のある順で尽くす**——(1)逆順フォールバック(順序対が無ければ逆順で同じタイプに落ちる同点は実は決着済)→(2)副軸・主副gap 等の**回答由来の二次規準**→(3)それでも残る分のみ決定的な最終規準。G3 はこの最終規準が配列順でないことを課す**最低条件**であって目標ではない(前例 science-thinking はアルファベット順=決定的だが恣意的。恣意決着はできる限り (1)(2) で消す)。恣意決着の"割合"は G5 で縛る。
- **G5(出荷ゲート・本件の芯=回答でなく機械が決める割合)**: 是正すべき Rule4 実害の本体は「36〜46% のトップ同点が回答でなく `results` の並び順で決まる」こと。**この割合そのものにゲートを置く**。**定義(符号化非依存・抜け道を塞ぐ)**=**恣意決着率**:ある回答組合せについて、**回答由来の規準(6アーキタイプ得点・主軸/副軸・主副gap)を尽くしてもなお最終タイプ候補が2つ以上残る(=回答が結果を一意に決めていない)回答の割合**。※「決着に使う順序を置換して結果が変わる割合」という機構依存の定義は採らない——恣意決着を定数写像で実装すると順序不変で0とカウントされ実害を見逃すため(定数写像で隠れた恣意も候補集合サイズ≥2として捕捉する)。加えて D1/D2 で「最終の恣意決着がすべて単一の置換可能順序として実装され、候補集合サイズ≥2 の回答と一致すること」を検定。**測定は G4 と同じ c∈{1.0, 0.85, 一様} のグリッドで行い、最悪(最大)恣意率でゲート ≤5%**(G4 と非対称にして数字の小さい測度を選ばない=AP-P02 回避。現行は一様36%→一貫度80%で46%と c で増える)。5% は導出値でなく**事前固定した目標水準(緩めない運用で担保・超過なら出荷しない)**——逆順フォールバックが約9pt を無償一意化(research §3)するため小さくできる見込みはあるが、principled な導出ではない。
- **測度Mの事前固定(重要・数値だけでなく物差しごと固定する)**: G4/G5 の合否を決める回答者モデル M を、**設計の数値を見る前に、関数形・パラメータ・対称性・signalベクトルの構成まで確定**する(数値だけ固定してモデルを後で作ると、一貫度や埋め込みを都合よく選んで同型タイプを持ち上げられ=AP-P02 の再演)。**M の定義(確定)**:
  - アーキタイプ親和ベクトル a∈R^6 を各成分 iid Uniform(0,1) で引く(**6アーキタイプ対称・タイプ事前分布を持たない=同型/異型の混合比という調整ノブが存在しない**)。
  - **signalベクトルの構成(固定・別ノブを作らない)**: 各選択肢の signal は **`determineCharacterPersonalityResult` が実際に集計するその選択肢のアーキタイプ配点ベクトルそのもの**(生の配点・正規化しない・主signalのみ抽出もしない)。これにより M は本番採点に束縛され、別途の埋め込みを事後に選ぶ自由度を持たない。
  - **総配点の均一制約(Major 対応・「配点総量」ノブを消す)**: a が全成分正(Uniform(0,1))のため、生配点の内積は総配点(L1)の大きい選択肢を機械的に優遇する。これを設計で悪用(苦戦する同型タイプの選択肢に副signalを足し識別力ゼロのまま到達率を上げる)できないよう、**全選択肢の総配点を同一に固定**する制約を設計側(A2/B2)に課す(現行データも全48選択肢が総量6で均一=既存不変量)。これにより M は方向のみで選ぶ健全な挙動になる。
  - 回答者は各設問で確率 c で「選択肢の signal・a の内積」を最大化する選択肢を選び(選択肢同点は一様)、確率 1−c で一様ランダムに選ぶ。**評価点は c∈{1.0, 0.85} と一様(c=0)に固定**。
  - **単調性の前提(明記)**: 同型タイプ到達率は c に単調増(一貫度が上がるほど一つのアーキタイプが突出しやすい)と仮定し、binding は c=0.85(同型≥1%)・c=1.0(異型の圧縮)・c=0(フロア)・恣意率は最悪 c。D3 で単調性が崩れる兆候があれば c をグリッドで掃いて中間の凹みを確認する。
- **G4(出荷ゲート・現実的到達性・M上で判定)**:
  - **G4a**: c=1.0 と c=0.85 の**両方で、24タイプすべてが ≥1.0% で到達**(根拠=一様理想なら各 4.17%・1%未満は完走100人に1人未満しか受け取らない死蔵コンテンツ)。
  - **G4b**: c=0.85 で**最大/最小タイプ比 ≤10×**(現行=一様20倍・§5案=68〜93倍・いずれも未達)。
  - **G4c(一様フロアのゲート)**: 一様(c=0)でも**全24タイプ ≥0.2%**(=どのタイプも最悪の測度でも死蔵にしない・報告でなくゲート)。
  - **G4/G5 未達なら出荷しない**——反復するか、達成不能と判明したら理由を明記して繰越す(基準を緩めて緑にしない・退行は出さない)。
- **注意(プロキシの罠)**: 「各アーキタイプが主軸になれる機会の均等化」は到達性の**プロキシに過ぎず G4 を保証しない**。type-slot は非対称(commander 主軸5タイプ・artist 主軸2タイプ等)で、機会を均等化しても多タイプ側は各タイプが薄くなる。設計の目的関数は**タイプ到達性(G4)+恣意決着率(G5)であって機会均等ではない**。

## 作業計画

### 目的

12問に答えてくれた人に、**その人の回答が決めた、その人のための結果**を返す。現状は3人に1人以上がファイルの並び順で決められた結果を受け取り、2タイプは本人にすら届かない。判定は性格診断の「中身」そのものであり、Rule 4(every aspect で最高品質)の最も荷重のかかる aspect。

### 作業内容

**結果先行で診断を組み直す。** 24タイプ(6アーキタイプの主軸×副軸)を所与とし、そこへ正しく判別できるよう設問を設計する。cycle-284 は全数検査で欠陥(同点36.20%・配列順決着・2タイプ到達不能)を確定し、6アーキタイプ方式(§3)が正しい判定モデルだと示した。だが §5 の重み表は既存設問を固定した配点調整に留まり、同型6タイプの潰れ・残る同点6.64% を消せなかった——**設問が結果から設計されていないから**。本サイクルは設問そのものを結果先行で再設計してこの根を断つ。判定は機械であり性質は全数検査で確定できる——**ただし cycle-284 自身が「自作ハーネスに主題そのもののバグ(配列順タイブレーク)を仕込んだまま合格設計を棄却した」ため、過去の数値は鵜呑みにせず、本番経路を呼ぶハーネスで再検証し、ハーネス自体を既知のゴミ設計で検定する**(D1)。合否は着手前に固定した G1-G5(上記)で判定し、未達なら出荷しない。

### 検討した他の選択肢と判断理由

- **cycle-284 §5(既存設問を固定して配点だけ振り直す)をそのまま出荷**: **不採用**。「設問が先・割り振りは後付け」の壊れた枠で cycle-294 の教訓に反する。実際 §5 は同型6タイプの潰れ・残る同点を消せなかった。ただし cycle-284 の分析(6アーキタイプが正しいモデル・受け入れ基準・ハーネスの罠)は土台として活かす。
