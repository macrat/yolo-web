# サイクル-276

**このサイクルは、8サイクル（cycle-268〜275）逃げ続けた本丸 B-545 に決着の方向を与える正念場である。** オーナー通告：10連続失敗（cycle-277 まで）でオーナー直接介入。それまでに B-545 を完了させること。

過去8サイクルに共通したただ一つの構造は「決定(a)＝根幹をどのコンセプトに統一するかから逃げ、楽な下位問題（A/B・デザイン移行・計測・シェア・演出）をやり、プロジェクト純正の語彙で覆う」ことだった（cycle-275 自己点検）。このサイクルはその逃避を繰り返さない。**決定(a)を開いた問いのままにせず、Goal・データ・一次史料から論理で下し、決定として確定する。そのうえで根幹の統一を実際に着手する。**

## 決定(a)：サイトの根幹を「自分を知り、楽しむ場所（診断中心）」へ一本化する

**これは開いた問いではない。下した決定である。** 根拠は以下の Goal 直結の論理・実測データ・一次史料であり、いずれもこのサイクルで一次確認した（下記「計画にあたって参考にした情報」）。

### 何に統一するか

サイトのコンセプトを **「自分を知り、楽しむ。そのための場所」**（＝診断・占い・ちょっとしたゲームを中心とした「自分を発見する体験」）に一本化する。文化コンテンツ（辞典・伝統色）はそれを支える層、オンライン道具（道具箱）は**実用層として残すが、サイト構造・自己定義・デザインシステムの核にはしない**。これで「占いと診断の遊園地／日常の傍にある道具／診断とゲームのサイト」に裂けた 3〜4 コンセプト（オーナー指摘）を一つに畳む。

### なぜか（Goal 直結）

- **Goal**（constitution）＝来訪者に最高の価値を提供して PV を最大化する。CLAUDE.md 決定原則＝工数を理由に劣った UX を選ばない。
- **実測データ（cycle-257・頑健・一度も撤回されていない）**：`/play`（診断・ゲーム）が organic 着地の最大かつ中央エンゲージが他セクションの 15〜20 倍＝**集客・滞在の両軸で 1 位**。トップ `/`（道具箱）は全期間 organic 集客ほぼ 0、道具箱の中核機能（並べ替え・プリセット・カスタマイズ）の利用は観測ゼロ。この判定は複数データ源・敵対的再検算・reviewer 2 名承認で確定し、以後 18 サイクル一度も逆戻りしていない。cycle-275 の GA 接地（診断が全 PV の 58%・character-personality 単独 34%・完走者は 138 秒読み込む）も同じ方向。
- **一次史料**：道具箱-as-core（cycle-167）は**来訪者価値データに基づかない**オーナーの戦略仮説（「AI が安定して作れる」「差別化」）だった。cycle-257 がそれを実測で覆した。だが cycle-257/259 は**意図的にコード不変・文書レイヤーのみ**で留め、根幹（トップ `/`・アーキテクチャ・デザインシステム本体）は道具箱前提のまま残した。これが B-545 膠着の構造的原因＝「文書は診断中心・根幹は道具箱」の宙吊り。

論理は一直線だ。データが価値の中心と示す診断を、根幹（アーキテクチャ＋デザインシステム）が道具箱前提のまま抑え込み、診断は §7 例外とドキュメント注記で継ぎ足されている。**Goal を最大化するには、根幹を価値の中心＝診断に合わせて据え直すのが唯一筋の通った選択である。** 道具箱を核に固定し続けるのは、データが否定した仮説（cycle-167）を根幹に温存することであり、Goal に背く。

### どう統一するか（根幹の作り直しの方向・撤去候補を含む）

道具箱-as-core を「根幹の前提」から降ろす。具体的に、以下を**所与にせず俎上に載せ**、診断中心へ据え直す：

1. **アーキテクチャ根幹**：`docs/design-migration-plan.md` は移行の軸を「コア機能『道具箱』のタイル化（Phase 2・7・8・10）」に置いている＝アーキテクチャが道具箱前提。この**軸を撤回**し、移行の目的を「デザイン体系の統一とコード健全化（概念非依存で有効）＋診断体験を構造の中心に据える」へ据え直す。タイル/道具箱の機構は**実用層（道具）に閉じた関心事**として残し、サイトの構造的背骨にはしない。
2. **デザインシステム根幹**：DESIGN.md は §1〜6 の austere（極限ミニマル＝道具箱抑制のために作られた基調）を土台とし、§7「診断の視覚言語」を**診断タッチポイント限定の例外**として重ねる構造。これが「診断を継ぎ足す」構造そのもの。**方向：austere を「サイトの基調 identity」から降ろす。** 抑制（けばけばしさ・煽り・ダークパターンを避ける品質バー）は保つが、サイトの基調表現は「自分を発見する体験」に資するもの（結果の identity・色・到達感）へ据え直し、§7 を例外でなく基調の一部へ引き上げる。§7 と cycle-275 の勲章は**撤去/作り直し候補**として扱い、「残っているから正しい」（現状追認 AP-P30）にしない。
3. **正面玄関の撤去（第一の構造ステップ・本サイクルで出荷）**：トップ `/` は現在=道具箱ダッシュボード（cycle-232）、かつサイトの metadata / WebSite JSON-LD が「無料のオンラインツールを集めたサイト」と自己定義している＝道具箱-as-core の正面玄関そのもの。これを撤去し、`/` を診断中心の着地面へ、サイト自己定義を「自分を知り、楽しむ場所」へ据え直す。道具箱は `/toolbox` へ降ろす（実用層として存続）。
