# サイクル-296

## 背景と芯

玄関 `/play/character-personality`(全PVの31%)は、cycle-295(B-589)で判定を結果先行に作り直し、**回答が結果を決める**ようになった。判定は本物になった。だが判定が正しくても、その先で読者が読む**結果テキスト**が薄ければ、判定を直した価値はそこで漏れる。

cycle-284 が全24タイプを人の目で読了して確定した欠陥(T2):

- **欠陥3(テンプレート)**: 主要4ブロックのうち2つが「あなた」でなく「型」を語る。
  - `archetypeBreakdown` **24/24** が「あなたの中で(は)その二つが同時に◯◯している」の同一文型(◯◯を類語で回すだけ)。
  - `characterMessage` **24/24** が同一三段構造で、うち **11/24 が逐語同一の助言**(「誰か一人に話して/見せて/教えて/巻き込んでみて」)=24人中11人が同じことを言われている。
- **欠陥2(水増し)**: 1タイプ内で同じ観察を最大5フィールド(description/catchphrase/behaviors/characterMessage)で言い換える(例 #18 eternal-dreamer は description と catchphrase が逐語同一/#3 blazing-schemer は反復クラスタ3本)。

原因は cycle-284 line 221-222 が名指しした構造——「**24体に同じブリーフを渡した結果**」。だから同じ工程(24エージェントに同一brief)で作り直せば同じ収束が再発する。**先に横断台帳で各タイプの固有核を割り当ててから書き直す**のが本サイクルの規律。

芯は「判定に答えてくれた人に、**その人の型にだけ言える言葉**を返す」。T2 核心基準(cycle-284 line 220)=**「この文は他の23タイプの誰かに言えるか。言えるなら捨てる」**。

## 受け入れ基準(**書き直しに着手する前に確定・自分の成果を見て動かさない**=AP-P02回避)

cycle-284/294 はこの領域で「基準を後から緩める」「実物確認を捏造する」事故を起こした。合否の物差しを設計前に固定する。

- **AC1(テンプレート脱却・機械可検)**: `archetypeBreakdown` の共有文型「あなたの中で(は)…二つが同時に◯◯」を **0/24** に。`characterMessage` の逐語共有助言(現 11/24)を **0**(相互非共有)。三段固定構造を崩し温度・長さにムラを残す。
- **AC2(横断非共有=T2核心)**: 各タイプの観察・あるある(behaviors)・助言・キャッチが「**他の23タイプの誰かに言えない**」。共有句・共有観察・共有助言=**0**。半自動(句/n-gramオーバーラップ検出=C2)+ reviewer の意味判定(C1)の二段で担保。
- **AC3(タイプ内非水増し)**: 1タイプ内で description/catchphrase/behaviors/characterMessage が**同じ観察を反復しない**(欠陥2)。逐語重複0・観察の重複0。
- **AC4(声の保存・非平坦化)**: 各タイプの**役割語・キャラ声を保存**する(japanese-ai-slop.md L13/168=架空キャラの声はスコープ外・「だ・である調で統一」を適用しない)。before/after でトーン維持を確認。同時に AI slop(japanese-ai-slop.md の5観点)を落とす——両立させる(役割語は約束事であって slop ではない)。
- **AC5(非退行・資産不変)**: `typeId`/`title`/`color`/`icon`/相性マトリクス(24×24)/24タイプ一覧/`detailedContent.variant`/フィールド構造は**不変**。文字数レンジは枠崩れ防止のため現行レンジ(description 約220-270字・behaviors 各1文)を大きく外さない。lint/format/test/build 全緑・実機で枠内(はみ出し無し)・ダーク可読・コンソールエラー0。

**基準未達なら出荷しない**——差し戻して直すか、達成不能と判明したら理由を明記して繰越す(緑にするために基準を緩めない)。

## 作業計画

### 目的

12問に答えてくれた人へ、**その人の型にだけ言える言葉**を返す。現状は主要4ブロックのうち2つ(archetypeBreakdown・characterMessage)が24人全員に言える型の文で、24人中11人が同じ助言を受け取る。判定(B-589)が正しい型を返すようになった今、その型の肖像が「他人にも言える」なら、判定を直した価値がここで漏れる。

### 作業内容

**結果先行で固有核を割り当ててから書き直す。** 先に横断台帳(A1)で「どの観察・助言をどのタイプが持つか」を非衝突に配分し、各タイプはその核を軸に本文を組む。T2 核心基準「他の23に言えるなら捨てる」を横断突合(C1)+機械検出(C2)の二段で担保。役割語は保存し、slop の5観点は落とす。判定・相性・一覧・タイプ名は資産として触らない。

### 検討した他の選択肢と判断理由

- **一部タイプだけ書き直す(代表スライス)**: 不採用。欠陥は 24/24 の系統的収束で、一部だけ鋭くすると「鋭い型」と「テンプレの型」が同一ページに混在=Rule4「every aspect で最高品質」に反する。本文は型ごとに個別で「系統的に一括適用」できない=24タイプ全部が作業単位。
- **24エージェントに同一briefで一斉発注**: 不採用。cycle-284 line 221-222 が名指しした収束の原因そのもの。先に台帳で固有核を配分する工程を挟む。
- **欠陥4(24タイプ一覧の処遇=T3)も同時に**: 本サイクル**スコープ外**(キャリーオーバー)。T3 はレイアウト/IA の判断(一覧の撤去/畳み込み/移設)で before/after 撮影と独立の価値判断を要し、prose の T2 と混ぜるとサイクルが肥大化。本文品質(T2)を先に完了させる。
- **判定はそのまま本文だけ改善を cycle-295 前にやる**: 既に不採用済(cycle-284 T1→T2 順序)。判定は B-589 で出荷済のため本サイクルで T2 に着手できる。
- **役割語を平坦化して統一トーンに**: 不採用。japanese-ai-slop.md L13/168=架空キャラの声はスコープ外・役割語はフィクションの約束事。24キャラの声を平坦化するのは破壊。

### 影響範囲(不変であることの明示)

- **判定ロジック(`determineCharacterPersonalityResult`)・設問・配点・相性マトリクス(24×24)・24タイプ一覧・typeId/title/color/icon・SEO/URL は不変**。変更対象は各タイプの `description`/`catchphrase`/`detailedContent`(archetypeBreakdown/behaviors/characterMessage)の**文面のみ**。
- **他11 personality 診断は不変**。
