## C2. 半自動オーバーラップ検出(機械)

`tmp/c2-overlap.py` で24タイプの結果本文(description/catchphrase/archetypeBreakdown/behaviors/characterMessage の日本語文字列)を横断し、型をまたぐ**8文字 n-gram**(区切り除去)を抽出。検出型ブロック=24(全数)。**ヒットは全件を人の目で読んだ**(cycle-284 L101=「スクリプト出力は実物でない」)。

### 判定: ヒットの3分類

1. **役割語(AC4で保存対象=非該当)**: 最多数のヒットは各グループの役割語マーカー。例=お嬢様声「わたくしですわ」(dreamer4)・策士声「いられないっしょ/こっち側の人間っしょ」(clever-guardian/ultimate-trickster)・博士声「のであります」(careful-scholar/dreaming-scholar)。これらは約束事であり slop でない(AC4)。**是正不要**。

2. **結果挨拶の定型(元テキスト由来・許容)**: 「あなたの守護キャラは◯◯」の開幕挨拶が複数型で共有。**git HEAD~3(B着手前)で同一に存在**する既存の結果挨拶装置(=その型の声で「あなたの守護キャラはこれ」と名乗る)で、builder の新規テンプレではない。むしろ現行は開幕を多様化(「よっしゃ来たな」「ふむ実に興味深い」「グラフの曲線を見て」等・挨拶なしの型も混在)。観察ではなく構造的挨拶なので **AC2 の「他人にも言える観察」欠陥に非該当**。是正不要。

3. **内容重複の要判定候補(→ C1 の意味判定へ回付)**:
   - **「言葉にしようとすると…」= #12 dreaming-canvas / #21 ultimate-artist / #23 vibe-rebel**。共有は動詞句「言葉にしようとすると」だが対象・帰結は分化(#21=美の「違う」が詰まる→見せる方が早い / #23=走った熱が冷める / #12=脳内映像がうまく言えずもどかしい)。台帳 §4-0 の「言語化できない2(#12/#21)」の棲み分け意図どおりだが、**#12 と #21 は表層(言葉にできない→もどかしい/見せる)が近い**ため C1 で「他方に言えるか」を意味判定。
   - **助言の締め「〜してみてほしい(のですわ/のであります)」= careful-scholar / dreaming-canvas / eternal-dreamer 等**。三段構造の締め動詞の残り香。助言内容は分化しているが締めの型が近い候補として C1 で確認。
   - その他ペア(例 blazing-canvas×blazing-poet「勢いがそのまま感性」・creative-disruptor×vibe-rebel「なんでこっちにした」)は短句・場面差で分化しているが C1 の目視対象に含める。

→ **結論(C2)**: 機械検出は逐語/n-gram レベルで**新規のテンプレ収束を導入していない**ことを確認(3テンプレ族の逐語共有は検出されず=AC1 の機械可検部分を裏取り)。言い換え共有の可能性がある候補3件を C1 へ回付。C2 は言い換えを拾えないため C1 の意味判定を代替しない。
