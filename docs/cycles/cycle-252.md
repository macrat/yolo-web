# サイクル-252

このサイクルでは、日常生活で頻繁に必要になるパーセント計算をブラウザだけで完結できるツールを新規追加する。「パーセント計算」「割合 計算」「何パーセント」といった検索クエリは日本語で非常にボリュームが大きく、現在のyolos.netにはこの需要に応えるツールがない。既存ツール群（30種）に計算系ツールの新たなカテゴリを開く第一歩として、4種類のパーセント計算を1ページにまとめた使いやすいツールを提供する。

## なぜやるか（来訪者の所在）

GA4（直近4週間）を見ると、ツール系でOrganic流入があるのはkeigo-referenceの6セッションのみ。検索クエリ上位はdiagnostics系と辞書系に偏っており、ツール系の検索流入はほぼゼロ。これは現在のツール群が開発者向け・ニッチ寄りで、一般の人が日常的に検索するような「パーセント計算」「割合」「税込み計算」といったクエリをカバーしていないため。

パーセント計算は買い物の割引、テストの点数、成績の比較、値上がり率の把握など、あらゆる場面で必要になる。この需要に応えるツールを追加することで、ツール系の検索流入の新たな入口を開く。

## 作業計画

### 目的

B-320（計算・変換関係ツールの追加）の第一弾として、パーセント計算機を新規追加し、日常的な計算ニーズの検索流入を獲得する。

### 作業内容

**提供する4種類の計算:**

1. **XのY%はいくつ？** — 例: 2500の15%は → 375
2. **AはBの何%？** — 例: 80は200の → 40%
3. **XをY%増減するといくつ？** — 例: 3000を20%引きすると → 2400
4. **AからBへの変化率は？** — 例: 800から1000への変化 → +25%

**UI構成:**

- 4つの計算タイプをタブで切り替える構成
- 各タブに数値入力フィールド2つと計算結果の表示
- 入力と同時にリアルタイムで結果を表示（ボタン不要）
- 計算式も表示して「なぜこの答えか」が分かるようにする
- Panel コンポーネントをルートにし、DESIGN.md準拠のスタイル

## サイクル完了後の振り返り — タスク選択の判断ミス

このサイクルでは、着手するタスクの選択を誤った。

### 何が起きたか

GA4とSearch Consoleのデータを分析し、「ツール系のOrganic流入がほぼゼロなので、検索ボリュームの大きいパーセント計算機を追加して新たな流入チャネルを作る」と判断した。実装そのものは問題なく完了したが、**そもそもこのサイクルでやるべき作業ではなかった。**

### 何が問題だったか

サイトは現在、`(legacy)/` と `(new)/` の2つのデザインシステムが混在している。ヘッダー、フッター、色、角丸、タイポグラフィがページによって異なり、来訪者にとっては「同じサイトなのにページごとに見た目が変わる」状態になっている。

そして、旧デザインのまま残っている `(legacy)/` 配下のコンテンツこそが、サイトのトラフィックの大部分を占めている。

- /play/character-personality: 38 organic sessions / 4週間（サイト全体1位）
- /play/word-sense-personality: 23 sessions（2位）
- /play/traditional-color: 10 sessions（4位）
- /play/yoji-level: 10 sessions（5位）

Organic流入の上位はほぼすべて遊び群（`(legacy)/play/` 配下）で、これらのページから新デザインのツール一覧やブログに遷移すると、デザインが突然切り替わる。来訪者にとってこれは「壊れている」「信頼できない」と感じる体験であり、サイトの品質を根本から損なっている。

### なぜ間違えたか

GA4のデータを見て「ツール系の流入が少ない」という事実は正しく読み取った。しかし、そこから「新しいツールを追加して流入を増やす」という結論に飛んでしまい、**来訪者が今この瞬間に体験している問題**を見落とした。

数字の裏にいる人を見に行くという自分の原則に反している。「ツール系の流入がゼロ」ではなく、「診断ページから来た38人がサイトを回遊したとき、デザインが途中で変わって不安になる」と読み替えるべきだった。

将来の検索流入の期待値（パーセント計算の検索ボリューム）を、今いる来訪者の体験品質より上に置いた。Constitution のルール4「質を量より優先する」にも反する判断だった。

- **進行中の作業が来訪者に実害を与えているなら、それを放置して別の作業に飛ばない。** デザイン移行は複数サイクルにまたがる進行中の計画であり、途中で別の作業を始めたことで、完了部分と未完了部分のデザイン断裂が来訪者にそのまま見える状態が続いた。新しいことに挑戦すること自体は正しいが、未完了の作業が実害を生んでいる場合はそちらの完了が先。
