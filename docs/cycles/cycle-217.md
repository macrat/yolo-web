# サイクル-217

このサイクルでは、デザイン移行計画 Phase 8.1（ツール群の新デザイン移行 + タイル化）の第 18 弾として、`traditional-color-palette`（伝統色カラーパレット）ツールを対象に、(1) ロジック CSS Module の新トークン置換、(2) 詳細ページ（`/tools/traditional-color-palette`）の `(new)/` 配下への移行とデザイン適用、(3) 道具箱で単独完結するタイルウィジェットの実装を行う。

### 検討した他の選択肢と判断理由

AP-P17 に従い、設計判断が分かれる 3 論点（A/B/C）について 3 案以上をゼロベースで列挙・比較し、M1a の最短 UX を基準に推奨案と根拠を示す。あわせて先行 SSoT の引用採否（論点 D）も整理する。

#### 論点 A: タイルサイズ（recommendedSize）

| 案                                                                             | 来訪者価値                                                                                                                                                                              | 400px 系収まり                                                                                                                                 | 採否       |
| ------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | ---------- |
| rows=2（400×264 / **推定値** = tile-grid 定数 calcTilePixels より要 T-3 確定） | 縦が短く一覧性は高いが、操作側（スウォッチ + 配色タイプ + コピー行）と膨張側（5 配色カード群）の二分を 264px に詰めると配色カードが潰れる。                                             | 窮屈 / 配色カードの可読性低下リスク。                                                                                                          | 不採用     |
| **rows=3（400×400）**                                                          | 操作側 + 配色カード群を二分しても各配色（5 種）と HEX/RGB/HSL が読める縦余白を確保できる。research 指針 2 の高さ配分（色選択 140 / 配色タイプ 36 / 結果 180 / 余白 44 ≒ 400px）に整合。 | 収まる見込み。                                                                                                                                 | **第一案** |
| rows=4（400×528 等）                                                           | 余白に最も余裕。                                                                                                                                                                        | 縦に長すぎてトップのタイルグリッドで突出し、M1a の「開いた瞬間に全部見える」を損なう。スクロールなしで全配色を見せる要件には rows=3 で足りる。 | 不採用     |

**判断**: 複雑度（操作側と 5 種配色カード群の二分）から **rows=3（400×400）を第一案**とする。research §4 指針 2 の高さ配分が約 400px に収まることが根拠。サイズ系は先行 SSoT が薄いため**推定値**として hedge し、実機の収まりを T-3/T-4 で検証する（AP-P20 = 計画段階で確定しない）。

#### 論点 B: タイル内の基準色選択方式

| 案                                                                          | 来訪者価値（M1a 最短 UX）                                                                                                                                                                                                       | 400px 収まり                                                                                                             | 採否       |
| --------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ---------- |
| 全 250 グリッド + scroll                                                    | 全色から選べるが、各セル 16px 角以下で WCAG タップターゲット未達（research §3-1）。タイルに 250 件詰め込みは最短動線に反する。                                                                                                  | 不可（タップ不能）。                                                                                                     | 不採用     |
| **カテゴリ絞り込み + スクロールスウォッチ ＋ おまかせ再生成（「別の色」）** | おまかせ初期表示で即価値 → 気に入らなければ「別の色」で再生成、または色相カテゴリで 30〜40 色に絞ってスクロール選択。research §4 指針 1（おまかせ + 色相カテゴリの 2 段構え）に整合。250 全件閲覧・名前検索は詳細ページに残す。 | 収まる（カテゴリで絞り横スクロール）。                                                                                   | **第一案** |
| おまかせ + 「別の色」再生成のみ（カテゴリなし）                             | 最小実装。                                                                                                                                                                                                                      | 収まるが、特定色相を狙う M1a（「青系の和色が欲しい」）に応えられず探索が運任せになる。                                   | 次点       |
| 名前検索優先                                                                | ピンポイント選択は可能。                                                                                                                                                                                                        | 伝統色名は一般知識でなく初回来訪者が色名を知らないと機能しない（research §3-1 C の注意点）。タイルでは補助に留めるべき。 | 不採用     |

**判断**: **「おまかせ初期表示 + 色相カテゴリ絞り込み + スクロールスウォッチ（加えて『別の色』再生成）」を第一案**とする。おまかせで空っぽ画面を避けて即価値（M1a needs 直撃）を提供しつつ、特定色相を狙う需要にはカテゴリ絞り込みで応える。research §4 指針 1 の 2 段構えが根拠。名前検索・全 250 グリッドはタイルから外し詳細ページに残す（最短動線優先 / AP-P07 = 認知モデルで決める）。

#### 論点 C: 配色 5 タイプのタイル内提示

| 案                                      | 来訪者価値                                                                                                                   | 400px 収まり                                                                                                                     | 採否       |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- | ---------- |
| **5 タブ常時表示（横タブ + 一行補足）** | 5 種を一目で見渡せ、配色理論を知らない M1a も補足文で選べる。Adobe Color/Paletton 型（research §2-3/2-4）。切替が 1 タップ。 | 横 5 タブは短縮ラベル（補色/類似/トライアド/テトラド/分裂補色）なら 400px 幅に収まる見込み。補足はサブテキスト or ツールチップ。 | **第一案** |
| select ドロップダウン                   | 省スペース。                                                                                                                 | 収まるが、5 種が畳まれて見渡せず「どれを選ぶか」の比較が 1 タップで不能。M1a の即時性を損なう。                                  | 次点       |
| 補色固定で省略                          | 最小実装。                                                                                                                   | 5 種生成というツールの核心価値を捨てるため不採用。詳細ページとの差別化も失われる。                                               | 不採用     |

**判断**: **「5 タブ常時表示 + 一行の平易な補足」を第一案**とする。5 種配色の一括提示がこのツールの核心価値であり、タブ常時表示なら 1 タップ切替かつ見渡しが可能（research §3-4 の専門用語一般化 + §4 指針 4 の補足表記に整合）。ラベルの短縮形 / 補足をサブテキストにするかツールチップにするかは T-3 実機の面積を見て確定する（AP-P20）。select は見渡し性で劣り次点、補色固定は核心価値喪失で不採用。

#### 論点 D: 先行 SSoT の引用採否（AP-WF14 / T-1 で並べ読み確定）

- **cycle-213 SSoT η（マウント時自動生成型）**: おまかせ初期表示の実装根拠として**引用適用**（useState 遅延初期化 / クライアントのみ生成 / hydration 安全）。本ツールのおまかせ初期表示は password-generator（cycle-213）と同型のマウント時自動生成。
- **cycle-216 c216-γ（操作側 40px 下限）/ cycle-210 (i)**: **引用適用**（操作側スウォッチ・タブ・コピー行の 40px 下限判定）。
- **cycle-211 (x)（コピーボタン文言変化 AP-P21 適用外）**: **引用適用（適用外として記録）**。
- **cycle-216 c216-δ（(v) 変化率 N/A / flex 固定設計起因）**: **引用適用の見立て**（おまかせ初期表示で空状態がなく同軸ではない参考値になる公算 / T-4 で確定）。
- **複合入力型 (v) ±15%（cycle-210）/ 単一 textarea ±10%（cycle-209）/ B-452 / B-456 / B-433（localStorage）**: **適用対象外**（計測軸が別物 / 別系統）。
