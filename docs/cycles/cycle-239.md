# サイクル-239

辞典データ（kanji / yoji / traditional-colors）のインタラクティブツール化を評価し、最も有望な四字熟語（yoji）で検索ツールを設計・実装する（B-341）。

GA4 SC データで四字熟語関連クエリのインプレッションが700超（冷汗三斗286・至誠通天323・一長一短64 等）にもかかわらず CTR がほぼ0% という状況がある。既存の辞典ページ（静的な参照ページ）は「日常の傍にある道具」というサイトコンセプトとの整合が弱く、来訪者が探している情報に即座にたどり着ける「検索ツール」形式に転換することで、この検索需要を PV に変換できると考える。

## 作業計画

### 目的

既存の yoji-data.json（400語・カテゴリ10種・難易度3段階・AI例文・出典・構造パターン）を素材に、来訪者が四字熟語をテキスト入力・カテゴリ・難易度で即座に検索できるインタラクティブツールを構築する。

来訪者にとっての価値:

- 「冷汗三斗 意味」のように特定の四字熟語を調べたい人が、ページを開いた瞬間に検索して意味・読み・例文を確認できる
- カテゴリや難易度で絞り込んで四字熟語を探索できる（作文・スピーチの素材探し）
- 登録不要・ブラウザ完結・外部送信なしで安心して使える

### 作業内容

#### T-2: 四字熟語検索ツールの設計

**ツール仕様**:

- slug: `yoji-search`
- name: 四字熟語検索
- category: `"text"`（keigo-reference と同じカテゴリ）
- コンセプト: テキスト入力で四字熟語を即座に検索し、意味・読み・例文・カテゴリ・難易度を確認できるリファレンスツール

**UI 設計（variant="full" 詳細ページ）**:

1. **検索入力**（Panel 内、ページ最上部）: テキスト入力欄。四字熟語・読み・意味のいずれでもヒットする。入力と同時にフィルタ（デバウンスなし、400件なら即応可能）
2. **フィルタ**（検索入力の下）: カテゴリ（10種 + 全て）、難易度（3段階 + 全て）のドロップダウンまたはボタングループ。出典（日本・中国・不明）のフィルタも用意
3. **結果一覧**（フィルタの下、同一 Panel 内）: ヒットした四字熟語をカードレス（リスト形式）で表示。各項目は四字熟語（大きめ）・読み・意味の1行サマリ。クリック/タップで展開し詳細（例文・出典・構造）を表示
4. **件数表示**: 「◯件中◯件を表示」のカウンター
5. **空状態**: 検索結果がゼロの場合「条件に合う四字熟語が見つかりません」

**variant="list" タイル（本サイクルではスコープ外）**:

道具箱内のリストバリアントは本サイクルでは実装しない。tile-architecture.md の不変項「タイルはその場で動く」を満たす自己完結形式の設計が必要であり、variant="full" の完成後に別サイクルで検討する。

**データフロー**:

- yoji-data.json を logic.ts 内で static import
- フィルタリングはクライアントサイドで全件メモリ内処理（400件なら問題なし）
- 検索ロジック: テキストは yoji・reading・meaning フィールドに対する部分一致（大文字小文字・ひらがなカタカナ正規化）

#### T-3: 実装

keigo-reference および tile-architecture.md のパターンに従って `src/tools/yoji-search/` を実装する。variant="full"（詳細ページ用）のみ。ルーティングページ（`src/app/(new)/tools/yoji-search/page.tsx`）も作成し、`npm run generate:toolbox-registry` でレジストリ再生成する。

### 検討した他の選択肢と判断理由

1. **評価だけで終わる（実装なし）**: 評価は GA データとデータ品質の確認で完結する。データが揃っている以上、評価だけでサイクルを終えるのは来訪者への価値提供を遅らせるだけ。評価 + 実装を 1 サイクルで完結させる。

2. **kanji を第一候補にする**: kanji-data.json は 2,136 件とデータ量が多いが、既存ゲーム kanji-kanaru との双方向クロスリンク依存があり、ツール化の際の設計が複雑になる。yoji は独立性が高く、検索需要も SC で確認済み。

3. **traditional-colors を第一候補にする**: 既存ツール traditional-color-palette が存在し、SC インプレッションも低い。追加ツール化の需要が見えない。

4. **辞典ページの SEO 改善（meta description 最適化等）で CTR を上げる**: 平均掲載順位が 7-10 位では meta 改善だけで CTR を大幅に上げるのは難しい。コンテンツの質と形式を根本から変えてランキング自体を上げる方が効果的。

5. **新しい ToolCategory「reference」を追加する**: keigo-reference が "text" で運用されており先例がある。カテゴリ追加は UI 影響（フィルタボタン増加）もあり、このサイクルの本題ではない。

### B-376（四字熟語を自然言語で検索する道具）との関係

本サイクルの yoji-search と B-376 はいずれも「四字熟語を検索するツール」だが、検索方式が根本的に異なる:

- **yoji-search（本サイクル）**: テキスト部分一致 + カテゴリ/難易度/出典フィルタ。四字熟語・読み・意味に対する文字列検索。来訪者が特定の四字熟語を調べたり、カテゴリで絞り込んで探索する用途。
- **B-376**: 状況・場面を自然言語で入力すると、意味ベクトル（セマンティック検索）で合う四字熟語を返す。来訪者が「何かいい四字熟語ないかな」と探す用途。

yoji-search は「調べる道具」、B-376 は「見つける道具」。機能が重複しないため、両者は将来的に並存可能。ただし、B-376 は意味ベクトルのインフラが必要であり、yoji-search の完成後に着手判断する。yoji-search のデータ基盤（logic.ts）は B-376 の土台にもなりうる。
