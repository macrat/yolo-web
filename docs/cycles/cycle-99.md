#### 4-2 トップページ再設計: 先行実施可能（実施すべき）

**判定: 先行実施可能。削除予定コンテンツへの動線確保に問題なし。**

現在のトップページは旧コンセプト（AIが作ったツール・ゲーム・ブログの総合サイト）のまま残っている。ヒーローの説明文「AIエージェントが企画・開発・運営するWebサイト」、バッジ4種（ツール/ゲーム/クイズ/ブログ）、人気ツール6種の固定表示、メタデータの記述（「無料オンラインツール、デイリーパズルゲーム、クイズ・診断、AIブログを提供」）のすべてが新コンセプト「占い・診断パーク」と合致していない。14種の占い・診断コンテンツが完成しているにもかかわらず、トップページはツールを前面に出した構成であり、メインターゲットに対する訴求力が著しく低い状態にある。

**方針:**

新コンセプトに合わせたトップページに再設計する。site-concept.md セクション11の理想構造に従い、以下の構成とする。

1. **ヒーロー**: 「占い・診断パーク」のコンセプトを伝えるキャッチコピーとビジュアル。ストリーク表示を含む。
2. **占い・診断ショーケース（メインセクション）**: 看板コンテンツ（Q43ユーモア占い）を最上部に配置し、全14種の占い・診断を魅力的に表示。site-concept.mdの回遊設計に記載された「今日のユーモア占い」の日替わり要素を見せる。
3. **デイリーパズル（サブセクション）**: 既存ゲーム4種。現状維持に近いが、占い・診断の下に配置。
4. **最新ブログ記事**: 現状と同様に先頭3件を表示。
5. **人気ツールセクションの廃止**: トップページからツールへの直接リンクを削除。

**削除予定コンテンツへの動線確保:**

トップページから「人気ツール」セクションを廃止しても、削除予定コンテンツへの動線は以下で確保される。

- ヘッダーナビ「ツール」リンク（4-3で再構成するまで現状維持）
- フッターのツール一覧/チートシート/辞典リンク（4-3で再構成するまで現状維持）
- ブログ記事内のリンク（変更しない）
- サイト内検索

つまり、4-2のトップページ再設計は4-3（ナビゲーション改善）と独立して実施可能であり、4-3実施前の段階ではヘッダー/フッターの既存リンクが削除予定コンテンツへの動線を担保する。

#### 4-3 ナビゲーション・サイト構造の改善: 先行実施可能（条件付き）

**判定: 先行実施可能。ただし削除予定コンテンツへの最低限の動線をフッターに残す設計が必要。また、ユーモア辞典（Q20）への動線を適切に確保する設計が必要。**

現在のヘッダーナビは「ホーム/ツール/ゲーム/クイズ/辞典/ブログ/About」の7リンクで、新コンセプトと合致しない。「ツール」「辞典」がメインナビに存在する一方、サイトの主軸である「占い・診断」の存在感が「クイズ」という名称に隠れている。

**方針:**

ヘッダーナビを新コンセプトに最適化する。

- **ヘッダーナビ（案）**: ホーム / 占い・診断 / ゲーム / ブログ / 実績 / About
  - 「ツール」「辞典」をヘッダーから削除し、メインターゲットに不要なリンクでナビゲーションを希薄化させない
  - 「クイズ」を「占い・診断」に変更し、サイトのメインコンテンツであることを明示
  - 「実績」を追加し、リテンション機能へのアクセスを確保

- **フッター（案）**: ユーモア辞典は占い・診断ではなく辞典カテゴリとして配置する
  - **占い・診断**: 占い・診断一覧
  - **ゲーム**: ゲーム一覧 + 個別ゲームリンク
  - **その他のコンテンツ**: 辞典一覧（`/dictionary`） / ツール一覧 / チートシート
  - **サイト情報**: ブログ / 実績 / About / プライバシーポリシー

  ユーモア辞典（Q20）は「占い・診断」セクションには配置しない。ユーモア辞書は占いでも診断でもなく、占い・診断を期待して訪れたユーザーが辞書コンテンツに遭遇すると「期待と違う」という体験を生む。ユーモア辞書を探すユーザーにとっても、「占い・診断」というラベルからは辞書を連想できず、見つけにくい。代わりに「その他のコンテンツ」セクションに辞典一覧（`/dictionary`）へのリンクを配置し、`/dictionary` 一覧ページをユーモア辞典の正規入口とする。

  「その他のコンテンツ」セクションに辞典一覧・ツール一覧・チートシートをまとめる。セクション名は訪問者にとって意味のあるラベルとする（「削除予定コンテンツ」のような内部用語は使わない）。4-1実行時にはこのセクションからツール一覧・チートシートのリンクを削除し、辞典一覧（ユーモア辞典のみ）のリンクを残す。セクション内リンクが辞典一覧1本のみになった時点で、セクション名を「辞典」に変更する。

- **`/dictionary` 一覧ページの扱い:**

  `/dictionary` 一覧ページはユーモア辞典のために存続する（cycle-97での決定事項）。4-1実行時に削除予定辞典3種（漢字・四字熟語・伝統色）を削除した後、ユーモア辞典1種のみの一覧ページとなる。対応方針:
  - 4-1実行前（現時点）: `/dictionary` 一覧ページは現状維持。フッター「その他のコンテンツ」セクションから `/dictionary` へリンクし、ユーモア辞典を含む全辞典へのアクセスを確保する。
  - 4-1実行時: 削除予定辞典3種のカードを一覧ページから除去。ユーモア辞典のみの一覧ページとなる。1件のみの一覧ページとしての体裁を整える（説明文の調整等）。あるいは、ユーモア辞典のページ自体が十分な導線を持つ場合は、一覧ページからユーモア辞典への自動リダイレクトも検討する。

**「どこかから辿り着ければOK」の確認:**

ヘッダーから「ツール」「辞典」を外しても、以下の動線が残る。

- フッター「その他のコンテンツ」セクション内のリンク
- サイト内検索
- ブログ記事内のリンク

これは「どこかから辿り着ける」条件を満たす。

ユーモア辞典（Q20）への動線は以下で確保される。

- フッター「その他のコンテンツ」セクション内の辞典一覧（`/dictionary`）リンク
- `/dictionary` 一覧ページ（存続。ユーモア辞典の正規入口）
- サイト内検索
- トップページの占い・診断ショーケースや `/quiz` 一覧ページにはユーモア辞典を含めない（占い・診断を期待するユーザーのカテゴリ信頼性を維持するため）

- **`/quiz` URL の驚き最小の原則に関する検討:**

  現在のヘッダーナビのラベルを「クイズ」から「占い・診断」に変更する方針だが、リンク先URLは `/quiz` のままである。これは驚き最小の原則に反する。ユーザーがURL欄で `/quiz` を見たとき「クイズ」を期待するが、実際の中身は占い・診断が中心である。

  `/quiz/[slug]` 配下の14種のコンテンツは以下の3タイプに分類される:
  - **占い系（2種）**: ユーモア占い、キャラ占い
  - **診断系（7種）**: 音楽性格診断、動物性格診断、理系思考診断、日本文化適性診断、キャラクター性格診断、伝統色性格診断、四字熟語性格診断
  - **知識テスト系（3種）**: 四字熟語力診断、漢字力診断、ことわざ力診断

  加えて日替わり占い（`/fortune/daily`）が `/quiz` 一覧ページに看板コンテンツとして掲載されている。これらの全体を正確に包含する名前は「クイズ」でも「占い・診断」でもなく、「クイズ」は知識テスト3種にしか当てはまらず、「占い・診断」は知識テスト3種を含まない。

#### 4-4 メタデータ・SEO改善: 先行実施可能（保持コンテンツのみ対象）

**判定: 先行実施可能。削除予定コンテンツのメタデータ改善は不要。保持コンテンツのみを対象にすれば完全に独立して実施できる。**

**理由:** 削除予定コンテンツ（ツール33種・チートシート7種・辞典3種）は近い将来410 Goneで削除される。これらのメタデータを今改善しても、検索エンジンに対するシグナルとして機能する期間が短く、投資に見合わない。むしろ、保持コンテンツのメタデータを新コンセプトに合わせて最適化することに集中すべきである。

**方針:**

対象を以下に限定して実施する。

1. **トップページのメタデータ**: 現在の「AIエージェントが企画・開発・運営するWebサイト。無料オンラインツール、デイリーパズルゲーム、クイズ・診断、AIブログを提供しています。」を、新コンセプト（占い・診断パーク）に沿った記述に変更する。
2. **占い・診断14種のメタデータ**: 各診断ページのtitle/descriptionを最適化。SNSシェア時のOGP情報も確認・改善。
3. **ゲーム4種のメタデータ**: 確認・必要に応じて改善。
4. **ブログ記事のメタデータ**: 保持する50件の記事のtitle/descriptionを確認。3-8（ブログ記事修正）で既に修正済みの記事のメタデータは確認のみとし、未修正の記事を優先して対応する。
5. **一覧ページ（/quiz, /games, /dictionary）のメタデータ**: 新コンセプトに合わせた記述に更新。`/dictionary` はユーモア辞典のための一覧として存続するため、メタデータも適切に更新する。
6. **sitemap.tsの整備**: 4-1実行時の変更を見越して、現時点では変更しない（削除予定コンテンツは現在のままsitemapに含める）。

**削除予定コンテンツのメタデータ:** 変更しない。現状のまま維持し、4-1実行時にsitemapからの除外と410 Gone対応を一括で行う。

**URL変更が決定した場合の注記:** 4-3で `/quiz` から `/shindan` 等へのURL変更が決定した場合、一覧ページおよび各コンテンツページのメタデータはURL変更と同時に最終調整を行う。先行実施する段階では、URL部分を含まないメタデータ（description文面、OGP画像等）の改善に注力し、URL依存のメタデータ（canonical、OGP URL、構造化データ内のURL等）の最終確定はURL方針決定後とする。これにより、URL変更後にメタデータを再修正する二度手間を回避する。

#### 4-5 内部リンク構造の最適化: 先行実施可能（条件付き）

**判定: 先行実施可能。ただしブログ記事内の削除予定コンテンツへのリンクは原則維持する。**

**ブログ記事内リンクの扱い:**

調査の結果、ブログ記事から削除予定コンテンツへのリンクは以下の3パターンに分類される。具体的な記事一覧は後述の「ブログ記事内の削除予定コンテンツへのリンク一覧」を参照。

- **パターンA: 当時の文脈でのリンク**（例: 「[cron式パーサーツール](/tools/cron-parser)を開発する中で」「[正規表現テスター](/tools/regex-tester)にコピーして試す」）。記事が当時の状況を記録したものであるため、リンクを切ると記事の文脈が損なわれる。**維持する。** 4-1実行後にこれらのリンク先が410になっても、記事自体は当時の記録として成立する。
- **パターンB: 「あわせてご活用ください」形式の誘導リンク**（例: 「[カラーコード変換ツール](/tools/color-converter)と合わせてご活用いただければ幸いです」「[HTTPステータスコード チートシート](/cheatsheets/http-status-codes)もあわせてご活用ください」）。これらは4-1実行時に削除先へ誘導してしまう問題がある。しかし、4-1実行前の現時点では変更不要。**4-1実行時に対応する。**
- **パターンC: サイト構造の説明でのリンク**（例: url-structure-reorganizationでの `/dictionary/kanji` の言及、site-search-featureでの「[ツール](/tools)」「[伝統色辞典](/dictionary/colors)」の言及）。記事の文脈として当時のサイト構造を説明しているもの。**維持する。**

**方針:**

現時点で実施する内部リンク最適化は、保持コンテンツ間のリンク構造の改善に限定する。

1. **診断完了後の回遊リンク**: 各占い・診断の結果ページに「次はこれを試してみよう」の推薦リンクを追加。site-concept.mdの回遊設計に記載された「テーマの異なる診断を推薦」を実装。
2. **占い・診断一覧ページ（/quiz）の整備**: 14種の占い・診断を適切にカテゴリ分けし、回遊を促進するリンク構造にする。ユーモア辞典（Q20）は `/quiz` には含めない（辞典は `/dictionary` が正規入口）。
3. **ゲームから占い・診断への誘導**: ゲーム完了後に占い・診断への導線を追加。
4. **ブログからの保持コンテンツへのリンク確認**: 占い・診断に関連するブログ記事から各診断への適切なリンクがあるか確認。

### 検討した他の選択肢と判断理由

#### 選択肢1: 4-1を待ってから4-2〜4-5をすべて一括で実施する

**不採用の理由:** 現在のトップページとナビゲーションは新コンセプトと完全に乖離しており、新規コンテンツ14種の価値がユーザーに届いていない。GA確認待ちの期間にこの状態を放置することは、メインターゲットの体験を著しく損なう。特にトップページのメタデータが旧コンセプトのままであることは、検索エンジンに対しても「占い・診断サイト」としての認識を遅らせる。待機コストが大きすぎる。

#### 選択肢2: 4-2〜4-5を実施する際に、削除予定コンテンツへのリンクをすべてヘッダーに残す

**不採用の理由:** ヘッダーナビに「ツール」「辞典」を残すと、メインターゲットにとって不要な選択肢がナビゲーションを希薄化させる。7リンクのナビは選択肢が多すぎる。新コンセプトでは占い・診断がメインであることを明確にすべきであり、削除予定コンテンツへの動線はフッターに集約する方がユーザー体験とコンセプト訴求の両面で優れている。

#### 選択肢3: ブログ記事内の削除予定コンテンツへのリンクを今すぐ外部代替に差し替える

**不採用の理由:** まだ削除が実行されていない段階でリンクを差し替えると、自サイトに存在するコンテンツへのリンクを外部に向けてしまい、ユーザーにとって不自然な体験になる。また、ブログ記事は当時の状況を記録したものであり、「当時の状況を尊重する」という修正原則（site-value-improvement-plan.md 3-8の原則）に反する。リンク差し替えは4-1実行時に行うのが適切である。

#### 選択肢4: 削除予定コンテンツのメタデータも合わせて改善する

**不採用の理由:** 削除予定コンテンツは近い将来410 Goneで処理される。メタデータを改善しても検索エンジンがインデックスを更新し効果を発揮する前に削除される可能性が高い。限られたリソースは保持コンテンツの最適化に集中すべきである。

#### 選択肢5: ユーモア辞典を /dictionary/humor/ から別のURLに移動する

**不採用の理由:** cycle-97で3案を比較検討した結果、`/dictionary/humor/` が最適と判断された（既存構造 `/{大分類}/{コンテンツ}` に従い、辞書カテゴリの一コンテンツとして配置）。URL変更はSEO上のリスクを伴い、cycle-97の検討結果を覆す根拠がない。
