## 作業計画

### 目的

移行で**移行前より劣化した**回遊の摩擦を取り除く。上下を罫で区切った行も、人はカード＝クリック可能な単位と認識する。多くのサイトは一覧のタイトル＋説明を含む単位全体をクリック可能にする。本サイトは移行後、回遊の最上流ハブ（一覧）で**タイトル文字列だけがクリック可能**の劣化型になっていた——「行のどこを押しても開くと思ったのに、タイトルの文字の上でしか反応しない」。来訪者が毎回体験するこの摩擦は、ページ遷移を1つずつ減らす＝Goal（PV 最大化）を直接損ねている。constitution Rule 4（全コンテンツをあらゆる面で最高品質に）に照らせば、回帰は明白な品質未達であり是正義務がある。

重要なのは、これが本丸 B-545（結果体験そのものの価値）のすり替えではない点だ。cycle-280 の事故記録が名指しした反復は「結果面のシェアを、動線・計測・装飾・A/B の機械的問いにすり替える」ことだった。本サイクルは結果面には触れない。移行が壊した回遊の標的を、独立に正しく直す。B-545 は開いたまま残し、過去の賭けの結末と確立知見を読む groundwork を踏んでから別サイクルで正面から取り組む。

### 来訪者（誰が・なぜ来るか・市場）

- **誰が**: 一覧から個別コンテンツへ進もうとする、サイト内を回遊する全来訪者。トップ `/`（診断/辞典/道具/読みもの の4棚）・`/play`・`/tools`・`/dictionary`・四字熟語/色の索引・`/about`・`/blog`（一覧/カテゴリ/タグ/ページ送り）・記事末の関連記事——回遊の最上流ほぼ全面が対象。
- **なぜ手が止まるか**: カードの見た目（罫で区切られた1行＝クリック可能な単位）とクリック可能領域（タイトル文字のみ）が食い違う。タイトルを外すと反応せず、来訪者は「押せない」と誤認するか、正確にタイトルを狙う余計な操作を強いられる。とくにモバイルでタイトルが短い行は標的が小さく、外しやすい。
- **市場/規模**: これは特定クラスタの獲得ではなく、既に来ている全来訪者の回遊効率の底上げ。低リスク（下流に良好型が既存＝揃える先が確定している）で全面に効く。

### 作業内容

1. **設計フェーズ（先に方式を確定）**: 揃える先は下流に既存する良好型 `PlayRecommendBlock`（`src/dictionary/_components/new/PlayRecommendBlock.tsx`。コメントに「行全体を 1 本のリンクにして大きいタップ領域を確保する（重複リンクを避ける）」と明記・行全体を単一 `<Link>` で包み 44px 確保）。ただし2種の行で条件が異なる:
   - **Shinagaki**（行内に対話要素なし＝note/値札/右端メタは非対話 span）→ 行全体を単一 `<Link>` で包む素直な形。RelatedArticles は Shinagaki に委譲するため自動的に是正される。
   - **BlogList**（`TagList` がタグページへのリンクを含みうる＝`<a>` の入れ子は不正）→ stretched-link 方式（主リンクに `::after{position:absolute;inset:0}` を当てて行全体を覆い、タグリンクを `position:relative`+`z-index` で上に出す）で、行全体クリックとタグの独立クリックを両立させる。

2. **実装（部品ごとに1エージェント）**: Shinagaki・BlogList を上記方式で全体クリック化。44px・focus 可視・キーボード・SR を担保。各実装で take-screenshot による変更前/変更後の視覚確認（light/dark×375/1280）。

3. **根本原因の予防（DESIGN.md 規定追加）**: 回帰を許した根本は、DESIGN.md §4「品書き」が行の中身（品名・ひとこと・値札）を規定する一方、**クリック領域＝視覚単位の全体か否かを規定していなかった**沈黙（cycle-280.md §157）。デザインシステムに「一覧/カードは視覚単位の全体をクリック可能にする・44px・主/副リンクの両立指針」を明文化し、次に一覧系部品を足すときの劣化を SSoT で塞ぐ。これは B-572 の是正と密結合（直す＋再発を止める）なので同時に行う（CLAUDE.md「Improve work and process」）。

## 実装結果

- **方式**: 当初案（Shinagaki を行全体で単一 `<Link>` に包む）は、アクセシブル名の汚染（品名＋note＋値札＋metaの連結を SR が読み上げる）・既存テスト破壊・BlogList では見出しアウトライン破壊/入れ子アンカー不正を招くため不採用。**両部品とも stretched-link に統一**——主リンク（品名）は `<a>` のまま残し、`.row{position:relative}` ＋ 主リンクの `::after{content:"";position:absolute;inset:0}` で**クリック/タップ標的だけ**を視覚単位（行）全体へ拡張。DOM 不変ゆえアクセシブル名は主リンク名のみに保たれる。
- **Shinagaki**（`Shinagaki.module.css` のみ・`index.tsx` 不変。RelatedArticles を委譲で内包）: `.row` に position:relative、`.name::after` を追加。行内に対話要素が無いため hover は既存 `.name:hover`（`::after` 経由で行全体で発火）をそのまま使用。モバイル 44px 残置。
- **BlogList**（`BlogList.tsx`／`BlogList.module.css`／`TagList.tsx`）: `.titleLink::after` で標的を行全体へ。タグリンクは `TagList` に汎用 `className?` プロップを新設して根 `<ul>` に `.tagRow`（`position:relative; z-index:1`）をマージし、`::after` の前面に維持して独立クリックを両立（`<div>` ラッパにすると空タグ行で余分な gap が出るため className 渡し）。hover はタグ上では発火しない（タグが `::after` の前面のため＝タイトル朱化しない）。
- **タグ右側デッドゾーンの是正**: `.tagRow{width:fit-content; max-width:100%}` を追加。タグ帯を内容幅に絞り、右側の空きを主リンクの `::after`（記事遷移）に戻す。`fit-content=min(max-content,available)` により狭幅では利用幅で頭打ち＝折り返し維持・overflow なし。デスクトップでデッドゾーン解消、モバイル 375 でタグ12個強制でも3行折り返し・overflow ゼロを実機検証（設計時の「fit-content が折り返しを壊す」懸念は実測で否定）。
- **根本原因の予防**: `DESIGN.md` §4 に原則「クリック標的＝視覚単位の全体」を追記（実装手段は書かない）。実装手段（stretched-link レシピ・z-index・デッドゾーン是正）は `docs/knowledge/frontend.md` を新設して記述。移行が最上流面で劣化型を選んだのは §4 がこの点に沈黙していたためで、SSoT に規定を足して再発を塞いだ。
- **テスト**: アクセシブル名＝品名のみ／タグ独立リンク／TagList `className` 付与・後方互換を固定するテストを追加。品質ゲート lint / format:check / test（**5444 passed**・新規5・既存全通過）/ build すべて緑。
- **視覚不変の確認**: stretched-link は挙動のみで見た目を変えないため、before/after（`tmp/shots-before`・`tmp/shots-after`・`/`・`/blog`・light/dark×6幅）が同一＝レイアウト回帰なしを PM 実見。
- **副産物: 新タグページの品質補完**: 本サイクルのブログが「アクセシビリティ」タグを付けたことで同タグが3記事以上に達し `/blog/tag/アクセシビリティ` が実ページ化。tag-page テストが要求する `TAG_DESCRIPTIONS`（`src/blog/_lib/blog.ts`）のエントリ未定義を検出→説明文（121字・既存文体準拠）を追加。閾値超えでインデックス対象タグが増えるのは望ましい方向で、説明文を補って品質を担保した。
