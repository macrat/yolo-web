# B-598 処遇判断の記録 — 検索機能は「撤去」

cycle-292 の核心タスク。未結線の検索機能（`src/components/search/` 一式＋索引生成）を **結線するか撤去するか** を、履歴とデータに基づいて判断した記録。

## 結論

**撤去（remove）する。** グローバル検索モーダル一式・索引生成スクリプト・`generate:static-assets` 配線・`public/search-index.json`・関連テスト・検索専用のランタイム依存 `fuse.js`（package.json / package-lock.json）・Header の UI 側トリガー残骸（`onSearchOpen` prop・SearchIcon・Cmd+K キーバインド・検索ボタン・関連CSS/テスト・layout の「Phase 5 で結線予定」コメント/テスト）を除去する。これは cycle-186 で確定済みの「作らない」判断を実装面で完遂する作業であり、新たな方針転換ではない。

## 判断の芯

### 1. 「結線しない」は既に確定済みの製品判断（cycle-186）

履歴再構成（サブエージェント調査・原文引用付き）で確定した事実:

- **cycle-174** で「① 新デザインに横断検索を作る」と一旦確定。
- **cycle-185** で暫定結線を全撤回。ただしこの撤回は【やり方（スコープ越境・ファクトチェック不実施）の否定】であって、結線という結論の否定ではなかった（当時は「Phase 5 で結線」を維持）。
- **cycle-186** で「② 作らない（Phase 5 スキップ）」に**再判断・確定**。cycle-174 が挙げた採用根拠6項目のうち A-13（Google サイトリンク検索ボックスによる SEO 便益）は、実は既に無効な便益だった——サイトリンク検索ボックスは2024年11月に廃止済みで、このサイトの発足（2026年2月）より前から存在しなかった（cycle-186 が一次資料で確認し、cycle-174 の見落としを是正）。A-13 を除いた残5項目も cycle-174 自身が弱点を認める中項目で来訪者価値は限定的なため②を確定。実装コスト等を判断材料に含めない来訪者価値ベースの判断（L75）。B-331 は Done 化。
  - **重要な訂正**: A-13 は「作っている間に失効した便益」ではない。サイト発足前から死んでいた便益を、cycle-174 が確認せず採用根拠に挙げ、cycle-186 が是正したもの。当初この点を「作る根拠が途中で失効した」と脚色した撤去記事の下書きを書いたが、事実に反するため取り下げた（下記「教訓記事の取り下げ」）。**このサイトの検索を作った理由は UX（横断的にコンテンツを探す導線）であり、サイトリンク検索ボックスとは無関係**（build 時の一次資料＝元記事 site-search-feature.md 本文が UX 理由のみを記し、サイトリンクに言及していない）。撤去の理由も UX/データ（未使用・外部検索が各ページへ直接誘導）であって、サイトリンク廃止は撤去の根拠ではない。
- **cycle-279** の legacy 一掃で呼び出し元（legacy Header）は消えたが `src/components/search/` 本体と `scripts/build-search-index.ts` が残置され、完全な dead code 化。cycle-186 が想定した「Phase 10.2 で legacy と一体で自然消滅」が部分的にしか実行されなかった。
- **cycle-287** 監査 C7 で検出 → **B-598** として管理下に。

→ **「結線」を今選ぶことは、cycle-186 の確定判断の再掘削（re-litigation）になる。** それを正当化するには cycle-186 の根拠を覆す新証拠が要る。

### 2. 現在のデータは cycle-186 の判断を覆さない（むしろ補強する）

GA4/BigQuery（2026-03-28〜07-19・113日・1,631セッション）＋ Search Console:

- **単一PV離脱が支配的**: 全セッションの 79.4%（10人中約8人）が1ページで離脱。グローバル検索が助ける「多ページ探索型」ではない。探索起点になりうるトップ（回遊48%）・辞典一覧（回遊36%）でも「探せず困る」異常な離脱集中は見られない。
- **ブランド/ナビゲーショナルクエリが皆無**: `yolos.net` 2clicks・`site:yolos.net` 0clicks・「yolos＋探索語」型 0件。「サイト内で探し直したい」需要の外部証拠なし。Google 検索（6,560種のクエリ）が事実上のサイト内検索として機能し、来訪者を目的ページへ直接着地させている。
- **検索モーダルの実利用ほぼゼロ**: `search_modal_open` 1回/1ユーザー、`view_search_results` 1回のみ（113日）。実質1〜2ユーザーで動作確認の公算。

→ 新証拠は「結線支持」ではなく「結線不要」の方向。cycle-186 の判断は現在も妥当。

### 3. 位置づけ（撤去は機能の全否定ではない）

撤去は「検索という機能の価値の全否定」ではなく「現状 dead code のまま毎ビルド 673KB を生む状態の解消」。将来フェーズC（クラスタ再設計）で改めて探索導線が必要になれば、その時の要件で作り直す判断を別途行う。

なお、データ調査の副次所見として、**一覧ページのローカルフィルタ**（辞典の `DictionarySearch`・`/blog` 一覧フィルタ・`/tools/keigo-reference` の絞り込み。いずれも撤去対象のグローバル検索モーダルとは別機能で自己完結しており、撤去スコープに含めない）は極小ながら実利用の痕跡がある。投資するなら「グローバル検索」ではなく「一覧の見やすい導線・ローカル絞り込み」——これは本タスクのスコープ外だが、気づきとしてキャリーオーバーに記す。

## 撤去に伴う公開記事の是正（レビュー指摘・必須）

白紙再レビューで、撤去だけでは来訪者に虚偽が残ると判明した。公開済みブログ記事 `2026-02-21-site-search-feature.md`（「サイト内検索を実装しました。Cmd+K で開けます」）が検索機能を来訪者に告知しており、コードを撤去すればこの記事は恒久的に誤りになる。撤去を「誠実な健全化」と位置づける以上、記事側の是正を撤去スコープに必ず含める。

対処方針は blog-writing.md の確立ルールに従う——**本文は当時の記録としてそのまま保ち、書き換えない**（撤去済み機能を現在形で紹介していても「当時こう書いた」という記録であり、本文改変は記録の改竄＝読者を欺く）。代わりに **冒頭に日付つき注記** を加え、現時点では機能が撤去済みであることを伝える。前例: `nextjs-route-handler-...md` のメモフィード廃止追記、`series-navigation-ui.md` 冒頭注記。

- `2026-02-21-site-search-feature.md`: 冒頭に日付つき注記を追加（本文不変）。撤去理由は UX/データ（未使用・外部検索が各ページへ直接誘導）と記す。※当初の注記はサイトリンク廃止を撤去理由に挙げていたが、事実に反するため削除・是正した。
- `2026-02-21-dark-mode-toggle.md` L29 の内部リンク `/blog/site-search-feature`: 記事は削除せず注記のみのため **リンクは依然有効。ルール上いじらない**（本文書き換え・リンク除去はしない）。

### 教訓記事の取り下げ（重大な自己批判）

撤去の経緯を語る記事 `2026-07-21-feature-rationale-expires.md` を一度執筆・公開しようとしたが、**取り下げた**（下書きは記録として commit 保全のうえ削除）。理由: 記事は、時系列を一次資料で確認しなかったために生まれた **2つの事実誤認** を骨格に据えていた。

1. **「作る根拠だったサイトリンク検索ボックスが、作っている間に失効した」**——一次資料（git log）で **サイトリンク廃止(2024-11-21) < サイト発足(2026-02-13) < 検索実装(2026-02-21)** が判明。サイトリンクはサイト発足前から死んでおり「作っている間に失効」は起きていない。作った理由も UX（build 時の元記事本文）でサイトリンクとは無関係。cycle-186 が持ち込んだ A-13 の扱いを検証せず「失効の物語」に脚色していた。
2. **「検索は最後まで UI に繋がらず、来訪者は一度も使えなかった」**——これも事実に反する。公開時のコミット `d7ec1f8e`（2026-02-21）で legacy Header が `<SearchTrigger />` を描画し、可視の検索ボタン（aria-label「サイト内検索」）＋Cmd+K を結線していた。**公開当初、検索は結線され使える状態だった**。dead code 化したのは後のデザイン刷新（新デザインが cycle-181/185 で未結線化）と cycle-279（legacy Header 消滅）以降であり、「未使用」の解析データ（2026-03-28〜）はその後期を映しているにすぎない。

いずれも AP-W16（一人称の経緯・因果を一次資料で確認せず物語に脚色）に該当。根本原因は「裏取りしない単一の時系列が複数の捏造を生む」ことで、語る前に git で全時系列を再構成すべきだった（Owner 指摘で発覚）。

本サイクルでは新規ブログを公開しない。撤去の題材から誠実に取り出せるのは「公開当初は稼働していた検索が、デザイン刷新で静かに UI から切り離され、ビルド時 dead code として残ったのを撤去した」という内部保守の経緯に留まる。小規模データ（113日1,631セッション）で強い一般論（例「Google が事実上のサイト内検索だから内部検索は不要」）を主張するのは過度の一般化で、単一の強い独自主張として成立しない。無理に書けば再び物語を捏造する。よって公開を見送る（＝backlog への延期タスクも作らない。延期は完了手順違反であり、そもそも書くべき確たる主張がない）。撤去という **製品判断（コードの撤去）自体は、現状 dead code である事実＋データで独立に妥当** であり、この訂正では揺らがない。
