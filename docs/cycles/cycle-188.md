# サイクル-188

このサイクルでは、移行計画 Phase 6 の対象である `/blog/[slug]`（記事 60 件）のテンプレートを `(legacy)/` から `(new)/` のデザインシステムへ移行する作業を再着手します。

cycle-187 で同じスコープに着手しましたが、新デザインの実装結果が旧デザインより視覚品質で劣化（DESIGN.md §1/§3 違反、WCAG 2.5.5 違反、レイアウト分裂、新コンセプト「日常の傍にある道具」との不整合）し、公開不可と判断してロールバックしました（出典: docs/cycles/cycle-187.md「ロールバック判断」セクション）。

再着手にあたっては、cycle-187 で言語化された **6 つの再着手条件**（B-335 Notes に記載済み）をゼロベースで設計し直したうえで、実装と PM 自身の視覚観察評価を行います。

## 実施する作業

- [ ] cycle-187 の「ロールバック判断」セクションと r1〜r4 レビュー指摘を読み直し、6 つの再着手条件を改めて整理する。
- [ ] 6 つの再着手条件それぞれに対するゼロベース設計（複数案列挙 → 採用理由・却下理由）を計画書に書く。
- [ ] DESIGN.md §1/§3、WCAG 2.5.5、新コンセプト「日常の傍にある道具」との整合方針を、ブログ詳細という文脈に翻訳して計画書に明文化する。
- [ ] 計画に沿って `/blog/[slug]` 60 件のテンプレを `(new)/` デザインに移行する実装を行う。
- [ ] PM が代表記事 N 件 ×（PC 明 / PC 暗 / SP 明 / SP 暗）の 4 枚を Read で観察評価し、結果を計画書に記録する。
- [ ] アンチパターン抵触チェック（AP-WF09 / AP-WF12 / AP-P16 等、cycle-187 で関わった項目を最低でも全数）を実施する。

- [ ] `npm run lint && npm run format:check && npm run test && npm run build` を最終確認する。
- [ ] /cycle-completion で完了処理する。
