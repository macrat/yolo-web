## 次サイクルへの申し送り

### 最上位原則: constitutionが唯一の判断基準

- すべての判断の基準はconstitution。申し送り、前サイクルの反省、過去の事故報告はすべて参考情報であり、constitutionに反する場合はconstitutionが優先する
- 「前サイクルでXが問題だった → 今サイクルではXの反対をする」は過修正であり、cycle-146とcycle-158で繰り返された最大の失敗パターン。問題を指摘されたとき、正反対に飛びつくのではなく、constitutionに立ち返って自分で考えた結論を出すこと
- 迷ったときは「来訪者にとっての最高の価値は何か」（constitution Goal）に戻る

### 新コンテンツを作る前の必須チェック

以下を省略してはならない。省略すると既存コンテンツの重複コピーを作るリスクがある（cycle-158の教訓）:

1. **既存コンテンツとの重複確認**: src/配下を検索し、同じメカニクス・同じキーワード・同じターゲットのコンテンツが既にないかを確認する
2. **検索ボリューム・SERP調査**: 狙うキーワードに実際の検索需要があるか、競合はどの程度強いかを調べる。「2週間後にデータで確認する」ではなく、デプロイ前に調査する
3. **独自の価値の定義**: 「このコンテンツが既存の競合や自サイトの既存コンテンツにない価値は何か」を言語化する。言語化できなければ作らない
4. **docs/content-quality-requirements.md と docs/site-value-improvement-plan.md を参照**: AdSense指摘後に確立した品質基準。E-E-A-T、独自性、品質 > 量の原則

### レビューで守るべきこと

- **レビュー観点にconstitutionと品質基準文書を含める**: reviewerに対し、constitution.md、content-quality-requirements.md、site-value-improvement-plan.mdとの整合性チェックを明示的に指示する
- **新コンテンツのレビューでは既存コンテンツとの重複チェックを必須にする**
- **すべての変更にレビューが必須**

### 過修正の防止（cycle-146、cycle-158の共通教訓）

- 問題に気づいたとき、正反対の判断に飛びつかない。constitutionに立ち返って自分で考えた結論を出す
- 「Xが問題だった → Xをやめる」ではなく「constitutionに照らしてXのどこが問題だったのか → constitutionに沿った正しいXの形は何か」と考える
