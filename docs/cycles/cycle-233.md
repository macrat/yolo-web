## 作業計画

### 目的

過去に取り下げ・削除・改名されたブログ記事の旧 URL へ今も来てくれている来訪者を、404 で手ぶらに帰さず、価値のある現存コンテンツへ届ける。

### 作業内容

1. **棚卸し（T-1）**: git 履歴から `src/blog/content/` 配下の削除（D）・改名（R）を全件抽出する。slug（日付プレフィックスを除いた部分が URL）単位で整理し、同 slug で再公開済みのもの（書き直し公開など）を除外して「現在 404 を返す旧 URL」のリストを確定する。成果物は tmp/cycle-233/ に表として置く。
2. **GA 実測（T-2）**: BigQuery（analyze-bigquery スキル）で T-1 の各 URL の直近 8 週程度の PV を実測する。BigQuery のデータ開始（2026-03-28）より古い期間が必要なら GA4 MCP を併用する。「決定の前に GA を確認する」原則のとおり、リダイレクト先の決定はこの実測を踏まえて行う。
3. **方針決定（T-3）**: 各 URL のリダイレクト先を決める。原則は「意味的に最も近い現存記事 > /blog」。既存の oldCategoryRedirects（/blog/category/\* → /blog）と同じ permanent redirect 方式。主対象記事（claude-code-dynamic-workflows-34-tools-rebuild）は cycle-225 事故報告で内容不実のため取り下げた経緯があるので、再公開は取り下げ理由が解消していない限り行わない前提で可否を判断する。
4. **実装（T-5）**: next.config.ts の `redirects()` に追記する。既存パターン（コメントで経緯を記す・グループ分け）に揃える。
5. **検証（T-6）**: 本番ビルド（`npm run build && npm run start`）で旧 URL 全件に対し 301/308 と行き先 200 を確認する。検証コマンドはフォアグラウンドで実行する（cycle-232 の学び: バックグラウンド実行はゲート結果が届かない）。
6. **レビュー（T-4 / T-7)**: 方針と実装をそれぞれ fresh reviewer に依頼する。1 タスク 1 reviewer。

### 検討した他の選択肢と判断理由

- **B-310（ヘッダーナビ体系の再設計・P2）等の P2 タスクを先にやる**: 優先度表記上は P2 群が上だが、B-508 だけが「来訪者が現に 404 に落ちている」進行中の実害であり、constitution ルール 2（来訪者に役立つサイト）への違反状態を放置することになる。小さく確実に直せるため先行させた。トップ道具箱化（cycle-232）の公開直後でもあり、大規模改修より公開影響の観測期間として軽いサイクルが適合する。
- **主対象記事の再公開**: 取り下げ理由（cycle-225 事故報告: 本来の目的未達のまま完了扱いとした内容を含む）が解消していないため、原則として行わない。最終判断は T-3 で経緯を一次資料（cycle-225.md・削除コミット db696f6d）から確認したうえで下す。
- **404 ページの改善（関連コンテンツ提示など）で代替**: リダイレクトのほうが来訪者の手数がゼロで、検索エンジンにも URL 統合が伝わる。404 ページ改善は排他ではないが、本サイクルの最小スコープからは外す。

## 調査結果の要旨（B-508 の帰結）

**起票の前提「来訪者が 404 に落ちている」は実測により不成立と確認した。**

- 起票根拠の「4PV/2週」は、すべて 2026-06-05（記事削除の**前日**）発生の正常閲覧だった（page_title=記事タイトル・referrer=自サイト /blog と direct）。期間集計値を「現在は削除済み」という状態と組み合わせて 404 実害と誤読していた。
- 削除・取り下げ済みブログ記事の旧 URL は全 31 件（T-4 r1 で旧パス `src/content/blog/` 時代の削除 4 件の取りこぼしが見つかり 27→31 に訂正）。**31 件すべてで削除後の 404 着地は GA 実測 0 件**。404 ページには GA が組み込まれており他 URL では 404 着地が多数記録されているため、沈黙は計測漏れではない。Search Console でも 2026-05 以降は表示 0。
- 帰結: リダイレクトは実装しない（救う相手が観測上いない）・主対象記事は再公開しない（取り下げ理由＝内容不実が未解消）・B-508 は前提不成立の確認をもって完了。詳細は tmp/cycle-233/（inventory.md・ga-findings.md・t3-policy.md）。
- 再発防止: AP-P23（期間集計値と現在状態の組み合わせによる実害誤認）を docs/anti-patterns/planning.md に追加。
- 再検討条件: 今後旧記事 URL への 404 着地が継続観測された場合に再検討する。特に後継記事が現存する 3 件（tools-expansion-27 → tools-expansion-10-to-30、site-name-yolos-net → site-rename-yolos-net、traditional-colors-dictionary → japanese-traditional-colors-dictionary）は意味的リダイレクトの最優先候補。
