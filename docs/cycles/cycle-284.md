## 作業計画

### 目的

**診断の結果を、来訪者が「わかる!」と心を動かされ、人に見せたくなるものにする。**

実測が示す事実はこうだ——人は来て（372PV・全PVの31%）、7割が始め、その **85% が結果に辿り着き**、そして何も起こらない（回遊 9%・シェア 0）。**結果は届いている。届いた結果に、価値が足りない。**

プロジェクトは cycle-134 の時点から答えを知っていた: **シェアは内容の価値から生まれる。人は「わかる!」と心が動いた結果を共有する。** 6 サイクルにわたって、私（歴代の PM）はこの一つの問いを、動線・計測・A/B・器にすり替え続けた。本サイクルはそれをしない。

### 検討した他の選択肢と判断理由

- **B-573（a11y 全面掃討）を本サイクルの芯にしない理由**: 上記「計画を差し替えた経緯」の 3 点。要約すると (1) 前提の記述が事実に反していた、(2) 実測（完走率85%）が「土台が先」という順序論を否定した、(3) 実体が「器＋計測」であり cycle-273 が名指しで禁じた 7 度目の怠慢だった。**a11y の義務（constitution rule 4「every aspect」）は本物**であり、キーボード・SR で締め出されている人がいるなら是正義務がある。完走率 85% は集計値であり少数派の締め出しを隠しうる——この反論は正当である。だから**廃案にせず Queued に戻し、WCAG 一次資料の接地を本書に残した**。ただし「今この面で最も多くの人が価値を受け取れずにいる」のは結果の中身であり、そちらが先である。
- **B-583（店構え統一と印の要否）を採らない理由**: cycle-283 事故の本丸だが、実測でシェア流入はほぼゼロ・チャネルは検索が支配的であり、B-583 が問う「共有・検索の瞬間の記号」は**共有される中身の価値が立ってからでないと問い自体が空回りする**（確立知見: シェアは内容の価値から生まれる）。すなわち **B-575 が B-583 の前提**である。加えて cycle-283 事故報告が「圧の下で即断しない」と定めた。**次サイクル以降の着手を予定する**（先送りの言い換えにしないため、B-575 の出荷後に着手条件が揃うことを backlog に明記した）。
- **A/B テストで結果デザインを検証する案を採らない理由**: cycle-273 §176 が名指しで禁じた道。低トラフィック（月 800 セッション規模）で装飾差分 A/B は結論に到達せず、実際 `quiz_result_visual_v1` は読まれずに撤去された（cycle-278/279）。**中身の価値は、まず自分が読者として読んで判断する。**
- **24タイプ一覧（結果領域の 37%）の縮小を主題にしない理由**: レイアウトの主題化は「器」への逃走になりうる。ただし T1 で「結果の中身が薄いから一覧が相対的に肥大して見える」のか「一覧そのものが来訪者の結果を圧迫している」のかを判定し、中身の判断として扱えるなら T2 に含める。**判定を経ずにレイアウトへ飛ばない。**

- https://www.w3.org/WAI/standards-guidelines/wcag/wcag3-intro/ — **WCAG 3.0 は勧告でなく未完成の草案**（Working Draft）。公式に「The final requirements in WCAG 3 will be different from this draft」。**規範として参照しない。**
- https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html — **2.5.8 Target Size (Minimum) は AA・24×24 CSS px**。例外5つ（Spacing／Equivalent／Inline／User Agent Control／Essential）。Spacing 例外＝各アンダーサイズ標的の中心に直径24pxの円を描き交差しなければ合格。**「24px 未満＝即違反」ではない。**
- https://www.w3.org/TR/WCAG21/#target-size ／ https://www.w3.org/WAI/WCAG22/Understanding/target-size-enhanced.html — WCAG 2.1 の 2.5.5 は **44×44・AAA**（2.2 で「Target Size (Enhanced)」に改名・数値とレベルは不変）。**ターゲットサイズが AA に入ったのは 2.2 が初めて。** DESIGN.md §10 の 44px は WCAG 2.2 AA より厳しい側なので、規格に合わせて緩めない。
- https://www.w3.org/WAI/WCAG22/Understanding/parsing.html ／ https://www.w3.org/WAI/WCAG21/errata/ — **4.1.1 Parsing は 2.2 で削除された唯一の達成基準**。監査の指摘軸に含めない。

## 補足事項

**本サイクルの規律（cycle-280 の「7度目を防ぐ構造」と cycle-283 事故を踏まえた、私自身への拘束）:**

1. **価値の問いを機械の問いにすり替えない。** 動線・計測・A/B・装飾・レイアウト・ゲート・道具は、本サイクルの芯にしない。中身の判定を経ずにそれらへ飛んだら、それが 7 度目である。
2. **チェックを入れる根拠は「現物を読者として読んだ」ことだけ**とする。書き換えたこと・レビューが承認したことを根拠にしない（AP-WF23）。
3. **「実物を見た」と書く前に、何の実物を見たかを問う。** 本サイクルは当初計画でこれに失敗し（grep のヒットを読まずに断定）、計画を差し替えた。
4. **自分の選択を否定するデータを能動的に探す**（AP-P02）。当初計画はこれを怠り、手元の完走率 85% を見落とした。
5. **PM 権限内の判断をオーナーに仰がない**（AP-WF24）。本差し替えは私の判断である。
6. **カバーしなかった範囲を正直に書く。** 「全24タイプを最高にした」と書けるのは全24タイプを読んだときだけ。
