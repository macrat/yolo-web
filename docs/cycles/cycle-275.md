# サイクル-275

## このサイクルで扱うこと

7サイクル(cycle-268〜274)未決着の本丸 **B-545** に、Goal 起点で正面から着手する。

本丸を引き継ぎ(cycle-274.md)は次のように据えた——**サイトの根幹(アーキテクチャとデザインシステム)が、撤退したはずの「道具箱-as-core」を前提に作られたまま**であり、データで判明した実際の価値の中心＝**診断**は、その上にドキュメントの継ぎ足し＋デザインの §7 例外として乗っているだけ、という根幹の構造的矛盾。austere（極限ミニマル）は「検証された土台」ではなく、撤退した道具箱-core のための抑制であり、診断への §7 再適合も含め全体が未検証（cycle-271 申し送り）。

過去7サイクルの失敗は2つの罠の反復だった——**Trap A**（楽な周辺を本丸と偽る）と **Trap B**（未検証の設計方向を勝利譚にする）。加えて cycle-273 は出荷ゼロの分析で「問いを駐車」と断罪され、cycle-274 は最大レバーを先送りして事故を起こした。

このサイクルの構え——**本丸を抽象論で「結論」しない（反射的結論は cycle-274 の事故そのもの）。かといって「トップページの矛盾」や「本文コピー一点」へ矮小化もしない（cycle-274 が owner に是正された過ち）。代わりに、根幹が来訪者価値を実際に損ねている“具体的な急所”を Goal とデータから特定し、そこで来訪者価値を高める改善を1つ出荷し、効果を正直に検証する。** これは accident report §4 が到達した中道（「核の価値の問いに接地から取り組み、効果を正直に確かめる」）の実行であり、Trap A（周辺の楽な作業ではなく根幹の急所）でも Trap B（未検証の壮大な設計賭けではなく、確立知見に裏打ちされた具体改善＋正直な検証）でも分析駐車（出荷する）でもない。

## この本丸の「具体的な急所」——GA 接地で特定した事実

cycle-274 が欠いた GA 生データ接地を最初に行った（`analyze-bigquery`、期間 2026-06-01〜30、詳細は下記 T1 と research doc）。判明した事実:

- **診断が価値の中心は確定**: /play が全PVの58%、character-personality 単独で全PVの34%(176PV)、エンゲージ 142秒/セッションで全セクション最長。
- **完走者は深く読み込む**: character-personality の漏斗は 到達148→開始114(77%)→完了108(**開始者の95%が完走**)。完走者はインライン結果を 138秒/セッション（サイト最高水準）読み込む。
- **にもかかわらず「見せたい」がほぼ起きていない**: 30日でシェアはサイト全体2件、character-personality **1件**。108人が完走し138秒読み込んでも、「これ自分だ!→人に見せたい」の行動はほぼ発生していない。
- **前提の方向修正**: 長年の設計前提「多数派は友人のシェアリンク(単独result URL)から来る」は**このデータでは成立しない**（単独result URL 30日で2PV・social流入0・着地はほぼ100%がオーガニック検索→インライン結果）。友人シェアのループは今“死んでいる”。

ここから見える急所は明確だ。**108人が2分かけて自分についての診断を完走し、138秒も読み込んでいるのに、設問を解き終えた瞬間に「自分はこれだ!」という到達・勲章・リビールの山場が無い**（cycle-274 Finding A・下記 T2 で実画面再確認）。結果は診断トップ説明文の下に淡々とインライン描画されるだけ。これは §7 が「結果面＝クライマックス／自分についての発見のごほうび／勲章感」と理想を書きながら、実画面ではその祝祭性が実装されていないこと＝**道具箱由来の抑制設計(austere)を“土台”として保つ §7 の構えが、診断のクライマックスから祝祭性を奪っている**という、本丸の構造的矛盾が来訪者価値として噴き出している一点である。

**なぜ今これが最優先の来訪者価値か**（Goal 直結の論理）:

1. site-concept の芯は「単発訪問の質＝この一回を最高にする」。108人の完走者に“ごほうびの瞬間”が無いのは、価値の中心コンテンツの一番おいしい瞬間を取りこぼしている。リビールは**シェアが増えるか以前に、この108人の“この一回”を良くする来訪者価値の底上げ**である（効果はシェア数に依存せず成立する）。
2. その上で、「これ自分だ!」の勲章の瞬間はシェア動機の源泉（確立知見 cycle-134/136/137/142）。深く読み込むのにシェア1件という漏れは、死んでいる友人ループを起こす鍵がこの瞬間にある可能性を示す（**仮説**。thin data ゆえ断定せず正直に検証する）。

## 作業計画

### 目的

本丸 B-545（根幹が道具箱前提のまま診断が継ぎ足された構造で良いか）に、Goal 起点でその具体的な急所——**価値の中心＝診断の完走者に、"自分についての発見のごほうびの瞬間"が無い**——から着手する。有界な達成点は、(1) 急所の一次接地（GA＋実画面＋競合）、(2) 根幹が来訪者価値を損ねる点の証拠マップ（矮小化しない担保）、(3) 完走者の“この一回”を高める結果リビール/クライマックスを1つ出荷（Trap A/B と分析駐車を同時に回避）、(4) 効果の正直な検証設計、(5) 本丸を開いた問いとして継続（反射的結論を出さない担保）。

### 作業内容

T0〜T9 を順に実施。T2（実画面接地）・T7（before/after）は MCP（Playwright）を使うため foreground サブエージェントで行う（CLAUDE.md）。実装（T6）はサブエージェントに委譲しトレーサビリティを保つ。設計（T5）は T2/T3 の接地を織り込んでから確定する（机上で先に固定しない）。

### この計画が罠と過去の是正を同時に回避する根拠

- **Trap A（楽な周辺を本丸と偽る）を避ける**: 出荷対象は cycle-274 が「最も価値が高い」と自認しながら先送りした最大レバー Finding A（結果リビール）そのもの。周辺の楽な作業ではない。決定原則（工数を理由に劣った UX を選ばない）に従う。
- **Trap B（未検証の設計賭けを勝利譚に）を避ける**: 出荷は「austere が正しい/壮大リデザインが正しい」という設計方向の賭けではなく、「完走者に到達の瞬間が無い」という**確立知見（これ自分だ!→満足/シェア）とデータ（完走95%・138秒読むのにシェア1件）に裏打ちされた具体欠陥**の是正。効果は仮説として明示し正直に検証する。§7 が未検証である事実も踏まえ、solved と主張しない。
- **分析駐車（cycle-273）を避ける**: 接地で止めず、1つの改善を出荷する。
- **矮小化（cycle-274 が owner に是正された過ち）を避ける**: 本丸を「本文一点」や「トップ矛盾」に縮めない。急所は根幹の症状として位置づけ、T4 の証拠マップで根幹の来訪者価値毀損を指差し、本丸は高さを保ったまま開いた問いとして継続する。
- **反射的結論（cycle-274 の事故）を避ける**: 根幹をどう据え直すかの結論は本サイクルで出さない。evidence を足し、次 PM が Goal から論理で決着させる開いた問いとして手渡す。A/B の是非も AP-P31 に従い、反射で却下しない。

### 検討した他の選択肢と判断理由

- **本丸を「根幹アーキテクチャの据え直し」の戦略決定として1サイクルで結論する**: 不採用。抽象的な設計方向の結論を1サイクルで出すのは cycle-274 の反射的決定事故の再来リスクが高く、かつ出荷ゼロの分析駐車（cycle-273）にも陥りやすい。根幹は evidence を積んで開いた問いとして扱う。
- **入口ファーストビュー改善（到達→開始の23%離脱＝§7 の入口）を出荷**: 有力だが不採用（今回は）。到達→開始離脱は実在の第2レバーだが、本丸の核（診断の“自分についての発見”のクライマックス＝結果面）からは一段遠い。結果リビール完了後の次サイクル候補として carryover。
- **シェア導線の機械的改修（B-550）**: 不採用。シェアが起きないのは導線位置ではなく結果体験に“見せたくなる価値”が未達だから（cycle-273 自己点検）。導線いじりは Trap A。リビールの下で扱う。
- **走行中 A/B の結論（B-526・7/21目安）を待ってから結果面に触る**: 不採用。引き継ぎが「Finding A の着手を B-526 の結論待ちに連結するな」と明記（回避の抜け道）。リビールは両 arm 等適用で A/B 対比を汚さず、待たずに進める。
- **B-537/B-540/B-524 等の他 P1/P2（回遊計測・メタ作業・内部リファクタ）**: 不採用。いずれも Goal 直結の本丸を先送りする理由にならない。
