# サイクル-222

cycle-220 で策定した立て直し計画のうち、Phase に属さない即時タスクである **B-487（tmp 重要知見の保全＋ docs からの tmp 参照解消）** を実施する。

cycle-220 ③-5 で「docs 配下からの tmp/ 参照多発」が問題点として記録された。`tmp/` は git 管理外であり、参照先のファイルが消えれば docs から辿れる重要知見が永久に失われ、将来サイクルでリンク切れになる（`.claude/rules/tmp-directory.md` 違反）。本サイクルは、docs が参照している tmp ファイルを棚卸しし、重要知見を永続場所（docs/）へ保全したうえで、docs 内の tmp 参照を解消する。

本タスクは独立（他タスクへの依存なし）で、立て直し本線（B-488→B-489→B-490）の前提ではないが、P0（最優先）として「知見消失という回復不能な損失を今すぐ止める」ことを目的とする。

## 実施する作業

- [ ] B-487: docs 配下からの tmp 参照を棚卸しし、重要知見を保全したうえで tmp 参照を解消する
  - [ ] docs/ 配下（backlog.md・docs/cycles/・docs/anti-patterns/・docs/research/・docs/archive/・docs/ 直下）から tmp/ への参照を grep で全件洗い出す
  - [ ] 各参照について「参照先 tmp ファイルが現存するか」「その内容に永続保全すべき重要知見があるか」を判定する
  - [ ] 重要知見がある場合は適切な docs/ 配下（research / knowledge / archive 等、内容に応じて）へ保全する
  - [ ] docs 内の tmp 参照を、保全先の docs パスへ置き換えるか、不要な参照（既に役目を終えたもの）であれば削除する
  - [ ] 保全・参照解消後、docs 配下に未解消の tmp 参照が残っていないことを grep で確認する
- [ ] 成果物を reviewer のレビューにかけ、指摘を解消する
- [ ] `npm run lint && npm run format:check && npm run test && npm run build` がすべて成功することを確認する
