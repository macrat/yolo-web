# サイクル-229

このサイクルでは、移行計画の SSoT である `docs/design-migration-plan.md` の逸脱を原設計へ訂正する（B-501）。

計画書は cycle-178/194（道具箱を Phase 10 へ後回し・順序反転）と cycle-220（1ツール n タイル廃止）で原設計から逸脱したまま正典の顔をしており、cycle-225 までの「ナビゲーションを作って生きたタイルを作らない」約50サイクルの失敗モードを支えた。cycle-226〜228 の B-497 完了により、原設計（道具箱に生きたタイルを並べる・1ツール n タイル・タイル＝詳細ページと道具箱の同一実装）が**動いた実物**（34/34 ツール・道具箱39枚・実機 Playwright 検証済み）で実証されたので、それに設計を合わせる形で計画書を訂正する。

訂正の歯止めは「承認」ではなく〔原典 site-concept.md へ戻す向き＋動いた実物＋fresh reviewer による原典忠実性の敵対的検証〕とする（机上で逸脱を正典化した原罪と、向き・根拠が逆になるように）。

同一文書の訂正タスクとして、B-492（B-490 で撤去済みの tile-definition.ts／tile-declarations.ts／/internal/tiles を現行/予定構成として記述した箇所の是正）と B-365（Phase 7.1/7.2 の数値不一致 30→34 ルート・13→20 ルートの訂正）も本サイクルで一括対応する。
