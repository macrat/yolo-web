### 重要な是正: design-gate の glob 空振り（機械検査が効いていなかった）

design-gate の `src/app/(new)/**` や `[param]` を含む glob が fast-glob の extglob 解釈で**0件マッチ**しており、トップ`/`・辞典トップ等の `(new)/` 配下が**実は機械検査されていなかった**（＝それまでの「gate緑」は空振りの緑）。f090d209 で paren escape＋`[param]`→`*` に修正し、**空振り検出テスト**（各globが最低1件マッチしないと落ちる）を追加。現在20 CSSファイルを実検査してERROR 0。教訓: glob ベースのゲートは「マッチ0件で緑」になりうる——空振り検出をゲート自身に組み込む。トップ等は視覚検証済みのため実害はなかったが、機械担保は今回から有効。

### 学び: 共有サブシステムは面ごとに並行変換できない（fan-out戦略の是正）

C2で辞典一覧5面をworkflowで並行変換した際、各エージェントを「自分の2ファイルだけ」に閉じたため、**辞典が共有する検索/閲覧の器（`SearchBox`/`DictionaryGrid`/`DictionaryCard`/`CategoryNav`/各`*IndexClient`）が分断**された。結果: kanji/yojiで検索UXが脱落・colorsは旧検索クライアントを温存・`KanjiIndexClient`/`YojiIndexClient`が孤児化・ファセット面(radical/grade/stroke/category)は未変換のまま。**是正方針**: 共有サブシステム（辞典の検索/閲覧・ツールのToolLayout・quizエンジン・ブログ一覧/ページネーション等）は、**共有の器を先に店構えへ変換してから、それを使う面を展開する**。面単位のfan-outはサブシステムを共有しない独立ページに限る。

### 学び: レシピの機械適用は「規約準拠だが平板」を生む（顔・見せたくなる結果面は手わざが要る）

トップv1は§4「一覧＝品書き」を機械適用し、4つのほぼ同一な罫区切りリストの羅列＝「色を替えただけの索引」になった（オーナー指摘で作り直し）。**規約遵守（design-gate緑・§準拠）は品質の下限であって、良いデザインの証明ではない**。とくに (a) サイトの顔（トップ）、(b) 見せたくなる結果面（診断・占い・ゲームの結果＝C4）は、焦点・組版の律動・成果物を"見せる"craft・視覚反復が要る。機械変換で済ませてよいのは、独立した実直な面（辞典詳細・実務ツール等）に限る。**C4の結果面は最も手をかける面**として扱う（包み/札/印/和色は構築済み・実結果で焦点と持ち帰りを立てる）。

### 検討した他の選択肢と判断理由

**ツールのプルーニングを撤回した判断(PM判断・kickoff時)**: rebuild-plan §2 R-1 は cycle-278 で PM が「SC表示数<閾値のツールを削除」と導出していた(§0のオーナー裁定には無く PM 導出)。これを kickoff で再検討し**撤回**した。根拠:

- (1) **論理の破綻**: SC表示数による一律削除は「最高の価値に達してなお構造的に負けた(削除妥当)」と「そもそも最高品質へ磨いていない=我々の怠慢(削除は不当)」を区別できない。constitution rule 4(品質優先)に照らせば、磨く努力をしていない段階での「外れ」判定に根拠が無い。
- (2) **実証**: 各ツールの作成日(`git log --follow`)と改変履歴を点検。33本は2/13〜28作成(4.5ヶ月)だが、percent-calculatorは6/19作成(3週間=評価窓未経過→一律削除は遅咲きの誤殺)。さらに各ツールへの「努力」の実体は legacy→new移設・トークン移行・cycle-160 ToolLayout一律再設計・cycle-225/228タイル化・trustLevel撤去等の**リポジトリ横断の機械的churn**であり、競合に勝つための一件ごとの品質改善ではなかった=中身を数ヶ月放置した。日数の経過そのものは評価窓の成立条件ではない。
- (3) **ゴール逸脱**: 削除を正当化する一件ずつの品質・競合調査+最高価値への磨き込みの費用は、共有ToolLayoutに載る全ツールを一律移行する費用を遥かに上回り、**本サイクルのゴール(デザイン移行)から外れる**。
- → 全ツールを新デザインへ一律移行し、コンテンツ削除はしない。ツールの処遇(改善orプルーニング)は「最高価値に磨いてから判断する」専用の取り組み(B-566)へ分離。

**B-521(辞書CTR検証)の結果**: cycle-251の漢字CTR施策(部首・画数の前置、6/18デプロイ)に帰属できるCTR改善は検出されず。狙った「部首」クエリは表示7倍化(394→2,835)なのにクリックは前後ともゼロ。漢字辞典全体は直近28日9,866imp/8clk=CTR0.08%(参照型の敗北兆候TW-A)。→ ADR001に記録し、辞典の処遇(B-564・フェーズC)の材料とする。辞典は撤去対象外なので本サイクルは現状のまま新デザインへ移行する。
