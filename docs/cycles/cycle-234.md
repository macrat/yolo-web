# サイクル-234

このサイクルでは、道具箱（トップ `/`）とツールタイルの利用計測（GA4 カスタムイベント）を整備する。

cycle-232 でトップを道具箱に置き換えて本公開したが、kickoff 時の実測で「道具箱の利用実態を観測する手段が存在しない」ことを確認した。タイルの追加・削除・リセット・プリセット選択・タイルの実利用は、いずれも GA に一切記録されていない（page_view と滞在時間しか見えない）。この観測空白を放置すると、(1) Phase 10.3「本公開」の成否を判定できない、(2) Phase 10.4（ツール間連携 B-324）/ 10.5（シェア B-313）の着手価値を判断できない、(3) B-448 等の「GA 観測駆動」backlog 項目が永遠にトリガーされない。公開直後の利用実態は今しか観測できないため、最優先で計測を整備する。

## 作業計画

### 目的

来訪者が道具箱をどう使っているか（使っていないか）を観測可能にする。計測そのものは来訪者への直接価値ではないが、constitution の goal（来訪者への最高の価値→PV）に対し「観測なしの機能投資は来訪者価値を外す」という構造で接続する（AP-P07 自覚済み）: 検索流入の主力は個別コンテンツ直行（cycle-232 実測で約 79%）であり、道具箱を使い込む人が実在するかが見えないまま連携・シェア機能を積むのは、救う相手がいるか不明な投資になる。まず見えるようにする。

### 作業内容

イベント設計の素案（確定は T-1。reviewer レビューで変更可）:

- `toolbox_tile_add` { item_id } — 「タイルを追加」パネルからの追加
- `toolbox_tile_remove` { item_id } — 「外す」操作
- `toolbox_reset` — リセット操作
- `toolbox_preset_select` { preset_id } — プリセット選択（インライン確認を経た適用のみ。確認キャンセルを別イベントにするかは T-1 で判断）
- `tile_first_interaction` { item_id, surface: "toolbox" | "detail" } — タイル内の最初のポインタ／キーボード操作をマウントごと 1 回だけ送信

プライバシー方針: イベントパラメータは操作種別・item_id（ツール slug）・preset_id のみ。**タイルへの入力内容・出力内容は一切送らない**（coding-rules「ユーザーを危険にさらさない」）。

実装方式: 既存の `src/lib/analytics.ts`（sendGaEvent + track 関数 + `__tests__`）のパターンに従う。タイル実利用は共通ラッパーでのイベント捕捉（capture phase）で実現し、34 ツールのタイル実装には手を入れない。ToolPageLayout はサーバーコンポーネントの可能性があるため、詳細ページ側は小さなクライアント境界コンポーネントの追加が必要になる見込み（実装時に確定）。
