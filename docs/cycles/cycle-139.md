# サイクル-139

静的結果ページ（`/play/[slug]/result/[resultId]/page.tsx`）のシェアボタンが、detailedContentの下（fold下約1253px）に配置されており、ユーザーが気づかない可能性が高い。シェアボタンを説明文の直後（detailedContentの前）に移動し、結果を見た直後の感情ピークでシェアを促す。

## 実施する作業

- [ ] 1. 静的結果ページのシェアボタン配置変更（page.tsx + page.module.css）
- [ ] 2. Playwright目視テスト（detailedContentあり/なし両パターン）
- [ ] 3. 既存テストの確認・修正（テストがある場合）

## 作業計画

### 目的

**誰のために**: 「手軽で面白い占い・診断を楽しみたい人」。結果をSNSでシェアしたいが、シェアボタンが見つからずにそのまま離脱しているユーザー。

**何のために**: シェアボタンをfold内（結果の感情ピーク直後）に移動し、シェア率を向上させる。NNGの調査によればユーザーはページ閲覧時間の57%をfold上に費やし、シェア意欲は結果を見た直後にピークを迎える。適切な配置改善でシェア率3-5倍の効果実績がある。

**提供する価値**:

- 結果を見た感情の高まりのタイミングでシェアボタンが目に入る
- シェアのためにページを下までスクロールする手間がなくなる
- detailedContentを読む前にシェアの選択肢が提示されることで、シェア→読み進めるという自然な導線が生まれる

### 変更するもの

| ファイル                                                | 変更内容                                                                                           |
| ------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `src/app/play/[slug]/result/[resultId]/page.tsx`        | シェアボタンのJSXをdetailedContentの前に移動。shareSectionのborder-topを削除し、自然な見た目にする |
| `src/app/play/[slug]/result/[resultId]/page.module.css` | `.shareSection`のスタイル調整（border-top削除、margin調整）                                        |

### 変更しないもの

以下は本サイクルでは一切変更しない:

- `ShareButtons.tsx` / `ShareButtons.module.css` — ボタンコンポーネント自体のデザインやサイズは変更不要。現在のpadding `0.625rem 1.25rem`（高さ約40px以上）はタップターゲット基準（44x44px）をほぼ満たしており、CTAボタンより一回り小さいサイズが維持されている
- `ResultCard.tsx` — 動的ResultCardは既にシェアボタンが説明文直後にあるため変更不要
- `ResultModal`関連 — モーダルなのでfold問題は該当しない
- detailedContent関連のデータ・コンポーネント — 表示内容自体は変更しない

### レイアウト変更の詳細

**現在のレイアウト順序:**

1. パンくずリスト
2. アイコン + タイトル(h1) + クイズ名 + 説明文
3. detailedContent（特徴リスト、あるある、アドバイス）← 長い
4. CTAボタン（「診断してみよう」）+ コスト情報
5. **シェアボタン** ← fold下約1253px
6. 相性表示
7. 関連クイズ
8. おすすめコンテンツ

**変更後のレイアウト順序:**

1. パンくずリスト
2. アイコン + タイトル(h1) + クイズ名 + 説明文
3. **シェアボタン** ← ここに移動（感情ピーク直後）
4. detailedContent（特徴リスト、あるある、アドバイス）
5. CTAボタン（「診断してみよう」）+ コスト情報
6. 相性表示
7. 関連クイズ
8. おすすめコンテンツ

### 作業内容

#### 作業1: 静的結果ページのシェアボタン配置変更

**page.tsx の変更:**

- `<div className={styles.shareSection}>` + `<ShareButtons .../>` のブロックを、`description` の直後、`detailedContent` の前に移動する
- 元の位置（CTAボタンの後）からは削除する

**page.module.css の変更:**

- `.shareSection` から `border-top: 1px solid var(--color-border)` を削除する（説明文直後に区切り線は不要）
- `.shareSection` の `margin-top` と `padding-top` を調整する。説明文の `margin-bottom: 1.5rem` があるので、shareSection側は `margin-top: 0` 程度で十分
- detailedContentがある場合にシェアボタンとdetailedContentの間に適切な余白を確保する（shareSectionの `margin-bottom` を設定）

#### 作業2: Playwright目視テスト

以下のパターンを目視確認する:

- **detailedContentありのページ**（例: `/play/music-personality/result/rhythmic-groove`）: シェアボタンが説明文直後、detailedContentの前に表示されること
- **detailedContentなしのページ**（例: detailedContentがない結果ページがあれば）: シェアボタンが説明文直後、CTAボタンの前に表示されること
- ダークモードでの表示確認

#### 作業3: 既存テストの確認・修正

- `src/app/play/[slug]/result/[resultId]/` 配下にテストがあれば、レイアウト変更に伴う修正が必要か確認する
- テストがない場合は、本サイクルでは新規テスト作成は行わない（レイアウト変更のみのため）
