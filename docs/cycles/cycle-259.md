# サイクル-259（コンセプト再センタリングの実行：site-concept改訂と下流再編）

cycle-257 で確定した戦略判定 **B-535「道具箱-as-core を降ろし、診断（と文化系コンテンツ）中心へ再センタリングする」**（reviewer 2名承認）を、文書と backlog に落とし込んで実行する。

判定そのものは確定済みであり、本サイクルでそれを再議論はしない。本サイクルがやるのは「決定の実装」——ただしビルドではなく、**サイトの設計を規定する一次文書（`site-concept.md`）の改訂と、それに連なる下流タスクの再編**である。

## なぜ今これをやるのか

`site-concept.md` は現在こう書いている（L9, L26）:

- 「あくまで『道具』がコアであり、息抜きは世界観を補完する要素である。息抜きコンテンツの割合はコンテンツ全体の1割を超えない。」
- 「既存の診断・占いコンテンツは将来的に整理する。毎日やるものではなく、サイトコンセプト『日常の傍にある道具』と合致しないためである。」

これは cycle-257 の確定判定（来訪者価値は診断 `/play` に頑健に集中・全期間84日で集客と滞在の両軸1位／`/`道具箱は organic 集客ほぼ0・中核機能利用ゼロ）の**真逆**である。最も価値が届いている診断を「整理対象の息抜き1割以下」と規定する文書が、サイトの一次設計文書として生き続けている。

`site-concept.md` はサブエージェント・将来の全計画が参照する一次文書だ。辞典の「80字」誤情報（B-536）と同型の**内部の確定誤情報**であり、CLAUDE.md が警告する「未検証情報がサブエージェントに渡り将来サイクルを汚染する」状態を、最上位の文書で起こしている。約50サイクルにわたり未計測の前提が固定され続けた（cycle-257 A-1）のと同じ構造を、放置すれば再生産する。だから改訂が要石であり、急ぐ。

## 作業計画

### 目的

確定した戦略判定（診断中心への再センタリング）を、サイトの一次設計文書とタスク体系に正しく反映し、「道具箱が核」という今や偽の枠組みがサブエージェント・将来計画を誤導し続ける状態を解消する。来訪者に直接見える変更（デザイン・導線）は本サイクルでは行わない——それらは下流タスク（B-539 はA/B計測駆動で B-526 のデータ読み 2026-07-21 待ち、B-537 は 06-25 追計測後）として、データに基づいて順に実行する。本サイクルは「中心を正しく書き直し、次に何を作るかの順序を整える」段。

### 作業内容

1. **AP-P27 点検 → 一次根拠の読み直し** を先に行い、改訂の事実基盤と「宙づり実害なし」を確認する。
2. **`site-concept.md` 改訂**を本サイクルの中心に置く。これは高判断のPM中心作業だが、改訂案のドラフトと整合監査はサブエージェントに分担させ、PM が判定との整合を確かめて確定する。
3. **トップ `/` 位置づけ**と**下流再編**を、改訂と同じ判定軸で行う。
4. **他文書波及点検**で、最上位文書だけ直して下位文書に矛盾を残す事態を防ぐ。
5. **白紙 reviewer 再レビュー**で確定。

### 検討した他の選択肢と判断理由

- **B-540（AP集 規約準拠クリーンアップ・P1キャリーオーバー）を先にやる**: 不採用。B-540 はプロセス文書の整備で重要だが来訪者価値の連鎖からは遠く、`site-concept.md` の確定誤情報を放置すると本サイクル以降の全計画が誤導され続ける緊急性に劣る。B-540 は次サイクル以降で単独実施する（無関係な作業を束ねない＝タスクを小さく保つ）。
- **B-538 と B-539（DESIGN.md 再適合）を一体で同サイクル実施する**: 不採用。cycle-257 は「B-538のsite-concept改訂と一体で扱う」と書くが、B-539 の核は走行中A/B `quiz_result_visual_v1`（旧カラフル/絵文字 vs 新ミニマル）の一次証拠に基づくデザイン再適合で、その結果読み（B-526）は 2026-07-21 目安。データ未着のいま DESIGN.md をブラインド改訂するのは「未計測の前提の上に作業を積む」cycle-257 A-1 の再演になる。本サイクルでは B-539 の位置づけ・順序を下流再編で確定するに留め、実作業はデータ着後に回す。
- **site-concept 改訂と同時にコード（トップ`/`・導線）も変える**: 不採用。判定の確定根拠は計測だが、具体的な新トップ形態・回遊強化は「単発訪問の質を損なわず付加価値を足すか／回遊数値を上げる導線か」の境界判断（AP-I04 / cycle-256 ダークパターン撤回の教訓）を要し、A/B・追計測で確かめるべきもの。文書で中心と方向を固め、実装は計測駆動で別サイクルに分ける。
- **判定自体を本サイクルで再検討する**: 不採用。判定は cycle-257 で reviewer 2名承認により確定済。本サイクルは実行であって再議論ではない（蒸し返しは AP-WF24 的な堂々巡りを生む）。新たな反証データが出たら別途扱う。
