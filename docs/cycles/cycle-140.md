## 作業計画

### 目的

静的結果ページ（`/play/[slug]/result/[resultId]`）を「第三者来訪者にとっての最高の価値」を提供するランディングページとして再設計・実装する。現在のページは「クイズ受験者本人」向けのコンテンツ構成になっており、実際の来訪者である「シェアリンクや検索から来た第三者」に対して適切な体験を提供できていない。この根本的な設計ミスを解消し、第三者が「この診断は面白そうだ、自分もやってみたい」と感じてCTAをクリックする確率を最大化する。

### 設計判断

#### 判断1: 共通テンプレート vs コンテンツ別設計

**結論: 共通テンプレートを維持し、コンテンツ固有の「見出し」「セクション構成」をデータ駆動でカスタマイズする方式を採用する。**

理由:

- 調査レポートの競合分析で、成功サービスの多くが「共通テンプレート＋コンテンツ固有のビジュアル要素」を採用している（16Personalitiesはタイプ別に色・アイコンを変えるが構造は共通、Kuizyは完全共通テンプレート）
- yolos.netは既にアクセントカラー・アイコン絵文字・OGP画像でコンテンツ差別化を行っており、これらは有効に機能している
- 15種のクイズそれぞれに完全専用デザインを作ると保守コストが極めて高くなり、品質の一貫性も維持困難になる
- ただし「あなたの特徴」「こんなところ、ありませんか?」という全コンテンツ共通の見出しは、コンテンツの世界観を破壊しているため、コンテンツの特性に合わせた見出しをデータ側から指定可能にする

具体的な差別化方針:

- detailedContentセクションの見出しテキスト（traits/behaviors/advice）を各クイズのデータで指定できるようにする（`QuizMeta`または`QuizResultDetailedContent`にオプショナルフィールド追加）
- 指定がない場合のデフォルト値は第三者向けに書き直した汎用的な見出しを使う
- accentColor、icon、OGP画像による差別化は現状のまま維持

#### 判断2: ページ構成の再設計（情報アーキテクチャ）

**結論: fold上にコンテキスト→結果→CTAの最短経路を実現し、detailedContentはCTAの後に配置する。CTAはダブル配置（detailedContent前後）とする。**

新しいページ構成（上から下）:

```
[Breadcrumb]
[コンテキスト: クイズ名 + shortDescription（1行）]     ← 新規追加
[結果ビジュアル: icon + h1(結果タイトル)]
[結果説明: description（200-270文字）]
[CTA1: ボタン + "全N問 / 登録不要"]                    ← fold上に移動
[detailedContent: traits + behaviors + advice]          ← CTA1の後に配置
[CTA2: ボタン（detailedContentを読んだ人向け）]        ← 新規追加
[ShareButtons]
[CompatibilityDisplay（?with=パラメータ時のみ）]
[RelatedQuizzes]
[RecommendedContent]
```

根拠:

- 調査で第三者来訪者のスクロール深度は25-35%（モバイルで約200px分）と判明。CTAがfold上にないと大部分の来訪者に見せられない
- fold上CTAはfold下比で304%のパフォーマンス差（Landing Page Flow調査）
- detailedContentは読み物としての価値があり、検索からの来訪者のSEO価値もあるため削除はしない。ただしCTAより後に配置して、第三者の最短経路を阻害しないようにする
- detailedContent読了後の第2CTAで、コンテンツを読み込んだ高関与ユーザーの取りこぼしを防ぐ

**モバイル(375x812)でのfold上収まりの検証と対応方針:**

CTA1の上に配置される要素の高さを見積もる（375px幅、現在のCSS基準）:

- wrapper padding-top: 16px
- Breadcrumb: 約32px（フォントサイズ0.75rem + padding）
- card padding-top: 32px
- コンテキスト（クイズ名 + shortDescription 1行）: 約40px
- icon（3rem = 48px）+ margin: 約56px
- h1（1.75rem、1-2行）: 約50-80px
- description（200-270文字、1rem、line-height 1.7、375px幅で約10-14行）: 約270-380px
- CTA1ボタン + tryCost: 約70px

合計見積もり: 566-706px。foldライン812pxに対して最良ケースで約250pxの余裕があるが、descriptionが長い場合（270文字、14行相当で380px）は合計706pxとなりギリギリになる。

**対応方針: descriptionの表示を最大4行に制限し、「続きを読む」で展開する方式を採用する。**

理由:

- descriptionを4行（約108px）に制限すれば、合計は約404-434pxとなりCTA1が確実にfold上に収まる
- 第三者にとってdescription全文よりもCTAの視認性が重要。description全文はタップで展開できればよい
- 4行あれば結果の概要は十分に伝わる（日本語で約80文字相当）
- CSS `line-clamp` で実装可能。展開はクライアントコンポーネントにする必要があるが、descriptionの展開/折りたたみだけなので軽量

実装時にPlaywright（Step 4）で実機確認し、4行で不十分であれば5行に調整する。

#### 判断3: 主語問題（第三者向けテキスト）の解決

**結論: 見出しのデータ駆動化に加え、adviceセクションに「このタイプの人へのアドバイス」という見出しを付与して第三者視点のフレーミングを行う。シェアテキストに問いかけを追加する。**

現状の問題:

- 「あなたの特徴」→ 第三者にとって「あなた」は友人の結果であり、「あなたの」と呼ばれても「私の話じゃない」と混乱する
- 「こんなところ、ありませんか?」→ 同様に第三者への問いかけとして不自然
- **advice文は一貫して「あなた」への直接的な語りかけになっている**（例: 「あなたは計画している時間もちゃんと楽しめる人だから」「今日あなたが「これいい」と思った曲を、まず一人だけに送ってみて」）。traits/behaviors本文にも「あなた」が散在（detailedContent内で「あなた」を含む行が主要5ファイルだけで合計66行）

セクション別の問題の深刻度と対応方針:

- **traits/behaviors**: 本文は概ね観察的記述。見出しを「このタイプの特徴」「このタイプのあるある」に変えれば第三者が自然に読めるレベル。本文中の散在する「あなた」は見出しのフレーミングでカバーできる範囲
- **advice**: 本文が一貫して二人称の語りかけ（「〜してみて」「あなたは〜」）であり、見出し変更だけでは不十分。ただし、advice文は各結果タイプにつき1-2文と短いため、「このタイプの人へのアドバイス」という明示的な見出しを付ければ、第三者は「これは本人向けのアドバイスなんだな」と理解した上で読める

**advice文の対応として選択肢Aを採用する（見出しによるフレーミング）。理由:**

- 選択肢A（フレーミング）: advice文に「このタイプの人へのアドバイス」という見出しを付け、第三者が「本人向けの文章である」と理解できるようにする。第三者は友人の結果を見に来ているので、「友人のタイプにはこういうアドバイスがあるのか」という読み方は自然である
- 選択肢B（adviceを非表示）: adviceはSEOコンテンツの一部であり、また読み物として面白みがある。非表示にするとコンテンツの価値が下がる。不採用
- 選択肢C（三人称に書き換え）: 根本解決だが、advice文は「あなた」への語りかけだからこそ親密さや温かみがある。三人称にすると「このタイプの人は〜するとよいでしょう」のような冷淡な文体になり、コンテンツとしての魅力が低下する。また、第三者が「自分もこのタイプかも」と思ったとき、二人称の語りかけがそのまま自分ごとになるという効果もある。来訪者価値の観点からフレーミング（選択肢A）の方が優れている

解決策:

- `QuizMeta`に`resultPageLabels`（オプショナル）を追加。各クイズが自身のコンセプトに合った見出しを設定可能にする
  - `traitsHeading`: traits セクションの見出し（例: 音楽性格診断→「このタイプの音楽との向き合い方」、動物性格診断→「このタイプの特徴」）
  - `behaviorsHeading`: behaviors セクションの見出し（例: 「日常のワンシーン」「このタイプのあるある」）
  - `adviceHeading`: advice セクションの見出し（例: 「このタイプの人へのアドバイス」）
- デフォルト値（`resultPageLabels`未設定時）は第三者向けの中立的な表現にする:
  - traits: 「このタイプの特徴」
  - behaviors: 「このタイプのあるある」
  - advice: 「このタイプの人へのアドバイス」
- **adviceセクションにはデフォルトで見出しを表示する**（現在は見出しなしのカードだが、第三者向けフレーミングとして見出しが必須）
- シェアテキスト: 末尾に「あなたは?」を追加 → `${quiz.meta.title}の結果は「${result.title}」でした！あなたは? #${...}`

#### 判断4: 「同じクイズの他の結果タイプ一覧」の表示について

**結論: 今回のサイクルでは不採用とし、次サイクル以降の検討事項とする。**

理由:

- 調査レポートは「中優先（できれば実施）」として挙げているが、これは単体で独立した改善項目であり、今回の「第三者視点への再設計」の核心ではない
- 他の結果タイプへのリンクは、UIデザインの検討が必要（一覧表示の形式、リンク先が同じ静的結果ページ同士の回遊になる、クイズ受験のCTAとの注意の奪い合いなど）。CTA最適化と同時に入れると、効果測定が困難になる
- 現時点での結果ページのトラフィックが極めて少ない（3月で2ページ各3PVのみ）。まずはCTA配置・コンテキスト・フレーミングという基盤改善を行い、トラフィックが増えた段階で他タイプ一覧の効果を検証する方が合理的
- キャリーオーバーに記載し、次サイクル以降で検討する

#### 判断5: 社会的証明（受検者数等）の表示について

**結論: 現時点ではデータがないため不採用。**

理由:

- yolos.netはサーバーサイドのデータベースやユーザー管理を持たない（CLAUDE.mdのコーディング原則2「外部APIの呼び出し、データベース、認証・ユーザー管理は実装しない」）
- 受検者数やタイプ比率をリアルタイムで表示するには、何らかのデータ集計基盤が必要だが、現在のアーキテクチャ（静的サイト生成）では実現できない
- GA4のデータから手動で「累計N人が受検」のような固定値を埋め込むことは技術的に可能だが、現時点のトラフィック規模（クイズページ全体で月10-20PV程度）では、社会的証明として機能する数値にならない
- トラフィックが十分に成長した段階で改めて検討する

#### 判断6: コンテキスト情報の追加

**結論: クイズのshortDescriptionをページ上部に表示し、第三者に「このコンテンツが何であるか」を即座に伝える。**

現状のquizName（「○○の結果」）だけでは、第三者は「何問の診断か」「どんな軸で診るのか」が分からない。quiz.meta.shortDescriptionは既に各クイズに設定済みであり、これをquizNameの下（またはquizNameに代えて）表示することで、第三者のコンテキスト理解を3秒以内にする。

### 検討した他の選択肢と判断理由

#### 選択肢A: コンテンツ別に完全に異なるテンプレートを作成する

- 15種のクイズそれぞれに専用のpage.tsxやCSSを用意する
- **不採用理由**: 保守コストが極めて高く、品質の一貫性が保てない。調査で成功サービスの多くが共通テンプレート＋カスタマイズ要素の方式を採用していることが確認された。コンテンツ固有の差別化はデータ駆動の見出しカスタマイズとaccentColor/iconで十分な効果が得られる

#### 選択肢B: detailedContentを完全に削除し、短い結果説明＋CTAのみにする

- ページを極限まで軽量化し、CTAへの最短経路だけを提供
- **不採用理由**: detailedContentはSEOインデックス対象（79ページがindex:true）であり、検索からの来訪者にとっての価値がある。また、読み物としての面白さは「自分もやってみたい」動機を強化する効果があり、削除は来訪者価値の低下につながる

#### 選択肢C: detailedContentの内容自体を第三者向けに全面書き換えする

- 「あなたの特徴」→「このタイプの人は...」のように本文テキストも三人称に変更
- **不採用理由**: detailedContentは各クイズの全結果タイプ（計100件以上）に設定されており、本文を全面書き換えすると膨大な作業量になる。実際の本文を確認したところ、traits/behaviorsは概ね観察的記述で見出し変更によるフレーミングでカバーできる範囲だが、advice文は一貫して「あなた」への直接的な語りかけになっている（「〜してみて」「あなたは〜だから」等）。しかしadvice文の二人称表現はコンテンツとしての親密さと温かみの源泉であり、三人称に書き換えると冷淡な文体になりコンテンツの魅力が低下する。第三者が「自分もこのタイプかも」と思ったとき、二人称の語りかけがそのまま自分ごとになるという効果もある。そのため、本文の書き換えではなく「このタイプの人へのアドバイス」という見出しによるフレーミングで、第三者に「これは本人向けの文章である」と理解させるアプローチを採用する（判断3参照）

#### 選択肢D: CTAをfold上に固定表示（sticky）にする

- スクロールしても常に画面下部にCTAボタンが追従する
- **不採用理由**: 調査でダークパターン研究の知見として「操作を強いられていると感じると信頼感が低下する」ことが確認されている。stickyなCTAは広告と混同されやすく、来訪者の信頼を損なうリスクがある。CTAのダブル配置（fold上＋detailedContent後）で十分にカバーできる

#### 指摘1（重大）: 選択肢Cの不採用理由が事実に反しており、主語問題の解決策が不十分

計画の判断3および選択肢Cの不採用理由に、以下の記述がある:

> 本文自体は「このタイプの人はこういう傾向がある」という観察的な記述であり、見出しを変更すれば第三者が自然に読めるコンテキストになる。見出しの変更で「このタイプの特徴を第三者として観察している」というフレーミングを与えれば、本文の「あなた」表記も「このタイプのあなた」として読める

**この主張は事実に反している。** 実際のdetailedContent本文を確認したところ、traitsとbehaviorsの文章の多くは観察的記述（三人称的）だが、advice文は一貫して「あなた」への直接的な語りかけになっている。例:

- 「次に行きたいライブの候補を今すぐ3つ書き出してみて。**あなた**は計画している時間もちゃんと楽しめる人だから。」（music-personality）
- 「今日**あなた**が「これいい」と思った曲を、まず一人だけに送ってみて。」（music-personality）
- 「最近「面白そう」と思ったけど踏み出せていないことに、今週一歩だけ近づいてみて。**あなた**の行動力は周りの人にも勇気を与えている。」（traditional-color）

また、traits/behaviors本文にも「あなた」が散在している:

- 「自分の好きなものを誰かと共有する行為そのものが、**あなた**にとってのエネルギー補給になっている。」（music-personality traits）

定量的に確認したところ、detailedContentを持つ主要5ファイルだけでも、detailedContent内に「あなた」を含む行が合計66行存在する。

cycle-139の教訓「根拠のない判断をしない」に明確に違反している。「見出しを変えれば本文も自然に読める」という主張は、本文を実際に確認していれば出てこない結論である。

**対応方針として検討すべきこと**: adviceセクションは第三者にとって「クイズ受験者本人へのアドバイス」であり、見出しの変更だけでは主語問題を解消できない。少なくとも以下のいずれかの対応が必要:

- (a) adviceセクションの見出しを「このタイプへのアドバイス」のように明示し、「本人に向けた文章である」ことを第三者に理解させるフレーミングにする（最小コスト案）
- (b) adviceセクションの本文を三人称に書き換える（100件以上あるが、advice文は各1-2文なので作業量はtraits/behaviorsの書き換えより遥かに少ない）
- (c) traits/behaviors本文中の「あなた」も含めて書き換える（最大コスト案）

いずれを選ぶにせよ、「見出しだけ変えれば自然に読める」という根拠のない主張を計画の前提にしてはならない。実際の本文を確認した上で、どのレベルの対応が必要かを判断し直すこと。

#### 前回指摘への対応確認

**指摘1（重大）: advice文の主語問題** --- 対応済み

修正後の判断3は、advice文が一貫して二人称の語りかけであるという事実を正しく認識した上で、「このタイプの人へのアドバイス」という見出しによるフレーミング（選択肢A）を採用している。選択肢B（非表示）・選択肢C（三人称書き換え）の不採用理由も、前回の「見出しだけ変えれば自然に読める」という根拠のない主張ではなく、コンテンツの魅力（親密さ・温かみ）の維持や、第三者が「自分もこのタイプかも」と感じたときの効果という観点から論じられており、妥当である。

advice文の実例（「次に行きたいライブの候補を今すぐ3つ書き出してみて。あなたは計画している時間もちゃんと楽しめる人だから。」等）を確認したが、「このタイプの人へのアドバイス」という見出しがあれば、第三者は「これは診断結果として本人に送られたメッセージだ」と理解できる。診断サービスにおいて「タイプ別アドバイス」は一般的な構成要素であり、見出しによる文脈付与で違和感は解消される。

**指摘2（中）: 調査改善案の取捨選択が不透明** --- 対応済み

判断4（他の結果タイプ一覧）と判断5（社会的証明）が新設され、不採用理由が明記された。いずれも合理的である。特に判断5のアーキテクチャ制約（CLAUDE.mdのコーディング原則2に基づくDB不使用）の言及は適切。

**指摘3（中）: CTA1のモバイルfold上収まり検証不足** --- 対応済み

CSSからの高さ積算（566-706px）と、descriptionの4行制限による対応方針（合計404-434px、fold上に約380pxの余裕）が追加された。line-clamp + 「続きを読む」展開の方式は技術的に妥当であり、Step 4のPlaywrightテストで実機確認する計画も維持されている。

#### 計画全体の再確認

**来訪者価値の最大化**: 第三者来訪者の思考フロー（「何の診断か理解 -> 結果を読む -> 自分もやりたい」）に沿ったページ構成（コンテキスト -> 結果 -> CTA1 -> detailedContent -> CTA2）になっている。CTA配置、コンテキスト追加、見出しのデータ駆動化、adviceのフレーミングの4点はいずれも来訪者視点から出発した改善であり、constitutionの「来訪者にとっての最高の価値」に合致している。

**descriptionの4行制限について**: 来訪者にとって「途中で切れる」体験は理想的ではないが、「CTA1がfold上に確実に収まる」ことの方が来訪者価値が高い（fold上CTA vs fold下CTAで304%のパフォーマンス差）。4行あれば結果の概要は伝わり、興味を持った来訪者は「続きを読む」で展開できるため、許容範囲の妥協である。

**共通テンプレートの妥当性**: 判断1で競合分析に基づく共通テンプレート維持の理由が示されており、見出しのデータ駆動カスタマイズとaccentColor/iconによる差別化を組み合わせる方針は妥当。各コンテンツの固有の魅力は見出しテキストで表現可能（例: 音楽性格診断 -> 「このタイプの音楽との向き合い方」）。

**cycle-139の教訓の遵守**: 「来訪者視点から出発する」（判断2の根拠がすべて来訪者行動データに基づく）、「根拠のない判断をしない」（前回指摘で問題だったadvice文の実態認識が修正済み）、「多角的に検討する」（6つの判断 + 4つの不採用選択肢を検討）のいずれも守られている。
