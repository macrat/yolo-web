## 作業計画

### 目指す姿

静的結果ページ（`/play/[slug]/result/[resultId]`）は、「シェアリンクや検索から第三者が訪れるカジュアル診断の結果ページ」である。このページの来訪者は100%が第三者（シェアリンク経由の友人、または検索経由の来訪者）であり、クイズ受験者本人は見ない（本人は `/play/[slug]` 上の動的ResultCardで結果を確認する）。

ページの性質は、Truity・CrystalKnowsのような「汎用タイプ解説ページ」（不特定多数が読む図鑑的コンテンツ）とは異なり、ホイミー・MIRRORZのような「受験者のシェアを起点に第三者も閲覧する結果ページ」に近い。ただし、来訪者が100%第三者である以上、第三者にとっての読みやすさと価値提供を最優先に設計する必要がある。

yolos.netはカジュアルなエンタメ診断サイトであるため、16PersonalitiesやTruityのような学術的深さは不要だが、来訪者に最高の価値を提供しページビューを最大化するという目標に照らして、以下のゴールの姿を定義する。

#### ゴール1: 第三者が「このタイプ」を素早く理解できる

調査レポートの事実:

- Truityは「Key Facts About INTJ」（箇条書き5項目）をページ上部に配置し、タイプの本質を端的に示している
- CrystalKnowsは「At a Glance」カード（Focus / Style / Strength / Approach）で一目でタイプを把握できるようにしている
- ホイミーは本文300-400字 + 特徴箇条書き3項目という短さでテンポ感を重視している

yolos.netの目指す姿: 第三者がページを開いた瞬間に「何の診断のどんなタイプか」を理解でき、タイプの特徴を箇条書きで素早く把握できる状態。

#### ゴール2: 第三者が「自分もやってみたい」と感じてCTAをクリックする

調査レポートの事実:

- ホイミーは「この診断をやってみる」ボタンを結果タイトルの直下（ページ上部）に配置している
- CrystalKnowsはページ冒頭（ヒーローエリア）と最下部の2カ所にCTAを配置し、社会的証明（"1M+ people have discovered their type"）を付している
- CrystalKnowsはCTA近くに「Free assessment / No credit card required / Results in 5 minutes」という摩擦除去コピーを配置している

yolos.netの目指す姿: CTAがfold上（スクロールなしで見える範囲）に配置され、「全N問 / 登録不要」の摩擦除去コピーが付いている。detailedContentを読み込んだ来訪者向けに、コンテンツ後にも第2のCTAがある。

#### ゴール3: 見出しとフレーミングが第三者視点で書かれている

調査レポートの事実:

- 成功しているすべての英語サイト（Truity、CrystalKnows）は一貫して三人称（「INTJs are...」「They tend to...」）を使用している
- Truityの「How Others See the INTJ」は、第三者の視点から書かれた外見的特徴の説明であり、第三者向け価値の典型とされる
- CrystalKnowsの「How to Communicate with INTJ Personality Types」は完全に第三者向けのセクション
- ホイミーは二人称（「あなた」）を使用しているが、レポートの分析によれば「来訪者が第三者であることを考慮した設計ではない」

yolos.netの目指す姿: 見出しが「このタイプの特徴」「このタイプのあるある」のように三人称フレーミングされ、第三者が「友人のタイプについて読んでいる」と自然に理解できる。advice文（二人称の語りかけ）には「このタイプの人へのアドバイス」という見出しが付き、第三者が「本人向けのメッセージだ」と文脈を理解できる。

#### ゴール4: 第三者に「このコンテンツが何であるか」のコンテキストが即座に伝わる

調査レポートの事実:

- Truityはタブナビゲーション（Overview / Strengths / Careers / Relationships）でコンテンツの全体像を示している
- CrystalKnowsはヒーローエリアに100語以下の概要を配置し、CTAボタン2つと社会的証明を並べている
- ホイミーはパンくずリスト（ホイミー > 診断名 > 診断結果）で文脈を提供し、「この診断のトップへ」リンクをページ上部に配置している

yolos.netの目指す姿: クイズ名に加えて短い説明文（shortDescription）が表示され、第三者がシェアリンクを開いた瞬間に「これは○○な診断で、友人の結果はこのタイプだ」と3秒以内に理解できる。

### ゴールと現状のギャップ

#### ギャップ1: 見出しが受験者本人向けに書かれている

**現状**: 見出しが「あなたの特徴」「こんなところ、ありませんか?」であり、二人称で受験者本人に語りかけている。adviceセクションには見出しすらない（カード表示のみ）。

**ゴール3との乖離**: 第三者が読むと「あなた」が誰を指すのか混乱する。advice文は一貫して二人称の語りかけ（「〜してみて」「あなたは〜」）であり、見出しによるフレーミングがなければ第三者にとって文脈が不明。detailedContent内に「あなた」を含む行が主要5ファイルだけで合計66行存在する。

#### ギャップ2: CTAがfold下に隠れている

**現状**: CTA（「あなたはどのタイプ? 診断してみよう」ボタン）はdetailedContentの後に配置されている。detailedContentがある場合、traits 4項目 + behaviors 4項目 + adviceカードの後にCTAが来るため、モバイルではかなりスクロールしないとCTAが見えない。

**ゴール2との乖離**: ホイミーは結果タイトル直下にCTAを配置し、CrystalKnowsはページ冒頭と最下部の2カ所に配置している。現在のyolos.netはCTAが1カ所のみかつfold下であり、スクロールせずに離脱する来訪者にCTAを見せられない。

#### ギャップ3: コンテキスト情報が不足している

**現状**: ページ上部には「○○の結果」というクイズ名の表示しかない。診断がどんな内容で何を測るのかの説明がない。

**ゴール4との乖離**: 第三者はシェアリンクから来るため、事前にこの診断について知らない。「音楽性格診断の結果」とだけ書かれても、それが何問の診断でどんな切り口で診断するのかが分からない。quiz.meta.shortDescriptionは既に全クイズに設定済みだが、結果ページでは使われていない。

#### ギャップ4: 見出しが全クイズで画一的

**現状**: 全クイズ共通で「あなたの特徴」「こんなところ、ありませんか?」という見出し。音楽性格診断でも動物性格診断でもキャラ診断でも同じ見出し。

**ゴール1との乖離**: 各クイズは異なるコンセプトと世界観を持っている。例えば「守護キャラ診断」のtraitsに「このタイプの特徴」という汎用見出しは合うが、音楽性格診断なら「このタイプの音楽との向き合い方」のような見出しの方がコンテンツの魅力を引き出せる。Truityは各タイプで共通テンプレートを使いつつ、セクション名をタイプ固有にしている（"INTJ Values and Motivations"など）。

### 設計判断

#### 判断1: 共通テンプレートを維持し、見出しをデータ駆動でカスタマイズする

**根拠**:

- 調査レポートで確認された事実: Truity、CrystalKnows、16Personalitiesはすべて共通テンプレート + タイプ固有のビジュアル要素（色、アイコン等）を採用している
- yolos.netは既にアクセントカラー、アイコン絵文字、OGP画像でコンテンツ差別化を実施済み
- 見出しテキストをデータ側から指定可能にすることで、各クイズの世界観に合った見出しを設定できる

**実装方針**:

- `QuizMeta`に`resultPageLabels`（オプショナル）を追加
  - `traitsHeading?: string`
  - `behaviorsHeading?: string`
  - `adviceHeading?: string`
- デフォルト値（未設定時）は第三者向けの三人称表現:
  - traits: 「このタイプの特徴」
  - behaviors: 「このタイプのあるある」
  - advice: 「このタイプの人へのアドバイス」
- detailedContentを持つ9種のクイズに、各コンテンツの世界観に合った見出しを設定する

#### 判断2: ページ構成をCTA-first設計に変更する

**根拠**:

- 調査レポートの事実: ホイミーは結果タイトル直下にCTAを配置。CrystalKnowsはページ冒頭と最下部の2カ所にCTA
- 現在のyolos.netはCTAがdetailedContentの後（1カ所のみ）であり、fold下に隠れている

**新しいページ構成（上から下）**:

```
[Breadcrumb]
[クイズ名 + shortDescription]                          ← コンテキスト追加
[icon + h1(結果タイトル)]
[description（4行制限 + 展開）]
[CTA1: ボタン + "全N問 / 登録不要"]                    ← fold上に移動
[detailedContent: traits + behaviors + advice]          ← CTA1の後
[CTA2: テキストリンク形式（detailedContent読了者向け）] ← 新規追加
[ShareButtons]
[CompatibilityDisplay（?with=パラメータ時のみ）]
[RelatedQuizzes]
[RecommendedContent]
```

**モバイル(375x812)でのfold上収まりの検証**:

CTA1の上に配置される要素の高さ見積もり:

- wrapper padding-top + Breadcrumb: 約48px
- card padding-top: 32px
- コンテキスト（クイズ名 + shortDescription 1行）: 約40px
- icon（3rem = 48px）+ margin: 約56px
- h1（1.75rem、1-2行）: 約50-80px
- description（4行制限、line-height 1.7）: 約108px
- CTA1ボタン + tryCost: 約70px

合計: 約404-434px。foldライン812pxに対して約380pxの余裕がある。descriptionの4行制限により確実にfold上に収まる。

**descriptionの4行制限について**: CSSの`line-clamp: 4`で表示を制限し、超過分は「続きを読む」で展開する。4行（約80文字）あれば結果の概要は十分に伝わる。展開/折りたたみのインタラクションが必要なため、この部分のみClient Componentとして切り出す。実装時にPlaywrightで実機確認し、4行で不十分なら5行に調整する。

#### 判断3: advice文は見出しによるフレーミングで対応する（本文書き換えはしない）

**現状の問題**:

- advice文は一貫して二人称の語りかけ（例: 「あなたは計画している時間もちゃんと楽しめる人だから」「次に行きたいライブの候補を今すぐ3つ書き出してみて」）
- traits/behaviors本文にも「あなた」が散在（主要5ファイルで合計66行）
- 現在adviceセクションには見出しがなく、第三者は文脈を把握できない

**対応方針**: adviceセクションに「このタイプの人へのアドバイス」という見出しを追加し、第三者が「これは本人向けのメッセージだ」と理解できるフレーミングを行う。traits/behaviorsは見出し変更（「このタイプの特徴」「このタイプのあるある」）により三人称フレーミングを与える。

**本文書き換え（三人称化）を不採用とする根拠**:

人称選択の判断にあたり、以下の2つの調査レポートを参照した。

1. 人称調査レポート（`docs/research/2026-03-31-diagnosis-result-page-person-pronoun-analysis.md`）の結論:
   - 「受験者向け結果（主要読者が受験者本人）: 二人称が業界標準」「両方の読者が混在するページ（ホイミー・MIRRORZ型）: 二人称を維持しCTAで来訪者を誘導」
   - 「yolos.netの静的結果ページは「主要なユースケースが受験者によるシェア」であるため、ホイミー・MIRRORZ型（二人称維持）の方が業界実態に沿っている」
   - ただし「二人称の方が必ず良いという絶対的なエビデンスも存在しない」「消極的論拠に留まる」

2. 第三者向け設計調査レポート（`docs/research/2026-03-31-third-party-type-result-page-design.md`）の事実:
   - Truity・CrystalKnows（汎用タイプ解説ページ）は一貫して三人称を使用
   - ホイミー（カジュアル診断の結果ページ）は二人称を使用するが、「来訪者が第三者であることを考慮した設計ではない」「文体が二人称のため、純粋に第三者として読む場合にわずかな違和感がある」

yolos.netの静的結果ページは「汎用タイプ解説ページ」ではなく「シェアを起点に第三者も訪れるカジュアル診断の結果ページ」であるため、正しい参照先はホイミー・MIRRORZ型のカジュアル診断ページである。ただし、ホイミーが第三者を考慮した設計ではないという調査レポートの指摘を踏まえ、ホイミーの設計をそのまま踏襲するのではなく、「二人称本文を維持しつつ、見出しによる三人称フレーミングで第三者の読みやすさを確保する」という改善を加える。

二人称維持の積極的根拠は限定的であり（業界慣行として標準的であること、第三者が二人称テキストを拒否するという問題が確認されていないこと）、「消極的論拠に留まる」（人称調査レポート結論）点は認識している。三人称への全面書き換えは、Truity・CrystalKnowsのような汎用タイプ解説ページを参照先とした場合に整合する選択肢だが、yolos.netの結果ページはそれとは性質が異なるため不採用とする。加えて、advice文の二人称表現（「〜してみて」「あなたは〜だから」）は、三人称に書き換えると指示・観察の文体になりカジュアル診断らしい語りかけのトーンが失われるため、コンテンツの魅力維持の観点からも二人称を維持する。

- 「このタイプの人へのアドバイス」という見出しがあれば、第三者は「診断結果として本人に送られたメッセージだ」と理解した上で読める。診断サービスにおいて「タイプ別アドバイス」は一般的な構成要素であり、違和感は生じない
- 今後A/Bテスト等で二人称と三人称の効果を実測検証する余地はある（人称調査レポートの示唆）

### 作業内容

#### Step 1: データ構造の拡張

**目的**: 各クイズが結果ページの見出しをカスタマイズできるようにする。

**作業**:

1. `src/play/quiz/types.ts`の`QuizMeta`インターフェースに`resultPageLabels`オプショナルフィールドを追加する
   - `traitsHeading?: string` — traitsセクションの見出し
   - `behaviorsHeading?: string` — behaviorsセクションの見出し
   - `adviceHeading?: string` — adviceセクションの見出し

#### 指摘1（重大）: 選択肢Cの不採用理由が事実に反しており、主語問題の解決策が不十分

計画の判断3および選択肢Cの不採用理由に、以下の記述がある:

> 本文自体は「このタイプの人はこういう傾向がある」という観察的な記述であり、見出しを変更すれば第三者が自然に読めるコンテキストになる。見出しの変更で「このタイプの特徴を第三者として観察している」というフレーミングを与えれば、本文の「あなた」表記も「このタイプのあなた」として読める

**この主張は事実に反している。** 実際のdetailedContent本文を確認したところ、traitsとbehaviorsの文章の多くは観察的記述（三人称的）だが、advice文は一貫して「あなた」への直接的な語りかけになっている。例:

- 「次に行きたいライブの候補を今すぐ3つ書き出してみて。**あなた**は計画している時間もちゃんと楽しめる人だから。」（music-personality）
- 「今日**あなた**が「これいい」と思った曲を、まず一人だけに送ってみて。」（music-personality）
- 「最近「面白そう」と思ったけど踏み出せていないことに、今週一歩だけ近づいてみて。**あなた**の行動力は周りの人にも勇気を与えている。」（traditional-color）

また、traits/behaviors本文にも「あなた」が散在している:

- 「自分の好きなものを誰かと共有する行為そのものが、**あなた**にとってのエネルギー補給になっている。」（music-personality traits）

定量的に確認したところ、detailedContentを持つ主要5ファイルだけでも、detailedContent内に「あなた」を含む行が合計66行存在する。

cycle-139の教訓「根拠のない判断をしない」に明確に違反している。「見出しを変えれば本文も自然に読める」という主張は、本文を実際に確認していれば出てこない結論である。

**対応方針として検討すべきこと**: adviceセクションは第三者にとって「クイズ受験者本人へのアドバイス」であり、見出しの変更だけでは主語問題を解消できない。少なくとも以下のいずれかの対応が必要:

- (a) adviceセクションの見出しを「このタイプへのアドバイス」のように明示し、「本人に向けた文章である」ことを第三者に理解させるフレーミングにする（最小コスト案）
- (b) adviceセクションの本文を三人称に書き換える（100件以上あるが、advice文は各1-2文なので作業量はtraits/behaviorsの書き換えより遥かに少ない）
- (c) traits/behaviors本文中の「あなた」も含めて書き換える（最大コスト案）

いずれを選ぶにせよ、「見出しだけ変えれば自然に読める」という根拠のない主張を計画の前提にしてはならない。実際の本文を確認した上で、どのレベルの対応が必要かを判断し直すこと。
