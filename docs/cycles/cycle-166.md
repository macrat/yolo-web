**選定理由:**

主ターゲットとして候補1を選ぶ理由は、検索需要の大きさである。診断系のKWは他の候補と比較して桁違いに大きい（サイコパス診断16万、ストレス診断2.6万、前世診断6,480、動物診断5,280）。競合は強〜中だが、ニッチ・ロングテールKW（おもしろ診断1,920/月・競合弱、暇つぶし診断576/月・競合弱、脳タイプ診断136/月・競合弱）では十分に勝機がある。KW機会調査のTOP10のうち5つが診断系（サイコパス診断、ストレス診断、前世診断、動物診断、おもしろ診断）である事実が、この選定を裏付ける。加えて、既存17種の診断/クイズ基盤があり、AIによる質問・結果パターンの大量生成が可能なため制作力も高い。

副ターゲットとして候補3（ツール）を含める理由は、SEOのエバーグリーン性とAI Overview代替困難性である。診断系がSNSバイラルに依存する面があるのに対し、ツール系はSEO安定流入を提供する。既存36種のツール基盤があり、JSON整形（2.1万）、カラーコード変換（2,320）など中程度の需要がある。

副ターゲットとして候補7（エンジニア）を含める理由は、独自性である。AI自律運営の一次体験は他サイトでは得られない情報であり、GA4でもblogのPV（103）はplay（50）やtools（17）を上回っている。ただし検索ボリュームが小さいため、PV貢献は限定的と認識している。

**サイトの一貫性について:**

3つのターゲットは「自分を知る・試す楽しさ」（診断）、「作業を効率化する便利さ」（ツール）、「AIの実践を学ぶ知見」（エンジニア向けブログ）と、一見バラバラに見える。しかし、これらを束ねるコンセプトは「AIが作った、すぐに使えるコンテンツ」である。来訪者はAI運営であること自体には関心がないかもしれないが、結果としてAIが高速・大量に質の高いコンテンツを生成できることが、3領域すべてで競争力の源泉になる。詳細は第2部で定義する。

**この選定の不確実性:**

- 診断系の高PVはSNSバイラルが主要チャネルであるという調査結果がある（市場調査レポート参照）。SNSフォロワーゼロの状態でバイラルを起こせるかは未検証。ただし、SEO経由のロングテール流入（おもしろ診断、暇つぶし診断等の低競合KW）で段階的にPVを積み上げる経路は存在する
- ドメイン年齢2ヶ月のため、SEO効果が本格化するのは6〜12ヶ月後。短期的なPV増は期待できない
- 選定が最適でなかった場合、施策選定方法論（第3部）の手順4で実績データに基づき修正する

#### 第2部: サイトの方向性とコンセプト

ターゲット検討の結果を踏まえ、以下の形式でコンセプトを定義する。

**コンセプトが満たすべき条件:**

1. **具体性**: 「楽しくて役に立つ」のような、全てのサイトに当てはまる形容ではなく、yolos.netならではの方向性を示す
2. **排他性**: このコンセプトに合致しないコンテンツを明確に除外できる（取捨選択基準として機能する）
3. **成長性**: 月6万PVに到達するまでコンテンツを拡充し続けられる十分な広がりがある
4. **実現性**: AI自律運営という制約の下で、質の高いコンテンツを持続的に作れる

**コンセプト定義:**

- **yolos.netは何のサイトか（1文）**: 「おもしろ診断とすぐ使える便利ツールがたくさんある無料サイト」
  - 来訪者が友人に紹介するとき、「AIが運営している」とは言わない。「面白い診断がある」「便利なツールがある」と言う。来訪者にとっての価値は「おもしろさ」と「便利さ」であり、AIはその手段に過ぎない。

- **このサイトでしか得られない価値は何か**: 診断メーカーや4ndan.comにはない「ニッチで質の高い診断」と、cman.jpにはない「デザインが洗練されたツール」の両方が1つのサイトにあること
  - 診断メーカーはUGCプラットフォームであり、質にばらつきがある。yolos.netはAI生成による質の担保（質問設計、結果パターンの網羅性、UI品質）で差別化する
  - cman.jp等の既存ツールサイトはデザインが古く機能優先。yolos.netはモダンなUIで同じ機能を提供する
  - 「診断もツールもある」という組み合わせ自体が、診断専業サイトやツール専業サイトとの差別化になる

- **コンセプトに合致するコンテンツの例**:
  1. サイコパス度診断（高需要ニッチ診断、SNSシェアされやすい自己表現系）
  2. ストレスレベル診断（エンタメ系ストレスチェック、医療系との差別化が明確）
  3. JSON整形ツール（開発者向け実用ツール、AI Overviewに代替されない）
  4. カラーコード変換ツール（伝統色辞典とのシナジー、実用性が高い）
  5. 「AI自律運営サイトを作ってわかったこと」系のブログ記事（一次情報としての独自性）

- **コンセプトに合致しないコンテンツの例**:
  1. 毎日更新の星座占い（占い専業サイトの領域。AI運営開示がマイナスに働く。運用負荷も高い）
  2. MBTI診断の模倣（16Personalitiesが世界規模で独占。後追いしても価値が出ない）
  3. 履歴書テンプレート配布（転職大手の領域。yolos.netの強みが全く活きない）
  4. プログラミング入門講座（Progate等の学習サービスが確立。長文記事の網羅性で競合に勝てない）
  5. ビジネスマナー記事（ビジネス情報メディアの領域。差別化要素がない）

- **月6万PVに到達するときのサイトの姿**:
  - コンテンツ量: 診断/クイズ50〜80種、ツール60〜100種、ブログ100〜150記事、辞典データ（既存4種を維持・改善）
  - 流入構成: Organic Search 60〜70%（ツール系のロングテールKWとニッチ診断KWの蓄積）、SNS/Referral 15〜25%（診断結果のシェア経由）、Direct 10〜15%（リピーター）
  - 主要な流入源: ニッチ診断KW（おもしろ診断、動物診断、前世診断等）のSEO上位表示と、一部の診断コンテンツのSNSシェアによるバイラル
  - この姿に到達するまでの期間: 12〜18ヶ月（SEO効果の本格化に6〜12ヶ月、コンテンツ蓄積にさらに数ヶ月）。ただし、バイラルが発生すれば大幅に短縮される可能性がある（再現性は低い）
  - 注意: この数値は市場調査（バズ部調査: 月50万PVメディアの平均174記事、上位25%が76%のPVを生成）からの概算であり、保証ではない。実績データに基づいて随時修正する

**4条件の充足確認:**

1. **具体性**: 「おもしろ診断と便利ツール」は全てのサイトに当てはまる表現ではなく、yolos.netが注力する2領域を明示している
2. **排他性**: 上記の「合致しない例」5つを実際に排除できる。占い、MBTI模倣、履歴書、プログラミング講座、ビジネスマナーはコンセプト外と判断可能
3. **成長性**: 診断系だけでも検索需要の合計は月数十万。ツール系のロングテールKWも広範。コンテンツの種類を増やし続けられる
4. **実現性**: 診断のAI生成（質問・結果パターン）、ツールのフロントエンド実装、ブログの一次情報執筆のいずれもAI自律運営で質を担保できる

#### 第3部: 施策選定の方法論

個別施策のリストは作らない。代わりに、「次に何を作るか」を判断するための手順と基準を定める。cycle-165の結論「網羅的調査で候補を洗い出し、1本ずつ質を重視して実行、実績データで次を判断」を具体化する。

**手順1: 候補の発見**

「何を作るか」の候補をどうやって見つけるか。

- aramakijakeで関連KWの検索ボリュームを実測する
- 実測対象のKWは、コンセプトに合致する全カテゴリから偏りなく選ぶ（AP-P14: 調査範囲の恣意的限定を避ける）
- 「まだ調べていない候補は何か」を調査開始前に確認する
- 既存コンテンツと重複しないか確認する（AP-P06）

**手順2: 候補の評価**

見つかった候補を、以下の基準で評価する。フレームワークや理論ではなく、各候補を個別にデータで評価する（AP-P13）。

| 評価基準                                | 確認方法                                               | 判断の目安                                                                               |
| --------------------------------------- | ------------------------------------------------------ | ---------------------------------------------------------------------------------------- |
| 検索需要がどれだけあるか                | aramakijake実測値                                      | 数値そのものに閾値は設けない。他の候補との相対比較で判断する                             |
| 競合がどれだけ強いか                    | 対象KWでGoogle検索し、上位10件の顔ぶれを確認           | 上位に個人サイト・小規模サイトがいれば勝機あり。大手企業・専門サイトが独占していれば困難 |
| 来訪者にとっての具体的な価値は何か      | 検索意図の分析（SERPの上位ページが何を提供しているか） | 来訪者が求めているものを特定し、それを既存の上位ページより良い形で提供できるか           |
| yolos.netで質の高いコンテンツを作れるか | 既存の技術基盤・データ資産との整合性                   | 既存資産を活かせれば有利だが、新規でも質を担保できるなら候補に含める                     |

**手順3: 1本の選定と制作**

- 評価の結果、最も「需要あり x 競合弱い x 来訪者に明確な価値 x 制作可能」の候補を1本選ぶ
- constitution.mdの「質 > 量」に従い、その1本を最高品質で制作する
- 制作前にSERPの上位ページを実際に使い、「何が足りないか」「どう差別化するか」を具体的に特定する

**手順4: 実績データに基づく次の判断**

- 公開後、GA4とSearch Consoleで検索順位・流入数・エンゲージメントを観測する
- 実績データが出るまで（最低2〜4週間）は、次の施策の制作に進んでよい。ただし、実績データが出た時点で振り返りを行い、手順2の評価基準が妥当だったかを検証する
- 「うまくいった施策」のパターンが見えてきたら、そのパターンを活用する。ただし、成功体験への親近感で他の候補を過小評価しないよう注意する（AP-P15）

**成功/失敗の判断基準:**

個別コンテンツの成否を判断するための目安を以下に示す。ただし、ドメイン年齢2ヶ月の段階では、SEO効果の発現自体に数ヶ月かかるため、短期的な数値で「失敗」と断定しないよう注意する。

- **検索流入の有無（公開後4〜8週間）**: Google Search Consoleで対象KWの表示回数・クリック数を確認する。表示回数がゼロ（インデックスされていない）なら技術的な問題。表示回数はあるがクリック数がゼロなら、タイトル・ディスクリプションの改善余地がある。少しでもクリックが発生していれば、SEOの芽はある
- **コンテンツのエンゲージメント（公開直後から）**: GA4で直帰率と平均滞在時間を確認する。現在のOrganic Search経由の指標（直帰率37.5%、平均滞在7分17秒）を基準値として、これを大きく下回るコンテンツは来訪者の期待と提供価値のギャップがある可能性が高い
- **診断コンテンツのSNSシェア**: シェアボタンのクリック数（または結果ページからの外部遷移）を計測する。診断系コンテンツでシェアが発生しない場合、結果ページの設計（おもしろさ、シェアしたくなる要素）に問題がある可能性がある

**方法論レベルの失敗判断:**

- 3本以上のコンテンツを制作し、全てで公開後8週間経過しても検索流入がゼロのままなら、KW選定（競合の強さの評価）が甘い可能性がある。より低競合なKWに絞り、SERP分析をより厳密に行う
- 検索流入はあるがPVが伸びないなら、コンテンツの質（来訪者の期待と実際の提供価値のギャップ）を見直す。SERPの上位ページと自サイトのコンテンツを具体的に比較し、不足している要素を特定する
- 診断コンテンツを5本以上公開してもSNSシェアが全く発生しないなら、バイラルチャネルへの依存度を下げ、SEOロングテール戦略に比重を移す
- そもそも候補が見つからない（コンセプトに合致するKWで需要のあるものが枯渇する）なら、コンセプト自体を見直す必要がある

### 本サイクルの失敗の詳細記録

#### 概要

本サイクルはB-305「PV成長に向けた網羅的な候補調査と最初の1本の実行」として、サイトの方向性・コンセプトの策定と施策選定方法論の確立を行った。R1（M4件）→全面再設計、R2（M1件）→結論確定、R3→承認の3回のレビューサイクルを経て承認されたが、Ownerとの質疑により構造的欠陥が判明し、**失敗**と判定された。cycle-155以降10サイクル連続の成長戦略失敗。

#### 策定した戦略の内容

- **コンセプト**: 「おもしろ診断とすぐ使える便利ツールがたくさんある無料サイト」
- **ターゲット**: 主ターゲット「診断・心理テストで遊びたい人」、副ターゲット「実用ツールを使いたい人」「エンジニア」
- **施策選定方法論**: aramakijakeでKW実測 → 需要×競合×価値×制作力で評価 → 1本制作 → 実績データで検証

#### 根本的な失敗: コンセプト集中の原則の無視

**cycle-66（2026-03-04〜05）で確立された原則を、本サイクルは検討すらしなかった。**

cycle-64でAdSense審査が「有用性の低いコンテンツ」で却下されたことを受け、cycle-66で根本原因分析が行われた。特定された問題の一つが「S2: コンテンツカテゴリの過剰分散」であり、ツール33個・チートシート7個・ゲーム4種・クイズ5種・辞典3種・ブログ53記事が散在することで専門性（Expertise）が希薄化し、E-E-A-T評価で不利になるという分析だった。

この分析に基づき、cycle-66では1,728通りの強制発想法と7案の定量比較を経て「笑えて、シェアしたくなる占い・診断の遊園地」にコンセプトを集中させ、合わないコンテンツ（汎用ツール約23個、チートシート7種、辞典3種）を410 Goneで削除した。

本サイクルで策定した「おもしろ診断と便利ツール」は、この判断を覆し、cycle-66が意図的に解消したカテゴリ分散（診断 + ツール）を再び作り出している。しかし、PMはcycle-66の経緯を調査しておらず、コンセプト集中の原則の存在自体を認識していなかった。

#### 失敗の経緯と判断の過程

**1. 調査フェーズでの見落とし**

3本の調査（GA4分析、KW機会調査、競合・市場調査）を並行実行したが、以下が欠落していた:

- cycle-66のコンセプト決定経緯の調査（コンセプト集中の原則を知らないまま新コンセプトを策定）
- 既存コンテンツのSearch Consoleパフォーマンス分析（既存118件から学べることを飛ばして新規制作前提で進行）
- 「占い・診断の遊園地」コンセプトの成否分析（コンセプト自体の問題か、実行の問題かの切り分け）
- 辞典系KW、ブログ個別記事のKW、既存診断のKW調査（現在PVを稼いでいるカテゴリの深掘り）

**2. ターゲット選定の過程**

7候補を比較表で評価したが:

- 7候補の生成が体系的でなかった。既存3ターゲット + R1レビュアーが例示した4候補の合算であり、網羅的な洗い出しではない
- 比較の定量性は検索需要の1軸のみ。競合・制作力・来訪者価値の3軸はplannerの定性的判断に依存
- 比較評価の内部整合性チェックを行っていない
- 診断系（20万超/月）とツール系（15万超/月）の差を「桁違い」と表現したが実際は33%差。さらにツール系15万のうち13.2万はフォント変換1KWで検索意図未確認
- コンセプト集中の原則を考慮していないため、「診断 + ツール」という2カテゴリ選定に疑問を持てなかった

**3. コンセプト定義の過程**

ターゲット選定からの一方向導出であり、コンセプトレベルでの代替案比較を行っていない:

- cycle-66では7案を定量比較（スコア2.4〜4.0）して選定したが、本サイクルではコンセプト案は1つしか存在しなかった
- 「おもしろ診断と便利ツール」は、主ターゲット（診断）と副ターゲット（ツール）を結合しただけ
- 「日本語の言葉で遊ぶサイト」「AIが毎日新しいコンテンツを作る実験サイト」等の代替案は検討していない
- コンセプトの排他性テスト（合致/非合致例各5つ）を行ったが、「脳トレゲーム」「漢字クイズ」等の境界ケースでは判断できず、コンセプトの定義が不十分

**4. 施策選定方法論の問題**

「KW需要があるものを作る」が実質的な基準であり、コンセプトによるフィルタリングが弱い:

- コンセプトの排他性が不十分なため、方法論を適用すると「需要があれば何でも作る」になるリスク
- これはcycle-64で特定された「何でもサイト」問題への回帰
- 大量生成コンテンツのSEOペナルティリスクも未検討

**5. レビュープロセスの限界（再発）**

R1（M4件）→R2（M1件）→R3（承認）の3回のレビューサイクルで品質を担保したはずが、構造的な問題を検出できなかった:

- cycle-66のコンセプト集中原則は、計画に書かれていない「検討すべきだったが検討されなかったこと」であり、計画の内部整合性をチェックするレビュー設計では検出不能

- Ownerとの質疑でのみ発見された（「比較評価の矛盾チェックは？」「桁違いか？」「既存コンテンツからの学習は？」「コンセプト集中の原則は？」等の質問で逐次判明）

#### Ownerとの質疑で判明した具体的な問題点

1. **比較評価の内部整合性を確認していない** — 7候補の競合評価基準が統一されていない可能性
2. **「桁違い」は不正確** — 診断系20万 vs ツール系15万は33%差であり、桁の差ではない
3. **既存コンテンツからの学習を飛ばした** — 118件のコンテンツと2ヶ月分のデータがあるのに、新規制作前提で進行
4. **KW調査が網羅的でない** — 辞典系KW、ブログ個別記事KW、既存診断KWが未調査
5. **ツール不振（36種で月17PV）の原因分析なし** — 副ターゲットに据えたのに原因を調べていない
6. **6万PVへの具体的な経路がない** — 「方法論を繰り返す」だけで月数千PVから6万PVへのジャンプを説明できない
7. **コンセプトの確証なし** — 代替案比較なし、検証幅の評価なし
8. **大量生成コンテンツのSEOリスク未検討**
9. **コンセプト集中の原則を無視** — cycle-66の経緯を調査していない（最も根本的な問題）

#### 何が有効で何が無効か

**有効（事実データとして次サイクルで参照可能）:**

- KW実測値55件（aramakijake。ただし網羅性に欠け、フォント変換の検索意図未確認等の留意点あり）
- バイラル事例データ（VONVON 4ヶ月で1.6億PV、Peing 1ヶ月で2億PV等）
- Ownerとの質疑で明らかになった問題点のリスト（上記9点）

**無効（破棄すべき）:**

- コンセプト「おもしろ診断と便利ツール」（コンセプト集中の原則に反する）
- ターゲット選定（主: 診断、副: ツール・エンジニア）（検討プロセスが不十分、コンセプト集中原則未考慮）
- 施策選定方法論（コンセプトなしに使うとカテゴリ分散を加速する）

**判断が必要:**

- 施策選定方法論の手順部分（aramakijake実測→SERP確認→1本制作→実績検証）は、コンセプトが確定した後にその枠内で適用できる可能性がある

#### この失敗の構造的原因

**PMがプロジェクトの歴史を把握していなかった。** cycle-66でのコンセプト集中の判断、その背景にあるE-E-A-T分析、コンセプト外コンテンツの意図的削除という一連の経緯を知らないまま、ゼロベースと称して過去の判断を無視したコンセプトを策定した。

これは「ゼロベースで検討する」ことと「過去の判断を無視する」ことを混同した結果である。ゼロベースとは、前提を所与としないことであり、過去の知見を捨てることではない。cycle-66のコンセプト集中原則は「検討した上で採用するか却下するかを判断すべき知見」であって、「知らなくてよい過去の判断」ではなかった。

#### レビュープロセスの構造的限界

本サイクルでもcycle-165と同じ問題が発生した: reviewerは計画の内部整合性を検証できるが、「検討すべきだったが検討されなかったこと」は検出できない。cycle-66のコンセプト集中原則は、PMが調査して計画に含めるべき情報だったが、PMが調査しなかったためreviewerのチェック対象にならなかった。

この問題への対策案:

- レビュー時に「プロジェクトの過去のコンセプト変更履歴を確認したか」をチェック項目に追加する
- コンセプト策定時には、過去のコンセプト決定の経緯と理由を必ず調査するプロセスを義務化する
- ただし、チェック項目の追加だけでは形骸化のリスクがある（AP-WF09）。根本的には、PMがプロジェクトの歴史を理解した上で計画を策定するという能力・姿勢の問題
