## 作業計画

### 目的

**派生元**: B-336（backlog・P1）＝ `docs/design-migration-plan.md` Phase 10.3「道具箱ページの本公開」。その原典は `docs/site-concept.md` L7「気に入ったツールを1つの画面にまとめられるダッシュボード的な機能を通じて、『日常の傍にある』存在を目指す」。

cycle-226〜231 で道具箱は「39枚の生きたタイル・タイル選択・localStorage 永続化・ペルソナ別プリセット5本」まで完成したが、`/toolbox` は noindex のままで、ターゲット「気に入った道具を繰り返し使っている人」（docs/targets/気に入った道具を繰り返し使っている人.yaml）が実際にこの価値に触れる経路が存在しない。プリセットが「迷わない出発点」として「特定の作業に使えるツールをさっと探している人」に価値を発現するのも本公開時点（cycle-231 計画 r1 SF-1 で正確化済み）。**作った価値が来訪者に届いていない状態を解消することが本サイクルの目的**であり、constitution の Goal（来訪者への価値提供による PV 向上）への最短経路である。

GA 実測（2026-05-29〜06-11、GA4 property 524708437、`mcp__google-analytics__run_report`、本サイクル kickoff 時に PM が実測）: ページ別 PV 上位は `/play/character-personality` 21PV・トップ `/` 17PV（17セッション）・`/play/word-sense-personality` 14PV で、`/tools` 一覧は 5PV、`/toolbox` は上位25行に現れない（cycle-231 kickoff 実測で 1PV）。トップ `/` の現行内容を変更しても影響を受ける既存トラフィックは 14日で17PV と極小であり、URL=トップ案を含めた再設計の自由度は高い。一方でサイト全体の流入の大半は個別コンテンツ（`/play/*`・`/tools/*`・`/blog/*`・`/dictionary/*`）への検索直行であり、道具箱公開の狙いは「個別ツールに検索到達した来訪者が『また使う』ために戻ってくる場所を作る」ことにある（site-concept.md「検索→使う→離脱」からの差別化）。

### 作業内容

Phase 10.3 を「URL 決定 → default content 組み込み → 正式公開 → about 書き換え」と定義し、T-2〜T-7 で実施する。

1. **公開形態の決定**（T-2）: 移行計画が Phase 10.3 の責務と明記する URL 決定（「URL 構成（未決定）」節: トップ `/` か専用 URL か。Phase 10.3 着手 PM が「Phase 2.1 で決まった」と読まないこと、に従う）。URL=トップの場合は Phase 4.4 で移行した現行トップ内容の処遇（別ページに移す／統合する／廃棄する）も決める。default content は cycle-231 の見立て（第一候補 daily-life・対抗 writing。docs/cycles/cycle-231.md 補足事項⑥）を出発点に最終判断する。
2. **本公開の実装**（T-3・T-5）: noindex 解除・OGP/メタデータ・sitemap・ナビ導線・default content。既存の localStorage 構成保持者（プレビュー利用者・バージョン付きスキーマ）を壊さない。
3. **about の書き換え**（T-4）: B-336 の明示スコープ。旧コンセプト（占い・診断パーク）由来の記述を新コンセプトに更新する。

**Phase 10.4 / 10.5 との順序の解釈**: 移行計画 Phase 10 は「サブタスクを順に」として 10.1 → 10.2 → **10.3（本公開）** → 10.4（連携）→ 10.5（シェア）の順で列挙しており、10.3 の本公開は 10.4/10.5 の実装を前提としない。10.3 内の記述「10.4 / 10.5 も本公開時に有効化する」は、未実装の現状では有効化対象が存在せず、Phase 10.1 で確立された整理（「公開後に順次追加する機能」。原設計の核は「気に入ったツールを1画面にまとめられる」であり、複雑機能の全揃えより道具箱が動くことが優先）に従って **10.4/10.5 は公開後の順次追加機能**と解釈する。B-336 の着手条件が「Phase 10.1 + 10.2 完了」のみである（cycle-231 T-10 で reviewer 承認込みで記録済み）こととも整合する。この解釈は T-1 で reviewer の敵対的検証を受ける。

**DESIGN.md 恒久要件の本サイクルへの翻訳**（AP-P22 / B-498 対応・builder への指示書へ必ず含める。対象: ナビ導線・道具箱ページの公開化・about・トップ変更が入る場合はトップ）:

- §1: すべてのコンテンツはパネルに収まる（src/components/ の Panel）
- §2: 色は機能を伝えるためだけに使い装飾色禁止。フォーカスは `outline: 2px solid var(--accent); outline-offset: 2px;`。選択中状態の塗りに `--accent` を直接使わない（収束チェックリスト B-3）
- §3: アイコン原則なし（必要なら Lucide 線画 1.5px / 16・20・24px ＋ aria-label）、絵文字禁止、フォント16px以上（補足ラベルの扱いは B-507 未決のため新規の小さめフォントを増やさない）、本文太字原則なし
- §4: 画面幅は最大1200px。パネルは入れ子にしない。影をつけない。どの画面幅でも自然なレイアウト
- §5: UI 部品は src/components/ の共通部品を使う。Panel padding は prop 規定、角丸はトークン（`--r-normal` / `--r-interactive`）のみ
- §6 Do/Don't: 最小限の見出し・単色パネル・過度な区切りなし
- §7 暫定対応: 新グローバルスタイル（src/app/globals.css）と新コンポーネント（src/components/）のみを使う。旧（src/components/common/・old-globals.css）を使わない
- タイルのルートは `<Panel>` のまま、既存タイル（XxxTile）には手を入れない
- タップターゲットは44px以上

**スコープ外**: DnD 並べ替え・編集モード・Undo・モーダル（公開後に順次追加）、タイル間連携（10.4＝B-324）、シェア（10.5＝B-313）、タイル入力値の保持（B-433）、legacy 撤去（Phase 11＝B-337）、ナビゲーション全体のゼロベース再設計のうち道具箱導線に不要な部分（B-310 に残す。境界は T-2 ③で確定）。

**完了条件**: 来訪者が検索エンジン経由・サイト内導線経由の双方で道具箱に到達でき（noindex 解除・sitemap 掲載・ナビ導線）、初回来訪者は default content（プリセット）入りの道具箱を見る（空状態を見ない）。既存のプレビュー利用者の localStorage 構成が壊れない。about が新コンセプトを説明している。tile-architecture.md §4 の合格条件と w360 横はみ出しゼロが回帰していない。4ゲート全通過。
