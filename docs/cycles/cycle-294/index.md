## 作業計画

### 目的

来訪者が診断に求めるのは「自分はどのタイプか」という答えそのもの。その答えが配列の並び順という無関係な実装都合で歪んでいるなら、玄関のコンテンツで静かに来訪者を裏切っていることになる(constitution rule 2・4)。実測で歪みの有無と大きさを確かめ、あれば正す。

### 作業内容

1. **棚卸し**: 全12 personality診断(animal/character-fortune/character-personality/contrarian-fortune/impossible-advice/japanese-culture/music/science-thinking/traditional-color/unexpected-compatibility/word-sense/yoji)の配点構造を確認。
2. **計測**: 回答空間の列挙(設問数×選択肢数が現実的なら全列挙、大きければ層化サンプリング。列挙規模は harness が自動判定し、サンプリング時はその旨を findings に明記=silent cap 禁止)。指標=勝者分布・同点率・到達不能タイプ・配列順バイアスの寄与。
3. **判定**: 診断ごとに実害度を記録。「同点はほぼ起きない/起きても到達不能タイプを生まない」なら無害、「特定タイプが配列順だけで恒常的に勝つ/負ける」なら実害。
4. **是正**: 実害があれば方針決定のうえ最小修正。無害なら「検証の結果無害」を根拠付きで記録してクローズ(是正しないことも正当な結論)。
