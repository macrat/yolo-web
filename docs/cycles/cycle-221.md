# サイクル-221

cycle-220 で策定した立て直し計画の最初の実装サイクル。立て直しの本線（B-488→B-489→B-490 の土台構築と全ツール再構築）に入る前に、**公開中のツールが今この瞬間、来訪者に矛盾・誤った答えを返している緊急バグ（P0）を止める**ことを目的とする。

対象は cycle-220 で「緊急」ラベルが付いた2件：

- **B-485（①-1）**: char-count と byte-counter が同一入力で文字数・単語数が食い違う。さらに char-count の meta（「コードポイント単位でカウント」）と実装（`text.length`）が矛盾。
- **B-486（①-3）**: cron-parser が `9-17`（範囲）・`9,12,15`（リスト）指定で日本語説明を誤り、次回実行リストと自己矛盾する。

両件とも cycle-220 の立て直し計画に従い、**介入は logic 層の正答化に限定し、Component の UI には一切手を入れない**（UI リフォームは廃止予定の家のリフォームになるため禁止。正答仕様は B-490 の各タイル再構築に継承される）。
