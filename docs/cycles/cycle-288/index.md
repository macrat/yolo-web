# サイクル-288

このサイクルでは **B-593: ツールのセクション見出しの階層是正(h1→h3飛び)** を行う。

cycle-287 の動的a11y監査(C1)で見つけた実欠陥。ツールページの見出し順序は次のようになっている:

```
h1  ツール名            (ToolPageLayout/index.tsx:56)
  h3  日付の差分         ← h2 を飛ばす
  h3  日付の加算・減算
  h3  和暦・西暦変換
h2  このツールについて   (ToolPageLayout:80・本体の後)
h2  共有                 (ToolPageLayout:96)
```

**本体のセクション見出し(h3)が、その後に来るメタ見出し(h2)より重要なのに階層が深い。** スクリーンリーダーの見出しナビ(h キー等)で辿ると `h1 → h3(飛び) → … → h2 → h2` となり、一番大事なツール本体の構造が正しく伝わらない。WCAG 1.3.1 Info and Relationships(A)/2.4.6 Headings and Labels(AA) 違反。

**芯**: 抽象評価ではなく「どのツールのどの見出しが何レベルになるべきか」を指で差せる形で是正する。単純な一律 h3→h2 ではない——cron-parser のように「次回実行予定」が「フィールド詳細」の下に**正当にネスト**している箇所があり、tile ごとにトップレベルのセクションか正当なサブ見出しかを判断する。判断を誤ると正しい階層を壊す(cycle-287 の教訓: grep ヒットでなく実描画パスで判断する)。

**スコープ**: h3 をハードコードする7ツールファイル(実測18箇所)に限定。ToolPageLayout 側の h1/h2(メタ見出し)は既に正しいので触らない。ページ h1 が存在しない面(diagnosis の daily 等)は別チケット(B-594)。

### 検討した他の選択肢と判断理由
