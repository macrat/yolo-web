## 作業計画

### 目的

メインターゲット（手軽で面白い占い・診断を楽しみたい人）にとって、/playページのカテゴリ名が「機能的だが感情を喚起しない」状態にある。カテゴリ名をユーザーの動機や感情に寄り添う表現に変更することで、「このカテゴリを開いてみたい」という気持ちを引き出し、回遊率の向上を図る。

変更はあくまで表示ラベルのみであり、英語キー（URL、データファイル、内部ロジック）には一切手を入れない。影響範囲を最小化しつつ、ユーザー体験に直接効く改善を行う。

### 作業内容

#### 1. カテゴリ名の最終決定

以下のカテゴリ名に変更する。

| 内部キー    | 現在のラベル | 新ラベル             | 選定理由                                                                                                                                                                              |
| ----------- | ------------ | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| fortune     | 占い         | 今日の運勢           | 占いカテゴリは実質「今日の運勢」1種のみ。「今日の」が毎日来る動機を喚起し、検索意図（"今日の運勢"）とも一致する。                                                                     |
| personality | 性格診断     | あなたはどのタイプ？ | 「あなた」と問いかけることで好奇心を刺激する。11種と最多カテゴリであり、ユーザーの自己理解欲求に直接訴えかける。                                                                      |
| knowledge   | 知識テスト   | どこまで知ってる？   | 「テスト」は敬遠されやすい（調査結果より）。疑問形にすることで挑戦心を煽り、クリック意欲を高める。                                                                                    |
| game        | ゲーム       | 毎日のパズル         | ゲームカテゴリの4種はすべてパズル系。「毎日の」でリピート動機を喚起する。占いの「今日の」と合わせて「今日の〇〇」の多用を回避（「今日の」は1カテゴリ、「毎日の」は1カテゴリに限定）。 |

「今日の〇〇」系フレームの適用ルール:

- 「今日の」を使うのはfortune（1カテゴリ）のみ
- 「毎日の」を使うのはgame（1カテゴリ）のみ
- personality/knowledgeは疑問形で差別化し、単調さを回避

#### 3. UIへの影響確認

カテゴリナビのタブ幅に注意が必要。現在のラベル（最大4文字「性格診断」）に対し、新ラベルは最大10文字（「あなたはどのタイプ？」）になる。

CSS確認結果（`src/app/play/page.module.css`）:

- `.categoryNav` は `display: flex; flex-wrap: wrap; overflow-x: auto; scrollbar-width: none`
- `.categoryNavTab` は `white-space: nowrap; padding: 0.35rem 1rem; font-size: 0.85rem`
- タブは折り返さず、収まらなければ横スクロール（スクロールバー非表示）になる設計
- 10文字程度のラベルでもスクロールで対応可能だが、375px幅で全タブが一覧できるかはPlaywrightで確認する

収まらない場合の代替ラベル（事前に決定しておく）:

- 「あなたはどのタイプ？」→「どのタイプ？」（6文字）
- 「どこまで知ってる？」→「知識クイズ」（5文字）
- 上記の短縮版でもモバイルで収まらない場合は横スクロールを許容する（既にCSS対応済み）

#### 4. SEOへの影響への配慮

- /playページのmeta description: カテゴリ名を新ラベルそのままではなく、SEOに適した一般名詞で記述する（例: 「運勢占い・タイプ診断・クイズ・パズルゲーム」）
- トップページのmeta description: 同様に一般名詞を維持する
- `seo.ts` の `resolveDisplayCategory()`: titleタグ用なので短い名詞形を使う（疑問形は不可）

### 検討した他の選択肢と判断理由

#### 選択肢A: 疑問形を全カテゴリに適用する

- 案: 「今日の運勢は？」「あなたはどのタイプ？」「どこまで知ってる？」「今日のパズルは？」
- 却下理由: 全て疑問形にすると単調になり、かえってスキャン性が下がる。また「今日の〇〇」の多用回避の原則にも反する。

#### 選択肢B: 全て名詞形の新ラベルにする

- 案: 「今日の運勢」「タイプ診断」「知識チャレンジ」「毎日のパズル」
- 却下理由: 「タイプ診断」「知識チャレンジ」は現在のラベルからの変化が小さく、感情喚起の効果が限定的。personality（11種・最多カテゴリ）とknowledge（3種・テスト敬遠の課題あり）こそ疑問形で好奇心を刺激する効果が大きい。

#### 選択肢C: 現在のラベルを維持し、サブテキストで補足する

- 案: カテゴリ見出しの下に「あなたはどのタイプ？試してみよう」等のサブテキストを追加
- 却下理由: カテゴリナビのタブにはサブテキストを入れる余地がなく、ナビとセクション見出しでラベルが不一致になる。表示ラベル自体を変更する方がシンプルで一貫性が高い。
