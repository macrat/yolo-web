# サイクル-172

このサイクルでは `docs/design-migration-plan.md` の **Phase 1: 着手前判断** を実施する。Phase 1.1（検索方針）と Phase 1.2（既存コンテンツ整理方針）の 2 つの判断を完了させ、後続 Phase 2〜10 のスコープを確定する。

本サイクルは判断のみで実装作業はゼロ。来訪者から見える変化はないが、ここで決まることが Phase 4（一覧設計）・Phase 5（検索実装の要否）・Phase 8（既存コンテンツの削除/移行範囲）のスコープを決める。

## 実施する作業

- [ ] **Phase 1.1: サイト内検索の方針判断（B-332）**
  - [ ] GA4 で legacy search の利用率（検索モーダル開閉数、検索実行数、検索結果クリック数）を分析
  - [ ] 検索ワード傾向（横断的な検索が多いか、カテゴリ内検索で十分か）を分析
  - [ ] 候補 (a) 横断検索維持 / (b) カテゴリ別検索に分割 / (c) 検索撤去 のいずれかを選択し、根拠とともに記録
  - [ ] 判断結果を `docs/design-migration-plan.md` の Phase 1.1 セクションに反映、または別途判断記録ドキュメントを `docs/research/` に保存

- [ ] **Phase 1.2: 既存コンテンツ整理の方針確定（B-315）**
  - [ ] 辞典（colors 系・humor 系・kanji 系・yoji 系の 12+ ルート）について「移行 or 削除」を個別判断
  - [ ] cheatsheets（cron / git / html-tags / http-status-codes / markdown / regex / sql の 7 ルート）について「移行 or 削除」を個別判断
  - [ ] achievements（単一ページ）について「移行 or 削除」を個別判断
  - [ ] 各判断の根拠（GA4 PV、新コンセプトとの整合、運用コスト等）を記録
  - [ ] 判断結果を `docs/design-migration-plan.md` の Phase 1.2 / Phase 8 のスコープに反映
