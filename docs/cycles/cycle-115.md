# サイクル-115

このサイクルでは2つのタスクに取り組みます。B-220ではGoogle Discover掲載の前提条件である`max-image-preview:large`メタタグを追加します。B-111では辞典コンテンツの差別化として、AIやロボット目線のユーモアのある例文追加の価値を調査・検討します。

## 実施する作業

- [x] B-220: max-image-preview:largeメタタグの追加
- [x] B-111: 辞典への面白い例文追加の価値検討

## 作業計画

### 目的

#### B-220: Google Discover掲載のための技術的前提条件を満たす

Google Discoverに掲載されるためには、robotsメタタグに `max-image-preview:large` を指定する必要がある。現在のサイトにはこの設定が存在しない。ルートレイアウトに追加することで全ページに伝播させ、Google Discoverからの流入経路を確保する。これはサイトのPV最大化（constitution.md Goal）に直接貢献する施策である。

**誰のために**: サイトへの来訪者全体（Google Discoverで大きなプレビュー画像を見て興味を持ち、クリックする潜在的な来訪者）

**提供する価値**: Google Discoverで画像が大きく表示されることでCTRが向上し、サイトへの流入が増加する。来訪者にとっては、検索結果やDiscoverフィードで視覚的に魅力的なプレビューが表示され、コンテンツの内容を判断しやすくなる。

#### B-111: 辞典への面白い例文追加で差別化できるか検討する

漢字・四字熟語・色の3辞典はsite-concept.mdで削除候補に挙がっているが、低価値コンテンツの削除（Phase 4-1）はGA分析が前提条件であり、GA分析（B-114）が未実施の現時点では削除できない。一方、**constitution.mdルール4はサイト上のすべてのコンテンツが来訪者にとって最高の品質であることを求めている**。辞典がサイトに存在する以上、最高品質にする義務がある。面白い例文の追加によって辞典コンテンツの独自価値を最大化する方法を検討し、具体的な改善方針を提示する。

**誰のために**: サイトの来訪者全体。特に四字熟語・漢字・伝統色に興味を持つ来訪者。ユーモアのある例文はSNSシェアの動機にもなりうる。

**提供する価値**: 辞典コンテンツを「大手サイトにはない独自の価値を持つコンテンツ」に改善する具体的な方針を示す。

### 作業内容

#### B-220: max-image-preview:largeメタタグの追加（実装タスク）

1. **ルートレイアウトへのrobots設定追加**
   - `src/app/layout.tsx` の `metadata` オブジェクトに `robots` フィールドを追加する
   - `index: true`, `follow: true`, `'max-image-preview': 'large'` を設定する
   - Next.js Metadata APIの型安全な方法で実装する

2. **既存のnoindexページとの整合性確認**
   - `src/app/play/[slug]/result/[resultId]/page.tsx` が独自に `robots: { index: false, follow: true }` を設定している
   - Next.jsのmetadata merging動作により、子ページの設定がルートレイアウトの設定を上書きする
   - noindexページはDiscoverに掲載されないため `max-image-preview` が未指定でも影響なし
   - この動作が期待通りであることをビルド後に確認する

3. **検証**
   - `npm run build` が成功すること
   - ビルド成果物のHTMLに `<meta name="robots" content="index, follow, max-image-preview:large">` が出力されていること
   - noindexページのHTMLの `robots` メタタグの内容を確認する。Next.jsのmetadata mergingでは子ページの `robots` 設定が親を完全に上書きするため、`max-image-preview:large` は含まれない想定。万が一含まれていても、noindexページはDiscoverに掲載されないため実害はない
   - lint, format, testが全て通ること

4. **注意点**
   - 変更は1ファイルのみ（`src/app/layout.tsx`）であり、影響範囲が明確
   - 既存のテストを壊さないこと
   - `googleBot` の設定は不要（通常の `robots` 設定で十分）

#### B-111: 辞典への面白い例文追加の価値検討（検討タスク）

成果物は**推奨事項を含む検討結果レポート**である。実装は行わない。

1. **選択肢の整理と評価**

   辞典はサイト上に存在しており、GA分析完了まで削除できない。constitution.mdルール4に基づき、**サイト上のコンテンツはすべて最高品質でなければならない**。以下の選択肢は、辞典コンテンツの品質を最大化するための改善方針である。
   - **選択肢A**: ユーモア例文を追加して辞典を差別化する（四字熟語・漢字・色それぞれの具体的な改善案を含む）
   - **選択肢B**: ユーモア例文に加え、B-092の拡充（書き順・成り立ち・由来等）も組み合わせて辞典の独自価値を高める
   - **選択肢C**: 既存のユーモア辞典（Q20）を辞典素材で拡充する方向に注力する（例: 四字熟語にユーモア的再定義を付与しQ20フォーマットで追加、または「AI的四字熟語解説」としてQ20のテーマカテゴリ化）。辞典自体の改善は別途検討する

2. **評価の観点**

   各選択肢について以下を検討する。
   - **来訪者価値**: 辞典の来訪者にとって面白い例文がどの程度の価値を持つか。辞典コンテンツ自体が検索流入の入口になりうるかも評価する
   - **独自性と差別化効果**: 新明解国語辞典の「個性的な語釈」バズ事例を踏まえ、AI/ロボット視点のユーモア例文が実際に差別化要因になりうるか。大手辞書サイト（Weblio, コトバンク等）との差がどの程度縮まるか
   - **サイトコンセプトとの整合性**: AI運営サイトとしてのブランドとの整合性。辞典が「占い・診断パーク」コンセプトの中でどう位置づけられるか
   - **既存資産との関係**: ユーモア辞典（Q20、30エントリ、評価ボタン・SNSシェア機能あり）との重複・相乗効果
   - **B-092との関係**: 辞典データ拡充タスク（B-092）との問題意識の重複をどう整理するか
   - **実現可能性**: 各辞典の現在のデータ構造と件数（漢字2,136件、四字熟語400件、色250件）を踏まえた実装工数

3. **推奨事項の策定**

   評価結果に基づき、明確な推奨事項を1つ提示する。推奨事項にはその判断根拠を含める。

4. **backlogの更新**

   検討結果に基づいて以下のbacklog操作を行う。
   - B-111を完了にする
   - B-092（辞典データ拡充）との関係について言及する。ただしB-092は「書き順・成り立ち・由来の追加」という別の切り口であるため、B-092自体の最終判断はB-111の結論だけでは決定しない。B-111の結論が選択肢B（ユーモア例文+B-092の拡充を組み合わせる）の場合のみB-092をB-111の後続タスクとして位置づけ、それ以外の場合はB-092は独立して価値判断する旨を記載する
   - 必要に応じて新規タスクをbacklogに追加する

5. **注意点**
   - 「来訪者にとっての最大価値」を判断基準の最上位に置く。実装の容易さやコードの都合で判断しない
   - 辞典はサイト上に存在しており、削除はGA分析（Phase 4-1前提条件）完了まで不可。constitution.mdルール4に基づき、存在するコンテンツはすべて最高品質にする義務がある。「削除するかどうか」ではなく「どう最高品質にするか」が検討の主軸
   - ユーモア辞典（Q20）は既にPhase 3のコンテンツとして位置づけられており、相乗効果を活かす方向で検討する

### 検討した他の選択肢と判断理由

#### B-220について

- **個別ページごとに設定する案**: ルートレイアウトで一括設定するほうが漏れがなく保守性が高い。noindexページでは子の設定が優先されるため問題なし。ルートレイアウト一括設定を採用する
- **googleBotも別途設定する案**: 通常のrobots設定でGooglebotにも適用されるため不要。過剰な設定は避ける

#### B-111について

- **実装まで行う案**: タスクの定義が「価値検討」であり、削除予定コンテンツへの実装投資は検討結果を待つべき。検討のみに留める
- **検討を省略して即座に「削除」判断とする案**: 削除はGA分析（Phase 4-1前提条件）が完了するまで不可。サイトに存在する以上、constitution.mdルール4に基づき最高品質にする義務がある
