# サイクル-165

cycle-163で策定したPV成長計画には構造的欠陥（現状固定の前提で上限を算出、フェーズ分けなし、新規ジャンル未検討、施策の組み合わせ未検討、過去失敗の分析未実施）が指摘された。cycle-155以降8サイクル連続で成長戦略が失敗している。このサイクルでは、これらの欠陥を補完する調査を実施し、検証済みの事実に基づいた実行可能なPV成長計画を確定させる。

## 作業計画

### 目的

yolos.netのPV成長を「来訪者への価値提供」から出発して実現するための、フェーズ別実行計画を策定する。constitutionのゴール「achieve higher page views by providing the best value for visitors」に直接対応し、過去8サイクル連続の失敗パターンを構造的に回避する。

**核心的前提（この計画が成立するための条件）:**

1. **低競合ロングテールKWでは、ドメイン年齢2ヶ月でもSERP上位を狙える**: cycle-164で「改行削除 オンライン」のSERP上位がGitHub個人ページ・netlifyページだったことで部分的に実証済み。ただし実際の検索順位変動はこれから検証される（推測）
2. **デイリーコンテンツはリピーター獲得に有効**: Wordle/Spotle/Globale等の事例で訪問者の88-90%がDirect/Returnとされている（Medium記事等のサードパーティ推定値）。ただしyolos.netで同様の効果が出るかは未検証（推測）
3. **診断コンテンツはシェアされやすい**: BuzzFeedクイズのトラフィック75%がシェア由来は実測データ。yolos.netの診断コンテンツの直帰率0%はGA実測値。ただしyolos.netの診断でシェアが発生するかは未検証（推測）
4. **テーマ特化タイピングKW（四字熟語タイピング等）の直接SEO流入は極めて限定的（月100未満）。ただしタイピングゲーム市場全体（月8.8万）へのロングテール流入と、リテンション・バイラルループでの価値が主要な貢献経路（aramakijake実測済み）**

### 作業内容

#### 全体設計: 3ループ複合フライホイール

個々の施策を単独で機能させるのではなく、3つのミクロループが相互に強化し合うフライホイールを設計する。各施策は「どのループのどのステップを担うか」で位置づけを明確にする。

**ループ1: SEOコンテンツループ（新規流入の獲得）**

```
低競合KWで検索 → コンテンツ体験（ツール・診断・ゲーム）→ 長滞在・低直帰 → Googleエンゲージメント評価改善 → SEOランキング向上 → 流入増加
```

**ループ2: バイラルループ（来訪者の拡散）**

```
診断・ゲーム体験 → 感情的反応（「自分は○○タイプ」「スコアXX達成」）→ SNSシェア → フォロワーが流入 → 体験 → さらにシェア
```

**ループ3: リテンションループ（来訪者の定着）**

```
デイリーコンテンツ → 「また明日来よう」習慣形成 → ブックマーク/Direct流入 → サイト内回遊 → 深いエンゲージメント → ループ2へ接続
```

**3ループの接続点:**

- ループ1→3: SEOで新規流入した来訪者がデイリーコンテンツを発見しリピーターに転換
- ループ3→2: リピーターが蓄積した体験（スコア、診断結果）をシェア
- ループ2→1: シェア由来の流入がサイト全体のエンゲージメント指標を改善しSEO評価向上

#### フェーズ1（0-6ヶ月: 2026年4月〜10月）— 基盤構築・最初のループ回転

**ドメインの状態:** ドメイン年齢2ヶ月。Google評価フェーズ（いわゆるサンドボックス期）。低競合KWでのみ表示機会あり。OS月299PV。

**フェーズ1の目標:**

- OS PV: 299 → 600〜1,000/月（根拠: Semrush 28,000ドメイン研究では全体の4.2%がTop10を経験し、そのうちの約41%が6ヶ月時点で到達（全体の約1.7%）。ただしこれは全難易度KWの平均であり、低競合KWでは成功率が高い。yolos.netは低競合KW（改行削除ツール等）に集中するため、新規コンテンツの一定割合が6ヶ月以内にTop10入りすると想定）
- Direct PV: 238 → 400〜800/月（ただしフェーズ1後半に効果が集中）（根拠: 現状DAU約8人（Direct 238PV÷30日）。施策別の追加DAU見込み: タイピングゲーム（デイリーチャレンジモード）+2-5人/日（cycle-166で導入、3ヶ月運用後の値）、デイリークイズ+3-8人/日（cycle-168で導入、3ヶ月運用後の値）、シェア経由の新規→リピーター転換+1-3人/日。合計でフェーズ1末のDAU 14-24人。月間Direct PV: 14-24人×30日×1.5PV/訪問=630-1,080PV。ただしデイリーコンテンツの導入がcycle-166/168であるためフェーズ1の前半はDAU増加なし）
- 合計: 608 → 1,200〜2,000/月

**フェーズ1の戦略方針: 「来訪者が毎日来る理由」と「来訪者がシェアする理由」を作る**

現時点で最も不足しているのは、ループ2（シェア機構）とループ3（デイリーコンテンツ）。ループ1（SEO）は既に部分的に機能している（OS 299PV）ため、ループ2・3の構築を優先する。

**具体的アクション（優先順序）:**

1. **テーマ特化タイピングゲームの新規作成**（ループ1+2+3の全てに寄与）
   - 来訪者の何の問題を解決するか: 「楽しみながら四字熟語・漢字を覚えたい」「タイピング練習をしたい」という来訪者に、既存サイトにないテーマ特化型のタイピング体験を提供する
   - 具体的成果物: 「四字熟語タイピング道場」— 四字熟語辞典（400語）を問題データベースとして活用したタイピングゲーム
   - 差別化要素: aramakijake実測で「四字熟語 タイピング」はデータなし（月100未満）。ただし「タイピング ゲーム」（月8.8万）「タイピング 練習」（月24万）の巨大市場内でテーマ特化型のニッチとして差別化。既存の四字熟語辞典400語が問題データベースになる
   - ループへの貢献: ループ1のSEO流入の主軸は「四字熟語 タイピング」ではなく「タイピング ゲーム 四字熟語」等のロングテールKWと「タイピング 練習 無料」（月2.6万）への間接的貢献。ループ2（スコアシェア）、ループ3（デイリーチャレンジモード）
   - 期待効果: SEO流入は限定的（月50-200PV）だが、リテンションループ（デイリーチャレンジモード）とバイラルループ（スコアシェア）による間接効果が主要な貢献。合計で3ヶ月後+200〜500PV/月（推測）

2. **既存診断コンテンツへのシェア機構追加**（ループ2の着火剤）
   - 来訪者の何の問題を解決するか: 「診断結果を友達に見せたい・比べたい」という来訪者の欲求を満たす
   - 具体的成果物: 全診断の結果ページに、事前入力テキスト付きX(Twitter)シェアボタンを追加。結果をポジティブにフレーミングし、シェアしたくなる表現に改善
   - 根拠: 診断コンテンツの直帰率0%（GA実測値）はエンゲージメントの高さを示す。BuzzFeedクイズではトラフィックの75%がシェア由来（実測データ）。パーソナリティテストは他コンテンツより40%多くシェアされる（Quizlead調査）
   - ループへの貢献: ループ2の「体験→シェア」ステップの離脱率を下げる
   - 期待効果: シェア機構を追加してバイラルループの起点を作る。シェア率（診断完了者のうちシェアする割合）は業界平均で2-5%程度だが、yolos.netでの実効値は未知。効果はSNSシェア数（実装時にカウント機構追加）とReferral/Organic Social流入の月次追跡で測定する。フェーズ1の目標はシェア率を測定可能にすること自体である

3. **毎日更新型クイズの導入**（ループ3の構築）
   - 来訪者の何の問題を解決するか: 「毎日ちょっとした知識チャレンジを楽しみたい」という来訪者に、習慣的に楽しめるコンテンツを提供する
   - 具体的成果物: 「今日の四字熟語クイズ」— 毎日1問、四字熟語の意味を当てるクイズ。既存辞典データを問題ソースとして使用。正解すると翌日の問題が楽しみになる設計
   - ループへの貢献: ループ3の「デイリーコンテンツ→習慣形成→Direct流入」を実現
   - 期待効果: 6ヶ月でデイリーリピーター10-30人 → 月300〜900PV増（推測。保守的試算: 10人×30日×1PV=300、楽観的試算: 30人×30日×1PV=900）

4. **低競合KWをターゲットとした新規ツール2-3本の追加**（ループ1の強化）
   - 来訪者の何の問題を解決するか: aramakijake実測で需要を確認し、SERPで競合が弱い領域のツールを作成。cycle-164の改行削除ツールと同じ手法
   - 具体的成果物: 各ツールの選定はcycle-163の調査データ（docs/research/2026-04-07-keyword-serp-research-all-categories.md）の「需要あり×競合弱い」マトリクスから選択。実行時にaramakijake再実測+SERP確認を必ず行う
   - ループへの貢献: ループ1の入口を増やす
   - 期待効果: ツールあたり月100-300PV増（推測。改行削除ツールの実績データが出てから精度が上がる）

5. **診断結果ページのテキスト量充実**（ループ1の改善）
   - 来訪者の何の問題を解決するか: 「自分の診断タイプについてもっと詳しく知りたい」という来訪者に、深い解説を提供する。cycle-159の教訓を踏まえ、SEOのためのテキスト追加ではなく、来訪者が求める情報を充実させる
   - 具体的成果物: 結果ページの本文を現在の300-400文字から1,200-2,000文字に拡充（競合平均の水準）。ただし追加コンテンツはツール本体の下に配置（cycle-160の3ゾーン設計に準拠）
   - 根拠: 競合（hoyme.jp, uranaino.net等）の結果ページは1,200〜5,500文字。yolos.netの結果ページへのOS流入はゼロ（GA実測値）
   - 注意: cycle-159の失敗を反復しないこと。追加テキストが「来訪者がその瞬間に求めている情報」であることを設計時に検証する。ファーストビューにツール/結果表示を維持する。具体的なチェックポイント（実行サイクルで必須）:
     - 追加するテキストの各段落について、その段落が解決する来訪者の疑問を1文で明記できるか検証する。明記できない段落は来訪者価値がない
     - モバイル375x667pxで診断結果がファーストビューに収まることをPlaywrightスクリーンショットで確認（結果ページの上部に結果表示、追加テキストはスクロール後に配置）
     - 追加テキストの草案をreviewerに来訪者目線でレビューさせ、「この情報を今知りたいか」の観点で評価する
   - 期待効果: 6ヶ月後に診断結果ページからのOS流入が月50-200PV（推測。ドメイン年齢が6ヶ月に達し、ロングテールKWでランクインし始める前提）

**フェーズ2の戦略方針: 「フライホイールの加速」**

フェーズ1で構築した3ループが接続し始めるフェーズ。各ループ間の接続を強化し、フライホイールの回転速度を上げる。

**具体的アクション:**

1. **MBTI/16タイプ深掘りコンテンツ群の展開**（ループ1+2の大幅強化）

1. **MBTIコンテンツをフェーズ1で着手する案**

#### 指摘事項

**M-01（修正必須）: タイピングゲームの期待効果「3ヶ月で+500〜1,000PV/月」の根拠が不十分**

計画にも「推測。根拠: ニッチKWのSEO流入+デイリーリピーター。ただし類似施策の実績データはyolos.net上にない」と正直に書かれているが、問題はこの推測値が計画の優先順位付けの根拠として使われていることである。新規ジャンル調査レポートでは「四字熟語タイピング」のニッチKWは「数百〜数千/月規模」と記載されており、aramakijakeでの実測値が記載されていない（「ニッチKWは数百〜数千/月規模」は推測）。cycle-166の「実行前に検証すべき前提」にaramakijake実測が含まれているのは良いが、**計画策定時点で実測できるデータを「実行前に検証」と先送りしている構造はAP-P01に該当する**。

修正案: 「四字熟語 タイピング」「漢字 タイピング」等のKVをaramakijakeで実測し、その結果を計画に反映するか、実測前であることを核心的前提に追加する（現在の3つの核心的前提に追加して4つ目とする）。

**M-02（修正必須）: フェーズ1の目標値「OS PV 299→600〜1,000/月」の根拠が曖昧**

根拠として「Semrush研究『6ヶ月でTop10入り41%』を低競合KWに限定した場合の現実値」と記載されているが、Semrush研究の41%は「28,000ドメインのうちTop10に少なくとも1回入った4.2%の中で、6ヶ月時点で到達した割合」であり、28,000ドメイン全体の41%ではない。ドメイン成長フェーズ研究レポートの元データ（行42-43）を確認すると、「少なくとも1回Top10圏内に入ったドメイン: 4.2%」「6ヶ月時点でTop10に到達したドメイン: 約41%」と並記されており、後者は前者の中での比率（つまり28,000ドメインの4.2%=約1,176ドメインの41%=約482ドメイン）と読める。この場合、全体の約1.7%しか6ヶ月でTop10に到達していない。「6ヶ月でTop10入り41%」という引用は原文のニュアンスを歪めている可能性がある。

修正案: Semrush研究データの正確な引用に修正する。「28,000ドメインのうち4.2%がTop10を経験し、そのうちの約41%が6ヶ月時点で到達」のように、母集団を明示する。これに基づいてOS PV目標の根拠を再検討する。

**M-03（修正必須）: デイリーリピーター「10-30人」獲得の根拠が提示されていない**

フェーズ1のDirect PV目標の根拠が「デイリーコンテンツ導入によるリピーター10-30人獲得の保守的試算」とあるが、この10-30人がどのように獲得されるかの試算が示されていない。フライホイール研究レポート（セクション3.4）では「現状のDirect流入238PVを30日で割ると約8人/日（DAU換算）」であり、10-30人は現状の1.25〜3.75倍に相当する。この増加を6ヶ月で達成できる根拠が必要。特に、デイリーコンテンツ（毎日更新クイズ）はcycle-168で導入予定であり、フェーズ1の6ヶ月のうち最初の数サイクルはデイリーコンテンツが存在しない期間がある。

修正案: リピーター獲得の経路と時間軸を明記する。例えば「cycle-166のタイピングゲーム（デイリーチャレンジモード付き）で+3-5人/日、cycle-168のデイリークイズで+5-10人/日、合計でフェーズ1末に10-20人/日が現実的」のように、施策ごとの寄与を分解する。

**M-04（修正必須）: 「バイラル係数0.05〜0.1」の根拠が薄い**

シェア機構追加（施策2）の期待効果として「バイラル係数を0.0から0.05〜0.1に引き上げ（100人の訪問者のうち5〜10人がシェアする想定）」とあるが、バイラル係数の定義（1人のユーザーが生み出す新規ユーザー数）と「シェア率5-10%」は異なる概念。シェア率5-10%であっても、シェアを見た人がクリックする率（CTR）を掛ける必要がある。フライホイール研究レポートで引用されたBuzzFeedの「トラフィック75%がシェア由来」はBuzzFeedの圧倒的なリーチが前提であり、月間608PVのサイトに適用できない。

修正案: バイラル係数の定義を正確に使い、「シェア率」と「シェアからの流入率」を分けて記載する。あるいはバイラル係数の具体的数値を削除し、「シェア機構を追加してバイラルループの起点を作る。効果の測定はSNSシェア数とReferral流入の月次追跡で行う」のような表現に変更する。

**S-01（提案）: フェーズ2のMBTIコンテンツ「ロングテールKW 100ページ規模で月5,000〜20,000PV」の幅が広すぎる**

4倍の幅（5,000〜20,000）は予測としての有用性が低い。フェーズ2はまだ先であり不確実性は高いが、「ドメイン権威の成長次第で大幅に変動する」とあるなら、変動要因をもう少し具体的に記述する（例: 「DA 10到達時は5,000PV、DA 15到達時は10,000-15,000PV」など）か、フェーズ2の計画はフェーズ1終了時のデータで再策定する旨を明記する方がよい。

**S-02（提案）: フェーズ3の目標「月間10,000〜30,000PV」の根拠として引用されている事例の適用可否が不明確**

「日本語ブログ複利事例『1年後1万PV→2年後3万PV』」の出典はweb4mom.comだが、これは記事を月数十本ペースで投稿するブログの事例。yolos.netはインタラクティブコンテンツ（ゲーム・ツール・診断）中心であり、記事ベースのブログとは成長ダイナミクスが異なる可能性がある。「omnius.so事例『7ヶ月で月6万訪問』」はSaaS AIツールであり、yolos.netにより近い参考事例だが、305キーワードに集中した大量記事投稿（月20記事）が戦略の柱であった。この違いを計画内で認識しておくべき。

**S-03（提案）: 施策5「診断結果ページのテキスト量充実」のcycle-159との差異をより明確にする**

現在の記述では「SEOのためのテキスト追加ではなく、来訪者が求める情報を充実させる」「ファーストビューにツール/結果表示を維持する」と注意書きがあるが、cycle-159でも計画段階では同様の注意書きがあった（cycle-159失敗分析レポート: 「来訪者にとって実際に役立つ内容にすること」）。注意書きだけでは実行時に同じ失敗を繰り返すリスクがある。**具体的なチェックポイント**（例: 「追加するテキストの各段落について、その段落が解決する来訪者の疑問を1文で明記できるか検証する」「モバイル375x667pxで結果表示がファーストビューに収まるかのスクリーンショット確認を必須とする」）を計画に含めることを推奨する。

**N-01（軽微）: 核心的前提2の記述の正確性**

「Wordle/Spotle/Globale等の事例で訪問者の88-90%がDirect/Returnだったことは実測データ」とあるが、フライホイール研究レポートでは出典が「Medium記事（Money Tent）」であり、これは一次データ（Spotle/Globale運営者の公式データ）ではなく二次データ（サードパーティの推定）である。「実測データ」ではなく「サードパーティ推定値」と記載すべき。

**N-02（軽微）: アンチパターンチェック AP-P01の対応状況**

「cycle-166実行前にaramakijake+SERP確認を必須化」と記載しているが、これはまさにAP-P01が警告する「実装フェーズで検証する」先送りパターンに該当する。M-01と同根の問題。AP-P01の対応状況は「対応済み」ではなく「計画策定時点で実測可能なデータ（四字熟語タイピングのKV）は未検証。cycle-166の実行前検証で補完予定」と正直に記載すべき。

**N-03（軽微・修正不要）: Semrush研究の41%の母集団が依然として不正確**
