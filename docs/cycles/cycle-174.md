# サイクル-174

このサイクルでは、**B-332「サイト内検索の方針判断（移行計画 Phase 1.1、設計判断版）」** を実施します。
新コンセプト「日常の傍にある道具」下のターゲット（M1a 初回ツール利用者 / M1b リピーター）に対して、新デザインに横断検索（サイト内全コンテンツを跨いだ検索）を作るかどうかを、二択の設計判断として答えます。判断結果が後続フェーズ（B-334 Phase 4 / B-331 Phase 5 / B-335 Phase 6 …）の着手スコープを決めるため、ここを通さないと移行が進みません。

## 背景（経緯と前提）

- **cycle-172 / cycle-173 は両方失敗認定**（2026-05-01 Owner 指摘）。
  - cycle-172: 「データを 30 日蓄積してから判断」と決めたが、この問いはデータで答えられる種類ではなく、ターゲット動線の設計判断だった。
  - cycle-173: 失敗の前提のまま GA4 計測強化を実装したため、「30 日後に判断する」という根拠ごと崩れた。
  - 旧 (a)/(b)/(c) の 3 分割（横断検索を作る / 作らない / セクション内検索のみ）は誤りで、実態は **二択**（横断検索を「新デザインに」作るか / 作らないか）。
  - 旧分割では「移行期間中の動線継続性」が考慮されておらず、② を選ぶ場合は Phase 7 以降で横断検索アクセスが段階的に縮小されることを許容できる設計か確認する必要がある。
- **本サイクルの問い**:
  - ① 新デザインに横断検索を作る（B-331 Phase 5 で実装する）
  - ② 新デザインに横断検索を作らない（B-331 Phase 5 をスキップ）
- **本判断とは独立に必須実装される項目**:
  - ツール内・遊び内の絞り込み（一覧ページ内フィルタ）は Phase 4 の通常スコープで必ず実装する。本判断とは無関係。
- **②（作らない）を選んだ場合の自然な帰結**:
  - 移行期間中、横断検索にアクセスできるページは Phase 7 以降で段階的に減る（legacy ページが新デザインに置き換わるたびに横断検索エントリが消える）。これが許容できる設計か確認する。
  - レガシー検索の物理撤去は Phase 10.2 のタイミングで行われる自然な帰結。
- **B-340 で実装した GA4 計測**（search_modal_open / close / result_click / abandoned）は **本サイクルの判断材料ではなく**、新コンセプト公開後の利用実態観測 infrastructure として残置している。本判断は計測データを根拠としない（cycle-173 の失敗を踏まえる）。コードを残すかは本サイクルの判断結果次第（① なら有用、② なら次サイクルで legacy 撤去と一緒に削除を検討）。
