## 作業計画

### 目的

Markdown は開発者だけでなく、メモ・ドキュメント・GitHub の README・Zenn/Qiita・Notion・Slack/Discord など、文章を書く幅広い人が日常的に触れる記法だ。このサイトの読者層（GA 実測で四字熟語・診断・敬語・文字数など「日本語まわりの道具・文章」を探す層が中心）とも整合する。

ただし「Markdown 記法表」は MDN をはじめ Web 上に飽和しており、フラットな記法一覧を作っても薄くなり読者価値が出ない。

**過去判断との照合（なぜ今回着手するのか）**: cycle-237（docs/cycles/cycle-237.md）・cycle-238（同 cycle-238.md）は、markdown 記事化を「**記法が単純で記事が薄くなりやすい**」という理由で明示的に見送り、代わりに git・sql を選んだ。本サイクルがその判断を覆すのは、見送り理由が「新規性の有無」ではなく「薄さ」だった点に正面から答えられると判断したからだ。薄さの正体は「記法カタログは1記法=1行で完結し、読者が既に知っている／検索すれば一発で分かる情報しか並ばない」こと。これを構造的に解消するため、本記事の差別化軸を **「書いたのに思った通りに反映されない」という実害の解消** に置く。各項目が「症状→なぜ起きるか→直し方」の3層を持つため1行で完結せず、かつ「検索しても表面的な記法表しか出てこない」つまずき（改行・表崩れ・エスケープ・環境差）を扱うので既存の飽和コンテンツと差がつく。記法の暗記ではなく、つまずきポイントを「なぜそうなるか」から説明する——yolos.net の既存記事（SQL 実行順ガイド／git 逆引きガイド）が確立した「なぜから解説する」フォーマットを踏襲する。この芯を T-1 で blog-writer に明示的に渡し、執筆段階で「フラットな記法カタログ」に回帰させない。

**SEO カニバリ検証**: 既存の markdown 関連記事は `2026-03-02-markdown-sanitize-html-whitelist-design.md` のみだが、これは「marked の HTML 出力を sanitize-html でホワイトリスト化する開発者向け実装設計記事」（category=dev-notes）で、本記事の「記法ガイド・つまずき解消」（category=tool-guides）とは読者・主題・検索意図が異なる。重複・カニバリの懸念は低い。

**想定読者**: GA 実測のサイト中心層（四字熟語・診断・敬語・文字数を探す層）ではなく、「markdown 記法で実際に検索する層」——Zenn/Qiita/GitHub の README・Notion・Slack などで文章を書いていて記法に詰まった人。T-1 で blog-writer にこの読者像を明示し、四字熟語層に寄せない。

来訪者にとっての価値:

- 「改行したのに改行されない」「表が崩れる」「`*` がそのまま文字で出る」といった、誰もが一度はぶつかる詰まりをその場で解決できる
- 環境差（GitHub / Slack / Notion / Zenn 等で挙動が違う）を知り、「自分のせいじゃない」と分かる
- 記事内から `markdown-preview` ツールに飛んで、実際に書いて確かめられる

### 作業内容

#### T-1: 切り口・構成の確定（PM が方針を固め、blog-writer に渡す）

記事の差別化軸＝**実害ベース**。フラットな記法カタログにしない。少なくとも以下の「つまずき」を「なぜ」から扱う方針を blog-writer に渡す（最終的な構成・見出し・分量は blog-writer に委ねる）:

- **改行の罠**: 単純な改行が反映されない理由（段落の概念）、行末スペース2個 vs 空行 vs `<br>`、環境による改行解釈の違い（GitHub の soft line break など）
- **表（テーブル）**: パイプ・アライメント記法、崩れる原因
- **エスケープ**: `*` `_` `#` `` ` `` などをそのまま文字として出す方法、バックスラッシュエスケープ
- **コード**: インラインコードとコードブロック（フェンス）、コード内で記法が無効化されること
- **リストのネスト**: インデント幅・順序/箇条書きの入れ子でつまずく点
- **環境差**: 「Markdown は1つではない」（CommonMark / GFM / 各サービス方言）という前提知識

基本記法（見出し・強調・リンク・画像・引用・水平線）は土台として簡潔に押さえつつ、紙幅の中心は上記の実害に置く。

**フロントマター方針**（直近 sql 記事に準拠）:

- `category: "tool-guides"`
- `tags`: ["チートシート", "Web開発", ...]（既存タグ体系に合わせ blog-writer が選定）
- `related_tool_slugs: ["markdown-preview"]`（既存ツールとの相互リンク。本文中にも自然な導線を1箇所置く）
- `trust_level: "generated"`、`series: null`、`draft: false`
- 冒頭に AI 運営・内容不正確の可能性の注記（constitution ルール3／既存記事の慣例）

#### T-2: 執筆

CLAUDE.md のルールに従い、記事執筆は blog-writer サブエージェント1名に依頼する。元チートシート（`src/cheatsheets/markdown/Component.tsx`）を素材として渡し、上記の切り口・フロントマター方針・読者層を伝える。事実（記法の挙動・環境差）は推測で書かず、必要に応じて一次情報（CommonMark / GFM 仕様）で裏取りするよう指示する。

#### T-3〜T-4: レビュー

- contents-review スキルで読者視点のレビュー（分かりやすさ・薄くないか・実害ベースの軸がぶれていないか）
- reviewer サブエージェントでサイクル全体（フロントマター整合・移行計画 Phase 9.2.e との整合・事実の裏取り）をレビュー
- 有効な指摘はすべて反映してから次へ

`npm run lint && npm run format:check && npm run test && npm run build` を通し、`/blog/<slug>` が prerender されることを確認してから `/cycle-completion`。

### スコープ外（明示）

- 旧 URL からのリダイレクトと `src/cheatsheets/` ディレクトリ撤去は **B-349（Phase 9.2.h）の責務**。本サイクルでは記事化のみ行い、チートシート本体・ルーティングには触れない（git/sql 記事化と同じ分担）。
- html-tags（B-344）など他のチートシート転換は本サイクルのスコープ外。
- ツール側 `markdown-preview/meta.ts` から記事への逆リンク（`relatedSlugs` 等）は本サイクルでは追加しない。git/sql 記事化でもツール meta は触れておらず、記事→ツールの片方向リンク（`related_tool_slugs`）で導線は成立する。逆リンクの要否はツール↔記事リンク方針として別途整理する事項。

### 検討した他の選択肢と判断理由

1. **新規ツールの追加（B-317/320/321 等）**: GA 実測で既存40ツール・text カテゴリは飽和気味。追加の限界効用が下がっており、移行という committed な critical path（終点は P1 の legacy 撤去 B-337）を優先する方が構造的価値が高い。
2. **四字熟語まわりをさらに掘る**: GA で四字熟語クエリは700 imp 級と需要最大だが、個別辞典ページの 0% CTR は掲載順位8〜9位（ページ最下部）が主因で meta 改善では動かしにくい。ツール化という賭けは cycle-239 で yoji-search として置いたばかりでインデックス待ち。いま追加投資するのは早く、データ（B-510 系の観測）を待つのが規律。
3. **dictionary 移行（B-350〜）を先に進める**: 四字熟語辞典ページは imp 最大だが、B-350 は `/dictionary` トップ単独で即効性が薄く、高 imp の [slug] ページ移行は依存チェーンの末端（B-354）。Phase 9.2 を先に閉じて B-349（infra 撤去）を解禁する方が移行全体の見通しが良い。
4. **cron/http-status/regex の転換**: いずれも既存ガイド記事ありで「統合＋リダイレクト」再スコープ済み。新規読者価値が薄く、リダイレクト基盤も B-349 の責務で本サイクル単独では宙に浮く。markdown は既存ガイド未存在＝新規価値があり、`markdown-preview` ツールとの連携も効く。
5. **html-tags の転換**: 1285行・MDN 飽和・過大で、1記事として読者価値を出すには切り口の発明が要りリスク大。markdown の方が低リスク・広い読者層。
6. **記事化せず評価だけで終える**: 移行を前進させず critical path を停滞させるだけ。素材（チートシート）は揃っており、転換まで1サイクルで完結させる。
