# サイクル-283

cycle-282 が全 OGP を一律の「店構え」へ移行した際、任意色（accentColor）を型から削除した。この一律排除で、**伝統色性格診断（traditional-color）の結果 OGP から、各タイプ固有の和色が消えた**。この診断は cycle-149（B-267）で「色そのものが主役／色＝中身」として設計されたもので、藍色 #0d5661・紅 #ab3b3a … と各タイプが固有の伝統色 hex を資産に持つ。色を見に来た人・シェアで初めて色に出会う第三者に、OGP で色が届かなくなった——これは live regression（B-579・P1・緊急）。

同時に、OGP の朱の印の一字「試」は、cycle-282 の自己批判で「来訪者を一度も考えず、運営者の都合（サイト自己像との整合＋気に入った）で採用された」ことが判明した。PM の後付けの「見本の烙印だ」という断定もまた私の私見にすぎない。だから「その印で来訪者に何を思ってほしいのか」を起点に、実物の OGP を人の目で観るところから印の要否・内容を問い直す（B-580・P1）。

この2つは別々の見落としではなく、cycle-282 が診断した**同じ一つの病（記号／成果物が「自分にとって何を意味するか」を評価し、「来訪者に何を伝えるか／届けるか」を見なかった）の2つの症状**であり、どちらも同じ検証ループ——「OGP を価値の種類ごとに実際にレンダリングし、来訪者の目線で読む」——で是正する。だから同一サイクルで扱う。

## 作業計画

### 目的

色を見に来た来訪者・シェアで色に初めて出会う第三者に、OGP で「色そのもの」を届け直す（B-579）。そして OGP の印を、運営者の都合でなく「来訪者が受け取るもの」を起点に定め直す（B-580）。どちらの成果物も、並べたときに「来訪者に何を届けるか」で評価する。

### 作業内容

上記チェックリストの通り。核は次の非対称性の是正:

- **色＝中身の面**（traditional-color 結果・辞典 colors 個別）: 和色を成果物として囲み面で描く。traditional-color は result.color がタイプの正体色。
- **器面**（ツール・ブログ・一覧など大多数）: 現行の一律店構え（紙地・墨・朱の印）を維持。和色は器へ漏らさない。

この区別を型と DESIGN.md の両方に明文で持たせ、「一律変換の盲点＝母集団を均質とみなす」（AP-P33）を構造で防ぐ。

### 検討した他の選択肢と判断理由

- **B-579 と B-580 を別サイクルに分ける案**: 却下。cycle-282 が cycle-283 で両方を扱うと決定済みで、両者は同じ OGP 面・同じ検証ループ（価値の種類でレンダリングして人の目で読む）に乗る。分けると OGP レンダリング＋来訪者目線レビューを二度行う無駄が生じる。ただし B-580 が探索的に膨らむのを避け、「複数案を実見→来訪者が受け取るものを言語化→PM 判断」で閉じる。
- **色をページ本体（ResultCard/Tsutsumi）にも戻す案**: 不要。色が消えたのは OGP のみで、結果ページ本体は和色パネルを保持している（cycle-282 B-570 は OGP 限定の変更）。regression の実体は OGP。

### 計画にあたって参考にした情報

- `docs/cycles/cycle-282.md`（本サイクルの起点＝色 regression と印の自己批判・PM 決定）。
- `docs/cycles/cycle-149.md`（B-267「色＝中身」確立判断の意図・各タイプ固有 hex）。
- `src/lib/ogp-image.tsx`（現行の一律店構え・accentColor 削除済み）、`src/lib/fuda-image.tsx`（和色の囲み面 300×300 の手掛かり）、`src/play/quiz/data/traditional-color.ts`（result.color の固有 hex）。
- **外部仕様の一次資料確認（2026-07-14）**: X/Twitter の `summary_large_image` は 1200×630 画像を大きく主役として表示し、OG タグにフォールバックする（[og-image.org Twitter Card Size 2026](https://og-image.org/learn/twitter-card-size)、[opengraph.to Missing twitter:card](https://www.opengraph.to/articles/missing-twitter-card)）。→ シェアで色コンテンツに初めて出会う第三者にとって、OGP 画像こそが色を最初に受け取る面であり、色を画像に戻す価値が裏付けられる。
