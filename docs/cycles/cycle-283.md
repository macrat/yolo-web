# サイクル-283

cycle-282 が全 OGP を一律の「店構え」へ移行した際、任意色（accentColor）を型から削除した。この一律排除で、**伝統色性格診断（traditional-color）の結果 OGP から、各タイプ固有の和色が消えた**。この診断は cycle-149（B-267）で「色そのものが主役／色＝中身」として設計されたもので、藍色 #0d5661・紅 #ab3b3a … と各タイプが固有の伝統色 hex を資産に持つ。色を見に来た人・シェアで初めて色に出会う第三者に、OGP で色が届かなくなった——これは live regression（B-579・P1・緊急）。

同時に、OGP の朱の印の一字「試」は、cycle-282 の自己批判で「来訪者を一度も考えず、運営者の都合（サイト自己像との整合＋気に入った）で採用された」ことが判明した。PM の後付けの「見本の烙印だ」という断定もまた私の私見にすぎない。だから「その印で来訪者に何を思ってほしいのか」を起点に、実物の OGP を人の目で観るところから印の要否・内容を問い直す（B-580・P1）。

この2つは別々の見落としではなく、cycle-282 が診断した**同じ一つの病（記号／成果物が「自分にとって何を意味するか」を評価し、「来訪者に何を伝えるか／届けるか」を見なかった）の2つの症状**であり、どちらも同じ検証ループ——「OGP を価値の種類ごとに実際にレンダリングし、来訪者の目線で読む」——で是正する。だから同一サイクルで扱う。

2. **最大レバレッジは dictionary/colors 個別**（色面で最大流入 63PV・検索駆動・OGP ゼロ）。ここへ固有色 OGP を新設するのが、色コンテンツ OGP で最も価値の高い一手（最大面かつ純増・拡散ループの種）。regression 修復と同じ「固有色の囲み面」機構で共有できる。

> **⚠️ この節の結論（試→屋への変更）は撤回済み。** 下記末尾「事故報告」を参照。「屋」は「どの字が一番『店』を伝えるか」という運営者目線の判断で来訪者価値がゼロだった（cycle-282 の病の再演）。コードごと撤回し着手前の "試" へ戻し、印の要否・店構え統一自体の再検討は B-583 へ回した。以下は撤回された経緯の記録として残す。

- **決定: 生成看板・辞典の家印を「試」→「屋」に変更。診断結果は「診」（製品の印）を維持。** コード変更あり（`ogp-image.tsx` SHOP_SEAL_CHAR・辞典 opengraph の定数・関連テストの assertion）。
- **屋を採る来訪者起点の根拠**: (1) 印の仕事は店の身元・家印（出所は店号 yolos.net が担い、印が装飾的に補強）。「屋」はその身元＝よろず屋の「屋」を誤読なく即座に伝える。(2)「試」は真下の副題と意味が重複し、印が独自の情報を足していない。「屋」は副題（＝実験の気配）と非重複で補い合う。(3)「試（＝試作／暫定）」は参照面（伝統色辞典＝色データの信頼を掲げる面）に最も座りの悪い字で、微妙に中身の信頼を削る——私と実装外レビューが独立に指摘した実問題。「屋」はこれを解消。
- **却下理由（実物を見た上で）**: 印なし＝店構えの署名が消え汎用化（E）。円環のみ＝未完成に見える（D）。萬＝数字誤読の危険（B）。試＝上記の重複＋参照面の座りの悪さ（A）。
- **cycle-282 との違い（今度こそ本物）**: 前回は「気に入った」だけで来訪者を見ずに採用。今回は**印の扱い自体の 5 案を実 OGP でレンダーし**、各案が来訪者に届けるものを言語化した上で、旧字「試」を能動的に退けて「屋」へ変えた。「記号 vs 自分」でなく「人 vs 成果物（実物 5 枚）」の照合。
- **正直な限定**: OGP 印の露出は現状ほぼゼロ（シェア流入実質ゼロ）。よってこれは craft／誠実さ／整合の判断で、測定されたトラフィックの判断ではない。ただし辞典色 OGP は「拡散ループの種」として露出を育てる意図で新設した面であり、そこに座りの悪い「試」を残さず「屋」に整えることは、育てる面の質を先に整える意味で整合する（初版にあった「露出を育てる面に不適字を残す」矛盾を解消）。
