## なぜ今これをやるのか

`docs/targets/` は cycle-kickoff スキル自身が毎サイクル参照し、「ターゲットユーザーを確認し、彼らのために最高の価値を提供せよ」と命じる**生きた一次文書**だ。ここが間違っていれば、以降の全サイクルが間違った相手に向けて作られ続ける。

現状、targets/ は次の点で実測とコンセプトに反している（cycle-259 の「重大発見」）:

- **メイン2件がいずれもツール利用者**で、実測の主役である**診断来訪者を表すターゲットが存在しない**。cycle-257 実測: `/play`（診断・ゲーム）が organic 着地 184/201（最大）・滞在の深さも他の15〜20倍で**集客と滞在の両軸で頑健1位**。直近の organic 流入増も「似てるキャラ診断」系で、`/play/character-personality` への一点突破。一方ツール詳細 `/tools` は organic n小で判断保留。
- **メイン M1b「気に入った道具を繰り返し使っている人」は反復利用を前提**にしているが、実測では再訪はサイト全体で不成立（returning が全セクション一桁人）。コンセプトも「価値は単発訪問の質で測る／『また来たくなる仕掛け』より『この一回を最高にする』を優先」と確定済み。前提が崩れている。
- 「サイトの中心は『道具』」「息抜きは全体の1割を超えない」という旧枠組みの記述が README・S3 説明に残る。

cycle-259 は README に警告注記を入れて即時の誤導を止めたが、**定義本体は旧コンセプトのまま**。これは辞典の「80字」誤情報（B-536）や site-concept の旧枠組み（B-538）と同型の**内部の確定誤情報**で、CLAUDE.md が警告する「未検証情報がサブエージェントに渡り将来サイクルを汚染する」状態を、最上位の参照文書の一つで起こしている。だから本体の是正を急ぐ。

**なぜチェックを回避したか（観測事実ベース）**: タスクを「cycle-257 で確定済みの判定を文書へ整合させるだけの内部文書整備」と枠付けし、市場判断の要素を伴わない作業とみなした。その枠付けの結果、計画段階で「外部一次資料の確認は不要」と書き、需要×競合の確認を設計から外した。なお、このサイトはツール制作前には competitor research を毎回実施している（password-generator・text-diff・regex-tester 等の調査ファイルが存在）にもかかわらず、最重要の主軸ターゲット決定でその確立した実務を適用しなかった。

**是正（実施済み）**:

1. 性格診断・キャラ診断の日本市場について、需要規模・競合ランドスケープ・自サイトSWOT を2エージェントで Web 調査（出典URL＋確認日付き）し、`docs/research/2026-06-22-diagnostic-market-competitive-analysis.md` に統合。
2. 調査結果（需要は巨大かつ定着／主動機は会話のネタ・シェア／head語と相性はSEO寡占で勝ち筋は「文化×診断＋ロングテール＋シェア＋登録不要モバイル1回完結」）を、診断メインペルソナの knows/interests/likes/dislikes と README の主軸説明に反映。
3. planning.md の AP-P13 に発生記録を追記。

**結果として何が変わったか**: 主軸＝診断の判断は内部119PVだけでなく外部の大きく定着した需要でも裏付けられ、かつ「head語の正面突破は非現実的・勝ち筋は文化×診断とロングテール」という競合理解が加わって、ペルソナが市場の現実に即して鋭くなった。文化層ターゲットを残す判断も「文化×診断が競合空白」という市場面で補強された。

**構造的教訓**: 「既存判定の文書整合」に見えるタスクでも、**それが将来投資の対象（誰に作るか）を確定するなら市場判断であり、需要×競合の外部調査を計画段階の必須ステップに含める**。内部実測が方向を示していても、それは市場規模・競合の代替にならない。「外部仕様の確認不要」と「市場・競合の調査不要」は別物で、混同しない。

## 事故報告2（AP-WF24: 是正の駆動源を Owner に帰属／meta 再発）
