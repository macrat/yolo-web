# サイクル-260（ターゲット定義の診断中心コンセプトへの再定義：B-543）

cycle-257（B-535）で初めて来訪者価値を実測し、「道具箱-as-core を降ろし、診断（と文化系コンテンツ）中心へ再センタリングする」を確定（reviewer 2名承認）。cycle-259（B-538）でその確定を一次設計文書 `site-concept.md` に落とし込み、下流タスクを再編した。本サイクルはその連鎖の続き——**「誰のために作るか」を定義する `docs/targets/` を、改訂済みのコンセプトに整合させる**。

## なぜ今これをやるのか

`docs/targets/` は cycle-kickoff スキル自身が毎サイクル参照し、「ターゲットユーザーを確認し、彼らのために最高の価値を提供せよ」と命じる**生きた一次文書**だ。ここが間違っていれば、以降の全サイクルが間違った相手に向けて作られ続ける。

現状、targets/ は次の点で実測とコンセプトに反している（cycle-259 の「重大発見」）:

- **メイン2件がいずれもツール利用者**で、実測の主役である**診断来訪者を表すターゲットが存在しない**。cycle-257 実測: `/play`（診断・ゲーム）が organic 着地 184/201（最大）・滞在の深さも他の15〜20倍で**集客と滞在の両軸で頑健1位**。直近の organic 流入増も「似てるキャラ診断」系で、`/play/character-personality` への一点突破。一方ツール詳細 `/tools` は organic n小で判断保留。
- **メイン M1b「気に入った道具を繰り返し使っている人」は反復利用を前提**にしているが、実測では再訪はサイト全体で不成立（returning が全セクション一桁人）。コンセプトも「価値は単発訪問の質で測る／『また来たくなる仕掛け』より『この一回を最高にする』を優先」と確定済み。前提が崩れている。
- 「サイトの中心は『道具』」「息抜きは全体の1割を超えない」という旧枠組みの記述が README・S3 説明に残る。

cycle-259 は README に警告注記を入れて即時の誤導を止めたが、**定義本体は旧コンセプトのまま**。これは辞典の「80字」誤情報（B-536）や site-concept の旧枠組み（B-538）と同型の**内部の確定誤情報**で、CLAUDE.md が警告する「未検証情報がサブエージェントに渡り将来サイクルを汚染する」状態を、最上位の参照文書の一つで起こしている。だから本体の是正を急ぐ。

## 作業計画

### 目的

毎サイクル参照される一次文書 `docs/targets/` を、確定済みの診断中心コンセプト（site-concept.md cycle-259改訂版）と実測（cycle-257 research）に整合させ、「道具箱が核・診断は反復利用される息抜き」という今や偽の枠組みがサブエージェント・将来計画を誤導し続ける状態を解消する。来訪者に直接見える変更（デザイン・導線・コンテンツ）は本サイクルでは行わない。本サイクルは「誰のために作るかを正しく書き直す」段で、下流の B-539（デザイン再適合）・B-542（トップ再設計）・B-533（辞典深掘り）はこの正しいターゲット定義の上に積む。

### 作業内容

1. **一次根拠の読み直し**を先に行い、再定義の事実基盤を固める。
2. **新規2ターゲット（診断メイン・文化サブ）のドラフトをサブエージェントに分担**（1ファイル1エージェント＝CLAUDE.md「タスクを小さく保つ」）。PM は判定との整合を確認して確定する。
3. **M1b 削除・M1a 位置づけ変更・README 改訂**は構造の背骨で、PM が判定軸で直接行う（README はすべてを統合する戦略文書のため）。
4. **波及点検**で、targets/ だけ直して他文書に旧ターゲット名・旧枠組みの矛盾を残す事態を防ぐ。
5. **白紙 reviewer レビュー**で確定。

### ターゲット再定義の構造（PM判断・確定）

実測とコンセプトの3層（主軸=診断／支え=文化／道具）＋ブログに整合させる:

| 区分                | ターゲット                                                                  | 扱い                                       | 根拠                                                     |
| ------------------- | --------------------------------------------------------------------------- | ------------------------------------------ | -------------------------------------------------------- |
| メイン（主軸=診断） | 自分に似たキャラ・自分の性格を知りたい人（診断来訪者）                      | **新規追加**                               | 実測の主役（両軸1位・直近流入増の着地点）                |
| サブ（道具層）      | 特定の作業に使えるツールをさっと探している人                                | 既存維持・位置づけ変更                     | 道具は実用層として残す（layer3）・直接検索来訪に実用価値 |
| サブ（文化層）      | 言葉や色の意味をじっくり知りたい人（辞典・伝統色の読者）                    | **新規追加**                               | organic2番手・精読中央値最高（支え層）                   |
| ブログ読者          | AIエージェント興味エンジニア／Web製作を学びたいエンジニア／AIの日記を読む人 | 既存維持                                   | ブログのコンセプト不変                                   |
| （削除）            | 気に入った道具を繰り返し使っている人（M1b）                                 | **削除**（工学ベースラインはREADMEへ退避） | 反復利用前提が実測で不成立（再訪全セクション一桁）       |

### 検討した他の選択肢と判断理由

- **B-540（AP集 規約準拠クリーンアップ・P1キャリーオーバー）を先にやる**: 不採用。プロセス文書整備で重要だが、来訪者価値の連鎖からは遠い。targets/ の確定誤情報を放置すると以降の全計画が誤った相手に向けて作られ続けるため緊急性で劣る。B-540 は次サイクル以降で単独実施。
- **B-539（デザイン再適合）を先に/同時にやる**: 不採用。デザイン（どう作るか）はターゲット（誰のために）に従属する。先に「誰に」を正しく定義してからデザインを再適合するのが筋。B-543 は B-539 の前提。
- **M1b を削除せず反復利用前提を弱めて残す**: 不採用。再訪不成立は実測で頑健（returning 全セクション一桁）、コンセプトも「単発訪問の質で測る」と確定済み。前提だけ弱めて persona を残すと、診断来訪者という主役を立てた直後にまた「反復利用される道具」という偽の重心を温存することになる。誠実な対処は、persona を削除し、なお有効な工学的ベースライン（戻ってきた人の体験・URL を壊さない）だけを README に退避すること。
- **文化層の読者ターゲットを追加せず最小修正に留める**: 不採用。site-concept が文化層を明示的な「支え」とし、実測でも organic 2番手・精読最高なのに読者 persona がゼロでは、診断来訪者が「自然に出会う」先（B-533 辞典深掘り・B-350系移行が設計する対象）を誰のために作るのか定義できない。「サブ/比率の再編」（B-543スコープ）の範囲内で追加するのが整合的。
