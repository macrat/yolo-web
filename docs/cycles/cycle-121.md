## 作業計画

### 目的

**誰のために**: ブログ読者（サブターゲット「AIエージェントやオーケストレーションに興味があるエンジニア」）

**何のために**: 長い技術記事を読む際に、目次を通じて記事の全体構成を把握し、特定セクションへ素早くジャンプできるようにする。現状ではデスクトップでstickyが壊れており、モバイルでは目次自体が表示されないため、この重要なナビゲーション機能が事実上使えない状態になっている。

**提供する価値**:

- デスクトップ: スクロールしても常にサイドバーに目次が表示され、現在位置の把握と任意セクションへのジャンプが可能になる
- モバイル: 記事冒頭で目次を確認でき、記事の全体像を把握してから読み始められる。セクションへの直接ジャンプも可能になる

### 作業内容

#### 作業1: デスクトップのsticky動作修正

**対象ファイル**: `src/app/globals.css` (57行目付近)

**内容**: `body` に設定されている `overflow-x: hidden` を `overflow-x: clip` に変更する。

**理由**: CSS仕様上、`overflow-x: hidden` は暗黙的に `overflow-y` を `auto` に変更し、bodyにスクロールコンテキストを作成する。これが `position: sticky` の動作を妨げている。`overflow-x: clip` はオーバーフローするコンテンツを非表示にしつつ、スクロールコンテキストを作成しないため、stickyとの互換性がある。

**注意点**:

- `html` にも同じ `overflow-x: hidden` が設定されているため、`html` 側も同様に `clip` に変更する必要がある（html側が `hidden` のままだと同じ問題が残る可能性がある）
- `overflow-x: clip` のブラウザサポート: Chrome 90+、Firefox 81+、Safari 16+。ターゲットユーザー層（エンジニア）を考慮すると十分なサポート範囲
- 変更後、他のページでの水平方向オーバーフロー制御に副作用がないか確認が必要。特にコードブロックやテーブルなど横スクロールが必要な要素への影響を確認する

#### 作業2: モバイル向けインラインTOCの追加

**対象ファイル**:

- `src/app/blog/[slug]/page.tsx` — インラインTOCの配置
- `src/app/blog/[slug]/page.module.css` — レスポンシブ表示切り替え
- `src/blog/_components/TableOfContents.tsx` — コンポーネントの拡張（必要に応じて）
- `src/blog/_components/TableOfContents.module.css` — スタイルの追加（必要に応じて）

**内容**:

1. **インラインTOCの配置**: `page.tsx` の `.layout` div の前（SeriesNavの後）に、モバイル用のインラインTOCブロックを追加する。HTML標準の `<details>/<summary>` 要素を使い、折りたたみ可能な目次にする。既存の `TableOfContents` コンポーネントを `<details>` の中に配置する形で再利用する。

2. **レスポンシブ切り替え**: CSSメディアクエリで表示を切り替える。
   - 900px未満: インラインTOCを表示、サイドバーTOCを非表示（現状通り）
   - 900px以上: サイドバーTOCを表示（現状通り）、インラインTOCを非表示

3. **UIデザイン方針**:
   - `<details>` はデフォルトで閉じた状態にする（記事本文の邪魔にならない）
   - `<summary>` には「目次」というラベルを表示する
   - 既存のTOCコンポーネントのスタイル（背景色、ボーダー等）を活用し、統一感を保つ
   - headingsが空の場合はインラインTOCも非表示にする

**注意点**:

- `TableOfContents` コンポーネント自体は変更せずに再利用するのが望ましいが、インラインTOC用に `<nav>` の `<h2>` タイトルが重複する場合はpropsで制御するか、別途ラッパーで対応する
- `<details>/<summary>` はすべてのモダンブラウザでサポートされており、JavaScriptなしで動作するため、サーバーコンポーネントのまま実装可能
- アクセシビリティ: `<details>` にはネイティブのキーボード操作とARIAセマンティクスが組み込まれているため追加対応は不要

#### 作業3: 動作確認

**内容**:

- `npm run build` が成功することを確認
- `npm run lint && npm run format:check && npm run test` がすべてパスすることを確認
- デスクトップ幅（900px以上）でサイドバーTOCがsticky動作することを確認
- モバイル幅（900px未満）でインラインTOCが表示され、サイドバーTOCが非表示であることを確認
- デスクトップ幅でインラインTOCが非表示であることを確認
- コードブロックやテーブルの横スクロールが `overflow-x: clip` で壊れていないことを確認

### 検討した他の選択肢と判断理由

1. **モバイルTOCをフローティングボタン + ドロワーで実装する案**: JavaScriptが必須となり、クライアントコンポーネント化が必要。コーディング原則の「静的最優先」に反し、実装の複雑さに対して得られるUXの向上が限定的。`<details>/<summary>` でシンプルに実現できるため不採用。

2. **モバイルTOCをデフォルトで開いた状態にする案**: 長い目次の場合、記事本文が画面外に押し出されてしまい、最初のスクロールで離脱するリスクがある。目次を確認したいユーザーだけがタップで開ける方が、記事を読みに来たユーザー全体にとって邪魔にならない。

3. **`overflow-x: hidden` を単純に削除する案**: 横スクロールバーが表示される可能性があり、レイアウト崩れのリスクがある。`clip` に置き換えることで、オーバーフロー制御とsticky互換性の両方を確保できる。

### 計画にあたって参考にした情報

- CSS仕様: `overflow: hidden` が暗黙的にスクロールコンテキストを作成し `position: sticky` を妨げる仕様上の挙動
- `overflow: clip` の仕様: スクロールコンテキストを作成しないオーバーフロー制御プロパティ
- ブラウザサポート: `overflow: clip` は Chrome 90+, Firefox 81+, Safari 16+ でサポート
- Playwrightでの検証結果: スクロール後にサイドバーが画面外に出てしまうことを確認済み
- 現状のコード: `globals.css` 54-58行目の `html, body { overflow-x: hidden }` がsticky阻害の原因
