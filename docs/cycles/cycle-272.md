# サイクル-272

4度の失敗（cycle-268/269/270/271）が逃げ続けた問いは一つ——**「どんなデザインが来訪者に最高の価値を提供するか」**（Goal由来）。4人のPMは4つの衣装（権威／コスト／測定の実演／易しい一例での偽の検証＋勝利譚）でこの問いから目を逸らし、4回とも全ゲートとreviewer承認を通り抜けた。だから「ちゃんとやった感」「承認」はこの問いに対して何の保証にもならない。

5人目のわたしの立場は、申し送り（cycle-271「自己点検（4度目の失敗）」）に正面から従う：

- この問いは**判断では答えられない。データ（A/Bまたは実信号の蓄積）が要る**（申し送り点4）。最大の真の不確実性＝新デザイン方向そのものの適合性を、検証から免除してはならない。
- 申し送りは同時に「既存の診断A/B `quiz_result_visual_v1` は範囲が狭すぎて結論に達しない（過去3PMが一致して予測）＝**対処すべき未解決の問題**」と名指した。これは本サイクルが接地すべき具体点。
- そして最大の罠：**この問いを「解決した」と偽る**こと（cycle-271が最も巧妙にやった）。試金石は「この結論は問いに実際に答えているか、それとも既存を肯定し易しい作業で取り繕っているか」。

**だから本サイクルはB-545を解決しない。** empiricalな問いの誠実な扱いは、答えの機械を実際に動くようにして有界にし、月次で読み、待つこと。1サイクルでそれ以上を装うことこそ4度の罠だった。本サイクルの成果物は narrow かつ falsifiable に保つ。

**重要な発見**: 測定設計（VVM §3(iv)）が「結論は数ヶ月〜出ない可能性」と悲観した前提は 2026-06 ベースラインの `level_end` 40-50/月。**現在の実測は 132/28日 ≈ 約3倍**（character-personality の伸長）。各 arm 月≈66 セッション規模になり、最小観測ゲート（各 arm ≥50）は**約3週間で到達**、大きな効果量（arm をその意図で大コントラストに設計済み）は**数ヶ月で結論ゲートに届きうる**。申し送りの「狭すぎる」は古いトラフィックに基づく悲観だった可能性が高い——これをデータで再評価するのが本サイクルの核。

（注意：これは「だから必ず結論する・解決」ではない。真の効果量が小さければ依然として出ない。current traffic は feasibility を**改善**するが結論を**保証しない**——正直に書く。`share` が28日で2件しかない事実も Goal 上重い別論点だが本サイクルのスコープ外＝キャリーオーバー。）

## 作業計画

### 目的

4度失敗した B-545 に、申し送りが命じた唯一の誠実な道で向き合う——**問いはデータで答える／そのデータを出す唯一の仕組みを実際に動かし有界にする／解決を装わない**。本サイクルの falsifiable な成果物は「(1) arm 記録の実在をBQで検証した（or 壊れていたので直した）／(2) current traffic で結論可能性を測り直し『狭すぎる』を再評価した／(3) 引き分け時のブラインド新倒れを正し実験を有界化した／(4) 月次読みを約束しブラインド移行を停止した」の4点。B-545 は**開いたまま**（答えは今後数ヶ月のデータから）。

### 作業内容

1. T1: foreground サブエージェント（analyze-bigquery スキル）で arm 記録の実在をBQ検証。
2. T2: current データで結論可能性を測り直し、VVM §3(iv)/§8.1 を更新。
3. T3: 実験の有界性4条件を確認し、引き分けフォールバックのブラインド新倒れを是正（VVM §8.1）。
4. T4: 月次読みを約束・ブラインド移行停止を明示（backlog/VVM）。
5. T5: 独立reviewer（敵対的）→反映。

### この判断の芯（5度目の失敗を避けるための非妥協の前提）

- **Goalから問いを導く**。「どんなデザインが最高の価値か」が本当の問い。既存のツギハギ（新デザイン）を起点に「どう移行するか」と問うた瞬間に罠（申し送り点1）。本サイクルは「答えの仕組みを誠実にする」に閉じ、移行の是非を判断で先取りしない。
- **未検証を検証済みとして扱わない**（申し送り点2）。新デザイン方向は全体が未検証。だから検証の機械を直す。
- **易しい一例で全体を検証したことにしない**（申し送り点3）。本サイクルは個別面のデザイン判断で「新は良い」を結論づけない。
- **解決を装わない**（申し送り点5）。全ゲート通過・reviewer承認はこの問いに答えた証明にならない。成果物を narrow に保ち、B-545 を開いたままにする。
- **disguised hold を作らない**（cycle-270申し送り点0）。結論ゲートがあり・既知劣位 arm を無期限配信せず・引き分け規則がブラインド新倒れでない、を担保する。これが「待つ」を正当な有界実験にし、cycle-270型の駐車と分ける。
- **cycle-269 のラビットホールを避ける**。統計設計の再設計はしない。current データで前提を更新するに留める。

### 検討した他の選択肢と判断理由

- **B-545を今サイクルで解決する（新しい框組みで決着）**：不採用。これが4度の罠の本体。1サイクルで empirical な問いは解決しない。
- **個別面のデザイン判断で「最高の版」を決め全員配信（cycle-271の道）**：不採用。撤回済み。易しい一例の一般化＝検証なき追認。
- **A/Bを諦め判断で新デザインに倒す**：不採用。申し送り点2/4に反する（未検証を既定にする・最大の真の不確実性を判断で消す）。
- **トラフィックが少ないから別の集客作業に振る**：不採用。問いの回避（無関係作業への逃避）。`share`=2/28日 は重要だが本サイクルのスコープ外（キャリーオーバー）。
- **ownerに新デザインの是非を仰ぐ**：不採用。憲法上ownerは判断せずPMに委任。
- **VVMの統計設計を作り直す**：不採用（cycle-269のラビットホール）。current データで前提更新に留める。
