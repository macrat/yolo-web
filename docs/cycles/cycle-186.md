#### 横断検索撤去で失われる便益が来訪者最高基準で薄い理由

cycle-174 ① 採用根拠 6 項目（cycle-174.md L262-275）を A-13 廃止後に再評価する:

- **A-13**: 2024-11-21 廃止により本サイクル時点で実在しない便益（PM が WebFetch で一次確認、`https://developers.google.com/search/blog/2024/10/sitelinks-search-box`）。除外。
- **A-7「M1b 名前忘れ救済」**: cycle-174.md L109 が自ら認める弱点引用「M1b の主再到達経路はブックマーク・履歴・OS ランチャーで `search_intents = []`（targets/README.md / `気に入った道具を繰り返し使っている人.yaml` 明記）。サイト内検索なしでも経路成立、保険として ① 固有便益が薄く残る」。M1b yaml が `search_intents = []` を明示する以上、検索を能動的に使う場面そのものが M1b の動線想定の外側にある。
- **A-4「M1a 未分化キーワード探索」**: cycle-174.md L106 が自ら認める弱点引用「M1a の `search_intents` は『文字数カウント』『日付計算』『カラーコード 変換』など全て修飾付きの具体語で、未分化抽象語を打つ場面そのものが targets 定義から想像しにくい。あれば便利レベル。ツール一覧内絞り込み + 辞典 B-341 ツール化で代替可能」。M1a の yaml 定義が示す `search_intents` の全てが「修飾付きの具体語」である以上、A-4 が想定する「未分化抽象語を打つ場面」自体が yaml 定義から外れる。

#### 影響範囲の比較（CLAUDE.md L9 Decision Making Principle 接続）

- **横断検索を新デザインに移行する場合の効力範囲**: サイト内検索を能動的に使う来訪者（cycle-174.md ターゲット別動線想像から推測される範囲では、5 ターゲット中で能動的検索利用が yaml に明示されているターゲットは 0、補助動線として残差価値があるのは S1 / S3 の読後関連記事探索場面のみ）。

### 検討した他の選択肢と判断理由

#### 検討対象として浮上したが排除した中間案

AP-P17 列挙網羅性のため、本サイクル検討中に浮上したが Owner 厳命との不整合により排除した中間案を記録する。

- **中間案 C: 横断検索を「やや軽く」実装する（モーダル維持で最小コストで結線、独立ページ・ファーストビュー入力欄・URL シェア性は実装しない）**:
  - **排除理由**: Owner 厳命「横断検索の新デザイン移行を『やや軽く』実装するなどの中間案を提示しない」に明示的に違反。また「実装容易性」「現状維持しやすさ」を判断材料に含むことで CLAUDE.md L9（来訪者価値最大化）を読み替えており、Owner 厳命 3（工数で判断しない）にも違反。
- **中間案 D: 横断検索を「将来再開する可能性を残して保留」する**:
  - **排除理由**: Owner 厳命「『将来再開する可能性』『保留』のような曖昧な表現で残さない（明確に ② 採用、Phase 10.2 で自然消滅）」に明示的に違反。② 採用は「やらない」の明確な確定判断であり、保留状態とは構造的に異なる。

#### CLAUDE.md L9 との整合確認

CLAUDE.md L9 Decision Making Principle:

> When multiple approaches exist, always choose the one that maximizes value for the user (visitor), even if it requires significantly more implementation effort. The ultimate goal of this project is to maximize page views by providing the best possible value to visitors, as defined in `docs/constitution.md`. Implementation cost (time, number of files, complexity of changes) must never be a reason to choose an approach that delivers inferior UX.

本判断は **実装コストを判断材料に含めていない**。仮に選択肢 A（横断検索新デザイン移行）の方が実装サイクル数が少ないとしても、来訪者価値で選択肢 B が優位であれば B を採用する構造で論証している。実体として、本サイクル枠を ② 採用でドキュメント整合のみに使うことで Phase 6 / 7 / 8 / 9 着手を **早める** 効果は実装容易性ではなく **来訪者が新デザイン体験を得るまでの時間短縮** という来訪者価値として位置づけられる（サイトデザイン切替こそが全来訪者・全ページに効く来訪者価値の主経路だから）。CLAUDE.md L9 と整合する。
