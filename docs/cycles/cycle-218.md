### 目的

#### 誰のために

- **主: 特定の作業に使えるツールをさっと探している人**（`docs/targets/特定の作業に使えるツールをさっと探している人.yaml` / M1a）: 検索からの初回来訪者で、cron 式を「今書いた / これから書く」が、それが自分の意図どおりのスケジュールか・次にいつ動くかを今すぐ確かめたい人。likes「開いた瞬間に入力欄が見えてすぐ使える」「コピペで結果を受け取りすぐ元の作業へ戻れる」「余計な説明・装飾がない」「結果の根拠や前提が必要最小限だけ添えられている」に直撃する。提供価値の核心は「**タイルを開く → 式を入力（または貼り付け）→ 人間可読の説明と次回実行時刻が即座に表示される**」が、スクロールも解析ボタンの明示クリックもなしに完結すること。とくに cron 特有の「分・時の順序が直感に反する」落とし穴（research §4）を、入力欄直下のフィールドラベルと人間可読説明で即座に気付かせる点が M1a の「ミスが起きやすい部分を素早く確認する」needs に応える。

| --------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- | ---------- |

#### 論点 G: 次回実行のタイムゾーンの扱い（MAJOR 対応 / 虚偽表示リスク）

`getNextExecutions` は実行ブラウザのローカル TZ で計算し（実装値 = `logic.ts:421-425` の `getHours()` 等）、既存 `formatDate` も `toLocaleString("ja-JP")` で timeZone 未指定 = ローカル TZ 表示（実装値 = `Component.tsx:85`）。これをそのまま再利用して「（JST）」と付けると、JST 以外の TZ の来訪者には実際はローカル時刻なのに「JST」と書かれた虚偽表示になる（憲法 rule 2,3）。

| ------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------- |
