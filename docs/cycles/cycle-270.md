# サイクル-270

cycle-268（信仰でブラインド出荷）と cycle-269（A/Bを「実現不可能/測定不能」と偽装して保留）は、**唯一の基準=来訪者価値**の代わりに自分に都合のよい基準（確立方向・楽さ・コスト・慎重さ）で判断し、二度とも失敗した。cycle-269 は最後に判断を未了として次PMへ託した。本サイクルでその判断を**逃げずに下す**。

託された判断は明確だ——**「ページ／機能単位で (a) テストせず移行する か (b) A/B に混ぜる かを、面ごとに来訪者価値で判断する」**（`docs/visitor-value-measurement.md` §8.2 がその判断表の器）。確かなことは一点、**(c) 保留（割れた UX を据え置いて待つ）は constitution（ルール2・4・Goal）上の最悪手で採れない**（cycle-269 事故報告）。そして「全面一律 (a)」「全面一律 (b)」もまた個別判断の放棄（sweep）であり禁止（cycle-269 の二度目の失敗）。

実害ある「割れ」が残るのは **(legacy) の4ゲームだけ**（daily/irodori/nakamawake/yoji-kimeru）。残り76面は (new) 単独で一貫しており割れていない。よって本サイクルの判断対象は、この4ゲームを**1つずつ実際に見て** (a)/(b) に分類すること。前PMが三度逃げたのはまさにこの「面ごとに実際に見て判断する」工程である。

## 作業計画

### 目的

cycle-268/269 が二度失敗した B-545（(new)移行の検証方針）に、**面ごとの実判断**で決着をつける。具体的には、実害ある割れが残る (legacy) 4ゲームを1つずつ来訪者価値で (a)/(b) に分類し、(b) はシステム単位の単一 A/B にプールする方針を確定する。これにより、ブラインド移行の再発（cycle-268）も、判断を装う sweep／保留（cycle-269）も止め、B-493（残ゲーム移行）が来訪者価値に沿って即座に動ける状態を作る。

### 作業内容

方針決定なので成果物は「決定＋ドキュメント更新」であり、本番コードは変更しない（4ゲートは docs 変更のみで通過）。手順：

1. T1：ゲームごとに foreground サブエージェントで (legacy) と (new) 移行の差分を接地（コード＋スクリーンショット）、GA4 実測。`tmp/` に記録。
2. T2：PM が §8.2 基準で各ゲームを (a)/(b) に実分類し、(b) のプール形を決める。決定を planning.md 全 AP に照合。`tmp/cycle-270-decision.md` に草案。
3. T3：決定を §8.2／backlog／（必要なら §8.3）へ固定。
4. T4：独立 reviewer に敵対的レビューを依頼し、反映。

### この判断の芯（cycle-269 の失敗を踏まえた非妥協の前提）

- **唯一の基準は来訪者価値**。楽さ・コスト・整合・慎重さで代替しない。
- **(c) 保留は採れない**。4ゲームの割れ（一部 legacy・大半 new）は AP-P27 の実害。動かす（=(a) か (b)）以外にない。
- **全面一律 sweep も採れない**。面ごとに実分類する。判断の器は §8.2。
- **コストは却下理由にならない**（CLAUDE.md 意思決定原則）。「legacy 復活が大変」は (b) を退ける理由にしてはならない（そもそも4ゲームは legacy が手元にあり復活不要）。
- **観測量の小ささは (b) 却下の理由にならない**（AP-P31）。小規模だからこそ一人ひとりを丁寧に観る＝A/B が手段。範囲を狭めるほど信号は減る。
- (b) なら**システム単位の独立変数**で全トラフィックを1本に集約（per-game に割らない）。

### 検討した他の選択肢と判断理由

- **cycle-269 の「決定」（保留三段・全面A/B棄却・全面一律(b)・偽の per-item 表）を引き継ぐ**：不採用。事故報告で全て撤回済み・無効。判断は未了。
- **owner に (a)/(b) を仰ぐ**：不採用。憲法上 owner は判断せず PM に委任（事故報告でも明記）。本決定は PM 責務。高リスクゆえ接地と独立レビューで裏取りする。
- **B-545 と B-493（実装）を本サイクルで同時にやる**：不採用。決定→レビュー→実装の順序を守る（cycle-268 のブラインド出荷再発を防ぐ）。ただし B-493 は実験結論を待たず直近サイクルで着手（割れの早期解消＝AP-P27）。
- **§8.1 統計フレーミングの再設計まで踏み込む**：不採用（scope 外）。cycle-269 がこれに踏み込んで rabbit hole 化・失敗した。本サイクルは per-game (a)/(b) 判断に閉じる。§8.1 訂正ノートの宿題は別 backlog として残す。
