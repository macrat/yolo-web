# サイクル-256

このサイクルでは **B-532「診断の共有導線（バイラルの環）の改善」** を行う。サイトの来訪者の大半は /play の診断に集まり、そこで長く遊んでいる（character-personality 73PV/142秒/PV、word-sense-personality 32PV/123秒/PV）。クイズの完了率も健全（level_start 118 → level_end 94 ＝ 80%）。けれど、その楽しさを**新しい来訪者に変える環＝共有**が回っていない。憲法 Goal（来訪者価値による PV 向上）に最も効くのはここだ。

## なぜやるか（データに立脚した判断）

GA4（property 524708437）直近28日の実測：

- `level_start` 118 → `level_end` **94**（完了率 80%・体験自体は健全）
- `share` イベント = **2件**（完了94に対し 2.1%）
- 共有結果ページ（`/play/*/result/*`）の外部流入はほぼ0（上位でも fuji 3PV、多くは 2PV 未満）

**94人が結果に到達したのに、共有したのは2人だけ。** 診断は「友達と相性を調べよう」と共有を前提に設計されている（InviteFriendButton・相性機能）のに、そのバイラルの環が事実上死んでいる。エンゲージは取れているのだから、ここを直せば同じ来訪者数からより多くの新規来訪者＝PV が生まれる。これは移行バックログのどれよりレバレッジが高い。

### 確実な事実と、検証する仮説の区別

- **確実な事実**:
  1. 完了94に対し共有2件（share イベント実測）。
  2. 共有導線（`ShareButtons`）は `ResultCard` の**最下部**（title→description→detailedContent の**すべての後**、`ResultCard.tsx:628`）にある。character-personality の詳細本文は8タイプ一覧・キャラ解説・相性まで長い。
  3. 共有結果ページの外部流入はほぼ0＝環が新規来訪者を生んでいない。
- **検証する仮説（自信はあるが断定しない）**: 結果を見た瞬間の感情のピークは画面**上部**にあるのに、共有導線は遠い**下部**にある。多くの人は共有導線に辿り着く前に離れている。→ **感情のピーク（結果カード上部）に共有導線を出せば、共有率は上がる。**
  - 注: GA4 `scroll`（90%深度）は28日で33件だが、これはサイト全体集計で結果ページ限定でないため「○%が最下部を見ていない」とは断定できない。配置仮説は before/after の share 率で検証する。
- 二段の漏れがある可能性: 最下部に到達した人（少数）でも共有率は低い（動機づけの問題）。本サイクルは**最大多数に効く配置（上部露出）**に絞り、動機づけ（共有文言・OGP）は効果を見てから別サイクルで判断する。

### 走行中 A/B との関係

cycle-255 で仕込んだインライン結果 A/B（`quiz_result_visual_v1`、絵文字/カラフル vs ミニマル）は character-personality で走行中。その独立変数は `renderDetailedContent` 内の結果**本文の視覚**。共有導線を結果カード**上部の arm 中立な chrome**（`renderDetailedContent` の外、両 arm で同一の title/description 帯）に置けば、両 arm 等価で実験を汚染しない。`share` は A/B の主 KPI（エンゲージ時間）でもないため干渉しない。

### なぜ A/B でなく before/after コホートで検証するか

cycle-255 の原則は「来訪者に見える変更はブラインドでなく測って出す」。本変更も測る。ただし `share` は二値イベントで、完了94/月・現状 2.1% という低トラフィック・低基準率では A/B が原理的に検出力に達しない（cycle-255 設計 論点8.1 が二値イベントの検出力不足を理由に連続量 KPI を主に据えたのと同じ理由）。さらに本変更は「美しさの賭け」ではなく**計測で壊れていると分かった導線の欠陥修正**（共有導線が長い本文の後ろに隠れている）である。よって cycle-255 が A/B 不能な変更のために併設した補完経路＝**release 識別子による before/after コホート比較**（`docs/sql/ab-value-metrics.sql` SECTION 2）で share 率の変化を観測する。これは cycle-255 の計測規律と整合する。
