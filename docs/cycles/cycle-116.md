# サイクル-116

このサイクルでは、B-114（Google Analyticsデータ活用方法の調査・提案）に取り組みます。B-165（サイト全面価値向上プロジェクト）のPhase 4-1「低価値コンテンツの削除」はGA分析完了が前提条件であり、本タスクはその前提を満たすための重要な調査です。MCP/REST APIでのアクセスデータ取得方法、人気コンテンツランキング、アクセス数の活用方法等の実現可能性・手段・リスクを調査し、ownerへの提案書を作成します。

## 作業計画

### 目的

B-114は「調査・提案」タスクであり、成果物はownerへの提案書（`docs/research/` 配下に作成）である。実装は行わない。提案書をownerに提示し、承認を得てから実装に進む。

このタスクには2つの目的がある。

#### 目的1: 内部利用 -- Phase 4-1の削除判断に必要なGAデータ分析環境の確立

B-165 Phase 4-1「低価値コンテンツの削除」は、新コンテンツ（占い・診断パーク）がサイトのトラフィックを十分に支えているかをGAデータで検証してから実施する前提条件がある。このためには以下が必要である。

- **どのデータを見て、どう判断するかのフレームワーク**: PV数、セッション数、エンゲージメント率、検索流入数、コンバージョン（シェア、実績解除等）のうち、削除判断に必要な指標の選定と閾値の設計
- **MCP経由でのGA4データアクセス環境の構築提案**: AIエージェント（Claude Code）がGA4データに自然言語でアクセスし、分析を行える環境の構築方法

**誰のために**: サイト運営チーム（AIエージェント + owner）。データに基づいた削除判断を行うことで、価値のあるコンテンツを誤って削除するリスクを防ぎ、結果として来訪者にとって最適なサイト構成を実現する。

#### 目的2: 外部公開 -- GAデータをサイト上で来訪者向けに活用する方法の提案

GAデータを来訪者に見える形で活用することで、サイトの価値向上に貢献できるかを検討する。

- **人気コンテンツランキングの表示**: 「今週の人気診断TOP5」のような動的ランキングの表示
- **アクセス数公開ページ**: サイトの透明性を高めるための統計公開ページ（「AI運営サイト」の実験的性格を活かした公開）
- **来訪者にとっての価値を最重視**: 単なる数字の羅列ではなく、来訪者の行動を助ける情報提供として設計する

**誰のために**: サイトの来訪者。「みんなが遊んでいる人気の診断」という社会的証明が、初回訪問者のコンテンツ選択を助け、サイト内回遊を促進する。また「AIが運営しているサイトの裏側」としてアクセス数を公開することで、サブターゲット（エンジニア）にとっての知的好奇心も満たせる可能性がある。

### 作業内容

成果物は **提案書1本**（`docs/research/2026-03-20-ga4-data-utilization-proposal.md`）である。提案書は以下の構成で作成する。

#### 1. GA4 Data API技術調査のまとめ

背景情報として既に判明している事項（GA4 Data APIが完全無料であること、Node.js公式SDK `@google-analytics/data` が利用可能であること、サービスアカウント認証が推奨であること等）を確認した上で、以下を追加調査し提案書にまとめる。

- **認証方式の詳細**: サービスアカウントの作成手順、GA4プロパティへの閲覧者ロール付与、JSONキーファイルの管理方法
- **API クォータの正確な把握**: 公式ドキュメントの最新値を確認して記載する（2026年3月時点の参考値: 日次200,000コアトークン、時間40,000コアトークン、同時リクエスト10）。本サイトの想定利用量での十分性の評価も行う
- **利用可能なディメンション・メトリクス**: Phase 4-1の判断に必要なデータ（ページパス別PV、セッション数、エンゲージメント率、イベント数等）がAPIで取得可能かの確認
- **カスタムイベントデータの取得可否**: 現在のGA4実装で送信しているカスタムイベント（level_start, level_end, unlock_achievement, search, share, content_rating）がAPIから取得・分析可能かの確認
- **Search Consoleデータの取得方法の確認**: GA4 Data API経由でSearch Consoleデータ（検索クエリ、検索流入数等）を直接取得可能か、Search Console APIの別途利用が必要かを調査する

#### 2. MCP GA4サーバーの調査と推奨

AIエージェント（Claude Code）がGA4データにアクセスするためのMCPサーバーを調査し、推奨案を提示する。

- **既存OSSの評価**: google-analytics-mcp、mcp-server-ga4、Stape GA4 MCP Server等の主要実装を比較評価する。評価軸は、機能（利用可能なツール・クエリ能力）、認証方式、メンテナンス状況、セットアップの容易さ
- **自前実装の要否判断**: 既存OSSで要件を満たせるか、カスタマイズが必要かを判断する
- **セットアップ手順の提案**: ownerが実施する認証設定（サービスアカウント作成、GA4への権限付与、キーファイルの配置）と、Claude Codeの `.mcp.json` 設定内容を具体的に提示する
- **B-113提案書のMCPサーバー方式との整合性確認**: Imagen 4のMCPサーバーと同様のアーキテクチャパターン（サービスアカウントキーの隔離、環境変数管理）が適用可能かを確認する

#### 3. Phase 4-1削除判断フレームワークの提案

GAデータを用いて低価値コンテンツの削除判断を行うためのフレームワークを提案する。

- **評価対象コンテンツの明確化**: site-concept.mdで削除候補とされている辞典3種（colors, kanji, yoji）、チートシート7種、ツール33種が対象。B-111の検討結果（四字熟語辞典へのAI例文全件導入、漢字辞典の小学1-3年生への限定導入の推奨。色辞典は見送り）も考慮する
- **判断に使用する指標の選定**: PV数だけでなく、以下の多面的な指標を提案する
  - ページビュー数（絶対値とサイト全体に占める割合）
  - セッション数とページ/セッション
  - エンゲージメント率（GA4のengaged_sessions比率）
  - 検索流入数（Search Console連携データまたは既存のトラフィック分析データ）
  - カスタムイベント（share, content_rating等の来訪者アクション）
  - 直帰率/離脱率
- **判断基準（閾値）の提案**: 「削除すべき」「存続を検討」「存続」の3段階の判断基準を提案する。ただし閾値は現時点のサイト規模（月間PV約500）を前提とした相対的な基準とし、サイト成長に伴い見直すことを前提とする
- **新コンテンツとの比較評価**: 新コンテンツ（占い・診断パーク）のトラフィックデータと旧コンテンツを比較し、「新コンテンツがサイトの価値を十分に支えているか」を判断する方法
- **既存トラフィックデータとの整合**: `docs/research/2026-03-17-traffic-analysis-for-url-restructuring.md` に90日間のデータが既にある。このデータとGA4 APIから取得する最新データの比較方法、データの鮮度の扱いを提案する

#### 4. 外部公開機能の検討

GAデータをサイト上で来訪者向けに公開する方法を検討する。

- **人気コンテンツランキング**
  - 表示形式: 「今週の人気診断TOP5」「みんなが遊んでいるゲーム」等
  - 来訪者にとっての価値: 社会的証明による初回訪問者のコンテンツ選択支援、回遊促進
  - 技術的実現方法: ISR（revalidate指定）でGA4 APIを呼び出す方法と、GitHub Actions cronで日次取得してJSON化しビルドトリガーする方法の比較
  - **coding-rules.md ルール2との整合性**: ルール2は「外部APIの呼び出し、データベース、認証・ユーザー管理は実装しない」と包括的に定めている。GA4 Data APIの呼び出しはビルド時/ISR/GitHub Actionsでのサーバーサイド処理となるが、この区別がルール2の例外に該当するかはowner判断事項である。B-113提案書と同様にowner確認事項として提示する

- **アクセス統計公開ページ**
  - 表示内容: 月間PV推移、人気ページランキング、デバイス比率、カスタムイベント集計等
  - 来訪者にとっての価値: 「AIが運営するサイトの裏側を覗ける」という知的好奇心の充足（サブターゲット向け）。constitution.md Rule 3（AI運営の透明性）との高い整合性
  - プライバシー配慮: 個人を特定できるデータは一切公開しない。集計データのみの公開であることを明示

- **来訪者価値の総合評価**: 各機能が来訪者にとってどの程度の価値を持つか、実装コストに見合うかを評価する。「人気ランキング」は初回訪問者の回遊を助ける高い実用価値がある一方、「統計公開ページ」はサブターゲット向けの限定的な価値であり、優先度に差がある点を明確にする

#### 5. セキュリティ・プライバシーの整理

### 検討した他の選択肢と判断理由

- **調査を省略してPhase 4-1を直感で実施する案**: site-concept.mdの削除候補リストに従って機械的に削除する方法。データに基づかない判断は、value-improvement-plan.mdの心構え7「データに基づく判断」に反する。また、B-111でAI例文追加による辞典の独自性付与が推奨されており、削除判断はそれらの施策の効果も含めてデータで検証すべきである
- **GA4 Data APIのみ調査してMCPは対象外とする案**: MCP環境がなければ、GAデータの分析は手動でGA4管理画面を操作するか、APIを直接叩くスクリプトを書く必要がある。AIエージェントがGA4データに自然言語でアクセスできるMCP環境は、Phase 4-1以降も継続的なデータ分析に活用でき、長期的な価値が高い
- **外部公開機能を調査対象から除外する案**: backlog.mdにも「人気ツールランキング、アクセス数公開ページ等の実現可能性」が明記されている。内部利用のみでは来訪者に直接的な価値を提供できない。特に「人気ランキング」は社会的証明としてPV向上に直結する可能性があり、constitutionのGoal（PV最大化）の観点から検討すべきである
- **提案書ではなく直接実装する案**: B-114は「外部連携のためowner承認必須」と定義されている。サービスアカウントの作成やAPIキーの管理はownerの判断・実行が必要であり、提案書なしに実装を進めることはできない
