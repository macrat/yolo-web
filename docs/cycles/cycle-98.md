## 作業計画

### 目的

**誰のために**: 手軽で面白い占い・診断を楽しみたい人。友達との会話ネタになるコンテンツを求め、診断結果を比較して盛り上がりたい人。

**何の価値を提供するか**: Q25の6アーキタイプを2軸組み合わせで展開した24体のオリジナルキャラクターの中から「あなたに似たキャラクター」を発見する体験を提供する。自分の性格の複数面が1つのキャラクターに凝縮されている面白さと、キャラクターごとのユーモラスな性格描写による「くすっと笑える自己発見」が核となる価値。

**Q25との差別化ポイント**:

- **Q25「守護キャラ診断」**: 自分に足りないものを補う存在を占う。6アーキタイプから1つ選ばれる。「あなたの守護キャラはこの人」
- **Q10「キャラクター性格診断」**: 自分の鏡となるキャラクターを診断する。24体の中から最も似たキャラが選ばれる。「あなたに似てるのはこのキャラ」
- Q25は8問・6結果でシンプルな体験、Q10は12問・24結果でより細かい性格分解と多様な結果による「何が出るかわからない」楽しさを提供する

### 作業内容

#### タスク1: データ設計・キャラクター体系設計

**特性軸の定義（内部設計ツール）**

Q25の6アーキタイプを性格の2面として組み合わせるため、6アーキタイプを4つの特性軸上に位置づける。この4軸はキャラクターの性格付けガイドおよびタスク2でのprimary/secondary配分を決めるための内部設計ツールであり、スコアリングエンジンには組み込まない（スコアリングは方式Bの直接ポイント方式で行う）。

- **軸1: 行動 (Action)**: 即行動 (active) vs 熟考 (reflective) — commander/tricksterが即行動寄り、guardian/professorが熟考寄り
- **軸2: 社交 (Social)**: 外向 (outward) vs 内向 (inward) — commander/guardianが外向寄り、dreamer/artistが内向寄り
- **軸3: 知覚 (Perception)**: 論理 (logical) vs 感性 (sensory) — professor/tricksterが論理寄り、artist/dreamerが感性寄り
- **軸4: エネルギー (Energy)**: 爆発 (burst) vs 持続 (steady) — commander/artistが爆発寄り、guardian/professorが持続寄り

4軸の活用方法:

1. **キャラクター性格付け**: 各キャラの主アーキタイプ・副アーキタイプから4軸上の位置を導出し、性格描写テキストの方向性を決定する
2. **primary/secondary配分の設計**: 各質問が測定する軸を明確にし、その軸上で近い位置にあるキャラにprimaryポイント、やや近いキャラにsecondaryポイントを配分する。これにより「質問の意図 → 測定軸 → 該当キャラ」の論理的な対応関係が保証される
3. **均等性の検証**: 4軸のカバレッジが均等かどうかを確認するためのフレームワークとして機能する

**24体のキャラクター体系**

6アーキタイプの2軸組み合わせ（6C2 = 15通りの異型組み合わせ + 6通りの同型強化 = 21通り）から、品質を担保できる24体を選定する。

- **異型組み合わせ15通り**: commander-professor, commander-dreamer, commander-trickster, commander-guardian, commander-artist, professor-dreamer, professor-trickster, professor-guardian, professor-artist, dreamer-trickster, dreamer-guardian, dreamer-artist, trickster-guardian, trickster-artist, guardian-artist — 各1体 = 15体
- **同型強化6通り**: commander-commander, professor-professor, dreamer-dreamer, trickster-trickster, guardian-guardian, artist-artist — 各1体 = 6体
- **追加バリエーション3体**: 15+6=21体に対し、以下の選定基準に基づいて3体を追加し、合計24体とする
  - 選定基準1: **4軸空間上の疎密バランス** — 21体を4軸上にプロットした際に、カバーが薄い領域を補う組み合わせを優先する
  - 選定基準2: **結果文の差別化余地** — 同じ組み合わせの別解釈（例: commander-tricksterの「攻めの策士」vs「守りの策士」）として独自の性格描写が成立するかどうか
  - 選定基準3: **アーキタイプ出現頻度の均等化** — 21体時点で各アーキタイプが主/副として出現する回数を集計し、出現回数が少ないアーキタイプを含む組み合わせを優先する

各キャラには以下を設計する:

- ID（slug形式、例: `blazing-strategist`）
- 名前（日本語、Q25と同様のユーモラスなキャッチフレーズ付き）
- 主アーキタイプと副アーキタイプ（例: commander + professor）
- 4軸スコアベクトル（性格付けと配分設計のためのメタデータ）
- 性格テンプレート（タスク3で詳細テキストを制作する前の骨格）

#### タスク2: 質問・スコアリング設計

**質問設計**

- 12問、各4択
- Q25の質問と重複しないシチュエーションを採用する
- 各質問は4軸のうち主に1-2軸を測定する設計とし、12問で4軸すべてが十分にカバーされるようにする（各軸を主軸として測定する質問が最低3問）
- 質問のトーンはQ25と同様にユーモラスで意外性のあるシチュエーションとする

**スコアリング設計（方式B: 直接ポイント方式）**

既存のQuizDefinitionの型（`points: Record<string, number>`）をそのまま使い、24体のIDに直接ポイントを付与する。animal-personality（12体）、music-personality（8体）と同じ方式。

配分ルール:

- 各選択肢にprimary(2pt)を2体、secondary(1pt)を2体に付与する
- 12問x4択=48選択肢で合計96のprimary配分と96のsecondary配分を24体に均等配分する（目標: 各体primary 4スロット、secondary 4スロット）

4軸設計からの配分導出プロセス:

1. 各質問が主に測定する軸を決定する（タスク1の4軸定義に基づく）
2. 各選択肢が測定軸上でどの方向を示すかを定義する
3. その方向に近い4軸ベクトルを持つキャラにprimary(2pt)を、やや近いキャラにsecondary(1pt)を配分する
4. 全48選択肢の配分を集計し、均等性を検証する

**均等性の許容範囲と代替策**:

- **目標**: 各キャラのprimaryスロット数が4（= 96配分 / 24体）
- **許容範囲**: primary 3-5スロット、secondary 3-5スロット。合計ポイント（primary x 2 + secondary x 1）が全キャラで12 +/- 2の範囲（10-14）に収まること
- **均等配分が達成できなかった場合の代替策**:
  1. まず質問・選択肢の文言を微調整して配分を修正する
  2. それでも収まらない場合、一部の選択肢でprimary 1体 + secondary 3体、またはprimary 3体 + secondary 1体のように配分比率を変えて全体のバランスを取る
  3. 最終手段として、24体のうち到達困難なキャラを統合して22-23体に削減する（品質優先の原則に基づき、無理に24体を維持しない）

**マッチングの均等性検証**

- 全24体への到達可能性を確認する（特定のキャラが出にくい/出やすい偏りがないこと）
- 各キャラへの合計ポイント配分が許容範囲内であることを設計段階で数値的に検証する
- 配分表（キャラID x 質問ID のマトリクス）をタスク2の成果物として作成し、レビューに供する

#### タスク3: キャラクターコンテンツ制作（バッチ1: 10体）

最初の10体のキャラクターについて、以下のコンテンツを制作する:

- **タイトル**: Q25と同等のキャッチフレーズ（例: 「締切3分前に本気出す炎の司令塔」のような独自性のある一文）
- **性格描写テキスト**: 200-300文字。Q25のresultsと同等の品質。一人称の語りかけ調で、ユーモアと共感を兼ね備えた文章
- **色（color）**: キャラクターのイメージカラー（hex値）
- **アイコン（icon）**: 絵文字1つ

**品質チェック基準**:

1. 読んで「くすっと笑える」かどうか
2. 主アーキタイプと副アーキタイプの特徴が自然に融合しているか（無理なつぎはぎ感がないか）
3. Q25の6アーキタイプの語り口・世界観との一貫性があるか
4. 24体の間で個性が重複していないか
5. SNSでシェアしたくなる結果文かどうか

制作後、レビュー・修正を行う。

#### タスク6: 相性テキスト制作（300エントリ）

24体の全組み合わせ（24x25/2 = 300エントリ）の相性テキストを制作する。各エントリは短いラベル（label）と説明文（description）で構成される。

**パターン化方針**:

300エントリを以下の3カテゴリに分け、カテゴリごとに制作方針を定める:

1. **自己ペア（24エントリ）**: 同一キャラ同士の組み合わせ。「似た者同士あるある」をテーマに、長所の増幅と短所の共鳴をユーモラスに描く
2. **共通アーキタイプペア（同じ主or副アーキタイプを共有するペア）**: 共通のアーキタイプが生む「わかる!」感と、異なるアーキタイプが生むズレを対比させる
3. **完全異型ペア（共通アーキタイプなし）**: 2つのキャラの最大の違いを焦点に、正反対だからこそ生まれる化学反応を描く

**品質チェック基準**:

1. ラベルが具体的で面白いか（「相性抜群」のような汎用表現を避ける）
2. 2つのキャラの個性が両方とも活きているか（一方だけが目立つ記述になっていないか）
3. 同カテゴリ内で表現が単調になっていないか（似た構文の繰り返しを避ける）
4. 読んで「この二人が会ったらどうなるか」が想像できるか

**制作・レビューの工程**:

- バッチ分け: 自己ペア24件 → 共通アーキタイプペア → 完全異型ペアの順に制作
- 各バッチ制作後にレビュー、フィードバック反映

### 検討した他の選択肢と判断理由

**キャラ数の検討**:

- **50-100体（site-concept.mdの当初案）**: 品質優先の原則（constitution.md Rule 4）に基づき下方修正。各キャラにQ25と同等の品質で性格描写・ユーモアテキストを付与するには、50体以上では品質管理が困難。また相性マトリクスのエントリ数が爆発する（50体なら1,275エントリ）
- **24体（採用）**: 6アーキタイプの全組み合わせ（15異型+6同型=21）+ バリエーション3体で、体系的かつ管理可能な数。animal-personality（12体）の2倍であり、十分な多様性を提供しつつ品質を担保できる
- **12-16体**: 体系性が弱くなり、「24体もある中から自分だけのキャラが見つかる」というワクワク感が薄れる

**スコアリング方式の検討**:

- **方式A（軸スコア + ベクトル類似度マッチング）**: 理論的にはより精密なマッチングが可能だが、QuizDefinitionの型を拡張するかカスタムスコアリング関数の実装が必要。既存のpersonality型クイズとの一貫性が崩れる
- **方式B（直接ポイント方式、採用）**: 既存のanimal-personality（12体）、music-personality（8体）と完全に同じ方式。型の拡張もカスタムロジックも不要。24体でも各選択肢にprimary 2体 + secondary 2体を配分すれば均等なスコアリングが設計可能。4軸設計はキャラクターの性格付けとprimary/secondary配分の論理的根拠として活用する（内部設計ツール）

**Q25との差別化方法の検討**:

- **「守護キャラ」のバリエーション追加**: Q25と同じコンセプトの拡張は差別化にならない
- **「似たキャラ」診断（採用）**: Q25の「自分に足りないものを補う守護キャラ」と対をなす「自分の鏡としてのキャラ」というコンセプト。2つの診断を両方受けることで「守護キャラ（足りないもの）」と「似たキャラ（自分の鏡）」の両面から自己発見できる

## レビュー結果

### 計画レビュー（第1回）

レビューで6件の指摘があり、すべて対応済み:

1. **ファイルサイズの見積もりとファイル分割方針が欠落（軽微）** → タスク7にファイル分割方針を追加。character-personality.ts（約800-1,000行）と character-personality-compatibility.ts（約1,500-2,000行）に分割する方針を明記
2. **300エントリの相性テキストの品質担保方針が不十分（重要）** → タスク6を独立タスクとして新設。3カテゴリへのパターン化方針（同型ペア・共通アーキタイプペア・完全異型ペア）と品質チェック基準4項目を追加
3. **スコアリング均等配分が達成できなかった場合の代替案が欠落（重要）** → タスク2に許容範囲（primary 3-5、合計ポイント10 +/- 2）と3段階の代替策（文言微調整 → 配分比率変更 → キャラ数削減）を追加
4. **相性テキスト制作がタスク6（実装）に混在（重要）** → 相性テキスト制作をタスク6として独立化し、元のタスク6（実装）をタスク7、元のタスク7（レビュー・テスト）をタスク8に繰り下げ
5. **4軸特性設計と方式Bの関係性が不明瞭（中程度）** → タスク1に4軸の活用方法3点を明記。タスク2に「4軸設計からの配分導出プロセス」4ステップを追加。4軸は内部設計ツールであり、スコアリングエンジンには組み込まないことを冒頭で明示
6. **追加バリエーション3体の選定基準の具体化（軽微）** → 3つの具体的な選定基準（4軸空間上の疎密バランス・結果文の差別化余地・アーキタイプ出現頻度の均等化）を追加

### 計画レビュー（第2回）

2件の指摘があり、すべて対応済み:

1. **合計ポイント許容範囲の数値誤り（重要）** → 「10 +/- 2」を「12 +/- 2（範囲: 10-14）」に修正。中央値は primary 4スロット x 2pt + secondary 4スロット x 1pt = 12pt
2. **「同型ペア」の呼称が不明確（軽微）** → 「同型ペア」を「自己ペア（同一キャラ同士の組み合わせ）」に修正

### 計画レビュー（第3回）

1件の軽微な指摘があり、対応済み:

1. **「同型ペア」の修正漏れ（軽微）** → バッチ分けの記述で「同型ペア24件」が残っていたのを「自己ペア24件」に修正
