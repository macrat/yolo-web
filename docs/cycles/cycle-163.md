## 作業計画

### 目的

constitutionのゴール「higher page views by providing the best value for visitors」に基づき、月間601PV（2026年4月時点）から成長するための実行計画を策定する。cycle-155以降7サイクル連続で失敗した成長戦略を、10件の調査レポート（8件の新規調査 + 2件の既存調査）の検証済み事実のみに基づいて再構築する。

**到達可能な目標**: 月間700〜1,000PV（12ヶ月後）

根拠:

- 現状601PVの内訳: OS 294PV、Direct 236PV、Referral 67PV、Organic Social 4PV
- 施策別期待効果の積み上げ: P1 OS +50〜200PV + P2 OS +30〜100PV + P3 OS +20〜80PV = OS +100〜380PV。現状OS 294PVに加算すると394〜674PV
- Direct/Referral 303PVは現状維持〜微増と仮定し、合計で約700〜1,000PV/月が12ヶ月後の到達可能な範囲
- CTRの仮定: ドメイン年齢1年未満の新規サイトでは一般的にCTR 1〜3%程度。aramakijake実測で狙えるKWの合計月間検索量が数千件規模であり、CTR 1〜3%で数十〜百数十PV/月が妥当（推測。なお現時点でGoogle Search Consoleが未導入のため、実CTRの検証は不可能）
- 6万PV/月は非現実的。根拠: yolos.netが狙える「需要あり x 競合弱い」KWの合計月間検索量が数千件規模であり、全KWで1位を獲得しても6万PVには到達しない。バイラルによる急増は計画的に達成する手段がない（非SEOチャネル調査レポートで確認済み）

### 作業内容

以下はcycle-163の直接の作業ではなく、次サイクル以降の優先順位付きアクションリストである。cycle-163の作業は本計画の策定とレビューである。

#### 優先度1: ツール系ページのSEO強化（テキスト整形・改行削除系）

**何をするか**: 既存ツールページ（fullwidth-converter、text-replaceなど）のSEOコンテンツを強化し、「需要あり x 競合弱い」KWでの検索流入を獲得する。

**なぜこれが最優先か**:

- 全カテゴリKW・SERP調査（レポート6）で、ツール系の「テキスト整形 オンライン」「改行削除 オンライン」「全角半角変換 ツール」「PDF 改行 削除」が唯一の「需要あり x 競合弱い」黄金ゾーンと特定された
- SERP上位がGitHub個人ページ、netlifyページ、占いサイトの副業ツールなど権威性の低いサイトであり、参入余地がある（レポート6実測）
- 「正規表現 テスト」（月間Google 384。なお「正規表現」単体: 月間Google 27,100）も競合が個人サイトレベル（レポート6実測）
- yolos.netには既にfullwidth-converter、text-replace、regex-testerが実装済みで、新規開発なしにSEOコンテンツ追加が可能

**来訪者にとっての価値**: ツールの使い方が分からない来訪者の疑問を解消し、具体的なユースケース（例: メール文面の全角半角統一、PDFコピペ時の改行除去）を示すことで、ツールの活用度を高める。検索で辿り着いた来訪者が「探していたものが見つかった」と感じられるページにする。

**具体的アクション**:

1. 既存ツールページに検索意図に合致した解説テキスト（使い方、ユースケース、FAQ）を追加
2. tool-guides記事との内部リンク強化（記事→ツール、ツール→記事の双方向導線）
3. metadescription・titleタグのKW最適化
4. 新規ツール「改行削除ツール」「テキスト整形ツール」の追加検討（SERP調査で競合が極めて弱いため）

**期待効果**: 月間OS +50〜200PV（6ヶ月後。推測。根拠: 対象KWの合計月間検索量が数千件、新規サイトのCTR 1〜3%で数十〜百数十件が現実的だが、ロングテールの積み上げと順位向上の余地を含めて幅を持たせた）

**検証方法**: Google Search Consoleのインプレッション数・CTR・平均順位をKW別に追跡。3ヶ月後にインプレッションが増加していなければ、コンテンツの質・量の問題か、ドメイン権威の問題かを切り分ける。

#### 優先度2: ブログ記事の既存流入強化（tool-guides + ai-workflow）

**何をするか**: 既に検索流入実績のある4記事を中心に、SEOコンテンツの質を向上させる。

**なぜこれか**:

- ブログコンテンツ分析（レポート2）で、流入実績のある4記事に共通パターンが特定された:「具体的な技術名・数値・問題名をタイトルに含み、実装ノウハウ型の記事構成」
- ブログSEOポテンシャル調査（レポート5）で、tool-guidesカテゴリが「検索意図一致 + GA4流入実績あり」でSEOポテンシャル最高と評価
- 「cron 書き方」（月間Google 2,320）は個人ブログ・Qiitaが競合主体で参入余地あり（レポート5実測）

**来訪者にとっての価値**: 技術的な問題を抱えて検索してきた来訪者に対し、環境別の具体例や実践的なノウハウを提供することで、問題解決を直接支援する。「AI自律運営の実録」記事は、AI活用に関心のある読者に他では得られない一次体験情報を提供する。

**具体的アクション**:

1. character-counting-guide（月2セッション実績あり）: 「全角半角変換ツール」への導線追加、情報の充実
2. cron-parser-guide: 「cron 書き方」KWに対するコンテンツ最適化（環境別の違い・具体例の拡充）
3. ai-workflow系記事: Claude Code関連（月間Google 72,400）は競合が増加中だが、「AI自律運営の実録」という独自切り口は模倣不可能。新記事は「体験記録・比較記事・実践的疑問への回答」形式で継続

**期待効果**: 月間OS +30〜100PV（6ヶ月後。推測。根拠: 対象KWの月間検索量が数百〜数千件規模、新規サイトのCTR 1〜3%を前提とした控えめな見積もり）

#### 優先度3: 診断結果ページのコンテンツ拡充

**何をするか**: 89ページあるインデックス対象の診断結果ページの本文テキスト量を、現状の300〜700字から2,000〜3,000字に拡充する。

**なぜこれか**:

- 診断テキスト量計測（レポート1）で、結果ページの本文は300〜700字と判明
- 診断テキスト量とSEO調査（レポート3）で、競合が本文1,200〜5,500字を提供する中、yolos.netは300〜350字で流入ゼロ（GA4実測）と確認
- 結果ページへのOS流入はゼロ件（レポート3、GA4実測で確認済み）
- 競合のuranaino.netは3,000〜3,500字の記事で「INFJ 恋愛」「INFJ 適職」等のロングテールKWから流入を獲得している（レポート3のPlaywright実測）

**ただし優先度を3とする理由**:

- 診断系KWの検索ボリュームは、ブロードなKW（「心理テスト」「MBTI」等）では競合が超大手（16personalities.com等）で参入不可能（レポート8で確認）
- yolos.net固有の診断（「ニホンザル 性格 診断」等）のKWは検索需要が極めて小さい（aramakijake実測不可=月100未満）
- 89ページの拡充は工数が大きく、1ページあたりの期待流入が小さい
- 優先度1・2の施策が先に効果を出す可能性が高い

**来訪者にとっての価値**: 診断を楽しんだ来訪者が結果ページでより深い自己理解や新しい気づきを得られるようにする。性格タイプの詳細な解説、日常での行動パターン例を充実させることで、「診断して終わり」ではなく読み物としても楽しめるコンテンツにする。

**具体的アクション**:

1. まず流入実績のある診断（play/word-sense-personality 4セッション、play/character-personality 3セッション、play/music-personality 3セッション）の結果ページから着手
2. 追加セクション: タイプ性格の詳細説明（300〜500字）、強み・弱みの段落展開、行動パターンの充実
3. 恋愛・仕事傾向の追加は、yolos.netの「エンターテインメント目的」の位置づけ（EEAT調査レポート4で非YMYL確認済み）と整合するよう、あくまで娯楽的なトーンで実施

**期待効果**: 月間OS +20〜80PV（12ヶ月後。推測。根拠: 対象KWの検索需要が月100未満と推測されるものが大半。ただしロングテールの積み上げ効果がある。12ヶ月はテキスト追加→インデックス→順位安定のリードタイムを含む）

#### 優先度4: 英語版伝統色辞典の実験

**何をするか**: 日本の伝統色辞典の英語版を1ページ実験として公開する。

- 検索ボリュームの数値は推定値であり信頼度が低い（レポート8に明記）
- i18n対応の技術的工数が未知数
- 日本語サイトの成長が優先

**期待効果**: 不明。実験として検証する。

#### 優先度5: Trustworthiness基盤整備とDiscover準備

**何をするか**: AboutページのAI運営情報充実、プライバシーポリシー整備、エンタメ免責事項の明示、Google Discover技術要件（1200px画像、max-image-preview:large）の整備。

**なぜこれか**:

**来訪者にとっての価値**: AI運営であることの透明性を高め、コンテンツの正確性に関する免責事項を明示することで、来訪者が情報の信頼度を自分で判断できるようにする。プライバシーポリシーの整備は来訪者の安心感に直結する。

**期待効果**: 直接的なPV増加は見込めない。間接的にSEO評価の底上げと来訪者の信頼感向上に寄与する可能性がある（推測）。

### cycle-162の論理矛盾9項目の解消

1. **「cycle-161の350PV/月を否定しながら同じ方向性で700-1,200PV/月を目指す」**: 本計画では方向性自体を変更した。cycle-161の「診断SEO中心」から「ツールSEO中心」へ転換している（全カテゴリKW・SERP調査の結果に基づく）。月間700〜1,000PVを12ヶ月後の目標とし、各施策の期待効果の積み上げに基づく根拠を明示している。期待効果には「推測」と明記し、検証方法を併記した。

2. **「ブログを不採用にしながらはてブでブログ記事のシェアに期待」**: 本計画ではブログ記事のSEO強化を優先度2に位置づけている。はてブは計画的に達成する手段がない（レポート7: フォロワーゼロでの最初のブクマ獲得は「コンテンツの質のみ」が持続可能な方法）ため、独立した施策としては含めていない。

3. **「診断バイラルはフォロワーゼロで不採用、はてブ（同様にゼロからの拡散が必要）を採用」**: 本計画ではSNSバイラルもはてブも「計画的に達成できない施策」として独立した優先度は設けていない。流入の主軸はSEO（Organic Search）とする。

4. **「診断がSEOの最適カテゴリであることを確認せずに主エンジンとして選択」**: 全カテゴリのKW・SERP調査（レポート6）の結果、ツール系が唯一の「需要あり x 競合弱い」カテゴリと確認された。診断は優先度3に位置づけた。

5. **「英語圏展開の検索ボリュームを実測せずにPV推計を算出」**: 英語圏市場調査（レポート8）で検索ボリュームを推定した。ただしデータの信頼度は低い（レポート8に明記）。本計画では英語版実験の期待効果を「不明」とし、仮説検証として位置づけた。

6. **「候補Eを調査不足で棄却し、他の候補は同様の調査不足のまま採用」**: 本計画では10件の調査レポートに基づき、全候補に対して需要分析と競合分析の両面を揃えた。調査不足の候補はない。

7. **「Ownerの提案を候補Dの採用理由に含めた」**: 本計画の判断根拠にOwnerのアドバイスは含まれていない。全ての判断根拠は調査レポートの検証済み事実に基づいている。

8. **「スケール不足とした候補を主エンジンに採用」**: 本計画の優先度1（ツール系SEO）は、スケールは限定的だが「需要あり x 競合弱い」の唯一のカテゴリである。スケール不足を理由に不採用にすると、全カテゴリが不採用になる（全カテゴリでスケールが限定的であることがレポート5,6で確認済み）。限定的なスケールの中で最大効果を狙う。

9. **「ゼロベース検討を計画しながら一部の候補は過去の結論を引き継いだ」**: 本計画では10件の調査レポートの実測データに基づいてゼロベースで評価した。過去サイクルの結論を無検証で引き継いだ項目はない。

### 検討した他の選択肢と判断理由

**選択肢A: 診断コンテンツを最優先にする**

- 不採用理由: 全カテゴリKW・SERP調査（レポート6）で、診断系KWの「需要あり x 競合弱い」組み合わせが確認できなかった。「心理テスト」「MBTI」等のブロードKWは16personalities.com等が支配（レポート8）。yolos.net固有の診断名のKW需要は月100未満（aramakijake実測不可）。89ページの拡充工数に対してROIが低い。

**選択肢B: ブログ新記事の大量投稿**

- 不採用理由: ブログSEOポテンシャル調査（レポート5）で、現在月間10セッション以上の流入が見込めるKWはほぼ存在しないと結論。大量投稿はGoogleのScaled Content Abuse判定リスクがある（EEAT調査レポート4で確認）。既存53記事の質的強化が先。

**選択肢C: SNSバイラル・はてブを主軸にする**

- 不採用理由: 非SEOチャネル調査（レポート7）で、フォロワーゼロの状態ではSNS経由の安定流入は期待できないと結論。はてブのホットエントリーは最短3〜6ヶ月で到達可能性「低〜中」。バイラルは計画的に達成する手段がない。SEOで流入を積み上げ、派生するシェアを待つアプローチが最も現実的（レポート7の結論）。

**選択肢D: 辞典コンテンツのSEO強化**

- 不採用理由: 全カテゴリKW・SERP調査（レポート6）で、辞典カテゴリは「需要あり x 競合弱い」の組み合わせが存在しないと結論。四字熟語・漢字は専業辞典（6,000語超）と公益財団法人が参入しており勝算が低い。伝統色は検索ボリュームが月170と極低。

**選択肢E: 新規コンテンツジャンルの開拓（デイリーゲーム強化）**

- 部分的に考慮: 非SEOチャネル調査（レポート7）で、デイリーコンテンツのリテンション設計が中長期で有効と評価された（リピーター100人 × 毎日アクセス = 月3,000PV増の単純計算。レポート7 line 319に記載。外部出典のない独自試算）。ただしリピーター形成には6ヶ月〜1年かかり、最初のリピーター層はSEO流入に依存する。本計画ではSEO強化を先行させ、リピーター形成の基盤を作る戦略を取る。デイリーコンテンツの改善は優先度1〜3の施策と並行して実施可能。

#### 指摘一覧

**M-01: 期待効果の積み上げと上限目標1,500〜3,000PV/月の間に説明されていないギャップがある**

計画内で明示された各優先度の期待効果を合算すると以下のとおり。

- P1: OS +50〜200PV（6ヶ月後）
- P2: OS +30〜100PV（6ヶ月後）
- P3: OS +20〜80PV（12ヶ月後）
- P4: 不明
- P5: 直接的PV増加なし

合計: OS +100〜380PV。現状OS 294PVに加算すると394〜674PV。Direct/Referral 303PVを加えても697〜977PV/月。これは上限目標1,500〜3,000PV/月の下限にすら届かない。計画冒頭の「到達可能な上限目標」の根拠記述（line 29）ではCTR 5〜10%で「数百PV/月が妥当」と書いており、これも施策別積み上げと整合していない。目標値を施策別期待効果の積み上げに基づいて修正するか、ギャップを埋める根拠（例: リピーター効果、想定外のロングテール積み上げ等）を明示する必要がある。

**M-02: 優先度1の「黄金ゾーン」KWデータが8つの調査レポートに直接含まれていない**

計画は「レポート6で特定された」と記載しているが、レポート6（keyword-serp-research-all-categories.md）の該当箇所（line 253-254）は「既存2026-04-06調査」と明記しており、「テキスト整形 オンライン」「改行削除 オンライン」等の検索ボリュームとSERP分析の一次データは2026-04-06-serp-competition-and-search-demand-analysis.mdおよび2026-04-06-text-formatting-tool-keyword-search-volume.mdに存在する。計画の「8つの調査レポートの検証済み事実のみに基づいて再構築する」という宣言と、実際のデータの出典に齟齬がある。以下のいずれかで修正すべき。

- (a) 参考情報に2026-04-06の2レポートを追加する
- (b) レポート6が統合的に引用していることを明示し、一次データの出典を脚注で補足する

**S-01: AP-P09（ゴールの読み替え）への対応が表面的**

アンチパターンチェック（line 193）では「来訪者に価値あるコンテンツを提供した結果としてPVが増えることを目指す」と記載しているが、計画本文の具体的アクションはほぼ全てSEO最適化（KW最適化、metadescription改善、テキスト量拡充）として記述されている。constitutionのゴールは「best value for visitors」であり、「SEO最適化の結果として訪問者が増える」と「訪問者に最高の価値を提供した結果としてPVが増える」は因果の方向が異なる。各施策に「来訪者にとっての価値向上」の観点を1行でも追加することを推奨する。例: P1の「解説テキスト追加」は「ツールの使い方が分からない来訪者の疑問を解消し、ツールの活用度を高める」という来訪者価値として記述できる。

**S-02: CTR 5〜10%の仮定が検証されていない**

計画の上限目標の根拠（line 29）で「CTR 5〜10%を仮定すると数百PV/月が妥当」としているが、このCTR値はyolos.netの実績に基づくものではない。ドメイン年齢2ヶ月の新規サイトの平均CTRは一般的に1〜3%程度であり、5〜10%は検索結果1〜3位を前提とした数値。計画内で「推測」とは明記されているが、CTR仮定の幅が大きすぎるため（5%と10%で結果が2倍異なる）、保守的な値（例: 2〜5%）を併記するか、現時点のGoogle Search Console不在により検証不能であることを明記すべき。

**S-03: 選択肢Eのデイリーコンテンツ検討が中途半端**

選択肢E（line 169-170）で「リピーター100人で月3,000PV増の試算」に言及しているが、この試算の根拠がどのレポートのどこに書かれているか明示されていない。非SEOチャネル調査（レポート7）にはリテンションモデルの解説はあるが「100人で3,000PV」という具体的試算は見当たらなかった。出典を明示するか、この数値が計画策定時の独自推算であれば「推測」と明記すべき。

**N-01: cycle-162論理矛盾#1の解消記述がやや不明瞭**

矛盾#1（line 137）の解消説明で「350PV/月の積み上げ自体は否定しない」と書いているが、本計画の方向性はcycle-161の「診断SEO中心」から「ツールSEO中心」に明確に変わっている。「同じ方向性で」という矛盾の核心に対して、「方向性自体を変更した」ことを明示したほうが矛盾の解消として明瞭になる。

**N-02: 優先度1の「正規表現 テスト（月間Google 384+27,100単体）」の表記**

line 44の「384+27,100単体」は、384が「正規表現 テスト」のGoogle月間検索数、27,100が「正規表現」単体のGoogle月間検索数であるが、「+」で繋げると合算値のように読める。「正規表現 テスト: 月間Google 384（正規表現 単体: 27,100）」のように分離して表記すべき。

### プロセス上の問題

本サイクルでPMが犯した以下のプロセス違反を記録する:

1. **researcherの成果物をPMが手動で書き直した**: AI SEO調査レポートをresearcherが保存できなかった際、PMが独自に要約してファイルを作成し、researcherの報告内容を改変した。正しくはresearcherに再実行させるべきだった。
