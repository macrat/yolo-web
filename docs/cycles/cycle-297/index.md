# サイクル-297

## 背景と芯

来訪者が診断に求めるのは「自分はどのタイプか」という**答えそのもの**。その答えが、回答内容ではなく `quiz.results` の**配列の並び順**という無関係な実装都合で決まっているなら、玄関のコンテンツで静かに来訪者を裏切っている(constitution rule 2・4)。

`scoring.ts:90-100` の汎用 personality 判定は strict `>` で累計配点が最大のタイプを選ぶ。同点時は「配列で先に現れるタイプ」が必ず勝つ。character-personality はこの構造が Rule4 違反(回答でなく並び順が結果を決める)と確定し(cycle-294)、cycle-295 で専用判定へ移して是正済み。だが**同じ経路を残り10診断がそのまま使っている**。

cycle-294 は全12診断を計測し「配列順タイブレークによる**到達不能タイプはゼロ**」と結論した。しかしこれは**機械的到達性**(=どれかの回答組合せで到達できるか)の問いであり、character-personality で撤回された理屈そのもの。本当に問うべきは:

- **∀理想回答者テスト**: 各タイプ T について、「T に当てはまる人が正直に答えたとき、T を**同点に頼らず**(strict に)勝者にできる回答が存在するか」。存在しなければ、T は**本人に届かない**(配列順の運でしか手に入らない)。cycle-295 は character-personality でこのレンズを使い academic-artist/clever-guardian が本人に届かないと判明させた。

cycle-294 実測では、この10診断の同点率は **17〜31%**——4〜5人に1人が、正直に答えても配列順のコイン投げで結果を受け取っている可能性がある。

芯は「診断に答えてくれた人が受け取る型は、**その人の回答が決めた型**か。並び順の運で決まっていないか」を10診断すべてで確かめ、裏切りがあれば是正の的を定める。

## 本サイクルの成果物(スコープの明確化=cycle-294 事故の回避)

cycle-294 は「B-589 の価値の問いを、機械的タイブレーク検証にすり替え、道具を1本作って『是正不要』で出荷」した事故サイクルだった。本サイクルはその轍を踏まないため、成果物と非成果物を先に固定する:

- **成果物**: 10診断の**トリアージ評決**([triage.md](./triage.md))。各診断につき「健全 / 要是正(重篤度)」を、(a)理想回答者による strict 到達可能性の悉皆実測・(b)同点依存率・(c)診断が来訪者に約束するもの、の3点から判断。要是正診断は個別 backlog 項目として起票し、GA トラフィックで優先順位付け。トリビアルに是正できる欠陥(データ起因の恒常敗退等)があれば本サイクルで是正。
- **非成果物(明示的スコープ外)**: 各診断の判定作り直し(B-589 スケールの設問設計・軸判定への移行)。1診断=1サイクル規模のため、要是正と判明した診断ごとに個別サイクルで実施する。本サイクルで10診断分の作り直しを束ねるとレビュー不能に肥大化する。
- **「健全」評決の縛り**: ある診断を「是正不要」とするには、∀理想回答者テストが**実際に PASS**(全タイプが strict 到達可能 かつ 同点依存が来訪者の約束を裏切らない)することを根拠にする。「実害が来訪者に不可視だから無害」は根拠に使わない(=cycle-294 の撤回済み理屈)。

## 判定基準(トリアージ着手前に確定)

各診断の評決は以下の観測から機械的に導く。基準は着手前に固定し、結果を見て緩めない(AP-P02回避)。

- **観測1(strict 到達可能性・悉皆)**: 各タイプ T について、回答空間を悉皆列挙(規模が悉皆不能なら決定的サンプリングで、その旨と規模を triage.md に明記=silent cap 禁止)し、「T が strict 単独最大になる回答が1つ以上存在するか」を判定。**存在しない T=person-unreachable**(本人に届かない)。
- **観測2(同点依存率)**: 回答空間全体のうち、最大得点タイプが複数=同点になり配列順で決着する回答の割合。
- **観測3(配列順の集計歪み)**: 同点決着により配列先頭タイプが受け取る過剰勝率(%pt)。
- **観測4(来訪者への約束)**: その診断が UI/結果テキストで「これがあなた」と本人性を約束するか、それとも娯楽くじ(その日の運勢・相性の出目)として本人性を約束しないか。約束の強さで、同点決着が裏切りになるかが変わる。
- **評決の合成**:
  - **要是正(重篤)**: person-unreachable なタイプが1つ以上ある(=本人性を約束しているのに、その型の人に結果が届かない)。
  - **要是正(中)**: person-unreachable は無いが、本人性を約束する診断で同点依存率が高く(cycle-284 の character-personality 水準=概ね30%超を目安・具体閾値は観測後に triage で確定)、配列先頭への歪みが顕著。
  - **健全**: 全タイプ strict 到達可能 かつ(本人性を約束しない娯楽くじ or 同点依存が低く歪みが軽微)。

## 作業計画

### 目的

玄関の性格診断群(10診断)で、来訪者が受け取る型が「回答が決めた型」か「配列の並び順の運で決まった型」かを悉皆実測し、本人性を約束しているのに型が本人に届かない/高頻度で運任せになっている診断を特定して、是正の的を定める。

### 作業内容

各診断の回答空間を悉皆列挙し、∀理想回答者テスト(strict 到達可能性)・同点依存率・配列順歪みを実測。診断が来訪者に約束するもの(本人性 or 娯楽くじ)と突合して診断別評決を出す。要是正は個別 backlog へ。判定作り直しは各診断の別サイクル。

### 検討した他の選択肢と判断理由

- **B-603 T3(character-personality 24タイプ一覧の処遇)**: 不採用(今回)。1ページの結果領域レイアウト/IA の判断で、玄関1ページの presentation。一方 B-604 は10診断横断の「判定が本物か」という correctness の問いで、cycle-295→296 の芯(判定の実在化)を site 全体へ広げる。4〜5人に1人が運任せの結果を受け取っている可能性の方が来訪者への裏切りが深い。T3 は Queued に残す。
- **10診断すべてを本サイクルで作り直す**: 不採用。B-589 は1診断の判定作り直しで1サイクル(大)を要した。10診断束ねはレビュー不能に肥大化し Rule4 の品質担保を崩す。まずトリアージで的を定め、要是正は個別サイクルへ。
- **cycle-294 の『実害なし』を再確認して閉じる**: 不採用。cycle-294 は機械的到達性(どれかの回答で到達可能か)を測って「実害なし」としたが、これは character-personality で撤回された理屈。本人に届くか(理想回答者が strict 勝者にできるか)という**別の問い**を立てていない。同じ機械の問いを繰り返せば同じすり替えになる。
- **判定ロジックを一律ランダムタイブレークに変える**: 不採用(少なくとも本サイクルでは)。再現性(同じ回答→同じ結果=シェア/再受験の前提)を壊す。かつ person-unreachable の是正にはならない(運の偏りを別の運に変えるだけ)。是正は診断ごとの設問/配点設計の問題で、トリアージ後に個別判断。
