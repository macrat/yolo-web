# サイクル-297

## 背景と芯

来訪者が診断に求めるのは「自分はどのタイプか」という**答えそのもの**。その答えが、回答内容ではなく `quiz.results` の**配列の並び順**という無関係な実装都合で決まっているなら、玄関のコンテンツで静かに来訪者を裏切っている(constitution rule 2・4)。

`scoring.ts:90-100` の汎用 personality 判定は strict `>` で累計配点が最大のタイプを選ぶ。同点時は「配列で先に現れるタイプ」が必ず勝つ。character-personality はこの構造が Rule4 違反(回答でなく並び順が結果を決める)と確定し(cycle-294)、cycle-295 で専用判定へ移して是正済み。だが**同じ経路を残り10診断がそのまま使っている**。

cycle-294 は全12診断を計測し「配列順タイブレークによる**到達不能タイプはゼロ**」と結論した。しかしこれは**機械的到達性**(=どれかの回答組合せで到達できるか)の問いであり、character-personality で撤回された理屈そのもの。本当に問うべきは:

- **∀理想回答者テスト**: 各タイプ T について、「T に当てはまる人が正直に答えたとき、T を**同点に頼らず**(strict に)勝者にできる回答が存在するか」。存在しなければ、T は**本人に届かない**(配列順の運でしか手に入らない)。cycle-295 は character-personality でこのレンズを使い academic-artist/clever-guardian が本人に届かないと判明させた。

cycle-294 実測では、この10診断の**回答空間**の同点率は **17〜31%**——回答空間を一様に見ると、そのくらいの割合の回答が配列順のコイン投げで決着する。ただしこれは一様回答空間の割合であって、実来訪者の同点遭遇率ではない(実来訪者は一貫回答者=cycle-295 の測度 c で、一様回答者より同点は通常低い)。評決は同点率の大きさではなく「回答でなく並び順が結果を決めうる」という構造の有無で行う。

芯は「診断に答えてくれた人が受け取る型は、**その人の回答が決めた型**か。並び順の運で決まっていないか」を10診断すべてで確かめ、裏切りがあれば是正の的を定める。

## 本サイクルの成果物(スコープの明確化=cycle-294 事故の回避)

cycle-294 は「B-589 の価値の問いを、機械的タイブレーク検証にすり替え、道具を1本作って『是正不要』で出荷」した事故サイクルだった。本サイクルはその轍を踏まないため、成果物と非成果物を先に固定する:

- **成果物**: 10診断の**トリアージ評決**([triage.md](./triage.md))=全10診断がタイブレーク品質欠陥を共有することの明示+重篤度3階層+ dead type(wakakusa)の**本サイクル是正**。共通是正 B-606 を起票し GA トラフィックで優先順位付け。
- **非成果物(明示的スコープ外=最小スコープ維持)**: 10診断分の結果先行の再設計(B-606・タイブレーク差替はその一部・共有 scoring の挙動変更を含む)。波及が大きく focused な review を要するため束ねず繰越。**唯一の例外**=調査中に発見した dead type(wakakusa)は本サイクルで是正した(トリビアルなデータ是正で、放置は『調べたが直さない』cycle-294 型になるため)。
- **品質基準(constitution Rule4)**: 唯一の判断基準は constitution。「エンタメだから品質欠陥を放置してよい」はしない(Rule4=every aspect で最高品質)。約束の強さは優先順位を決めるだけで免除の根拠にしない。「実害が来訪者に不可視だから無害」も根拠に使わない(cycle-294 の撤回済み理屈)。

## 判定基準(トリアージ着手前に確定)

各診断の評決は以下の観測から機械的に導く。基準は着手前に固定し、結果を見て緩めない(AP-P02回避)。

- **観測1(理想回答者テスト)**: 各タイプ T について「**T を各設問で最大化する正直回答**で T が strict 単独最大になるか」を判定。ならなければ T=person-unreachable(本人に届かない)。※当初これを「回答空間のどこかに単独最大が存在するか」という弱い到達性で実装し誤ったため、正しい理想回答者テストへ是正した(補足事項)。
- **観測2(同点依存率)**: 回答空間全体のうち、最大得点タイプが複数=同点になり配列順で決着する回答の割合。
- **観測3(配列順の集計歪み)**: 同点決着により配列先頭タイプが受け取る過剰勝率(%pt)。
- **観測4(来訪者への約束)**: その診断が UI/結果テキストで「これがあなた」と本人性を約束するか、それとも娯楽くじ(その日の運勢・相性の出目)として本人性を約束しないか。約束の強さで、同点決着が裏切りになるかが変わる。
- **評決の合成(constitution Rule4 に照らした最終版)**: **配列順タイブレークは全診断で Rule4 品質欠陥**(エンタメでも免除しない)。重篤度で優先順位を付ける:
  - **dead type(最重篤)**: 正直な理想回答者でも strict に負け、その型が本人に永久に届かない(=配点データ欠陥・タイブレークでは直らない)。本サイクルで是正。
  - **高優先**: 強い本人性(「これがあなた」と言い当てる)を約束する診断のタイブレーク=裏切りが最も可視。
  - **低優先(免除ではない)**: 免責つき娯楽くじのタイブレーク=可視性は低いが品質欠陥は残る。B-606(結果先行の再設計)で是正。
  - ※当初は「person-unreachable 無し=健全」という枠組みだったが、(a)harness の person-unreachable 判定が誤り(弱い到達性)で wakakusa を見逃した・(b)「エンタメだから健全」は Rule4(every aspect 最高品質)の免除にならない——の2点で是正した((b)は Owner が Rule4 を指差したのを機に PM が是正=駆動源は Rule4)。

## 作業計画

### 目的

玄関の性格診断群(10診断)で、来訪者が受け取る型が「回答が決めた型」か「配列の並び順の運で決まった型」かを悉皆実測し、本人性を約束しているのに型が本人に届かない/高頻度で運任せになっている診断を特定して、是正の的を定める。

### 作業内容

各診断の回答空間を悉皆列挙し、∀理想回答者テスト(strict 到達可能性)・同点依存率・配列順歪みを実測。診断が来訪者に約束するもの(本人性 or 娯楽くじ)と突合して診断別評決を出す。要是正は個別 backlog へ。判定作り直しは各診断の別サイクル。
