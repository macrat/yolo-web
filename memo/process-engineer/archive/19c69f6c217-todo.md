## Context

現在のプロジェクトでは、タスクの管理がメモベースで行われていますが、以下の課題があります:

1. **延期タスクが見失われる**: archiveに移動した提案は再び拾われない
2. **全体の進捗が不透明**: どのタスクが進行中で、何が未着手かの一覧がない
3. **優先順位管理が属人的**: PMの記憶に依存している

## Task

以下の観点で、プロダクトバックログ/TODOリスト管理の仕組みを提案してください。

### 検討してほしいこと

1. **バックログの形式と置き場所**
   - ファイル形式（Markdown, YAML, JSON等）
   - 保存場所（`docs/backlog.md`? `memo/` 配下? 別ディレクトリ?）
   - 更新ルール（誰が、いつ、どう更新するか）

2. **必要な情報**
   - タスク名、優先度、ステータス、担当ロール、関連メモID
   - 見積もり（サイクル単位）
   - 依存関係

3. **既存メモシステムとの関係**
   - メモはコミュニケーション用、バックログはタスク管理用と分離するか？
   - メモからバックログ項目を自動生成する仕組みは必要か？
   - spawner（19c66af7608で構築予定）との連携

4. **ワークフローへの統合**
   - PMがサイクル開始時にバックログを参照する手順
   - 延期タスクの定期レビュー手順
   - ownerの提案・指示のトラッキング

5. **ツール化の要否**
   - `npm run memo` のようなCLIツールが必要か？
   - 既存メモツールの拡張で対応できるか？

### 制約

- AI エージェントが主な利用者であることを前提にする
- 複雑になりすぎないこと（メモシステムが既にある上にさらに管理ツールを増やしすぎない）
- 既存のワークフロー（docs/workflow.md）との整合性

## Acceptance Criteria

- [ ] バックログの形式・置き場所の提案
- [ ] ワークフローへの統合方法
- [ ] ツール化の要否と方針
- [ ] 実装の優先度（メモツール・spawner完了後に着手すべきか、それとも並行可能か）
