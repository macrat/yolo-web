サイトに新しいサービスを追加したとき(最近だと色関連など)や、コンテンツを大幅に増やしたとき(ツール群の追加など)は、必ずそれに関するブログを書くようにしてください。
その記事では、どうしてそのコンテンツを選んだのか、どのような意図で今の設計にしたのか、今後の展望はあるのか、などを詳しく書いてください。
最近行った新規サービス追加や大幅なツールの追加などの履歴を調べて、その時点の日時でブログ記事を書いてください。
今後に関しては、ワークフロー等のドキュメントにブログ記事の追加や更新を含めるようにしてください。

また、サイト名の決定も大きな変更なので、これについても記事を書いてください。
新しいサイト名である「yolos.net」は、 **YOLO (You Live Only Once)** と **よろず** を掛けたもので、AI主導であらゆるコンテンツを扱うという意図が込められています。
プロジェクト立ち上げ直後に使っていたYolo-Webという名前は検索性が悪かったので、ある程度検索ボリュームが少ない名前として付けました。
また、元々のドメインである<https://yolo.macr.app>というドメインは、ownerが作ったツールを公開するのに使っている macr.app のサブドメインなので、自分が管理しきれているわけではないコンテンツを置くことによるレピュテーションリスクを考えて別のドメインに切り替えたという意図もあります。
これらの情報を魅力的な記事にまとめて、サイト名変更を告知する記事にしてください。

これらの重要な変化や意思決定に関する記録は、このプロジェクトの目的である「AIに企画・開発・運営を任せたらどうなるのか」という実験の成果を分かりやすく残すものになるはずです。
よりよい記録を残してより成果を挙げられるように、何を考えてどのように工夫してここに至ったのか、経緯を克明に記してください。
