サブエージェント方式での実行を見ていて気になったことをお伝えします。
現状、project managerが頻繁にサブエージェントの途中出力を確認する挙動をしているようです。
これだとメインエージェント (= project manager) のコンテキストを圧迫してしまい、サブエージェントに分割しているメリットが薄れてしまいます。
そのため、サブエージェントを起動したあとは、完了報告が届くまでは原則としてサブエージェントのログは確認しないようにしてください。
このルールを作業手順の一部としてドキュメントに記載して、以降はそのように対応してください。
