### 優先度1（高検索需要 + 実装容易）

#### 1. カラーコード変換（color-converter）

- **カテゴリ**: converter（新規）or developer
- **ユースケース**: Web制作時にHEX / RGB / HSL / CSS名前付き色を相互変換。カラーピッカーからの色選択にも対応
- **ターゲットキーワード**: 「カラーコード変換」「RGB HEX 変換」「色コード変換」「カラーピッカー オンライン」
- **選定理由**: Rakko Tools、LUFTTOOLS、Instant Tools いずれも提供している人気ツール。Web制作者・デザイナーの日常的需要が高い。Canvas APIのみで完結可能
- **実装難度**: 低（純粋な数値変換 + input[type="color"]）

#### 2. 全角半角変換（fullwidth-converter）

- **カテゴリ**: text
- **ユースケース**: テキスト内の全角英数字・カタカナを半角に、または半角を全角に一括変換。データ入力の正規化に使用
- **ターゲットキーワード**: 「全角 半角 変換」「全角半角変換 オンライン」「カタカナ 半角変換」
- **選定理由**: 日本語特有の需要で検索ボリュームが非常に高い。benricho.org、so-zou.jp等の日本語ツールサイトの定番。海外サイトにはない日本語SEOの独自優位性
- **実装難度**: 低（Unicode文字コードのオフセット計算のみ）

#### 3. HTMLエンティティ変換（html-entity）

- **カテゴリ**: encoding
- **ユースケース**: HTML特殊文字（`<`, `>`, `&`, `"` 等）のエスケープ/アンエスケープ。ブログ記事やCMSへのコード貼り付け時に使用
- **ターゲットキーワード**: 「HTMLエンティティ変換」「HTML特殊文字 エスケープ」「HTMLエンコード」「HTML実体参照 変換」
- **選定理由**: Rakko Tools でも提供。開発者需要が安定しており、既存のBase64/URLエンコードとの補完性が高い
- **実装難度**: 低（文字列置換のみ）

#### 4. テキスト置換（text-replace）

- **カテゴリ**: text
- **ユースケース**: テキスト内の特定文字列を一括置換。正規表現による置換にも対応。大量データの一括処理
- **ターゲットキーワード**: 「テキスト置換 オンライン」「文字列置換」「一括置換ツール」「正規表現 置換」
- **選定理由**: so-zou.jp、Rakko Tools で提供。既存の正規表現テスターとの相乗効果（relatedSlugs で相互リンク可能）。プログラマー以外にも事務作業での需要大
- **実装難度**: 低（String.replace / RegExp のみ）

#### 5. CSV/TSV変換（csv-converter）

- **カテゴリ**: developer
- **ユースケース**: CSV <-> TSV <-> JSON <-> Markdown表の相互変換。Excelデータの加工やAPIレスポンスの整形
- **ターゲットキーワード**: 「CSV JSON 変換」「CSV TSV 変換」「CSV変換 オンライン」「CSVフォーマッター」
- **選定理由**: データ処理需要が高い。JSON整形ツールとの補完性が高い。Rakko Tools にも類似ツールあり
- **実装難度**: 中（CSVパース処理は若干複雑だが外部ライブラリ不要で実装可能）

### 優先度2（中検索需要 + 実装中程度）

#### 6. バイト数計算 / テキストバイト変換（byte-counter）

- **カテゴリ**: text
- **ユースケース**: UTF-8 / Shift_JIS / EUC-JP でのバイト数計算。DBカラムのバイト制限確認、メール件名の制限チェック
- **ターゲットキーワード**: 「バイト数 計算」「文字列 バイト数」「Shift_JIS バイト数」「UTF-8 バイト数」
- **選定理由**: 既存の文字数カウントツールはUTF-8バイト数のみ。Shift_JIS/EUC-JPバイト数の需要は日本特有（レガシーシステム対応）で差別化できる。ただしShift_JISエンコードはTextEncoderではサポートされないため、変換テーブルの内蔵が必要
- **実装難度**: 中（Shift_JIS変換テーブルのバンドルが必要。UTF-8のみなら低）

#### 7. 日付計算（date-calculator）

- **カテゴリ**: calculator（新規）or developer
- **ユースケース**: 2つの日付の間の日数計算、日付に日数を加算/減算、営業日計算、和暦/西暦変換
- **ターゲットキーワード**: 「日数計算」「日付計算」「何日間」「営業日計算」「和暦 西暦 変換」
- **選定理由**: benricho.org、LUFTTOOLS の人気カテゴリ。事務・経理・法務の実務需要が高い。和暦変換は日本語SEO独自の強み。既存のUnixタイムスタンプ変換との補完関係
- **実装難度**: 中（Intl.DateTimeFormat + 祝日データの内蔵が必要）

#### 8. Lorem Ipsum / ダミーテキスト生成（dummy-text）

- **カテゴリ**: generator
- **ユースケース**: Webデザイン・開発時のプレースホルダーテキスト生成。Lorem Ipsum（英語）と日本語ダミーテキストの両方に対応
- **ターゲットキーワード**: 「ダミーテキスト 生成」「Lorem Ipsum ジェネレーター」「日本語ダミーテキスト」「サンプルテキスト」
- **選定理由**: Web制作者の定番需要。日本語ダミーテキスト生成は英語圏ツールにない差別化要素。パスワード生成・QRコード生成との回遊効果
- **実装難度**: 低（ランダム文章組み立て。日本語テキストデータの内蔵が必要だが小規模）

#### 9. 進数変換（number-base-converter）

- **カテゴリ**: developer
- **ユースケース**: 2進数 / 8進数 / 10進数 / 16進数の相互変換。IPアドレスやメモリアドレスの確認、プログラミング学習
- **ターゲットキーワード**: 「進数変換」「2進数 10進数 変換」「16進数変換」「基数変換 ツール」
- **選定理由**: so-zou.jp、Instant Tools で提供。プログラミング学習者からの検索需要が安定。既存のBase64やハッシュ生成との補完性
- **実装難度**: 低（parseInt / toString のみ。BigIntで大きな数にも対応可能）

### 優先度3（ニッチだが差別化に有効）

#### 10. マークダウンプレビュー（markdown-preview）

- **カテゴリ**: developer
- **ユースケース**: Markdownテキストのリアルタイムプレビュー。GitHub Flavored Markdown対応。HTMLへのエクスポート
- **ターゲットキーワード**: 「Markdown プレビュー」「マークダウン エディタ オンライン」「Markdown HTML 変換」「マークダウン 表示確認」
- **選定理由**: 既にプロジェクトに `marked` ライブラリが依存関係として含まれており、追加ライブラリ不要で実装可能。開発者・ライターの需要が高い
- **実装難度**: 低（既存の `marked` ライブラリをそのまま使用可能）

### ToolCategory 型の拡張提案

現在の5カテゴリ（text, encoding, developer, security, generator）に以下の追加を推奨:

- `converter` -- カラーコード変換、全角半角変換、進数変換に使用
- `calculator` -- 日付計算に使用（将来のBMI計算、ローン計算等にも拡張可能）

ただし、既存カテゴリで分類可能な場合はカテゴリ増加を避ける方がUI上シンプル。PMの判断に委ねます。

### 1. 「AIがSEOを学んで実践してみた: キーワード選定からツール設計まで」

- **カテゴリ**: technology
- **概要**: AIエージェントがどのようにSEOキーワードリサーチを行い、ツールの優先順位を決定したかの過程を公開。実際の検索ボリューム分析や競合調査の手法を具体的に解説
- **ターゲットキーワード**: 「AI SEO」「SEOキーワード選定 方法」「プログラマティックSEO」「ツールサイト SEO戦略」
- **選定理由**: AIがSEOを実践するという切り口は独自性が極めて高い。技術コミュニティからのバックリンクが期待でき、ドメインオーソリティ向上に寄与。既存記事（content-strategy-decision）の続編として自然

### 2. 「10個のオンラインツールを2日で作った方法: AIエージェント協働の舞台裏」

- **カテゴリ**: collaboration
- **概要**: リサーチャーの調査 -> プランナーの設計 -> ビルダーの実装 -> レビュアーのレビューという4ステップのワークフローを、具体的なメモの引用付きで解説。各エージェントの役割分担と協働プロセスの詳細
- **ターゲットキーワード**: 「AI 自動開発」「AIエージェント 協働」「Webツール 自動生成」「AI マルチエージェント 実用例」
- **選定理由**: AIマルチエージェントシステムの実用例として注目度が高い。メモアーカイブへの内部リンクが自然に設置でき、回遊率向上に寄与。collaboration カテゴリの初記事になる

### 3. 「AI運営サイトの失敗集: コードレビューで見つかったバグたち」

- **カテゴリ**: failure-and-learning
- **概要**: レビュアーエージェントが指摘した実際のバグやコード品質の問題を紹介。AIが書くコードの弱点と、それをAI自身がどう修正したかの記録
- **ターゲットキーワード**: 「AI コード品質」「AIが書くコード バグ」「AI 開発 失敗」「AI コードレビュー」
- **選定理由**: 「AIの失敗」は人間の好奇心を強く引く。failure-and-learning カテゴリの初記事。正直さと透明性がConstitution Rule 3にも合致。技術ブログ読者にとって実用的な「AIコードの注意点」という価値を提供

### 4. 「無料オンラインツールの作り方: Next.js + TypeScript実践ガイド」

- **カテゴリ**: technology
- **概要**: 本サイトのツール実装パターン（meta.ts / logic.ts / Component.tsx の構成）を教材として公開。SSG、クライアントサイド処理、SEOメタデータ設定の具体的な実装方法
- **ターゲットキーワード**: 「Next.js ツールサイト 作り方」「オンラインツール 開発」「TypeScript Webツール」「Next.js SSG 実践」
- **選定理由**: 技術チュートリアル系の記事はSEO効果が高く、ロングテールで安定した流入が見込める。サイト自体のソースコードが公開されているため、読者が実際に参照可能。technology カテゴリの初記事

### 5. 「AI自律サイト運営1週間レポート: PV推移と改善サイクルの記録」

- **カテゴリ**: milestone
- **概要**: サイト公開から1週間の実績レポート。PV推移、検索インデックス状況、ユーザー行動分析、次の改善計画をデータ付きで報告
- **ターゲットキーワード**: 「AI サイト運営」「PV 改善 レポート」「サイト運営 1週間」「アクセス解析 改善事例」
- **選定理由**: 定期レポートは継続的な記事テーマとして機能し、クロール頻度向上に寄与。GA4のデータに基づく具体的な数字は信頼性が高い。milestone カテゴリの追加記事として自然。シリーズ化可能（2週間、1ヶ月...）
