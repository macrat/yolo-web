# 計画レポート: B-190 漢字カナル カテゴリ分類の修正

## 1. 目的と提供価値

**ターゲットユーザー**: 漢字カナルのプレイヤー（常用漢字の基本知識を持つ日本語話者）

**提供価値**: カテゴリフィードバックが「一般的な漢字の常識」で解釈可能なヒントとして機能すること。プレイヤーが「近い」と言われた時に「なるほど、同じ系統か」と納得でき、次の推測の手がかりにできること。

**解決する問題**: 現状の部首番号範囲による機械的分類（女が「天文」、心が「動作」、木水火が同一グループ）をプレイヤー目線で意味の通るグルーピングに置き換える。

## 2. 採用アプローチ

**方向性A「部首番号の意味的グルーピング」を採用する。**

理由:
- 確定的・再現可能（AI不要、部首番号からグループへの1対1マッピング）
- 作業対象は214部首であり、2,136字の個別分類は不要
- KANJIDIC2の部首番号データは完全に確定的
- 旧セマンティックカテゴリ（AI生成）の品質問題を回避できる
- 旧機械的分類（番号範囲）の意味不整合を解消できる

## 4. スーパーグループ設計（「近い」フィードバック用）

| スーパーグループ | 名前 | 含まれるグループ |
|---|---|---|
| A | 人体系 | 1(人・家族), 2(体・器官), 3(手・力), 4(心・感情), 5(口・言語), 6(足・移動) |
| B | 自然系 | 7(水), 8(火), 9(木・植物), 10(土・山), 11(天・気象), 12(動物) |
| C | 文明系 | 13(建物), 14(布・衣), 15(刀・武器), 16(金・素材), 17(食・飲), 18(乗り物), 19(社会) |
| D | 抽象系 | 20(抽象・記号) |

**直感的解釈の例**:
- 「海」(水)と「山」(土・山) → 両方自然系 → 「近い」 ... 納得できる
- 「話」(口・言語)と「走」(足・移動) → 両方人体系 → 「近い」 ... 体の機能として納得
- 「刀」(刀・武器)と「食」(食・飲) → 両方文明系 → 「近い」 ... 人間の道具・文化として納得
- 「火」(火・光)と「心」(心・感情) → 自然系 vs 人体系 → 「遠い」 ... 異なる世界

## 5. ゲーム性の検証: 具体的なプレイシナリオ

### シナリオ1: 答えが「海」(部首:水、グループ7:水・液体)
- 1手目「山」→ カテゴリ「近い」(土・山=自然系) → 「自然系か。水か木か動物か...」
- 2手目「花」→ カテゴリ「近い」(木・植物=自然系) → 「まだ自然系。水か火か天か...」
- 3手目「池」→ カテゴリ「正解」(水) → 「水グループだ」→ 部首・画数等で絞り込み
- プレイヤーが自然な連想で次の手を考えられる

### シナリオ2: 答えが「読」(部首:言、グループ5:口・言語)
- 1手目「走」→ カテゴリ「近い」(足・移動=人体系) → 「人体系か」
- 2手目「思」→ カテゴリ「近い」(心・感情=人体系) → 「まだ人体系。口か手か体か...」
- 3手目「話」→ カテゴリ「正解」(口・言語) → 「言語系だ」→ 絞り込み続行

### シナリオ3: 答えが「無」(部首:火、グループ8:火・光)
- これは「不可避な違和感」の例。「無」の部首が火であることを知らないプレイヤーもいる
- しかし「無」の下部に「灬」(火の変形)があることは見ればわかる
- 旧システムの「女が天文」とは本質的に異なり、漢字の構成要素に基づく分類として説明可能

### 情報量の検証
- カテゴリ20グループ(約4.1 bits) + grade 7段階(約2.4 bits) + strokeCount(約1.5 bits) + radical(バイナリだが198種でヒット時の情報量大) + onYomi + kunYomiCount
- log2(2136) = 11.1 bits → 6回の推測で十分到達可能
