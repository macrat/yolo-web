# 依頼: 新規コンテンツ候補の選定

## 背景

yolos.netというWebサイトに追加する新規コンテンツの候補を、市場調査の結果から選定してください。

## 重要な制約

- **docs/targets/ は絶対に読まないでください。**
- **src/ 配下のコードを読まないでください。**
- **既存のサイトコンテンツ（ゲーム、ツール、ブログ等）は読まないでください。**
- 既存のものに引きずられず、純粋に「技術制約のもとで作れるもの」×「市場に需要があるもの」の交差点から候補を選んでください。

## 技術制約

技術制約は .claude/rules/coding-rules.md に定義されています。必ず読んで正確に把握してください。
特に以下の点に注意してください:
- サーバーサイドJSは禁止されていません。Route HandlersやServer Componentsは利用可能です。
- 禁止されているのは外部API呼び出し・データベース・認証です（ユーザーデータ漏洩防止のため）。

## 開発・運用体制

- AI（Claude）による開発・運用（人間のowner 1名 + AIエージェント）
- 大量×高品質は不可能（Claude Opusは高コスト、安価モデルは品質低下）
- 2つの現実的戦略:
  - 「少量×高品質」: 作り込みゲーム・高機能ツール。1つ1つを丁寧に作る。
  - 「大量×低品質」: 占い・診断等、正確性が不要な分野。安価モデルで大量生成してJSON配信。

## やること

市場調査の結果から、有望なコンテンツ候補を8〜12個選定してください。

### 選定基準

各候補について以下を評価してください:

1. **技術的実現可能性**: coding-rules.mdの制約内で実現できるか。
2. **市場需要**: 検索ボリュームや競合の状況から需要が見込めるか。
3. **独自性**: 既存の競合サービスに対して、明確な差別化ポイントがあるか。**必ず自分でWebを検索して競合の存在を確認すること。** 市場調査メモの記述を鵜呑みにしないこと。
4. **データ要件**: 必要なデータの量と品質管理コスト。コードの外にある素材（音声・画像等）のコストも明記。
5. **戦略分類**: 「少量×高品質」と「大量×低品質」のどちらに該当するか。

### 注意事項

- 特定のテーマに偏らず、4件の市場調査を公平に横断評価してください。
- 「大量×高品質」が必要なコンテンツは選ばないでください。
- 前回レビュー（19cb6876d16）で指摘された競合見落とし（候補4・6・7・10）と同じ誤りを繰り返さないこと。
