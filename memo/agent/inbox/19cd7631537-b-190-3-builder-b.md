# B-190タスク3 Builder B: クライアントコア改修

## 計画
メモ 19cd75a6de2 のタスク3e, 3f, 3i, 3j, 3n を読んでください。

## 作業内容の要約

### 1. 軽量バリデーションデータの作成（3n）
- src/games/kanji-kanaru/data/joyo-kanji-set.ts を新規作成
- src/data/kanji-data.json から2,136字の文字のみを抽出して Set<string> としてexport
- 約6KBのデータ

### 2. types.ts の変更（3i）
- GameState.targetKanji: KanjiEntry → KanjiEntry | null
- 新しいAPI型定義を追加:
  - EvaluateRequest: { guess: string, puzzleDate: string, difficulty: Difficulty, guessNumber: number }
  - EvaluateResponse: { feedback: GuessFeedback, isCorrect: boolean, targetKanji?: { character: string, onYomi: string[], kunYomi: string[], meanings: string[], examples: string[] } }
  - HintsResponse: { puzzleNumber: number, hints: { strokeCount: number, onYomiCount: number, kunYomiCount: number } }

### 3. storage.ts の変更（3j）
- GameHistoryのエントリにfeedbacks: GuessFeedback[] フィールドを追加
- saveHistory/loadHistory: フィードバックデータも保存・読み込み
- 後方互換性: feedbacksフィールドがないデータも正常に読み込める

### 4. GameContainer.tsx の大幅改修（3e）
これが最大の変更。以下を実施:

a) **import変更**:
   - evaluateGuess, lookupKanji, allKanjiData のimportを削除
   - kanji-data.json, puzzle-schedule-*.json のimportを削除
   - joyo-kanji-set.ts をimport

b) **初期化フローの非同期化**:
   - buildGameState関数を廃止
   - マウント時に GET /api/kanji-kanaru/hints をfetchしてpuzzleNumber + ヒント情報取得
   - ローディング状態管理: const [loading, setLoading] = useState(true)
   - localStorageからguesses + feedbacksを復元（APIコール不要）
   - 旧形式（feedbacks未定義）の場合はAPIフォールバック

c) **handleGuessの非同期化**:
   - 戻り値: string | null → Promise<string | null>
   - バリデーション: JOYO_KANJI_SET.has(input) で常用漢字チェック
   - パス後: POST /api/kanji-kanaru/evaluate をfetch
   - レスポンスからfeedbackを取得しstate更新
   - guess送信中: const [submitting, setSubmitting] = useState(false)
   - isCorrectはAPIレスポンスで判定
   - ゲーム終了時: targetKanjiをstateにセット

d) **targetKanjiの管理変更**:
   - ゲーム開始時はnull
   - ゲーム終了時にAPIレスポンスからセット
   - HintBar: hints APIの結果を別state（hintsData）で管理

e) **ローディング・エラー状態**:
   - 初期化中: ゲームエリアにローディング表示
   - guess送信中: GuessInputをdisable + ローディング
   - API失敗: エラーメッセージ + リトライ

f) **難易度変更時**: hints APIを再fetch

### 5. GuessInput.tsx の変更（3f）
- onSubmitの型: (kanji: string) => string | null → (kanji: string) => Promise<string | null>
- handleSubmit内でawait
- submitting中のdisable制御

## APIインターフェース（Builder Aが実装するもの）

```
POST /api/kanji-kanaru/evaluate
Request: { guess, puzzleDate, difficulty, guessNumber }
Response: { feedback: GuessFeedback, isCorrect: boolean, targetKanji?: {...} }

GET /api/kanji-kanaru/hints?date=YYYY-MM-DD&difficulty=xxx
Response: { puzzleNumber, hints: { strokeCount, onYomiCount, kunYomiCount } }
```
