# 匿名化コンテンツリスト作成依頼

## 目的

サイトコンセプトをゼロベースで検討するため、「既存コンテンツ」と「新規コンテンツ候補」を区別なく匿名化した統一リストを作成する。このリストは別のエージェント（planner）に渡され、どれが既存でどれが新規かを知らない状態でコンセプト案を策定させるために使う。

## 作業内容

### A. 既存コンテンツの調査と評価

サイトの既存コンテンツを調べ、それぞれを以下の5軸で評価（各1-5点）してください:
1. 独自性: 競合と比べて独自の価値があるか
2. 需要: ユーザーの需要がどの程度あるか
3. 実装可能性: （既に実装済みなので5点固定）
4. 継続性: リピート利用・習慣化が見込めるか
5. 品質達成可能性: 高品質を維持できるか

調べるべき既存コンテンツ:
- src/app/games/ 配下の全ゲーム（各ゲームのpage.tsxを読んで内容を把握）
- src/app/tools/ 配下のうち、日本語・日本文化に特化したツール（汎用開発ツールは除外）
- src/app/tools/ 配下の汎用開発ツール群（まとめて1項目として扱う）
- src/app/blog/ のブログ
- src/app/dictionary/, src/app/quiz/, src/app/cheatsheets/ の内容

各コンテンツについて、簡潔な説明（何をするものか、どんな価値を提供するか、差別化ポイント）も記載してください。

### B. Phase E結果の取り込み

/tmp/phase_e_all_results.txt にPhase E市場調査結果（Aランク31件）が記録されています。
各Aランクアイテムの以下の情報を抽出してください:
- コンセプト名と説明
- 5軸スコア（独自性、需要、実装可能性、継続性、品質達成可能性）
- 総合スコア
- 差別化ポイント

### C. 匿名化統一リストの作成

以下のルールで統一リストを /tmp/phase_f_anonymized_items.txt に出力してください:

1. 既存コンテンツ（Aで調査した分）と新規候補（Bの31件）をすべてまとめる
2. 各アイテムにランダムなID（P01, P02, ..., PXX）を振る。Python random.shuffle(seed=123)でシャッフル
3. 「既存」「新規」「現行」などの出自を示す情報は一切含めない
4. 各アイテムのフォーマット:

```
## P01: [コンセプト名]

**概要**: [何をするものか、1-2文]
**提供価値**: [ユーザーにとっての価値]
**差別化ポイント**: [競合との差別化要素]
**テーマ領域**: [関連するテーマ（地理、言語、色彩、etc.）]
**コンテンツ形式**: [デイリーゲーム/ツール/診断/ブログ/etc.]

| 評価軸 | スコア |
|---|---|
| 独自性 | X |
| 需要 | X |
| 実装可能性 | X |
| 継続性 | X |
| 品質達成可能性 | X |
| **総合** | **X.X** |
```

5. シャッフル後の全アイテムを通しで書き出す
6. ファイルの冒頭に「# サイトコンテンツ候補一覧」とだけ記載。出自に関する説明は一切書かない

### 重要な注意

- このリストを見るplannerは、どれが既存でどれが新規か区別できてはいけない
- 既存コンテンツの説明に「現在運用中」「既にリリース済み」等を書かないこと
- 新規候補の説明に「市場調査の結果」「Phase Eで評価」等を書かないこと
- 全アイテムが同じトーンで記述されていること
