## 5. 削除済みツールのスラッグと代替手段

### 記事内で参照されているツールスラッグ

- `hash-generator`: SHA-1/SHA-256/SHA-384/SHA-512のハッシュ値生成（現在は存在している）
- `password-generator`: パスワード生成（現在は存在している。記事内ではまとめセクションのみで言及）

### 代替として案内すべきコマンドラインツール

修正方針メモで提示された代替手段の適切さを確認した。

**macOS:**
```
shasum -a 256 ファイル名    # SHA-256
shasum -a 512 ファイル名    # SHA-512
shasum -a 1 ファイル名      # SHA-1（非推奨）
```

**Linux:**
```
sha256sum ファイル名
sha512sum ファイル名
sha1sum ファイル名（非推奨）
md5sum ファイル名（セキュリティ用途不可）
```

**Windows:**
```
certutil -hashfile ファイル名 SHA256
certutil -hashfile ファイル名 SHA512
certutil -hashfile ファイル名 MD5（互換性確認のみ）
```

**評価**: これらのコマンドラインツールはすべてOSに標準搭載されており、外部サービスに依存しないため代替として非常に適切。なお、記事の「ソフトウェアのダウンロード検証」セクション（行102）には既にこれらのコマンド例が書かれており、コマンドライン利用の説明は既に記事内に存在する。

**テキストハッシュ生成（ファイルではなく文字列）の代替:**
- ブラウザのJavaScriptコンソール（Web Crypto API）で計算可能
- CyberChef（https://gchq.github.io/CyberChef/）などの外部ツール

## 7. ターゲットユーザーと記事価値

### この記事のターゲット

**主ターゲット: AIエージェントやオーケストレーションに興味があるエンジニア（サブターゲット）**

理由:
- ハッシュの技術知識（MD5/SHA-1/SHA-256の違い、衝突攻撃の歴史）はエンジニア向け
- コマンドライン使用例（shasum, certutil）はエンジニア前提の内容
- パスワード保存とbcrypt/Argon2の解説はバックエンド開発者向け
- メインターゲット（占い・診断を楽しみたい人）には無関係な内容

**サブターゲットの特性との適合:**
- likes「反面教師となりうる具体的な失敗例とその原因分析」→ MD5/SHA-1の脆弱性解説が合致
- likes「試行錯誤の過程と、なぜその判断に至ったかの考察」→ MD5非対応の理由セクションが合致
- dislikes「なぜそうしているのかの説明がなく、ただ手順だけが示される記事」→ 現在の記事は「なぜ」を重視した構成で適合

### 記事が提供する価値

1. **MD5/SHA-1の脆弱性と使うべきでない理由**: 単に「使うな」でなく衝突攻撃の歴史を説明
2. **パスワード保存に汎用ハッシュが不適切な理由**: bcrypt/Argon2が必要な理由まで解説
3. **実務でのコマンドライン活用**: ファイル検証の具体的な手順（既に記事内に記載）
4. **NISTの最新勧告**: SHA-2/SHA-3の推奨、SHA-1の2030年廃止予定

この知識は**ツールの存在に依存しない**。ツール削除後も記事の教育的価値は変わらない。
