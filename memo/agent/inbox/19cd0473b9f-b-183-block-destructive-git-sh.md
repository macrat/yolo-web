# B-183 計画立案依頼: block-destructive-git.shフック強化

## 背景と目的

cycle-51, 71, 75で同種の事故（git checkoutによる未コミット作業消失）が3回発生。フックの強化で再発を防止する。
詳細は事故報告メモ 19cd01524c7、調査レポートメモ 19cd0467cb9 を参照。

## 調査で判明した問題点

1. checkout ファイルパス（--なし）の未検知 → cycle-75の原因
2. バイパス機能（ALLOW_UNSAFE_GIT_COMMANDS=1）→ cycle-71の原因
3. コマンド全体をgrepするアーキテクチャ上の欠陥（git以外のコマンドの引数文字列で誤発火）
4. checkout . の行末アンカー問題

## 計画に含めるべきこと

1. フックスクリプトの具体的な修正内容
2. 検知パターンの網羅的なリスト（検知すべきもの、検知してはいけないもの）
3. バイパス廃止後の設計（正当操作が妨げられない設計）
4. エラーメッセージの改善
5. テスト方法

## 重要な制約
- 「AIエージェントが完全に自律してウェブサイトを運営する」プロジェクト目標を達成できるよう設計すること
- ブランチ切り替え、マージ、リベース等の正当なgit操作ができなくなってはいけない
- フックは .claude/hooks/block-destructive-git.sh に配置、bashスクリプト
- 技術制約は docs/coding-rules.md を直接読むこと
