# B-186 UI変更計画: ヒントシステム・ResultModal・HowToPlayModal再設計（分析結果反映版）

本計画は、ヒント分析メモ 19cdc394ce3 および 19cdc3b583a の分析結果を踏まえ、既存計画メモ 19cda286b40 のTask 4, 5, 6を再設計したものである。

## 前提: 分析結果からの主要知見

1. **読みの文字数ヒントが最も効果的**: 最初から表示すると1件絞り込み率が57.2%から85.5%に改善。最悪ケース9件が3件に解消される
2. **カテゴリヒントは主観性リスクあり**: knowledgeカテゴリで約25%が境界曖昧。プレイヤーが「答えを知っているのにカテゴリが直感と一致しない」ことで逆効果になるリスクがある
3. **originは客観的だが効果は限定的**: 難易度3はほぼ中国由来で区別効果が低い。全体の残1件率は72.2%で読み文字数（85.5%）に劣る
4. **structureは効果が低い**: 「組合せ」が60%を占め、絞り込みにほぼ寄与しない
5. **「々」バグは修正済み**: engine.tsのisValidYojiInputにU+3005が既に追加されている

## 設計方針

「プレイヤーにとっての価値最大化」と「答えを知っていれば確実に辿り着ける保証」の2つを軸とする。

### ヒントの段階設計方針

分析に基づき以下の原則を採用する:
- **客観的で逆効果リスクのないヒントを優先する**: 読みの文字数 > 読みの1文字目 > origin > カテゴリの順
- **カテゴリヒントは維持するが、最終段階に留める**: 主観性リスクはあるが、57.2%の問題では正確に絞り込める有用な情報でもある。廃止せず最終手段として残す
- **structureヒントは不採用**: 60%が「組合せ」で絞り込み効果が極めて低く、表示しても混乱するだけ

## Task 4: ヒントシステム再設計

### 新しいヒント段階設計（4段階）

| タイミング | ヒント内容 | 根拠 |
|-----------|-----------|------|
| 常時表示 | 難易度（★/★★/★★★）+ 読みの文字数（例: 「7文字」） | 読み文字数は最も客観的かつ効果的（残1件率85.5%）。最初から表示してもゲーム性を損なわない（文字数だけでは答えは分からない） |
| 3回目推測後 | 読みの1文字目（例: 「い...」） | 既存ヒントを維持。読み文字数との組み合わせでさらに絞り込み可能 |
| 4回目推測後 | 出典（中国/日本/不明） | originは客観的。「不明」の場合も「不明」と表示する（情報として有効）。カテゴリより逆効果リスクが低い |
| 5回目推測後 | カテゴリ（例: 「人生・生き方」） | 主観性リスクを認識しつつ、最終手段として維持。既存の10カテゴリ分類をそのまま使う |

### 旧計画（19cda286b40 Task 4）からの変更点

1. **読みの文字数を常時表示に追加**（新規）: 分析で最も効果的と判明したため
2. **structureヒントを不採用**（旧: 2回目後に表示）: 60%が「組合せ」で絞り込み効果がほぼゼロ
3. **段階数を5段階から4段階に削減**: structureを除外したため。ヒントが多すぎると情報過多になるリスクもある
4. **originの「不明」もそのまま表示**: 旧計画では「不明はスキップ」だったが、「不明」自体が情報になる（出典が不明確＝日本で独自に発展した可能性など）

### HintBarコンポーネントの変更内容

- propsにoriginを追加（YojiOrigin型）
- 常時表示エリアに「読み N文字」を追加
- 4回目推測後にorigin表示を追加
- structureは使わない
- 予告テキスト「(3回目で読みヒント)」は維持。追加で段階ごとの予告を簡潔に表示

### GameContainerの変更

- HintBarへのprops渡しにoriginを追加: `origin={gameState.targetYoji.origin}`

## Task 5: ResultModal拡充

### 表示する情報

ゲーム終了後（正解・不正解問わず）に以下を表示する:

1. **四字熟語**（既存）
2. **読み**（既存）
3. **意味**（既存）
4. **出典**: origin値を表示（「中国の古典に由来」「日本で生まれた表現」「出典不明」のような自然な文言）
5. **カテゴリ**: categoryLabelsの日本語表記を表示

「不明」値の場合も表示する（省略すると情報の欠落になるため）。

### structureについて

結果画面でもstructureは表示しない。60%が「組合せ」で情報としての価値が低く、プレイヤーに意味のある学びを提供しない。

### 旧計画からの変更点

- structureの表示を取りやめ（旧計画では表示予定だった）
- origin表示の文言を自然な日本語に変換して表示
