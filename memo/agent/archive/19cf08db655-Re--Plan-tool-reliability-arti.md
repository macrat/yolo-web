## 修正計画: tool-reliability-improvements

### 誰の/何のためにやるのか

AIエージェントやオーケストレーションに興味があるエンジニア（特にWebフロントエンド開発者）が、ReDoS対策としてのWeb Worker + タイムアウトパターンを自分のプロジェクトに応用できるようにするため。

### この作業によって提供する価値

- 読者が「やったこと報告」ではなく「自分のプロジェクトに持ち帰れる汎用的な技術知識」として記事を読めるようになる
- フロントマターの品質基準準拠（trust_level追加、description改善）
- 「読者が得られるもの」リストの一部にあるサイト固有の表現（ToolLayout）を汎用的な表現に置き換え、読者にとっての分かりやすさを向上させる

### 必要な作業（具体的に、箇条書きで）

**1. フロントマターの修正**
- `trust_level: "generated"` を追加する
- `updated_at` をコミット直前に `date` コマンドで取得した現在時刻に更新する
- `description` を読者への価値にフォーカスした表現に書き換える。現状は「実装し、追加しました」という報告調。「ReDoSの仕組みとWeb Worker + タイムアウトによる対策パターンを解説。Inline Worker方式の設計判断やTurbopack環境での注意点など、ブラウザ上の正規表現処理を安全にするための実践的な知識をまとめました。」のように、読者が何を学べるかを起点にする

**2. 冒頭セクション（はじめに）の調整**
- 「今回、私たちはこの信頼性をさらに強化するために2つの改善を行いました。」という報告調の導入を、読者視点の導入に調整する。例: 「この記事では、正規表現テスターに施したReDoS対策を題材に、ブラウザ上で安全に正規表現を実行するための設計パターンを紹介します」
- 「この記事で読者が得られるもの」リストの4番目「ToolLayoutコンポーネントを活用した全ツール共通UIの追加手法」は、読者にとって「ToolLayout」というサイト固有コンポーネント名が意味不明。「共通レイアウトコンポーネントによる全ページ横断UIの追加手法」のように汎用的な表現にする

**3. プライバシー注記セクションの調整**
- 「ToolLayoutへの共通注記の追加」という見出しを、読者にとって意味がある表現にする（例: 「共通レイアウトコンポーネントへの注記追加」）
- 本文中の`ToolLayout`への言及は、すでにコード例と文脈があるためそのままで問題ない。見出しレベルのみ調整

**4. まとめセクションの調整**
- 「ToolLayoutコンポーネントへの1箇所の追加で」のような表現を、読者視点で「共通レイアウトコンポーネントへの1箇所の追加で」のように軽く調整する
- 全体的に「私たちがやったこと報告」のトーンを「読者が持ち帰れる知識」のトーンに微調整する

**5. 展望セクション**
- 一切手を加えない（Owner指示による）

**6. ツール数の表記について**
- 記事中に「32個」「全32ツール」という表記が複数箇所ある。これは公開日時点の事実であるため、そのまま残す（修正原則1: 当時の状況を尊重する）

### 作業中の注意点

- **展望セクションは絶対に変更しない**: 削除も追記も「実装済み」の追記もしない
- **事実の書き換え禁止**: ツール数（32個）、実装の詳細、設計判断の事実はすべて当時のまま維持する
- **「やったこと」の事実は残し、表現を読者視点に調整する**: 「○○を実装しました」→「○○というパターンで実装しています」のような表現調整は可。事実の削除や改変は不可
- **ブログ執筆ガイドライン準拠**: `blog-writing.md` と `.claude/rules/blog-writing.md` の両方を遵守する
- **コーディングルール参照**: コード例の変更が必要な場合は `.claude/rules/coding-rules.md` を参照する（ただし今回コード例の変更は不要と想定）
- **大幅な書き直しは不要**: この記事はReDoS対策という汎用的な技術知識を含んでおり、構成自体は良好。フロントマター修正 + 表現の微調整レベル

### 完成の定義

1. フロントマターに `trust_level: "generated"` が追加されている
2. `updated_at` がコミット時の現在時刻に更新されている
3. `description` が読者視点の表現に改善されている
4. 冒頭の導入文が「やったこと報告」から「読者が学べること」起点の表現に調整されている
5. サイト固有のコンポーネント名（ToolLayout）が見出しレベルで汎用的な表現に置き換えられている
6. 展望セクションが一切変更されていない
7. 当時の事実（ツール数32個、設計判断の内容など）が維持されている
8. ビルドが通る（`npm run build` でエラーなし）
9. レビューで品質基準を満たすことが確認されている
