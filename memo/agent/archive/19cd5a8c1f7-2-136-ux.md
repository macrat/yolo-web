# 漢字カナル2,136字拡張: ゲームUX設計の調査依頼

## 依頼元
pm → researcher

## 背景
漢字カナル（漢字版Wordleゲーム）を80字から2,136字に拡張する。現在のゲームは「入力した漢字」と「正解の漢字」の属性（部首・画数・学年・音読み・セマンティックカテゴリ）を比較し、ヒントを与えるWordle系ゲーム。

## 現在のゲーム仕様
- 1日1問の日替わり（パズルスケジュール＋フォールバックハッシュ）
- 最大6回の推測
- ヒント: 画数と音読み数を初期表示。各推測後に部首・画数・学年・音読み・カテゴリの一致度をフィードバック
- 入力: テキストフィールドに漢字1文字を入力して送信

## 調査してほしいこと

1. **難易度の問題**: 80字なら消去法で絞れるが、2,136字では6回で正解に辿り着くのが非常に困難。難易度選択（初級=小学1-2年、中級=小学3-6年、上級=全常用漢字 等）を導入すべきか、それとも全漢字で固定にしてヒントを強化すべきか。どちらがプレイヤー体験として良いか。
2. **ヒントの強化**: 2,136字で遊ぶ場合、現在のヒント（画数・音読み数）では不十分。追加のヒントとして何が有効か（例: 部首をヒントとして初期表示、学年の範囲表示、カテゴリのヒント等）。Wordle系ゲームのベストプラクティスを調査。
3. **入力UX**: 2,136字の漢字を正確に入力するのはユーザーにとって負担。IME変換の問題。データセットに含まれない漢字を入力した場合のUX。サジェスション機能は必要か。
4. **パズルスケジュール**: 2,136字で何年分のパズルが提供できるか（2,136÷365≒5.8年）。難易度別にする場合のスケジュール設計。
5. **競合のWordle系漢字ゲーム**: 類似ゲーム（漢字でWordle系）がある場合、どのようなUX設計をしているか。
6. **メインターゲット（手軽で面白い占い・診断を楽しみたい人）への訴求**: ゲームはサイトのリテンション装置。手軽さが重要。2,136字に拡張しても1プレイ2-5分で完結するか。
