## C. NIST SP 800-63-4 ファクトチェック

### 主張1: 「NISTの最低要件は8文字」

**不正確です。**

NIST SP 800-63B-4（最終版、2025年7月31日公開）の正確な要件は以下の通りです。

- 単一要素認証（パスワードのみ）: **SHALL（必須）: 最低15文字**
- 多要素認証の一部として使うパスワード: SHALL（必須）: 最低8文字

出典: [NIST SP 800-63B-4 本文](https://pages.nist.gov/800-63-4/sp800-63b.html)、[csrc.nist.gov 最終版確認](https://csrc.nist.gov/pubs/sp/800/63/b/4/final)

記事の「NISTの最低要件は8文字」という記述は、最終版における単一要素認証のケースでは**誤り**です。記事が参照しているSP 800-63-4の最終版では、単一要素認証に対して15文字が必須要件（SHALL）に引き上げられました。記事が推奨している「15文字以上」という実践的なアドバイスは正しいのですが、「NISTの最低要件は8文字」という事実記述が現在の最終版と矛盾しています。

### 主張2: パスワードマネージャーへの「SHOULD」言及

**不正確です。**

最終版では「SHALL（必須）」に格上げされています。

- 正確な記述: 「Verifiers SHALL allow the use of password managers and autofill functionality」（SHALL = 義務的要件）
- 記事の記述: 「NISTが『すべきだ（SHOULD）』という強い推奨でパスワードマネージャーに言及」

SHOULDは「推奨」、SHALLは「義務」であり、格が異なります。最終版では義務的要件に強化されています。

### 主張3: パスキー/FIDO2への言及

記事にはパスキーやFIDO2への言及がありません。NIST SP 800-63B-4ではWebAuthnを含むフィッシング耐性認証が広く記述されており、2026年時点の「実践的セキュリティガイド」としてパスキーへの言及がないことは、記事の現代性の観点で欠落があります。

## D. 判断

### 書き直しで独自の価値を出せるか

書き直しによって独自価値を出せる可能性は**低い**と判断します。

理由は以下の3点です。

**1. ツール削除により根拠が崩れる**: 記事の独自性は自サイトのパスワード生成ツール・ハッシュ生成ツールとの連携に依存していましたが、これらのツールは削除対象となっています。ツールなしに汎用パスワードガイドとして立つためには、より強固な独自視点が必要ですが、セキュリティ分野ではAIが生成した実験サイトがその信頼性優位を確立することは困難です。

**2. 事実誤認が構造的問題を示している**: 「NISTの最低要件は8文字」「SHOULDでパスワードマネージャーに言及」の2点は最終版の公式文書と矛盾しています。セキュリティ分野での事実誤認は constitution.md Rule 1（法令遵守）および Rule 2（人を傷つけない）の観点からリスクがあります。これをAI生成コンテンツが常に正確に維持し続けることは構造的に困難です。

**3. 評価ルーブリックで独自性スコア2**: 大手セキュリティベンダーとGoogle公式が上位を占める分野で、RISKフラグが立つ品質達成可能性2との組み合わせは、書き直しの投資対効果が低いことを示します。

### 削除が適当か

**削除が適当と判断します。**

`site-concept.md` では「削除するコンテンツ」の判断基準として「大手サイト（略）に品質・データ量で大差があり、独自性がない」を挙げています。この基準に照らすと、本記事はそれに該当します。

削除が適当な根拠を以下にまとめます。

| 根拠 | 詳細 |
|---|---|
| ツール削除で独自性消失 | 内部ツールへの誘導が独自性の核だったが、ツールが削除対象 |
| 競合に対する優位性なし | トレンドマイクロ・カスペルスキー・Google公式が上位を占める分野 |
| 事実誤認がある | NISTの最低要件（8文字 vs 15文字）、パスワードマネージャーの推奨レベル（SHOULD vs SHALL）が最終版と矛盾 |
| ターゲット不一致 | メイン・サブターゲット双方の search_intents にない |
| サイトコンセプトとの距離 | 「笑えて、シェアしたくなる占い・診断の遊園地」のコンセプトと無関係な汎用ガイド |
| 修正コスト対効果 | 事実誤認修正 + パスキー追加 + ツールリンク差し替えを行っても、独自性スコア2の構造は変わらない |
