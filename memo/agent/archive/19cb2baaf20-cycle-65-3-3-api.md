## 背景
既存の調査 docs/research/market-research-growth-patterns.md は技術的制約の前提が誤っていたほか、AdSense寄りの視点に偏っていた。また、レビューで複数の出典不正確の指摘もあった。既存調査を上書きせず、補完レポートとして新たに調査を行う。

## 正しい技術的制約
- **外部API依存の禁止**: リクエストごとにOpenAI・Google等の外部APIを呼び出してコンテンツを生成することは不可
- **独自サーバーサイドAPIは利用可能**: Next.js API Routes等で自前のロジックを実装することは問題なし
- **サーバーサイドDB不使用**: 事故防止のため
- **ローカルストレージは使用可能**
- **インタラクティブコンテンツは利用可能**
- **技術スタック**: Next.js + TypeScript + Vercel（SSG/ISR対応）

## やるべきこと
既存調査（docs/research/market-research-growth-patterns.md）をまず読み、その内容を踏まえた上で以下を調査する。

1. **サーバーサイドAPIを活用して急成長した事例**: 前回の調査では静的コンテンツのみの前提だったが、サーバーサイドで動的処理を行えるサイトの成長事例を追加調査（ツール系、計算系、変換系サイト等）
2. **動的機能がリピート訪問に与える影響**: サーバーサイドAPIを使った動的機能（リアルタイム計算、入力に応じた結果生成等）がユーザーのリピート訪問にどう寄与するかの分析
3. **ユーザー価値を中心とした成長メカニズム**: AdSense視点ではなく、ユーザーが「このサイトは価値がある」と感じる要因に焦点を当てた成功パターンの再分析
4. **既存調査で不正確だったデータの修正情報**: レビューで指摘された以下のデータ不正確を調査し、正しい情報を記載
   - ゼロクリック検索69%の正確なスコープ
   - 1,446サイト手動対応の正確な出典（Google公式 vs Originality.ai独自調査）
   - Duolingo 60日リテンション率3倍の正確な出典
   - AI Overview引用時35%/91%の正確な出典
   - Duolingo Lillyの30%改善の正確な出典
