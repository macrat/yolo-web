## B-106 実施計画: ワークフロー改善連載の「完結」表現修正

### 背景と方針

ownerの指示により、プロジェクトが続く限り連載も続くため、「全5回」「既に完結」などの表現を修正する。連載は今後も記事が追加される可能性があるため、回数を固定しない表現に変更する。

### 修正対象ファイル（5ファイル）

以下の全5ファイルに共通の「ナビゲーションブロック」が存在し、そこに「全5回」の表現がある。

1. `/mnt/data/yolo-web/src/content/blog/2026-02-13-how-we-built-this-site.md`（第1回）
2. `/mnt/data/yolo-web/src/content/blog/2026-02-18-spawner-experiment.md`（第2回）
3. `/mnt/data/yolo-web/src/content/blog/2026-02-18-workflow-evolution-direct-agent-collaboration.md`（第3回）
4. `/mnt/data/yolo-web/src/content/blog/2026-02-19-workflow-simplification-stopping-rule-violations.md`（第4回）
5. `/mnt/data/yolo-web/src/content/blog/2026-02-23-workflow-skill-based-autonomous-operation.md`（第5回）

### 修正箇所と修正案

#### A. 全5ファイル共通: ナビゲーションブロック（各記事の冒頭）

**現状（各ファイルの19行目付近）:**
```
> **ワークフロー連載**（全5回）
```

**修正後:**
```
> **ワークフロー連載**
```

理由: 「（全5回）」を削除し、回数を固定しない。今後記事が増えても修正不要になる。

#### B. 第1回記事のみ: セクション見出し（217行目）

**現状:**
```
## この連載で語られること -- 全5回の見取り図
```

**修正後:**
```
## この連載で語られること -- これまでの見取り図
```

理由: 「全5回の見取り図」は回数を固定している。「これまでの」にすることで、今後記事が追加されても自然に読める。

#### C. 第1回記事のみ: 見取り図セクション本文（219行目）

**現状:**
```
この記事はワークフロー連載の第1回です。連載は全5回が完結しており、プロジェクトの開始から現在の安定運用に至るまでの軌跡を記録しています。
```

**修正後:**
```
この記事はワークフロー連載の第1回です。プロジェクトの開始から現在の運用に至るまでの軌跡を記録しており、今後も連載は続きます。
```

理由: 「全5回が完結しており」を削除し、連載継続を明示する。

#### D. 第1回記事のみ: 目次項目（33行目）

**現状:**
```
- 連載全5回で語られることの見取り図
```

**修正後:**
```
- 連載のこれまでの見取り図
```

理由: セクション見出しの変更（修正B）に合わせる。

#### E. 第5回記事のみ: おわりにセクション（278行目）

**現状:**
```
この記事は、[ワークフロー連載](/blog/how-we-built-this-site)の第5回にあたります。連載を通じて私たちが辿ってきた道のりを振り返ると、驚くほどの変遷がありました。
```

**修正後:**
```
この記事は、[ワークフロー連載](/blog/how-we-built-this-site)の第5回にあたります。ここまでの連載を通じて私たちが辿ってきた道のりを振り返ると、驚くほどの変遷がありました。
```

理由: 「連載を通じて」だと全体が完結した印象になるため、「ここまでの連載を通じて」に変更し、継続中であることを暗示する。

### 修正不要と判断した箇所

- 各記事内の「第N回」の表記 -- これは単に記事番号を示しているだけで、完結を意味しない
- 第5回記事末尾（286行目）の「失敗も成功もすべて記録し、改善し続けることが〜」 -- 既に継続的ニュアンスがある
- 第5回記事末尾（288行目）の「今後も私たちはサイクルを回し続け〜」 -- 既に継続を示している

### 実施手順

1. builderが上記A-Eの修正を全5ファイルに適用する
2. updated_atを修正日時に更新する
3. npm run buildでビルドが通ることを確認する
4. reviewerが修正内容を確認する（全箇所が漏れなく修正されていること、文脈が自然であること）

### 完了条件

- 全5ファイルから「全5回」の表現が除去されていること
- 「完結」「既に完結」などの完結を示唆する表現が除去されていること
- 修正後の文章が自然で、連載継続中であることと矛盾しないこと
- ビルドが正常に通ること
- reviewerによる確認が完了していること
