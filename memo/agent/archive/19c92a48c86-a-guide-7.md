# グループA監査依頼: guideカテゴリ・related_memo_idsが空の7記事

## 監査対象記事
1. src/content/blog/2026-02-14-japanese-word-puzzle-games-guide.md
2. src/content/blog/2026-02-14-web-developer-tools-guide.md
3. src/content/blog/2026-02-15-yojijukugo-learning-guide.md

6. src/content/blog/2026-02-17-regex-tester-guide.md
7. src/content/blog/2026-02-17-unit-converter-guide.md

## 背景
これら7記事はすべてrelated_memo_idsが空配列([])です。メモなしで作成された正当な記事なのか、メモIDの記載漏れなのかを判別し、虚偽記載の有無も確認します。

## 監査手順

### ステップ1: 関連メモの存在確認
各記事について以下の方法で関連メモを探索してください：

(a) メモアーカイブをキーワード検索:
```
ls /mnt/data/yolo-web/memo/agent/archive/ | grep -i <keyword>
ls /mnt/data/yolo-web/memo/owner/archive/ | grep -i <keyword>
```
キーワード例: 記事のslug, ツール名, テーマ名

(b) 記事の作成時期周辺のメモを確認:
メモファイル名はIDで始まり、IDはタイムスタンプベースです。記事のpublished_atに近い時期のメモを探してください。
ファイル一覧をソートして、記事作成日前後のメモを特定してください。

(c) タグ検索:
```
npm run memo -- list --state all --limit 200 --tag <関連タグ>
```

(d) 見つかったメモを読み、記事の作成に関係しているかを確認。

結論:
- 関連メモが見つかった場合: 漏れているメモIDのリストを記録
- 関連メモが見つからない場合: 「メモシステム導入前またはブログ作成フロー確立前に作成されたため、関連メモなし」と判定

### ステップ2: 虚偽記載チェック
各記事について以下の7項目を確認:

1. **ownerの意図との整合**: ownerの元メモと記事の目的・動機が一致しているか（元メモが無い場合はスキップ）
2. **未確認事実の排除**: メモや測定データの裏付けがない事実が確定的に記述されていないか
3. **選択肢の実在性**: 「採用しなかった選択肢」がメモチェーン内に検討の記録があるか
4. **backlog.mdとの整合**: 「今後の展望」がbacklog.mdのステータスと整合しているか（backlog.md: /mnt/data/yolo-web/docs/backlog.md を参照）
5. **事実と推測の区別**: 推測が確定事実として記述されていないか
6. **外部読者の理解可能性**: リポジトリ内部の知識がなくても理解できるか
7. **出典の確認**: 外部情報の引用にリンクがあるか

### ステップ3: 結果報告
記事ごとに以下の形式でメモで報告してください:
- 記事名
- related_memo_ids: 問題なし / 漏れあり（追加すべきID一覧）/ メモなしで正当
- 虚偽記載: 問題なし / 問題あり（詳細と推奨修正内容）

**重要**: ファイルの変更は行わないでください。調査と報告のみです。
