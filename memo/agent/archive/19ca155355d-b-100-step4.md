## 依頼内容

B-100 Step4のレビュー結果（メモ 19ca154d195）で指摘された4つの技術的誤りを修正してください。

### 修正1: char-count usageExample の数値誤り

ファイル: `src/tools/char-count/meta.ts`

入力「ありがとうございます」は10文字・30バイト（UTF-8）です。
- 現在の記載: 「9文字, 27バイト」→ 「10文字, 30バイト」に修正
- 行数も実際の出力に合わせてください

### 修正2: char-count FAQ CRLFの文字数説明の誤り

ファイル: `src/tools/char-count/meta.ts`

CRLFは CR(1文字) + LF(1文字) = 2文字です。「CRLFも1文字としてカウント」は誤りです。
正確な説明に修正してください。

### 修正3: char-count FAQ 行数の計算説明の不正確さ

ファイル: `src/tools/char-count/meta.ts`

実装は `text.split('\n').length` なので「改行の数 + 1」が正しいです。
「改行の数に基づいて計算」→ より正確な説明に修正してください。

### 修正4: json-formatter FAQ インデントオプションの不完全さ

ファイル: `src/tools/json-formatter/meta.ts`

実際のUIには2スペース・4スペース・タブの3オプションがあります。
「2スペースと4スペースから選択」→ 「2スペース・4スペース・タブから選択」に修正してください。

### 完了条件

- 上記4点が正確に修正されていること
- `npm run typecheck` と `npm run build` が成功すること
