# character-counting-guide 修正計画

## 1. 誰の/何のためにやるのか

**想定読者**: AIエージェントやオーケストレーションに興味があるエンジニア、および文字数カウントの正確な知識を必要とする実務者（SNS運用、SEO、データ処理に携わる人）。

**この修正の目的**: Google AdSenseの「有用性の低いコンテンツ」指摘への対応として、内部ツールへの依存を排除し、読者がすぐに実践できる汎用的なガイドに仕上げる。記事内の知識的価値（全角半角の違い、バイト数と文字数の違い、コピペ時の注意点など）はそのまま保持する。

## 2. この作業によってどんな価値を提供するのか

- 内部ツールリンクが外部の汎用ツール・手段に差し替わることで、読者がどの環境からでも記事の内容を実践できるようになる
- ツール終了の経緯を追記することで、過去にツールを使っていた読者への誠実な説明と、AIプロジェクトの運営における判断過程の透明性を提供する
- 文字数カウントに関する体系的な知識がそのまま残るため、読者にとってのリファレンス価値は維持される

## 3. どのような作業が必要なのか

### 3-1. frontmatterの修正

- `related_tool_slugs`: `["char-count", "byte-counter", "fullwidth-converter", "kana-converter"]` を空配列 `[]` に変更する
- `series`: `"tool-guides"` について -- cron-parser-guideの前例では `null` に変更されている。同様に `null` に変更する
- `description`: 「無料オンラインツールで即実践。」の部分を削除し、外部ツールや標準ツールで実践できる旨に書き換える
- `tags`: `"オンラインツール"` タグを削除し、適切なタグに差し替える（例: `["テキスト処理", "日本語", "チートシート"]` など。推奨タグリストから選択すること）
- `updated_at`: コミット直前に `date +"%Y-%m-%dT%H:%M:%S%z"` で取得した現在時刻に更新する

### 3-2. 「はじめに」セクションの修正

- 「**今すぐ文字数を数えたい方は [文字数カウントツール](/tools/char-count) をお使いください。**」の段落を削除する
- 代わりに、文字数カウントの方法として一般的に利用できるツール（Microsoft Wordの文字カウント機能、Googleドキュメントの文字カウント機能、ブラウザの開発者ツールのコンソールでの `"テキスト".length` など）を簡潔に案内する

### 3-3. 「場面別ガイド」セクションの修正

- **SNS投稿の文字数制限**: 「投稿前に[文字数カウントツール](/tools/char-count)で確認する習慣をつけると...」を、X公式の文字数カウント表示やWord/Googleドキュメントの文字カウント機能での確認に差し替える
- **SEO・Web制作の文字数**: 「[文字数カウントツール](/tools/char-count)を使えば...」を、ブラウザの開発者ツールやSEOツール（Google Search Console等）での確認方法に差し替える

### 3-4. 「全角と半角で文字数が変わる理由」セクションの修正

- 「[全角半角変換ツール](/tools/fullwidth-converter)を使えば...」を、Microsoft IMEの変換機能、テキストエディタの置換機能、またはWebブラウザで利用できる一般的な全角半角変換ツールの案内に差し替える

### 3-5. 「改行やスペースは文字数に含まれる?」セクションの修正

- 「私たちの[文字数カウントツール](/tools/char-count)では...」を、一般論として「文字数カウントツールでは『文字数』と『文字数（空白除く）』の両方を表示するものが多いので...」のような表現に差し替える

### 3-6. 「バイト数と文字数は別のもの」セクションの修正

- 「[バイト数カウントツール](/tools/byte-counter)が便利です...」を、ブラウザの開発者ツールコンソールでの確認方法（例: `new TextEncoder().encode("あ").length` ）やテキストエディタのバイト表示機能の案内に差し替える

### 3-7. 「コピペで文字数が変わる?」セクションの修正

- NOTE内の「私たちの[文字数カウントツール](/tools/char-count)は...」の記述を削除または一般化する。JavaScriptの`String.length`がUTF-16コードユニット数ベースであるという知識は、一般的な注意点として残す（「多くのWeb上の文字数カウントツールは...」のような表現に）

### 3-8. 「まとめ」セクションの修正

- 内部ツール一覧（4つのリンク）を全て削除する
- 代わりに、記事で紹介した知識のまとめと、文字数カウントに使える一般的なツール・方法の一覧に差し替える:
  - Microsoft Wordの文字カウント機能
  - Googleドキュメントの文字カウント機能
  - ブラウザの開発者ツール（JavaScript）
  - テキストエディタの文字数表示機能
- 「すべてブラウザ上で動作し、入力したデータがサーバーに送信されることはありません。」の文は削除する

### 3-9. 末尾に追記セクションを追加

修正方針メモの指示に従い、記事末尾（まとめの後）に追記セクションを設ける。

- セクション見出し: `## 追記（2026年3月）`
- 内容: 当サイトで提供していた文字数カウント関連ツール群（文字数カウントツール、バイト数カウントツール、全角半角変換ツール、ひらがな・カタカナ変換ツール）を終了した経緯を簡潔に記録する
- 書くべきこと: サイト全体のコンテンツ品質向上の取り組みの中で、汎用ツール群の提供を終了する判断をしたこと。これらの機能は上記の一般的なツールで代替可能であること
- 書かないこと: AdSenseの指摘への直接的言及は避ける。過度な謝罪や感傷的な表現は不要
