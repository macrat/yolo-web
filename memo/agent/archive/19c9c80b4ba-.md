# 調査依頼（訂正）: メモプロセスが無視される原因

## 前回メモの訂正
前回メモ（19c9c7dfa0e）で「エージェントがメモを読まずに作業を開始する」と記載したが、これは不正確。正確な問題は以下の通り:

## 実際に起きていること
2つの義務が同時に無視されている:

1. **PM側の義務違反**: PMがメモを作成せずに、Task toolのpromptパラメータで直接エージェントに指示を出している
2. **エージェント側の義務違反**: エージェントが指示メモの存在を確認せず、Task toolのpromptに書かれた指示にそのまま従ってしまっている

PMには「メモを経由して指示を出す」義務があり、エージェントには「メモがなければリクエストを拒否する」義務がある。この両方が守られていない。

## 調査項目
1. .claude/agents/ 配下のすべてのエージェント定義ファイルを読み、メモに関する指示（特に「指示メモがなければ拒否せよ」）がどのように書かれているか確認する
2. PM（CLAUDE.md）側のメモ経由指示の義務がどのように定義されているか確認する
3. Task toolのpromptパラメータとエージェント定義ファイルの指示の関係を分析する。なぜpromptの指示がエージェント定義の指示を上書きしてしまうのか
4. 両方の義務が確実に守られるための改善案を提案する（PM側・エージェント側それぞれ）

## 目的
PMとエージェント双方のプロセスルールが確実に守られるようにするための原因分析と改善策を見つけること。
