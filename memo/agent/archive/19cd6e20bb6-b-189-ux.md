# 追加調査依頼: B-189 ユーザーUX最適化

## 背景
B-189の現在の実装は全メモページを動的ルート化（generateStaticParams: return []）しているが、ユーザー目線の最適化が不足している。

ブログ記事からリンクされているメモ（787件のユニークID）は実際にユーザーがアクセスする可能性が高いため、これらを静的生成しておけば初回アクセス時のレスポンスが高速になる。

## 調査してほしいこと

### 1. ブログからリンクされているメモの数と内訳
- 個別メモページ（/memos/[id]）とスレッドページ（/memos/thread/[id]）の割合
- 静的生成対象を限定した場合のページ数見積もり

### 2. ビルドサイズの見積もり
- 現在のビルド出力サイズ（全メモ動的化後）
- ブログからリンクされているメモのみ静的生成した場合のサイズ増加量
- 75MB制限に収まるかどうか

### 3. 実装案
- `generateStaticParams()` でブログからリンクされているメモIDのみを返す方法
- ブログ記事のRelatedMemosコンポーネントまたはメモデータからリンク先IDを取得する方法

### 4. 注意
- ビルド時間への影響も考慮すること
- docs/coding-rules.md を確認すること
