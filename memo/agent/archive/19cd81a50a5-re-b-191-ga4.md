# B-191: GA4推奨イベント調査結果

## 1. GA4推奨イベント仕様と当サイトへのマッピング

### 対象イベントの公式仕様

| イベント名 | トリガー | 公式パラメータ |
|---|---|---|
| `level_start` | ゲームのレベル開始時 | `level_name` (string) |
| `level_end` | ゲームのレベル終了時 | `level_name` (string), `success` (boolean) |
| `unlock_achievement` | 実績アンロック時 | `achievement_id` (string) |
| `search` | サイト内検索実行時 | `search_term` (string) |
| `share` | 外部SNSへのシェア時 | `method` (string), `content_type` (string), `item_id` (string) |

注: `content_type` と `content_id` はGA4のスタンダードディメンションとして組み込み対応されている（`customEvent:` プレフィックス不要）。

### コンテンツタイプ別マッピング

#### ゲーム（4種: irodori, kanji-kanaru, nakamawake, yoji-kimeru）

```
ゲーム開始:
  gtag('event', 'level_start', {
    level_name: 'irodori',       // ゲームID
    content_type: 'game'
  })

ゲーム完了:
  gtag('event', 'level_end', {
    level_name: 'irodori',
    success: true,               // クリア=true, 失敗=false
    content_type: 'game'
  })
```

#### クイズ/診断（13種: quiz-{slug}）

```
クイズ開始:
  gtag('event', 'level_start', {
    level_name: 'quiz-kanji-level',
    content_type: 'quiz'         // personality quiz → 'diagnosis'
  })

クイズ完了:
  gtag('event', 'level_end', {
    level_name: 'quiz-kanji-level',
    success: true,               // 知識クイズは常にtrue（完走した）、診断はtrue固定
    content_type: 'quiz'
  })
```

#### 実績システム

```
バッジアンロック時:
  gtag('event', 'unlock_achievement', {
    achievement_id: 'streak-7'   // BADGE_DEFINITIONS の id
  })
```

#### サイト内検索（SearchModal）

```
検索実行時（デバウンス後に結果が出た時点）:
  gtag('event', 'search', {
    search_term: 'query_string'
  })
```

#### シェア（ShareButtons）

```
X（Twitter）シェア:
  gtag('event', 'share', {
    method: 'twitter',
    content_type: 'quiz',        // または 'game'
    item_id: 'quiz-kanji-level'  // コンテンツID
  })

LINEシェア:
  gtag('event', 'share', {
    method: 'line',
    content_type: 'quiz',
    item_id: 'quiz-kanji-level'
  })

Web Share API:
  gtag('event', 'share', {
    method: 'web_share',
    content_type: 'quiz',
    item_id: 'quiz-kanji-level'
  })

クリップボードコピー:
  gtag('event', 'share', {
    method: 'clipboard',
    content_type: 'quiz',
    item_id: 'quiz-kanji-level'
  })
```

#### ツール（30種以上）

ツールはゲームとは性質が異なるため `level_start`/`level_end` は不適切。代わりに以下を検討：
- ページビューのみ（現状維持、ツールはパラメータの少ないシンプルな利用）
- または独自カスタムイベント `tool_use` を追加（GA4非推奨イベントになるがツール固有分析には有効）

#### 占い（fortune-daily）

```
占い閲覧（recordPlayの呼び出しタイミングと合わせる）:
  gtag('event', 'level_start', {
    level_name: 'fortune-daily',
    content_type: 'fortune'
  })
```

## 2. Next.js App RouterでのGA4 gtag実装ベストプラクティス

### 既存実装の評価

`src/components/common/GoogleAnalytics.tsx` は `strategy="afterInteractive"` を使用しており適切。ただし現状は `gtag('config', ...)` のみでイベント送信機能がない。

### クライアントコンポーネントでの呼び出し方法

`window.gtag` は `"use client"` コンポーネントから直接呼び出せる。注意点：

1. `window` オブジェクトはSSR環境で存在しないため、`typeof window !== 'undefined'` ガード必須
2. ただしGA4スクリプトが `afterInteractive` でロードされるため、実質的にクライアントのみで動作する
3. TypeScript型安全のために `@types/gtag.js` をdevDependenciesに追加すること

### 推奨アーキテクチャ

新しいユーティリティモジュール `src/lib/analytics.ts` を作成し、全イベント送信をここに集約する。

## 3. カスタムパラメータの設計指針

### 推奨パラメータセット

```typescript
// 全イベントに共通で付与できる拡張パラメータ
interface BaseEventParams {
  content_type: 'game' | 'quiz' | 'diagnosis' | 'fortune' | 'tool' | 'dictionary' | 'blog';
  content_id: string;  // 例: 'irodori', 'quiz-kanji-level', 'fortune-daily'
}
```

**注意**: `content_type` と `content_id` はGA4のスタンダードディメンション（`contentType`、`contentId`）に対応しており、カスタムディメンション登録なしでデータAPIから直接参照できる。

### GA4 Admin での設定

カスタムパラメータ（上記以外のもの）はGA4 Admin → Property → Custom definitions で「Event-scoped custom dimension」として登録が必要。登録しないとレポートUIには表示されないが、Data APIからは `customEvent:param_name` として参照できる。

### 既存の `ContentType` との整合

`src/lib/search/types.ts` に定義された `ContentType` 型（tool, game, cheatsheet, kanji, yoji, color, blog, quiz）がある。GA4の `content_type` パラメータとは完全一致しないため、analytics.ts 内でマッピング変換するか、別途 GA4専用の型を定義することを推奨する。

## 4. B-114（将来のランキング機能）のためのイベント設計

### Data API での参照方法

GA4 Data APIの `runReport` で以下のディメンション・メトリクスが使える：

```
dimensions:
  - eventName        # "level_end", "unlock_achievement" 等
  - contentId        # GA4スタンダードディメンション（content_id パラメータ）
  - contentType      # GA4スタンダードディメンション（content_type パラメータ）

metrics:
  - eventCount       # イベント発生回数
  - activeUsers      # ユニークユーザー数
```

カスタムディメンションは `customEvent:achievement_id` のように参照できる。

### ランキング用に取得すべきイベントとパラメータ

| 目的 | イベント | 必須パラメータ |
|---|---|---|
| コンテンツ人気度（開始数） | `level_start` | `content_id`, `content_type` |
| コンテンツ完走率（完了/開始比） | `level_end` | `content_id`, `content_type`, `success` |
| シェア率 | `share` | `item_id`, `content_type`, `method` |
| 実績アンロック頻度 | `unlock_achievement` | `achievement_id` |
| 検索キーワード | `search` | `search_term` |

ランキング用には `level_start` + `level_end` の `content_id` 別の `eventCount` を集計するのが最も直接的。`success=true` の `level_end` 件数でゲームクリア率の計算も可能。

## 5. イベント送信ユーティリティ関数の設計パターン

### 推奨設計: `src/lib/analytics.ts`

```typescript
/** GA4 Measurement IDが存在し、windowが利用可能な場合のみ送信 */
function sendGaEvent(
  eventName: Gtag.EventNames | string,
  params?: Gtag.CustomParams,
): void {
  if (typeof window === 'undefined') return;
  if (!window.gtag) return;
  window.gtag('event', eventName, params);
}

/** ゲーム・クイズ・診断の開始 */
export function trackContentStart(params: {
  contentId: string;
  contentType: 'game' | 'quiz' | 'fortune';
}): void {
  sendGaEvent('level_start', {
    level_name: params.contentId,
    content_type: params.contentType,
    content_id: params.contentId,
  });
}

/** ゲーム・クイズ・診断の完了 */
export function trackContentEnd(params: {
  contentId: string;
  contentType: 'game' | 'quiz' | 'fortune';
  success: boolean;
}): void {
  sendGaEvent('level_end', {
    level_name: params.contentId,
    success: params.success,
    content_type: params.contentType,
    content_id: params.contentId,
  });
}

/** 実績アンロック */
export function trackAchievementUnlock(achievementId: string): void {
  sendGaEvent('unlock_achievement', {
    achievement_id: achievementId,
  });
}

/** サイト内検索（デバウンス後・結果あり時のみ） */
export function trackSearch(searchTerm: string): void {
  if (!searchTerm.trim()) return;
  sendGaEvent('search', {
    search_term: searchTerm.trim(),
  });
}

/** SNSシェア */
export function trackShare(params: {
  method: 'twitter' | 'line' | 'web_share' | 'clipboard';
  contentType: string;
  itemId: string;
}): void {
  sendGaEvent('share', {
    method: params.method,
    content_type: params.contentType,
    item_id: params.itemId,
  });
}
```

### 既存コードへの統合ポイント

1. **実績アンロック**: `AchievementProvider.tsx` の `recordPlay` 内で `result.newlyUnlocked` ループ時に `trackAchievementUnlock` を呼ぶ。実績システムと一元管理できる。
2. **クイズ完了**: `QuizContainer.tsx` の `handleStart`（start）と、`setPhase('result')` 前（end）に挿入。
3. **ゲーム完了**: 各ゲームの `GameContainer.tsx` の `recordPlay` 呼び出し箇所と合わせて挿入。
4. **検索**: `useSearch.ts` の `performSearch` 内で結果が出たタイミングに挿入。
5. **シェア**: `ShareButtons.tsx` の各ハンドラーに追加。

### 型定義のセットアップ

`@types/gtag.js` をdevDependenciesに追加し、`tsconfig.json` または `globals.d.ts` でグローバル型を参照する。これにより `window.gtag` が型安全に呼び出せる。
