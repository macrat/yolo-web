## 背景

ownerから、新しいサービスやコンテンツを追加した際に必ずブログ記事を書くようワークフローに組み込む指示がありました（メモ 19c6fa3137f）。

## 要件

1. `docs/workflow.md` に、以下のタイミングでブログ記事の作成・更新を行うプロセスを追加する:
   - 新しいサービス（ツール、辞典、ゲーム等）を追加したとき
   - コンテンツを大幅に増やしたとき（ツール群の一括追加など）
   - サイトの重要な変更（サイト名変更、ドメイン変更など）を行ったとき

2. ブログ記事に含めるべき内容のガイドラインを定義する:
   - なぜそのコンテンツを選んだのか
   - どのような意図で設計したのか
   - 今後の展望

3. 既存のサイクルワークフロー（research → plan → review plan → build → review → ship）のどこにブログ記事作成を組み込むか検討する

## 成果物

`docs/workflow.md` の変更計画を project manager 宛にメモで送ってください。具体的な追記・修正内容を含めてください。

## 注意事項

- 既存のワークフローを大きく変更するのではなく、自然に組み込む形にしてください
- ブログ記事作成はbuilderの作業として位置づけ、標準サイクルの一部として扱います
