## 今回のレビュー観点

### 最重要: ゲームバランスの検証（6回で正解に辿り着けるか）

Owner指示: 「6回の入力で確実に回答に辿り着けるようにゲームバランスを検討すること」

以下の手順で検証してください:

1. **現在のフィードバック属性を整理する**: 部首（部首グループ）、画数、学年、音読み、（新）部首グループカテゴリ。各属性のフィードバックレベル（correct/close/wrong）を確認。

2. **属性の絞り込み能力を定量的に分析する**:
   - 2,136字の常用漢字データについて、「部首グループ + 画数 + 学年 + 音読み数」が完全一致する漢字のグループを調査する
   - 最悪ケース（最大グループサイズ）と最善ケース（最小グループサイズ）を特定する
   - KANJIDIC2の実データを使って分析すること（推定ではなく実測）

3. **6回で正解に辿り着けるか判断する**:
   - 最悪ケースで、フィードバックの「close」判定（画数±2、学年±1、部首グループのスーパーグループ一致）を活用して何回で絞り込めるか
   - 6回で絞り込めない場合、どのようなヒント追加や属性変更が必要か

4. **難易度別の分析**:
   - 初級（240字）、中級（1,026字）、上級（2,136字）それぞれで分析
   - 特に上級（2,136字）で6回以内に正解できるかが重要

### 分析方法の提案

KANJIDIC2のデータをダウンロードしてNode.jsスクリプトで分析するのが最も確実。以下のようなスクリプトを書いて実行してください:

```
1. KANJIDIC2からgrade 1-8の漢字を抽出
2. 各漢字の(radicalGroup, strokeCount, grade, onYomiCount)のタプルを作成
3. 同一タプルの漢字をグルーピング
4. グループサイズの分布（最大、平均、中央値）を出力
5. 最大グループの漢字一覧を出力
```

jmdict-simplifiedのkanjidic2-en.jsonを使うとJSONで直接扱える: https://github.com/scriptin/jmdict-simplified
