# サイトコンセプト文書v4.2: 数値修正と長期参照文書化

## タスク
サイトコンセプト文書v4.1（メモ 19cbc8c4a72）を以下の2点で修正してv4.2を作成してください。

## 修正1: ブログ件数の数値不整合の修正

### 問題
- 実際のブログ記事ファイル数: src/blog/content/ 配下に55件
- v4.1の数値: 保持24 + 修正して保持32 + 削除3 = 59件（4件多い）
- 原因: 再検討で一部記事が二重カウントされた

### 修正方法
1. src/blog/content/ 配下の全ファイルを一覧する
2. 元のブログ評価メモ3件（19cbc40d9f0, 19cbc40e9bb, 19cbc420351）の判定を確認
3. 再検討メモ3件（19cbc84fade, 19cbc851c88, 19cbc85c7b7）の判定を確認
4. PM統合修正メモ（19cbc50bb28）の変更も確認
5. 全55件の最終判定を確定し、正確な内訳（保持/修正して保持/削除）を算出
6. 文書内の数値を修正

## 修正2: 長期参照文書としての品質向上（最重要）

### Ownerの指摘
「今後長期間参照するドキュメントとして適切な書き方になっているか確認してください。サイクル66の手順に固有の情報や、策定経緯の話、経緯を知らないと理解できない説明、などがないか注意深く確認してください。ドキュメント単体で見て理解でき、役に立つ資料としてください。」

### 具体的に除去・修正すべきもの
1. **メモIDの参照**: 「メモ 19cbc70b725」等のメモID参照はすべて除去。メモは内部的な通信手段であり、長期参照文書に含めるべきでない。
2. **サイクル番号への過度な依存**: 「cycle-66で策定」等の表現は最小限に。策定経緯を知らなくても理解できるようにする。
3. **策定プロセスの詳細**: 「強制発想法でN件のアイデアから選定した」等のプロセス説明は不要。結論と根拠のみ記載。
4. **「v3からの変更点」等の差分情報**: 変更履歴セクションは削除するか、最小限の付録にする。読者にとって重要なのは最終的な方針であり、前バージョンとの差分ではない。
5. **中間成果物への参照**: evaluation-rubric.md等、アーカイブされたファイルへの参照は除去。
6. **文脈依存の説明**: 「Ownerの提案に基づき」「レビュー指摘を受けて」等、プロジェクト内部の文脈がないと理解できない表現は、根拠を直接記載する形に書き換え。

### 目指すべき品質
- この文書を初めて読む人（新しいAIエージェント含む）が、他の文書を参照せずに理解できる
- サイトの方針、コンテンツ戦略、技術方針、品質基準が自己完結的に記述されている
- 「なぜそうするのか」の根拠は、外部参照ではなく文書内で完結している
- 実用的なリファレンスとして、判断に迷ったときに参照して答えが得られる
