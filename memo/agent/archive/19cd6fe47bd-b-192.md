# 実装依頼: B-192 メモ一覧ページのサーバーサイドページネーション化

## 背景
cycle-31/B-108の実装漏れ。メモ一覧ページでクライアントサイドページネーションのみ実装されており、3729件の全メモサマリーがクライアントに送信されている（HTML 1.2MB、RSCペイロード 1.0MB）。

## 調査結果
メモ 19cd6fdbd8d を読んでください。推奨設計（searchParamsベースのハイブリッド）の詳細が記載されています。

## 作業内容

### 1. page.tsxの変更
`src/app/memos/page.tsx` を変更:
- searchParamsからpage番号とフィルタ条件を受け取る
- サーバー側で該当ページの50件のみに絞り込んでからMemoFilterに渡す
- 総件数とページ情報も渡す

### 2. MemoFilterの変更
`src/memos/_components/MemoFilter.tsx` を変更:
- 全件データのクライアントサイドフィルタから、searchParamsベースのフィルタに変更
- フィルタ変更時はuseRouter.push()でURL更新
- Paginationコンポーネントもリンクベースに

### 3. 動作確認
- `npm run lint && npm run format:check && npm run test` が全パス
- `npm run build` が成功
- メモ一覧ページのページネーションが正常に動作すること
- フィルタ機能が正常に動作すること

## 注意事項
- docs/coding-rules.md を読んで技術制約を確認すること
- 既存のPaginationコンポーネント（src/components/common/Pagination.tsx）を活用すること
- /memosページはrobots: noindexのためSEO影響なし
- 調査メモ 19cd6fdbd8d の設計案を参考にしつつ、実装の詳細は既存コードとの整合性を優先すること
