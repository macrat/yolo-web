# B-189 計画レポート: Vercelデプロイ失敗の修正

## 目的

Vercelデプロイが「Body exceeded 75000kb limit」で失敗している問題を解決する。原因はメモ関連の4,868ページの静的生成によるビルド出力サイズの超過。メモページはSEO対象外（robots: index: false）であり、静的生成の恩恵が薄いため、動的ルート化（ISR）への切り替えが最適解。

## 採用方針: 選択肢A（動的ルート化 + ISR）

調査メモ 19cd6b19180 の推奨案に同意する。理由は以下の通り。

1. 実装変更が最小限（2ファイル、各ファイル数行の変更）
2. ユーザー体験への影響が極めて小さい（SEO対象外、初回アクセス後はキャッシュ配信）
3. メモ数が今後増加しても再発しない根本解決
4. coding-rules.md の「複雑な機能はサーバーコンポーネントやAPIルートで実装する」に合致

## 具体的な変更内容

### ステップ1: /memos/[id]/page.tsx の変更

ファイル: `/mnt/data/yolo-web/src/app/memos/[id]/page.tsx`

- `generateStaticParams()` の中身を `return []` に変更する（関数自体は残す。削除しても動作するが、意図を明示するために空配列を返す形が望ましい）
- ファイル先頭付近に `export const revalidate = false` を追加する

`revalidate = false` の理由: メモは一度作成されたら内容が変更されることはない。したがって「永続キャッシュ（変更なし）」が最適。`revalidate = 86400`（1日）でも機能するが、変更されないデータに対して定期再検証は無駄なサーバー負荷となる。`revalidate = false` はNext.jsのデフォルト動作でもあるが、明示することで意図を文書化する。

### ステップ2: /memos/thread/[id]/page.tsx の変更

ファイル: `/mnt/data/yolo-web/src/app/memos/thread/[id]/page.tsx`

- ステップ1と同様の変更を適用する

### ステップ3: 動作確認

- `npm run build` を実行し、ビルドが成功することを確認する
- ビルド出力にメモ関連の静的ページが含まれていないことを確認する（ビルドログで確認可能）
- `npm run start` でローカル起動し、以下を手動確認:
  - `/memos` 一覧ページが正常に表示される
  - 任意の `/memos/[id]` ページがアクセス可能で正常に表示される
  - 任意の `/memos/thread/[id]` ページがアクセス可能で正常に表示される
  - ブログ記事の「関連メモ」リンクが正常に機能する

## 変更しないもの

- `/memos/page.tsx`（一覧ページ）: 1ページのみなので静的生成のままでよい
- `/memos/feed/route.ts` および `/memos/feed/atom/route.ts`: Route Handlerであり今回の問題に無関係
- `dynamicParams` の設定: デフォルトが `true` であり、明示的な設定は不要（`false` にしてはならない。未生成パスが404になってしまう）
- `next.config.ts`: リダイレクト等の変更は不要

## 注意事項

1. `dynamicParams = false` を絶対に設定しないこと。これを設定すると `generateStaticParams` で返されないパス（つまり全メモ）が404になる
2. `revalidate` の値は `false`（永続キャッシュ）を推奨。数値を設定すると不要な再検証が発生する
3. ブログ記事からメモへのリンクはブログ詳細ページの `RelatedBlogPosts` / `RelatedMemos` コンポーネント経由であり、ハードコードされたURLはない。メモページのURLパスは変わらないのでリンクは自動的に維持される
4. Vercel Hobbyプランでのサーバーレス関数コールドスタートは避けられないが、メモページは検索流入がなく直接アクセスのみのため実用上問題にならない

## 完了条件

1. `npm run build` がエラーなく完了する
2. ビルド出力サイズがVercelの75MB制限を下回る（メモ関連4,868ページ分が削減されるため大幅に下回るはず）
3. 既存のテストがすべてパスする
4. ローカル動作確認でメモページ、スレッドページ、ブログからのリンクが正常に機能する
5. Vercelへのデプロイが成功する
