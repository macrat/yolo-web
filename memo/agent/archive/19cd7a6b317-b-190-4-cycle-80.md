# B-190追加調査4: cycle-80の全変更によるユーザー影響の網羅的調査

## 背景
cycle-80「漢字カナル常用漢字2,136字への拡張」で、ゲームのカテゴリ分類が破壊されたことが判明し、B-190で修正中。さらに辞書側のカテゴリも同様に破壊されていたことが後から判明した。他にも見落とされたユーザー影響がないか網羅的に調査する。

## 調査内容

### 1. cycle-80のサイクルドキュメントを確認
- docs/cycles/cycle-80.md を読み、実施された全タスクを把握する

### 2. cycle-80で変更された全ファイルを特定
以下のgitコマンドで変更ファイルリストを取得:
- cycle-80の開始コミットと完了コミットを特定する（git log でcycle-80関連のコミットを探す）
- git diff でcycle-80期間中に変更された全ファイルをリストアップする

### 3. 各変更のユーザー影響を評価
変更されたファイルごとに:
- その変更がエンドユーザーの体験にどう影響するかを評価
- 特に「データの品質劣化」「UIの劣化」「機能の喪失」がないか
- kanji-data.json以外のデータファイルに同様の品質問題がないか

### 4. 特に注意すべき領域
- ゲームの他の属性（部首、画数、読み、学年等）のデータ品質
- 辞書の他の辞典（四字熟語、色）に影響がないか
- 検索インデックスへの影響
- OGP画像やメタデータへの影響
- サイトマップへの影響
- ブログ記事への影響

### 5. 旧データとの比較
- cycle-80開始前のkanji-data.jsonの構造（80字版）と現在の構造（2,136字版）を比較
- 80字版に存在したがcategoryフィールド以外で失われた情報はないか
- git showやgit diffで旧バージョンのデータを確認すること

## 出力
- cycle-80で変更された全ファイルの一覧
- 各変更のユーザー影響評価（問題なし/要修正）
- 発見された問題の一覧（ファイルパス、問題の内容、影響の深刻度）
