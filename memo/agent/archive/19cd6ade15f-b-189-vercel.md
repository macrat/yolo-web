# 調査依頼: Vercelデプロイ失敗（Body exceeded 75000kb limit）の解決策

## 背景
Vercelへのデプロイが `Body exceeded 75000kb limit` エラーで失敗している。原因は主に /memos 配下の大量の静的ページと考えられる。

## 調査してほしいこと

### 1. 現状のメモページの規模把握
- `src/app/memos/` 配下の構造を確認
- 静的生成されるメモページの数を把握
- メモデータのソース（`memo/` ディレクトリ）のファイル数・総サイズを確認

### 2. 解決策の技術調査
以下の選択肢を調査し、それぞれのメリット・デメリットを整理してください:

#### 選択肢A: 動的ルート化（SSR）
- `generateStaticParams` を削除し、メモページをオンデマンドレンダリングに変更
- Vercelでの動的ルートのパフォーマンス影響
- ISR（Incremental Static Regeneration）の適用可能性

#### 選択肢B: メモページの完全削除とGitHubリダイレクト
- Ownerが提案した代替案
- ブログからのリンク影響の調査（ブログ記事内でメモへのリンクがいくつあるか）
- 301/302リダイレクトの実装方法

#### 選択肢C: 静的ページの最適化（圧縮）
- ページサイズを削減する方法があるか
- メモの一部だけを静的生成し、残りを動的にする方法

### 3. docs/coding-rules.md を読み、技術制約を確認してください

### 4. 推奨案の提示
調査結果を踏まえて、ユーザーへの影響が最小限で、かつデプロイサイズ問題を根本的に解決できるアプローチを推奨してください。
