# B-190サーバーサイドAPI計画修正依頼: レビュー指摘3点

## レビュー結果
メモ 19cd71f661f を読んでください（B+評価、3点修正）。

## 修正が必要な点

### 1. HintBar方式の矛盾解消
計画内で方式C（targetKanjiをクライアントに維持）を推奨しつつ、3dではtargetKanjiをnull化する方針を提示しており矛盾。GameContainer.tsx 352-355行目でHintBarがtargetKanjiを直接参照しているため、null化すると動作しない。方式を1つに統一すること。

サーバーサイドAPI方式の本来の目的（ターゲットをクライアントに送らない）を考慮すると、HintBarの表示方式も変更が必要。APIレスポンスにHintBar用の情報を含めるか、HintBar自体のUXを再設計する必要がある。

### 2. ResultModalのtargetKanji依存
ゲーム終了時にtargetCharacter（1文字）のみ返す設計だが、ResultModal.tsx 31-38行目ではKanjiEntryの全フィールド（onYomi, kunYomi, meanings, examples等）を使用。ゲーム終了レスポンスにKanjiEntryの表示用フィールドも含める必要がある。

### 3. lookupKanjiの保持
クライアント側のバリデーション（常用漢字チェック、重複チェック）にlookupKanjiは必要。evaluateGuessのimportのみ削除し、lookupKanjiは保持すること。
