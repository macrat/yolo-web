# Phase D やり直し依頼

## 背景

ひねり強制発想法で273件の成立アイデアが出た（Phase C完了）。
これを17コンセプトに統合したが（Phase D完了）、この統合は機能別（A=占い・診断、B=ゲーム、C=辞書）でのみ分類されており不十分。

## Ownerの指摘

「セグメント特化でゲーム・占い・診断を揃える」のように、テーマ別にして機能横断にする形でも考えるべき。
つまり、ひねりの種類（キャラ付き、セグメント特化、ユーモア、方言、テーマ差し替え、デイリー等）をテーマとして、占い・ゲーム・辞書を横断して統合する案も作るべき。

## 入力データ

- 273件の成立アイデア: docs/forced-ideation/twist/phase-c-viable-ideas.txt
- 現行の17コンセプト（参考、ただしこれに引きずられないこと）: docs/forced-ideation/twist/phase-d-consolidated-concepts.txt

## 依頼内容

273件の成立アイデアを読み、以下の2つの観点で再統合してください：

### 観点1: テーマ軸（機能横断）
ひねりの種類をテーマとして、占い・ゲーム・辞書を横断する統合を行う。
例:
- 「キャラ付きコンテンツ」: キャラ付き占い + AI性格対戦ゲーム + キャラテーマ辞書
- 「セグメント特化コンテンツ」: 職業別占い + 業界ネタゲーム + 文脈テーマ辞書
- 「ユーモア全開コンテンツ」: 逆張り占い + ルール変更ゲーム + 逆引きゲーム辞書
- 「デイリーコンテンツ」: 日替わり占い + デイリーチャレンジゲーム + 日替わり辞書

### 観点2: 機能軸（従来型、改良版）
現行の17コンセプトをベースにしつつ、より実用的な粒度に再整理する。
ただし、「キャラ付き占い」と「ユーモア系占い」は実質的に同じ方向性なのでまとめる等、本当に別コンセプトとして成立するものだけを残す。

## 出力

docs/forced-ideation/twist/phase-d-v2-consolidated.md に以下を含む形で出力：
- テーマ軸統合コンセプト一覧（各コンセプトに含まれるアイデア数、概要、提供価値、実装イメージ）
- 機能軸統合コンセプト一覧（同上）
- 各コンセプトがどの元アイデア番号を含むかの対応表

## 注意事項

- 「大量×高品質は不可能」原則を忘れないこと。辞書系コンセプトは少量データでも成立するか要検討。
- テーマ軸と機能軸は別のレンズで同じ273件を見たもの。同じアイデアが両方に含まれてよい。
- 無理に全アイデアをコンセプトに含めなくてよい。コンセプトとして成立しないものは除外して構わない。
