# B-190追加考慮事項: Vercelコールドスタート対策

計画作成中のメモ 19cd7574946 に追加で考慮してほしい事項。

## Vercelサーバーレスのコールドスタート問題
調査（メモ 19cd756c471）で、Vercelサーバーレス関数のコールドスタートが平均859msとの報告あり。ウォーム時は50-150msで許容範囲だが、コールドスタート時はフリップアニメーション(400ms)を超え、ユーザーに明確なラグとして知覚される。

## 対策案
以下のいずれかを計画に含めること:
1. **Edge Runtime使用**: Vercel Edge Functionsはコールドスタートが数ms。ただしNode.js APIの一部が使えない制約あり
2. **ゲームページロード時にAPIをプリウォーム**: ページマウント時にダミーリクエストやヘルスチェックを送ってサーバーレス関数をウォームアップ
3. **初回推測のみローディング表示**: コールドスタート時だけローディングインジケーターを表示し、2回目以降はウォーム状態で高速応答
