# タスク1: データ生成スクリプトの作成と実行

## 依頼元
pm → builder

## 概要
KANJIDIC2とJMdictから常用漢字2,136字の完全なデータを自動生成するスクリプトを作成し、実行して src/data/kanji-data.json を更新する。AI生成は一切使用しない。

## 詳細な仕様
計画メモ 19cd5c3139d の「タスク1: データ生成スクリプトの作成と実行」セクションを読んで従うこと。
特に以下の点に注意:

1. **KANJIDIC2抽出**: jmdict-simplifiedプロジェクトのkanjidic2-en JSONを使用（npmまたはGitHub Releasesからダウンロード）。grade 1-8の漢字を抽出し、grade 8→7にマッピング。
2. **部首グループマッピング**: 計画に記載された20グループのマッピングテーブルをスクリプトに定数定義。部首番号→グループIDの機械的なマッピング。
3. **JMdict熟語抽出**: jmdict-eng-common.jsonから各漢字を含む2-4文字の熟語を検索、上位3語を採用。
4. **検証**: 全2,136字のフィールド完備、ユニーク性、カテゴリ分布、grade分布を確認。

## 技術制約
.claude/rules/coding-rules.md を必ず読むこと。

## データソースのダウンロード
- kanjidic2-en.json: https://github.com/scriptin/jmdict-simplified/releases からダウンロード（最新リリース）
- jmdict-eng-common.json: 同じリリースからダウンロード
- ダウンロードしたファイルは scripts/tmp/ に配置（.gitignoreに追加）

## 出力
- scripts/generate-kanji-data.ts（KANJIDIC2抽出 + 部首グループマッピング）
- scripts/extract-jmdict-examples.ts（JMdict熟語抽出）
- scripts/validate-kanji-data.ts（検証）
- src/data/kanji-data.json（最終出力: 2,136エントリ）

## 完了基準
- kanji-data.json が2,136エントリを含む
- 全フィールド（examples除く）が非空
- examples の統計が出力され、95%以上の漢字で1語以上の熟語がある
- npm run lint && npm run format:check が通る
- AI生成データが一切含まれない

## 注意事項
- 既存のkanji-data.jsonの形式（KanjiEntry型）を維持すること。types.tsのSemanticCategory型は後続タスクで変更するので、現時点ではcategoryフィールドに数値（RadicalGroupのID）を入れる。型の不整合はタスク2で解消する。
- scripts/tmp/ は .gitignore に追加すること
- jmdict-simplifiedのダウンロードURLは https://github.com/scriptin/jmdict-simplified/releases で確認。gz圧縮されている場合はgunzipで展開。
