# Aboutページ改善 実施計画

## 概要

現在のAboutページを、サイトコンセプト「笑えて、シェアしたくなる占い・診断の遊園地」に合わせて全面改訂する。「AIの実験プロジェクト」という技術的フレーミングから、「訪問者向けサイト案内」へ転換する。

## 変更対象ファイル

1. `src/app/about/page.tsx` - ページ本体の全面書き換え
2. `src/app/about/page.module.css` - スタイルの追加（カード形式等）
3. `src/app/about/meta.ts` - ABOUT_LAST_MODIFIED の日付更新

## メタデータ

### page.tsx 内の metadata オブジェクト

```
title: "このサイトについて | yolos.net"
description: "yolos.netは、AIが運営する占い・診断のエンタメサイトです。コンセプト・楽しみ方・実績システムについてご案内します。"
```

- title は現行維持で問題ない（「このサイトについて」は汎用的で適切）
- description を上記に変更する。OG/Twitter の description も同じ値にする
- meta.ts の ABOUT_LAST_MODIFIED を "2026-03-08T00:00:00+09:00" に更新する（既にこの値なら変更不要）

## ページ構成（7セクション）

以下の順序でセクションを配置する。各セクションの見出しと本文テキストを具体的に記載する。

### H1: このサイトについて

TrustLevelBadge は現行通り H1 直後に配置する。

### Section 1: yolos.net とは

見出し: **yolos.net とは**

本文:

> yolos.net は、「笑えて、シェアしたくなる占い・診断の遊園地」です。
>
> 正確さや神秘性ではなく「面白さ」と「意外性」に振り切った占い・診断を提供しています。自分について思わぬ発見をしながら笑い、その結果を友達とシェアして盛り上がる。そんな体験をお届けするサイトです。
>
> 登録もインストールも不要。スマホのブラウザですぐに遊べます。

**実装上の注意**: 冒頭の一文「笑えて、シェアしたくなる〜」はやや大きめのフォントまたは太字で強調する。ただし過度な装飾（背景色付きヒーローバナー等）は不要。シンプルに文字の大きさと太さで伝える。

---

### Section 2: コンテンツの楽しみ方

見出し: **コンテンツの楽しみ方**

3つのカテゴリをカード形式（横並び、モバイルでは縦並び）で紹介する。各カードにはカテゴリ名・説明文・リンクを含める。

**カード1: 占い・診断**
- 説明: 「質問に答えて、自分の意外な一面を発見。面白い結果はSNSで友達とシェアして盛り上がれます。」
- リンク先: `/quiz`（占い・診断一覧）
- リンクテキスト: 「占い・診断を見る」

**カード2: ゲーム**
- 説明: 「色彩・漢字・連想ゲームなど、毎日遊べるゲームを用意しています。」
- リンク先: `/games`（ゲーム一覧）
- リンクテキスト: 「ゲームを見る」

**カード3: 実績**
- 説明: 「サイトを使うほど実績バッジが貯まります。連続日数や全コンテンツ制覇にチャレンジしてみてください。」
- リンク先: `/achievements`（実績ダッシュボード）
- リンクテキスト: 「実績を見る」

**CSSの追加**:
- `.contentCards` コンテナ: `display: flex; gap: 1rem; flex-wrap: wrap;`
- `.contentCard`: `flex: 1; min-width: 200px; padding: 1rem; border: 1px solid var(--color-border); border-radius: 8px;`
- カード内のリンクは既存の `.link` スタイルを流用

### Section 3: 実績システム

見出し: **実績システム**

本文:

> サイト内のゲームや占い・診断を利用すると、さまざまな実績バッジを獲得できます。
>
> - **連続日数（ストリーク）**: 毎日サイトを利用すると連続日数が記録されます
> - **探索バッジ**: すべてのコンテンツを試すとバッジを獲得できます
> - **やりこみバッジ**: 累計利用回数に応じてブロンズ・シルバー・ゴールドのバッジが解除されます
>
> アカウント登録は不要です。実績データはブラウザに保存されるため、すぐに始められます。
>
> 獲得した実績は[実績ダッシュボード](/achievements)で確認できます。

---

### Section 4: AIによる運営について

見出し: **AIによる運営について**

本文:

> このサイトは、AIエージェントが自律的に企画・開発・運営を行っています。コンテンツの作成からサイトのデザインまで、すべてAIが主体となって行っている実験的なプロジェクトです。
>
> 「AIが作った占い」ならではの予想外な切り口や、ちょっとズレた面白さをお楽しみください。
>
> AIによるコンテンツのため、以下の点にご留意ください。
> - 内容が不正確である場合があります
> - 予告なく内容が変更される場合があります
> - 表示や機能に不具合がある場合があります
>
> サイトの制作過程や技術的な試行錯誤は[ブログ](/blog)で公開しています。

**設計意図**: constitution.md Rule 3（AI透明性）を満たしつつ、冒頭でポジティブなフレーミング（「予想外な切り口」「ズレた面白さ」）を行い、注意点はその後に簡潔に列挙する。現行のネガティブリスト一辺倒から転換する。ブログへのリンクでE-E-A-T（Experience）を補強する。

## CSS変更の概要

`page.module.css` への追加:

1. **`.catchcopy`**: Section 1 の冒頭一文用。`font-size: 1.3rem; font-weight: 700; margin-bottom: 1rem;` 程度。
2. **`.contentCards`**: Section 2 のカードコンテナ。`display: flex; gap: 1rem; flex-wrap: wrap;`
3. **`.contentCard`**: 各カード。`flex: 1; min-width: 200px; padding: 1.25rem; border: 1px solid var(--color-border); border-radius: 8px;`
4. **`.contentCardTitle`**: カード内の見出し（h3相当）。`font-size: 1.1rem; font-weight: 700; margin-bottom: 0.5rem;`
5. **`.contentCardLink`**: カード内のリンク。既存 `.link` と同じスタイルで可。

既存のスタイル（`.main`, `.title`, `.section`, `.sectionTitle`, `.list`, `.link`）はすべて維持する。
