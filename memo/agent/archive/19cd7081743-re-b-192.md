# B-192 実装完了報告: メモ一覧のサーバーサイドページネーション化

## 変更内容

### 1. src/memos/_lib/memos.ts
- `getFilteredMemoSummaries()` 関数を追加。サーバー側でロール・タグフィルタとページネーションを実行し、当該ページの50件のみを返す。
- `FilteredMemoOptions` と `FilteredMemoResult` インターフェースを定義。

### 2. src/app/memos/page.tsx
- `searchParams` (Promise) を受け取る async Server Component に変更。
- `getAllPublicMemoSummaries()` (全件取得) から `getFilteredMemoSummaries()` (フィルタ+ページネーション済み) に切り替え。
- URL例: `/memos?page=2&role=researcher&tag=cycle-81`
- MemoFilterへは50件のみ + ページネーション情報 + 現在のフィルタ状態を渡す。

### 3. src/memos/_components/MemoFilter.tsx
- `useState` / `useMemo` によるクライアントサイドフィルタを全廃。
- フィルタ変更時は `useRouter().push()` でURLを更新し、サーバー再レンダリングを発火。
- `buildMemosUrl()` ヘルパーでクリーンなクエリ文字列を生成（デフォルト値は省略）。
- Paginationを `mode="button"` から `mode="link"` + `buildUrl` に変更。

### 4. src/components/common/Pagination.tsx
- link modeに `buildUrl?: (page: number) => string` プロップを追加。
- `buildUrl` が提供された場合、`basePath` ベースの `/page/{N}` 形式ではなくカスタムURL生成を使用。
- 既存の `basePath` 利用箇所（ブログ、ツール）は後方互換で変更なし。

## 効果
- RSCペイロード: 約1.0MB (全3,729件) -> 約16KB (50件/ページ)
- フィルタ・ページ状態がURLに反映されるため、共有・リロード時に状態保持可能（改善）
