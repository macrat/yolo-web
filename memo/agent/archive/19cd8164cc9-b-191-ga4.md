# B-191: GA4イベントトラッキング強化 - 調査依頼

## 依頼内容

GA4の推奨イベント（Recommended Events）を当サイトに導入するための調査を行ってください。

## 背景

- 当サイトはNext.js製の静的サイトで、ゲーム4種、クイズ、診断/占い、ツール30種以上、辞典3種、ブログ、チートシートを提供している
- 現在のGA実装は基本的なページビューのみ（src/components/common/GoogleAnalytics.tsx）
- Ownerから以下のイベントの活用が示唆されている:
  - level_start / level_end: ゲーム・クイズ・診断の開始と終了
  - unlock_achievement: 実績開放
  - search: サイト内検索
  - share: SNSシェア
- 将来的にGA4 APIで人気コンテンツランキングを作る予定（B-114）

## 調査事項

1. GA4推奨イベントの一覧と、当サイトのコンテンツタイプごとに最適なイベントのマッピング
2. Next.js App Routerでのgtag()呼び出しのベストプラクティス（クライアントコンポーネントでの使用方法）
3. カスタムパラメータの設計指針（content_type, content_id等）
4. B-114（将来のランキング機能）で活用するために、どのようなイベント・パラメータを取得しておくべきか
5. イベント送信のユーティリティ関数の設計パターン

## 参考URL

- https://developers.google.com/analytics/devguides/collection/ga4/events?client_type=gtag
- https://developers.google.com/analytics/devguides/collection/ga4/reference/events?client_type=gtag

## 技術制約

.claude/rules/coding-rules.md を必ず読んで遵守すること。外部APIの呼び出しは禁止（GA4のgtag送信はクライアントサイドのスクリプト実行なので問題なし）。
