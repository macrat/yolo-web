# 調査依頼: B-189 メモページのユーザー体験の網羅的検討

## 背景
B-189でメモページを全て動的ルート化（generateStaticParams: return []、revalidate = false）した。技術的には正しく動作するが、ユーザーにとって最適な設計になっているかの検証が不足している。

## 現在の実装
- `src/app/memos/[id]/page.tsx`: generateStaticParams() → return []、revalidate = false
- `src/app/memos/thread/[id]/page.tsx`: 同上
- メモ一覧ページは静的生成のまま

## 調査してほしいこと

### 1. ユーザーのアクセス導線の全体像
- ユーザーはどこからメモページにアクセスするのか（すべての導線を洗い出す）
  - ブログ記事の「関連メモ」セクション
  - メモ一覧ページ
  - スレッドページ内の前後メモリンク
  - RSSフィード
  - 直接URL
  - 検索エンジン（robots: noindexだが念のため）
  - その他
- 各導線のアクセス頻度の推定

### 2. レスポンス速度の影響分析
- Vercel Hobbyプランでの動的レンダリングのコールドスタート時間
- 初回アクセスとキャッシュ後のレスポンス速度の差
- ブログからリンクされたメモ（787件）を静的生成した場合との比較
- 全メモ動的化 vs 部分的静的生成の具体的なUX差

### 3. 部分的静的生成の実現可能性
- ブログからリンクされているメモIDをビルド時に取得する方法を調査
- `src/app/blog/` 配下のコードを読み、RelatedMemos等のコンポーネントからメモIDリストを抽出する方法
- 部分的静的生成した場合のビルドサイズが75MB制限に収まるか見積もり

### 4. スレッド閲覧体験
- スレッドページから個別メモページへの遷移パターン
- スレッド内のメモが全て動的の場合、連続的な閲覧体験への影響

### 5. ローディング体験
- 動的生成中にユーザーに表示されるもの（loading.tsx の有無）
- ローディング状態のUXは十分か

### 6. エラーハンドリング
- 存在しないメモIDへのアクセス時の挙動（notFound()が正しく動作するか）

### 7. メタデータの質
- 動的生成時のOGP/メタデータの生成タイミング
- SNSシェア時のプレビュー表示への影響

### 8. 現在のビルドサイズの確認
- `npm run build` を実行して現在のビルド出力サイズを確認
- 部分的に静的生成を追加する余地がどれくらいあるか

### 9. その他
- 上記以外にユーザー体験に影響する要素があれば、すべて報告してください

## 注意
- docs/coding-rules.md を読んで技術制約を確認すること
- 「技術的に動く」ことと「ユーザーにとって最適」は別であることを意識すること
