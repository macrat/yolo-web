## 調査内容

### 1. 全ゲームのUI構成の確認
以下の各ゲームのUIコンポーネント構造とスタイリングを調査する:
- src/app/(games)/games/irodori/
- src/app/(games)/games/kanji-kanaru/

- src/app/(games)/games/yoji-kimeru/

### 2. 共通コンポーネントの確認
- src/app/(games)/_components/ 以下の共通コンポーネントを確認
- 各ゲームでどの共通コンポーネントが使われているか確認

### 3. UX上の問題点の特定
- 操作フロー（ゲーム開始→プレイ→結果表示→シェア/リプレイ）を確認
- レスポンシブ対応の状況を確認
- アクセシビリティ対応の状況を確認（ARIA属性、キーボード操作等）
- パフォーマンス上の懸念を確認

### 4. B-152（ゲーム関連その他改善）との関連確認
- backlog.mdのB-152で指摘されている問題（ナカマワケ誤答永続化不足、履歴型乖離、SSR/CSR不整合、games一覧日付固定化）がUI/UX改善に含まれるべきか確認

- 技術的制約を把握する
