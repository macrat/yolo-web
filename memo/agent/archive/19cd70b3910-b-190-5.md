# B-190計画修正依頼: レビュー指摘対応

## レビュー結果
メモ 19cd70aa4d6 を読んでください（B+評価、5点の修正事項）。

## 修正が必要な点

### 1. correct閾値の再設計（最重要）
同一文字のみをcorrectとする設計では、正解時にゲーム終了するため、category列のcorrectがプレイ中に出現しない。3段階が実質2段階に退化する。
→ correct閾値をコサイン類似度の上位（例: 0.85以上）に設定し、「非常に近い意味」をcorrectとする設計に変更すること。

### 2. 128次元を第一選択に
Vercelデプロイサイズ制限の問題が既に発生中。384次元（約1.1MB）のリスクは高い。
→ 128次元を基本とし、タスク2で品質差を検証する手順に変更。

### 3. プリロード方式をメイン設計に
代替案として記載されていたプリロード方式の方が明確に優れている（engine.ts同期維持、テスト影響最小）。
→ メインの設計として採用。

### 4. close率の定量的根拠
15-25%の根拠が不足。
→ タスク2で、1回のカテゴリフィードバックで候補が何%に絞り込まれるか計算し、6回で収束可能か検証する手順を追加。

### 5. プレイスルーテストの具体化
→ 初級/中級/上級各1シナリオ以上でカテゴリフィードバックがヒントとして解釈可能か検証する合格基準を追加。

## 追加で対応すべき点
- ブラウザキャッシュ一貫性: ファイル名にハッシュを含めるか検討
- 量子化時のクランプ処理を明記
- HowToPlayModalの具体的な説明文案をタスク3に含める
