# 修正依頼: web-developer-tools記事のツール紹介方式の変更

## 背景
ownerの指摘により、記事執筆時の20ツールを尊重しつつ、後から追加された12ツールも紹介する方式に変更する。

## 方針
現在の「32ツール」一括紹介を廃止し、以下の構成にする:
- 本文: 元の20ツールの記述をそのまま維持（title/descriptionも「20ツール」のまま）
- 追記セクション: 記事末尾（まとめの前）に「追記: その後追加されたツール」のような追記ボックスを設けて、12個の新ツールを簡潔に紹介

## 具体的な修正内容

### 1. frontmatter
- title: 「32ツールの使い分け」→「20ツールの使い分け」（元に戻す）
- description: 「32個」→「20個」（元に戻す）
- related_tool_slugs: 追加した12個を削除し元の20個に戻す

### 2. 本文
- 冒頭のツール数の記述を元の20に戻す
- 現在の「その他のツール（12個）」セクションを削除
- 代わりに、まとめセクションの直前に追記ボックスを追加:

```markdown
:::note[追記（2026年2月更新）]
この記事の公開後、以下の12個のツールが新たに追加されました。

**テキスト処理**: 文字列反転、Lorem Ipsum生成、テキスト差分比較
**データ変換**: 単位変換、Cron式パーサー
**エンコード・デコード**: HTMLエンティティ変換、URLパーサー
**開発支援**: IPアドレス情報、HTTP Status一覧、カラーコード変換
**セキュリティ**: TOTP生成
**その他**: QRコード生成

各ツールの詳細は[ツール一覧](/tools)をご覧ください。また、ツール拡充の経緯は[ツール拡充の記事](/blog/tools-expansion)で解説しています。
:::
```

※ このサイトで :::note 記法が使えるか確認すること。使えない場合は、blockquoteや通常のセクションで代替する。

### 3. まとめセクション
- ツール数を20に戻す
- 追記ボックスとの整合性を確認
