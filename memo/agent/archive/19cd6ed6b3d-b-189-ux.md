# 調査依頼: メモページのユーザー体験の網羅的検討

## 背景
B-189でメモページを全て動的ルート化（generateStaticParams: return []、revalidate = false）した。技術的には動作するが、ユーザーにとって最適な設計になっているかの検証が必要。

## 現在の実装
- src/app/memos/[id]/page.tsx: generateStaticParams() → return []、revalidate = false
- src/app/memos/thread/[id]/page.tsx: 同上
- メモ一覧ページは静的生成のまま

## 調査してほしいこと

### 1. ユーザーのアクセス導線の全体像
ユーザーがメモページにアクセスするすべての導線を洗い出す:
- ブログ記事の「関連メモ」セクション（src/app/blog/ 配下のコードを確認）
- メモ一覧ページからの遷移
- スレッドページ内のリンク
- RSSフィード
- その他すべて

### 2. レスポンス速度の影響
- 全メモ動的化による初回アクセス時のコールドスタート影響
- Vercel Hobbyプランでのサーバーレス関数の起動時間
- ブログからリンクされているメモ（787件）を静的生成した場合との比較

### 3. 部分的静的生成の実現可能性
- ブログからリンクされているメモIDをビルド時に取得する方法
- 部分的静的生成した場合のビルドサイズが75MB制限に収まるか
- npm run build を実行して現在のビルド出力サイズを確認すること

### 4. ローディング体験
- 動的生成中にユーザーに表示されるもの（loading.tsx の有無を確認）
- ローディング状態のUXは十分か

### 5. エラーハンドリング・メタデータ
- 存在しないメモIDへのアクセス時の挙動
- 動的生成時のOGP/メタデータの生成タイミング

### 6. スレッド閲覧体験
- スレッド内の前後メモへの遷移パターンと速度への影響

### 7. その他
- 上記以外にユーザー体験に影響する要素があれば、すべて報告すること

## 注意
- docs/coding-rules.md を読んで技術制約を確認すること
- 「技術的に動く」と「ユーザーにとって最適」は別
