# プライバシーポリシー要件 調査レポート

## 1. Google AdSenseのプライバシーポリシー必須要件

### 必須記載事項（公式ヘルプより）

Google AdSense利用サイトのプライバシーポリシーには、以下を明記する必要がある。

**[A] Cookie使用の開示**
「Google などの第三者配信事業者が Cookie を使用して、ユーザーがそのウェブサイトや他のウェブサイトに過去にアクセスした際の情報に基づいて広告を配信すること」

**[B] パーソナライズ広告のオプトアウト**
ユーザーがパーソナライズ広告を無効にできることと、その方法を記載する。
- Googleの広告設定: https://www.google.com/settings/ads
- NAI（Network Advertising Initiative）のオプトアウトツール: https://www.aboutads.info

**[C] 第三者ベンダーの開示**
第三者がユーザーのブラウザにCookieを設置・読み取りしたり、ウェブビーコンやIPアドレスを収集したりする可能性があることを開示する。

**[D] データ収集・共有・利用の総合的な開示**
Googleのサービス利用に伴うデータ収集・共有・利用について明記する。

**[E] プライバシーポリシーURLの登録**
AdSenseアカウントの「プライバシーとメッセージ」設定でプライバシーポリシーのURLを登録する必要がある（審査通過後も引き続き必要）。

**[F] アクセシビリティ**
プライバシーポリシーページはトップページから1クリック（ワンクリック）で到達できる場所に設置する必要がある。フッターへのリンク設置が標準的。

### 重要な注意点

- Googleは国や地域によって法律が異なるため、具体的な文面の提案は行っていない。
- 2025年2月16日にパブリッシャー向けポリシーが更新されたが、ユーザーへの透明性確保の義務は継続している。

参考URL: https://support.google.com/adsense/answer/1348695?hl=ja


## 2. 日本の個人情報保護法で求められる記載事項

### yolos.netの個人情報取扱事業者としての立場

2017年改正により5000件の件数制限が撤廃されたため、1件でも個人情報を取り扱えば個人情報取扱事業者に該当する。yolos.netはGoogle Analyticsを使用しており、IPアドレスやCookieを通じたアクセス情報を収集しているため、個人情報保護法の対象となる可能性がある。

ただし、LocalStorageはサーバーに送信されずブラウザ内に完結するため、個人情報の「収集」には該当しない。

### 必須または推奨の記載事項

**個人情報保護法第21条（利用目的の通知）・第32条（保有個人データの開示）に基づく記載:**

1. **事業者の名称・住所** - 運営者情報（yolos.netの場合はGitHubリポジトリへの言及が現状）
2. **個人情報の利用目的** - アクセス解析、サービス改善など具体的に記載
3. **保有個人データの利用目的** - 同上
4. **第三者提供** - Googleへのデータ送信（Analytics）
5. **安全管理措置** - 2022年改正で追加。HTTPS使用、LocalStorage限定のゲームデータなどを記載
6. **開示・訂正・削除の手続き** - ユーザーからの請求対応方法（GitHubのIssue経由等）
7. **苦情申し出先** - 問い合わせ窓口

**2024年4月改正で新たに追加された事項:**
- 安全管理措置の対象が「個人データとして取り扱うことを予定している個人情報」にも拡大された。ウェブスキミング対策などの記載が必要。

### yolos.netが実際に収集・使用しているデータ

コードベースの調査結果:

**Google Analytics（サーバー側収集）:**
- IPアドレス（Googleに送信）
- ページビュー・セッション情報
- ブラウザ・デバイス情報
- `NEXT_PUBLIC_GA_TRACKING_ID`環境変数で管理
- `analytics_storage: 'granted'`がデフォルト設定（ユーザーの同意なしに有効）

**LocalStorage（ブラウザ内のみ・サーバー非送信）:**
- ゲーム統計データ（ゲームプレイ回数、連続プレイ日数、正解率など）
- ゲーム履歴（日付別プレイ記録）
- 対象ゲーム: 仲間分け、いろどり、漢字かなる、四字決め
- クロスゲーム進捗管理データ

**収集していないデータ:**
- 氏名・メールアドレス等の直接的な個人識別情報（お問い合わせフォームなし）
- アカウント・認証情報（認証機能なし）
- 支払い情報（決済機能なし）


## 3. 同規模の日本語Webサイトのプライバシーポリシー事例

個人ブログ・小規模サイトで一般的に採用されている構成:

### 典型的なページ構成（セクション順）

1. **基本方針** - プライバシー保護への取り組み姿勢
2. **収集する情報** - Cookieやアクセスログの説明
3. **情報の利用目的** - サービス提供・改善・広告配信
4. **第三者への提供** - GoogleアナリティクスとAdSense
5. **Cookieについて** - Cookie利用と無効化方法の説明
6. **アクセス解析ツール（Google Analytics）** - GAの利用とオプトアウト
7. **広告について（Google AdSense）** - AdSenseのCookie利用と広告設定
8. **免責事項** - リンク先・内容の正確性
9. **個人情報の管理と開示請求** - 問い合わせ窓口
10. **プライバシーポリシーの変更** - 変更時の告知方法
11. **制定日・改定日**

### 設置場所

- フッターにテキストリンクを配置（最も一般的）
- URL: `/privacy-policy` または `/privacy` が慣例
- 検索エンジンへのインデックス: サイトによって異なる（AdSense審査時はインデックスを推奨する場合もあり）

### 小規模サイト特有の傾向

- 運営者の住所・氏名は省略し、GitHubやお問い合わせフォームへの誘導のみにするケースあり
- 「このサイトはAIが運営」などの注記をAboutページに集約し、プライバシーポリシーはシンプルに保つ傾向


## 4. 技術制約の確認

### coding-rules.md（`/mnt/data/yolo-web/.claude/rules/coding-rules.md`）の制約

- **禁止**: 外部APIの呼び出し、データベース、認証・ユーザー管理
- **可能**: クライアントサイドLocalStorage、静的コンテンツ、サーバーコンポーネント、APIルート
- **方針**: 静的コンテンツ優先、シンプルで一貫したコンポーネント設計

### プライバシーポリシーページの実装方針

以下の方針が技術制約と整合的:

- **実装方法**: 静的ページ（`/app/privacy/page.tsx`）として作成
- **外部依存なし**: Cookie同意バナー等の外部サービス不要（現時点では）
- **コンテンツ**: Markdownではなく直接JSXで記述（他のAboutページと同様）
- **フッターへのリンク追加**: `src/components/common/Footer.tsx` に `/privacy` リンクを追加
- **AdSense URLの登録**: AdSenseアカウント管理画面で別途登録が必要


## 5. まとめと推奨事項

### プライバシーポリシーページに必要な記載内容

yolos.netのプライバシーポリシーとして以下のセクション構成を推奨する:

1. **はじめに** - サイト名・運営概要
2. **収集する情報** - Google Analytics経由のアクセス情報（Cookie含む）、LocalStorageに保存するゲームデータ（ブラウザ内のみ）
3. **利用目的** - アクセス解析・サービス改善
4. **第三者サービスの利用**
   - Google Analytics: データ収集・オプトアウト方法
   - Google AdSense（将来的）: Cookie使用・パーソナライズ広告・オプトアウト方法（広告設定、aboutads.info）
5. **Cookieについて** - 利用するCookieの種類と無効化方法
6. **情報の管理と安全管理措置** - HTTPS使用、サーバー側での個人情報不保持
7. **個人情報の開示・訂正・削除** - GitHub Issueを通じた対応
8. **プライバシーポリシーの変更** - 変更時のサイト上告知
9. **お問い合わせ** - GitHub リポジトリへの誘導
10. **制定日**

### 特記事項

- yolos.netはお問い合わせフォームを持たず、ユーザーから直接個人情報を収集する仕組みがない。ゲームのLocalStorageデータはサーバーに送信されない。
- Google Analyticsのデフォルト設定が `analytics_storage: 'granted'` になっているため、プライバシーポリシーでその旨を明確に説明し、オプトアウト方法を案内することが重要。
- AdSense申請を見据える場合、プライバシーポリシーのURLをAdSenseアカウントに登録する必要がある。
