# タスク3: 辞書UIページの再構築

## 目的
無意味なcategoryベースのナビゲーションを、学年別・部首別・画数別の3軸ナビゲーションに置き換える。

## 前提
タスク2が完了済み（types.ts, kanji.tsに新しい関数・型が定義済み）。

## 変更・削除ファイル

### 削除
- `src/app/dictionary/kanji/category/` ディレクトリを丸ごと削除

### 新規作成
1. `src/app/dictionary/kanji/grade/[grade]/page.tsx` — 学年別一覧
2. `src/app/dictionary/kanji/radical/[radical]/page.tsx` — 部首別一覧
3. `src/app/dictionary/kanji/stroke/[count]/page.tsx` — 画数別一覧

### 修正
4. `src/app/dictionary/kanji/page.tsx` — メインページのナビゲーション変更
5. `src/app/dictionary/kanji/KanjiIndexClient.tsx` — カテゴリラベルを学年表示に変更

## 具体的な変更内容

### 1. 学年別ページ (grade/[grade]/page.tsx)
- 既存のcategory/[category]/page.tsxをベースにする
- generateStaticParams: `getKanjiGrades()` を使用
- generateMetadata: 「小学1年の漢字一覧 - 漢字辞典」のようなタイトル
- `KANJI_GRADE_LABELS` を使用してラベル表示
- `getKanjiByGrade(Number(grade))` でデータ取得
- ナビゲーション: 学年別のCategoryNavを表示（basePath="/dictionary/kanji/grade"）

### 2. 部首別ページ (radical/[radical]/page.tsx)
- generateStaticParams: `getKanjiRadicals()` で全部首を取得し `{ radical: encodeURIComponent(r) }` を返す
- パラメータ受け取り時: `decodeURIComponent(radical)` でデコード
- generateMetadata: 「部首「木」の漢字一覧 - 漢字辞典」のようなタイトル
- `getKanjiByRadical(decodedRadical)` でデータ取得
- ナビゲーション: 部首別のCategoryNavを表示。ラベルは部首文字そのまま

### 3. 画数別ページ (stroke/[count]/page.tsx)
- generateStaticParams: `getKanjiStrokeCounts()` で全画数を取得し `{ count: String(c) }` を返す
- generateMetadata: 「5画の漢字一覧 - 漢字辞典」のようなタイトル
- `getKanjiByStrokeCount(Number(count))` でデータ取得
- ナビゲーション: 画数別のCategoryNavを表示。ラベルは「N画」形式

### 4. メインページ (page.tsx)
- 既存のCategoryNavを3つのセクションに変更:
  - 「学年別」セクション: 7つのリンク（basePath="/dictionary/kanji/grade"）
  - 「部首別」セクション: 全部首のリンク（basePath="/dictionary/kanji/radical"）。部首が多い（~180）ので、主要な部首のみ表示するか、折りたたみにするか検討
  - 「画数別」セクション: 1画〜29画のリンク（basePath="/dictionary/kanji/stroke"）
- imports: KANJI_CATEGORY_LABELS → KANJI_GRADE_LABELS、getKanjiCategories → getKanjiGrades, getKanjiRadicals, getKanjiStrokeCounts

### 5. KanjiIndexClient.tsx
- `KANJI_CATEGORY_LABELS[k.category]` → `KANJI_GRADE_LABELS[k.grade]` に変更
- import変更

## 注意事項
- CategoryNavは汎用コンポーネントなので変更不要
- 部首別は部首文字をURLに使うため、encodeURIComponentが必要
- 各ページのBreadcrumb・SEOメタデータ・OGデータも正しく設定すること
- 既存のcategory/[category]/page.tsxのパターンに合わせる

## 完了条件
- 旧categoryページが削除されている
- 3種類の分類ページが正しいメタデータ・Breadcrumb・ナビゲーション付きで作成されている
- メインページに3つの分類軸のナビゲーションがある
- KanjiIndexClientが学年表示になっている
