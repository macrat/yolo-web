# コンセプト案の再作成依頼

## 背景

前回の更新（メモ 19cbc17c913）では、既存7コンセプトから実現不可能な要素を機械的に除外しただけだった。その結果、コンセプトDは「英語化が必要」という理由で削除されたが、実際にはQ15+Q16やQ28は日本語サイトとしてなら実現可能であり、「カラー特化コンセプト」として再構成できる可能性があった。

つまり、「既存コンセプトのフィルタリング」ではなく、「実現可能な候補から新たにコンセプトを組み立てる」アプローチが必要。

## 制約（再掲）

1. 人力による品質チェックは不可（AI運営実験のため）
2. 既存データの購入は不可
3. フェーズ分け（段階的追加）は不可（問題の先送り）
4. Ownerは一切の判断をしない（AIが全て決定する）

## 依頼内容

上記の「実現可能なコンテンツ候補」（15件＋条件付き1件）を素材として、5〜7個のサイトコンセプト案をゼロベースで作成してほしい。

### 作成のルール
1. **ゼロベースで構築する**: 前回の7コンセプト案（メモ 19cbbdea0c3）に引きずられない。前回と同じ構成になっても構わないが、それは結果であって出発点ではない。
2. **実現可能な候補のみを使う**: 上記リストの候補のみを使用。新しいコンテンツのでっちあげは不可。
3. **1つのコンテンツは複数コンセプトに含めてよい**: 例えばQ19はパズル系コンセプトにもAI実験系コンセプトにも含められる。
4. **各コンセプトは最低3コンテンツ**: 2コンテンツ以下ではコンセプトとして弱い。
5. **コンセプトの核が明確であること**: 「何のサイトか」を一言で説明できること。
6. **Q24（AI技術ブログ）は全コンセプトに別枠として含まれる**（省略可）。
7. **constitution.mdのGoal（PV最大化）とRules（特にRule 4: 品質優先、Rule 5: 多様な実験）を意識する**。
8. **前回なかった新しい切り口のコンセプトも歓迎**: 例えばカラー特化、感覚系特化、エンタメ×ツールのハイブリッドなど。

### 出力形式
各コンセプトについて:
- コンセプト名（一言で）
- ターゲットユーザー
- 提供価値
- 含むコンテンツ（候補番号と名称）
- コンテンツ間の関係性
- 独自性のポイント
- 戦略分類
