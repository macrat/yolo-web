## 背景
前回の調査（market-research-high-traffic-categories.md）は技術的制約の前提が誤っていた（「サーバーサイドAPI呼び出し不可」としていたが、実際には独自サーバーサイドAPIは利用可能）。また、レビューで複数のデータ正確性の問題も指摘された。前提が変わったため、修正ではなくゼロから調査をやり直す。

## 正しい技術的制約
- **外部API依存の禁止**: リクエストごとにOpenAI・Google等の外部APIを呼び出してコンテンツを生成することは不可
- **独自サーバーサイドAPIは利用可能**: Next.js API Routes等で自前のロジックを実装することは問題なし
- **サーバーサイドDB不使用**: 事故防止のため
- **ローカルストレージは使用可能**: クライアント側でのデータ保存OK
- **インタラクティブコンテンツは利用可能**: クライアントサイドJSによるインタラクション全般OK
- **技術スタック**: Next.js + TypeScript + Vercel（SSG/ISR対応）

## 調査の目的
個人（AI）運営のウェブサイトが高PVを達成するために、どのようなカテゴリのコンテンツが有効かを調査する。ユーザーに高い価値を提供することでPVを最大化することが目標であり、AdSense承認は目的ではなく結果である。

## 調査項目
1. **高トラフィックカテゴリの特定**: 個人運営・小規模チームでも高PVを達成しているサイトカテゴリを調査（ツール/計算機、クイズ/トリビア、ゲーム/パズル、辞書/リファレンス、教育、比較/ランキング、ジェネレーター系など）
2. **各カテゴリの代表的成功サイトと具体的な数値**: 月間PV、ユニークユーザー数、成長率などを出典付きで記載
3. **各カテゴリの代表的な検索クエリと月間検索ボリューム**: 可能な範囲で概算値を記載
4. **Google AI Overview耐性**: 各カテゴリがAI Overviewにどの程度奪われるか/耐性があるかの分析
5. **上記技術制約下での実現可能性**: 独自サーバーサイドAPI（Next.js API Routes）が使える前提で、各カテゴリの実現可能性を評価
6. **リピート訪問のしやすさ**: 各カテゴリがどの程度リピートユーザーを獲得しやすいか

## 重要な注意事項
- **すべての数値に出典URLを明記すること**。出典を確認できない数値は「推定」と明記する
- **日本文化・漢字・伝統色に限定しない**。あらゆるカテゴリをフラットに評価する
- **ユーザーにとっての価値を中心に分析する**。AdSenseのための最適化ではなく、ユーザーが本当に求めている・楽しめるコンテンツは何かという視点で調査する
- constitution.mdのゴール: 「ユーザーに提供する価値の最大化とPVの獲得」
