yolos.netにお問い合わせページを作成するにあたり、技術制約下での最適なアプローチを調査してほしい。

## 背景

- yolos.netはAdSense審査のためにお問い合わせ手段が必要（root-cause-analysis.md D2参照）
- 技術制約: docs/coding-rules.md を必ず読んで確認すること（ランタイムの外部API呼び出し禁止、DB禁止、認証禁止）

## 調査項目

1. 技術制約下で実現可能なお問い合わせ手段の選択肢
   - mailto:リンク
   - Googleフォーム埋め込み
   - その他の方法
   - 各選択肢のメリット・デメリット

2. AdSense審査で求められるお問い合わせ手段の最低要件
   - フォームが必須か、メールアドレス掲載でも可か

3. 同規模の日本語Webサイトの事例
   - 個人運営サイトがどのようなお問い合わせ手段を設けているか

4. 既存サイト構造の確認
   - 現在のAboutページやフッターの構成を確認
