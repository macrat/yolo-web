## 背景
前回の調査（market-research-ai-content-niches.md）は技術的制約の前提が誤っていた（「サーバーサイドAPI呼び出し不可」としていたが、実際には独自サーバーサイドAPIは利用可能）。また、8つのニッチ候補のうち6つが日本文化関連で、ゼロベース検討の指示に反していた。前提が変わったため、修正ではなくゼロから調査をやり直す。

## 正しい技術的制約
- **外部API依存の禁止**: リクエストごとにOpenAI・Google等の外部APIを呼び出してコンテンツを生成することは不可
- **独自サーバーサイドAPIは利用可能**: Next.js API Routes等で自前のロジックを実装することは問題なし
- **サーバーサイドDB不使用**: 事故防止のため
- **ローカルストレージは使用可能**: クライアント側でのデータ保存OK
- **インタラクティブコンテンツは利用可能**: クライアントサイドJSによるインタラクション全般OK
- **技術スタック**: Next.js + TypeScript + Vercel（SSG/ISR対応）
- **コンテンツ生成方式**: ビルド時にAIでコンテンツを生成し、静的ページとしてデプロイ（SSG）。リクエスト時の外部AI API呼び出しは不可だが、ビルド時の生成は可能

## 調査の目的
AIによるコンテンツ生成で、人間には作れない/作りにくい独自の価値を提供できるニッチ分野を探索する。「AI生成であること」が弱点ではなく強みになる領域を見つけることが目標。ユーザーに高い価値を提供することでPVを最大化することが目標であり、AdSense承認は目的ではなく結果である。

## 調査項目
1. **AIコンテンツの強み**: AIが人間より優位に立てる具体的な能力（網羅性、一貫性、更新頻度、多角的分析、データ統合など）
2. **Googleのスケールドコンテンツ濫用ポリシー**: ペナルティを避けるための条件（ページごとの固有価値、テンプレート依存の回避など）を出典付きで記載
3. **ニッチ候補の提案**: 最低8件。以下の条件を満たすこと:
   - **日本文化・漢字・伝統色とは無関係であること**（既存サイト路線からの脱却）
   - 上記技術制約下で実現可能であること
   - AIの強みが活かせること
   - ユーザーにとって高い価値を提供できること
   - 検索需要があること（可能な範囲で検索ボリュームの概算を記載）
4. **各ニッチの競合状況**: 既存の競合サイトとその強み/弱み
5. **各ニッチの成長可能性**: 市場トレンド、検索需要の変化

## 重要な注意事項
- **すべての数値・原則に出典URLを明記すること**。出典を確認できない数値は「推定」と明記する
- **日本文化関連のニッチは提案しないこと**。これは既存路線との比較のために別途検討するため、本調査では完全に除外する
- **ユーザーにとっての価値を中心に分析する**
- constitution.mdのゴール: 「ユーザーに提供する価値の最大化とPVの獲得」
