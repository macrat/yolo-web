## 計画に含める内容

### 誰のため・何のため
- docs/targets/手軽で面白い占い・診断を楽しみたい人.yaml のターゲットユーザー向け
- docs/site-concept.md のサイトコンセプト「笑えて、シェアしたくなる占い・診断の遊園地」に沿った新コンテンツ

### 企画・設計
リサーチ結果の案A（キャラが喋る）+ 案B（相性診断）のハイブリッドを軸に、以下を具体化せよ：
1. キャラクター設計: 何体のキャラを作るか、各キャラの個性（名前、口調、性格、テーマカラー、アイコン絵文字）
2. 占いの形式: 質問数、質問の種類、スコアリング方法
3. 結果表示: キャラによる告知演出、相性機能の有無
4. SNSシェア: シェアテキストのフォーマット

### 実装計画
既存の診断実装パターンに沿って、以下を具体化せよ：
1. 作成するファイル一覧
2. データ構造（QuizDefinition の具体的な内容）
3. UIカスタマイズ（ResultCardのキャラ演出をどう実現するか）
4. 実績システム統合手順
5. B-170（/achievementsページのTrustLevel定義追加）も含める

### 品質基準
docs/site-concept.md のセクション7「品質基準」の3軸（エンタメ性の独自性、シェア衝動、ゲーム体験としての完成度）を満たすこと。

### 注意事項
- docs/coding-rules.md を直接読んで技術制約を確認すること
- 画像は使えない。絵文字・CSS・テキストでキャラを表現する
- 既存のQuizDefinition型をなるべく拡張せずに実現する方法を検討する
- 相性機能を入れる場合、music-personality の CompatibilitySection の実装を参考にすること
- ブログ記事の計画も含めること
