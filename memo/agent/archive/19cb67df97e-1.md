# 依頼: 新規コンテンツ候補の選定

## 背景

yolos.netというWebサイトに追加する新規コンテンツの候補を、市場調査の結果から選定してください。

## 重要な制約

- **docs/targets/ は絶対に読まないでください。**
- **src/ 配下のコードを読まないでください。**
- **既存のサイトコンテンツ（ゲーム、ツール、ブログ等）は読まないでください。**
- 既存のものに引きずられず、純粋に「技術制約のもとで作れるもの」×「市場に需要があるもの」の交差点から候補を選んでください。

## サイトの技術制約

- Next.js + TypeScript の静的サイト（クライアントサイド処理中心）
- サーバーコストは最小限（APIコール等のランタイムコストは避ける）
- AI（Claude）による開発・運用（人間のowner 1名 + AIエージェント）
- 大量×高品質は不可能（Claude Opusは高コスト、安価モデルは品質低下）
- 2つの現実的戦略:
  - 「少量×高品質」: 作り込みゲーム・高機能ツール。1つ1つを丁寧に作る。
  - 「大量×低品質」: 占い・診断等、正確性が不要な分野。安価モデルで大量生成してJSON配信。

- 19cb66282a0: 占い・診断市場調査
- 19cb6641692: ゼロベース広域市場調査（特に重要: 特定テーマに限らない幅広い候補あり）

## やること

市場調査の結果から、有望なコンテンツ候補を8〜12個選定してください。

### 選定基準

各候補について以下を評価してください:

1. **技術的実現可能性**: クライアントサイドJS + 静的サイトで実現できるか。ランタイムのAPIコストが不要か。
2. **市場需要**: 検索ボリュームや競合の状況から需要が見込めるか。
3. **独自性**: 既存の競合サービスに対して、明確な差別化ポイントがあるか。具体的に何が違うのか。
4. **データ要件**: 必要なデータの量と品質管理コスト。少量のデータで成立するか、大量のデータが必要か。データの正確性の検証はどの程度困難か。
5. **戦略分類**: 「少量×高品質」と「大量×低品質」のどちらに該当するか。

### 注意事項

- 特定のテーマ（日本文化等）に偏らないでください。4件の市場調査を公平に横断評価してください。
- 最終的にある分野が多くなるのは構いませんが、それは公平な評価の結果でなければなりません。
- 各候補には必ず「なぜこの候補を選んだのか」「競合と比べてどう差別化するのか」を具体的に記載してください。
- 「大量×高品質」が必要なコンテンツは選ばないでください。それは技術制約に反します。
- 音声ファイルや画像ファイルなど、コードの外にある素材の品質管理が必要なものは、そのコストを明記してください。
