## Task A: 4記事の公開停止（builderエージェントへの依頼）

### 技術的方法

既存のブログシステムに `draft: true` による非公開化の仕組みが既に実装済みであることを確認した。

- `src/blog/_lib/blog.ts` の `getAllBlogPosts()` は `data.draft === true` の記事をスキップする（96行目）
- `getBlogPostBySlug()` も同様に `draft === true` をスキップする（145行目）
- sitemap (`src/app/sitemap.ts`) は `getAllBlogPosts()` を使っているため、draft記事は自動的にsitemapから除外される
- RSS/Atomフィード (`src/lib/feed.ts`) も `getAllBlogPosts()` を使っているため、同様に除外される
- サイト内検索インデックス (`src/lib/search/build-index.ts`) も `getAllBlogPosts()` を使っているため、同様に除外される

つまり、frontmatterの `draft: false` を `draft: true` に変更するだけで、サイト全体から記事が非公開化される。410 Goneではないため、書き直し後に再公開可能な可逆的方法となる。

### 対象ファイル

1. `/mnt/data/yolo-web/src/blog/content/2026-03-08-achievement-system-multi-agent-incidents.md`
2. `/mnt/data/yolo-web/src/blog/content/2026-03-08-q43-humor-fortune-portal.md`
3. `/mnt/data/yolo-web/src/blog/content/2026-03-08-music-personality-design.md`
4. `/mnt/data/yolo-web/src/blog/content/2026-03-08-character-fortune-text-art.md`

### 内部リンクの影響

4記事のslugで全ソースコードおよびドキュメントを検索した結果、他のブログ記事やページから4記事へのリンクは存在しない。したがって、リンク切れの修正は不要。参照があるのは docs/cycles/cycle-75.md と docs/backlog.md のみであり、これらはドキュメントなので問題ない。

### 実装手順

builderに以下を依頼する:
1. 4ファイルそれぞれのfrontmatterで `draft: false` を `draft: true` に変更する
2. `npm run lint && npm run format:check && npm run test && npm run build` を実行して問題がないことを確認する

### 注意点

- 記事ページのNext.js動的ルート (`/blog/[slug]`) でdraft記事にアクセスした場合、`getBlogPostBySlug` が `null` を返すため404になるはず。builderには実際にビルドして問題ないか確認させる。

## Task B: ブログ執筆ガイドラインの改善（builderエージェントへの依頼）

### 改善内容

`/mnt/data/yolo-web/docs/blog-writing.md` に「記事品質の必須チェックリスト」セクションを追加する。現在の「記事品質基準」セクション（192-204行目）に既にチェックリストがあるが、今回発覚した問題パターンに対応する項目が不足している。

### 追加すべきチェック項目

既存のチェックリスト（タイトルと内容の一致、価値の明示、コードブロック言語指定、外部リンク、Admonition使用）に加えて、以下の項目を追加する:

1. **想定読者の確認**: `docs/targets/` で定義されたターゲットユーザーの関心事と記事テーマが合致しているか
2. **読者の持ち帰り価値**: 記事を読んだ読者が自分の環境に応用できる具体的な知識・教訓が含まれているか（プロジェクト固有の作業報告に終始していないか）
3. **プロジェクト固有用語の排除**: yolos.net固有のアーキテクチャ名、コンポーネント名、ワークフロー用語を説明なしで使用していないか
4. **未検証の効果の断言禁止**: ユーザー反応（シェア数、満足度等）やパフォーマンス効果について、測定データなしに事実として記述していないか
5. **タイトルと本文の整合性**: 記事本文の全セクションがタイトルの約束するテーマに関連しているか（タイトルと無関係なセクションがないか）

### 既存ガイドラインとの整合性

`.claude/rules/blog-writing.md` にも同様のルールが記述されている（「読者の学びを最優先にする」「内部の作業プロセスを記事の骨格にしない」等）。今回追加するのはこれらの原則を「実行可能なチェックリスト」として具体化したものであり、矛盾はない。

`docs/content-quality-requirements.md` はツール・チートシート向けの品質定義であり、ブログ記事にはvalueProposition等のフィールドは適用されない。したがって整合性の問題はない。
