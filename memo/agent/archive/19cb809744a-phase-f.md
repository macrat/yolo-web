# Phase Fやり直し: 実現可能性フィルタリング後のコンセプト案再策定

## 背景

前回のPhase Fでは40候補（P01〜P40）をそのまま入力としてコンセプト案4案を策定した。しかし結果として4案中3案が実現可能性に重大な問題を抱えていた（メモ 19cb7f9bd0f でD評価）。

Ownerの判断: **コンセプト案作りの段階で問題があった。非現実的な候補が入力に含まれていたことが根本原因であり、推奨変更（対症療法）ではなくプロセスのやり直し（根本治療）が必要。**

## 今回のプロセス

### ステップ1: 実現可能性フィルタリング

メモ 19cb7f9bd0f の品質保証分類に基づき、40候補を以下の3群に分類する:

**A群: 高品質保証可能（9件）** — コンセプト案の中核に使える
- P33: 創作インスピレーション・ランダムプロンプト（テンプレート組み合わせ、正解なし）
- P09: 創作制約チャレンジ（お題生成、正解なし）
- P37: 俳句・短歌ツール（ルールベース処理）
- P27: 日本語テキスト変換ツール群（アルゴリズム処理）
- P04: 呼吸法・瞑想ガイド（SVGアニメーション、データ不要）
- P08: カラーメディテーション（色データ、正確性問題なし）
- P20: 色彩感覚チャレンジ（HSLスライダー、色データ）
- P23: ストーリー・プロット生成（テンプレート組み合わせ）
- P18: キャラクター・シナリオジェネレーター（テンプレートベース）

**B群: 条件付きで品質保証可能（16件）** — 条件を満たせばコンセプトに含められる
- P05, P15, P02, P11, P28, P26, P31, P07, P14, P13, P01, P10, P17, P24, P32, P35

**C群: 品質保証困難（12件）** — コンセプト案から除外する
- P29, P40, P34, P16, P25, P21, P30, P03, P12, P36, P22, P39

### ステップ2: コンセプト案の再策定

**入力**: A群9件 + B群16件 = 25候補 + 既存コンテンツ（P06等）
**除外**: C群12件（入力に含めない）

この25候補+既存コンテンツを使って、コンセプト案を3〜4案策定してください。

### コンセプト案策定のルール

1. **コンセプトの中核コンテンツは必ずA群から選ぶこと**: 中核（初期スコープで最優先のもの）にB群を据えない
2. **B群は補完的な役割として組み込むのはOK**: ただし条件（何が満たされれば品質保証できるか）を明記する
3. **C群の候補は一切含めないこと**: 「将来の拡張候補」としても記載しない
4. **既存コンテンツの扱いは公平に**: 既存ゲーム（漢字カナール、四字キメル、ナカマワケ、イロドリ）やツール群は既に稼働しているので、新規DB構築の品質問題とは無関係。公平に扱う
5. **各案の差別化を明確に**: 似たような案を複数作らない。各案が異なるビジョンを提示すること

### 評価基準

各コンセプト案を以下の基準で自己評価してください:

| 基準 | 説明 |
|------|------|
| 来訪者への価値 | 来訪者はこのサイトをなぜ使うのか？他にはない何が得られるのか？ |
| 実現可能性 | A群中心で構成されているか？品質を構造的に保証できるか？ |
| ターゲット規模 | 想定利用者はどの程度いるか？ニッチすぎないか？ |
| コンセプトの一貫性 | サイト全体として統一感があるか？ |
| 差別化 | 既存の競合サイトとどう異なるか？ |
| constitution.md準拠 | 5ルール全てに適合しているか？ |

### 前回の教訓

前回のPhase Fの問題点を踏まえて、以下に注意:
- 「品質維持の現実性」を甘く評価しない。漢字80字DB構築の失敗を常に念頭に置く
- デイリーゲーム形式は大量データが前提なので、中核に据える場合はデータ生成の実現可能性を厳密に検討する
- 「有限データセット」と「高品質DB構築の容易さ」を混同しない
