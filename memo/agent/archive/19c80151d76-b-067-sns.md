## 依頼内容

サイクル22の最初のタスクとして、SNS拡散最適化のベストプラクティスを調査してください。
調査結果はB-065（シェアボタン設置）とB-066（OGP最適化）の実装方針の基盤となります。

## 調査項目

### 1. シェアボタンのUI/UXベストプラクティス
- どのSNSに対応すべきか（日本向けサイトとして: X/Twitter, LINE, Facebook, はてブ等）
- ボタンの配置場所（記事上部・下部・固定サイドバー等）
- コンテンツタイプ別の最適なシェア方法（ブログ記事 vs ツール vs ゲーム結果）
- Web Share API のフォールバック戦略
- ライブラリ不要のシンプルな実装方法（intent URL等）

### 2. OGP/メタ情報のベストプラクティス
- og:title, og:description, og:image の最適な設定方法
- Twitter Card (summary vs summary_large_image) の使い分け
- コンテンツタイプ別のOGP最適化（ブログ、ツール、ゲーム、辞書等）
- Next.js App Router での generateMetadata ベストプラクティス
- OGP画像の推奨サイズ・フォーマット

### 3. 日本のSNS事情
- 日本で主要なSNSのシェア率・ユーザー数
- 各SNSでのシェアURL仕様（intent URL形式）
- はてなブックマークの対応は必要か

## 現在のサイト状況

- **既にシェアボタンがある箇所**: ゲーム結果画面（漢字カナール、四字キメル、仲間分け、イロドリ）、クイズ結果画面
- **シェアボタンがない箇所**: ブログ記事、ツール、チートシート、辞書、色辞典
- **対応SNS（既存）**: X(Twitter)、LINE、コピー、Web Share API
- **OGP**: ルートに基本設定あり。各ページはgenerateMetadataで設定。opengraph-image.tsxで汎用OGP画像生成。
- **技術スタック**: Next.js 15 App Router, TypeScript
