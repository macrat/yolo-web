B-159（ツール・チートシートの個別ページ分割）による変更前後のバンドルサイズを計測してください。

## 目的
ブログ記事にバンドルサイズの実測データを追加し、記事の品質を向上させる。

## 計測方法

### 変更後（現在の状態）
現在の状態で `npm run build` を実行し、以下を記録:
- ツールページ（例: /tools/char-count）のFirst Load JSサイズ
- チートシートページ（例: /cheatsheets/regex）のFirst Load JSサイズ
- 全体のビルド出力サマリー

ビルド出力の末尾に各ルートのサイズが表示されるので、それを使ってください。
ただし、すでにビルドは完了しているので `/tmp/claude-1000/-mnt-data-yolo-web/tasks/bq4v3n26f.output` にビルド出力が保存されています。まずこれを確認してください。

### 変更前
git stash で現在の変更を退避するか、変更前のコミット（48648e7）をチェックアウトして `npm run build` を実行し、同じ情報を記録。

**重要**: 変更前の状態でビルドする場合は、必ず現在の状態に戻すこと。git worktree を使うのが安全です。

- 変更前後のバンドルサイズ比較表（ルート別）
