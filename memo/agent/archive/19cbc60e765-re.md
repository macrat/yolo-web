# 競合調査結果: 画像リサイザーツール

## 調査概要

yolos.net の image-resizer（/tools/image-resizer）について、日本語・英語の主要競合と比較し、保持/削除の推奨を評価した。

---

## 1. yolos.net 現在のツールの機能

- 対応入力フォーマット: PNG, JPEG, GIF, WebP（最大20MB）
- 出力フォーマット: PNG, JPEG, WebP
- リサイズモード: ピクセル指定・パーセント指定
- アスペクト比ロック機能: あり
- 品質スライダー（JPEG/WebP用）: あり
- バッチ処理: **なし（1枚のみ）**
- サーバー送信: **なし（ブラウザ完結）**
- アカウント登録: 不要
- プレビュー: 元画像・変換後画像ともに表示

**特筆すべき不足機能**: バッチ処理、圧縮特化機能、AVIF/SVG対応なし

---

## 2. 主要競合サイト

### 英語圏

#### Squoosh（squoosh.app）— Google製
- **ブラウザ完結**: あり（WebAssembly使用、画像はデバイス外に出ない）
- 対応フォーマット: JPEG, PNG, WebP, AVIF, JPEG XL, OxiPNG, MozJPEG, SVG
- リサイズ機能: あり（リサイズ+圧縮の同時処理）
- バッチ処理: **なし**
- 差別化点: Before/Afterスライダーによる視覚的比較、WebAssemblyによる高速処理、Google製の信頼性
- 評価: ブラウザ完結かつGoogle製。「ブラウザ完結でプライバシー重視」という切り口ではSquooshが圧倒的優位。

#### Bulk Resize Photos（bulkresizephotos.com）
- **ブラウザ完結**: あり（「プライベート — アップロードなし」を明示）
- バッチ処理: **あり（無制限）**
- リサイズモード: %指定、幅×高さ、ファイルサイズ指定、幅指定、高さ指定、最長辺指定（6種類）
- 対応フォーマット: JPG, PNG, WebP
- 評価: ブラウザ完結かつバッチ処理対応。yolos.netより明確に高機能。

#### TinyPNG（tinypng.com）
- **サーバー処理**: あり（サーバーに送信して圧縮）
- バッチ処理: 無料で20枚まで
- 特化機能: PNG/JPEG/WebPの圧縮最適化
- リサイズ: 有料版のみ
- 評価: 圧縮特化。知名度・信頼性が高く、リサイズ目的のユーザーにはやや不向き。

#### iLoveIMG（iloveimg.com）
- **サーバー処理**: あり（サーバーにアップロード、HTTPS暗号化、2時間後自動削除）
- 日本語対応: あり
- バッチ処理: 無料30枚、プレミアム120枚
- 対応フォーマット: JPG, PNG, SVG, GIF
- その他機能: 比率/ピクセル指定、アスペクト比維持、拡大防止オプション
- Google Drive/Dropboxからのインポート: あり
- 評価: 機能豊富・多機能スイート（圧縮、変換、切り抜きも提供）。日本語対応あり。

### 日本語圏（独立サービス）

#### さるわかくん（saruwakakun.com/tools/image-resize/）
- **ブラウザ完結**: あり（「サーバーなどにも送信されない」を明示）
- バッチ処理: **あり**（複数枚一括、ZIPで一括ダウンロード）
- 対応フォーマット: JPEG, PNG, GIF
- リサイズモード: 最大幅/最大高指定（アスペクト比維持）
- 品質: 低・中・高・最高の4段階
- 評価: 日本語・ブラウザ完結・バッチ処理の三拍子が揃う。yolos.netの直接的な競合。

#### Adobe Express（adobe.com/jp/express）
- **サーバー処理**: あり（Adobeのクラウド処理）
- 対応フォーマット: JPEG, PNG, WebP, HEIC（最大40MB）
- バッチ処理: 1枚のみ（「1回につき1枚」と明示）
- AI機能: AIリサイズ機能あり（プレミアム）
- 評価: Adobeブランドの強力な競合。無料で使えるが機能は制限あり。

#### Simple Image Resizer（simpleimageresizer.com）
- **サーバー処理**: あり（HTTPS、24時間後削除）
- バッチ処理: 無料3枚、プレミアム10枚
- 評価: サーバー処理のため「ブラウザ完結」優位性あり。ただし機能は平凡。

## 4. evaluation-rubric.md 独自性スコア評価

### 独自性（日本語Web市場での競合状況）

判定: **スコア 1（VETO）**

根拠:
- 日本語対応かつブラウザ完結の同種サービスが複数確認される（さるわかくん、Bulk Resize Photos日本語版）
- その中でも「バッチ処理+ブラウザ完結+日本語」を同時に満たすさるわかくんがyolos.netより明確に高機能
- ブラウザ完結という差別化軸はSquoosh（Google製）が既に実現済みで、かつSquooshはWebAssembly・高画質・多形式対応（AVIF含む）で圧倒的
- yolos.netの現ツールは「ブラウザ完結・1枚リサイズ・PNG/JPEG/WebP対応」という機能セットだが、この機能は大手（Squoosh、Bulk Resize Photos、さるわかくん）が全て提供済み
- 新規ユーザーが能動的にyolos.netのimage-resizerを選ぶ理由が存在しない

evaluation-rubric.mdのスコア1定義:「大手が圧倒的に市場を支配しており、新規参入の余地がほぼない」に該当。

### その他の軸（参考）

- **需要（スコア4程度）**: 「画像 リサイズ」「画像 サイズ変更」は検索需要が高い。ただし独自性スコア1のため需要は意味をなさない。
- **継続性（スコア4）**: 使う機会は定期的に発生するツールで継続性は高い。ただし他サービスに移行されやすい。
- **実装可能性（スコア5）**: 現状実装済み。追加コストは不要。
- **品質達成可能性（スコア5）**: アルゴリズムで正確性が保証されるロジックツール。

**総合スコア（参考）**: (1 + 4 + 5 + 4 + 5) / 5 = 3.8 — ただし独自性スコア1のため **VETOフラグ付与**

## 5. 結論と推奨

### 推奨: **削除（410エラー）**

#### 根拠

1. **「ブラウザ完結」という差別化軸が競合に潰されている**: yolos.netが主張する最大の強み（プライバシー・サーバー送信なし）は、Squoosh（Google製）・Bulk Resize Photos・さるわかくんが全て実現済み。差別化にならない。

2. **機能で明確に劣位**: 競合はバッチ処理（複数枚一括）を提供しているが、yolos.netは1枚のみ。日本語ユーザーが実際のユースケース（SNS投稿用に複数枚一括処理等）で使う場合、競合に流れることが確実。

3. **VETOフラグ**: evaluation-rubric.mdの独自性スコアが1に該当し、ルーブリック規定のVETOフラグを受ける。「致命的な問題があり、そのままでは採用不可」の状態。

4. **改善余地の限界**: 仮にバッチ処理・AVIF対応・Before/After比較を追加したとしても、Squoosh・Bulk Resize Photosがその改善後の機能セットを既に提供しており、後追い改善は意味をなさない。

5. **AdSense「有用性の低いコンテンツ」への影響**: 大手が同一機能を無料提供している中での劣位ツールは、Googleの観点で「代替品より価値が低い」と判断されやすく、AdSense審査に悪影響を与えるリスクがある。

#### 削除時の注意

- **410エラー（Gone）を使用すること**（301リダイレクトは不可、owner指示に基づく）
- related-slugsから image-resizer を削除（image-base64, base64, qr-code が参照している）

## 参考: 保持を検討できる条件（現状では該当しない）

以下の差別化があれば再評価の余地があるが、現状では実現していない:
- AVIF/HEIC/TIFF等の希少フォーマット対応で専門家向け差別化
- リサイズ+圧縮+EXIF削除+WebP変換を一括で行う「一括最適化」機能でSquooshを上回るUX
- 日本語圏向けの特殊用途（SNSごとの最適サイズプリセット等）での特化

ただしこれらの追加開発コストと競合との差別化持続可能性を考えると、リソースをより独自性の高いコンテンツ（ゲーム等）に集中させる方が全体戦略として優れる。
