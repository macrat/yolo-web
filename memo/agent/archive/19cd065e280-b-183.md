# B-183計画 追加修正依頼: サニタイズ設計の根本的な問題

## 問題（Owner指摘 メモ 19cd055e935）

現在の計画ではサニタイズで「クォート内文字列を除去」するが、これだと git checkout "src/" のようにクォートされたファイルパスも除去されてしまい、パス検知が効かなくなる。

すり抜けるコマンド例:
- git checkout "src/"
- git checkout "docs/cycles/cycle-75.md"
- git checkout 'page.tsx'

## 修正方針

サニタイズの対象を見直す必要がある。以下のいずれかのアプローチを検討して採用すること:

**案A: gitコマンドの場合はサニタイズしない**
- コマンドがgitで始まる場合: サニタイズせずにそのままパターン検査（クォートを除去してから検査するのはOK、内容まで除去してはいけない）
- コマンドがgitで始まらない場合: コマンド全体をスキップ（gitコマンドでなければチェック不要）

**案B: サニタイズの代わりに、コマンドの先頭がgitかだけを確認**
- そもそもフックの目的は「git の破壊的操作をブロック」なので、gitコマンド以外は早期リターンでよい（既に実装済み）
- 問題はヒアドキュメント等の中に「git」が含まれるケースだが、これはコマンド先頭のgitだけを正確に判定すれば解決する

## 要求
- 修正版計画v2にこの問題への対応を反映すること
- テストケースに git checkout "src/" のようなクォート付きファイルパスを追加すること（ブロックされるべき）
- 修正版は計画全体を含めること
