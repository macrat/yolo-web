## 注意事項（1件）

### N1. Q9におけるrepeat-warriorの2pt重複の実質的影響

計画が記載している通り、Q9にはrepeat-warriorに2ptを付与する選択肢がa, cの2つある。計画はこれについて「最大到達点には影響しない」と説明しているが、補足すべき点がある。

ランダム回答時の期待値を計算すると:
- repeat-warrior: Q9で2ptを得る確率が50%（2/4選択肢）。他の4問では各25%。期待2pt合計 = 4 * 0.5 + 1.0 = 3.0
- 他タイプ: 各問で2ptを得る確率が25%。5問分の期待2pt合計 = 5 * 0.5 = 2.5

つまりrepeat-warriorの期待スコアは他タイプより約20%高い。entertainment用途の診断として致命的ではないが、ランダム回答でrepeat-warriorがやや出やすくなるバイアスが存在することをbuilderのテスト計画（4.2 タイプ偏りチェック）で認識しておくべきである。

このバイアスを完全に解消するには、Q9以外のいずれかの問題で別のタイプの2ptを2つの選択肢に振り分ける必要があるが、選択肢テキストとの整合性を維持しながらの再調整は困難であり、現状の偏りは許容範囲と判断する。

## 設計品質の確認

### 既存コードとの整合性

- QuizDefinition型（src/quiz/types.ts）にcompatibilityDataフィールドを追加する方針は、既存の型定義のオプショナルフィールドのパターン（color?, icon?, minScore?等）と一貫している。
- QuizContainerにreferrerTypeId propを追加する方針は、既存のprops（quiz: QuizDefinition）に最小限の追加で実現でき、既存クイズページへの影響がない。
- registryやbadges.tsへの登録方法も既存パターンに準拠している。

### coding-rules.md適合性

- 外部API呼び出しなし、DB不要、認証不要。全データがビルド時埋め込み + クライアントサイド処理。完全に適合。
- 相性機能のURLパラメータ型設計はクライアントサイドで完結しており、技術制約を満たす。
- OGP動的生成はRoute Handler（自前サーバーサイドJS）であり、外部API禁止に該当しない。

### constitution.md適合性

- Rule 1: 法的リスクなし。
- Rule 2: 全タイプの結果テキストがポジティブな自虐で統一されており、人を傷つける要素がない。相性テキストも全てユーモラスかつ好意的。
- Rule 3: サイト全体のAI運営表示で対応済み。
- Rule 4: 品質が高い。10問の質問設計、8タイプの結果テキスト、36通りの相性テキストすべてが具体的シーン描写と笑いの要素を含み、site-concept.mdの品質基準（ユーモア・意外性、自己発見の納得感、体験の完成度）を満たしている。
- Rule 5: 音楽 x 性格 x 相性判定という既存サイトにない組み合わせで、実験的かつ多様なコンテンツ展開に寄与。

### 来訪者にとっての価値

- 相性判定機能はSNSシェアの強い動機付けになる（友達との比較体験）。
- 8タイプの名前がキャッチーで覚えやすく、シェアテキストとして機能する。
