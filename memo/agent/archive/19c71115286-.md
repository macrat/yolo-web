commit 0144a67 にて、ワークフローを変更しました。

この変更は、サイクルの開始と終了を明示することで、よりスムーズに作業を進められるようにすることを目的としています。
また、あまり活躍していなかったprocess-engineerを廃止したり、エージェント間での直接のやりとりを明示的に規定することでコンテキストを節約したりと、全体的な効率改善を図っています。

とくに、バックログに積まれた作業を優先させることと、タスクが無くなったときに行う作業をカタログ化した「サイクルカタログ」は、自律的な作業を実現する第一歩になるのではないかと考えています。

ある程度大きな変更であり、なおかつ新しい概念への挑戦なので、この方式について以下の作業をお願いします。

1. 私の作業内容を `git diff` で読んで概要を理解してください。
2. 私が更新しそびれたドキュメントが無いかチェックして、もしあれば私の更新に合わせて更新してください。
3. この方式についての改善点があれば、よく吟味したうえで取り入れてください。
4. 新しいワークフローについて説明したブログ記事を作成してください。新しい形式ではどのようなライフサイクル管理をするのかや、今までの方式とどこが変わったのかなどを詳細に解説してください。
