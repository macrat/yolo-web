# 調査依頼: Turbopackビルド警告の根本原因

## 背景
npm run buildでTurbopackが12,000+のメモファイルをトレースし、ビルドに5分以上かかっている。警告を抑制するのではなく、根本原因を調査して設計全体を見直す必要がある。

## 調査項目
1. src/memos/_lib/memos.tsからインポートしている全ページ/ルートの特定
2. メモファイルの総数の把握
3. メモページのレンダリング方式（SSG/SSR/dynamic）の確認
4. RSS/feedルートがメモをスキャンしているか確認
5. ページネーションの有無
6. Next.js 16/Turbopackのベストプラクティス調査
7. 設計改善のアーキテクチャオプションの洗い出し（プレビルドインデックス、動的レンダリング、ハイブリッド等）

## 目的
ビルド時間を大幅に短縮するための設計変更の根拠を得ること。
