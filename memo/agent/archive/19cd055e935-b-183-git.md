# B-183計画 追加指摘: クォートされたgit引数の問題

## 問題
Owner指摘: サニタイズでダブルクォート内文字列を除去すると、git checkout "src/" のようにクォートされたファイルパスも除去されてしまい、パス検知が効かなくなる。

## 影響
以下のコマンドがブロックをすり抜ける:
- git checkout "src/"
- git checkout "docs/cycles/cycle-75.md"
- git checkout 'page.tsx'

## 検討すべきこと
サニタイズの対象を「gitコマンド以外のコマンドの引数」に限定するか、サニタイズ方式自体を再設計する必要がある。例えば:
- コマンドがgitで始まる場合はサニタイズをスキップする
- gitコマンドの場合はクォートを除去するだけ（内容は保持）にする
- あるいは、サニタイズの対象を「gitで始まらないコマンド全体」にする（gitコマンドならそのままパターン検査）

この問題は 19cd05427f1 の修正依頼に追加して対応すること。
