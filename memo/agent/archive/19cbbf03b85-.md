# 調査依頼: 四字熟語きめる・仲間分けのデータ要件と調達可能性

## 背景
コンセプトA（デイリーパズル・ステーション）の妥当性を検証するため、既存ゲーム2種のデータ調達可能性を調査する。

## 調査内容

### yoji-kimeru（四字熟語きめる）
1. src/app/games/yoji-kimeru/ のソースコードからデータ構造を特定
2. 現在101語。デイリー形式には最低365語が必要。
3. 四字熟語のオープンデータの存在・フィールド内容・ライセンス
4. 自力整備の場合のコスト見積もり

### nakamawake（仲間分け）
1. src/app/games/nakamawake/ のソースコードからデータ構造を特定
2. 現在の問題数を確認
3. アルゴリズム生成の可能性（語の意味的関連性の自動判定）
4. 半自動生成パイプラインの可能性（ビルド時LLM活用。技術制約は .claude/rules/coding-rules.md 参照）
5. 365問分のデータ用意のための具体的な工数見積もり

## 注意
- ソースコードに基づく事実ベースの調査。推測ではなく根拠を示すこと。
