## 依頼概要

ownerから「日本文化に引っ張られすぎている」というフィードバックを受けた。既存サイト（yolos.net）の現在のコンテンツや方向性と**一切関係のない**、完全に新しいサイト方向性を4つ提案してほしい。

これらの4案と、既存路線の延長（日本文化+AI）を合わせた5案で比較検討を行う。

## 重要な制約

1. **既存サイトの資産を一切考慮しないこと**: 漢字、四字熟語、伝統色、ツール、チートシート、デイリーゲーム、辞典 — これらの既存コンテンツは忘れること。白紙から考える。
2. **constitution.mdのゴールは守ること**:
   - 来訪者にとって有用または楽しいサイトを作り、PV数を最大化する
   - 品質を量より優先する
   - AI運営であることを訪問者に通知する（削除不可）
   - 創造的なアイデアで多様なことを試みる
3. **AIが運営するWebサイト**という前提は変えられない（AIエージェントが企画・開発・運営する）
4. **日本語サイト**である必要はない（英語や多言語も選択肢）
5. **技術スタック**: Next.js + TypeScript + Vercelでホスティング。これは変更可能だが、変更コストは考慮すべき。

## 求めるアウトプット

### 4つの新方向性案

各案について以下を調査・記載すること:

1. **コンセプト**: サイトが何を提供するのか（一言で説明）
2. **ターゲットユーザー**: 誰のためのサイトか
3. **提供価値**: なぜユーザーがこのサイトを使うのか
4. **PV最大化の根拠**: なぜこの方向性がPVを伸ばせるのか（市場規模、検索需要、リピート性など）
5. **AIの強みの活用**: AIが運営するからこそ実現できる価値は何か
6. **競合状況**: 主要な競合と差別化ポイント
7. **実現可能性**: 技術的・運用的に実現可能か（Next.js + Vercelの制約内で）
8. **リスクと課題**: 最大のリスクは何か

### 案の多様性

4つの案はできるだけ異なるカテゴリから選ぶこと。例えば:
- コンテンツ型（読み物・情報提供）
- ツール型（実用的なWebアプリ）
- コミュニティ型（ユーザー参加・UGC）
- エンタメ型（ゲーム・娯楽）
- 教育型（学習・スキル向上）
- データ型（データ分析・可視化）

ただし上記は例示であり、これに縛られる必要はない。本当にPVを最大化できるアイデアを自由に考えること。

## 調査のアプローチ

1. まず「AIが運営するWebサイトでPVを最大化する」という目標に対して、世界中の成功事例やトレンドをWeb検索で調査する
2. 日本語圏に限定せず、英語圏・グローバルの市場も視野に入れる
3. 2025-2026年のWebトレンド、AI活用トレンドを踏まえる
4. 各案に対して、具体的な競合サイトとその規模を調べる
5. **既存サイトの資産やスキルにバイアスされないよう注意する**
