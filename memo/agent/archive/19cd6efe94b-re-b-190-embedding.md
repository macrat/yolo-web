# 実験結果: 入力テキストパターン別のEmbedding品質比較

## 実験概要

モデル: `sirasagi62/ruri-v3-30m-ONNX`（@huggingface/transformers経由）
ベクトル次元: 256次元、normalize=true
評価指標: 同グループ内コサイン類似度平均（Within）、異グループ間コサイン類似度平均（Between）、分離度（Separation = Within - Between）

テストグループ（5グループ + 無関係5字）:
- 水系: 海, 川, 池, 湖, 泳, 波
- 火系: 火, 炎, 燃, 灯, 熱
- 植物系: 花, 草, 木, 林, 森
- 人体系: 手, 足, 目, 耳, 口
- 感情系: 喜, 怒, 悲, 楽, 愛
- 無関係（対照）: 机, 鉄, 空, 食, 走

## パターン定義

- パターンA: 漢字1文字のみ（"海"）
- パターンB: meaningsのみ（"sea ocean"）
- パターンC: 漢字 + meanings（"海 sea ocean"）
- パターンD: 漢字 + 部首 + meanings（"海 水 sea ocean"）
- パターンE: 漢字 + 読み + meanings（"海 カイ うみ sea ocean"）
- パターンF: 例語のみ（"海岸 海外 海底"）
- パターンG: 漢字 + meanings + 例語（"海 sea ocean 海岸 海外"）
- パターンH: 日本語説明文（「海」はsea oceanを意味する漢字）
- パターンI: 漢字 + meanings + 読み + 例語（全情報結合）

## 実験結果（分離度順）

| パターン | Within | Between | Separation |
|---------|--------|---------|------------|
| **D**   | 0.8636 | 0.8080  | **0.0556** |
| G       | 0.8315 | 0.7885  | 0.0430     |
| C       | 0.8463 | 0.8038  | 0.0426     |
| B       | 0.8269 | 0.7865  | 0.0404     |
| I       | 0.8335 | 0.7949  | 0.0386     |
| E       | 0.8458 | 0.8094  | 0.0364     |
| A       | 0.8929 | 0.8600  | 0.0329     |
| H       | 0.8739 | 0.8419  | 0.0320     |
| F       | 0.8434 | 0.8117  | 0.0317     |

## ベストパターン: D（漢字 + 部首 + meanings）

### パターンDの入力例

```
海  → "海 水 sea ocean"
火  → "火 火 fire"
花  → "花 艸 flower"
手  → "手 手 hand"
喜  → "喜 口 rejoice take pleasure in"
机  → "机 木 desk table"
```

### パターンDのグループ別Within類似度

| グループ | Within |
|---------|--------|
| 火系    | 0.9043 |
| 植物系  | 0.8785 |
| 水系    | 0.8689 |
| 感情系  | 0.8380 |
| 人体系  | 0.8254 |

## 考察

### なぜパターンDが最良か

- **部首の追加が意味の絞り込みに寄与**: 同じグループの漢字は部首を共有する傾向（水系: 氵/水, 植物系: 艸/木 など）。部首情報がベクトル空間での凝集を強化する。
- **最小限の情報量**: パターンD（漢字+部首+meanings）はパターンI（全情報）より分離度が高い。余分な情報（読み、例語）はノイズになる。
- **なぜ漢字1字（パターンA）が最低水準か**: Within=0.8929と高いが、Between=0.8600も高く、全体的に類似度が高すぎて分離できていない。モデルが1文字の日本語文字を意味的に区別しにくいと考えられる。

### 2位・3位（G, C）との比較

| 順位 | パターン | 特徴 | Separation |
|-----|---------|------|------------|
| 1位 | D | 漢字+部首+meanings | 0.0556 |
| 2位 | G | 漢字+meanings+例語 | 0.0430 |
| 3位 | C | 漢字+meanings | 0.0426 |

DとCの差はほぼ部首情報の有無（+0.013）。部首はシンプルな追加情報だが分離度への貢献が大きい。

## 推奨事項

### B-190（漢字関連機能）への適用

**推奨入力フォーマット**: `${character} ${radical} ${meanings.join(" ")}`

- 部首がcharacterと同じ場合（火→"火 火 fire"）も重複させて問題なし
- meaningsは全件結合（スペース区切り）
- 読み・例語は不要（ノイズになる）

### 留意点

- 分離度の差は小さい（最良0.0556 vs 最低0.0317）。パターン選択は重要だが劇的な違いではない
- 感情系・人体系のWithin類似度が相対的に低い（0.82〜0.84）。意味的な近さの定義自体が曖昧なグループは、どのパターンでもEmbeddingの凝集が弱い
- 本実験はruri-v3-30m-ONNXに特化した結果。他モデルでは最適パターンが変わる可能性がある
