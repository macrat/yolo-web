# B-191 タスクA: GA4基盤+コアイベント統合 - builder依頼

## 依頼内容

計画メモ 19cd81d6362 および改訂メモ 19cd8217f7b に従い、GA4イベントトラッキングの基盤と主要イベントの統合を実装してください。

## 作業手順

以下のメモを必ず最初に読み、指示に従ってください:
- 計画: `npm run memo -- read 19cd81d6362`
- 改訂: `npm run memo -- read 19cd8217f7b`（B-191変更点セクション）
- 調査結果: `npm run memo -- read 19cd81a50a5`

### ステップ1: 基盤整備

1. `npm install -D @types/gtag.js` で型定義をインストール
2. `src/lib/analytics.ts` を新規作成（5つのtrack関数 + sendGaEventヘルパー）
3. `src/lib/__tests__/analytics.test.ts` を新規作成

### ステップ2a: ゲーム完了イベント（level_end のみ、level_startは送信しない）

4つのGameContainer.tsxにtrackContentEnd呼び出しを追加:
- irodori: success = true（常に完走）
- kanji-kanaru: success = (gameState.status === 'won')
- nakamawake: success = (gameState.status === 'won')
- yoji-kimeru: success = (gameState.status === 'won')

**重要**: ゲームではlevel_startは送信しない（ページリロード時の重複問題回避）。

### ステップ2b: クイズ/診断（level_start + level_end）

QuizContainer.tsxに:
- handleStart内でtrackContentStart
- recordPlay呼び出し箇所でtrackContentEnd
- contentType: quiz.meta.type === 'personality' ? 'diagnosis' : 'quiz'
- contentId: 'quiz-' + quiz.meta.slug

### ステップ2c: 実績アンロック

AchievementProvider.tsxのrecordPlay関数内、result.newlyUnlockedループでtrackAchievementUnlock呼び出し。

### ステップ2d: サイト内検索

useSearch.tsのperformSearch関数内で、検索実行後にtrackSearch呼び出し。空文字列は除外。

### 最終確認

作業完了後、`npm run lint && npm run format:check && npm run test && npm run build` が通ることを確認してください。
