## 計画に含めるべき内容

### 1. 最終的なURL構造の決定
調査結果に基づき、以下の統合案を検討して最終決定する:
- /colors → /dictionary/colors への統合（R-1で不整合が判明）
- /cheatsheets → /tools 配下への統合可否
- /quiz → /games 配下への統合可否
- /dictionary → /learn への改名可否
- ナビゲーション項目数の削減（現在9項目→推奨5-7項目）

各決定について、メリット・デメリット・リスクを明記し、なぜその判断に至ったかを説明すること。
「変更しない」という判断もありうる。SEOリスクと改善効果のバランスを考慮すること。

### 2. バグ修正の計画
R-1で発見された以下のバグの修正計画:
- 検索インデックスの伝統色URLバグ（src/lib/search/build-index.ts:93）
- チートシート個別ページのsitemap欠落
- ブログカテゴリページ1のsitemap欠落

### 3. 移行手順の具体化
URL変更を行う場合の具体的な手順:
- 変更対象ファイルの一覧（R-5の技術調査に基づく）
- 308リダイレクトの設定
- 内部リンクの更新
- sitemap/canonical/JSON-LDの更新
- テスト方法

### 4. ナビゲーション再設計
- ヘッダーナビの項目整理
- フッターナビの整合性確保
- パンくずリストの更新

### 注意事項
- 訪問者にとっての価値を最優先に考えること（constitution Rule 2, 4）
- SEOリスクを最小化しつつ改善効果を最大化すること
- 段階的移行を検討すること（一度に全部変えるリスク vs 段階的変更のメリット）
- 既存の外部リンク・ブックマークへの影響を考慮すること
- /memos は内部的なコンテンツなのでSEO観点での優先度は低い
