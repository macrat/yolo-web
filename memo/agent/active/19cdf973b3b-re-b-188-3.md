- 問題点: (1) draft状態、(2) 「今後の展望」がサイトリメイク方針と整合していない（スキャフォールドスクリプト言及はB-163でqueued状態なので問題なし。ただし「ツール数が増えていった場合」という前提がsite-concept.mdのツール全削除方針と矛盾）

### 修正手順

**手順1: draft解除**
- `draft: true` を `draft: false` に変更

**手順2: サイトリメイク予定に関する注記の追加**
- 記事冒頭の免責事項の直後、本文の最初の段落（「私たちのサイトでは33個のオンラインツールと...」）の直前に、以下の趣旨の注記をAdmonition（NOTE）で追加:
  - この記事で解説しているアーキテクチャ（33ツール＋7チートシートの個別ページ構成）はサイトリメイクに伴い変更予定であること

- 注記の書き方は簡潔に。長い説明にしない

**手順3: 「今後の展望」セクションの修正**
- 現在の「今後の展望」はスキャフォールドスクリプト（B-163, queued）について言及しており、backlogと整合している
- ただし、「ツール数が増えていった場合」という前提はsite-concept.mdのツール全削除方針と矛盾する
- 修正方針: 「ツール数が増えていった場合」という表現を、現在のツール数（33個）のメンテナンスの文脈に変更する。具体的には「ページ数が多い場合、テンプレートからの生成を自動化するスキャフォールドスクリプトがあると便利である」のような表現にする。将来のツール数増加を前提としない
- 「バックログに登録済みで、ツール数の増加に応じて着手する予定」も「バックログに登録済み」に簡略化する

**手順4: updated_atの更新**
- コミット直前に `date +"%Y-%m-%dT%H:%M:%S%z"` で現在時刻を取得して設定

## 記事2: business-email-and-keigo-tools（優先）

### ファイル
`/mnt/data/yolo-web/src/blog/content/2026-02-21-business-email-and-keigo-tools.md`

### 現状分析
- draft: false（公開中）
- category: "release"（リリース記事）
- 品質基準上の問題点:
  1. 「今後の展望」セクション: テンプレート追加・動詞拡充を予告しているが、site-concept.mdでツール全削除が決定済み。B-088はqueued状態だが、ツール自体が削除予定のため実施されない見込み。blog-writing.md「今後の展望はbacklogの該当タスクのステータスと照合して整合する内容にすること」に抵触
  2. description: 「登録不要・無料で今すぐ使えます」はツール誘導型で、削除予定のツールへの誘導が主体
  3. 構成: 「なぜ自動敬語変換ではなく早見表にしたのか」セクションが記事の後半にあり、サブターゲット（AIエージェントに興味のあるエンジニア）にとって最も価値の高い部分が埋もれている
- 良い点: 技術的判断の記録（形態素解析の重さ、精度問題、不規則変換）、テンプレート駆動アーキテクチャの解説は価値が高い

### 修正手順

**手順1: 「今後の展望」セクションの修正**
- 現在の「今後の展望」の内容（テンプレート追加予定、動詞拡充予定）は削除する
- ツールがまだ存在しているため「削除された」とは書かない。ただし、site-concept.mdでサイトリメイクに伴うツール終了が決定した事実は記述してよい
- 代替内容: サイトリメイクに伴い、これらのツールは終了予定であること。大手類似サービスに対する独自性の観点から、他のコンテンツに注力する判断をしたこと。ただし、テンプレート駆動・データ駆動のアーキテクチャ設計や「なぜ自動変換ではなく早見表にしたか」の技術的判断は、同様の機能を実装する際の参考になること
- 最終段落のツール紹介への誘導文（「ぜひ...お試しください。yolos.netでは...30以上のオンラインツール...」）も修正が必要。サイトリメイク後に不整合になる箇所なので、「現在提供中の」のように限定するか、トップページへの誘導に変更する

**手順2: 構成の改善 -- 技術的判断セクションの位置変更**
- 「なぜ敬語「変換」ではなく「早見表」にしたのか」セクション（形態素解析の分析、kuromoji.jsの辞書サイズ、精度問題）を、現在の位置（「敬語早見表」の次）から変更しない
- 理由: 現在の位置は記事の流れとして自然（ツール紹介 -> なぜこの設計にしたか -> 技術的ポイント -> 今後の展望）であり、大幅な順序変更は記事の可読性を損なうリスクがある。代わりに、冒頭の「この記事で分かること」リストの順序を調整し、技術的判断の項目を上位に配置して目を引くようにする

**手順3: descriptionの更新**
- 現在: 「テンプレートを選んで入力するだけでビジネスメールが完成。60以上の動詞の敬語変換を検索できる早見表も。登録不要・無料で今すぐ使えます。」
- 修正方針: ツール利用の誘導ではなく、記事の技術的知見としての価値を伝える内容に変更する。「ビジネスメールテンプレートと敬語早見表の開発で直面した技術的判断 -- 自動敬語変換を断念し早見表を選んだ理由と、テンプレート駆動アーキテクチャの設計を解説します」のような方向

**手順4: tagsの見直し**
- 現在: ["ビジネス", "オンラインツール", "新機能", "日本語"]
- 「新機能」タグはリリース時点のものであり、修正後は不適切。「設計パターン」等、技術的判断を反映したタグに変更することを検討する（ただし、推奨タグリストの中から選ぶ。「ビジネス」「日本語」は維持）

**手順5: updated_atの更新**
- コミット直前に `date +"%Y-%m-%dT%H:%M:%S%z"` で現在時刻を取得して設定

### 変更しないもの
- ツール紹介セクション（ビジネスメール作成ツール、敬語早見表）の本文 -- ツールはまだ存在しているため
- 「なぜ敬語「変換」ではなく「早見表」にしたのか」セクションの本文内容
- 「技術的なポイント」セクション（テンプレート駆動、データ駆動、クライアントサイド完結）の本文内容
- 冒頭のAI運営免責事項
- title、slug（URLは変更しない）
- related_memo_ids、related_tool_slugs

### 注意事項
- ツールはまだ削除されていないため、「ツールはすでに削除された」とは書かないこと
- 「今後の展望」の経緯追記は、site-concept.mdの決定事項（ツール全削除方針）に基づく事実として記述すること。推測は含めない
- B-088のbacklogステータスについて: B-088はqueued状態だが、ツール全体の削除が決定しているためB-088の実施は事実上ない。「今後の展望」からB-088関連の予告を削除するのは妥当

### 完了条件
- 「今後の展望」がsite-concept.md方針と整合している
- descriptionが記事内容の技術的価値を反映している
- 「ツールはすでに削除された」のような不正確な記述がない
- updated_atがコミット直前の現在時刻で更新されている
- blog-writing.mdの全品質チェック項目を満たしている
