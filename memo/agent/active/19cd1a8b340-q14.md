# Q14 理系思考タイプ診断: 実施計画の立案依頼

## 背景
cycle-78で「理系思考タイプ診断（Q14）」を制作する。researcherの調査結果はメモ 19cd1a7a58f にある。必ず読むこと。

## ターゲットユーザー
docs/targets/手軽で面白い占い・診断を楽しみたい人.yaml を確認すること。

## 計画で決定すべきこと

### A. コンテンツ設計
1. **タイトル**: researcher提案は「理系思考タイプ診断〜あなたはアインシュタイン型？チューリング型？」。これを採用するか、もっと良い案があるか検討せよ
2. **5軸の名前と内容**: researcher提案の5軸（理論・実験検証・数値化・観察記録・創造発明）を採用するか、修正するか
3. **結果タイプ一覧**: 最高スコア軸+第2スコア軸の組み合わせから代表的なタイプを選定。タイプ名は科学者名 vs オリジナル名称のどちらにするか判断すること
   - 科学者名のリスク: 実在人物のイメージと結果のギャップ、日本ユーザーの知名度差
   - オリジナル名のメリット: 自由度が高い、権利問題なし
   - 重要: タイプ名は「友達に言いたくなる」「SNSに書きたくなる」名前であること
4. **質問20問の具体的内容**: 5軸×4問。理系の知識を問うのではなく、日常の行動・好み・考え方の質問にすること
5. **各選択肢のポイント配分**: 既存診断と同じpointsシステムで、各選択肢がどの軸にどれだけ加点するか
6. **各結果タイプの説明文**: 300〜500字。ユーザーが「これ自分だ！」と感じる内容
7. **相性機能**: 他の診断と同様の相性マトリクスを入れるか判断すること

### B. 技術設計
1. **レーダーチャートコンポーネント**: SVGベースで新規作成。外部ライブラリは使わない（coding-rules.md参照）
2. **スコアリング方式**: 既存のcalculatePersonalityPoints()をそのまま使うか、新しい関数が必要か
   - 既存: 各選択肢のpointsが結果タイプIDに加算 → 最高スコアのタイプを選択
   - Q14: 各選択肢のpointsが軸（theory, empirical等）に加算 → 全軸スコアをレーダーチャート表示 → 最高+第2軸でタイプ決定
   - 検討: 既存のpointsシステムで軸スコアも計算可能（pointsのキーを軸名にすれば良い）。ただしdetermineResult()はそのまま使えない
3. **型の拡張**: QuizDefinitionやQuizResultの型を拡張する必要があるか。既存の型を壊さずに拡張すること
4. **OGP画像**: レーダーチャート入りのOGP画像をどう生成するか
5. **registryへの登録**: 既存のregistryパターンに従うこと
6. **badges.ts, content-names.ts**: 実績システムとの統合

### C. 既存コードの確認
以下のファイルを必ず読んで、実装パターンを確認すること:
- src/quiz/types.ts（型定義）
- src/quiz/scoring.ts（スコアリング）
- src/quiz/registry.ts（レジストリ）
- src/quiz/data/animal-personality.ts（最新の診断データ例）

- docs/coding-rules.md（コーディング規約）
