# PM判断: 既存コンテンツ評価結果の統合と修正

6件のエージェント評価結果を統合し、PMとして厳しくレビューした上での最終判断。

## エージェント評価からの修正点

### 修正1: 漢字カナル → 「修正して保持」に変更（エージェントは「保持」としていた）
**理由**: 現在80字（小1配当漢字のみ）しかデータがない。Wordle型ゲームでは推理のために入力する漢字からヒントを得ることがゲーム体験の核心であり、候補が80字しかないとヒントの情報量が極めて少なく、ゲームとして成立しない。
**必要な修正**: KANJIDIC2（CC BY-SA 4.0）を使って常用漢字2,136字に拡張する。リサーチメモ19cbbf7bd1dで技術的に実現可能と確認済み（categoryフィールドをradicalGroupに置換すれば完全自動化可能）。

### 修正2: チートシート4件（Git, HTML, HTTP, Markdown）→ 「削除」に変更（エージェントは「保持（優先度低）」としていた）
**理由**: エージェントは辞典を「大手に勝てない」で削除する一方、チートシートを「品質は問題ない」で保持としたが、判断基準の一貫性がない。
- Git: 公式ドキュメント（git-scm.com/docs）、Qiita/Zennの多数の記事、GitHub公式チートシートに対して独自性なし
- HTMLタグ: MDN Web Docsが圧倒的。HTMLタグ一覧も多数のサイトに存在
- HTTPステータスコード: MDN、RFC原文の日本語解説が多数存在
- Markdown: GitHub Docs、Qiita公式記事が充実
これらは辞典と同じく「大手・公式ドキュメントに品質で劣り、独自性がない」に該当する。

### 修正3: チートシート3件（Cron, Regex, SQL）→ 独自性の根拠を精査して保持維持
- Cron: プラットフォーム別比較（GitHub Actions/AWS/Quartz）をまとめた日本語リソースが少ない → 保持
- Regex: regex-testerツール（ただしツール自体は削除予定）との連携 → ツール削除後は独自性が消える → **削除に変更**
- SQL: 統一サンプルデータによる一貫した教育的説明 → 一定の独自性あり → 保持

### 修正4: Regex チートシート → 「削除」に変更
regex-testerツールが削除される場合、「ツール連携」という差別化要素が消失する。正規表現チートシート単体では公式ドキュメントやQiita記事と差別化できない。
