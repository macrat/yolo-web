# サイクル実行手順

サイクルドキュメントと、plannerが立案した作業計画に従って作業を進めてください。

## 作業の進め方

以下の手順で作業してください。

1. 各タスクごとに builder エージェントを起動し、計画に沿って作業を進めさせてください。
2. reviewer に成果物をレビューさせてください。
3. 指摘事項が1つでもあれば、 builder にそのメモを読ませて修正させてください。
4. 指摘事項なしでレビューが通るまで2と3を繰り返してください。再レビュー時は、前回の指摘事項だけでなく全体を見直すように指示してください。これにより、見落していた点も修正され、品質の高い成果物ができるようになります。

作業はすべてサブエージェントを通じて行ってください。PMが `src/` 配下のファイルを直接 Edit/Write ツールで変更することは禁止です。修正が軽微であっても、必ず builder エージェントを通じて行ってください。
サブエージェントに依頼するときはまず依頼メモを作成し、それからツールを使ってエージェントを起動してください。
ツール起動時には、メモのIDだけを指定してください。依頼内容を直接伝えてはいけません。

なお、作業にあたってownerの承認を得る必要はありません。独自の判断で進めてください。ownerはメモのやり取りを監視し、必要に応じてフィードバックを提供します。

## 作業の心構え

作業やレビューにあたっては、以下の点を意識してください。

- 計画が立てられた本来の目的を常に意識すること。
- ユーザーにとっての価値を常に意識すること。
- もし目的を果せていなかったり、十分な価値を提供できていないようなら、ゼロからやり直すことも辞さない覚悟で作業またはレビューすること。

## 作業上の注意点

- **タスク粒度の厳守**: 大量のコンテンツ調査を単一エージェントに集約しない。LLMはコンテキスト圧迫で急激に性能劣化する。1メモ1タスク1エージェントのルールを厳守し、カテゴリや件数で細かく分割する。
- **レビュー指示は価値重視**: reviewerへのレビュー依頼時は、細部の正確性よりも「全体の価値」を最優先で確認するよう指示する。「来訪者にとっての価値」「競合にない独自性」「目的達成の最善手か」を確認させる。
- **Owner発言は示唆であり命令ではない**: Ownerの指摘をそのままコピーするのは責務の放棄。PMの責務はResearcherに調査させ、Plannerに検討させ、総合的に判断すること。Owner発言は示唆やアドバイスであり、検証した上で最終判断する。

## 作業が完了したら

すべての作業が完了したら、`/cycle-completion` スキルを実行してサイクルを完了させてください。
