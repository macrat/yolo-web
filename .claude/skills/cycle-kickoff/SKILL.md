# サイクルの実施

## サイクル全体の流れ

サイクルは以下のような流れで進みます。

1. サイクルを開始する。このサイクルで実施することを決める。
2. 作業計画を立てる。何をどのようにやるのかを具体的に決める。
3. 実際の作業をする。計画に沿って成果物を完成させる。
4. サイクルを完了させる。成果を報告し、デプロイ等の必要な手続きを行う。

## 手順

以下の手順に従って新しいサイクルを開始してください。

### 1. 状態の確認

/docs/cycles/ にある最新のサイクルドキュメントを読んで、前回のサイクルが完了していることを確認してください。
前回のサイクルが完了していない場合は、まずはそちらを完了させてください。

### 2. Backlogの更新

`/docs/backlog.md` を確認し、Deferredに分類されている項目のうち着手可能になっているものがあればQueuedに移動してください。

また、Queuedに分類されている項目のうち、まだ着手できないものがあれば、着手できるようになる条件を記してDeferredに移動してください。

### 3. 実施する作業の選択

`/docs/backlog.md` を確認し、Queuedに分類されている項目から着手するものを選んでActiveに移動してください。
サイクルの作業は最小限にしてください。密接に関係していて2つ以上同時にやる必要がある場合に限っては、複数のタスクを選んでも構いません。

なお、すべての項目が必ずしも必要とは限りません。
必要性が疑わしいものがあれば、「本当に実施する価値があるか検討する」という形でActiveにしても構いません。

着手できる項目がない場合は、 `/new-cycle-idea` スキルを実行して次にやることを考えてください。

### 4. サイクルドキュメントの作成

サイクルの内容が決まったら、 `/docs/cycles/TEMPLATE.md` をコピーして新しいサイクルドキュメントを作成してください。
サイクルドキュメントのファイル名は `cycle-XX.md` としてください。XXはサイクル番号で、前回のサイクルドキュメントの番号に1を足したものになります。

`started_at` には !`date +"%Y-%m-%dT%H:%M:%S%z"` を使ってください。

ここで、作業計画の欄を埋めます。
必要に応じてネットやドキュメントを調べて前提情報を集めてください。

### 5. 開始時点の状態の記録

ここまでできたら、一度 `git commit` して、開始時点の状態を記録してください。

### 6. 作業の実施

4で作成したサイクルドキュメントの「作業計画」に沿って、「実施する作業」に記載されたタスクを実施してください。
「実施する作業」はチェックリストを兼ねるので、それぞれのタスクが完了するごとにチェックを入れてください。

### 7. レビューの実施

作業が完了したら、サブエージェントにレビューを依頼してください。
レビューで指摘があれば、それが本当に必要な指摘かどうかを判断して、必要なものはすべて対応してください。
有効な指摘事項が残っている状態でサイクルを完了させることはできません。

### 8. サイクルの完了

`/cycle-completion` スキルを実行してサイクルを完了させてください。

## 心構え

すべての作業は、このサイトの来訪者に提供できる価値を最大化するために実施します。
/docs/targets/ で定義されているターゲットユーザーを確認し、彼らのために最高の価値を提供してください。
