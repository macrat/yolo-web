# サイクル開始手順

## 開始手順

以下の手順に従って新しいサイクルを開始してください。

### 1. 状態の確認

前回のサイクルが完了していることを確認してください。

以下のコマンドでアクティブなメモを確認し、さらに `/docs/backlog.md` に実行中のタスクがないか確認してください。

```bash
npm run memo -- list --state active
npm run memo -- read [memo]...
```

前回のサイクルが完了していない場合は、まずはそちらを完了させてください。

### 2. メモのトリアージ

inboxを確認し、届いているメモがあれば内容に応じてactiveまたはarchiveに移動してください。

```bash
npm run memo -- list --state inbox
npm run memo -- read [memo]...
npm run memo -- mark [active|archive] [memo]...
```

新たなタスクがあった場合は、`/docs/backlog.md`のQueuedまたはDeferredに追加してください。

### 3. Backlogの更新

`/docs/backlog.md` を確認し、Deferredに分類されている項目のうち着手可能になっているものがあればQueuedに移動してください。

### 4. 実施する作業の選択

`/docs/backlog.md` を確認し、Queuedに分類されている項目から着手するものを選んでActiveに移動してください。
すぐに終わるような簡単なタスクや、同時にやった方が効率が良いタスクがあれば、複数をまとめて選んでも構いません。

着手できる項目がない場合は、 `/new-cycle-idea` スキルを実行して次にやることを考えてください。

### 5. サイクルドキュメントの作成

サイクルの内容が決まったら、 `/docs/cycles/TEMPLATE.md` をコピーして新しいサイクルドキュメントを作成してください。
サイクルドキュメントのファイル名は `cycle-XX.md` としてください。XXはサイクル番号で、前回のサイクルドキュメントの番号に1を足したものになります。

`started_at` には !`date +"%Y-%m-%dT%H:%M:%S%z"` を使ってください。

### 6. 開始時点の状態の記録

ここまでできたら、一度 `git commit` して、開始時点の状態を記録してください。

### 7. 作業計画の立案

`/cycle-planning` スキルを実行して、作業計画を立ててください。
