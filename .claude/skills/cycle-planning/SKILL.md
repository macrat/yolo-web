# サイクルプランニング

サイクルドキュメントに記載された内容を具体化するために、調査・分析を行って実施計画を立ててください。

## 作業の進め方

以下の手順で作業してください。

1. サイクルで予定されている作業内容をいくつかのタスクに分割してください。タスクはできるだけ小さく、互いに独立しているものにしてください。
2. /docs/targets/ で定義されているターゲットユーザーを確認し、どのユーザーのためにどのような価値を提供するのかを明確にしてください。
3. タスクごとに、必要な情報やベストプラクティスを調査するための researcher エージェントを起動してください。1タスクにつき1エージェントを起動してください。並行で進められるものは並行で、依存関係がある場合は順番に起動します。なお、解決方法が自明で、調査の余地がないタスクについてはこのステップを省略して構いません。
4. タスクごとに planner を起動し、調査結果を踏まえた実施計画を立てさせてください。
5. reviewer を起動し、 planner が立てた計画をレビューさせてください。
6. 指摘事項が1つでもあれば、planner にそのメモを読ませて修正させてください。
7. 指摘事項なしでレビューが通るまで5と6を繰り返してください。再レビュー時は、前回の指摘事項だけでなく全体を見直すように指示してください。これにより、見落していた点も修正され、品質の高い計画ができるようになります。

作業はすべてサブエージェントを通じて行ってください。

なお、作業にあたってownerの承認を得る必要はありません。独自の判断で進めてください。ownerは作業の状況を監督し、必要に応じてフィードバックを提供します。

## 計画に含めるべき内容

計画には以下のような内容を含めるべきです。

- 誰の/何のためにやるのか（想定利用者/読者は誰で、何を求めているのか）
- この作業によってどんな価値を提供するのか
- どのような作業が必要なのか
- 作業中どのような点に注意すべきか
- どうなったら完成といえるのか

## レビューの観点

計画をレビューするときは、そのタスクの本来の目的を考えて、それを達成できる内容になっているかを確認してください。
ハック的な解決策を可能な限り避けて、根本的な解決策を追求するようにしてください。

また、ユーザーにとっての価値を常に意識するようにしてください。
もしもユーザーにとって十分な価値を提供できないような内容になっていれば、ゼロからやり直すことも検討してください。

## 計画が完成したら

計画が完成したら、plannerからのメモをactiveにした状態で一度 `git commit` してください。
次に、`/cycle-execution` スキルを実行して作業を開始してください。

もし今サイクルで予定されているタスクが調査や計画だけで実際の作業を伴わないのであれば、`/cycle-completion` スキルを実行してサイクルを完了させてください。
