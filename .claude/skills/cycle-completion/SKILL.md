# サイクル完了手順

サイクルの全タスクが完了したら、以下の手順を順番に実行してください。

## 1. 実装完了確認

サイクルドキュメントに記載されたチェックリストがすべて完了していることを確認してください。
1つでも未完了の項目がある場合は、サイクルを完了させてはいけません。すべての項目を完了させてから完了手続きをやりなおしてください。

問題なければ、 `completed_at` を !`date +"%Y-%m-%dT%H:%M:%S%z"` で更新してください。

## 2. 残存タスクの確認

`npm run memo -- list --state inbox,active` を実行して、未処理のメモがないことを確認してください。
未処理のメモがある場合は、内容を確認して処理してからこの手順を最初からやりなおしてください。

## 3. ブログ記事確認

以下の条件に該当する場合、マイルストーンを知らせるブログ記事が作成されていることを確認してください。

1. 新機能や新サービスを追加した場合（新しい機能を知らせたいとき）
2. 既存コンテンツを大幅に変更した場合（ユーザーの使い勝手が大きく変わることを告知したいとき）
3. デザインや設計を大幅に変更した場合（サイトの工夫をシェアしたいとき）
4. ワークフローやドキュメントを大幅に変更した場合（ワークフローの工夫をシェアしたいとき）

マイルストーンブログには、以下の内容が含まれている必要があります。

- **背景**: どのような背景や状況があってその変更を行うことになったのか
- **変更内容や新サービスの概要**: どのような変更を行ったのか、新しいサービスがどのようなものなのか
- **選定理由**: なぜそのコンテンツ・変更を選んだのか（背景と動機）
- **設計意図**: どのような意図で現在の設計にしたのか（技術的判断やUXの考慮）
- **採用しなかった選択肢**: 他に検討した選択肢と、それらを採用しなかった理由
- **今後の展望**: 今後の改善予定や発展の方向性
- **経緯の記録**: AI実験プロジェクトとしての意思決定プロセスの記録

まだ作成されていなければ、ブログ記事を作成して、この手順を最初からやりなおしてください。

また、「今後の展望」を書いた場合は、同じ内容が backlog.md にも記載されていることを確認してください。
もし記載されていないものがあれば、必ず記載してください。

## 4. キャリーオーバー項目の整理

サイクルドキュメントにキャリーオーバー項目がある場合は、内容を整理して /docs/backlog.md に登録してください。
このとき、関連するメモを探して、そのIDも記載するようにしてください。

## 5. ownerへの完了報告

PMからowner宛てにサイクル完了報告メモを作成してください。
報告メモには以下の内容を含めてください。

- 完了したタスクの一覧
- 各タスクの成果物（作成・変更したファイル）
- テスト結果のサマリー（テスト数、ビルドページ数）
- レビュー結果のサマリー
- 未完了・キャリーオーバー項目があればその説明

報告メモは、サイクル開始報告メモへの返信という形で作成してください。

## 6. 最後のフォーマット

コミット前に、最終のフォーマットを行ってください。

```bash
npm run format
```

## 7. コミットとプッシュ

すべての変更をmainブランチにコミットしてプッシュしてください。
これにより、サイクルの成果がデプロイされます。
