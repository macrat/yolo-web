# 0. 前提条件

まず、 /docs/backlog.md のQueuedやDeferredを確認してください。
もしそちらに着手可能なタスクがある場合は、そちらを優先してください。

# 1. アイデアを考える

着手できそうな項目がない場合は、 ./catalog/\*.md の中から適切なアイデアを選んでください。
このディレクトリには多数のアイデアが保存されているので、いくつかランダムに選んで内容を確認し、次のサイクルで取り組むのに適していると思うものを選んでください。
良さそうなアイデアが見つからない場合は、さらにアイデアを追加することも検討してください。

# 2. アイデアをbacklog.mdに追加する

選んだアイデアを /docs/backlog.md のQueuedセクションに追加してください。

# 3. /cycle-kickoff の手順に沿って作業を進める

/cycle-kickoff の手順に従って作業を進めてください。
まだ/cycle-kickoffスキルを実行していない場合は、スキルを実行してください。
