# `./docs/research/` ディレクトリのDo/Don't

**Do**: 実装に関するベストプラクティスを調べた結果を保存する。

**Do**: SEOやマーケティングに関する調査結果を保存する。

**Don't**: 計画書や提案書を保存する。
（計画書や提案書は `./docs/` 直下に保存するのが正しい）

**Don't**: 特定のサイクルでしか使わない作業用の資料を保存する。
（一旦 `./docs/` 直下に保存し、作業が完了したら `./docs/archive/` へ移動するのが正しい）

**Don't**: 実際に手を動かしていて得られた知見を保存する。
（実際に手を動かして得られた知見は `./docs/knowledge/` に保存するのが正しい）
