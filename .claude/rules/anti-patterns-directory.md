# `./docs/anti-patterns/` ディレクトリのDo/Don't

**Do**: 計画や作業の中で失敗に至る典型的なパターンを記録する。

**Do**: 一般化した説明として広く捉えられるように記録する。

**Do**: 「何をやってはいけないか」を記録する。

**Do**: 同じアンチパターンを繰り返してしまった場合は `（cycle-XXX, XXXで実際に発生）` という部分にサイクル番号のみを追記する。このとき、詳細はサイクルドキュメントに記録する。

**Don't**: 「どのような手順でやるべきか」を記録する。
（どうやるべきかは `./docs/` 直下、 `./docs/knowledge/` 、または `./.claude/skills/` にある手順書に書くのが正しい）

**Don't**: 特定の状況に特化した具体的な事例を記録する。
（具体的な事例は `./docs/cycles/` にあるサイクルドキュメントに記録し、サイクル番号だけを書くのが正しい）

**Don't**: あとから気付いても回復できないようなアンチパターンを記録する。たとえば、「コミットしていないファイルを削除していないか？」というアンチパターンを記録しても、気付くのは削除後だけなので意味がない。
（このような問題はアンチパターンではなく手順書に書くか、フック等で防止する方が正しい）
