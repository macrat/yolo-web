# ドキュメントディレクトリのガイド

`./docs/` ディレクトリはドキュメントを保存するための場所です。
以下のようなディレクトリがあります。

- `./docs/archive/` - もはや利用しなくなったが、将来参照する価値がある知見が含まれているドキュメント。
- `./docs/cycles/` - サイクルごとのドキュメント。サイクルドキュメント以外は保存しないでください。
- `./docs/research/` - 調査や分析の結果。計画や提案などは保存しないでください。
- `./docs/targets/` - ターゲットユーザーの定義。書式などは `./docs/targets/README.md` を参照してください。
- `./docs/anti-patterns/` - 過去の失敗から抽出したアンチパターンのチェックリスト。レビュー時に参照する。Ownerから失敗を指摘されたときはここを確認し、カバーされていなければ追記する。
- `./docs/knowledge/` - 技術的な知見の蓄積（Next.js固有の挙動、LLMの実証的知見など）。困ったときに参照する。アンチパターンではカバーできない性質の新たな知見を得たら追加する。
- `./docs/`直下 - 上記以外のドキュメント。古くなったドキュメントは削除するかarchiveに移動してください。

## 注意

- ドキュメントは将来も参照されることを前提に、内容が古くなったものは削除するか `./docs/archive/` に移動してください。誤解を生む可能性があるものや間違っているものは、内容を更新するか削除してください。
- 新しいドキュメントを作成するときは、既存のファイルと重複していないか確認してください。重複している場合は既存のドキュメントを更新するか、古いドキュメントを削除またはarchiveに移動してください。
- ソースコードを読めば分かることはドキュメントにしないでください。代わりに、ソースコード内のコメントを充実させてください。
- ある作業の途中で一時的に必要なドキュメントやファイルは `./tmp/` ディレクトリに保存してください。 `./docs/` に保存するものは、次以降のサイクルでも参照されることを前提にしてください。
