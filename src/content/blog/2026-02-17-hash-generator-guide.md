## ハッシュとは

ハッシュ関数とは、任意の長さの入力データから固定長の値（ハッシュ値）を生成する一方向の変換関数です。「一方向」とは、ハッシュ値から元のデータを逆算することが計算上極めて困難であることを意味します。

ハッシュ関数には以下の重要な特性があります。

- **決定性**: 同じ入力からは常に同じハッシュ値が生成される
- **高速性**: ハッシュ値の計算が効率的に行える
- **一方向性**: ハッシュ値から元の入力を復元することが困難
- **衝突耐性**: 異なる入力から同じハッシュ値が生成される確率が極めて低い
- **雪崩効果**: 入力の微小な変化がハッシュ値の大きな変化を引き起こす

たとえば「hello」と「Hello」のように1文字の大文字小文字が異なるだけでも、まったく異なるハッシュ値が生成されます。

## 主要アルゴリズムの比較

### MD5

MD5（Message Digest 5）は128ビットのハッシュ値を生成するアルゴリズムです。かつては広く使われていましたが、[衝突攻撃](https://www.kb.cert.org/vuls/id/836068)（異なる入力から同じハッシュ値を意図的に生成する攻撃）が現実的に可能であることが示されており、セキュリティ用途には推奨されません。ファイルの簡易的な同一性チェックなど、セキュリティが問題にならない場面では今でも使われることがあります。

### SHA-1

SHA-1（Secure Hash Algorithm 1）は160ビットのハッシュ値を生成します。MD5より安全性は高いものの、2017年に[GoogleがSHA-1の衝突を実証](https://shattered.io/)しており、セキュリティ用途には不適切とされています。

### SHA-256

SHA-256はSHA-2ファミリーに属し、256ビットのハッシュ値を生成します。現在最も広く使われている安全なハッシュアルゴリズムの1つです。SSL/TLS証明書やブロックチェーン（ビットコイン）など、高い安全性が求められる場面で採用されています。

### SHA-384 / SHA-512

SHA-384とSHA-512もSHA-2ファミリーに属し、それぞれ384ビット、512ビットのハッシュ値を生成します。SHA-256より長いハッシュ値を生成するため理論上はより安全ですが、計算コストも高くなります。特に高いセキュリティレベルが求められる場面や、64ビットCPUでの処理効率を重視する場合に使われます。

| アルゴリズム | ハッシュ長 | 安全性                 | 用途                      |
| ------------ | ---------- | ---------------------- | ------------------------- |
| MD5          | 128ビット  | 低い（衝突攻撃が可能） | 簡易チェックサム          |
| SHA-1        | 160ビット  | 低い（衝突が実証済み） | レガシーシステム          |
| SHA-256      | 256ビット  | 高い                   | SSL/TLS、ブロックチェーン |
| SHA-512      | 512ビット  | 高い                   | 高セキュリティ要件        |

## セキュリティ上の注意

### パスワードのハッシュ化

パスワードの保存にはハッシュ化が不可欠ですが、SHA-256をそのまま使うだけでは不十分です。実際のパスワード保存には、bcryptやArgon2など、意図的に計算を遅くする専用のアルゴリズムが推奨されます（詳しくは[OWASP Password Storage Cheat Sheet](https://cheatsheetseries.owasp.org/cheatsheets/Password_Storage_Cheat_Sheet.html)を参照）。これらのアルゴリズムは、ブルートフォース攻撃（総当たり攻撃）のコストを大幅に引き上げます。

また、ハッシュ化の際には「ソルト」と呼ばれるランダムな文字列を付加することで、同じパスワードでもユーザーごとに異なるハッシュ値が生成されるようにする必要があります。

### ファイルの整合性検証

ソフトウェアをダウンロードした際に、公式サイトに記載されたSHA-256チェックサムとダウンロードしたファイルのハッシュ値を比較することで、ファイルが改ざんされていないことを確認できます。

## ツールでの生成方法

[ハッシュ生成ツール](/tools/hash-generator)では、テキストからSHA-1、SHA-256、SHA-384、SHA-512の4種類のハッシュ値を生成できます。Web Crypto APIを使用しており、すべての処理がブラウザ内で完結します。

### 使い方

1. テキスト入力欄にハッシュ化したい文字列を入力します
2. アルゴリズム（SHA-1、SHA-256、SHA-384、SHA-512）を選択します
3. 出力形式（16進数またはBase64）を選択します
4. ハッシュ値がリアルタイムで生成・表示されます

同じ文字列を入力すれば必ず同じハッシュ値が得られること、1文字変えるだけで全く異なるハッシュ値になることを実際に確認してみてください。ハッシュ関数の特性を体感的に理解できます。
